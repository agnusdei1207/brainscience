# 524. 스토리지 클래스 메모리 (SCM) 계층화 (SCM Tiering)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: DRAM의 빠른 속도(Byte 단위 접근 가능)와 NAND Flash의 비휘발성(전원이 꺼져도 데이터 유지) 특징을 모두 결합한 차세대 메모리인 SCM을 메모리-스토리지 계층 구조(Hierarchy)의 중간에 삽입하는 아키텍처 기술이다.
> 2. **가치**: DRAM과 NVMe SSD 사이의 엄청난 지연 시간 격차(100배 이상)를 메워, 인메모리 데이터베이스의 영속성(Persistence)을 보장하고 대용량 데이터를 저렴하게 메모리처럼 활용할 수 있게 한다.
> 3. **융합**: 인텔의 Optane(3D XPoint), NVDIMM, CXL 인터커넥트, 그리고 운영체제의 DAX(Direct Access) 파일 시스템과 결합하여 폰 노이만 구조의 메모리 병목을 혁신적으로 개선한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
스토리지 클래스 메모리(Storage Class Memory, SCM) 계층화는 기존 컴퓨터 아키텍처의 이분법적 구조(휘발성이지만 빠른 RAM vs 비휘발성이지만 느린 Disk/SSD)를 타파하고, 이 둘의 장점을 섞은 새로운 매체(SCM)를 메모리 캐시 또는 초고속 스토리지 티어로 배치하여 데이터 병목을 해소하는 기법이다.

### 💡 비유
- **DRAM (메인 메모리)**: "내 책상 위". 물건을 1초 만에 꺼낼 수 있지만, 퇴근할 때(전원 Off) 무조건 다 치워야 하고 공간이 좁다.
- **SSD (스토리지)**: "지하 창고". 물건을 영원히, 많이 보관할 수 있지만 가져오려면 엘리베이터를 타고 한참 다녀와야 한다.
- **SCM (스토리지 클래스 메모리)**: "책상 바로 옆 서랍장". 책상만큼 빠르지는 않지만 손만 뻗으면 닿는 위치에 있고, 퇴근해도 물건이 그대로 남아있다.

### 등장 배경 및 발전 과정

#### 1. 메모리 장벽(Memory Wall)과 I/O 격차
현대 CPU는 나노초(ns) 단위로 연산하지만, 가장 빠른 NVMe SSD조차 데이터를 가져오는데 수십 마이크로초(μs)가 걸린다. DRAM과 SSD 사이에는 약 **1,000배 이상의 속도 차이**가 존재하여, CPU가 데이터를 기다리느라 멈춰있는 병목(I/O Gap) 현상이 극심했다.

#### 2. 패러다임 변화 (새로운 매체의 탄생)
이 격차를 메우기 위해 PRAM(상변화 메모리), MRAM(자기 저항 메모리) 등의 신소재가 연구되었고, 인텔과 마이크론이 **3D XPoint (Optane)**라는 이름으로 상용화에 성공했다. 이를 어떻게 시스템에 편입시킬 것인가(메모리로 쓸 것인가? 디스크로 쓸 것인가?)에 대한 설계 논의가 SCM 계층화 아키텍처로 발전했다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 구성 요소 및 SCM 매체 특성 (표)

| 메모리/스토리지 계층 | 매체 예시 | 접근 방식 | 지연 시간(Latency) | 용량/가격 | 비휘발성 |
|----------------------|-----------|-----------|--------------------|-----------|----------|
| **L1/L2/L3 Cache** | SRAM | Byte-addressable | 1 ~ 10 ns | 수십 MB (최고가) | X |
| **Main Memory** | DRAM | Byte-addressable | 50 ~ 100 ns | 수백 GB (고가) | X |
| **SCM (Memory Mode)**| **NVDIMM (Optane)** | Byte-addressable | **100 ~ 300 ns** | **수 TB (중가)** | **O** |
| **SCM (Storage Mode)**| **NVMe Optane SSD** | Block-addressable | **5 ~ 10 μs** | **수 TB (중가)** | **O** |
| **Fast Storage** | NAND NVMe SSD | Block-addressable | 50 ~ 100 μs | 수십 TB (저가) | O |

### 정교한 구조 다이어그램 (ASCII Art)

```ascii
================================================================================
[ 1. SCM in Memory Mode vs 2. SCM in App-Direct (DAX) Mode ]
================================================================================

[ 1. Memory Mode (투명한 용량 확장) ]
  ┌─────────────────────────────────┐
  │         Application             │  <-- 앱은 아무 수정 없이 거대한 RAM으로 인식
  └────────────────┬────────────────┘
  ┌────────────────▼────────────────┐
  │         OS Memory Manager       │
  └────────────────┬────────────────┘
          (DRAM을 L4 캐시처럼 사용)
      ┌────────────▼────────────┐
      │   DRAM (Cache Tier)     │ 32GB (빠른 접근 버퍼)
      └────────────┬────────────┘
                   │ (Cache Miss)
      ┌────────────▼────────────┐
      │ SCM NVDIMM (Main Tier)  │ 512GB (실제 데이터 거주지)
      └─────────────────────────┘
  * 장점: 기존 S/W 변경 불필요. 단점: 재부팅 시 데이터 날아감(비휘발성 미활용)

[ 2. App-Direct Mode / DAX Mode (비휘발성 완벽 활용) ]
  ┌─────────────────────────────────┐
  │ In-Memory DB (Redis, SAP HANA)  │  <-- SCM을 인식하고 설계된 S/W
  └───────┬─────────────────┬───────┘
          │ (Load/Store)    │ (mmap)
          ▼                 ▼
     [ DRAM ]       [ OS DAX File System ]  <-- Page Cache (Kernel) 우회!
    (휘발성 변수)           │
                            ▼ (Byte 단위 직접 접근)
                    ┌───────────────┐
                    │ SCM NVDIMM    │ (비휘발성 DB 로그, 인덱스)
                    └───────────────┘
  * 장점: 커널 I/O 스택 우회(Zero-copy), 정전 시 즉시 복구(Instant Recovery).
```

### 심층 동작 원리

#### ① DAX (Direct Access) 파일 시스템
기존 운영체제(Linux ext4 등)는 디스크(Block 장치)에서 데이터를 읽을 때 무조건 RAM의 Page Cache로 복사본을 퍼올린 후 CPU에 전달했다. SCM은 RAM 슬롯(DIMM)에 꽂혀 있어 CPU가 직접 Byte 단위로 읽고 쓸 수 있다. 따라서 OS에 **DAX 옵션**을 켜면, 커널의 Page Cache를 거치지 않고 사용자 애플리케이션(DB)이 SCM의 물리적 주소에 직접 Load/Store 명령어를 쏘게 되어 오버헤드가 제로(0)가 된다.

#### ② 2-Tier 메모리 아키텍처 관리
CPU의 메모리 컨트롤러(iMC)가 DRAM을 캐시로, SCM을 메인 메모리로 관리한다. 자주 쓰이는 Hot Data는 DRAM에 남겨 고속 접근을 보장하고, 잘 쓰이지 않는 Cold Data는 SCM으로 내린다. 하드웨어 레벨에서 일어나는 캐싱이므로 OS가 개입할 필요가 없다.

## Ⅲ. 융합 비교 및 다각도 분석

### SCM 폼팩터(Form Factor) 장착 위치 비교

| 폼팩터 | 연결 버스 | 단위 | 지연 시간 | 특장점 및 한계 |
|--------|-----------|------|-----------|----------------|
| **NVDIMM-N / P** | DDR 슬롯 (메모리 버스) | Byte | ~ 300ns | 최고의 성능, 영속성 보장. 보드 슬롯 개수에 확장성 제한 |
| **NVMe SCM SSD** | PCIe 버스 | Block | ~ 10μs | NAND SSD 대비 10배 이상 빠르고 수명이 무한에 가까움 (메모리 버스보다는 느림) |
| **CXL Type 3** | PCIe (CXL 버스) | Byte | ~ 200ns | 메모리 버스의 용량 한계를 극복하는 차세대 SCM 장착 표준 |

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

### 기술사적 판단 (실무 시나리오)

**상황**: 금융권의 핵심 실시간 사기 탐지 시스템(FDS). 수 테라바이트(TB)의 데이터를 메모리에 올려놓고 인메모리 DB(Redis)로 구동 중이다. 서버에 장애가 발생해 재부팅되면, 디스크에서 5TB의 데이터를 다시 RAM으로 퍼올리는 데(Warm-up) 1시간 이상이 소요되어 막대한 금전적 손실이 발생함.
**전략적 SCM 도입**:
1. **App-Direct 모드 SCM 장착**: 서버의 DRAM 슬롯에 인텔 Optane PMem(Persistent Memory)을 장착하고 DAX 파일 시스템으로 포맷한다.
2. **인메모리 DB 재설계**: Redis의 핵심 데이터(Value)를 SCM 영역에 `mmap()`으로 할당하여 비휘발성 영역에 기록하고, 빠른 검색 인덱스(Key)만 일반 DRAM에 둔다.
3. **결과**: 서버 전원이 나가도 5TB 데이터는 SCM에 그대로 남아있다. 재부팅 시 인덱스만 DRAM에 재구성하면 되므로 서비스 복구 시간(RTO)이 1시간에서 **수십 초 이내로 단축**되며, 값비싼 DRAM을 수십 개 꽂을 필요가 없어 인프라 비용(TCO)도 절감된다.

### 주의사항 및 안티패턴 (Anti-patterns)
- **SCM의 비대칭적 성능**: SCM(예: Optane)은 읽기(Read) 속도는 DRAM에 근접하지만, 쓰기(Write) 속도는 상대적으로 훨씬 느리고 대역폭(Bandwidth) 제한이 있다. 극단적인 Write-heavy 애플리케이션을 SCM Memory 모드로 돌리면 오히려 일반 DRAM 서버보다 성능이 폭락하는 안티패턴을 겪게 된다.

## Ⅴ. 기대효과 및 결론

### 정량적 기대효과
- **용량당 비용(Cost per GB)**: DRAM 대비 약 1/2 수준의 비용으로 대규모 메모리 풀 구축.
- **장애 복구 시간(Recovery Time)**: 페타바이트급 인메모리 환경에서 디스크 로딩 시간을 완전히 없애 복구 시간 99% 단축.

### 미래 전망 및 진화 방향
비록 인텔이 Optane 사업을 철수(2022년)하는 등 시장의 부침이 있었으나, 메모리와 스토리지 사이의 "계층화"라는 아키텍처적 요구는 더욱 거세지고 있다. 현재 이 빈자리는 **CXL (Compute Express Link)** 기술이 완벽히 대체하고 있다. CXL은 서버 밖의 별도 메모리 랙(Memory Appliance)에 DRAM과 차세대 SCM을 섞어서 꽂아두고, 필요할 때마다 여러 서버가 이 메모리를 스위치 패브릭을 통해 공유(Memory Pooling)하는 진정한 클라우드 네이티브 메모리 아키텍처 시대를 열고 있다.

## 👶 어린이를 위한 3줄 비유 설명
1. **SCM이 뭔가요?**: 책상(DRAM)처럼 꺼내 쓰기 편하면서도, 책상 서랍장(SSD)처럼 전기를 꺼도 물건이 사라지지 않는 아주 신기한 마법의 서랍장이에요.
2. **왜 필요해요?**: 책상은 좁아서 물건을 많이 못 놓고, 지하 창고(디스크)는 다녀오기 너무 멀잖아요. 내 옆에 크고 빠른 서랍장이 있으면 일이 훨씬 빨라져요.
3. **무엇이 좋아지나요?**: 컴퓨터가 갑자기 꺼져서 다시 켰을 때, 지하 창고에서 장난감을 다시 다 꺼내올 필요 없이 옆 서랍장에서 바로 꺼내서 놀던 것을 계속 놀 수 있답니다!
