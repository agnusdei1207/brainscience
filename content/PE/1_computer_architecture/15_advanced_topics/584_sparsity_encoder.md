# 584. 딥러닝 텐서 희소성 (Sparsity) 인코더

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 딥러닝 연산에서 발생하는 방대한 텐서(Tensor) 데이터 중 대부분을 차지하는 **값이 0인 데이터(Zero elements)를 하드웨어 레벨에서 압축하여 전송 및 연산에서 제외**시키는 전용 하드웨어 가속 유닛이다.
> 2. **가치**: 불필요한 0 데이터의 이동을 차단하여 메모리 대역폭을 최대 10배 이상 절약하며, 0과의 곱셈(Mult-Accumulate) 연산을 생략함으로써 AI 추론 성능을 비약적으로 높이고 전력 소모를 획기적으로 낮춘다.
> 3. **융합**: 비트맵(Bitmap) 인코딩, CSR(Compressed Sparse Row) 포맷, 그리고 하드웨어 **제로 스키핑(Zero-skipping)** 로직과 융합되어 최신 NPU 및 GPU 아키텍처의 필수 구성 요소가 되었다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
딥러닝 모델, 특히 ReLU 활성화 함수를 통과한 데이터나 가지치기(Pruning)를 마친 가중치 데이터는 전체의 50~90% 이상이 '0'으로 채워져 있다. 이를 **'희소성(Sparsity)'**이라 한다. 희소성 인코더는 이 0들을 싹 무시하고 진짜 의미 있는 숫자들만 골라 담는 장치다.

### 💡 비유
희소성 인코더는 **"과대 포장을 걷어내는 지능형 압축기"**와 같다.
- **일반 텐서 (비효율)**: 거대한 질소 과자 봉지. 알맹이는 몇 개 없는데 봉지만 커서 트럭(메모리 버스) 자리를 다 차지한다. 연산할 때도 공기(0)를 입에 넣고 씹는 헛수고를 한다.
- **희소성 인코더**: 과자 봉지에서 질소를 쫙 빼고 알맹이(Non-zero)만 모은다. "몇 번째 자리에 과자가 있었는지" 번호표만 붙여서 보낸다. 트럭 한 대에 훨씬 많은 과자를 실을 수 있고, 먹을 때도(연산) 알맹이만 쏙쏙 골라 먹어 배가 빨리 부른다.

### 등장 배경 및 발전 과정

#### 1. 대규모 언어 모델(LLM)의 폭증
모델 파라미터가 수천억 개로 늘어나면서, 이를 메모리에서 가져오는 것 자체가 불가능한 수준에 도달했다. "0은 옮기지도 말고 계산하지도 말자"는 절박함이 하드웨어 설계를 바꿨다.

#### 2. NPU의 경쟁력
단순히 연산 유닛(ALU)을 늘리는 것보다, 0 데이터를 거르는 '희소성 처리' 능력이 AI 반도체의 실질적인 성능(TOPS/W)을 결정하는 핵심 지표가 되었다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 주요 희소성 데이터 포맷 (표)

| 포맷 명칭 | 작동 원리 | 장점 | 비유 |
|-----------|-----------|------|------|
| **Bitmap** | 데이터가 0이면 0, 아니면 1로 표시한 지도 생성 | 구현이 매우 단순함 | 좌석 배치도 |
| **CSR / CSC** | 0이 아닌 값의 위치(Index)를 리스트로 저장 | 압축률이 극도로 높음 | 보물찾기 지도 |
| **2:4 Structured**| 4개 덩어리 중 무조건 2개는 0으로 만듦 (NVIDIA) | 하드웨어 정렬 연산에 최적화 | 2인 1조 짝 맞추기 |

### 정교한 구조 다이어그램 (Sparsity Encoder/Decoder Flow)

```ascii
================================================================================
[ Hardware Sparsity Encoder & Zero-Skipping Logic ]
================================================================================

 [ Original Tensor ] : [ 5, 0, 0, 12, 0, 7, 0, 0 ] (8 elements)
           │
           ▼ (1) 🔢 Sparsity Encoding (Bitmask Logic)
 ┌──────────────────────────────────────────────────────────┐
 │  ⚡️ Sparsity Encoder Unit                                │
 │  - Value Array: [ 5, 12, 7 ] (알맹이만 추출)               │
 │  - Bitmask    : [ 1, 0, 0, 1, 0, 1, 0, 0 ] (위치 기록)     │
 └─────────────┬────────────────────────────────────────────┘
               │
               ▼ (2) 🛣️ Compressed Transfer (Save Bandwidth!)
 [ Memory Bus ] : [ 10100100 ] [ 5 ] [ 12 ] [ 7 ] 
               │
               ▼ (3) ⚡️ Zero-Skipping MAC (Multiply-Accumulate)
 ┌──────────────────────────────────────────────────────────┐
 │  🚀 Computation Unit (ALU)                               │
 │  - "Bitmask is 0? Skip Multiplication!"                  │
 │  - Only 3 Multiplications instead of 8.                  │
 └──────────────────────────────────────────────────────────┘

 * 핵심 포인트: 메모리 통로(Bus)에서는 8개 대신 3개만 나르고, 
   계산기(ALU)도 8번 대신 3번만 일함. 효율 2.6배 향상!
```

### 심층 동작 원리

#### ① 비트맵 인덱싱 (Bitmap Indexing)
데이터 한 세트(보통 16~32개)마다 16~32비트의 짧은 비트맵을 만든다. 이 비트맵을 보고 하드웨어의 '게이트(Gate)'가 열리고 닫힌다. 비트맵이 0이면 해당 데이터를 읽으려던 전기 신호 자체가 차단되어 전력을 아낀다.

#### ② 구조적 희소성 (Structured Sparsity)
무작위로 0이 박혀 있으면 하드웨어가 길을 찾기 힘들다. 엔비디아(NVIDIA) Ampere 아키텍처는 "4개 중 2개는 무조건 0이어야 한다"는 규칙을 세워, 하드웨어가 규칙적으로 0을 건너뛰게 만들었다. (성능 2배 향상).

#### ③ 메타데이터 관리 오버헤드
데이터를 압축하면 "원래 위치" 정보를 담은 메타데이터(Index)가 추가된다. 희소성이 50% 이하로 낮으면 오히려 메타데이터 때문에 용량이 늘어날 수 있다. 따라서 모델 설계 단계에서 희소성을 70% 이상으로 높이는 **'Sparse-aware Training'**이 병행된다.

## Ⅲ. 융합 비교 및 다각도 분석

### Dense 연산 vs Sparse 연산 하드웨어 비교

| 비교 지표 | Dense Accelerator (일반) | **Sparse Accelerator (특화)** |
|-----------|--------------------------|--------------------------------|
| **자원 낭비** | 0 연산에도 전력/시간 소모 | **0 연산 원천 차단** |
| **메모리 활용**| 낮음 (공백 데이터 포함) | **최상 (유효 데이터만 적재)** |
| **구현 복잡도**| 낮음 | **매우 높음 (인덱싱 로직 추가)** |
| **성능 (TOPS)**| 고정됨 | **희소성에 따라 폭발적 증가** |
| **비유** | 성실하지만 융통성 없는 일꾼 | **중요한 일만 골라 하는 천재 일꾼** |

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

### 기술사적 판단 (실무 시나리오)

**상황**: 온디바이스 AI 칩셋을 설계 중이다. 배터리 소모를 줄이면서 초거대 언어 모델(LLM)의 추론 속도를 높여야 한다. 하지만 칩 면적은 한정되어 있다.
**판단 및 아키텍처 적용 전략**:
1. **Weight Pruning 연동**: 학습 단계에서 중요도가 낮은 가중치를 강제로 0으로 만든 뒤, 하드웨어 **희소성 인코더**를 탑재한다.
2. **Zero-skipping MAC 적용**: ALU 입구에 제로 감지 회로를 넣어, 가중치가 0인 경우 활성화 함수 연산을 생략하고 즉시 다음 단계로 넘어가게 한다.
3. **Index Compression**: 메타데이터 용량을 줄이기 위해 `Run-length Encoding` 기법을 인코더에 추가한다.
4. **결과**: 동일한 전력 소모 내에서 추론 속도를 4배 향상하고, 모델이 차지하는 메모리 점유율을 60% 절감하여 저가형 모바일 칩에서도 고성능 AI를 구동한다.

### 주의사항 및 안티패턴 (Anti-patterns)
- **불규칙한 희소성 (Unstructured Sparsity)**: 0이 사방에 랜덤하게 흩어져 있으면 하드웨어가 데이터를 퍼 오는 주소(Address)가 계속 바뀌어 캐시 효율이 엉망이 된다. 반드시 하드웨어 설계팀과 협의하여 **'하드웨어가 좋아하는 0의 배치 규칙(Structured Sparsity)'**을 모델 학습팀에 가이드해야 한다.

## Ⅴ. 기대효과 및 결론

### 정량적 기대효과
- **전력 효율**: 불필요한 스위칭 제거로 연산 당 에너지 소모 **50~70% 절감**.
- **가속 성능**: 0 데이터 건너뛰기를 통해 실질 연산 속도(Effective TFLOPS) **2~5배 향상**.

### 미래 전망 및 진화 방향
희소성 인코더는 이제 **'동적 희소성 (Dynamic Sparsity)'** 단계로 진화하고 있다. 가중치뿐만 아니라, 입력되는 영상이나 음성 데이터의 0(Silence/Black)을 실시간으로 인지하여 칩의 리소스를 동적으로 끄고 켜는 기술이다. 이는 AI가 24시간 켜져 있어도 배터리가 닳지 않는 'Always-on AI' 기기의 핵심 경쟁력이 될 것이며, 인공지능 하드웨어를 '숫자만 계산하는 기계'에서 '데이터의 의미를 이해하고 반응하는 지능형 회로'로 바꿀 것이다.

## 👶 어린이를 위한 3줄 비유 설명
1. **희소성 인코더가 뭔가요?**: 인공지능이 공부하는 책(데이터)에 "아무 의미 없는 낙서(0)"가 너무 많을 때, 그 낙서들을 싹 지우고 진짜 공부할 내용만 골라서 요약해주는 '지능형 지우개'예요.
2. **왜 좋나요?**: 책이 얇아지니까 가방에 더 많은 책을 넣을 수 있고(대역폭 절약), 낙서를 안 읽어도 되니까 숙제하는 시간도 훨씬 빨라지기 때문이에요.
3. **가장 좋은 점은요?**: 로봇이 힘을 덜 써도 되니까 배터리가 아주 오래가고, 눈 깜짝할 사이에 정답을 맞힐 수 있게 된답니다!
