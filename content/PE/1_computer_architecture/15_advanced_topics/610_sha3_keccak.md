# 610. 보안 해시 함수 회로 (SHA-3 / Keccak) - 스펀지 구조가 만들어낸 철벽의 암호

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 임의의 길이를 가진 데이터를 고정된 길이의 고유한 '디지털 지문(Hash)'으로 변환하는 암호화 알고리즘으로, 기존 SHA-2의 구조적 취약점을 극복하기 위해 설계된 **Keccak(케착)** 기반의 최신 표준(SHA-3)이다.
> 2. **혁신**: 기존의 Merkle-Damgård 구조를 버리고, 데이터를 흡수(Absorb)하고 짜내는(Squeeze) **'스펀지 구조(Sponge Construction)'**를 채택하여 다양한 해시 길이와 스트림 암호화까지 유연하게 지원한다.
> 3. **가치**: 양자 컴퓨터의 공격에 대한 내성(Post-Quantum)이 뛰어나며, 하드웨어 회로(FPGA/ASIC)로 구현했을 때 기존 SHA-2보다 속도는 빠르면서 면적은 적게 차지하는 극강의 전력 효율을 보여준다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
해시 함수는 데이터의 무결성(원본이 변조되지 않았음)을 증명하는 함수다. 'A'라는 글자를 넣든 10GB짜리 영화를 넣든 항상 256비트의 무작위 같은 글자가 튀어나온다. SHA-3(Secure Hash Algorithm 3)는 미국 국립표준기술연구소(NIST)가 2015년에 공모전을 통해 선정한 차세대 해시 표준으로, 원래 이름은 Keccak(케착)이다.

### 💡 비유
- **해시 함수 (고기 분쇄기)**: 소고기를 넣든 돼지고기를 넣든 다진 고기로 만들어버린다. 다진 고기를 보고 원래 소의 모양을 절대 알 수 없고(단방향성), 소 한 마리에 파리 한 마리가 섞여 들어가도 다진 고기의 맛(해시값)은 완전히 달라진다(눈사태 효과).
- **스펀지 구조 (물감 스펀지)**: 스펀지(Keccak의 내부 상태)를 준비한다. 빨간 물감, 파란 물감을 차례대로 스펀지에 떨어뜨려 흡수시킨다(Absorb). 그다음 스펀지를 꾹 짜면(Squeeze) 두 물감이 오묘하게 섞인 보라색 물이 나온다.

### 등장 배경: 'SHA-1/2의 붕괴 위협'
과거의 표준이었던 MD5와 SHA-1은 이미 해커들에 의해 뚫렸다(충돌 공격 성공). 현재 널리 쓰이는 SHA-2는 안전하지만 SHA-1과 근본적인 뼈대(Merkle-Damgård)가 같아 언젠가 뚫릴지 모른다는 공포가 컸다. 이에 NIST는 아예 뼈대부터 완전히 다르고 양자 컴퓨터 공격에도 버틸 수 있는 새로운 백업 플랜을 찾았고, 그 결과가 스펀지 구조의 Keccak(SHA-3)이다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 스펀지 구조 (Sponge Construction) (ASCII Diagram)

```ascii
      [ Message (M) ] (잘게 쪼갬)
        M1      M2       M3
        │       │        │
      (XOR)   (XOR)    (XOR)
        ▼       ▼        ▼
 ┌────> ⊕ ────> ⊕ ─────> ⊕ ─────┐  [ Rate (r) : 흡수/짜내기 통로 ]
 │      │       │        │      │   <-- 사용자가 접근 가능한 영역
 │    [ f ]   [ f ]    [ f ]    │  <---------------------------------
 │      │       │        │      │   <-- 감춰진 내부 상태 (보안의 핵심)
 └──────┴───────┴────────┴──────┘  [ Capacity (c) : 용량 ]

   <----- Absorb (흡수) ----->    <----- Squeeze (짜내기) ----->
                                         │       │        │
                                         ▼       ▼        ▼
                                         Z1      Z2       Z3
                                      [ Hash Output (Z) ]
```

### 핵심 매커니즘 (Keccak-f 순열 함수)

스펀지 구조 내부에서 섞어주는 함수 `f`는 매우 독특한 3D 비트 배열(State)을 사용한다.
1. **상태(State) 배열**: $5 \times 5 \times 64$ 비트(총 1600비트)의 3차원 블록으로 구성된다.
2. **5단계 비트 연산 (θ, ρ, π, χ, ι)**: 복잡한 수학(더하기, 곱하기)을 쓰지 않고, 오직 `XOR`, `AND`, `NOT`, `Shift` 같은 가장 단순한 논리 게이트 연산만으로 24라운드를 섞어버린다.
3. **하드웨어 친화성**: 단순 논리 연산만 쓰기 때문에, CPU(소프트웨어)에서는 SHA-2보다 조금 느릴 수 있지만 **FPGA나 ASIC(하드웨어)로 구우면 SHA-2보다 면적은 작으면서 속도는 훨씬 빠르다.**

## Ⅲ. 융합 비교 및 다각도 분석

### SHA-256 (SHA-2) vs SHA-3 (Keccak)

| 비교 항목 | SHA-256 (현재 대세) | **SHA-3 (미래의 대세)** |
|-----------|---------------------|-------------------------|
| **기본 구조** | Merkle-Damgård 구조 | **Sponge 구조** |
| **길이 확장 공격**| 취약함 (부가적인 방어 필요) | **원천적으로 면역 (Capacity 영역 덕분)** |
| **소프트웨어 속도**| 매우 빠름 (Intel CPU 가속 명령어 존재) | 상대적으로 약간 느림 |
| **하드웨어(HW) 성능**| 보통 | **압도적으로 우수 (칩 면적, 전력 효율 최고)**|
| **유연성** | 고정된 출력 (256, 512 등) | **SHAKE 등 임의의 길이(XOF) 무한 출력 가능**|

### XOF (eXtendable-Output Function)
SHA-3 스펀지 구조의 최대 장점이다. 입력된 데이터를 다 흡수한 뒤 스펀지를 계속 짜내면(Squeeze), 256비트, 1000비트, 10000비트 등 원하는 길이만큼 무한히 난수(해시)를 뽑아낼 수 있다. 이를 이용해 해시 함수 하나로 '스트림 암호' 기능까지 덤으로 쓸 수 있다.

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

### 기술사적 판단 (실무 하드웨어 보안 모듈 설계)

**상황**: IoT 스마트 홈 기기에 들어갈 초저전력 SoC를 설계하고 있다. 기기가 서버와 통신할 때 펌웨어 위변조를 막기 위해 하드웨어 해시 가속기(IP Core)를 넣어야 하는데, 실리콘 면적(비용)이 극도로 제한적이다.
**판단 및 SHA-3 적용 전략**:
1. **IP 코어 선정**: 면적을 많이 차지하는 SHA-2 블록 대신, 단순 논리 게이트로 이루어진 **오픈 소스 Keccak(SHA-3) 코어**를 선택하여 실리콘 면적을 30% 이상 줄인다.
2. **SHAKE128 활용**: 펌웨어 무결성 검증뿐만 아니라, 기기 고유의 난수 생성기(RNG)와 스트림 암호화용 키 도출 함수(KDF) 역할까지 **SHA-3의 XOF 기능 하나로 퉁쳐서** 하드웨어 재사용성을 극대화한다.
3. **결과**: 극소형 칩 안에서 최상위 보안 등급(양자 내성)을 유지하면서도 배터리 소모를 극한으로 낮춘 스마트 기기 출시 성공.

### 안티패턴 및 고려사항
- **성급한 전면 교체**: 현재 웹 생태계의 SSL/TLS 인증서는 99%가 SHA-256을 쓴다. 보안에 더 좋다고 서버 애플리케이션의 기본 해시를 강제로 SHA-3로 바꾸면, 구형 클라이언트 기기(Intel 가속기 미지원 등)에서 로그인 속도가 눈에 띄게 느려질 수 있다. SHA-2가 완전히 뚫리기 전까지는 소프트웨어 진영에서 점진적 도입을 권장한다.

## Ⅴ. 미래 전망 및 결론

### 결론: 아름답고 우아한 수학적 승리
SHA-3(Keccak)는 복잡하고 무거운 수학 공식을 버리고, 비트 단위의 회전과 섞임이라는 가장 '컴퓨터다운(하드웨어 친화적인)' 방식으로 완벽한 보안을 이뤄냈다. 

### 미래 전망
가까운 미래에 양자 컴퓨터가 상용화되면 기존의 RSA, ECC 같은 공개키 암호가 모두 깨지게 된다. 하지만 해시 함수는 양자 컴퓨터(Grover 알고리즘)로도 깨기 어려워, 차세대 **'양자 내성 암호(PQC)'**의 뼈대는 대부분 이 SHA-3(Keccak) 해시 함수를 기반으로 만들어지고 있다. 결국 향후 10년 내 모든 디지털 보안의 심장부는 SHA-3로 대체될 것이다.

3. **가장 좋은 점은?**: 기계가 너무 작아서 우리 손목시계나 작은 로봇 안에도 쏙 들어가고, 미래의 슈퍼컴퓨터(양자 컴퓨터)가 해킹하려고 해도 절대 뚫을 수 없답니다!
