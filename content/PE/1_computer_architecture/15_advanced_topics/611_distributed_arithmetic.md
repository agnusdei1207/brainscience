# 611. 분산 산술 (Distributed Arithmetic) - 곱셈기를 버리고 룩업 테이블(LUT)을 취하다

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 디지털 신호 처리(DSP)나 필터(FIR) 연산에서 필수적인 '곱셈과 덧셈의 반복(MAC)' 연산을, **실제 곱셈기(Multiplier) 하드웨어를 쓰지 않고 메모리(LUT, Look-Up Table)와 덧셈기만으로 수행**하는 독창적인 하드웨어 설계 기법이다.
> 2. **혁신**: 곱해야 할 상수(계수) 값들을 미리 계산하여 메모리(ROM/LUT)에 구워놓고, 입력 데이터의 비트(Bit)들을 주소로 삼아 메모리에서 답을 찾아 더하는 방식으로 병목을 제거한다.
> 3. **가치**: 값비싸고 면적을 많이 차지하는 하드웨어 곱셈기가 부족한 저가형 FPGA나 초소형 ASIC에서, **극강의 칩 면적(Area) 절약과 고속 연산(High Throughput)**을 동시에 달성하게 해준다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
분산 산술(Distributed Arithmetic, DA)은 $Y = C_1X_1 + C_2X_2 + C_3X_3$ 와 같은 내적(Inner Product) 연산을 할 때, $C$(상수)와 $X$(입력)를 직접 곱하는 대신 연산의 방식을 '비트 레벨'로 분해(Distribute)하여 재조합하는 기술이다. 곱셈 대신 덧셈과 쉬프트(Shift) 연산만 남는다.

### 💡 비유
- **일반 곱셈 (계산기 두드리기)**: 153 × 7을 할 때, 머릿속으로 구구단을 외워서 일일이 곱하고 더해서 답을 구한다. 시간이 오래 걸리고 머리(곱셈기)가 아프다.
- **분산 산술 (답안지 훔쳐보기)**: 아예 "× 7"에 대한 모든 정답(1×7=7, 2×7=14...)을 종이(LUT)에 미리 다 적어놓는다. 153이 들어오면 계산할 필요 없이, 종이에서 1일 때 답, 5일 때 답, 3일 때 답을 손가락(주소)으로 짚어서 그냥 더하기만 한다. 엄청나게 빠르고 곱셈할 줄 몰라도 된다.

### 등장 배경: 'Multiplier'의 한계
과거 초창기 FPGA나 칩에는 하드웨어 곱셈기가 아예 없거나 개수가 매우 적었다. 하지만 디지털 필터(음성, 영상 신호 처리)는 초당 수백만 번의 곱셈을 해야 했다. 엔지니어들은 "곱셈기가 없으면 메모리로 곱셈을 대체하자!"라는 기발한 수학적 변환 기법을 고안해 냈고, 이것이 분산 산술(DA)이다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 분산 산술(DA) 기반 FIR 필터의 구조 (ASCII Diagram)

```ascii
[ Input Data X (e.g., 4-bit width) ]
   b3  b2  b1  b0  (비트 슬라이스)
   │   │   │   │
┌──▼───▼───▼───▼──────────────────────┐
│  Shift Registers (입력 데이터 지연)   │
└─────────────────────────────────────┘
         │ (특정 자리수의 비트들만 모아서 주소 생성)
         ▼
┌─────────────────────────────────────┐
│  Look-Up Table (LUT / ROM)          │
│  [ Address ]      [ Pre-computed ]  │
│     0000    --->      0             │
│     0001    --->      C1            │
│     0010    --->      C2            │
│     0011    --->    C1 + C2         │ <-- 상수를 미리 다 더해놓음!
│     ....            .......         │
└─────────────────────────────────────┘
         │ (메모리에서 찾은 값)
         ▼
┌─────────────────────────────────────┐
│ Shift & Accumulate (덧셈 및 쉬프트)   │
└─────────────────────────────────────┘
         │
         ▼ [ 최종 출력 Y ]
```

### 수학적 매커니즘 (Bit-level Distribution)

내적 식 $Y = \sum C_i X_i$ 에서, 입력 $X_i$를 2진수 비트($b_0, b_1, b_2...$)로 분해한다.
1. 식을 완전히 분해한 뒤 괄호로 묶는 순서를 바꾼다 (분산 법칙).
2. 같은 자리(예: $2^0$ 자리)에 있는 비트들끼리 묶어서, 상수 $C$들의 합을 표현한다.
3. 나올 수 있는 **모든 상수 합의 조합($2^N$개)을 미리 계산해서 메모리(LUT)에 저장**한다.
4. 런타임에는 $X$의 0번째 비트들을 모아 메모리 주소를 만들어 값을 찾고, 다음 클럭엔 1번째 비트들로 값을 찾아 이전 값에 Shift하여 더한다(Accumulate).

## Ⅲ. 융합 비교 및 다각도 분석

### MAC 연산기 (곱셈기) vs DA (분산 산술) 비교

| 비교 항목 | 전용 MAC 하드웨어 (DSP Slice) | **분산 산술 (Distributed Arithmetic)** |
|-----------|-------------------------------|----------------------------------------|
| **핵심 부품** | 덧셈기 + **곱셈기(Multiplier)** | 덧셈기 + **메모리(LUT/ROM)** |
| **연산 속도** | 1 클럭에 1 데이터 완료 (매우 빠름) | 입력 데이터의 비트 수만큼 클럭 소요 (예: 16비트면 16클럭) |
| **칩 면적(Area)** | 엄청나게 큼 | **매우 작음 (LUT는 어디에나 널려있음)** |
| **상수 유연성** | 실시간으로 계수(C) 변경 가능 | **계수(C)가 고정되어야 유리함** |
| **최적 사용처** | 고성능 CPU/GPU 부동소수점 연산| **초소형 FPGA 기반 FIR 필터, 오디오 신호처리** |

### 병렬 처리의 진화 (Fully Parallel DA)
DA는 원래 비트 수만큼 16클럭을 기다려야 하는 직렬 처리 구조다. 하지만 LUT를 16개로 복제해서 동시에 찾아내면(Fully Parallel), 곱셈기를 쓰지 않고도 1클럭 만에 답을 내는 기적의 아키텍처를 구성할 수 있다. (면적과 속도의 궁극적 트레이드오프).

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

### 기술사적 판단 (실무 FPGA 최적화 전략)

**상황**: 통신 기지국에 들어가는 싸구려 저가형 FPGA(예: Xilinx Spartan 시리즈)를 사용하여 노이즈 캔슬링용 64-Tap FIR 필터를 구현해야 한다. 칩 내부에 내장된 하드웨어 곱셈기(DSP) 블록은 10개밖에 없어 64개의 연산을 감당할 수 없다.
**판단 및 DA 적용 전략**:
1. **계수 고정(Fixed Coefficients)**: 노이즈 캔슬링 필터의 상수(C)는 미리 계산되어 바뀌지 않는 점을 이용한다.
2. **LUT 테이블 생성**: 64개의 상수를 조합한 결과값을 파이썬 스크립트로 미리 계산하여 FPGA 내부의 남는 **SRAM(BRAM/LUT)** 영역에 굽는다.
3. **DA 매크로 설계**: 고가의 DSP 블록(곱셈기)을 단 한 개도 쓰지 않고, 오직 쉬프트 레지스터와 덧셈기만으로 64-Tap 필터를 구현한다.
4. **결과**: 고가형 칩으로 올려야 했던 프로젝트를 저가형 칩에서 100% 동작하게 만들어, 대량 생산 시 칩 원가를 수십억 원 절감함.

### 안티패턴 및 고려사항
- **변동 계수 필터 (Adaptive Filter)**: 입력 데이터뿐만 아니라 곱해야 하는 상수(C)마저 실시간으로 계속 바뀌는 머신러닝 학습 모델 등에 DA를 적용하면, 상수가 바뀔 때마다 거대한 LUT 테이블 전체를 매번 다시 계산해야 하므로 시스템이 폭파된다. DA는 **'고정된 상수'**를 곱할 때만 빛을 발한다.

## Ⅴ. 미래 전망 및 결론

### 결론: 실리콘의 연금술
분산 산술(DA)은 "비싼 부품(곱셈기)이 없으면, 싼 부품(메모리)으로 우회해서 풀면 된다"는 컴퓨터 공학의 철학을 가장 아름답게 보여주는 하드웨어 트릭이다.

### 미래 전망
현대의 최신 FPGA에는 하드웨어 곱셈기(DSP Slice)가 수천 개씩 박혀 나오기 때문에 과거만큼 DA가 절실하지는 않다. 그러나 최근 초저전력 엣지 AI(TinyML)나 웨어러블 칩(ASIC) 등 **전력과 면적을 극한으로 쥐어짜야 하는 분야**에서, 가중치(Weight)가 고정된 딥러닝 추론용 인공신경망 코어를 설계할 때 DA 방식이 다시 각광받으며 부활하고 있다.

## 👶 어린이를 위한 3줄 비유 설명
1. **분산 산술이 뭔가요?**: 엄청 복잡한 곱하기 문제를 풀어야 할 때, 곱하기를 아예 하지 않고 '정답이 적힌 커다란 수첩'을 보고 덧셈만 해서 문제를 푸는 똑똑한 꼼수예요.
2. **왜 곱하기를 안 하나요?**: 컴퓨터 부품 중에 '곱하기 기계'는 너무 비싸고 뚱뚱해서 자리를 많이 차지하거든요. 수첩(메모리)은 작고 싸서 잔뜩 넣을 수 있어요.
3. **어디에 쓰나요?**: 아주 작고 건전지도 조금만 들어가는 귀 안의 '보청기' 같은 기계에서 소리를 깨끗하게 만들어 줄 때 이 꼼수가 최고랍니다!
