# 614. 페이지 랭크 알고리즘 하드웨어 맵핑 - 구글의 검색 엔진을 실리콘에 새기다

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 구글 검색의 핵심이자 대표적인 그래프 알고리즘인 '페이지 랭크(PageRank)'의 복잡한 행렬-벡터 곱셈(SpMV) 연산을, **CPU나 GPU의 소프트웨어가 아닌 커스텀 하드웨어(FPGA/ASIC)의 물리적 회로로 직접 구현(Mapping)**하는 기술이다.
> 2. **혁신**: 웹 페이지 간의 링크(Edge) 구조는 극단적으로 흩어져 있는 '희소 행렬(Sparse Matrix)' 형태를 띠므로, 이를 하드웨어로 가속하기 위해 **캐시 적중률을 높이는 정점 정렬(Vertex Sorting) 기법과 여러 개의 처리 요소(PE)를 병렬 연결한 파이프라인 아키텍처**를 사용한다.
> 3. **가치**: 수십억 개의 웹 문서나 소셜 네트워크 사용자들의 중요도를 실시간으로 평가해야 하는 초대형 그래프 분석 시스템에서 전력 소모를 획기적으로 낮추고 스루풋(Throughput)을 수십 배 끌어올린다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
페이지 랭크는 "중요한 페이지가 링크를 건 페이지 역시 중요할 것이다"라는 가정하에, 전체 네트워크 노드들의 점수를 반복적으로 계산하여 수렴시키는 알고리즘이다. 하드웨어 맵핑이란 이 $P_{i} = \sum (P_{j} / L_{j})$ 라는 반복 계산 공식을 트랜지스터들의 데이터 흐름(Data-flow)으로 물리적으로 구성하는 것이다.

### 💡 비유
- **소프트웨어 방식 (우체부의 반복 배달)**: 우체부 1명이 1번 집부터 100만 번 집까지 돌아다니며 "당신에게 온 편지가 3통이니 3점을 주겠소"라고 적어준다. 이 과정을 점수가 변하지 않을 때까지 수십 번 반복한다. 너무 넓고 무작위라 지친다.
- **하드웨어 맵핑 (물리적 파이프 연결망)**: 100만 개의 물통을 모두 파이프로 연결한다. 물(점수)을 펌프로 쏘아 보내면, 파이프의 두께(링크 수)에 따라 물이 알아서 쪼개져서 흘러간다. 물이 더 이상 출렁이지 않고 잔잔해질 때(수렴) 각 물통의 수위를 재기만 하면 끝난다. CPU의 명령어가 개입할 틈도 없이 물리 법칙에 의해 연산이 완료된다.

### 등장 배경: 'Sparsity'의 덫
웹 링크 구조를 행렬로 만들면 99.99%가 '0'으로 채워진 희소 행렬(Sparse Matrix)이 된다. 일반 GPU는 꽉 찬 행렬(Dense)을 한 번에 곱하는 데는 신이지만, 0이 대부분인 희소 행렬을 만나면 연산기의 99%가 놀면서 멍을 때리고, 메모리만 미친 듯이 왔다 갔다 하는 '메모리 장벽'에 갇힌다. 이 구조적 결함을 깨기 위해 특수 하드웨어가 필요했다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 하드웨어 기반 PageRank 가속기 아키텍처 (ASCII Diagram)

```ascii
[ Off-chip Memory (DRAM) ]
  - Vertex Data (노드 점수)
  - Edge Data (CSR 포맷 등 압축된 링크 정보)
        │
        ▼ (메모리 대역폭 최적화)
┌────────────────────────────────────────────────────────┐
│ [ FPGA / ASIC Accelerator ]                            │
│                                                        │
│  1. Dispatcher: Edge 데이터를 읽어와서 타겟 주소 분석    │
│           │                                            │
│  2. On-chip Buffer (BRAM): 자주 참조되는 노드 캐싱      │
│           │                                            │
│  3. Processing Elements (PE Array)                     │
│    ┌─────┐  ┌─────┐  ┌─────┐  ┌─────┐                  │
│    │ PE0 │  │ PE1 │  │ PE2 │  │ PE3 │  <-- 덧셈, 나눗셈│
│    └─────┘  └─────┘  └─────┘  └─────┘      파이프라인  │
│           │                                            │
│  4. Accumulator / Update Unit: 새로운 점수로 누적 갱신   │
└────────────────────────────────────────────────────────┘
        │
        ▼
[ Off-chip Memory (DRAM - 다음 반복(Iteration)을 위해 저장) ]
```

### 3대 핵심 맵핑 및 최적화 기술

#### ① CSR (Compressed Sparse Row) 포맷 해독기
0이 수억 개인 행렬을 그대로 메모리에 올릴 수 없다. '0이 아닌 값만 골라서 (열 주소, 행 포인터, 값)' 형태로 압축한 CSR 포맷을 하드웨어가 실시간으로 압축 해제(Decode)하며 읽어 들이는 전용 파서를 칩 입구에 둔다.

#### ② 파이프라인 데이터 포워딩 (Data Forwarding)
페이지 랭크는 A 노드의 점수가 업데이트되면 그 즉시 B 노드 계산에 반영되어야 수렴이 빠르다. 메모리에 썼다가 다시 읽어오면 지연이 생기므로, PE(계산 유닛) 간에 방금 계산된 점수를 곧바로 토스해 주는 포워딩 네트워크(버퍼)를 구축한다.

#### ③ 랜덤 메모리 접근 회피 (Graph Partitioning)
하드웨어 맵핑의 가장 큰 적은 무작위 메모리 읽기다. 전체 웹 지도를 수천 개의 작은 '동네(Sub-graph)'로 쪼개어, 칩 내부의 초고속 SRAM 크기에 딱 맞게 데이터를 꽉 채워 넣고 계산한 뒤 다음 동네로 넘어가는 블로킹(Blocking) 최적화 기법을 적용한다.

## Ⅲ. 융합 비교 및 다각도 분석

### CPU vs GPU vs FPGA(하드웨어 맵핑) 성능 비교

| 플랫폼 | 범용 CPU | 범용 GPU | **FPGA / ASIC (커스텀 HW)** |
|:---:|:---|:---|:---|
| **연산 유닛** | 적음 (코어 16~64개) | 매우 많음 (수천 개) | **필요한 만큼 맞춤형 (PE 수백 개)** |
| **메모리 접근** | 랜덤 접근에 다소 강함 | **랜덤 접근 시 효율 급감** | **SRAM 캐싱 및 커스텀 라우팅으로 극복** |
| **제어 로직** | 복잡한 조건문(If) 잘함 | 조건문(If) 만나면 멈춤 | **조건문 자체를 물리적 선(Wire)으로 분기** |
| **전력 소모** | 높음 | 매우 높음 (300W 이상) | **매우 낮음 (10~50W)** |
| **효율성** | 낮음 | 중간 | **최상 (에너지 대비 성능 압도적)** |

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

### 기술사적 판단 (실무 소셜 네트워크 분석 서버 설계)

**상황**: 초거대 SNS 기업에서 20억 명의 사용자 사이의 '인플루언서 랭킹(PageRank)'을 실시간 1초 이내로 업데이트하는 백엔드를 만들려 한다. 기존에는 하둡 클러스터 수백 대가 야간에 10시간씩 돌려서 계산했다.
**판단 및 하드웨어 가속기 도입 전략**:
1. **플랫폼 선정**: 전력 소모를 줄이면서도 실시간성을 확보하기 위해, PCIe 슬롯에 장착하는 **클라우드형 FPGA(AWS F1 등)**를 도입하여 페이지 랭크 코어를 맵핑한다.
2. **소프트웨어 전처리 (Pre-processing)**: CPU가 전체 20억 명의 그래프를 FPGA 내부 SRAM에 쏙 들어가는 사이즈(예: 1MB 단위)로 잘게 쪼개어(Partitioning) FPGA의 DDR 메모리로 밀어 넣는다.
3. **오프로딩 (Offloading)**: FPGA는 CPU 개입 없이 스스로 쪼개진 뭉치들을 읽어와서 파이프라인 방식으로 점수를 누적(Accumulate)한다.
4. **결과**: 서버 수백 대가 하던 작업을 랙 1대의 FPGA 클러스터가 전력 소모 1/10 수준으로 1초 만에 실시간 랭킹 연산을 끝내버림.

### 안티패턴 및 고려사항
- **부동소수점(Floating Point) 강박**: 페이지 랭크 점수는 정확한 실수 값이 아니어도 상대적인 크기만 비교하면 된다. 이를 완벽한 IEEE-754 64비트 Double 정밀도로 하드웨어를 짜면 면적만 엄청나게 잡아먹고 느려진다. 하드웨어 맵핑 시에는 **16비트나 8비트 Fixed-point(고정 소수점)**로 데이터 크기를 깎아내어(Quantization) 속도와 면적을 극한으로 이득 보는 것이 정석이다.

## Ⅴ. 미래 전망 및 결론

### 결론: 알고리즘의 화석화
소프트웨어로 구현된 알고리즘을 실리콘으로 맵핑하는 것은 '소프트웨어를 굳혀서 화석으로 만드는 작업'이다. 유연성은 잃지만 불멸의 속도를 얻는다. 페이지 랭크 맵핑은 그래프 분석이 더 이상 SW의 전유물이 아님을 증명했다.

### 미래 전망
앞으로는 페이지 랭크뿐만 아니라 GNN(그래프 신경망), 랜덤 워크(Random Walk) 등 다양한 그래프 알고리즘들을 하나의 칩에서 동시에 가속할 수 있는 **'CGRA(Coarse-Grained Reconfigurable Architecture)'** 형태의 범용 그래프 AI 반도체가 시장을 주도할 것이다. 또한, 차세대 메모리인 CXL과 결합하여 수 테라바이트의 그래프 데이터를 칩으로 퍼 올리는 대역폭의 혁명이 일어날 것이다.

## 👶 어린이를 위한 3줄 비유 설명
1. **하드웨어 맵핑이 뭔가요?**: 똑똑한 사람(소프트웨어)이 종이에 수학 문제를 계속 푸는 대신, 아예 "수학 문제만 푸는 전용 계산기 기계(하드웨어)"를 쇳덩이로 만들어버리는 거예요.
2. **페이지 랭크는 뭔가요?**: 세상에서 가장 유명한 친구가 누구인지, 친구들의 연결된 선을 보고 점수를 매기는 구글의 아주 유명한 계산법이에요.
3. **왜 기계로 만드나요?**: 지구상의 모든 사람의 연결고리를 사람이 계산하려면 백 년이 걸리지만, 이 전용 기계에 넣고 버튼을 누르면 1초 만에 "제일 유명한 사람은 1번 친구!"라고 튀어나오거든요!
