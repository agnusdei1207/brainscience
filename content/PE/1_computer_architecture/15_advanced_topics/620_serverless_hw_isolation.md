# 620. 서버리스 컴퓨팅 컨테이너 분리 하드웨어 기술 - 구름 속의 철벽 요새

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 수천 명의 사용자가 동시에 함수(Function)를 실행하는 서버리스 환경에서, **서로 다른 사용자의 코드가 물리적으로 같은 하드웨어를 공유하더라도 완벽하게 격리(Isolation)되도록 보장하는 칩 레벨의 보안 기술**이다.
> 2. **혁신**: 기존 가상 머신(VM)의 무거운 오버헤드와 일반 컨테이너의 보안 취약점을 동시에 해결하기 위해, **마이크로VM(Firecracker 등)과 하드웨어 기반 가상화 가속(Intel VT-x, VT-d) 기술**을 융합하여 밀리초 단위의 실행 속도와 강력한 보안 격리를 달성했다.
> 3. **가치**: "신뢰할 수 없는 코드"를 내 서버에서 돌려야 하는 클라우드 사업자의 리스크를 최소화하며, 사용자가 보안 걱정 없이 0.1초 단위로 자원을 빌려 쓸 수 있는 진정한 서버리스 경제의 물리적 토대를 제공한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
서버리스 컴퓨팅에서는 내 코드와 옆집 사람 코드가 같은 CPU, 같은 RAM에서 동시에 돌아간다. 만약 내 코드가 옆집 사람의 메모리를 훔쳐볼 수 있다면 대재앙이 일어난다. 서버리스 컨테이너 분리 기술은 하드웨어가 직접 "여기서부터 여기까지는 너만 써, 절대 선 넘지 마!"라고 물리적인 벽을 쳐주는 기술을 말한다.

### 💡 비유
- **일반 서버 (내 집)**: 내가 집 전체를 소유한다. 내가 뭘 하든 내 마음이고 남이 들어올 걱정이 없다. (보안 100%, 효율 10%).
- **일반 컨테이너 (고시원)**: 한 건물(커널)을 공유하며 얇은 가벽(Software 격리)을 친다. 옆방 사람이 벽에 귀를 대면 소리가 들릴 수도 있고(보안 취약), 한 명이 불을 내면 건물 전체가 위험하다.
- **서버리스 하드웨어 분리 (최첨단 캡슐 호텔)**: 각 방이 특수 방음 및 방화 소재(Hardware 격리)로 되어 있다. 옆 방 사람이 무슨 짓을 해도 내 방은 완벽히 안전하며, 각 방은 필요할 때만 0.1초 만에 뚝딱 생겨났다가 사라진다.

### 등장 배경: 'Cold Start'와 'Security'의 딜레마
초기에는 완벽한 보안을 위해 무거운 가상 머신(VM)을 썼다. 하지만 VM은 부팅에 수 초가 걸려 서버리스의 핵심인 '즉시 실행'이 불가능했다. 반대로 가벼운 컨테이너(Docker)를 쓰자니 보안이 불안했다. 결국 "VM처럼 안전하면서 컨테이너처럼 가벼운" 하드웨어 가속 기반의 마이크로VM 기술이 필요해졌다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 하드웨어 기반 격리 아키텍처 (ASCII Diagram)

```ascii
[ User A's Function ]      [ User B's Function ]
┌──────────────────┐      ┌────────────────────┐
│ Guest OS (Light) │      │  Guest OS (Light)  │  <-- 최소한의 커널
├──────────────────┤      ├────────────────────┤
│   MicroVM        │      │     MicroVM        │  <-- 5ms 이내 실행
└────────┬─────────┘      └──────────┬─────────┘
         │ (MMU / VT-x)              │ (MMU / VT-x)
┌────────▼───────────────────────────▼──────────┐
│ [ Hypervisor (KVM / Firecracker) ]            │
├──────────────────────────────────────────────┤
│ [ Host Hardware (CPU, RAM, NIC) ]            │
│  - EPT (Extended Page Tables): 메모리 물리 격리 │
│  - VT-d: I/O 장치 직접 할당 및 격리            │
└──────────────────────────────────────────────┘

* 핵심 원리: 소프트웨어가 아닌 하드웨어의 '페이지 테이블'과 
  '가상화 명령어'를 사용하여 메모리 영역을 물리적으로 절단.
```

### 3대 핵심 기술 요소

#### ① 마이크로VM (MicroVM - 예: AWS Firecracker)
기존 VM에서 키보드, 마우스, 비디오 등 서버리스에 불필요한 장치 90%를 제거한 초경량 가상화 기술이다. 하드웨어 가속(KVM)을 직접 호출하여 5ms(0.005초) 만에 보안이 완벽한 독립 공간을 창조한다.

#### ② 하드웨어 메모리 보호 (EPT / RVI)
CPU 내부에 있는 메모리 관리 유닛(MMU)이 '이중 주소 변환'을 수행한다. 사용자 A의 함수가 메모리 1번지를 읽으려 해도, 하드웨어가 이를 '물리 주소 500번지'로 강제 매핑하여 사용자 B의 영역(1000번지)은 쳐다볼 수도 없게 차단한다.

#### ③ 인클레이브 (Enclave / TEE)
가장 강력한 격리 기술이다. **Intel SGX** 같은 기술을 사용하면, 심지어 클라우드 관리자(Root)나 운영체제조차 함수 내부의 데이터(비밀번호, 개인정보)를 절대 읽을 수 없도록 하드웨어가 특수 암호화된 메모리 영역을 따로 떼어준다.

## Ⅲ. 융합 비교 및 다각도 분석

### 격리 수준에 따른 컴퓨팅 기술 비교

| 비교 항목 | 일반 컨테이너 (Docker) | **마이크로VM (Firecracker)** | 전통적 VM (VMware) |
|-----------|------------------------|------------------------------|--------------------|
| **격리 주체** | 소프트웨어 (Kernel Namespace) | **하드웨어 (KVM / VT-x)** | 하드웨어 (Hypervisor) |
| **보안 수준** | 보통 (커널 탈취 시 위험) | **매우 높음 (물리적 격리)** | 매우 높음 |
| **시작 속도** | **수 밀리초 (매우 빠름)** | **수십 밀리초 (빠름)** | 수 초 ~ 수 분 (느림) |
| **오버헤드** | 거의 없음 | **매우 낮음 (5MB 내외)** | 높음 (GB 단위) |
| **적합한 용도**| 마이크로서비스 (MSA) | **서버리스 (AWS Lambda)** | 레거시 앱, 복잡한 OS 환경 |

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

### 기술사적 판단 (실무 멀티 테넌시 보안 전략)

**상황**: 핀테크 기업에서 외부 개발자들이 우리 금융 API를 활용해 자신만의 '커스텀 결제 로직'을 코딩해서 올릴 수 있는 플랫폼을 구축하려 한다. 외부인의 코드가 우리 회사 내부망을 공격할까 봐 걱정된다.
**판단 및 하드웨어 격리 도입 전략**:
1. **기술 선정**: 단순 도커 컨테이너는 컨테이너 탈출(Escape) 공격에 취약하므로, 하드웨어 격리를 보장하는 **Firecracker(마이크로VM)** 기반의 서버리스 환경을 구축한다.
2. **I/O 격리**: 외부 코어가 네트워크 장비에 직접 접근하지 못하도록 하드웨어의 **VT-d(Directed I/O)** 기능을 사용하여 가상 네트워크 카드만 제한적으로 할당한다.
3. **리소스 쿼터(Quota) 강제**: 한 명의 코드가 CPU를 100% 점유하여 다른 사용자를 방해하는 'Noisy Neighbor' 문제를 막기 위해, 하드웨어 수준에서 CPU 타임슬롯을 강제 할당한다.
4. **결과**: 외부인의 '신뢰할 수 없는 코드'를 우리 서버에서 돌리면서도, 물리적인 벽을 통해 내부망을 완벽히 보호하고 자원 효율은 극대화함.

### 안티패턴 및 고려사항
- **Shared Kernel의 맹신**: 일반 컨테이너 환경에서 `Root` 권한을 주는 것은 하드웨어 격리 없는 아파트에서 옆집에 열쇠를 주는 것과 같다. 서버리스 환경을 직접 구축할 때는 반드시 **하드웨어 가속(VT-x)**이 지원되는 서버 사양을 확인하고 하이퍼바이저와 연동해야 한다.

## Ⅴ. 미래 전망 및 결론

### 결론: 하드웨어가 보장하는 신뢰
서버리스 컴퓨팅의 성공은 소프트웨어가 아니라 하드웨어가 얼마나 빨리, 얼마나 안전하게 벽을 칠 수 있느냐에 달려 있었다. 하드웨어 분리 기술은 클라우드 상에서 "남의 떡이 내 떡보다 안전함"을 증명해 낸 핵심 인프라다.

### 미래 전망
앞으로는 **'컨피덴셜 서버리스(Confidential Serverless)'**가 표준이 될 것이다. 하드웨어가 격리만 해주는 것이 아니라, 연산 중인 모든 데이터를 실시간 암호화하여 클라우드 사업자조차 고객의 함수 내용을 알 수 없게 만드는 것이다. 또한, **[SmartNIC / DPU](./619_msa_traffic_hw.md)**가 함수 격리 작업을 전담(Offloading)하여, 메인 CPU는 오직 함수 로직에만 100% 집중하는 '무결점 무지연' 서버리스 시대가 도래할 전망이다.

## 👶 어린이를 위한 3줄 비유 설명
1. **격리 하드웨어가 뭔가요?**: 여러 명의 아이가 커다란 도화지(컴퓨터) 하나에 그림을 그리는데, 서로의 그림을 망치지 못하게 투명하고 단단한 '유리벽'을 하드웨어가 직접 쳐주는 거예요.
2. **왜 유리벽을 치나요?**: 나쁜 친구가 내 그림을 훔쳐보거나 지워버리는 걸 막기 위해서예요.
3. **어떤 점이 좋나요?**: 유리벽이 엄청나게 빨리 생겼다 사라지기 때문에, 우리는 벽이 있는지도 모른 채 아주 안전하고 편하게 내 그림만 그릴 수 있답니다!
