# 636. 연합 학습 (Federated Learning) - 데이터는 남기고 지능만 공유하는 분산 AI

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 개인의 민감한 로우 데이터(Raw Data)를 중앙 서버로 전송하지 않고, **수만 개의 단말기(Edge)에서 각자 모델을 학습시킨 뒤 계산된 '가중치(Weight) 업데이트분'만 서버로 보내 통합하는 분산형 머신러닝 아키텍처**이다.
> 2. **혁신**: 데이터 수집 과정에서의 프라이버시 침해 문제를 원천 차단하며, **차분 프라이버시(Differential Privacy)와 보안 합산(Secure Aggregation)** 기술을 결합하여 가중치 정보로부터 원본 데이터를 유추하는 역공학 공격까지 방어한다.
> 3. **가치**: 의료, 금융, 키보드 입력 예측 등 극도의 보안이 필요한 분야에서 거대 데이터를 활용한 고성능 AI 모델 구축을 가능케 하며, 네트워크 대역폭 비용 절감과 데이터 주권 보호를 실현한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
연합 학습(Federated Learning)은 "공부(학습)는 각자 집(단말기)에서 하고, 시험 정답(가중치)만 선생님(중앙 서버)에게 제출하는" 방식이다. 선생님은 학생들의 정답지를 모아 평균을 낸 뒤, 더 완벽한 해설지(Global Model)를 만들어 다시 학생들에게 나눠준다.

### 💡 비유
- **전통적 학습 (전교생 일기장 검사)**: 선생님이 학생 1,000명의 일기장을 교무실로 다 걷어간다. 일기장 속 비밀(개인정보)이 선생님에게 다 노출된다. (데이터 집중형).
- **연합 학습 (일기장 속 단어 통계)**: 학생들은 자기 방에서 일기장을 쓴다. 선생님이 "일기장에 '행복'이라는 단어가 몇 번 나오니?"라고 물으면, 학생들은 숫자만 적어서 보낸다. 선생님은 일기장 내용은 전혀 모르지만, 전교생이 요즘 얼마나 행복한지(전체 경향성)는 완벽히 알 수 있다.

### 등장 배경: 'Data Silo'와 'Privacy'의 갈등
AI 성능을 높이려면 많은 데이터가 필요하다. 하지만 병원들은 환자 정보를 공유하기 싫어하고, 사용자들은 내 스마트폰 속 사진이 구글 서버로 가는 것을 거부한다. "데이터는 가둬두고 지능만 합칠 순 없을까?"라는 고민이 구글에 의해 2017년 연합 학습이라는 논문으로 구체화되었다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 연합 학습 작동 프로세스 (ASCII Diagram)

```ascii
[ Central Aggregation Server ] (Global Model)
          ▲      │
          │ (2)  │ (1) Broadcast Initial Model
          │      ▼
┌─────────┴──────┴─────────┐ (Parallel Processing)
│ [ Device A ]  [ Device B ]  [ Device C ] ...
│ - Local Data  │ - Local Data  │ - Local Data
│ - Local Train │ - Local Train │ - Local Train
└─────────┬──────┬─────────┘
          │ (3)  │ Update Gradients / Weights
          ▼      ▼
[ Aggregation Algorithm (FedAvg) ]
  - New Global Model = Average(Updates)
  - Loop back to (1)

* 핵심 단계:
1. 전파: 서버가 최신 글로벌 모델을 단말기들에 배포.
2. 학습: 단말기가 로컬 데이터로 모델을 미세 조정(Fine-tuning).
3. 전송: 학습된 '가중치 차이값'만 서버로 전송.
4. 합산: 서버가 FedAvg 알고리즘으로 가중치를 합쳐 모델 갱신.
```

### 3대 핵심 보안 기술

#### ① 보안 합산 (Secure Aggregation)
서버가 개별 단말기의 가중치를 직접 보지 못하게 하는 기술이다. 여러 단말기의 데이터를 암호학적으로 섞어서 서버로 보내면, 서버는 "합계"는 알 수 있지만 "누가 무엇을 보냈는지"는 절대 알 수 없다.

#### ② 차분 프라이버시 (Differential Privacy)
가중치 데이터에 미세한 수학적 '노이즈'를 섞는다. AI 모델이 너무 똑똑해져서 특정 개인의 데이터를 기억해버리는 '과적합 유출'을 막는 방패다.

#### ③ 동형 암호 (Homomorphic Encryption)
가중치를 암호화된 상태 그대로 서버에서 평균을 내는 기술이다. 데이터 유출 가능성을 물리적으로 0%에 수렴하게 만든다.

## Ⅲ. 융합 비교 및 다각도 분석

### 중앙 집중형 학습 vs 연합 학습 비교 (Table)

| 구분 | 중앙 집중형 학습 (Centralized) | **연합 학습 (Federated)** |
|:---:|:---|:---|
| **데이터 위치** | 중앙 서버로 모두 수집 | **개별 단말기에 상주** |
| **개인정보 보호**| 유출 위험 높음 | **매우 우수 (수집 안 함)** |
| **통신 부하** | 원본 데이터 전송 (매우 큼) | **가중치 정보만 전송 (작음)** |
| **학습 환경** | 고성능 GPU 클러스터 | 저사양 단말기 (이기종 환경) |
| **데이터 품질** | 정제된 균일 데이터 | 편향되고 불균일한 데이터 (Non-IID) |
| **비유** | **중앙 집중 취사** | **각자 집에서 요리 후 레시피 공유** |

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

### 기술사적 판단 (실무 다기관 협력 의료 AI 구축 전략)

**상황**: 5개 대형 병원이 협력하여 '암 진단 AI'를 만들려 한다. 각 병원은 환자 개인정보 유출 우려로 데이터를 병원 밖으로 내보내는 것에 절대 반대한다.
**판단 및 구축 로드맵**:
1. **아키텍처 선정**: 데이터 이동이 불가능하므로 **연합 학습(Federated Learning)**을 표준 모델로 채택한다.
2. **이질성 극복**: 각 병원의 장비(MRI 사양 등)가 다르므로 데이터 분포가 불균일(Non-IID)하다. 이를 보정하기 위해 **FedProx** 알고리즘을 적용하여 학습 안정성을 높인다.
3. **인프라 보안**: 서버-병원 구간에 **mTLS 통신**을 기본으로 하고, 서버 내부에 **TEE(신뢰 실행 환경)**를 구축하여 가중치 합산 과정을 보호한다.
4. **결과**: 데이터 유출 없이 5개 병원의 방대한 임상 데이터를 학습한 고정밀 암 진단 모델 완성. 법적 리스크 제로화.

### 안티패턴 및 고려사항
- **가중치 오염 공격 (Poisoning Attack)**: 악의적인 단말기가 일부러 엉터리 가중치를 보내 전체 모델을 망가뜨릴 수 있다. 합산 시 **이상치 제거(Robust Aggregation)** 로직을 반드시 포함하여 오염된 가중치를 걸러내야 한다.

## Ⅴ. 미래 전망 및 결론

### 결론: 데이터의 벽을 넘는 지능의 공유
연합 학습은 "데이터 소유권"과 "AI 발전"이라는 두 마리 토끼를 잡는 유일한 기술적 해법이다. 데이터가 파편화된 세상에서 지능을 하나로 묶어주는 신뢰의 연결 고리다.

### 미래 전망
앞으로는 **'개인화 연합 학습(pFed)'**이 대세가 될 것이다. 전체의 지능을 공유받으면서도, 내 단말기에서는 내 습관에 더 맞춤화된 모델을 유지하는 것이다. 또한, **[6G](./896_5g_6g_mec.md)**와 **[온디바이스 AI](./635_on_device_ai.md)** 기술이 성숙해지면, 전 세계 수십억 개의 기기가 실시간으로 가중치를 주고받으며 1초마다 진화하는 '지구 규모의 실시간 집단 지성'이 탄생할 전망이다.

## 👶 어린이를 위한 3줄 비유 설명
1. **연합 학습이 뭔가요?**: 친구들이 각자 자기 집에서 요리 연습을 한 뒤, "설탕은 이만큼 넣는 게 맛있어!"라는 비법(레시피)만 모아서 최고의 요리책을 만드는 방법이에요.
2. **왜 좋나요?**: 내가 어떤 재료를 썼는지, 우리 집 주방이 어떻게 생겼는지 비밀로 지키면서도 세상에서 제일 맛있는 요리법을 배울 수 있기 때문이에요.
3. **미래에는요?**: 내 핸드폰이 내가 좋아하는 게임이나 말을 서버에 보내지 않고도, 혼자 공부해서 나를 더 잘 도와주는 척척박사가 된답니다!
