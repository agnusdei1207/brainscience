# 리플 캐리 가산기 (Ripple Carry Adder)

## 정의

**리플 캐리 가산기(Ripple Carry Adder)**는 여러 개의 전가산기를 **직렬로 연결**하여 자리올림이 왼쪽으로 물결처럼 전파되는 N비트 덧셈기다. 구조가 단순하지만 자리올림 전파로 인한 지연이 발생한다.

```
┌─────────────────────────────────────────────────────────────────┐
│           4비트 리플 캐리 가산기 구조                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   A₃ B₃  A₂ B₂  A₁ B₁  A₀ B₀                           │ │
│   │     │      │      │      │                             │ │
│   │     ▼      ▼      ▼      ▼                             │ │
│   │   ┌───┐  ┌───┐  ┌───┐  ┌───┐                         │ │
│   │   │FA │  │FA │  │FA │  │HA │                         │ │
│   │   └─┬─┘  └─┬─┘  └─┬─┘  └─┬─┘                         │ │
│   │     │      │      │      │                             │ │
│   │    S₃     S₂     S₁     S₀                            │ │
│   │     │      │      │      │                             │ │
│   │     │  C₃  │  C₂  │  C₁  │                             │ │
│   │     │◄─────┘◄─────┘◄─────┘                             │ │
│   │     │      자리올림 전파 (Ripple)                       │ │
│   │     ▼                                                  │ │
│   │    Cout                                                │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   💡 자리올림이 LSB에서 MSB로 순차적으로 전파                      │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 핵심 개념

### 1. 동작 원리

```
┌─────────────────────────────────────────────────────────────────┐
│           리플 캐리 가산기 동작 과정                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   예: A = 1111₂ (15), B = 0001₂ (1)                            │
│       결과: 10000₂ (16) → S = 0000, Cout = 1                   │
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   단계 1: A₀ + B₀ = 1 + 1 = 10₂                         │ │
│   │           → S₀ = 0, C₁ = 1                              │ │
│   │                                                         │ │
│   │   단계 2: A₁ + B₁ + C₁ = 1 + 0 + 1 = 10₂                │ │
│   │           → S₁ = 0, C₂ = 1                              │ │
│   │                                                         │ │
│   │   단계 3: A₂ + B₂ + C₂ = 1 + 0 + 1 = 10₂                │ │
│   │           → S₂ = 0, C₃ = 1                              │ │
│   │                                                         │ │
│   │   단계 4: A₃ + B₃ + C₃ = 1 + 0 + 1 = 10₂                │ │
│   │           → S₃ = 0, Cout = 1                            │ │
│   │                                                         │ │
│   │   최종 결과: Cout S₃S₂S₁S₀ = 10000₂                     │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   💡 최악의 경우: 모든 자리에서 자리올림 발생                      │
│      (자리올림이 MSB까지 전파되어야 함)                           │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2. 지연 시간 분석

```
┌─────────────────────────────────────────────────────────────────┐
│           리플 캐리 가산기 지연 시간                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   게이트 지연 가정:                                              │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │   • AND, OR: 1단위 지연                                 │ │
│   │   • XOR: 2단위 지연                                     │ │
│   │                                                         │ │
│   │   반가산기: Sum = 2, Carry = 1                          │ │
│   │   전가산기: Sum = 4, Cout = 3 (독립 구현 시)              │ │
│   │           Cout = 2 (Cin→Cout 경로)                      │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   N비트 RCA 지연 시간:                                           │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   T_RCA = T_HA_Carry + (N-1) × T_FA_Carry               │ │
│   │                                                         │ │
│   │   예 (4비트):                                            │ │
│   │   T_4bit = 1 + 3 × 2 = 7 단위                           │ │
│   │                                                         │ │
│   │   예 (32비트):                                           │ │
│   │   T_32bit = 1 + 31 × 2 = 63 단위                        │ │
│   │                                                         │ │
│   │   💡 지연 시간이 비트 수에 선형으로 증가 (O(N))             │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 기술사 시험 포인트

### 1. 핵심 정의

> **리플 캐리 가산기**: 전가산기를 직렬 연결한 N비트 가산기
> **지연 시간**: O(N), 자리올림 전파 지연이 주요 병목
> **구성**: LSB에 HA, 나머지에 FA

### 2. 빈출 키워드

- **자리올림 전파 (Carry Propagation)**: LSB→MSB로 전파
- **지연 시간**: T_RCA = T_HA + (N-1) × T_FA
- **O(N)**: 비트 수에 선형 비례
- **캐리 예측 가산기**: O(log N)으로 개선

### 3. 예상 문제

**Q1. 4비트 리플 캐리 가산기를 설계하고 지연 시간을 분석하시오.**

**Q2. 리플 캐리 가산기의 최악의 경우 지연 시간이 발생하는 입력 예시를 들고 설명하시오.**

**Q3. 리플 캐리 가산기의 단점을 극복하기 위한 고속 가산기 기법들을 설명하시오.**
