# 47. 레벨 트리거 (Level Trigger)

## Ⅰ. 개요

레벨 트리거(Level Trigger)는 디지털 논리 회로 및 마이크로프로세서 인터럽트 시스템 등에서 사용되는 동기화 및 신호 감지 방식으로, 제어 신호(예: 클럭, 인에이블, 인터럽트 요청)가 특정 전압 레벨인 논리 'High(1)' 또는 논리 'Low(0)' 상태에 머무는 시간 동안 지속적으로 소자의 상태 변화를 허용하거나 신호를 유효한 것으로 인식하는 메커니즘을 의미합니다.
에지 트리거(Edge Trigger)가 순간적인 변화율에 민감하다면, 레벨 트리거는 지속되는 상태 값 자체에 민감합니다. 이는 회로 구조가 단순하고 구현 면적이 작다는 장점이 있지만, 신호가 유지되는 동안 입력의 작은 변화나 노이즈에도 출력이 즉각적으로 반응하는 투명성(Transparency)을 가지므로 타이밍 제어에 각별한 주의가 요구되는 기술입니다.

## Ⅱ. 아키텍처 및 핵심 원리

### 1. 레벨 트리거의 파형 특성 및 종류
레벨 트리거는 활성화되는 전압 레벨에 따라 두 가지로 분류됩니다. 활성화 구간 내에서는 입력 신호의 변화가 즉시 출력에 반영(투명 상태)되며, 비활성화 구간에서는 이전 상태를 유지(불투명/저장 상태)합니다.

```text
[ 레벨 트리거 타이밍 다이어그램 ]

Clock / Enable Signal
     +-------------------+                       +-------------------+
     |                   |                       |                   |
H ---+                   +-----------------------+                   +---
     |                   |                       |                   |
L ---+                   +-----------------------+                   +---
     | <--- Active --->  | <----- Inactive ----> | <--- Active --->  |
         (High Level)          (Low Level)             (High Level)

[ Active-High 레벨 트리거의 동작 예시 ]
Input D  :   _--__---___----____---__
Output Q :   _--__---___----____---__ (Active 구간에서는 D의 형태를 그대로 따라감)
             |                   |
             |<- Transparent ->  |<- Hold (Opaque) ->|
```

### 2. 레벨 트리거 기반 래치(Latch) 회로 아키텍처
레벨 트리거 방식을 하드웨어 수준에서 구현한 가장 대표적인 소자가 바로 래치(Latch)입니다. 아래는 Enable(E) 신호에 의해 제어되는 D 래치의 아키텍처입니다.

```text
[ D Latch의 논리 게이트 아키텍처 (Active-High Level Triggered) ]

        Data (D) --------------------------+
                                           |
                                           V
                                     +-----\
                                     | AND  \
        Enable (E) -------+--------->| Gate 1|----+
                          |          |      /     |
                          |          +-----/      |
                          |                       |     +-----\
                          |                       +---->| NOR  \
                          |                             | Gate 1|-----> Q (Output)
                          V                       +---->|      /  |
                        +---+                     |     +-----/   |
                        |NOT|                     |               |
                        |   |o--+                 |   +-----------+
                        +---+   |                 |   | Feedback
                                |                 |   |
                                V                 |   |
                              +-----\             |   V
                              | AND  \            | +-----\
                  +---------->| Gate 2|----+      +-| NOR  \
                  |           |      /     |        | Gate 2|-----> Q' (Inverted Output)
                  |           +-----/      +------->|      /
                  |                                 +-----/
                  |
                  +-------------------------------- (Derived from Q logic or external D')
```

*   **동작 원리 (투명성 - Transparency)**:
    *   **E = 1 (Active High Level)**: Enable 신호가 High로 유지되는 동안, 입력 데이터 D의 상태 변화는 AND 게이트와 NOR 게이트를 거쳐 지연(Delay) 시간 이후에 즉각적으로 출력 Q에 반영됩니다. 이 상태를 회로가 투명(Transparent)하다고 표현합니다. 이 구간에서 D가 0에서 1로, 다시 0으로 바뀌면 Q도 동일하게 변화합니다.
    *   **E = 0 (Inactive Low Level)**: Enable 신호가 Low로 떨어지면, AND 게이트의 출력이 차단되어 입력 D의 변화가 내부 회로로 전달되지 못합니다. 이때 NOR 게이트의 교차 결합(Cross-coupled) 피드백 구조에 의해 E가 0으로 떨어지기 직전의 Q 상태가 계속 유지(Latch)됩니다.

## Ⅲ. 융합 비교 및 다각도 분석

### 1. 레벨 트리거 vs 에지 트리거 시스템 융합 관점 비교
레벨 트리거와 에지 트리거는 컴퓨터 아키텍처 내에서 서로 다른 목적을 위해 융합되어 사용됩니다.

| 비교 항목 | 레벨 트리거 (Level Trigger) | 에지 트리거 (Edge Trigger) |
| :--- | :--- | :--- |
| **감지 방식** | 신호의 전압 상태 (High / Low) | 신호의 변화 순간 (Rising / Falling) |
| **동작 상태** | 투명(Transparent) 구간 존재 | 비투명성, 에지에서만 순간 갱신 |
| **타이밍 제어** | 복잡함 (활성 구간 내 글리치 취약) | 단순하고 예측 가능함 (동기식 설계에 유리) |
| **면적/소모전력** | 게이트 수가 적어 작고 효율적임 | 래치 2개를 직렬 연결하여 크고 전력소모 큼 |
| **주요 사용처** | SRAM 메모리 셀, 인터럽트 신호선, 파이프라인 버퍼 | CPU 범용 레지스터, 카운터, 상태 머신 |

### 2. 레이스 컨디션(Race Condition) 위험성 분석
레벨 트리거 소자를 여러 개 직렬로 연결하여 시프트 레지스터를 구성할 경우 치명적인 레이스 컨디션이 발생합니다. 클럭(Enable)이 High인 상태가 지속되는 동안, 첫 번째 래치를 통과한 데이터가 두 번째, 세 번째 래치로 멈추지 않고 계속 전파(Propagation)되어 버리는 현상입니다. 이를 방지하기 위해서는 클럭 펄스 폭을 매우 좁게 유지해야 하지만, 하드웨어적으로 이를 정밀하게 제어하는 것은 불가능에 가깝습니다. 이것이 동기식 시스템에서 레벨 트리거 대신 에지 트리거를 기본 레지스터로 사용하는 근본적인 이유입니다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 1. 인터럽트(Interrupt) 처리에서의 실무 적용
마이크로프로세서의 외부 디바이스 인터럽트 요청(IRQ) 라인에서 레벨 트리거 방식이 광범위하게 사용됩니다. 여러 디바이스가 하나의 IRQ 라인을 공유(Open-Drain 또는 Open-Collector 구조 방식)할 때, 어떤 디바이스라도 라인을 Low(Active-Low Level Trigger)로 당기고 있으면 프로세서는 지속적으로 인터럽트를 인식할 수 있습니다. 이는 에지 트리거 방식의 인터럽트 라인 공유 시 발생할 수 있는 '인터럽트 누락(Interrupt Miss)' 문제를 방지하는 효과적인 설계 기법입니다.

### 2. 기술사적 판단: 타임 보로잉(Time Borrowing) 기법
레벨 트리거의 단점으로 지적되는 '투명성(Transparency)'은 고성능 반도체 설계에서 오히려 장점으로 승화될 수 있습니다. 이를 활용한 기법이 **타임 보로잉(Time Borrowing)** 또는 훔치기(Cycle Stealing)입니다.
*   파이프라인 단계에서 앞 스테이지의 논리 연산이 예상보다 늦게 끝나 데이터 도착이 지연되더라도, 에지 트리거 플립플롭과 달리 레벨 트리거 래치는 클럭이 High로 유지되는 구간 동안에는 늦게 도착한 데이터를 받아들여 출력으로 넘길 수 있습니다.
*   즉, 뒷 스테이지의 여유 시간을 앞 스테이지가 '빌려 쓰는' 효과를 창출하여 시스템 전체의 동작 주파수(Clock Frequency)를 높일 수 있습니다. 단, 이러한 설계는 타이밍 분석이 극도로 복잡해지므로 철저한 타이밍 시뮬레이션(STA 검증)이 수반되어야 합니다.

## Ⅴ. 기대효과 및 결론

레벨 트리거 메커니즘은 그 구현의 단순성과 낮은 전력 소모라는 강력한 무기를 가지고 있습니다. 비록 동기식 순차 회로의 주역 자리는 에지 트리거 플립플롭에게 내어주었지만, 메모리 기본 소자(SRAM), 인터럽트 처리 로직, 그리고 고도의 최적화가 필요한 고성능 파이프라인(타임 보로잉 기법 적용) 분야에서는 여전히 핵심적인 역할을 수행하고 있습니다.
설계자는 시스템의 타이밍 요구사항, 면적 제약, 전력 예산 등을 종합적으로 고려하여 레벨 트리거 소자와 에지 트리거 소자를 적재적소에 혼합 배치(Mix and Match)하는 아키텍처 역량을 발휘해야 합니다. 결론적으로 레벨 트리거의 특성인 투명성을 위험 요소(글리치 전파)로 볼 것인지, 아니면 기회 요소(타이밍 최적화)로 볼 것인지는 전적으로 엔지니어의 깊이 있는 회로 분석 및 설계 기술력에 달려 있습니다.
