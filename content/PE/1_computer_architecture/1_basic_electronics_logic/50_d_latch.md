# 50. D 래치 (D Latch)

## Ⅰ. 개요

D 래치(Data Latch 또는 Delay Latch)는 단일 데이터 입력선(D)과 제어 신호인 인에이블(Enable, E 또는 Clock, C) 입력선을 가지는 레벨 트리거 방식의 순차 논리 회로 기억 소자입니다. 기존 SR 래치가 지니고 있던 치명적인 단점, 즉 S와 R 입력이 동시에 활성화될 때 발생하는 논리적 모순과 시스템 불안정성(Invalid State)을 구조적으로 원천 차단하기 위해 고안되었습니다. 인에이블 신호가 활성화되어 있는 동안 입력 데이터 D의 상태를 그대로 출력 Q로 투명하게 전달하며, 비활성화되는 순간 마지막 데이터를 잠가서 유지하는 특성을 지닙니다. 이러한 특성 덕분에 투명 래치(Transparent Latch)라고도 불리며, 컴퓨터 구조에서 임시 데이터 버퍼 및 레지스터 파일 설계의 핵심 요소로 활용됩니다.

## Ⅱ. 아키텍처 및 핵심 원리

D 래치의 아키텍처는 기본적으로 SR 래치의 입력단에 게이팅(Gating) 로직과 인버터(NOT 게이트)를 결합하여 설계됩니다. 이 인버터가 핵심적인 역할을 수행하여 S와 R 입력이 항상 서로 반전된 값을 가지도록 강제합니다.

### 1. D 래치의 논리 게이트 결합 아키텍처

```text
[ D 래치 아키텍처 (게이티드 SR 래치 기반) ]

                         +-----------------------------------+
                         |         Basic SR Latch            |
        Data Input (D)   |                                   |
         |               |   +-----\                         |
         |-------+-------+-->| AND  \        +---\           |
         |       |       |   | G1(S) |------>|NOR \          |
         |       |       | +-|      /        | L1  |----+----|----> Output (Q)
         |       V       | | +-----/     +-->|    /     |    |
         |     +---+     | |             |   +---/      |    |
         |     |NOT|     | |             |              |    |
         |     |   |o--+ | | Enable(E)   |              |    |
         |     +---+   | | |             |              |    |
         |             | | |             +--------------|----+
         |             | | |                            |    |
         |             | | |             +--------------+    |
         |             | | |             |                   |
         |             V V V +-----\     |   +---\           |
         |              \ \->| AND  \    +-->|NOR \          |
         +---------------+-->| G2(R) |------>| L2  |----+----|----> Inverted Output (Q')
                             |      /        |    /          |
                             +-----/         +---/           |
                                                             |
                                                             |
Enable Input (E) --------------------------------------------+
```

### 2. 투명성(Transparency) 기반 핵심 동작 원리

*   **동작 금지 구역의 원천 차단**: 데이터 입력 D가 인버터를 거쳐 두 갈래로 나뉩니다. D=1 이면 AND 게이트 G1의 한쪽 입력은 1이 되고, G2의 한쪽 입력은 0이 됩니다. 반대로 D=0 이면 G1 측은 0, G2 측은 1이 됩니다. 따라서 SR 래치 영역으로 들어가는 내부 Set 신호와 내부 Reset 신호는 절대 동시에 1이 될 수 없어 금지된 상태(S=1, R=1)가 발생할 논리적 경로가 완벽히 소멸됩니다.
*   **E = 1 (투명 모드 / Transparent Mode)**: 인에이블 신호가 High 일 때, 두 AND 게이트(G1, G2)는 열린 문과 같이 동작합니다. 입력 D의 값이 그대로 내부 SR 래치로 전달되어 S=D, R=D'의 조건이 형성됩니다. 결과적으로 입력 D의 변화가 지연 시간(Propagation Delay)을 두고 출력 Q에 계속 반영되며 회로가 투명한 유리창처럼 동작합니다.
*   **E = 0 (유지 모드 / Hold Mode)**: 인에이블 신호가 Low로 떨어지면, 두 AND 게이트의 출력이 모두 강제로 0이 됩니다. 이는 내부 SR 래치 입장에서 S=0, R=0 인 유지 조건이 되므로, 외부 입력 D의 변화와 무관하게 E가 떨어지기 직전 마지막으로 저장된 Q의 상태를 견고하게 래치(잠금)합니다.

## Ⅲ. 융합 비교 및 다각도 분석

### 1. D 래치와 D 플립플롭(Flip-Flop)의 정밀 분석
D 래치와 D 플립플롭은 모두 단일 데이터 입력을 받는다는 공통점이 있으나 동기화의 철학이 다릅니다.
*   **면적 및 자원 효율성**: D 래치는 통상 4~5개의 게이트로 구현되는 반면, 에지 트리거 방식의 D 플립플롭은 D 래치 두 개(Master-Slave 구조)를 직렬로 연결하여 구성하므로 회로 면적이 두 배 이상 넓고 전력 소모도 큽니다.
*   **타이밍 민감도**: D 플립플롭은 클럭 에지 순간에만 노이즈에 노출되므로 시스템 전체의 동기적 안정성이 높습니다. 반면 D 래치는 Enable이 1인 전체 시간 동안 노이즈(글리치)가 출력으로 전파될 수 있는 위험성을 안고 설계됩니다.

### 2. 레이스 컨디션의 관점
D 래치 여러 개를 시프트 레지스터 구조로 직렬연결하고 Enable 신호를 동시에 가하면, Enable 신호가 High인 동안 앞 래치의 출력이 뒤 래치의 입력으로 순식간에 관통해버리는 치명적인 레이스 컨디션 현상이 발생합니다. 따라서 D 래치는 데이터를 저장하고 멈추게 하는 기능에는 탁월하나, 클럭에 맞춰 데이터를 차례로 이동시키는 동기식 파이프라인 구조에는 단독으로 사용하기 부적합합니다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 1. I/O 포트 버퍼 및 어드레스 디멀티플렉싱 응용
초기 마이크로프로세서(예: 인텔 8085) 시스템 설계에서 주소 버스(Address Bus)와 데이터 버스(Data Bus)는 핀 수를 줄이기 위해 물리적 핀을 시분할하여 공유(Multiplexing)했습니다. 이때 특정 사이클에 출력되는 주소 정보를 지속적으로 붙잡아두기 위해 외부에 74HC373과 같은 8비트 D 래치 IC를 병렬 버퍼로 광범위하게 사용하였습니다. 프로세서가 ALE(Address Latch Enable) 신호를 펄스로 주면, D 래치가 그 순간의 주소 정보를 낚아채어 다음 사이클 동안 안정적으로 메모리에 공급하는 원리입니다.

### 2. 기술사적 판단: LSSD (Level Sensitive Scan Design) 테스트 용이성 설계
VLSI 칩의 테스트 용이성을 위한 구조적 설계 기법인 LSSD에서 D 래치가 핵심적인 역할을 합니다. 시스템 동작 중에는 에지 트리거 플립플롭처럼 동작하도록 마스터-슬레이브 구조를 취하되, 테스트 모드에서는 여러 클럭 신호를 제어하여 내부의 D 래치들을 거대한 스캔 체인(Scan Chain)으로 엮어 칩 내부 깊숙한 곳의 논리 상태를 외부로 읽어내거나 핀을 통해 강제로 상태를 주입(Shift-in)하는 혁신적인 기법에 응용됩니다. 이는 칩의 결함 커버리지(Fault Coverage)를 극대화하는 고도화된 아키텍처 기술입니다.

## Ⅴ. 기대효과 및 결론

D 래치는 SR 래치가 가진 입력 조합의 불완전성을 하나의 인버터와 게이팅 로직의 조화로 완벽하게 극복해낸 진일보한 디지털 설계의 산물입니다. 사용자는 더 이상 입력 신호의 충돌을 걱정할 필요 없이, 단지 '어떤 데이터(D)'를 '언제(E)' 저장할 것인지에만 집중할 수 있게 되었습니다.
현대의 초고밀도 시스템 온 칩(SoC) 설계에 있어, 무거운 D 플립플롭 대신 면적이 작고 타임 보로잉(Time Borrowing) 기법 적용이 가능한 D 래치 기반의 커스텀 데이터 패스를 설계하는 것은 성능 극대화를 위한 하이엔드 설계 기법으로 평가받습니다. 결론적으로 D 래치는 디지털 메모리와 레지스터의 아키텍처를 이해하는 척추와도 같으며, 면적-전력-성능(PPA: Power, Performance, Area)의 최적화가 필수적인 차세대 반도체 설계에서도 지속적으로 그 가치를 증명할 핵심적인 논리 소자입니다.
