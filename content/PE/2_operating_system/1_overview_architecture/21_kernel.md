# 커널 (Kernel)

## 정의

**커널(Kernel)**은 운영체제의 **핵심 구성 요소**로, 하드웨어와 소프트웨어 사이의 중계 역할을 한다. 프로세스 관리, 메모리 관리, 파일 시스템, 장치 드라이버, 시스템 호출 등 시스템의 근본적인 서비스를 제공한다.

```
┌─────────────────────────────────────────────────────────────────┐
│           운영체제 계층 구조에서 커널 위치                          │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │              사용자 응용 프로그램               │    │ │
│   │   │         (User Applications)                    │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                         │                              │ │
│   │                    시스템 콜                           │ │
│   │                         │                              │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │              커널 (Kernel)                     │    │ │
│   │   │   ┌───────────────────────────────────────┐   │    │ │
│   │   │   │ • 프로세스 관리                        │   │    │ │
│   │   │   │ • 메모리 관리                          │   │    │ │
│   │   │   │ • 파일 시스템                          │   │    │ │
│   │   │   │ • 장치 드라이버                        │   │    │ │
│   │   │   │ • 네트워크 스택                        │   │    │ │
│   │   │   │ • IPC (프로세스 간 통신)               │   │    │ │
│   │   │   └───────────────────────────────────────┘   │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                         │                              │ │
│   │                    하드웨어 추상화                      │ │
│   │                         │                              │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │              하드웨어 (Hardware)               │    │ │
│   │   │   CPU | Memory | Storage | Network | I/O      │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2. 커널 유형

```
┌─────────────────────────────────────────────────────────────────┐
│           커널 아키텍처 유형                                       │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   ① 모놀리식 커널 (Monolithic Kernel):                    │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │                                                 │    │ │
│   │   │   ┌───────────────────────────────────────┐   │    │ │
│   │   │   │           Kernel Space                 │   │    │ │
│   │   │   │  ┌─────────────────────────────────┐  │   │    │ │
│   │   │   │  │ 스케줄러 │ 메모리 │ FS │ 드라이버│  │   │    │ │
│   │   │   │  │    IPC   │  네트워크 │  ...     │  │   │    │ │
│   │   │   │  └─────────────────────────────────┘  │   │    │ │
│   │   │   └───────────────────────────────────────┘   │    │ │
│   │   │                   │                            │    │ │
│   │   │              시스템 콜                          │    │ │
│   │   │                   │                            │    │ │
│   │   │   ┌───────────────────────────────────────┐   │    │ │
│   │   │   │           User Space                   │   │    │ │
│   │   │   └───────────────────────────────────────┘   │    │ │
│   │   │                                                 │    │ │
│   │   │   특징:                                         │    │ │
│   │   │   • 모든 기능이 커널 공간에 통합                 │    │ │
│   │   │   • 높은 성능 (시스템 콜 오버헤드 적음)           │    │ │
│   │   │   • 확장/유지보수 어려움                         │    │ │
│   │   │   • 버그가 전체 시스템에 영향                     │    │ │
│   │   │   • 예: Linux, BSD, Solaris                    │    │ │
│   │   │                                                 │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   ② 마이크로커널 (Microkernel):                          │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │                                                 │    │ │
│   │   │   ┌───────────────────────────────────────┐   │    │ │
│   │   │   │           User Space                   │   │    │ │
│   │   │   │  ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐     │   │    │ │
│   │   │   │  │ FS  │ │드라이버│ │네트워크│ │ ... │     │   │    │ │
│   │   │   │  │서버 │ │ 서버 │ │ 서버 │ │     │     │   │    │ │
│   │   │   │  └─────┘ └─────┘ └─────┘ └─────┘     │   │    │ │
│   │   │   └───────────────────────────────────────┘   │    │ │
│   │   │                   │ IPC                        │    │ │
│   │   │   ┌───────────────────────────────────────┐   │    │ │
│   │   │   │         Microkernel (Kernel Space)    │   │    │ │
│   │   │   │  ┌─────────────────────────────────┐  │   │    │ │
│   │   │   │  │ IPC │ 스케줄링 │ 스레드 │ MMU  │  │   │    │ │
│   │   │   │  └─────────────────────────────────┘  │   │    │ │
│   │   │   └───────────────────────────────────────┘   │    │ │
│   │   │                                                 │    │ │
│   │   │   특징:                                         │    │ │
│   │   │   • 최소 기능만 커널에 유지                      │    │ │
│   │   │   • 서버 모듈이 사용자 공간에서 실행              │    │ │
│   │   │   • 높은 안정성/보안성                           │    │ │
│   │   │   • IPC 오버헤드로 성능 저하 가능                 │    │ │
│   │   │   • 예: QNX, Minix, L4, Zircon (Fuchsia)       │    │ │
│   │   │                                                 │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   │   ③ 하이브리드 커널 (Hybrid Kernel):                      │ │
│   │   ┌───────────────────────────────────────────────┐    │ │
│   │   │                                                 │    │ │
│   │   │   • 모놀리식 + 마이크로커널 결합                  │    │ │
│   │   │   • 성능과 모듈화 균형                           │    │ │
│   │   │   • 일부 서비스는 커널 공간                      │    │ │
│   │   │   • 예: Windows NT, macOS (XNU)                │    │ │
│   │   │                                                 │    │ │
│   │   └───────────────────────────────────────────────┘    │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   비교:                                                         │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │  구분        │ 모놀리식  │ 마이크로커널 │ 하이브리드    │ │
│   │  ────────────┼──────────┼────────────┼───────────────  │ │
│   │  성능        │ 높음      │ 낮음       │ 중간           │ │
│   │  안정성      │ 낮음      │ 높음       │ 중간           │ │
│   │  크기        │ 큼        │ 작음       │ 중간           │ │
│   │  확장성      │ 낮음      │ 높음       │ 중간           │ │
│   │  복잡도      │ 높음      │ 낮음       │ 중간           │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 4. 시스템 콜 (System Call)

┌─────────────────────────────────────────────────────────────────┐

## 기술사 시험 포인트

### 1. 핵심 정의

> **커널**: OS 핵심, 하드웨어-소프트웨어 중계
> **모놀리식 vs 마이크로커널**: 성능 vs 안정성/모듈화
> **커널 모드 vs 사용자 모드**: Ring 0 vs Ring 3

### 2. 빈출 키워드

- **Process/Memory/FS Management**: 커널 핵심 기능
- **Monolithic/Microkernel**: 커널 아키텍처 유형
- **Kernel Mode (Ring 0)**: 완전한 하드웨어 접근
- **System Call**: 사용자→커널 모드 전환

### 3. 예상 문제

**Q3. 커널 모드와 사용자 모드를 구분하는 이유와 시스템 콜 처리 과정을 설명하시오.**
