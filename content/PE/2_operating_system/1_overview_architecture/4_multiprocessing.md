# 다중 처리 시스템 (Multiprocessing System)

## 정의

**다중 처리 시스템(Multiprocessing System)**은 두 개 이상의 CPU(중앙처리장치)를 단일 시스템에 탑재하여, 여러 프로세스를 진정한 의미의 병렬(Parallel)로 동시에 실행할 수 있는 컴퓨터 시스템 아키텍처이다. 시분할 시스템이 단일 CPU를 시간적으로 분할하는 것과 달리, 다중 처리 시스템은 물리적으로 다수의 CPU가 독립적으로 작업을 수행하여 시스템 전체의 처리 능력을 선형적으로 향상시킨다.

```
+------------------------------------------------------------------+
|                다중 처리 시스템 개념도                              |
+------------------------------------------------------------------+

    단일 프로세서 시스템 (시분할)
    |----|----|----|----|----|----|----|----|
    [ P1 ][ P2 ][ P3 ][ P1 ][ P2 ][ P3 ][ P1 ]  <- 시간 분할

    다중 처리 시스템 (병렬 처리)
    CPU0: |----|----|----|----|----|
          [ P1 ][ P1 ][ P2 ][ P3 ][ P1 ]

    CPU1: |----|----|----|----|----|
          [ P2 ][ P3 ][ P3 ][ P1 ][ P2 ]        <- 진정한 병렬

    CPU2: |----|----|----|----|----|
          [ P3 ][ P2 ][ P1 ][ P2 ][ P3 ]

    CPU3: |----|----|----|----|----|
          [ P4 ][ P4 ][ P4 ][ P4 ][ P4 ]


    +--------------------------------------------------+
    |                 공유 메모리 시스템                 |
    |  +--------+  +--------+  +--------+  +--------+  |
    |  |  CPU0  |  |  CPU1  |  |  CPU2  |  |  CPU3  |  |
    |  +---+----+  +---+----+  +---+----+  +---+----+  |
    |      |           |           |           |       |
    |      +-----------+-----------+-----------+       |
    |                        |                        |
    |                 +------v------+                 |
    |                 |  시스템 버스  |                 |
    |                 +------+------+                 |
    |                        |                        |
    |                 +------v------+                 |
    |                 |  공유 메모리  |                 |
    |                 +-------------+                 |
    +--------------------------------------------------+
```

## 핵심 개념

### 1. 다중 처리 시스템의 분류

```
+================================================================+
|             다중 처리 시스템 분류 체계                             |
+================================================================+

                    다중 처리 시스템
                          |
           +--------------+--------------+
           |                             |
    단일 명령어 스트림           다중 명령어 스트림
    (SISD, SIMD)               (MISD, MIMD)
           |                             |
           |                    +--------+--------+
           |                    |                 |
    데이터 병렬성           공유 메모리        분산 메모리
    (Vector, GPU)         (SMP, NUMA)       (MPP, Cluster)
                                |                 |
                         +------+------+   +------+------+
                         |             |   |             |
                       균등 메모리    불균등     클러스터   MPP
                       (UMP, SMP)    메모리               대규모
                                     (NUMA)    병렬처리
```

#### Flynn 분류법 (Flynn's Taxonomy)

```
+-------------------+-------------------------------------------+
|     분류          |                 설명                       |
+-------------------+-------------------------------------------+
| SISD              | 단일 명령어, 단일 데이터                    |
| (Single           | 전통적인 단일 프로세서                       |
|  Instruction,     | 순차적 실행                                 |
|  Single Data)     |                                            |
+-------------------+-------------------------------------------+
| SIMD              | 단일 명령어, 다중 데이터                    |
| (Single           | 벡터 프로세서, GPU                          |
|  Instruction,     | 데이터 병렬성 활용                          |
|  Multiple Data)   | 예: 이미지 처리, 머신러닝                   |
+-------------------+-------------------------------------------+
| MISD              | 다중 명령어, 단일 데이터                    |
| (Multiple         | 이론적 분류, 실제 구현 드묾                  |
|  Instruction,     | 예: 우주선 비행 제어 시스템                  |
|  Single Data)     |     (동일 데이터에 다중 알고리즘 적용)       |
+-------------------+-------------------------------------------+
| MIMD              | 다중 명령어, 다중 데이터                    |
| (Multiple         | 다중 처리 시스템의 일반적 형태               |
|  Instruction,     | 각 CPU가 독립적인 프로그램 실행              |
|  Multiple Data)   | SMP, NUMA, Cluster 모두 포함                |
+-------------------+-------------------------------------------+
```

### 2. SMP (Symmetric Multiprocessing) - 대칭형 다중 처리

**정의**: 모든 CPU가 동등한 지위를 가지며, 단일 공유 메모리와 단일 운영체제를 공유하는 다중 처리 아키텍처이다. 모든 프로세서가 메모리와 I/O 장치에 동등하게 접근할 수 있다.

```
+================================================================+
|                   SMP (Symmetric Multiprocessing)                |
+================================================================+

                    +----------------------------------+
                    |          운영체제 (단일)           |
                    |    (Single OS Instance)          |
                    +----------------------------------+
                                    |
                    +---------------v------------------+
                    |           시스템 버스              |
                    |        (System Bus)              |
                    +---------------+------------------+
                                    |
        +-----------+-----------+-----------+-----------+
        |           |           |           |           |
    +---v---+   +---v---+   +---v---+   +---v---+   +---v---+
    | CPU 0 |   | CPU 1 |   | CPU 2 |   | CPU 3 |   |  ...  |
    +-------+   +-------+   +-------+   +-------+   +-------+
        |           |           |           |           |
        +-----------+-----------+-----------+-----------+
                                    |
                    +---------------v------------------+
                    |         공유 메모리                |
                    |      (Shared Memory)             |
                    |  +----------------------------+  |
                    |  |  Uniform Memory Access     |  |
                    |  |  모든 CPU에서 동일 접근 시간  |  |
                    +----------------------------------+

    SMP 특성:
    1. 단일 운영체제 인스턴스
    2. 모든 CPU 동등한 지위 (Master-Slave 없음)
    3. 균등 메모리 접근 (UMA)
    4. 공유 I/O 장치
    5. 캐시 일관성 (Cache Coherence) 필수
```

#### SMP의 메모리 접근 특성

```
    균등 메모리 접근 (Uniform Memory Access, UMA):

    메모리
    +----------------------------------------+
    |              Main Memory               |
    | Address: 0x0000_0000 - 0xFFFF_FFFF     |
    +--------------------+-------------------+
                         |
         +---------------+---------------+
         |               |               |
     +---v---+       +---v---+       +---v---+
     | CPU 0 |       | CPU 1 |       | CPU 2 |
     |       |       |       |       |       |
     |접근시간|       |접근시간|       |접근시간|
     | = T   |       | = T   |       | = T   |
     +-------+       +-------+       +-------+

    모든 CPU에서 메모리 접근 시간이 동일 (T)

    장점:
    - 프로그래밍 모델 단순
    - 메모리 할당 용이
    - 부하 분산 자동

    단점:
    - 시스템 버스 병목 (Bus Contention)
    - CPU 수 증가 시 성능 향상 둔화
    - 일반적으로 8-16개 CPU가 실용적 한계
```

### 3. NUMA (Non-Uniform Memory Access) - 불균등 메모리 접근

**정의**: 각 CPU가 자신의 로컬 메모리를 가지며, 다른 CPU의 메모리에 접근할 때 추가 지연 시간이 발생하는 아키텍처이다. 대규모 다중 처리 시스템에서 버스 병목을 해결하기 위해 설계되었다.

```
+================================================================+
|                NUMA (Non-Uniform Memory Access)                  |
+================================================================+

    +-----------------------+       +-----------------------+
    |       NUMA Node 0     |       |       NUMA Node 1     |
    |  +-----------------+  |       |  +-----------------+  |
    |  |     CPU 0       |  |       |  |     CPU 2       |  |
    |  +--------+--------+  |       |  +--------+--------+  |
    |           |           |       |           |           |
    |  +--------v--------+  |       |  +--------v--------+  |
    |  |   Local Memory  |  |       |  |   Local Memory  |  |
    |  |     (Node 0)    |  |       |  |     (Node 1)    |  |
    |  |    0-64 GB      |  |       |  |    64-128 GB    |  |
    |  +--------+--------+  |       |  +--------+--------+  |
    |           |           |       |           |           |
    +-----------+-----------+       +-----------+-----------+
                |                               |
                |      NUMA Interconnect        |
                |    (QPI, HyperTransport)      |
                +-------------------------------+
                |                               |
    +-----------+-----------+       +-----------+-----------+
    |           |           |       |           |           |
    |  +--------v--------+  |       |  +--------v--------+  |
    |  |   Local Memory  |  |       |  |   Local Memory  |  |
    |  |     (Node 2)    |  |       |  |     (Node 3)    |  |
    |  |   128-192 GB    |  |       |  |   192-256 GB    |  |
    |  +--------+--------+  |       |  +--------+--------+  |
    |           |           |       |           |           |
    |  +--------v--------+  |       |  +--------v--------+  |
    |  |     CPU 4       |  |       |  |     CPU 6       |  |
    |  +-----------------+  |       |  +-----------------+  |
    |       NUMA Node 2     |       |       NUMA Node 3     |
    +-----------------------+       +-----------------------+
```

#### NUMA 메모리 접근 시간 차이

```
    메모리 접근 시간 매트릭스 (예시):

                Memory Node
              0      1      2      3
           +------+------+------+------+
         0 | 100ns| 150ns| 200ns| 250ns|
    CPU     +------+------+------+------+
         1 | 150ns| 100ns| 200ns| 250ns|
    Node    +------+------+------+------+
         2 | 200ns| 200ns| 100ns| 150ns|
           +------+------+------+------+
         3 | 250ns| 250ns| 150ns| 100ns|
           +------+------+------+------+

    - Local Access: 100ns (동일 노드)
    - Remote Access: 150-250ns (다른 노드)
    - 최악의 경우: 2.5배 느린 접근

    NUMA 최적화 전략:
    1. 메모리 지역성 (Memory Locality)
       - 데이터를 접근하는 CPU 노드에 메모리 할당

    2. NUMA 인식 스케줄링 (NUMA-Aware Scheduling)
       - 프로세스를 데이터가 있는 노드에서 실행

    3. 페이지 마이그레이션 (Page Migration)
       - 자주 접근하는 페이지를 로컬 노드로 이동
```

### 4. 캐시 일관성 (Cache Coherence)

다중 처리 시스템에서 각 CPU의 캐시에 저장된 데이터의 일관성을 유지하는 메커니즘이다.

```
+================================================================+
|                   캐시 일관성 문제                                 |
+================================================================+

    [문제 상황: 캐시 불일치]

    초기 상태:
    Memory: X = 10
    CPU0 Cache: X = 10
    CPU1 Cache: X = 10

    1. CPU0이 X를 20으로 변경:
       CPU0 Cache: X = 20
       CPU1 Cache: X = 10  <- 불일치!
       Memory: X = 10      <- 불일치!

    2. CPU1이 X를 읽음:
       CPU1은 여전히 10을 읽음 (잘못된 값)


    [해결: 캐시 일관성 프로토콜]

    MESI 프로토콜 상태:
    +--------+------------------------------------------+
    |  상태   |                 설명                      |
    +--------+------------------------------------------+
    | Modified| 이 캐시라인만 수정됨, 메모리와 다름        |
    |        | 다른 캐시에는 없음                        |
    +--------+------------------------------------------+
    | Exclusive| 이 캐시라인만 가짐, 메모리와 동일         |
    |        | 수정 가능                                 |
    +--------+------------------------------------------+
    | Shared | 여러 캐시가 공유, 메모리와 동일            |
    |        | 수정 불가 (먼저 배타적 획득 필요)          |
    +--------+------------------------------------------+
    | Invalid| 캐시라인 유효하지 않음                     |
    +--------+------------------------------------------+
```

#### MESI 프로토콜 동작

```
    MESI 상태 전이도:

                    Read Hit
              +-------------------+
              |                   |
              v    Read Miss     |
         +---------+ ----------> +---------+
         |Invalid  |             | Shared  |
         +---------+ <---------- +---------+
              ^    Write Upgrade    |
              |        (Other)      |
              |                     |
              |    Read Miss        |
              |   (Other Core)      | Read Hit
              |                     |
         +---------+ ----------> +----------+
         |Modified |   Write     |Exclusive |
         +---------+   Hit       +----------+
              ^                     |
              |    Write Miss       |
              |   (Other Core)      | Write Hit
              |                     |
              +---------------------+

    상태 전이 예시:

    1. CPU0이 X 읽기 (Read Miss)
       Memory -> CPU0 Cache: X (Exclusive)

    2. CPU1이 X 읽기 (Read Miss)
       CPU0: Exclusive -> Shared
       Memory -> CPU1 Cache: X (Shared)

    3. CPU0이 X 쓰기 (Write Hit)
       CPU0: Shared -> Modified
       CPU1: Shared -> Invalid (Invalidation)

    4. CPU1이 X 읽기 (Read Miss)
       CPU0: Modified -> Shared (Write-back to Memory)
       CPU1: Invalid -> Shared
```

### 5. 다중 처리 vs 시분할 vs 다중 프로그래밍

```
+================================================================+
|          다중 처리 vs 시분할 vs 다중 프로그래밍 비교                |
+================================================================+

+-------------------+-------------------------------------------+
|     구분          |   다중 프로그래밍   |   시분할   | 다중 처리 |
+-------------------+-------------------------------------------+
| CPU 수            | 1개                | 1개       | 2개 이상   |
+-------------------+-------------------------------------------+
| 실행 방식         | 순차적             | 시분할    | 병렬      |
+-------------------+-------------------------------------------+
| 핵심 목표         | CPU 활용도         | 응답시간  | 처리량    |
+-------------------+-------------------------------------------+
| 병렬성            | 없음               | 없음      | 있음      |
| (Parallelism)     | (시간적 분할만)    |(시간적 분할)| (공간적) |
+-------------------+-------------------------------------------+
| 동시성            | 있음               | 있음      | 있음      |
| (Concurrency)     |                    |           |           |
+-------------------+-------------------------------------------+
| 하드웨어 요구     | 단일 CPU           | 단일 CPU  | 다중 CPU  |
+-------------------+-------------------------------------------+
| 운영체제 복잡도   | 낮음               | 중간      | 높음      |
+-------------------+-------------------------------------------+
| 캐시 일관성       | 해당 없음          | 해당 없음 | 필수      |
+-------------------+-------------------------------------------+
| 적용 분야         | 일괄 처리          | 대화형    | 고성능    |
|                   |                    | 시스템    | 컴퓨팅    |
+-------------------+-------------------------------------------+

    병렬성(Parallelism) vs 동시성(Concurrency):

    병렬성 (Parallelism):
    - 물리적으로 동시에 여러 작업 수행
    - 다중 CPU 필요
    - 예: 4개 CPU가 4개 스레드 동시 실행

    동시성 (Concurrency):
    - 논리적으로 동시에 여러 작업 처리
    - 단일 CPU로도 가능 (시분할)
    - 예: 1개 CPU가 4개 스레드 시분할 실행
```

## 상세 설명

### 1. 프로세서 간 통신 (Inter-Processor Communication)

```
+================================================================+
|               다중 처리 시스템의 프로세서 간 통신                   |
+================================================================+

    1. 공유 메모리 통신 (Shared Memory Communication)

    +------------------+
    |    CPU 0         |
    |  +------------+  |
    |  | Cache      |  |
    |  +------+-----+  |
    +---------|--------+
              |
    +---------v--------+      +------------------+
    |   Shared Memory  | <--> |    CPU 1         |
    |  +------------+  |      |  +------------+  |
    |  |  Buffer    |  |      |  | Cache      |  |
    |  +------------+  |      |  +------------+  |
    +------------------+      +------------------+

    장점: 빠른 통신, 간단한 프로그래밍
    단점: 동기화 필요, 캐시 일관성 오버헤드


    2. 메시지 전달 통신 (Message Passing)

    +------------------+          +------------------+
    |    CPU 0         |          |    CPU 1         |
    |  +------------+  |          |  +------------+  |
    |  | Send Msg   |--+----------+->| Recv Msg   |  |
    |  +------------+  |  Message |  +------------+  |
    +------------------+   Queue  +------------------+

    장점: 분산 메모리 시스템에 적합, 캐시 일관성 불필요
    단점: 통신 오버헤드, 복사 비용


    3. 인터프로세서 인터럽트 (Inter-Processor Interrupt, IPI)

    CPU 0                              CPU 1
       |                                 |
       | --- IPI (인터럽트 신호) -------->|
       |                                 |
       |                           +-----v-----+
       |                           | IPI 핸들러 |
       |                           | 실행       |
       |                           +-----------+

    용도:
    - 캐시 무효화 (TLB Shootdown)
    - 스케줄링 요청 (Rescheduling)
    - 프로세서 간 동기화
```

### 2. 다중 처리 시스템의 스케줄링

```
+================================================================+
|              다중 처리 시스템 스케줄링 기법                         |
+================================================================+

    1. 다중 큐 스케줄링 (Multiple Queue Scheduling)

    CPU 0 전용 큐        CPU 1 전용 큐        CPU 2 전용 큐
    +-------------+     +-------------+     +-------------+
    | P1, P2, P3  |     | P4, P5, P6  |     | P7, P8, P9  |
    +------+------+     +------+------+     +------+------+
           |                  |                  |
           v                  v                  v
        +-----+            +-----+            +-----+
        |CPU 0|            |CPU 1|            |CPU 2|
        +-----+            +-----+            +-----+

    장점: 캐시 지역성 향상, 문맥 교환 감소
    단점: 부하 불균형 가능성


    2. 공유 큐 스케줄링 (Shared Queue Scheduling)

                          공유 준비 큐
                    +-------------------------+
                    | P1 | P2 | P3 | P4 | P5 |
                    +-------------------------+
                              |
            +-----------------+-----------------+
            |                 |                 |
            v                 v                 v
         +-----+           +-----+           +-----+
         |CPU 0|           |CPU 1|           |CPU 2|
         +-----+           +-----+           +-----+

    장점: 자동 부하 분산
    단점: 큐 접근 경합, 캐시 지역성 저하


    3. 스틸링 스케줄링 (Work Stealing)

    CPU 0 큐          CPU 1 큐 (비어있음)   CPU 2 큐
    +-----------+     +-----------+     +-----------+
    | P1 P2 P3  |     |   (empty) |     | P7 P8     |
    +-----------+     +-----------+     +-----------+
           ^                 |                 |
           |                 |                 |
           +----- steal -----+                 |
                             |                 |
                             v                 |
                          +-----+              |
                          |CPU 1| <--- steal --+
                          +-----+

    특징:
    - 각 CPU가 로컬 큐 유지
    - 유휴 CPU가 다른 CPU의 큐에서 작업 훔침
    - Linux CFS, Go 스케줄러 등에서 사용
```

#### Linux 다중 처리 스케줄링

```
    Linux CFS (Completely Fair Scheduler) 다중 처리:

    +------------------------------------------------------------------+
    |                     Linux 다중 처리 스케줄링                        |
    +------------------------------------------------------------------+

                Run Queue (CPU 0)           Run Queue (CPU 1)
    +-----------------------------+    +-----------------------------+
    |      Red-Black Tree         |    |      Red-Black Tree         |
    |                             |    |                             |
    |        Task A (100)         |    |        Task C (200)         |
    |        Task B (150)         |    |        Task D (250)         |
    |                             |    |                             |
    +-----------------------------+    +-----------------------------+

    로드 밸런싱 (Load Balancing):
    - 주기적으로 각 CPU의 부하 확인
    - 불균형 감지 시 작업 마이그레이션
    - NUMA 시스템에서는 NUMA 노드 간 마이그레이션 최소화

    스케줄링 도메인 (Scheduling Domains):
    +---------------------------+
    |       SMT Domain          |  <- 하이퍼스레딩 코어
    |  (Logical CPUs in Core)   |
    +---------------------------+
               |
    +---------------------------+
    |        MC Domain          |  <- 멀티코어
    |   (Cores in Socket)       |
    +---------------------------+
               |
    +---------------------------+
    |       NUMA Domain         |  <- NUMA 노드
    |   (Sockets in Node)       |
    +---------------------------+
```

### 3. 다중 처리 시스템의 동기화

#### 스핀락 (Spinlock)

```
+================================================================+
|                      스핀락 (Spinlock)                           |
+================================================================+

    정의: 락을 획득할 때까지 busy-waiting(회전)하는 동기화 기법

    CPU 0                              CPU 1
       |                                 |
       | -- lock() ----------------------|
       |    (락 획득 성공)                |
       |                                 |
       |    [임계 영역 실행]              | -- lock() --
       |                                 | (락 획득 대기)
       |                                 |    spinning...
       |                                 |    spinning...
       | -- unlock() --------------------|
       |                                 | (락 획득 성공)
       |                                 |
       |                                 | [임계 영역 실행]

    장점:
    - 짧은 대기 시간에 효율적
    - 문맥 교환 오버헤드 없음
    - 다중 처리 시스템에 적합

    단점:
    - 대기 중 CPU 낭비
    - 긴 대기 시간에 비효율적
    - 우선순위 역전 가능성
```

#### 뮤텍스와 세마포어

```
    스핀락 vs 뮤텍스 비교:

    +-------------------+-------------------------------------------+
    |     구분          |    스핀락 (Spinlock)  |    뮤텍스 (Mutex)  |
    +-------------------+-------------------------------------------+
    | 대기 방식         | Busy-waiting          | Block/Sleep       |
    +-------------------+-------------------------------------------+
    | 문맥 교환         | 없음                  | 있음              |
    +-------------------+-------------------------------------------+
    | 적합한 상황       | 짧은 임계 영역        | 긴 임계 영역      |
    +-------------------+-------------------------------------------+
    | CPU 사용          | 대기 중 CPU 사용      | 대기 중 CPU 양보  |
    +-------------------+-------------------------------------------+
    | 인터럽트 컨텍스트 | 사용 가능             | 사용 불가         |
    +-------------------+-------------------------------------------+
    | 단일 CPU          | 비효율적              | 효율적            |
    +-------------------+-------------------------------------------+

    사용 예시:
    - 스핀락: 인터럽트 핸들러, 짧은 임계 영역
    - 뮤텍스: 파일 입출력, 긴 연산, 사용자 요청 처리
```

### 4. 다중 처리 시스템 유형별 비교

```
+================================================================+
|              SMP vs NUMA vs MPP vs Cluster 비교                   |
+================================================================+

+-------------------+-------------------------------------------+
|     구분          |  SMP      |  NUMA    |  MPP    | Cluster  |
+-------------------+-------------------------------------------+
| 메모리 구조       | 공유      | 분산공유 | 분산    | 분산      |
+-------------------+-------------------------------------------+
| 메모리 접근       | 균등      | 불균등   | 로컬만  | 로컬만    |
+-------------------+-------------------------------------------+
| 운영체제          | 단일      | 단일     | 단일    | 다중      |
+-------------------+-------------------------------------------+
| CPU 수            | 2-16      | 16-256   | 1000+   | 무제한    |
+-------------------+-------------------------------------------+
| 통신 방식         | 버스      | 인터커넥트|인터커넥트|네트워크  |
+-------------------+-------------------------------------------+
| 확장성            | 낮음      | 중간     | 높음    | 매우 높음 |
+-------------------+-------------------------------------------+
| 비용              | 저가      | 중가     | 고가    | 가변      |
+-------------------+-------------------------------------------+
| 적용 분야         | 일반      | DB서버   | 슈퍼    | 웹서버    |
|                   | 서버      | ERP      | 컴퓨터  | HPC       |
+-------------------+-------------------------------------------+

    MPP (Massively Parallel Processing):
    - 수천 개의 프로세서
    - 각 프로세서가 자체 메모리 보유
    - 메시지 전달 방식 통신
    - 예: IBM Blue Gene, Cray 시스템

    Cluster:
    - 독립적인 노드들의 연결
    - 각 노드가 자체 OS 실행
    - 네트워크 기반 통신
    - 예: Google 데이터센터, AWS EC2 클러스터
```

## 실무 적용

### 1. 서버 다중 처리 최적화

```
    Linux 다중 처리 시스템 튜닝:

    1. CPU 선호도 설정
    # taskset -c 0,1,2,3 <command>  # 특정 CPU에서만 실행
    # taskset -pc 0,1 <pid>         # 실행 중인 프로세스 변경

    2. IRQ(인터럽트) 분배
    # apt-get install irqbalance
    # systemctl start irqbalance

    3. NUMA 정책 설정
    # numactl --cpunodebind=0 --membind=0 <command>
    # numactl --interleave=all <command>  # 메모리 분산

    4. RCU(Read-Copy-Update) 최적화
    # sysctl -w kernel.rcu_nocbs="0-3"  # 특정 CPU에서 RCU 콜백 실행

    5. 스케줄러 튜닝
    # sysctl -w kernel.sched_load_balance=1
```

## 기술사 시험 포인트

### 1. 핵심 암기 사항

```
+------------------------------------------------------------------+
|              다중 처리 시스템 핵심 정리                              |
+------------------------------------------------------------------+

1. 정의: 두 개 이상의 CPU를 단일 시스템에 탑재하여 병렬 처리

2. Flynn 분류:
   - SISD: 단일 명령어, 단일 데이터 (단일 프로세서)
   - SIMD: 단일 명령어, 다중 데이터 (벡터, GPU)
   - MIMD: 다중 명령어, 다중 데이터 (다중 처리 시스템)

3. SMP 특징:
   - 단일 OS, 공유 메모리
   - 균등 메모리 접근 (UMA)
   - 모든 CPU 동등한 지위

4. NUMA 특징:
   - 각 CPU가 로컬 메모리 보유
   - 불균등 메모리 접근
   - 로컬 접근 < 원격 접근 (지연 시간)

5. 캐시 일관성:
   - MESI 프로토콜 (Modified, Exclusive, Shared, Invalid)
   - Write-Invalidate vs Write-Update

6. 동기화 기법:
   - 스핀락: busy-waiting, 짧은 임계 영역
   - 뮤텍스: block/sleep, 긴 임계 영역
```

### 2. 빈출 문제 유형

```
1. SMP vs NUMA 비교
   Q: SMP와 NUMA의 차이점과 각각의 적용 분야
   A:
   - SMP: 균등 메모리 접근, 소규모, 단일 버스
   - NUMA: 불균등 메모리 접근, 대규모, 인터커넥트

2. 캐시 일관성 문제
   Q: 다중 처리 시스템에서 캐시 일관성 문제 발생 원인과 해결
   A:
   - 원인: 각 CPU 캐시의 데이터 불일치
   - 해결: MESI 프로토콜, Snooping, Directory 기반

3. 병렬성 vs 동시성
   Q: Parallelism과 Concurrency의 차이
   A:
   - 병렬성: 물리적 동시 실행 (다중 CPU)
   - 동시성: 논리적 동시 처리 (단일 CPU 가능)

4. 스핀락 vs 뮤텍스
   Q: 각 동기화 기법의 특징과 적합한 사용 시나리오
   A:
   - 스핀락: 짧은 대기, busy-waiting, 인터럽트 컨텍스트
   - 뮤텍스: 긴 대기, block/sleep, 프로세스 컨텍스트

5. 다중 처리 시스템 분류
   Q: Flynn 분류법에 따른 다중 처리 시스템 분류
   A: SISD, SIMD, MISD, MIMD 설명
```

### 3. 기술사 답안 작성 요령

```
[문제] 다중 처리 시스템의 개념을 설명하고, SMP와 NUMA의
      차이점을 기술하시오.

[답안 구성]
1. 개요
   - 다중 처리 시스템의 정의 및 등장 배경
   - 단일 프로세서의 한계와 병렬 처리의 필요성

2. 다중 처리 시스템 분류
   가. Flynn 분류법
      - SISD, SIMD, MISD, MIMD
   나. 메모리 구조에 따른 분류
      - 공유 메모리, 분산 메모리

3. SMP (Symmetric Multiprocessing)
   가. 정의 및 구조
   나. 특징 (UMA, 단일 OS, 동등한 CPU)
   다. 장단점

4. NUMA (Non-Uniform Memory Access)
   가. 정의 및 구조
   나. 특징 (로컬 메모리, 불균등 접근)
   다. 장단점

5. SMP vs NUMA 비교 (표 작성)
   - 메모리 접근, 확장성, 적용 분야

6. 캐시 일관성
   - MESI 프로토콜 간략 설명

7. 결론
   - 현대 시스템에서의 적용 사례
```
