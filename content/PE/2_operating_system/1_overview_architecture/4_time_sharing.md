# 시분할 시스템 (Time-Sharing System)

## 정의

**시분할 시스템(Time-Sharing System)**은 CPU 시간을 **작은 단위(타임 슬라이스)로 분할**하여 여러 사용자/프로세스에게 **순차적으로 할당**함으로써, 모든 사용자가 동시에 컴퓨터를 독점하는 것처럼 느끼게 하는 운영 방식이다.

```
┌─────────────────────────────────────────────────────────────────┐
│              시분할 시스템 개념도                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   타임 슬라이스 (Time Slice / Time Quantum): 통상 10~100ms      │
│                                                                 │
│   CPU 시간 분할:                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   시간 ───────────────────────────────────────────────▶ │ │
│   │   │─────│─────│─────│─────│─────│─────│─────│─────│    │ │
│   │   │  A  │  B  │  C  │  A  │  B  │  C  │  A  │  B  │    │ │
│   │   │─────│─────│─────│─────│─────│─────│─────│─────│    │ │
│   │     ▲     ▲     ▲                                         │ │
│   │     │     │     │   각 프로세스에                         │ │
│   │   TS=10ms TS   TS   타임 슬라이스 할당                     │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   사용자 관점:                                                   │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   사용자 A ──▶ [입력] ──▶ [응답] (100ms 내)              │ │
│   │   사용자 B ──▶ [입력] ──▶ [응답] (100ms 내)              │ │
│   │   사용자 C ──▶ [입력] ──▶ [응답] (100ms 내)              │ │
│   │                                                         │ │
│   │   → 모든 사용자가 즉각적인 응답을 경험                    │ │
│   │   → 각자 컴퓨터를 독점하는 것처럼 느낌                    │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 핵심 개념

### 1. 시분할 vs 다중 프로그래밍

```
┌─────────────────────────────────────────────────────────────────┐
│       다중 프로그래밍 vs 시분할 시스템 비교                       │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │        │  다중 프로그래밍  │      시분할 시스템          │ │
│   ├────────┼──────────────────┼────────────────────────────┤ │
│   │ 스위칭  │ I/O 대기 시       │ 타임 슬라이스 만료 시       │ │
│   │ 시점    │                  │                            │ │
│   ├────────┼──────────────────┼────────────────────────────┤ │
│   │ 목표    │ CPU 활용도 극대화 │ 응답 시간 최소화            │ │
│   ├────────┼──────────────────┼────────────────────────────┤ │
│   │ 환경    │ 일괄 처리(Batch) │ 대화형(Interactive)        │ │
│   ├────────┼──────────────────┼────────────────────────────┤ │
│   │ 스케줄링│ 비선점형 가능     │ 선점형 필수                 │ │
│   ├────────┼──────────────────┼────────────────────────────┤ │
│   │ 응답성  │ 낮음             │ 높음                        │ │
│   ├────────┼──────────────────┼────────────────────────────┤ │
│   │ 예시    │ 일괄 처리 시스템  │ Unix, Linux, Windows       │ │
│   └────────┴──────────────────┴────────────────────────────┘ │
│                                                                 │
│   💡 현대 OS는 두 개념을 결합: 다중 프로그래밍 + 시분할          │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2. 라운드 로빈 스케줄링

```
┌─────────────────────────────────────────────────────────────────┐
│              라운드 로빈(Round Robin) 스케줄링                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   시분할 시스템의 대표적 스케줄링 알고리즘                        │
│                                                                 │
│   준비 큐:                                                      │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │    ┌───┐   ┌───┐   ┌───┐   ┌───┐   ┌───┐              │ │
│   │    │ P1│──▶│ P2│──▶│ P3│──▶│ P4│──▶│ P5│──▶ ...       │ │
│   │    └───┘   └───┘   └───┘   └───┘   └───┘              │ │
│   │      ▲                                    │             │ │
│   │      └────────────────────────────────────┘             │ │
│   │                    순환 큐 (Circular Queue)             │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   동작 과정:                                                    │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │ ① P1이 타임 슬라이스 할당받아 실행                       │ │
│   │ ② 타임 슬라이스 만료 → P1을 큐 뒤로 이동                 │ │
│   │ ③ P2에게 CPU 할당                                        │ │
│   │ ④ 반복...                                               │ │
│   │                                                         │ │
│   │ 타임 슬라이스 내 완료 시: 즉시 다음 프로세스로 전환       │ │
│   │ I/O 요청 시: 대기 큐로 이동, 다음 프로세스 실행          │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   타임 슬라이스 크기 결정:                                       │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │ • 너무 큼: 응답 시간 증가 (일괄 처리처럼 동작)            │ │
│   │ • 너무 작음: 문맥 교환 오버헤드 증가                      │ │
│   │ • 적절값: 10~100ms (현대 시스템)                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3. 응답 시간과 공정성

```
┌─────────────────────────────────────────────────────────────────┐
│           시분할 시스템의 성능 지표                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ① 응답 시간 (Response Time)                                   │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │   정의: 요청 제출부터 첫 응답까지의 시간                  │ │
│   │                                                         │ │
│   │   사용자 입력 ──▶ [대기] ──▶ 첫 번째 출력                │ │
│   │                   │───────────────│                      │ │
│   │                     응답 시간                            │ │
│   │                                                         │ │
│   │   목표: 100ms 이내 (인간이 지각하지 못하는 수준)          │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   ② 공정성 (Fairness)                                           │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │   • 모든 프로세스가 동등한 CPU 시간 할당                  │ │
│   │   • 기아(Starvation) 현상 방지                          │ │
│   │   • 우선순위 역전 문제 해결 필요                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   ③ 처리량 (Throughput)                                         │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │   단위 시간당 완료되는 프로세스 수                        │ │
│   │   • 응답성과 트레이드오프 관계                           │ │
│   │   • 타임 슬라이스 ↓ → 응답성 ↑, 처리량 ↓                 │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 4. 선점형 스케줄링

```
┌─────────────────────────────────────────────────────────────────┐
│               선점형 vs 비선점형 스케줄링                         │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   비선점형 (Non-preemptive):                                    │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │ • 프로세스가 자발적으로 CPU 반납                          │ │
│   │ • I/O 요청, 작업 완료 시에만 스위칭                       │ │
│   │ • 구현 간단, 문맥 교환 적음                              │ │
│   │ • 응답성 낮음, 긴 작업 시 다른 프로세스 대기              │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   선점형 (Preemptive):                                          │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │ • OS가 강제로 CPU 회수 가능                              │ │
│   │ • 타임 슬라이스 만료, 높은 우선순위 도착 시               │ │
│   │ • 응답성 높음, 실시간 시스템에 적합                       │ │
│   │ • 문맥 교환 오버헤드, 동기화 복잡                        │ │
│   │                                                         │ │
│   │   시분할 시스템 = 선점형 스케줄링 필수                    │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   선점 발생 시점:                                                │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │ ① 타임 슬라이스 만료 (Timer Interrupt)                  │ │
│   │ ② 더 높은 우선순위 프로세스 도착                         │ │
│   │ ③ I/O 완료 인터럽트                                      │ │
│   │ ④ 시스템 콜 종료                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 기술사 시험 포인트

### 1. 핵심 정의

> **시분할 시스템**: CPU 시간을 타임 슬라이스로 분할하여 순차 할당
> **타임 슬라이스**: 각 프로세스에 할당되는 CPU 시간 단위 (10~100ms)

### 2. 빈출 키워드

- **선점형 스케줄링**: OS가 강제로 CPU 회수
- **라운드 로빈**: 시분할의 대표적 알고리즘
- **응답 시간**: 첫 응답까지의 시간, 100ms 이내 목표
- **인터랙티브**: 대화형 처리

### 3. 예상 문제

**Q1. 시분할 시스템이 다중 프로그래밍과 다른 점을 설명하시오.**

**Q2. 타임 슬라이스 크기가 시스템 성능에 미치는 영향을 기술하시오.**

**Q3. 시분할 시스템에서 선점형 스케줄링이 필요한 이유를 설명하시오.**
