# 비대칭 다중 처리 (ASMP)

## 정의

**비대칭 다중 처리(Asymmetric Multiprocessing, ASMP)**는 각 CPU가 서로 다른 역할을 수행하는다. 하나의 CPU가 마스터가 되 시스템 제어 및 I/O 처리를 담당하고 사용자 응용 프로그램 실행합니다.

```
┌─────────────────────────────────────────────────────────────────┐
│           ASMP 구조                                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌───────────────────────────────────────────────────────┐    │
│   │                  Master CPU                      │    │
│   │              (운영 체제, I/O 제어)                        │    │
│   └───────────────────────────────┬───────────────────────┘    │
│                          │                    │                          │
│                          ▼                    ▼                          │
│   ┌─────────────────────┐    ┌─────────────────────┐    │
│   │   Master CPU      │    │   Slave CPU        │    │
│   │   (시스템 제어)     │    │   (사용자 태스크)        │    │
│   └─────────────────────┘    └─────────────────────┘    │
│                                                                 │
│   특징:                                                │
│   • 마스터 CPU: 복잡한 OS 코드 실행                              │
│   • 슬레이브 CPU: 사용자 애플리케이션만 실행                     │
│   • 오버헤드 크지만, 통신 필요                             │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 핵심 개념

### 1. ASMP vs SMP

```
┌─────────────────────────────────────────────────────────────────┐
│           ASMP vs SMP 비교                                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │  구분       │  ASMP             │  SMP               │ │
│   ├─────────────┼─────────────────────┼─────────────────────┤ │
│   │ 구조      │ 마스터-슬레이브   │ 대칭적 구조          │ │
│   │             │ (Master-Slave)  │ (Symmetric)        │ │
│   ├─────────────┼─────────────────────┼─────────────────────┤ │
│   │ OS 역할  │ 마스터만 수행     │ 모든 CPU가 공통 수행     │ │
│   │             │ (Master only)  │ (All CPUs)          │ │
│   ├─────────────┼─────────────────────┼─────────────────────┤ │
│   │ 메모리     │ 마스터 관리      │ 모든 CPU가 공유 메모리  │ │
│   │             │ (Master manages)│ (Shared Memory)     │ │
│   ├─────────────┼─────────────────────┼─────────────────────┤ │
│   │ I/O        │ 마스터만 처리     │ 모든 CPU가 공유 I/O     │ │
│   │             │ (Master only)  │ (All CPUs)          │ │
│   ├─────────────┼─────────────────────┼─────────────────────┤ │
│   │ 복잡성   │ 낮음              │ 높음               │ │
│   │ (Complexity)│ (Simple)          │ (Complex)          │ │
│   ├─────────────┼─────────────────────┼─────────────────────┤ │
│   │ 성능      │ 마스터 병목 가능   │ 선형 확장            │ │
│   │ (Performance)│ (Bottleneck)     │ (Scalable)         │ │
│   ├─────────────┼─────────────────────┼─────────────────────┤ │
│   │ 적용       │ 소규모 임베디드 │ 대규모 서버          │ │
│   │ (Use Case) │ (Small Embedded)│ (Large Servers)    │ │
│   └─────────────┴─────────────────────┴─────────────────────┘ │
│                                                                 │
│   💡 기억: SMP이 대칭적 처리의 표준                              │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2. 장단점

```
┌─────────────────────────────────────────────────────────────────┐
│           ASMP 장단점                                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   장점 (Advantages):                                           │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │ • 설계 단순, 구현 용이                              │ │
│   │   (Simple design, easy implementation)                │ │
│   │                                                         │ │
│   │ • 특정 작업에 최적화 가능                           │ │
│   │   (Optimization for specific tasks)                │ │
│   │                                                         │ │
│   │ • 슬레이브 장애 시에도 서비스 계속 유지                   │ │
│   │   (Service continuity on slave failure)             │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   단점 (Disadvantages):                                         │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │ • 마스터 CPU가 병목 (SPOF)                         │ │
│   │   (Master CPU bottleneck)                         │ │
│   │                                                         │ │
│   │ • 마스터 부하 분산 어려움                              │ │
│   │   (Difficult to balance master load)               │ │
│   │                                                         │ │
│   │ • 소프트웨어 복잡성 증가                                │ │
│   │   (Increased software complexity)                    │ │
│   │                                                         │ │
│   │ • 슬레이브 CPU 간 동기화 복잡                             │ │
│   │   (Complex synchronization between slaves)             │ │
│   │                                                         │ │
│   │ • 비대칭적이 비용 효율 떨어질 수 있음                    │ │
│   │   (May have cost efficiency issues)                  │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 기술사 시험 포인트

### 1. 핵심 정의

> **ASMP**: 각 CPU가 고유 작업을 수행, 마스터-슬레이브 구조
> **SMP**: 모든 CPU가 공유 메모리, 대칭적 구조

### 2. 빈출 키워드

- **마스터 CPU**: 시스템 제어, 복잡한 OS
- **슬레이브 CPU**: 사용자 태스크만 실행
- **SPOF**: ASMP에서 마스터 CPU가 단일 장애점
- **선형 확장**: SMP에서 성능 향상

### 3. 예상 문제

**Q1. ASMP와 SMP의 차이점을 설명하고 장단점을 비교하시오.**

**Q2. ASMP가 적합한 시스템 유형을 설명하시오.**

**Q3. 멀티코어 시스템에서 ASMP 대칭적 SMP를 선택하는 기준을 설명하시오. (CPU 코어 수, 캐시 계층 구조 등)
