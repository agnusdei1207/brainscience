# 약결합 시스템 (Loosely Coupled System)

## 정의

**약결합 시스템(Loosely Coupled System)**은 여러 독립적인 컴퓨터가 **네트워크를 통해 연결**되어 통신하면서 하나의 시스템처럼 동작하는 분산 컴퓨팅 환경이다. 각 노드는 자체 메모리와 운영체제를 가지며, 메시지 전달을 통해 협력한다.

```
┌─────────────────────────────────────────────────────────────────┐
│           약결합 시스템 구조                                      │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌───────────────────────────────────────────────────────────┐│
│   │                    네트워크 (Network)                       ││
│   │   ══════════════════════════════════════════════════▶    ││
│   └───────────────────────────────────────────────────────────┘│
│            │                 │                 │                │
│            ▼                 ▼                 ▼                │
│   ┌─────────────┐   ┌─────────────┐   ┌─────────────┐        │
│   │   노드 A    │   │   노드 B    │   │   노드 C    │        │
│   │ ┌─────────┐ │   │ ┌─────────┐ │   │ ┌─────────┐ │        │
│   │ │   CPU   │ │   │ │   CPU   │ │   │ │   CPU   │ │        │
│   │ ├─────────┤ │   │ ├─────────┤ │   │ ├─────────┤ │        │
│   │ │  Memory │ │   │ │  Memory │ │   │ │  Memory │ │        │
│   │ ├─────────┤ │   │ ├─────────┤ │   │ ├─────────┤ │        │
│   │ │   OS    │ │   │ │   OS    │ │   │ │   OS    │ │        │
│   │ └─────────┘ │   │ └─────────┘ │   │ └─────────┘ │        │
│   └─────────────┘   └─────────────┘   └─────────────┘        │
│                                                                 │
│   특징: 각 노드가 독립적 메모리, 메시지 전달로 통신             │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 핵심 개념

### 1. 약결합 vs 강결합

```
┌─────────────────────────────────────────────────────────────────┐
│           약결합 vs 강결합 비교                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │  구분        │  강결합            │  약결합           │ │
│   ├──────────────┼─────────────────────┼─────────────────────┤ │
│   │ 메모리      │ 공유 메모리         │ 분산 메모리          │ │
│   │ (Memory)    │ (Shared Memory)    │ (Distributed Memory)│ │
│   ├──────────────┼─────────────────────┼─────────────────────┤ │
│   │ 통신 방식    │ 메모리 접근         │ 메시지 전달          │ │
│   │             │ (Memory Access)    │ (Message Passing)  │ │
│   ├──────────────┼─────────────────────┼─────────────────────┤ │
│   │ 동기화      │ 하드웨어 락         │ 소프트웨어 락        │ │
│   │             │ (Hardware Lock)    │ (Software Lock)    │ │
│   ├──────────────┼─────────────────────┼─────────────────────┤ │
│   │ 대역폭      │ 높음 (버스)         │ 낮음 (네트워크)      │ │
│   │ (Bandwidth) │ (Bus)              │ (Network)          │ │
│   ├──────────────┼─────────────────────┼─────────────────────┤ │
│   │ 지연 시간   │ 낮음 (ns~μs)       │ 높음 (μs~ms)        │ │
│   │ (Latency)   │                    │                    │ │
│   ├──────────────┼─────────────────────┼─────────────────────┤ │
│   │ 확장성      │ 제한적              │ 우수                │ │
│   │ (Scalability)│ (Limited)          │ (Excellent)       │ │
│   ├──────────────┼─────────────────────┼─────────────────────┤ │
│   │ 예시        │ SMP, 멀티코어       │ 클러스터, 분산 DB  │ │
│   └──────────────┴─────────────────────┴─────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2. 메시지 전달 방식

```
┌─────────────────────────────────────────────────────────────────┐
│           메시지 전달 방식                                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   동기식 메시지 전달 (Synchronous):                             │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   송신자                    수신자                       │ │
│   │     │                        │                          │ │
│   │     │─── Send() ───────────▶│                          │ │
│   │     │                        │                          │ │
│   │     │◀─── Ack ──────────────│                          │ │
│   │     │                        │                          │ │
│   │     │                        │                          │ │
│   │                                                         │ │
│   │   특징: 송신자가 확인 응답 대기 (블로킹)                  │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   비동기식 메시지 전달 (Asynchronous):                           │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │                                                         │ │
│   │   송신자                    수신자                       │ │
│   │     │                        │                          │ │
│   │     │─── Send() ───────────▶│                          │ │
│   │     │                        │                          │ │
│   │     │    (계속 진행)          │                          │ │
│   │     │                        │                          │ │
│   │                                                         │ │
│   │   특징: 송신자가 즉시 계속 진행 (논블로킹)                  │ │
│   │                                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

```
┌─────────────────────────────────────────────────────────────────┐

│   장점:                                                         │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │ ① 높은 확장성                                           │ │

│   │                                                         │ │
│   │ ③ 지리적 분산                                            │ │
│   │   • 물리적으로 떨어진 위치에 배치 가능                        │ │
│   │                                                         │ │
│   │ ④ 비용 효율성                                              │ │
│   │   • 일반 하드웨어 사용 가능                               │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   단점:                                                         │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │ ① 높은 통신 지연                                          │ │
│   │   • 네트워크 지연으로 성능 저하                             │ │
│   │                                                         │ │
│   │ ② 복잡한 프로그래밍                                            │ │
│   │   • 분산 동기화, 일관성 유지 어려움                      │ │
│   │                                                         │ │
│   │ ③ 부분 장애 가능성                                            │ │
│   │   • 네트워크 분할, 노드 장애                              │ │
│   │                                                         │ │
│   │ ④ 디버깅 어려움                                            │ │
│   │   • 분산 환경에서 문제 추적 복잡                             │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 기술사 시험 포인트

### 1. 핵심 정의

> **약결합 시스템**: 네트워크로 연결된 독립 노드들, 메시지 전달로 통신
> **강결합 시스템**: 공유 메모리를 통한 직접 통신

### 2. 빈출 키워드

- **분산 메모리**: 각 노드가 독립적 메모리
- **메시지 전달**: 노드 간 통신 방식
- **확장성**: 시스템 확장 능력
- **지연 시간**: 통신 지연

### 3. 예상 문제

**Q1. 약결합 시스템과 강결합 시스템의 차이점을 설명하시오.**

**Q2. 약결합 시스템의 장단점을 기술하시오.**

**Q3. 클러스터 시스템이 약결합 시스템인 이유를 설명하시오.**
