# 894. SASE (Secure Access Service Edge) - 네트워크와 보안의 클라우드 합체

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: SD-WAN의 **네트워킹 기능**과 SWG, CASB, ZTNA 같은 **클라우드 네이티브 보안 기능**을 하나의 통합된 클라우드 서비스로 결합한 아키텍처이다.
> 2. **혁신**: 사용자가 어디에 있든(집, 카페, 해외지사) 본사 방화벽을 거치지 않고 가장 가까운 클라우드 에지(Edge)에서 즉시 보안 검사와 네트워크 최적화를 동시에 수행한다.
> 3. **가치**: 복잡한 개별 보안 장비(Point Solutions)를 관리할 필요를 없애 운영 복잡성을 획기적으로 낮추고, '제로 트러스트' 보안 원칙을 전사적으로 일관되게 적용한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
SASE(새시)는 2019년 가트너(Gartner)가 제시한 개념으로, 데이터센터 중심의 네트워크 설계에서 벗어나 'ID(신원)' 중심의 보안과 네트워크를 지향한다. 모든 보안 정책을 클라우드에서 관리하고 실행하며, 사용자의 위치에 상관없이 동일한 보안 경험을 제공한다.

### 💡 비유
- **전통적 보안 (성문 검문소)**: 모든 사람(데이터)이 성(본사) 안으로 들어가려면 정문(방화벽) 하나를 반드시 통과해야 한다. 성 밖 카페에서 일하는 사람도 성문까지 왔다가 가야 하니 매우 비효율적이다.
- **SASE (모바일 보안 요원)**: 전 세계 어디에나 보안 요원(클라우드 에지)이 배치되어 있다. 내가 집 앞 카페에서 일하면, 바로 옆에 있는 보안 요원이 나를 검사하고 안전한 길로 안내해 준다. 본사 성문까지 갈 필요가 없어 빠르고 안전하다.

### 등장 배경: 'Work from Anywhere'
코로나19 이후 재택근무가 일상화되고 업무용 앱이 SaaS(Slack, Salesforce 등)로 이동했다. 사용자는 집에서 바로 클라우드로 접속하고 싶은데, 기존 VPN 방식은 모든 트래픽을 회사로 불러들였다가 다시 내보내는 구조라 속도가 너무 느렸다. 이 병목 현상을 해결하고 보안까지 챙기기 위해 SASE가 등장했다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### SASE 구성 요소 및 통합 구조 (ASCII Diagram)

```ascii
[ SASE Cloud Platform ]
┌───────────────────────────────────────────────────────────┐
│   Network (SD-WAN)    +    Security (SSE)                 │
│  - Path Optimization       - ZTNA (Zero Trust Access)     │
│  - Traffic Shaping         - SWG (Web Gateway)            │
│  - SaaS Acceleration       - CASB (Cloud Access Broker)   │
│                            - FWaaS (Firewall as a Service)│
└───────────────▲───────────────────────▲───────────────────┘
                │                       │
      (User Identity & Context)         │ (Policy Enforcement)
                │                       │
┌───────────────┴───────────────────────┴───────────────────┐
│     Users (Home) | Branches (Office) | IoT Devices        │
└───────────────────────────────────────────────────────────┘

* 핵심 원리: 보안과 네트워크 기능을 칩이나 장비가 아닌 
  '클라우드 서비스(Service Edge)' 레벨에서 통합 처리.
```

### 5대 핵심 기술 요소

#### ① SD-WAN (Software Defined WAN)
네트워크 경로를 최적화하고 지연 시간을 관리하는 기초 인프라 역할을 한다.

#### ② ZTNA (Zero Trust Network Access) - SASE의 심장
"아무도 믿지 마라"는 원칙하에, 접속을 허용하기 전 매번 신원과 기기 상태를 검증한다. 일단 접속하면 모든 권한을 주는 VPN과 달리, 허용된 특정 앱에만 접근할 수 있게 한다.

#### ③ SWG (Secure Web Gateway)
인터넷 웹사이트 접속 시 악성코드 유무를 검사하고 유해 사이트 접속을 차단한다.

#### ④ CASB (Cloud Access Security Broker)
기업이 사용하는 SaaS(G-Suite 등) 내에서의 데이터 유출(DLP)을 감시하고 비인가된 클라우드 사용(Shadow IT)을 제어한다.

#### ⑤ FWaaS (Firewall as a Service)
기존의 물리적 방화벽 장비 기능을 클라우드에서 수행하여 전 세계 지사에 일관된 방화벽 정책을 적용한다.

## Ⅲ. 융합 비교 및 다각도 분석

### 전통적 VPN/MPLS vs SASE 상세 비교

| 비교 항목 | 전통적 방식 (VPN + MPLS) | **SASE (SD-WAN + SSE)** |
|-----------|-------------------------|--------------------------|
| **중심축** | 데이터센터 (DC) | **신원 (Identity / User)** |
| **접속 경로** | DC 경유 (Tromboning 현상) | **최단 경로 직접 접속** |
| **보안 철학** | 경계 보안 (성벽 쌓기) | **제로 트러스트 (항시 검증)** |
| **관리 단위** | 장비별 (Firewall, IPS 등) | **통합 서비스 (One Policy)** |
| **사용자 경험**| 접속 위치에 따라 속도 다름 | **전 세계 어디서나 동일 속도** |
| **비유** | 성문 통과하기 | **스마트 보안 카드 소지** |

### SSE (Security Service Edge)와의 관계
SASE에서 네트워킹(SD-WAN)을 제외하고 보안 기능들만 모은 것을 **SSE**라고 부른다. 이미 고성능 네트워크망을 가진 기업은 SSE만 도입하여 SASE를 완성하기도 한다.

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

### 기술사적 판단 (실무 도입 전략)

**상황**: 글로벌 전사 직원이 1만 명인 기업에서 보안 유지를 위해 전 직원이 강제로 본사 VPN을 쓰게 했더니, "화상회의가 뚝뚝 끊긴다"는 불만이 폭주하고 있다.
**판단 및 SASE 도입 전략**:
1. **에이전트 배포**: 전 직원 노트북에 SASE 전용 클라이언트를 설치한다.
2. **ZTNA 적용**: VPN을 폐지하고 ZTNA를 통해 직원이 ERP에 접속할 때만 권한을 준다. Zoom 트래픽은 보안 검사 없이 가장 가까운 에지에서 바로 나가게(Bypass) 설정한다.
3. **CASB 설정**: 직원이 회사 계정의 구글 드라이브에 기밀 문서를 올리는지, 개인 계정으로 파일을 옮기는지 클라우드에서 실시간 감시한다.
4. **결과**: 네트워크 지연 시간 50% 단축으로 사용자 만족도 급상승, 동시에 모든 접속 시도에 대한 가시성과 보안 통제권 확보.

### 안티패턴 및 고려사항
- **벤더 종속성**: 모든 보안과 네트워크를 한 회사(Cisco, Zscaler, Palo Alto 등)에 맡기면 나중에 다른 솔루션으로 옮기기 매우 어렵다.
- **성능 과신**: 클라우드 에지의 성능이 본사 방화벽 장비보다 낮을 수도 있다. 도입 전 주요 거점에서의 실제 응답 속도(Latency) 테스트가 필수적이다.

## Ⅴ. 미래 전망 및 결론

### 결론: 네트워크는 이제 '서비스'다
SASE는 네트워크와 보안이 더 이상 장비 구매의 영역이 아니라, 전기나 수도처럼 구독하는 **'서비스(SaaS)'**의 영역으로 넘어왔음을 상징한다. IT 인프라 관리자는 전선을 연결하는 대신 정책(Policy)을 코딩하게 될 것이다.

### 미래 전망
앞으로는 **AI 기반 자동 위협 대응**이 결합될 것이다. 사용자의 평소 행동 패턴과 다른 접속 시도가 발견되면 AI가 즉시 접속을 차단하고 격리(Isolation)하는 지능형 SASE로 진화할 것이다. 또한, 5G/6G 모바일 엣지 컴퓨팅(MEC)과 결합하여 모바일 기기 자체의 보안 수준을 비약적으로 높일 것으로 기대된다.

## 👶 어린이를 위한 3줄 비유 설명
1. **SASE가 뭔가요?**: 인터넷을 안전하게 해주는 '보디가드'와 길을 잘 찾아주는 '가이드'가 하나로 합쳐져서 클라우드 속에 살고 있는 똑똑한 서비스예요.
2. **왜 필요하나요?**: 예전에는 보안 검사를 받으러 회사까지 멀리 가야 했지만, 이제는 내가 있는 곳 바로 옆에서 보디가드가 지켜주니까 아주 빠르고 안전해요.
3. **뭐가 제일 좋나요?**: 집에서 공부하든 학교에서 공부하든, 나쁜 해커가 내 정보를 훔쳐가지 못하게 항상 빈틈없이 지켜준답니다!
