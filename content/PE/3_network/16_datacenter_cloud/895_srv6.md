# 895. SRv6 (Segment Routing over IPv6) - IPv6 기반의 지능형 패킷 네비게이션

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 복잡한 MPLS 태그 대신 **IPv6의 확장 헤더를 활용하여 패킷의 이동 경로를 지정**하는 소스 라우팅(Source Routing) 기술로, 네트워크의 상태 정보(State)를 중간 노드가 아닌 패킷 헤더에 직접 담는 방식이다.
> 2. **혁신**: 네트워크 슬라이싱(Network Slicing)과 서비스 체이닝(SFC)을 소프트웨어적으로 완벽하게 구현하며, 복잡한 라우팅 프로토콜(LDP, RSVP-TE 등)을 제거하여 망 구조를 획기적으로 단순화한다.
> 3. **가치**: 5G 및 클라우드 네이티브 인프라의 핵심 엔진으로, 수십억 개의 단말과 서비스를 IPv6 주소 체계 안에서 직접 연결하고 트래픽을 정교하게 제어할 수 있는 기반을 제공한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
SRv6는 세그먼트 라우팅(SR)을 IPv6 데이터 평면 위에서 구현한 것이다. 패킷을 보내는 출발지(Source)에서 패킷이 거쳐 가야 할 지점들을 'Segment ID(SID)'라는 IPv6 주소 리스트로 만들어 헤더에 담아 보낸다.

### 💡 비유
- **전통적 라우팅 (이정표 방식)**: 운전자가 목적지만 알고 출발한다. 교차로(라우터)마다 멈춰 서서 이정표(라우팅 테이블)를 보고 어디로 갈지 물어봐야 한다. 이정표가 바뀌면 운전자는 길을 잃는다.
- **SRv6 (네비게이션 방식)**: 출발할 때 이미 상세 경로(A빌딩 앞 좌회전 → B마트 통과 → C주유소 우회전)가 적힌 리스트를 들고 출발한다. 중간 지점에서는 리스트의 다음 칸만 확인하고 즉시 이동하면 된다. 중간 라우터들은 전체 경로를 외울 필요가 없다.

### 등장 배경: 'Complexity'와 'Scalability'
기존의 MPLS 망은 터널을 뚫기 위해 RSVP-TE 같은 복잡한 프로토콜이 필요했고, 망 규모가 커질수록 관리 비용이 폭증했다. 5G 시대에는 수많은 '슬라이스'를 만들어야 하는데, MPLS로는 한계가 명확했다. IPv6 주소 자체를 '명령어'로 쓰는 SRv6는 이 문제를 깔끔하게 해결했다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### SRv6 패킷 구조 및 흐름 (ASCII Diagram)

```ascii
[ SRv6 Packet Structure ]
┌─────────────────────────────────────────────────────────────┐
│  IPv6 Header (Src: Start, Dst: Current Segment ID)          │
├─────────────────────────────────────────────────────────────┤
│  SRH (Segment Routing Header)                               │
│  - Segment List [SID 0, SID 1, SID 2...] (The Path)         │
│  - Segments Left: 2 (Pointer to next SID)                   │
├─────────────────────────────────────────────────────────────┤
│  Payload (Actual Data)                                      │
└─────────────────────────────────────────────────────────────┘

[ Forwarding Process ]
Node A (Source) ───> Node B (SID 1) ───> Node C (SID 2) ───> Node D (End)
 (Sets SRH)          (Updates Dst IP)     (Updates Dst IP)     (Process Data)
                     (Decrements Left)    (Decrements Left)
```

### 핵심 구성 요소 및 메커니즘

#### ① SID (Segment Identifier)
SRv6에서 SID는 단순한 주소가 아니라 **'함수(Instruction)'**이다. 128비트 IPv6 주소를 `Locator : Function : Argument` 형태로 나누어 쓴다.
- **Locator**: 패킷이 도달해야 할 라우터의 위치.
- **Function**: 해당 라우터에서 수행해야 할 동작 (예: 특정 인터페이스로 포워딩, 방화벽 통과 등).

#### ② SRH (Segment Routing Header)
IPv6 확장 헤더 중 하나로, 패킷이 거쳐야 할 경로(SID 리스트)가 저장되는 공간이다.

#### ③ TI-LFA (Topology-Independent Loop-Free Alternate)
장애 발생 시 50ms 이내로 경로를 우회시키는 기술이다. SRv6는 소스에서 경로를 지정하므로, 어떤 토폴로지에서도 루프 없이 즉시 백업 경로를 계산할 수 있다.

## Ⅲ. 융합 비교 및 다각도 분석

### MPLS vs SR-MPLS vs SRv6 상세 비교

| 비교 항목 | 전통적 MPLS | SR-MPLS | **SRv6** |
|-----------|-------------|---------|----------|
| **데이터 평면** | MPLS Labels | MPLS Labels | **IPv6 Address** |
| **상태 관리** | 모든 노드가 관리 | 소스 노드만 관리 | **소스 노드만 관리** |
| **프로토콜** | LDP, RSVP-TE 등 복잡 | 단순화 (IGP 확장) | **완벽 단순화 (IPv6)** |
| **서비스 체이닝**| 매우 어려움 | 어려움 | **매우 쉬움 (SID 함수 활용)** |
| **End-to-End** | Core 망 한정 | Core 망 한정 | **앱에서 앱까지 직접 연결 가능** |

### SRv6의 3대 혁신 가치
1. **Unification**: 언어(IPv6)가 통일되어 데이터센터, 코어망, 액세스망이 하나로 합쳐진다.
2. **Programmability**: 패킷 헤더에 네트워크 '프로그램'을 실어 보낼 수 있다.
3. **Application Awareness**: 어떤 앱인지에 따라 네트워크가 스스로 경로 품질(Latency, Jitter)을 보장한다.

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

### 기술사적 판단 (실무 도입 전략)

**상황**: 5G 망을 구축 중인 통신사에서 자율주행(초저지연)과 유튜브(대용량) 트래픽을 하나의 망에서 분리하여 처리(Network Slicing)해야 한다.
**판단 및 SRv6 적용 전략**:
1. **SID 설계**: 망 전체 라우터에 SRv6 능력을 부여하고, 용도별 SID(Function)를 정의한다. (예: `Func 100` = Low Latency Path).
2. **Network Slicing**: 자율주행 패킷에는 최단 거리 라우터들의 SID 리스트를 넣고, 유튜브 패킷에는 대역폭이 넓은 일반 라우터 리스트를 넣는다.
3. **Service Chaining**: 보안 검사가 필요한 트래픽은 SID 리스트 중간에 'Firewall 가상 함수'를 끼워 넣어 강제로 방화벽을 거치게 만든다.
4. **결과**: 별도의 터널 설정 없이 패킷 헤더 조작만으로 완벽한 서비스 분리 및 품질 보장 성공.

### 주의사항 및 안티패턴
- **MTU 오버헤드**: SID 리스트가 길어지면 IPv6 헤더가 너무 커져서 실제 데이터(Payload) 공간이 줄어든다. 이를 방지하기 위해 **SRv6-uSID (Micro-segments)** 기술을 적용하여 SID 크기를 압축해야 한다.
- **HW 지원 여부**: 기존 구형 라우터는 SRv6 헤더를 인식하지 못할 수 있다. 망 고도화 시 칩셋 수준의 지원 여부를 반드시 확인해야 한다.

## Ⅴ. 미래 전망 및 결론

### 결론: 네트워크의 '언어 통일'
SRv6는 네트워크가 복잡한 기술의 집합체가 아니라, 하나의 거대한 '컴퓨터'처럼 작동하게 만드는 운영체제와 같다. IPv6라는 공용어를 통해 모든 장치가 소통하는 진정한 **'IP-on-Everything'** 시대를 여는 열쇠다.

### 미래 전망
앞으로는 **'APN6 (Application-aware IPv6)'**와 결합하여 애플리케이션이 직접 네트워크에게 "나는 지금 10ms 지연시간이 필요해"라고 요구하면, 네트워크가 실시간으로 SRv6 경로를 코딩하여 할당하는 자율 지능형망(Self-Driving Network)으로 진화할 전망이다.
