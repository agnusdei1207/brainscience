# 912. IPFS (InterPlanetary File System) - 위치가 아닌 내용으로 찾는 분산 웹

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터가 저장된 서버의 위치(URL/IP)가 아니라, 데이터의 고유한 해시값인 **'콘텐츠 주소(Content Addressing)'를 기반으로 파일을 배포하고 공유하는 P2P 방식의 분산형 파일 시스템**이다.
> 2. **혁신**: 중앙 집중형 서버(HTTP)의 병목과 검열 문제를 해결하기 위해 **머클 DAG(Merkle DAG)**와 **분산 해시 테이블(DHT)** 기술을 결합하여, 전 세계 노드들이 조각난 파일들을 서로 주고받는 영구적이고 탄력적인 망을 구성한다.
> 3. **가치**: 웹의 중앙화를 탈피하여 데이터 주권을 사용자에게 돌려주며, 중복 저장 방지를 통한 대역폭 절감과 오프라인/저대역폭 환경에서도 데이터 접근성을 보장하는 차세대 웹3.0의 저장 표준이다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
IPFS는 "행성 간 파일 시스템"이라는 거창한 이름처럼, 지연 시간이 매우 긴 환경(예: 지구와 화성 사이)에서도 작동할 수 있도록 설계된 분산 프로토콜이다. 기존 HTTP가 "어디(Location)에 있는가?"를 묻는다면, IPFS는 "무엇(Content)을 찾는가?"를 묻는다.

### 💡 비유
- **기존 웹 (HTTP / 도서관 대출)**: 내가 "어린 왕자" 책을 보고 싶으면 반드시 서울중앙도서관(중앙 서버)에 가야 한다. 도서관이 문을 닫거나 길이 막히면 책을 볼 수 없다. 심지어 내 옆집 사람이 그 책을 갖고 있어도 나는 알 수 없다.
- **분산 웹 (IPFS / 동네 사람들)**: 내가 "어린 왕자" 책의 고유 번호(해시값)를 전단지에 적어 동네에 뿌린다. 그러면 그 책을 가진 옆집 사람, 뒷동네 사람(P2P 노드)이 책의 조각들을 조금씩 건네준다. 중앙 도서관이 없어져도 세상 어딘가에 책을 가진 사람만 있다면 나는 언제든 책을 읽을 수 있다.

### 등장 배경: 'HTTP의 중앙화'와 가용성 위기
현재의 웹(HTTP)은 아마존(AWS)이나 구글 같은 거대 기업의 서버에 종속되어 있다. 서버가 죽으면 서비스는 마비되고, 정부가 사이트를 차단하면 접근이 불가능하다. 또한, 똑같은 유튜브 영상을 1억 명이 볼 때 중앙 서버는 1억 번 데이터를 보내야 하는 막대한 대역폭 낭비가 발생한다. IPFS는 이러한 비효율과 통제를 하드웨어/네트워크 아키텍처 레벨에서 해결하고자 탄생했다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### IPFS 데이터 구조 및 검색 프로세스 (ASCII Diagram)

```ascii
[ 1. Content Addressing (CID 생성) ]
  원본 파일 ──(SHA-256 해시)──> QmXoyp... (고유 ID: CID)

[ 2. Data Structure (Merkle DAG) ]
  파일을 256KB 조각(Block)으로 쪼개고 트리 형태로 엮음
       [ Root Hash (QM...) ]
        /        |        \
    [Block A] [Block B] [Block C]  <-- 일부 조각만 바뀌어도 전체 해시 변경

[ 3. Network Search (Kademlia DHT) ]
  사용자: "QmXoyp... 가진 사람?"
           │ (DHT 쿼리 전파)
  ┌────────▼────────┐      ┌──────────────────┐
  │  Node A (Peer)  │ <──> │  Node B (Peer)   │
  │ (파일 조각 보유) │      │ (파일 조각 보유) │
  └────────┬────────┘      └────────┬─────────┘
           │                        │
           └────> [ BitSwap ] <─────┘
           (패킷 조각들을 P2P로 전송)
```

### 4대 핵심 기술 요소

#### ① 콘텐츠 주소 지정 (Content Addressing)
파일의 주소는 서버 도메인명이 아니라 파일의 '해시값(CID)' 그 자체다. 내용이 조금이라도 바뀌면 주소도 바뀐다. 따라서 주소만 알면 그 파일이 위조되지 않았음을 수학적으로 100% 확신할 수 있다.

#### ② 머클 DAG (Merkle Directed Acyclic Graph)
파일과 폴더 구조를 해시 트리의 일종인 DAG로 관리한다. 데이터 중복을 방지(Deduplication)하는 데 탁월하며, 대용량 파일의 부분적 전송과 무결성 검증을 동시에 수행한다.

#### ③ 분산 해시 테이블 (Kademlia DHT)
"어떤 노드가 어떤 파일을 갖고 있는가?"에 대한 거대한 장부를 전 세계 노드들이 조금씩 나누어 가진다. 중앙 서버 없이도 전 세계 인터넷망에서 수억 개의 파일 중 내가 원하는 것을 0.5초 만에 찾아낼 수 있는 나침반 역할을 한다.

#### ④ 비트스왑 (BitSwap)
비트토렌트와 유사한 패킷 교환 프로토콜이다. 내가 원하는 조각을 가진 노드에게 데이터를 요청하고, 내가 가진 조각을 남에게 빌려주며 신뢰 점수를 쌓는 인센티브 기반 전송 방식이다.

## Ⅲ. 융합 비교 및 다각도 분석

### HTTP vs IPFS 상세 비교

| 비교 항목 | HTTP (중앙 집중형) | **IPFS (분산형)** |
|-----------|--------------------|-------------------|
| **주소 지정** | 위치 기반 (Location-based) | **콘텐츠 기반 (Content-based)** |
| **신뢰 모델** | 서버 관리자를 믿어야 함 | **수학적 해시를 믿음 (무결성)** |
| **대역폭 비용** | 높음 (중앙에서 계속 쏴야 함) | **매우 낮음 (이웃 노드에서 가져옴)** |
| **영구성** | 서버 주인이 지우면 사라짐 | **누군가 한 명이라도 들고 있으면 영구 보존** |
| **보안** | SSL/TLS로 구간 암호화 | **해시 기반 위변조 원천 차단** |
| **비유** | 지정된 주소의 식당 방문 | **레시피로 전 세계 주방에서 요리** |

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

### 기술사적 판단 (실무 웹3.0 스토리지 구축 전략)

**상황**: NFT(대체 불가능 토큰) 서비스를 론칭하려 한다. 수백만 원짜리 디지털 아트 이미지를 구글 드라이브에 저장했더니 사용자들이 "구글이 지우면 내 NFT는 어떻게 되냐?"며 항의한다.
**판단 및 IPFS 적용 전략**:
1. **분산 저장**: 이미지를 중앙 서버가 아닌 **IPFS망**에 올린다.
2. **콘텐츠 식별자(CID) 활용**: 블록체인 스마트 컨트랙트의 `tokenURI` 값에 구글 주소 대신 `ipfs://Qm...` 해시 주소를 직접 박아 넣는다.
3. **Pinning 서비스 연동**: 전 세계 노드들이 내 파일을 지우지 않게 하려고 **Pinata**나 **Infura** 같은 피닝(Pinning) 서비스를 활용해 데이터를 영구 고정한다.
4. **결과**: 회사가 망해도, 구글이 서비스를 종료해도 전 세계 누군가의 PC에 내 그림 조각이 남아있는 한 NFT의 가치는 영원히 보존되는 신뢰성 확보.

### 안티패턴 및 고려사항
- **개인정보 저장**: IPFS는 한 번 올라가면 삭제가 매우 어렵고 누구나 볼 수 있는 '공공 장부'와 같다. 여기에 암호화되지 않은 고객의 이름이나 주민번호를 올리는 것은 **GDPR(잊힐 권리) 위반**이자 보안 대참사다. 민감 정보는 절대 올리지 않거나, 반드시 동형 암호화 등을 거쳐야 한다.

## Ⅴ. 미래 전망 및 결론

### 결론: 웹의 잃어버린 조각을 찾아서
인터넷은 원래 분산 시스템으로 설계되었으나, 자본의 논리에 의해 중앙화되었다. IPFS는 기술적 완결성을 통해 다시 웹을 민주화하고 있다. 서버 중심에서 데이터 중심으로의 패러다임 전환은 거스를 수 없는 흐름이다.

### 미래 전망
앞으로는 IPFS 위에 경제적 보상 체계(암호화폐)를 결합한 **파일코인(Filecoin)**과 같은 '인센티브 기반 분산 저장소'가 클라우드 시장의 상당 부분을 잠식할 것이다. 또한, 저궤도 위성 통신(스타링크) 환경에서 지구와 우주를 잇는 거대한 캐시망으로 작동하여, 화성에서도 지구의 최신 뉴스를 0.1초 만에(로컬 캐시) 볼 수 있게 하는 우주 인터넷의 근간 프로토콜이 될 전망이다.

## 👶 어린이를 위한 3줄 비유 설명
1. **IPFS가 뭔가요?**: "서울시 강남구..."라는 주소를 찾는 게 아니라, 물건에 붙은 '이름표'만 부르면 전 세계 어디서든 그 물건을 가진 사람들이 짠! 하고 가져다주는 택배 시스템이에요.
2. **왜 주소를 안 쓰나요?**: 주소는 집이 없어지면 못 찾지만, 이름표는 물건만 있으면 지구 반대편에서도 찾을 수 있기 때문이에요.
3. **가장 좋은 점은?**: 전 세계 모든 컴퓨터가 조금씩 힘을 합쳐서 정보를 나눠주기 때문에, 큰 컴퓨터 한 대가 고장 나도 인터넷이 절대 끊기지 않는답니다!
