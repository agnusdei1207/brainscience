# 934. 라우팅 프로토콜 인증 방어망 - 가짜 경로로부터 인터넷의 지도를 지키는 법

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 인접한 라우터들끼리 경로 정보를 교환할 때, 상대방이 신뢰할 수 있는 이웃인지 확인하고 메시지가 위조되지 않았음을 보증하기 위해 **MD5, SHA-256 등 해시 기반의 암호화 인증을 수행하는 네트워크 보안 체계**다.
> 2. **혁신**: 단순 비밀번호 대조를 넘어, **BGP TTL Security(GTSM), MD5 Signature Option(RFC 2385)** 등을 통해 TCP 세션 하이재킹과 리플레이 공격을 방어하며 라우팅 테이블의 오염(Poisoning)을 차단한다.
> 3. **가치**: 블랙홀 라우팅, 트래픽 우회 탈취와 같은 거대 네트워크 마비 사고를 예방하며, 전 세계 인터넷의 동맥인 BGP와 내부망의 OSPF/IS-IS 신뢰성을 보장하는 최후의 방어선이다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
라우팅 프로토콜 인증은 라우터들끼리의 '암호 통신'이다. "저쪽 길이 더 빨라요!"라고 말하는 라우터가 진짜 우리 편인지, 아니면 나쁜 해커가 가짜 정보를 흘리는 것인지 확인하는 절차다.

### 💡 비유
- **인증 없는 라우팅 (길 안내원 사칭)**: 모르는 사람이 길거리에 서서 "서울 가려면 저쪽 낭떠러지로 가세요!"라고 푯말을 세워두는 것이다. 차들이(패킷) 아무 의심 없이 낭떠러지로 떨어져 버린다. (블랙홀 공격).
- **인증 있는 라우팅 (암행어사 마패)**: 길 안내원이 "서울은 저쪽입니다"라고 말하면서, 품속에서 **'암행어사 마패(디지털 서명)'**를 보여주는 것이다. 마패가 진짜임을 확인한 뒤에야 운전자들은 그 길로 차를 몬다. 가짜 안내원이 아무리 속이려 해도 마패가 없으면 무시당한다.

### 등장 배경: 'BGP Hijacking'의 공포
인터넷은 원래 서로를 믿는다는 전제하에 설계되었다. 하지만 특정 국가나 해커가 가짜 경로 정보를 뿌려 전 세계 구글 트래픽을 자기 나라로 끌어온 뒤 훔쳐보는 사고(BGP 하이재킹)가 빈번해졌다. 이에 따라 라우팅 패킷 하나하나에 '지장'을 찍어야 한다는 보안 요구가 전 세계적으로 강화되었다.

## II. 라우팅 인증 및 방어 핵심 매커니즘 (Deep Dive)

### 라우팅 프로토콜별 인증 구조 (ASCII Diagram)

```ascii
[ Router A ]                               [ Router B ]
┌──────────────────┐                      ┌──────────────────┐
│  Routing Table   │                      │  Routing Table   │
├──────────────────┤                      ├──────────────────┤
│ [ Auth Engine ]  │                      │ [ Auth Engine ]  │
│  - Key: "Secret" │                      │  - Key: "Secret" │
└────────┬─────────┘                      └──────────▲───────┘
         │                                           │
         │ (1) Create Hash:                          │ (2) Compare Hash
         │     MD5(Packet + Key) = [ ABC... ]        │     If OK, Update Table
         └─────────── [ Auth Packet ] ───────────────┘
                       (Data + Hash)

* 주요 보안 기술:
1. BGP MD5 (RFC 2385): TCP 헤더에 해시를 넣어 세션 탈취 방지.
2. OSPFv3 IPsec: 패킷 전체를 암호화하여 기밀성까지 확보.
3. GTSM (TTL Security): TTL 값을 255로 고정하여 멀리서 온 공격 패킷 차단.
```

### 3대 핵심 방어 기술

#### ① 해시 기반 메시지 인증 (HMAC)
라우팅 정보를 보낼 때 미리 약속한 비밀키와 내용을 섞어 해시값(MD5, SHA)을 만든다. 해커가 중간에 경로를 1m만 바꿔도 해시값이 완전히 달라지므로 변조 여부를 즉시 알 수 있다.

#### ② GTSM (Generalized TTL Security Mechanism)
BGP는 보통 바로 옆 라우터(1 Hop)와 대화한다. TTL 값을 255로 설정해 보내고, 받을 때 254가 아니면 버린다. 해커가 인터넷 저 멀리서(수십 Hop 밖) 가짜 BGP 패킷을 쏴도 라우터에 도착할 땐 TTL이 낮아져 있어 자동으로 무시된다.

#### ③ 키 체인 (Key-chain) 관리
비밀번호가 유출될 경우를 대비해, 시간에 따라 자동으로 비밀번호를 교체하는 기능이다. 관리자가 일일이 라우터를 접속하지 않아도 정해진 스케줄에 따라 보안 강도를 유지한다.

## Ⅲ. 융합 비교 및 다각도 분석

### 주요 라우팅 프로토콜 인증 방식 비교 (Table)

| 프로토콜 | 인증 방식 | 주요 특징 | 보안 수준 |
|:---:|:---|:---|:---:|
| **RIPv2** | Plain-text / MD5 | 단순 암호 대조 | 낮음 |
| **OSPFv2** | MD5 / HMAC-SHA | 패킷별 해시 포함 | 높음 |
| **BGP** | **TCP MD5 (RFC 2385)** | TCP 세션 자체를 보호 | **매우 높음** |
| **IS-IS** | HMAC-SHA | TLV 필드에 인증 정보 삽입 | 높음 |
| **OSPFv3** | **IPsec AH/ESP** | IPv6 표준 보안 기술 활용 | **최고** |

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

### 기술사적 판단 (실무 대형 ISP 연동 보안 강화 전략)

**상황**: 해외 ISP와 신규 BGP 피어링(Peering)을 맺으려 한다. 최근 인접 국가에서 BGP 하이재킹 사고가 잦아 강력한 보안 대책이 필요하다.
**판단 및 설정 로직**:
1. **MD5 Signature 적용**: 모든 BGP 세션에 **MD5 인증**을 필수로 설정한다. (Neighbor Password 설정).
2. **GTSM 활성화**: 직접 연결된 이웃이므로 **`ttl-security`** 기능을 켜서 원격지 공격자의 세션 주입을 물리적으로 차단한다.
3. **Prefix List 필터링**: 인증뿐만 아니라, 상대방이 "나는 네이버 주소를 가졌어"라고 뻥을 치지 못하도록 실제 보유한 IP 대역만 받도록 **화이트리스트(Prefix-list)**를 엄격히 적용한다.
4. **결과**: 가짜 경로 주입으로 인한 트래픽 유실 사고 원천 봉쇄.

### 주의사항 및 안티패턴
- **Plain-text 인증 사용**: 비밀번호를 암호화하지 않고 보내는 방식은 스니핑에 그대로 노출된다. "안 하는 것보다야 낫겠지"라는 생각은 위험한 **안티패턴**이다. 반드시 **MD5나 SHA 방식**을 사용해야 한다.
- **CPU 부하 고려**: 너무 복잡한 인증 알고리즘을 수만 개의 패킷에 적용하면 라우터 CPU가 비명을 지른다. 하드웨어 가속이 지원되는 알고리즘을 선택해야 한다.

## Ⅴ. 미래 전망 및 결론

### 결론: 신뢰할 수 없는 인터넷을 지탱하는 기술
라우팅 인증은 "누구의 말을 믿을 것인가"에 대한 기술적 답변이다. 인터넷이 거대해질수록 보이지 않는 신뢰의 끈(인증)을 튼튼하게 유지하는 것이 네트워크 엔지니어의 핵심 과업이다.

### 미래 전망
앞으로는 **[935. RPKI](./935_rpki_security.md)**가 라우팅 인증의 최종 진화 형태가 될 것이다. 라우터 간 1:1 약속을 넘어, 공인된 기관(KISA 등)으로부터 "이 IP는 내 것"이라는 디지털 인증서를 발급받아 전 세계에 공표하는 방식이다. 또한, **[시맨틱 통신](./923_semantic_communication.md)** 기술이 적용되어, 경로의 수치뿐만 아니라 전송되는 트래픽의 '의도'를 파악해 비정상적인 우회 경로를 AI가 실시간 차단하는 지능형 방어망으로 발전할 전망이다.

## 👶 어린이를 위한 3줄 비유 설명
1. **라우팅 인증이 뭔가요?**: 지도 앱(라우터)이 친구들에게 "이 길이 빨라!"라고 알려줄 때, 진짜 믿을 수 있는 지도인지 '정품 도장'을 찍어서 보여주는 거예요.
2. **왜 도장을 찍나요?**: 장난꾸러기 악당이 가짜 지도를 만들어서 친구들을 엉뚱한 길로 유인하는 걸 막기 위해서예요.
3. **어떤 게 좋나요?**: 도장이 찍힌 진짜 지도만 따라가면 길을 잃지 않고 목적지까지 안전하고 빠르게 도착할 수 있답니다!
