# 935. RPKI (Resource Public Key Infrastructure) - 인터넷 주소의 소유권을 보증하는 디지털 신분증

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 인터넷의 핵심 경로 프로토콜인 BGP의 취약점을 보완하기 위해, **특정 IP 대역(Resource)을 어떤 자율 시스템(AS)이 소유하고 있는지를 공인된 기관(RIR 등)이 디지털 인증서로 증명하는 보안 프레임워크**다.
> 2. **혁신**: 경로 정보를 광고할 때 소유권을 입증하는 **ROA(Route Origin Authorization)**를 생성하고, 수신 측 라우터가 이 정보를 **ROV(Route Origin Validation)**를 통해 검증함으로써 BGP 하이재킹과 오설정(Misconfiguration)을 원천 차단한다.
> 3. **가치**: "누구나 자신이 주인이라고 주장할 수 있었던" 기존 BGP의 구조적 불신을 해결하여, 전 세계 인터넷 라우팅 테이블의 신뢰성을 확보하고 사이버 테러와 대규모 통신 장애를 방지한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
RPKI는 인터넷 주소(IP)에 대한 '등기부등본' 시스템이다. "이 IP 주소는 내가 주인이야!"라고 외치는 라우터들에게 "진짜 주인이 맞는지 국가 기관이 발급한 인증서를 보여줘"라고 확인하는 절차다.

### 💡 비유
- **인증 없는 인터넷 (가짜 명함 시대)**: 길거리에서 누군가 "내가 이건희 회장 비서야, 나한테 돈을 맡겨!"라고 하면 사람들이 믿고 돈을 주는 것과 같다. 나중에 알고 보니 사기꾼이라도 돈을 되찾을 길이 없다. (BGP 하이재킹).
- **RPKI (신분증 확인 시대)**: 누군가 비서라고 주장하면, 즉시 경찰청(RIR 기관)에 전화를 걸어 **'공인 신분증(디지털 서명)'**이 진짜인지 확인하는 것이다. 신분증이 가짜이거나 없으면 아무도 그 사람의 말을 듣지 않는다. 돈(데이터)은 진짜 주인에게만 안전하게 전달된다.

### 등장 배경: 'The Internet is Broken'
BGP는 효율성을 위해 '가장 짧은 경로'라고 주장하면 무조건 믿는 속성이 있다. 2008년 파키스탄이 전 세계 유튜브 트래픽을 가로챈 사건, 2018년 구글 트래픽이 나이지리아와 러시아로 우회된 사건 등은 인터넷의 구조적 신뢰 한계를 드러냈다. "말(Route)이 아닌 서류(PKI)를 믿자"는 국제적 합의가 RPKI를 탄생시켰다.

## II. RPKI 아키텍처 및 작동 원리 (Deep Dive)

### RPKI 검증 프로세스 (ASCII Diagram)

```ascii
[ Regional Internet Registry (RIR) ]  <─── (Trust Anchor)
          │
          ▼ (1) Issue ROA (디지털 인증서 발급)
[ IP Resource Owner (AS 12345) ]
          │
          ▼ (2) BGP Update (IP: 1.1.1.0/24, AS: 12345)
┌────────────────────────────────────────────────────────┐
│ [ Internet BGP Network ]                               │
└──────────────────────────┬─────────────────────────────┘
                           ▼ (3) Receive Route
[ Neighbor Router (AS 67890) ]
          │
          ├────────────────▶ [ Cache Server ] (RPKI Cache)
          │  (4) ROV Check   - List of Valid ROAs
          │ <───────────────┘
          │
          ▼ (5) Decision
- [ VALID ]   : 주인 맞음 ──▶ 라우팅 허용
- [ INVALID ] : 가짜 주인 ──▶ 차단 / 우선순위 강하
- [ NOT FOUND ]: 등록 안 됨 ──▶ 정책에 따라 처리

* 핵심 개념:
1. ROA: [IP Prefix, AS Number, Max Length]가 담긴 서명된 파일.
2. ROV: 도착한 BGP 정보를 ROA 리스트와 대조하는 검증 과정.
```

### 3대 핵심 구성 기술

#### ① ROA (Route Origin Authorization)
IP 소유자가 "내 주소(1.1.1.0/24)는 반드시 내 번호(AS 12345)에서만 시작된다"고 명시하고 서명한 문서다. RPKI의 가장 핵심적인 데이터다.

#### ② TAL (Trust Anchor Locator)
누가 이 신분증을 발급했는지 확인하는 '뿌리' 정보다. 전 세계 5대 대륙별 IP 관리 기관(APNIC, RIPE, ARIN 등)이 신뢰의 앵커가 된다.

#### ③ RTR (RPKI-to-Router) 프로토콜
라우터는 직접 복잡한 암호 연산을 하기엔 힘이 딸린다. 그래서 중간에 **캐시 서버**를 두어 인증 정보를 모아두고, 라우터는 이 서버와 가벼운 RTR 프로토콜로 대화하며 검증 결과만 받아본다.

## Ⅲ. 융합 비교 및 다각도 분석

### 일반 BGP 필터링 vs RPKI 비교 (Table)

| 구분 | Prefix List (전통 방식) | **RPKI (현대적 방식)** |
|:---:|:---|:---|
| **기반 원리** | 관리자의 수동 입력 (Static) | **디지털 인증서 기반 (Dynamic)** |
| **정확도** | 관리자의 실수 가능성 높음 | **수학적으로 증명된 소유권** |
| **유지보수** | 일일이 수정해야 함 (노동 집약) | **중앙 기관 정보 자동 업데이트** |
| **방어 범위** | 아는 상대만 방어 가능 | **전 세계 모든 가짜 경로 방어** |
| **비유** | **지인 명단 대조** | **공인인증서 확인** |

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

### 기술사적 판단 (실무 국가 기간망 라우팅 보안 고도화 전략)

**상황**: 국가 주요 행정망 라우터에서 해외로부터의 BGP 하이재킹 공격을 원천 차단하라는 지침이 내려왔다.
**판단 및 구축 전략**:
1. **인프라 구축**: 데이터센터 내부에 **RPKI 캐시 서버(예: Routinator, Krill)**를 이중화하여 설치한다.
2. **ROA 등록**: 기관이 보유한 모든 IP 대역에 대해 관리 기관(KISA/APNIC) 홈페이지에서 **ROA를 발행**한다. (최대 길이를 엄격히 설정하여 Sub-prefix 하이재킹 방지).
3. **ROV 정책 적용**: 라우터에 **'Invalid = Reject'** 정책을 적용한다. 인증서 정보와 맞지 않는 가짜 경로는 라우팅 테이블에 발도 못 붙이게 차단한다.
4. **결과**: 인접 국가는 물론 지구 반대편에서 발생하는 모든 BGP 하이재킹 시도로부터 국가 행정망 트래픽을 물리적으로 보호.

### 안티패턴 및 고려사항
- **Not Found 처리 주의**: RPKI에 등록되지 않은(Unknown) 정보가 아직 많다. 만약 "Unknown"까지 모두 차단해버리면, 아직 RPKI를 안 쓰는 선량한 사이트들까지 접속이 안 되는 **'Black-out'**이 발생한다. 반드시 **Invalid(명백한 가짜)**만 차단하고 **Not Found**는 통과시키는 단계적 적용이 필요하다.

## Ⅴ. 미래 전망 및 결론

### 결론: 거짓말할 수 없는 인터넷의 완성
RPKI는 인터넷의 가장 고질적인 약점인 '출처 불분명'을 해결했다. 이는 전 세계 네트워크 엔지니어들이 30년간 꿈꿔온 "신뢰할 수 있는 인터넷"으로 가는 가장 거대한 진보 중 하나다.

### 미래 전망
앞으로는 **'BGPsec'**으로 진화할 것이다. 현재 RPKI는 '출발지'만 보증하지만, BGPsec은 패킷이 거쳐 가는 **'모든 경로(Path)'**를 디지털 서명으로 보증한다. 또한, **[블록체인](./919_dlt_concept.md)** 기술과 결합하여, 특정 기관에 의존하지 않는 '완전 탈중앙화 IP 소유권 증명 체계'에 대한 연구가 활발히 진행되어 더욱 견고한 인터넷 생태계를 구축할 전망이다.
