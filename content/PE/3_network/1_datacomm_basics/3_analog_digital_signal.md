# 아날로그 신호 vs 디지털 신호

## 정의

**아날로그 신호(Analog Signal)**는 시간에 따라 연속적으로 변화하는 물리량(전압, 전류, 주파수 등)으로 정보를 표현하는 신호 형태이며, **증폭기(Amplifier)**를 통해 신호를 강화한다. 반면 **디지털 신호(Digital Signal)**는 이산적인 값(0과 1)으로 정보를 표현하는 신호 형태이며, 전송 과정에서 신호 감쇠 시 **리피터(Repeater)**를 통해 신호를 재생성한다.

```
+------------------------------------------------------------------+
|              아날로그 신호 vs 디지털 신호 비교                       |
+------------------------------------------------------------------+

    [아날로그 신호 - 연속적]

    진폭
      ^
      |      * * *
      |    *       *
      |   *         *
      |  *           *
      | *             *
      |*               *
      +-------------------> 시간

    특징: 연속적인 파형, 무한한 값 표현
    증폭: Amplifier (잡음도 함께 증폭)


    [디지털 신호 - 이산적]

    진폭
      ^  ___     ___     ___
      | |   |   |   |   |   |
    1 | |   |   |   |   |   |
      | |   |   |   |   |   |
    0 +-+---+---+---+---+---+---> 시간
        |   |   |   |   |
        0   1   0   1   0

    특징: 이산적인 단계, 유한한 값 (0, 1)
    재생: Repeater (신호 재생성, 잡음 제거)
```

---

## 핵심 개념

### 1. 아날로그 신호 (Analog Signal)

**정의**: 시간과 진폭이 모두 연속적인 신호로, 자연계의 물리적 현상을 직접 표현한다. 전압, 전류, 주파수 등의 연속적인 변화로 정보를 전달한다.

```
+================================================================+
|                    아날로그 신호 특성                              |
+================================================================+

    아날로그 신호의 수학적 표현:

    s(t) = A * sin(2πft + φ)

    where:
    - A: 진폭 (Amplitude)
    - f: 주파수 (Frequency)
    - t: 시간 (Time)
    - φ: 위상 (Phase)

    아날로그 신호 예시:
    +--------------------------------------------------+
    |                                                  |
    |     자연계 신호        |     아날로그 표현        |
    |                                                  |
    |     음성 (목소리)      |     마이크 -> 전압 변화  |
    |     온도             |     센서 -> 저항 변화     |
    |     빛의 밝기         |     포토다이오드 -> 전류  |
    |     압력             |     압력 센서 -> 전압     |
    |                                                  |
    +--------------------------------------------------+
```

#### 아날로그 신호의 특성

```
    장점:
    +------------------------------------------+
    | 1. 자연스러운 표현                        |
    |    - 자연계 현상을 왜곡 없이 표현          |
    |    - 연속적인 정보 처리에 적합             |
    +------------------------------------------+
    | 2. 높은 해상도                            |
    |    - 이론적으로 무한한 해상도              |
    |    - 정보 손실 없는 이론적 가능성          |
    +------------------------------------------+
    | 3. 단순한 처리                            |
    |    - 복잡한 변환 없이 직접 전송 가능        |
    +------------------------------------------+

    단점:
    +------------------------------------------+
    | 1. 잡음에 취약                            |
    |    - 외부 간섭에 의한 신호 왜곡            |
    |    - 전송 거리 증가 시 품질 저하           |
    +------------------------------------------+
    | 2. 증폭 시 잡음 누적                       |
    |    - 증폭기가 신호와 잡음을 함께 증폭       |
    |    - 다단 증폭 시 품질 급격히 저하          |
    +------------------------------------------+
    | 3. 보안 취약                              |
    |    - 도청이 쉬움                          |
    |    - 암호화가 어려움                      |
    +------------------------------------------+
    | 4. 저장 및 처리의 한계                     |
    |    - 정확한 복제 어려움                    |
    |    - 디지털 처리 불가능                    |
    +------------------------------------------+
```

### 2. 디지털 신호 (Digital Signal)

**정의**: 시간과 진폭이 모두 이산적인 신호로, 0과 1의 두 가지 상태(또는 제한된 수의 상태)로 정보를 표현한다. 컴퓨터 및 디지털 통신의 기본이 된다.

```
+================================================================+
|                    디지털 신호 특성                                |
+================================================================+

    디지털 신호 표현:

    이진 디지털 신호:
    진폭
      ^
    V |  +-----+     +-----+  +--+     +-----+
      |  |     |     |     |  |  |     |     |
    0 +--+     +-----+     +--+  +-----+     +----->
      |                                            시간
         1     0     1     0   0   1     0

    다레벨 디지털 신호 (예: 4-PAM):
    진폭
      ^
    +3 |      +--+
       |      |  |
    +1 |  +--+  +--+  +--+
       |  |        |  |
    -1 |--+        +--+  +--+
       |
    -3 |                +--+
       +-------------------------> 시간
           01  11  01  00  10

    디지털 신호의 정보 표현:
    +------------------------------------------+
    | 비트 수 (n) | 표현 가능한 값 | 예시       |
    +------------+----------------+------------+
    | 1 bit      | 2 (0, 1)       | 이진       |
    | 2 bits     | 4 (00~11)      | 4-PAM      |
    | 8 bits     | 256            | ASCII      |
    | 16 bits    | 65,536         | CD 오디오  |
    | 24 bits    | 16,777,216     | True Color |
    +------------+----------------+------------+
```

#### 디지털 신호의 특성

```
    장점:
    +------------------------------------------+
    | 1. 잡음 내성 (Noise Immunity)             |
    |    - 임계값 이상/이하로 판정              |
    |    - 작은 잡음은 무시됨                   |
    +------------------------------------------+
    | 2. 재생성 가능 (Regeneration)             |
    |    - 리피터에서 원본 신호 재생성           |
    |    - 장거리 전송에서도 품질 유지           |
    +------------------------------------------+
    | 3. 오류 검출 및 정정                       |
    |    - 패리티, CRC, FEC 등 활용             |
    |    - 전송 오류 복구 가능                   |
    +------------------------------------------+
    | 4. 보안성                                 |
    |    - 암호화 용이                          |
    |    - 인증 및 접근 제어 가능               |
    +------------------------------------------+
    | 5. 처리 용이성                            |
    |    - 컴퓨터로 처리 가능                   |
    |    - 압축, 변환, 저장 용이                |
    +------------------------------------------+
    | 6. 정확한 복제                            |
    |    - 무손실 복사 가능                     |
    |    - 원본과 동일한 복제품                  |
    +------------------------------------------+

    단점:
    +------------------------------------------+
    | 1. 양자화 오류 (Quantization Error)       |
    |    - 연속 신호를 이산화 시 정보 손실       |
    |    - 해상도에 따른 품질 제한              |
    +------------------------------------------+
    | 2. 높은 대역폭 요구                        |
    |    - 동일 정보 전송 시 더 많은 대역폭      |
    |    - 아날로그 대비 2~10배 대역폭 필요      |
    +------------------------------------------+
    | 3. 동기화 필요                            |
    |    - 클럭 동기화 필수                     |
    |    - 동기 오류 시 데이터 손실             |
    +------------------------------------------+
```

### 3. 증폭기 (Amplifier) vs 리피터 (Repeater)

```
+================================================================+
|              증폭기 vs 리피터 비교                                 |
+================================================================+

[증폭기 (Amplifier) - 아날로그 신호용]

    입력 신호              출력 신호
    (약함 + 잡음)          (강함 + 잡음 증폭)

    +-------+     +-------+     +-------+
    | 입력  | --> | 증폭기 | --> | 출력  |
    | (신호 |     |       |     | (신호 |
    |  +잡음|     | Gain=G|     |  +잡음|
    +-------+     +-------+     |  증폭)|
                              +-------+

    문제점:
    - 신호와 잡음을 모두 증폭
    - 다단 증폭 시 S/N 비 악화

    수식:
    출력 = Gain x (입력 신호 + 잡음)
         = Gain x 입력 신호 + Gain x 잡음
         (신호와 잡음이 모두 같은 비율로 증폭)


[리피터 (Repeater) - 디지털 신호용]

    입력 신호              재생성된 신호
    (감쇠 + 잡음)          (깨끗한 신호)

    +-------+     +-------+     +-------+
    | 입력  | --> |리피터 | --> | 출력  |
    | (감쇠 |     |       |     | (완전 |
    |  +잡음|     |Regen- |     |  복원 |
    +-------+     | erate |     |  신호)|
                  +-------+     +-------+

    동작 과정:
    1. 수신된 신호 샘플링
    2. 임계값 기준으로 0/1 판정
    3. 새로운 깨끗한 신호 생성
    4. 재전송

    장점:
    - 잡음이 제거된 신호 재생성
    - 다단 연결 시에도 품질 유지
    - 장거리 전송 가능
```

#### 증폭기와 리피터의 성능 비교

```
    아날로그 전송 (증폭기 사용):

    송신기 ---[100km]--- 증폭기1 ---[100km]--- 증폭기2 ---[100km]--- 수신기

    S/N 비 변화 (각 구간 3dB 감쇠, 증폭기 이득 3dB):
    - 송신기 출력: S/N = 40 dB
    - 증폭기1 후: S/N = 37 dB (잡음 누적)
    - 증폭기2 후: S/N = 34 dB
    - 수신기: S/N = 31 dB (품질 저하)


    디지털 전송 (리피터 사용):

    송신기 ---[100km]--- 리피터1 ---[100km]--- 리피터2 ---[100km]--- 수신기

    신호 품질 변화:
    - 송신기 출력: 깨끗한 신호
    - 리피터1 후: 깨끗한 신호 (재생성)
    - 리피터2 후: 깨끗한 신호 (재생성)
    - 수신기: 깨끗한 신호 (품질 유지)

    결론: 디지털 전송은 거리와 무관하게 신호 품질 유지 가능
```

## 상세 설명

### 1. 아날로그-디지털 변환 (ADC)

```
+================================================================+
|           아날로그-디지털 변환 (Analog-to-Digital Conversion)       |
+================================================================+

    ADC 변환 과정:

    1. 표본화 (Sampling)
       - 연속적인 시간 신호를 이산적인 시간 신호로 변환
       - Nyquist 정리: fs >= 2 x fmax

    2. 양자화 (Quantization)
       - 연속적인 진폭을 이산적인 레벨로 변환
       - 양자화 오류 (Quantization Error) 발생

    3. 부호화 (Encoding)
       - 양자화된 값을 이진 코드로 변환


    [변환 과정 시각화]

    아날로그 신호
    ^
    |    * * *
    |  *       *
    | *         *
    |*           *
    +--------------> 시간 (연속)

           |
           v (Sampling)

    표본화된 신호
    ^
    |    *   *
    |  *       *
    | *         *
    |*           *
    +--------------> 시간 (이산)

           |
           v (Quantization)

    양자화된 신호
    ^
    |    o   o
    |  o       o
    | o         o
    |o           o
    +--------------> 시간 (이산, 레벨)

           |
           v (Encoding)

    디지털 데이터: 1011 0110 1100 0011
```

#### 표본화 (Sampling) 상세

```
    Nyquist-Shannon 표본화 정리:

    "대역폭이 B인 신호는 최소 2B의 샘플링 주파수로
     완벽하게 복원될 수 있다."

    fs >= 2 x fmax

    예시:
    +-------------------+-------------------+-------------------+
    |     신호 유형      |   최대 주파수     |   최소 샘플링    |
    +-------------------+-------------------+-------------------+
    | 음성 (전화)        | 3.4 kHz          | 8 kHz            |
    | 오디오 (CD)        | 20 kHz           | 44.1 kHz         |
    | 오디오 (DVD)       | 20 kHz           | 48 kHz           |
    | 오디오 (Hi-Res)    | 48 kHz           | 96/192 kHz       |
    +-------------------+-------------------+-------------------+

    언더샘플링 (Undersampling) 문제:
    - fs < 2 x fmax
    - 에일리어싱 (Aliasing) 발생
    - 고주파 성분이 저주파로 나타남

    에일리어싱 방지:
    - Anti-Aliasing Filter (LPF) 사용
    - 샘플링 전 fmax 이상 주파수 제거
```

#### 양자화 (Quantization) 상세

```
    양자화 레벨과 비트 수:

    레벨 수 = 2^n

    +----------+-------------+-----------------+
    | 비트 수  | 레벨 수     | 동적 범위 (dB)  |
    +----------+-------------+-----------------+
    | 8 bits   | 256         | 48 dB          |
    | 12 bits  | 4,096       | 72 dB          |
    | 16 bits  | 65,536      | 96 dB (CD)     |
    | 24 bits  | 16,777,216  | 144 dB         |
    +----------+-------------+-----------------+

    양자화 오류 (Quantization Error):
    - 양자화 오류 <= ±(LSB/2)
    - 양자화 잡음 = (LSB^2) / 12

    신호 대 양자화 잡음비 (SQNR):
    SQNR(dB) = 6.02 x n + 1.76 dB

    예: 16-bit 오디오
    SQNR = 6.02 x 16 + 1.76 = 98.08 dB
```

### 2. 디지털-아날로그 변환 (DAC)

```
+================================================================+
|           디지털-아날로그 변환 (Digital-to-Analog Conversion)       |
+================================================================+

    DAC 변환 과정:

    1. 복호화 (Decoding)
       - 이진 코드를 양자화 레벨로 변환

    2. 재구성 (Reconstruction)
       - 이산 신호를 연속 신호로 변환
       - 로우패스 필터링


    [DAC 구조 예시: R-2R 래더]

              Vref
               |
               R
               +----+----+----+
               |    |    |    |
               2R   R    R    R
               |    |    |    |
    D3 --------+    |    |    |
                    2R   R    R
                    |    |    |
    D2 -------------+    |    |
                         2R   R
                         |    |
    D1 ------------------+    |
                              2R
                              |
    D0 -----------------------+
                              |
                              +---- Vout
                              |
                             2R
                              |
                             GND

    특징:
    - 정밀한 저항만 필요 (R과 2R)
    - 비트 수 확장 용이
    - 고속 변환 가능
```

### 3. 디지털 신호의 전송 품질

```
+================================================================+
|               디지털 신호 전송 품질 지표                            |
+================================================================+

    1. 비트 오류율 (Bit Error Rate, BER)

    BER = 오류 비트 수 / 총 전송 비트 수

    +-------------------+-------------------------------------------+
    |     시스템         |      목표 BER                              |
    +-------------------+-------------------------------------------+
    | 음성 전화          | 10^-3 (0.1%)                              |
    | 데이터 통신        | 10^-6 (1ppm)                              |
    | 광섬유 통신        | 10^-12 (1ppt)                             |
    | 실시간 비디오      | 10^-9                                    |
    +-------------------+-------------------------------------------+


    2. 에이 눈도 (Eye Diagram)

    진폭
      ^
      |   +---+       +---+
      |   |   |       |   |
    1 |   |   |       |   |
      |  /     \     /     \
      | /       \   /       \
    0 |/         \ /         \
      +--------------------------> 시간
        닫힘        열림

    눈이 클수록:
    - 낮은 지터 (Jitter)
    - 낮은 잡음
    - 좋은 신호 품질

    눈이 닫힐수록:
    - 높은 지터
    - 높은 잡음
    - 높은 BER


    3. 신호 대 잡음비 (Signal-to-Noise Ratio, SNR)

    SNR(dB) = 10 x log10(Psignal / Pnoise)

    +-------------------+-------------------------------------------+
    |     SNR (dB)       |      통신 품질                             |
    +-------------------+-------------------------------------------+
    | 10 dB             | 최소 가청 수준                             |
    | 20 dB             | 낮은 품질                                  |
    | 30 dB             | 보통 품질                                  |
    | 40 dB             | 양호한 품질                                |
    | 50 dB             | 우수한 품질                                |
    | 60 dB+            | 매우 우수한 품질                           |
    +-------------------+-------------------------------------------+
```

### 4. 아날로그 전송 vs 디지털 전송

```
+================================================================+
|              아날로그 전송 vs 디지털 전송 비교                       |
+================================================================+

+-------------------+-------------------------------------------+
|     구분          |    아날로그 전송      |    디지털 전송     |
+-------------------+-------------------------------------------+
| 신호 형태         | 연속파               | 펄스파            |
+-------------------+-------------------------------------------+
| 대역폭 효율       | 높음                 | 낮음              |
+-------------------+-------------------------------------------+
| 잡음 내성         | 낮음                 | 높음              |
+-------------------+-------------------------------------------+
| 장거리 전송       | 품질 저하            | 품질 유지         |
+-------------------+-------------------------------------------+
| 신호 강화 방식    | 증폭기               | 리피터            |
+-------------------+-------------------------------------------+
| 오류 제어         | 어려움               | 용이              |
+-------------------+-------------------------------------------+
| 보안              | 취약                 | 강력 (암호화)      |
+-------------------+-------------------------------------------+
| 멀티미디어 통합   | 어려움               | 용이              |
+-------------------+-------------------------------------------+
| 장비 복잡도       | 단순                 | 복잡              |
+-------------------+-------------------------------------------+
| 비용              | 저가                 | 고가 (과거)       |
|                   |                      | 저렴 (현재)       |
+-------------------+-------------------------------------------+
| 전력 소모         | 적음                 | 많음              |
+-------------------+-------------------------------------------+
| 대표 예시         | AM/FM 라디오         | 인터넷            |
|                   | 아날로그 TV          | 디지털 TV         |
|                   | 전화망 (과거)        | VoIP              |
+-------------------+-------------------------------------------+
```

### 5. 현대적 신호 처리 아키텍처

```
+================================================================+
|              소프트웨어 정의 라디오 (SDR) 구조                       |
+================================================================+

    아날로그와 디지털의 결합:

    안테나
      |
      v
    +---------------+
    | RF 프론트엔드  |   <- 아날로그 영역
    | (LNA, 필터)   |
    +-------+-------+
            |
            v (ADC)
    +---------------+
    |    ADC        |   <- 아날로그 -> 디지털
    | (고속 샘플링)  |
    +-------+-------+
            |
            v
    +---------------+
    |  DSP / FPGA   |   <- 디지털 처리
    |  - 필터링      |
    |  - 변복조      |
    |  - 코딩        |
    +-------+-------+
            |
            v
    +---------------+
    |  소프트웨어    |   <- 응용 계층
    |  (프로토콜)    |
    +---------------+

    장점:
    - 하드웨어 변경 없이 소프트웨어 업그레이드
    - 다양한 통신 표준 지원 가능
    - 유연성과 확장성
```

### 1. 오디오 시스템 설계

```
    [재생]
    저장 -> DSP -> DAC -> 증폭기 -> 스피커

    CD 오디오 사양:

    |     파라미터       |      값                                   |

    | 채널 수            | 2 (스테레오)                              |
    | 비트 전송률        | 1.411 Mbps                                |
    | 주파수 범위        | 20 Hz - 20 kHz                            |
    | 동적 범위          | 96 dB                                     |
    +-------------------+-------------------------------------------+

    Hi-Res 오디오 사양:
    +-------------------+-------------------------------------------+
    |     포맷           |      사양                                 |
    +-------------------+-------------------------------------------+
    | 24-bit/96kHz      | 동적 범위 144 dB, 48kHz 대역폭            |
    | 24-bit/192kHz     | 동적 범위 144 dB, 96kHz 대역폭            |
    | DSD64             | 2.8224 MHz 1-bit, 100kHz+ 대역폭          |
    +-------------------+-------------------------------------------+
```

### 2. 통신 시스템 설계

```
+------------------------------------------------------------------+
|               디지털 통신 시스템 블록 다이어그램                      |
+------------------------------------------------------------------+

    송신기
    +----------------------------------------------------------+
    |  정보원  -->  [압축]  -->  [암호화]  -->  [채널 부호화]    |
    |                                            |             |
    |  [변조]  <--  [펄스 성형]  <--  [인터리빙] <+             |
    +------------------+---------------------------------------+
                       |
                       v (전송 매체)
    +------------------+---------------------------------------+
    |  [등화]  -->  [복조]  -->  [디인터리빙]  -->  [채널 복호화] |
    |                                            |             |
    |  정보목적지  <--  [복호화]  <--  [복원]  <--+             |
    +----------------------------------------------------------+
    수신기

    각 단계의 기능:
    - 압축: 데이터 양 감소
    - 암호화: 보안 제공
    - 채널 부호화: 오류 정정 코드 추가
    - 인터리빙: 버스트 오류 분산
    - 펄스 성형: 대역폭 제한
    - 변조: 반송파에 실림
    - 등화: 채널 왜곡 보정
```

### 3. 네트워크 리피터/증폭기 배치

```
    광섬유 통신 시스템:

    +--------+     +--------+     +--------+     +--------+
    | 송신기 |====>| 광증폭기 |====>| 광증폭기 |====>| 수신기 |
    | (E/O)  |     | (EDFA) |     | (EDFA) |     | (O/E)  |
    +--------+     +--------+     +--------+     +--------+
         |              |              |              |
       0 km          80 km         160 km        240 km

    EDFA (Erbium-Doped Fiber Amplifier):
    - 광 신호를 직접 증폭 (O/E 변환 없이)
    - 1550nm 대역에서 약 30dB 이득
    - 80-100km 간격으로 배치

    전기적 리피터 vs 광 증폭기:
    +-------------------+-------------------------------------------+
    |     구분          |    전기적 리피터     |    광 증폭기       |
    +-------------------+-------------------------------------------+
    | 동작 방식         | O/E -> 재생 -> E/O   | 직접 광 증폭      |
    +-------------------+-------------------------------------------+
    | 처리 속도         | 제한적               | 매우 빠름         |
    +-------------------+-------------------------------------------+
    | 투명성            | 프로토콜 종속        | 투명 (어떤 포맷도)|
    +-------------------+-------------------------------------------+
    | 비용              | 높음                 | 상대적 저렴       |
    +-------------------+-------------------------------------------+
```

## 기술사 시험 포인트

### 1. 핵심 암기 사항

```
+------------------------------------------------------------------+
|              아날로그/디지털 신호 핵심 정리                           |
+------------------------------------------------------------------+

1. 아날로그 신호 특성:
   - 연속적 (시간, 진폭 모두)
   - 증폭기 (Amplifier) 사용
   - 잡음 누적 문제
   - 장거리 전송 시 품질 저하

2. 디지털 신호 특성:
   - 이산적 (0, 1)
   - 리피터 (Repeater) 사용
   - 잡음 제거 가능 (재생성)
   - 장거리 전송 품질 유지

3. ADC 변환 과정:
   - 표본화 (Sampling): fs >= 2 x fmax (Nyquist)
   - 양자화 (Quantization): 연속 -> 이산 레벨
   - 부호화 (Encoding): 이진 코드 변환

4. 양자화 오류:
   - SQNR = 6.02n + 1.76 dB
   - n비트당 약 6dB 동적 범위 향상

5. 증폭기 vs 리피터:
   - 증폭기: 신호+잡음 모두 증폭
   - 리피터: 신호만 재생성 (잡음 제거)
```

### 2. 빈출 문제 유형

```
1. 아날로그 vs 디지털 비교
   Q: 아날로그 신호와 디지털 신호의 차이점을 설명하고,
      각각의 장단점을 기술하시오.

2. ADC 변환 과정
   Q: 아날로그 신호를 디지털 신호로 변환하는 과정을 설명하시오.
   A: 표본화 -> 양자화 -> 부호화

3. Nyquist 정리
   Q: 20kHz까지의 오디오 신호를 디지털로 변환하기 위한
      최소 샘플링 주파수는?
   A: fs >= 2 x 20kHz = 40kHz (실제 CD: 44.1kHz)

4. 증폭기 vs 리피터
   Q: 장거리 전송에서 디지털 신호가 유리한 이유를 설명하시오.
   A: 리피터에 의한 신호 재생성으로 잡음 제거 가능

5. 양자화 비트 수 계산
   Q: 96dB 동적 범위를 갖기 위한 최소 양자화 비트 수는?
   A: n >= (96 - 1.76) / 6.02 = 15.6 -> 16비트
```

### 3. 기술사 답안 작성 요령

```
[문제] 아날로그 신호와 디지털 신호의 특성을 비교 설명하고,
      통신 시스템에서 디지털 신호가 선호되는 이유를 기술하시오.

[답안 구성]
1. 개요
   - 신호의 정의 및 분류
   - 아날로그/디지털 신호의 정의

2. 아날로그 신호 특성
   가. 정의 및 수학적 표현
   나. 장점 (자연스러운 표현, 높은 해상도)
   다. 단점 (잡음 취약, 증폭 시 잡음 누적)

3. 디지털 신호 특성
   가. 정의 및 표현
   나. 장점 (잡음 내성, 재생성 가능, 오류 제어)
   다. 단점 (양자화 오류, 높은 대역폭)

4. 증폭기 vs 리피터 (비교표)
   - 아날로그: 증폭기 (신호+잡음 증폭)
   - 디지털: 리피터 (신호 재생성)

5. 디지털 신호 선호 이유
   가. 잡음 내성 및 신호 재생성
   나. 오류 검출 및 정정
   다. 보안 (암호화)
   라. 멀티미디어 통합
   마. 장거리 전송 품질 유지

6. ADC/DAC 변환 과정 (간략)
   - Sampling, Quantization, Encoding

7. 결론
   - 현대 통신의 디지털화 추세
```
