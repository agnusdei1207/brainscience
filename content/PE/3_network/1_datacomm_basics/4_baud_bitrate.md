# 보드레이트 (Baud Rate) vs 비트레이트 (Bit Rate)

## 정의

**보드레이트(Baud Rate, 변조 속도)**는 1초당 전송되는 신호 변화(심볼)의 횟수를 의미하며, 단위는 **Baud**를 사용한다. 반면 **비트레이트(Bit Rate, 전송 속도)**는 1초당 전송되는 비트 수를 의미하며, 단위는 **bps(bits per second)**를 사용한다. 하나의 심볼이 여러 비트를 표현할 수 있으므로 비트레이트는 보드레이트보다 클 수 있다.

```
+------------------------------------------------------------------+
|              보드레이트 vs 비트레이트 개념도                          |
+------------------------------------------------------------------+

    [2진 변조 (1심볼 = 1비트)]

    시간 ->
    |----|----|----|----|----|
    [ 1 ][ 0 ][ 1 ][ 1 ][ 0 ]

    보드레이트 = 비트레이트
    1 Baud = 1 bps


    [4진 변조 (1심볼 = 2비트)]

    시간 ->
    |----|----|----|----|----|
    [11][01][10][00][11]

    보드레이트 = 1000 Baud
    비트레이트 = 2000 bps
    (2 bits per symbol)


    [16진 변조 (1심볼 = 4비트)]

    시간 ->
    |----|----|----|----|----|
    [A ][3 ][F ][0 ][7 ]
    (1010)(0011)(1111)(0000)(0111)

    보드레이트 = 1000 Baud
    비트레이트 = 4000 bps
    (4 bits per symbol)

    핵심 공식:
    비트레이트 = 보드레이트 x log2(M)
    where M = 심볼 당 상태 수 (변조 레벨)
```

---

## 핵심 개념

### 1. 보드레이트 (Baud Rate, 변조 속도)

**정의**: 1초당 신호 레벨이 변화하는 횟수, 즉 심볼(Symbol) 전송 속도를 나타낸다. J.M.E. Baudot(프랑스 통신 공학자)의 이름을 따서 명명되었다.

```
+================================================================+
|                    보드레이트 상세 설명                            |
+================================================================+

    심볼 (Symbol):
    - 디지털 통신에서 전송의 기본 단위
    - 진폭, 위상, 주파수 등으로 표현
    - 하나의 심볼이 여러 비트를 표현 가능


    보드레이트 계산:
    보드레이트 (Baud) = 심볼 수 / 시간 (초)

    예시:
    - 1초당 2400번 신호 변화 = 2400 Baud
    - 1초당 9600번 신호 변화 = 9600 Baud


    신호 레벨 변화 시각화:

    2레벨 신호 (Binary):
    진폭
      ^
    V |  +--+  +--+--+  +--+
      |  |  |  |  |  |  |  |
    0 +--+  +--+  +--+  +--+--> 시간
         1  0  1  1  0  1

    각 천이 = 1 심볼
    6 심볼 / T초 = 보드레이트


    4레벨 신호 (Quaternary):
    진폭
      ^
    +3|      +--+
      |      |  |
    +1|  +--+  +--+
      |  |        |
    -1|--+        +--+
      |
    -3|              +--+
      +-------------------+--> 시간

    각 레벨 = 1 심볼 (2비트 표현)
    00 = -3, 01 = -1, 11 = +1, 10 = +3 (Gray Code)
```

---

### 2. 비트레이트 (Bit Rate, 전송 속도)

**정의**: 1초당 전송되는 비트 수를 나타내며, 정보 전송의 실질적인 속도를 의미한다. 데이터 통신에서 가장 중요한 성능 지표이다.

```
+================================================================+
|                    비트레이트 상세 설명                            |
+================================================================+

    비트레이트 계산:
    비트레이트 (bps) = 비트 수 / 시간 (초)

    단위 환산:
    +----------+----------+----------+
    | 단위     | 비트/초  | 예시     |
    +----------+----------+----------+
    | bps      | 1        | 시리얼   |
    | Kbps     | 10^3     | 전화     |
    | Mbps     | 10^6     | 이더넷   |
    | Gbps     | 10^9     | 광섬유   |
    | Tbps     | 10^12    | 백본망   |
    +----------+----------+----------+

    비트레이트와 대역폭의 관계:
    (Nyquist 공식 - 무잡음 채널)

    C = 2 x B x log2(M)

    where:
    - C: 채널 용량 (bps)
    - B: 대역폭 (Hz)
    - M: 심볼 당 레벨 수

    (Shannon 공식 - 유잡음 채널)

    C = B x log2(1 + S/N)

    where:
    - S/N: 신호 대 잡음비
```

---

### 3. 비트레이트와 보드레이트의 관계

```
+================================================================+
|            비트레이트 = 보드레이트 x log2(M)                        |
+================================================================+

    M: 변조 레벨 수 (한 심볼로 표현 가능한 상태 수)

    +----------+------------+----------------+-------------------+
    |   M값    |  비트/심볼  |   변조 방식    |      예시         |
    +----------+------------+----------------+-------------------+
    |    2     |    1       | 이진 변조      | BPSK, OOK        |
    |    4     |    2       | 4진 변조       | QPSK, 4-ASK      |
    |    8     |    3       | 8진 변조       | 8-PSK, 8-QAM     |
    |   16     |    4       | 16진 변조      | 16-QAM           |
    |   32     |    5       | 32진 변조      | 32-QAM           |
    |   64     |    6       | 64진 변조      | 64-QAM           |
    |  256     |    8       | 256진 변조     | 256-QAM          |
    | 1024     |   10       | 1024진 변조    | 1024-QAM         |
    +----------+------------+----------------+-------------------+


    [실제 계산 예시]

    1. QPSK 변조, 1200 Baud
       - M = 4 (4상 위상 변조)
       - 비트/심볼 = log2(4) = 2
       - 비트레이트 = 1200 x 2 = 2400 bps

    2. 16-QAM 변조, 2400 Baud
       - M = 16 (16 상태)
       - 비트/심볼 = log2(16) = 4
       - 비트레이트 = 2400 x 4 = 9600 bps

    3. 64-QAM 변조, 5 MHz 대역폭
       - M = 64
       - 비트/심볼 = log2(64) = 6
       - 보드레이트 ≈ 대역폭 = 5 MBaud
       - 비트레이트 = 5M x 6 = 30 Mbps

    4. 256-QAM 변조, 8000 Baud
       - M = 256
       - 비트/심볼 = log2(256) = 8
       - 비트레이트 = 8000 x 8 = 64,000 bps = 64 Kbps
```

### 4. 변조 방식에 따른 비트/심볼 관계

```
+================================================================+
|              주요 변조 방식의 비트/심볼                             |
+================================================================+

    [ASK (Amplitude Shift Keying) - 진폭 편이 변조]

    2-ASK (OOK):
    0  1  0  1  1
    __|  |__|  |  |

    1심볼 = 1비트
    보드레이트 = 비트레이트


    [PSK (Phase Shift Keying) - 위상 편이 변조]

    BPSK (Binary PSK):
    - 2 상태 (0°, 180°)
    - 1심볼 = 1비트

    QPSK (Quadrature PSK):
    - 4 상태 (45°, 135°, 225°, 315°)
    - 1심볼 = 2비트

    8-PSK:
    - 8 상태 (45° 간격)
    - 1심볼 = 3비트


    [QAM (Quadrature Amplitude Modulation) - 직교 진폭 변조]

                    I (In-phase)
                    ^
                    |
           10     11     01     00
            x      x      x      x
            |      |      |      |
    --------+------+------+------+----> Q (Quadrature)
            |      |      |      |
            x      x      x      x
           110    111    101    100

    16-QAM: 16 점 = 4비트/심볼
    64-QAM: 64 점 = 6비트/심볼
    256-QAM: 256 점 = 8비트/심볼


    [FSK (Frequency Shift Keying) - 주파수 편이 변조]

    BFSK (Binary FSK):
    f1: 주파수 1 (비트 0)
    f2: 주파수 2 (비트 1)

    1심볼 = 1비트
    보드레이트 = 비트레이트

    4-FSK:
    f1, f2, f3, f4: 4개 주파수
    1심볼 = 2비트
```

## 상세 설명

### 1. 심볼 레이트 (Symbol Rate) = 보드레이트

```
+================================================================+
|                 심볼과 보드레이트의 이해                            |
+================================================================+

    심볼의 구조:

    +-------------------+-------------------------------------------+
    |     구성요소       |                 설명                       |
    +-------------------+-------------------------------------------+
    | 심볼 기간 (Ts)     | 한 심볼이 전송되는 시간                    |
    |                   | Ts = 1 / 보드레이트                        |
    +-------------------+-------------------------------------------+
    | 심볼 레이트 (Rs)   | 초당 심볼 수 (= 보드레이트)                |
    |                   | Rs = 1 / Ts                                |
    +-------------------+-------------------------------------------+
    | 심볼 에너지 (Es)   | 한 심볼에 포함된 에너지                    |
    |                   | Es = Pb x log2(M) (Pb = 비트 에너지)       |
    +-------------------+-------------------------------------------+

    예시:
    - 보드레이트 = 1200 Baud
    - 심볼 기간 = 1/1200 = 0.833 ms
    - 1초당 1200개 심볼 전송


    심볼 vs 비트:

    +-------------------+-------------------------------------------+
    |     구분          |     심볼 (Symbol)   |     비트 (Bit)      |
    +-------------------+-------------------------------------------+
    | 정의              | 신호 변화 단위       | 정보 최소 단위      |
    +-------------------+-------------------------------------------+
    | 물리적 의미        | 실제 전송 파형       | 논리적 데이터       |
    +-------------------+-------------------------------------------+
    | 크기              | 변조 방식에 따라     | 항상 0 또는 1      |
    |                   | 다름                 |                     |
    +-------------------+-------------------------------------------+
    | 속도 단위         | Baud                | bps                 |
    +-------------------+-------------------------------------------+
    | 관계              | 1심볼 = log2(M)비트  | 1비트 <= 1심볼     |
    +-------------------+-------------------------------------------+
```

### 2. 대역폭과 보드레이트의 관계

```
+================================================================+
|               대역폭(Bandwidth)과 보드레이트                        |
+================================================================+

    Nyquist 대역폭:
    이상적인 저역 통과 필터에서 필요한 최소 대역폭

    B = Rs / 2 = 보드레이트 / 2

    실제 필터 (Raised Cosine):
    B = Rs / 2 x (1 + α)

    where α = Roll-off Factor (0 ~ 1)

    +----------+-------------------------------------------+
    | α 값     |           설명                             |
    +----------+-------------------------------------------+
    | 0        | 이상적 (이론적 한계), 구현 불가             |
    | 0.5      | 일반적, 50% 여유 대역폭                    |
    | 1.0      | 보수적, 100% 여유 대역폭                   |
    +----------+-------------------------------------------+


    [대역폭 계산 예시]

    1. QPSK, 1 Mbps 비트레이트, α = 0.5
       - 보드레이트 = 1 Mbps / 2 = 500 kBaud
       - 대역폭 = 500k / 2 x (1 + 0.5) = 375 kHz

    2. 16-QAM, 10 Mbps 비트레이트, α = 0.25
       - 보드레이트 = 10 Mbps / 4 = 2.5 MBaud
       - 대역폭 = 2.5M / 2 x (1 + 0.25) = 1.5625 MHz

    3. 64-QAM, 100 Mbps 비트레이트, α = 0.2
       - 보드레이트 = 100 Mbps / 6 = 16.67 MBaud
       - 대역폭 = 16.67M / 2 x 1.2 = 10 MHz


    대역폭 효율 (Spectral Efficiency):
    η = 비트레이트 / 대역폭 (bps/Hz)

    +----------+------------+--------------------+
    | 변조방식 | 비트/심볼  | 대역폭 효율 (bps/Hz)|
    +----------+------------+--------------------+
    | BPSK     |    1       |     ~0.5           |
    | QPSK     |    2       |     ~1.0           |
    | 8-PSK    |    3       |     ~1.5           |
    | 16-QAM   |    4       |     ~2.0           |
    | 64-QAM   |    6       |     ~3.0           |
    | 256-QAM  |    8       |     ~4.0           |
    +----------+------------+--------------------+
```

### 3. 변조 레벨 증가에 따른 트레이드오프

```
+================================================================+
|           높은 M값의 장단점 (Trade-off)                            |
+================================================================+

    장점:
    +------------------------------------------+
    | 1. 높은 비트레이트                         |
    |    - 동일 대역폭으로 더 많은 데이터 전송    |
    |    - 스펙트럼 효율 향상                     |
    +------------------------------------------+
    | 2. 대역폭 절약                             |
    |    - 동일 비트레이트에 더 좁은 대역폭       |
    +------------------------------------------+

    단점:
    +------------------------------------------+
    | 1. 높은 SNR 요구                           |
    |    - 심볼 간 거리 감소                     |
    |    - 잡음에 취약해짐                       |
    +------------------------------------------+
    | 2. 복잡한 하드웨어                         |
    |    - 정밀한 변복조기 필요                  |
    |    - 높은 전력 소모                        |
    +------------------------------------------+
    | 3. 비선형 왜곡에 민감                       |
    |    - 진폭 변동에 취약                      |
    |    - 선형 증폭기 필요                      |
    +------------------------------------------+
    | 4. 에러율 증가                             |
    |    - 동일 SNR에서 더 높은 BER              |
    |    - 강력한 오류 정정 코드 필요             |
    +------------------------------------------+


    BER vs SNR 그래프 (개념적):

    BER
      ^
    10^-3 |   *
          |      *     (BPSK)
    10^-6 |         *
          |            *     (QPSK)
    10^-9 |               *
          |                  *     (16-QAM)
          |                     *
          +---------------------------> SNR (dB)
                10    15    20    25

    결론:
    - 낮은 SNR 환경 -> 낮은 M값 (BPSK, QPSK)
    - 높은 SNR 환경 -> 높은 M값 (64-QAM, 256-QAM)
```

### 4. 실제 통신 시스템 예시

```

    +-------------------+-------------------------------------------+

    | 라인 부호화         | 4비트 -> 5비트 -> 3레벨                   |
    +-------------------+-------------------------------------------+

    4. WiFi 802.11ac

    +-------------------+-------------------------------------------+
    |     파라미터       |      값                                   |
    +-------------------+-------------------------------------------+
    | 최대 비트레이트     | 6.933 Gbps                                |
    | 채널 대역폭         | 160 MHz                                   |
    | 변조 방식          | 256-QAM                                   |
    | MIMO              | 8 스트림                                  |
    +-------------------+-------------------------------------------+

    5. 5G NR (New Radio)

    +-------------------+-------------------------------------------+
    |     파라미터       |      값                                   |
    +-------------------+-------------------------------------------+
    | 최대 비트레이트     | 20 Gbps (이론적)                          |
    | 변조 방식          | 256-QAM (DL), 64-QAM (UL)                 |
    | 서브캐리어 간격     | 15-240 kHz                                |
    | 최대 대역폭         | 400 MHz                                   |
    +-------------------+-------------------------------------------+
```

### 5. 비트레이트와 보드레이트가 같은 경우

```
+================================================================+
|        비트레이트 = 보드레이트인 경우 (M = 2)                       |
+================================================================+

    조건: 1심볼 = 1비트 (이진 변조)

    해당 변조 방식:
    - NRZ (Non-Return-to-Zero)
    - BPSK (Binary Phase Shift Keying)
    - OOK (On-Off Keying) / ASK
    - BFSK (Binary Frequency Shift Keying)


    [NRZ 부호화 예시]

    데이터: 1 0 1 1 0 0 1 0

    신호:
    +V |  +--+     +--+--+        +--+
       |  |  |     |  |  |        |  |
     0 |  |  |     |  |  |        |  |
       |  |  |     |  |  |        |  |
    -V |  |  |     |  |  |        |  |
       +--+  +-----+  +  +--------+  +---->
         1   0     1   1   0   0    1   0

    비트레이트 = 보드레이트 = 1/T bps (T = 비트 기간)


    [BPSK 예시]

    위상 천이:
    0 -> 0° (cos(ωt))
    1 -> 180° (-cos(ωt))

    성운도 (Constellation):
               Q
               ^
               |
        1      |      0
        x------+------> I
        180°   |      0°
               |

    1심볼 = 1비트
    비트레이트 = 보드레이트
```

## 실무 적용

### 1. 통신 시스템 설계에서의 고려사항

```
    설계 의사결정 프로세스:

    1. 요구 비트레이트 결정
       예: 10 Mbps 데이터 전송

    2. 가용 대역폭 확인
       예: 2 MHz 채널

    3. 대역폭 효율 계산
       η = 10 Mbps / 2 MHz = 5 bps/Hz

    4. 필요한 변조 방식 선택
       - 16-QAM: ~2 bps/Hz (부족)
       - 64-QAM: ~3 bps/Hz (부족)
       - 256-QAM: ~4 bps/Hz (부족)
       - 1024-QAM: ~5 bps/Hz (적합)

    5. SNR 요구사항 확인
       - 1024-QAM: 약 35 dB SNR 필요
       - 가용 SNR이 40 dB이면 가능

    6. 오류 정정 코드 설계
       - LDPC 또는 Turbo 코드 추가
       - 코딩 레이트에 따른 비트레이트 조정
```

### 2. 적응형 변복조 (Adaptive Modulation)

```
+------------------------------------------------------------------+
|              적응형 변복조 시스템 (Adaptive Modulation)              |
+------------------------------------------------------------------+

    SNR 측정
        |
        v
    +-----------+     +-----------+
    | SNR < 10dB| --> | BPSK      | --> 6 Mbps
    +-----------+     | (1 bps/Hz)|     (1/2 코딩)
        |
    +-----------+     +-----------+
    | 10-15 dB  | --> | QPSK      | --> 12 Mbps
    +-----------+     | (2 bps/Hz)|     (1/2 코딩)
        |
    +-----------+     +-----------+
    | 15-20 dB  | --> | 16-QAM    | --> 24 Mbps
    +-----------+     | (4 bps/Hz)|     (1/2 코딩)
        |
    +-----------+     +-----------+
    | 20-25 dB  | --> | 64-QAM    | --> 36 Mbps
    +-----------+     | (6 bps/Hz)|     (2/3 코딩)
        |
    +-----------+     +-----------+
    | SNR > 25dB| --> | 256-QAM   | --> 48 Mbps
    +-----------+     | (8 bps/Hz)|     (3/4 코딩)


    적용 분야:
    - WiFi (802.11n/ac/ax)
    - LTE/5G 셀룰러
    - DSL (ADSL2+, VDSL2)
    - DOCSIS (케이블 모뎀)
```

### 3. 실제 성능 측정

```
    측정 파라미터:

    1. 실제 비트레이트 (Throughput)
       - 오버헤드 제외 실제 데이터 전송 속도
       - 비트레이트 x (1 - 오버헤드 비율)

    2. 유효 비트레이트 (Goodput)
       - 응용 계층에서의 실제 데이터 속도
       - 재전송 오버헤드 제외

    3. 대역폭 효율
       η = 비트레이트 / 대역폭 (bps/Hz)

    4. 에너지 효율
       η_E = 비트레이트 / 소비 전력 (bits/Joule)


    측정 도구:
    +-------------------+-------------------------------------------+
    |     도구          |      용도                                 |
    +-------------------+-------------------------------------------+
    | Spectrum Analyzer  | 주파수 스펙트럼 분석                       |
    | Vector Analyzer    | 변조 품질 분석 (EVM, Constellation)        |
    | Bit Error Tester   | BER 측정                                  |
    | Network Analyzer   | 처리량 측정 (iperf, netperf)              |
    +-------------------+-------------------------------------------+
```

## 기술사 시험 포인트

### 1. 핵심 암기 사항

```
+------------------------------------------------------------------+
|              보드레이트/비트레이트 핵심 정리                          |
+------------------------------------------------------------------+

1. 정의:
   - 보드레이트: 1초당 심볼(신호 변화) 수, 단위: Baud
   - 비트레이트: 1초당 비트 수, 단위: bps

2. 핵심 공식:
   비트레이트 = 보드레이트 x log2(M)

   where M = 변조 레벨 수 (상태 수)

3. M값에 따른 비트/심볼:
   - M = 2: 1 bit/symbol (BPSK, NRZ)
   - M = 4: 2 bits/symbol (QPSK)
   - M = 8: 3 bits/symbol (8-PSK)
   - M = 16: 4 bits/symbol (16-QAM)
   - M = 64: 6 bits/symbol (64-QAM)
   - M = 256: 8 bits/symbol (256-QAM)

4. 대역폭 관계:
   - Nyquist: B = Rs / 2
   - Raised Cosine: B = Rs / 2 x (1 + α)

5. 특수 경우:
   - 이진 변조 (M=2): 보드레이트 = 비트레이트
   - 다진 변조 (M>2): 비트레이트 > 보드레이트

6. 대역폭 효율:
   η = 비트레이트 / 대역폭 (bps/Hz)
```

### 2. 빈출 문제 유형

```
1. 비트레이트 계산
   Q: 2400 Baud의 QPSK 변조에서 비트레이트는?
   A: 2400 x log2(4) = 2400 x 2 = 4800 bps

2. 보드레이트 계산
   Q: 64-QAM으로 1 Mbps를 전송할 때 필요한 보드레이트는?
   A: 1 Mbps / log2(64) = 1 Mbps / 6 = 166.67 kBaud

3. 대역폭 계산
   Q: 16-QAM, 10 MBaud, α=0.5일 때 필요한 대역폭은?
   A: 10M / 2 x (1 + 0.5) = 7.5 MHz

4. 비교 분석
   Q: 동일 대역폭에서 QPSK와 16-QAM의 비트레이트 비교
   A: 16-QAM이 QPSK의 2배 (4비트 vs 2비트/심볼)

5. 변조 방식 선택
   Q: SNR이 15dB인 채널에서 가장 효율적인 변조 방식은?
   A: QPSK (16-QAM은 약 20dB 필요)
```

### 3. 기술사 답안 작성 요령

```
[문제] 보드레이트와 비트레이트의 차이점을 설명하고, 변조 방식에 따른
      관계를 기술하시오.

[답안 구성]
1. 개요
   - 보드레이트의 정의 (심볼 레이트, Baud)
   - 비트레이트의 정의 (데이터 속도, bps)

2. 핵심 관계식
   비트레이트 = 보드레이트 x log2(M)

3. 심볼과 비트의 관계
   가. 심볼의 정의 (신호 변화 단위)
   나. 변조 레벨 M에 따른 비트 수
   다. 예시 (M=2, 4, 8, 16, 64, 256)

4. 주요 변조 방식별 분석
   가. 이진 변조 (M=2): 보드레이트 = 비트레이트
      - BPSK, NRZ, ASK
   나. 다진 변조 (M>2): 비트레이트 > 보드레이트
      - QPSK, 8-PSK, 16/64/256-QAM

5. 실제 적용 예시
   - 다이얼업 모뎀 (56K)
   - 이더넷 (100BASE-TX)
   - WiFi (802.11ac)

6. 대역폭과의 관계
   - Nyquist 공식
   - 대역폭 효율

7. 결론
   - 높은 M값의 장단점 (Trade-off)
   - 적응형 변복조의 필요성
```
