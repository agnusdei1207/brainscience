# 075. 시분할 다중화 (TDM - Time Division Multiplexing)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 전송 매체의 전체 대역폭을 점유하되, 시간을 아주 짧은 간격(Time Slot)으로 나누어 여러 사용자가 순차적으로 매체를 공유하는 디지털 기반 다중화 기술입니다.
> 2. **가치**: 디지털 신호 처리에 최적화되어 있으며, 동기식 TDM은 일정한 지연(Deterministic Latency)을 보장하고 통계적 TDM은 유휴 슬롯을 제거하여 대역폭 효율을 극대화합니다.
> 3. **융합**: 현대의 SDH/SONET 광전송망, T1/E1 전화 회선, 그리고 컴퓨터 내부의 시스템 버스 및 인터리빙 기술의 근간이 됩니다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
시분할 다중화(Time Division Multiplexing, TDM)는 하나의 고속 전송 회선을 다수의 저속 채널이 공유하기 위해, **시간축을 여러 개의 슬롯(Slot)으로 분할**하여 각 채널에 할당하는 방식입니다. 모든 채널은 서로 다른 시간에 전체 대역폭을 사용합니다.

### 💡 비유
TDM은 **"회전 초밥집의 레일"**과 같습니다. 레일(전송 매체)은 하나지만, 주방장(Mux)이 연어 초밥, 참치 초밥, 새우 초밥을 차례대로 접시에 담아 레일에 올립니다. 손님(Demux)은 자기 앞에 오는 접시 중 원하는 종류(Time Slot)만 골라 가져갑니다. 레일 위에는 여러 종류의 초밥이 섞여 있지만, 시간적으로 순서가 정해져 있어 서로 섞이지 않습니다.

### 등장 배경 및 발전 과정

#### 1. 디지털 통신의 등장 (1960년대)
- 아날로그 FDM 방식은 필터 설계의 복잡성과 보호 대역으로 인한 효율 저하 문제가 있었습니다.
- PCM(Pulse Code Modulation) 기술로 음성이 디지털화되면서, 비트 단위나 바이트 단위로 시간을 쪼개어 전송하는 것이 훨씬 정밀하고 구현하기 쉬워졌습니다.

#### 2. 대규모 백본망의 요구
- 전화국 간 수천 개의 통화 선로를 효율적으로 통합해야 했고, 이를 위해 T1(North America) 및 E1(Europe)이라는 표준 TDM 계층 구조가 확립되었습니다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### TDM 방식의 분류 (표)

| 비교 항목 | 동기식 TDM (Synchronous) | 통계적 TDM (Statistical / Asynchronous) |
|-----------|-------------------------|-----------------------------------------|
| **슬롯 할당** | 고정적 (Fixed Assignment) | 동적 (Dynamic / On-demand) |
| **유휴 슬롯** | 데이터 없어도 슬롯 전송 (낭비 발생) | 데이터 있는 채널만 전송 (효율 극대화) |
| **주소 정보** | 불필요 (순서로 식별) | 필수 (어느 채널 데이터인지 헤더 필요) |
| **지연(Latency)** | 일정 (Deterministic) | 가변적 (Queuing 지연 발생 가능) |
| **주요 응용** | T1/E1, SONET, 고전적 전화망 | ATM, 패킷 스위칭, 현대적 데이터 통신 |

### TDM 프레임 구조 다이어그램

```ascii
[ Synchronous TDM Architecture ]

 입력 채널 (A, B, C)          Multiplexer (Mux)            공유 매체 (High-speed)
   ┌───┐                      (Scanning Switch)
 A │ A1│──────────┐                 /
   └───┘          │                /  Clock Sync
   ┌───┐          │          [  /  ]
 B │ B1│──────────┼───────────○  ○───────────────────> [Composite Stream]
   └───┘          │                \
   ┌───┐          │                 \
 C │ C1│──────────┘
   └───┘

[ TDM Frame Structure ]

  |<------ Frame 1 ------>|<------ Frame 2 ------>|
  +---+---+---+---+-------+---+---+---+---+-------+
  | S | A1| B1| C1| Empty | S | A2| B2| C2| D2    | ...
  +---+---+---+---+-------+---+---+---+---+-------+
    ^   ^   ^   ^
    │   └─┴─┴─┴─ Time Slots for each Channel
    └─ Synchronization Bit (Framing Bit)

[ Statistical TDM (ATDM) Structure ]

  +-------+-------+-------+-------+-------+
  | Addr  | Data  | Addr  | Data  | Addr  | ...
  | (Ch A)| (A1)  | (Ch C)| (C1)  | (Ch B)|
  +-------+-------+-------+-------+-------+
   * 데이터가 있는 채널만 주소와 함께 전송
```

### 심층 동작 원리

#### ① 인터리빙 (Interleaving)
Tux는 각 채널에서 데이터를 한 번에 한 단위(1비트 또는 1바이트)씩 가져와 프레임에 배치합니다.
- **Bit Interleaving**: 디지털 신호의 비트 단위로 쪼개어 배치.
- **Character(Byte) Interleaving**: 8비트(1바이트) 단위로 배치. (예: T1 시스템)

#### ② 프레임 동기화 (Framing)
수신측(Demux)이 어느 비트가 채널 A의 시작인지 알기 위해, 프레임의 시작 부분에 특수한 비트 패턴(Framing bit)을 삽입합니다. 이를 통해 송수신측의 시계를 맞춥니다.

#### ③ 슬롯 오버헤드와 효율
- **동기식**: 주소 정보가 없어 오버헤드가 적지만, 사용하지 않는 채널이 많으면 효율이 급격히 떨어집니다.
- **통계적**: 데이터가 발생할 때만 슬롯을 쓰므로 효율적이지만, 각 슬롯마다 '누구의 데이터인가'를 알리는 **주소(Address) 비트** 오버헤드가 발생합니다.

### TDM 계층 구조 (Digital Hierarchy)

|--------|------|---------|-------------|------|

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

### 기술사적 판단 (실무 시나리오)

**상황**: 금융권 전용 회선망 구축 시, 보안을 위해 물리적 채널 분리와 극도의 저지연(Low Latency)이 요구됨.
**판단**: 
- **동기식 TDM 기반 전용선(Leased Line)** 또는 **OTN(Optical Transport Network)** 솔루션 제안.
- **이유**: 통계적 방식(패킷 교환)은 트래픽 폭주 시 큐잉 지연(Queuing Delay)이 발생하여 금융 거래의 시급성을 저해할 수 있으나, 동기식 TDM은 타 채널의 부하와 상관없이 고정된 타임슬롯을 보장하여 지연 시간의 가변성(Jitter)을 제거함.

### 도입 시 고려사항 (체크리스트)
1. **클럭 동기화**: 망 전체의 클럭이 일치하지 않으면 데이터가 누락되는 **슬립(Slip)** 현상 발생. (GPS 또는 Primary Reference Clock 필요)
2. **트래픽 패턴**: 데이터가 불규칙적(Bursty)이라면 통계적 TDM이 훨씬 경제적.
3. **확장성**: 슬롯 수가 고정된 동기식 방식은 사용자 추가 시 장비 전체 교체가 필요할 수 있음.

## Ⅴ. 기대효과 및 결론

### 정량적 기대효과
- **선로 병합**: 24개의 음성 회선을 하나의 T1 라인으로 통합하여 물리적 배선 비용 95% 절감.
- **품질 보장**: 고정 대역폭 할당으로 QoS(Quality of Service) 보장 용이.

### 미래 전망
전통적인 회선 교환형 TDM은 점차 **IP 기반 패킷망**으로 통합되고 있습니다. 하지만 하단부의 광전송 계층(OTN)에서는 여전히 거대한 데이터를 시분할하여 실어나르는 TDM 기술이 핵심적인 역할을 수행하며, 더 정밀한 시간 관리를 위한 **PTP(Precision Time Protocol)** 등과 결합하고 있습니다.
