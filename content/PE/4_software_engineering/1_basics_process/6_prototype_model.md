# 프로토타입 모델 (Prototype Model)

## 정의

**프로토타입 모델(Prototype Model)**은 실제 소프트웨어와 유사한 **시제품(Prototype)을 먼저 개발**하여 사용자에게 보여주고 피드백을 받아 **요구사항을 명확화**하는 개발 방식이다.

```
┌─────────────────────────────────────────────────────────────────┐
│           프로토타입 모델 개념도                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌──────────────┐                                             │
│   │ 요구사항 수집 │                                             │
│   └──────┬───────┘                                             │
│          ▼                                                      │
│   ┌──────────────┐      ┌──────────────┐                       │
│   │  빠른 설계    │────▶ │ 프로토타입   │                       │
│   └──────────────┘      │    구축      │                       │
│          ▲              └──────┬───────┘                       │
│          │                     ▼                                │
│          │              ┌──────────────┐                       │
│          │              │ 사용자 평가  │                       │
│          │              └──────┬───────┘                       │
│          │                     │                                │
│          └─────────────────────┤                                │
│               피드백/수정       │ 만족                           │
│                                ▼                                │
│                         ┌──────────────┐                       │
│                         │  실제 개발    │                       │
│                         └──────────────┘                       │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 핵심 개념

### 1. 프로토타입 유형

```
┌─────────────────────────────────────────────────────────────────┐
│           프로토타입 유형                                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   폐기형 (Throw-away):                                          │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │ • 목적: 요구사항 파악                                    │ │
│   │ • 특징: 요구사항 확인 후 폐기, 본 개발은 새로 시작       │ │
│   │ • 장점: 요구사항 명확화                                  │ │
│   │ • 단점: 중복 개발 비용                                   │ │
│   │ • 적용: 요구사항이 매우 불명확한 경우                    │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   진화형 (Evolutionary):                                        │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │ • 목적: 점진적 개발                                      │ │
│   │ • 특징: 프로토타입을 지속적으로 개선하여 최종 제품 완성  │ │
│   │ • 장점: 시간/비용 절약                                   │ │
│   │ • 단점: 구조적 문제 누적 가능                            │ │
│   │ • 적용: 점진적 기능 확장이 필요한 경우                   │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   증분형 (Incremental):                                         │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │ • 목적: 기능별 분할 개발                                  │ │
│   │ • 특징: 핵심 기능부터 우선 개발, 순차적으로 기능 추가    │ │
│   │ • 장점: 조기 사용 가능, 위험 분산                        │ │
│   │ • 적용: 대규모 시스템                                    │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2. 프로토타입 모델 프로세스

```
┌─────────────────────────────────────────────────────────────────┐
│           프로토타입 모델 상세 프로세스                          │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   1단계: 요구사항 수집                                          │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │ • 사용자 인터뷰                                          │ │
│   │ • 불명확한 요구사항 식별                                  │ │
│   │ • 프로토타입 목표 정의                                    │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   2단계: 빠른 설계 (Quick Design)                               │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │ • 핵심 기능 중심 설계                                    │ │
│   │ • UI/UX 초안                                             │ │
│   │ • 세부 구현 생략 가능                                    │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   3단계: 프로토타입 구축                                        │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │ • 신속한 개발 (Rapid Development)                        │ │
│   │ • Mock 데이터 사용                                       │ │
│   │ • 주요 기능만 구현                                       │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   4단계: 사용자 평가                                            │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │ • 사용자 피드백 수집                                     │ │
│   │ • 요구사항 검증                                          │ │
│   │ • 수정 사항 도출                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   5단계: 프로토타입 정제 (반복)                                  │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │ • 피드백 반영                                            │ │
│   │ • 프로토타입 개선                                        │ │
│   │ • 만족할 때까지 반복                                     │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   6단계: 실제 제품 개발                                         │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │ • 확정된 요구사항 기반 개발                               │ │
│   │ • 완전한 기능 구현                                       │ │
│   │ • 품질 보증 활동                                         │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3. 장단점

```
┌─────────────────────────────────────────────────────────────────┐
│           프로토타입 모델 장단점                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   장점:                                                         │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │ ① 요구사항 명확화                                        │ │
│   │    - 사용자가 실제 형태를 보고 피드백 제공               │ │
│   │    - 오해/누락 방지                                       │ │
│   │                                                         │ │
│   │ ② 사용자 참여 증대                                       │ │
│   │    - 적극적인 요구사항 검증                              │ │
│   │    - 사용자 만족도 향상                                  │ │
│   │                                                         │ │
│   │ ③ 조기 오류 발견                                         │ │
│   │    - 개발 초기에 문제 식별                               │ │
│   │    - 수정 비용 절감                                      │ │
│   │                                                         │ │
│   │ ④ 위험 감소                                              │ │
│   │    - 기술적 타당성 검증                                  │ │
│   │    - 프로젝트 실패 위험 감소                             │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
│   단점:                                                         │
│   ┌─────────────────────────────────────────────────────────┐ │
│   │ ① 범위 관리 어려움                                       │ │
│   │    - 사용자가 계속 수정 요구                             │ │
│   │    - 프로젝트 종료 시점 불확실                           │ │
│   │                                                         │ │
│   │ ② 문서화 부족                                            │ │
│   │    - 빠른 개발로 문서 소홀                               │ │
│   │    - 유지보수 어려움                                     │ │
│   │                                                         │ │
│   │ ③ 비용 증가 가능                                         │ │
│   │    - 폐기형: 이중 개발                                   │ │
│   │    - 반복으로 인한 비용                                  │ │
│   │                                                         │ │
│   │ ④ 오해 유발                                              │ │
│   │    - 사용자가 프로토타입을 완제품으로 착각               │ │
│   │    - 품질 기대치 관리 필요                               │ │
│   └─────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 기술사 시험 포인트

### 1. 핵심 정의

> **프로토타입 모델**: 시제품을 통한 요구사항 명확화
> **폐기형**: 요구사항 파악 후 폐기, 새로 개발
> **진화형**: 프로토타입을 최종 제품으로 발전

### 2. 빈출 키워드

- **사용자 피드백**: 요구사항 검증 핵심
- **요구사항 명확화**: 불명확한 요구사항 해결
- **폐기형 vs 진화형**: 프로토타입 처리 방식
- **반복 개선**: 피드백 반영 프로세스

### 3. 예상 문제

**Q1. 폐기형 프로토타입과 진화형 프로토타입의 차이를 설명하시오.**

**Q2. 프로토타입 모델이 적합한 상황을 기술하시오.**

**Q3. 프로토타입 모델의 단점과 대응 방안을 설명하시오.**
