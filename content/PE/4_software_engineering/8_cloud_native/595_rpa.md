# 595. RPA (Robotic Process Automation) 봇 결합 아키텍처

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 사람이 컴퓨터 화면에서 수행하는 단순 반복적인 업무(클릭, 복사, 입력 등)를 **소프트웨어 로봇(Bot)이 UI 레벨에서 자동화**하여, 기존 시스템의 변경 없이 업무 프로세스를 개선하는 기술이다.
> 2. **가치**: API가 제공되지 않는 레거시 시스템 간의 데이터를 연결하는 '가교' 역할을 수행하며, 휴먼 에러를 제거하고 24시간 중단 없는 업무 처리를 통해 기업의 운영 효율성을 극대화한다.
> 3. **융합**: 인공지능(AI)의 판단 능력이 결합된 **IPA (Intelligent Process Automation)**, 중앙 통제를 위한 관리 서버(Orchestrator), 그리고 딥러닝 기반의 광학 문자 인식(OCR) 기술과 융합되어 지능형 자동화를 완성한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
RPA는 "디지털 사무직원"이다. 엑셀 데이터를 읽어서 사내 ERP 시스템에 입력하고, 결과 리포트를 이메일로 보내는 것과 같이 규칙이 명확한 업무를 로봇이 대신해준다. 소스 코드를 고치는 게 아니라 사용자가 쓰는 화면을 그대로 이용하는 것이 특징이다.

### 💡 비유
RPA는 **"컴퓨터 속의 매크로 장인"**과 같다.
- **일반 개발 (API)**: 건물의 벽을 뚫고 전용 파이프라인(API)을 설치하는 공사. 튼튼하지만 비용과 시간이 많이 든다.
- **RPA**: 로봇이 사람 대신 문으로 들어가서 직접 스위치를 누르고 서류를 옮기는 것. 건물 구조를 바꿀 필요가 없어 빠르고 간편하다.

### 등장 배경 및 발전 과정

#### 1. 레거시 시스템의 한계
많은 기업이 10~20년 된 낡은 시스템을 쓴다. 이 시스템들은 API가 없어 최신 앱과 대화할 수 없었고, 결국 사람이 중간에서 데이터를 퍼 나르는 '노가다'를 해야 했다.

#### 2. 주 52시간 근무제와 생산성 혁명
단순 업무에 투입되는 인력을 줄이고, 창의적인 업무에 집중하게 하려는 기업들의 수요가 폭발하며 RPA 시장이 급성장했다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### RPA 시스템의 3대 구성 요소 (표)

| 요소명 | 상세 역할 | 특징 | 비유 |
|--------|----------|------|------|
| **Bot (Studio/Robot)** | 실제 업무 수행 주체 | UI 객체 인식, 워크플로우 실행 | 일하는 로봇 직원 |
| **Orchestrator** | 봇의 스케줄링 및 지휘 | 봇들의 상태 감시, 작업 배분, 로그 관리 | 로봇 팀장님 |
| **Runtime Environment**| 봇이 돌아가는 PC/VM | 윈도우 기반 가상 환경 또는 실제 PC | 로봇의 책상 |

### 정교한 구조 다이어그램 (RPA 봇 운영 아키텍처)

```ascii
================================================================================
[ RPA Enterprise Architecture: Control and Execution ]
================================================================================

  [ 👑 RPA Orchestrator (Cloud/Server) ] ◀──▶ [ 👨‍💻 Admin / Developer ]
           │                                     (Work Design & Monitoring)
           │ (1) Assign Job & License
           ▼
  ┌──────────────────────────────────────────────────────────┐
  │  🤖 Digital Worker Pool (Virtual Machines)               │
  │                                                          │
  │  ┌───────────────┐      ┌───────────────┐                │
  │  │ 🤖 Bot A      │      │ 🤖 Bot B      │                │
  │  │ (Attended)    │      │ (Unattended)  │                │
  │  └───────┬───────┘      └───────┬───────┘                │
  └──────────┼──────────────────────┼────────────────────────┘
             │ (2) Interaction      │ (3) Data Fetching
             ▼                      ▼
  ┌──────────────────┐      ┌──────────────────┐      ┌──────────────────┐
  │ 🏢 Legacy ERP    │      │ 📊 Excel / PDF   │      │ ☁️ Cloud SaaS     │
  │ (No API)         │      │ (OCR Input)      │      │ (Salesforce)     │
  └──────────────────┘      └──────────────────┘      └──────────────────┘

 * 핵심 포인트: 봇은 사람이 쓰는 화면(UI)을 픽셀이나 태그 단위로 인식하여 
   키보드와 마우스 이벤트를 '흉내' 낸다.
```

### 심층 동작 원리

#### ① UI 객체 인식 (Selectors)
단순히 마우스 좌표를 외우는 게 아니다. 웹 페이지의 HTML 태그나 윈도우 프로그램의 ID 값을 인식한다. 화면 크기가 바뀌어도 "로그인 버튼"이라는 객체를 찾아내어 정확히 클릭하는 지능적인 매커니즘을 가진다.

#### ② 유인(Attended) vs 무인(Unattended) 봇
- **유인 봇**: 사람이 시킬 때만 작동한다. 콜센터 직원 옆에서 실시간 조회를 돕는다.
- **무인 봇**: 서버에서 24시간 혼자 돌아간다. 새벽에 정산 데이터를 모아두는 식이다.

#### ③ 지능형 OCR 결합
최신 RPA는 이미지에서 글자를 읽어내는 **OCR(광학 문자 인식)**과 결합한다. 종이로 된 영수증이나 스캔 된 계약서를 찍어서 보내면, AI가 내용을 읽고 RPA가 시스템에 입력하는 원스톱 자동화를 구현한다.

## Ⅲ. 융합 비교 및 다각도 분석

### API 자동화 vs RPA 자동화

| 비교 지표 | API (Direct) | **RPA (UI-based)** |
|-----------|---------------|---------------------|
| **개발 대상** | 백엔드 코드 | **프론트엔드 UI** |
| **개발 난이도** | 높음 (시스템 이해 필수) | **보통 (Low-code 지향)** |
| **안정성** | 최상 (안 변함) | 보통 (UI 바뀌면 오류 가능) |
| **비용/속도** | 구축 기간 긺, 비용 높음 | **구축 매우 빠름, 가성비 우수** |
| **비유** | 지하에 전용 수도관 매설 | **로봇이 물통 들고 나르기** |

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

### 기술사적 판단 (실무 시나리오)

**상황**: 한 제조 기업의 구매팀. 매일 아침 500개 협력사의 웹사이트에 들어가서 단가 정보를 긁어와 엑셀에 정리하고, 사내 SAP ERP에 입력하는 데 3명이 오전 시간을 다 쓴다. SAP는 폐쇄망이라 외부 API 연동이 안 된다.
**판단 및 RPA 도입 전략**:
1. **Unattended Bot 구축**: 새벽 4시에 봇 2대를 가동하여 500개 사이트의 정보를 자동으로 크롤링하게 한다.
2. **Exception Handling**: 웹사이트가 개편되거나 로그인이 실패할 경우, 즉시 담당자에게 '스크린샷'과 함께 텔레그램 알림을 보내게 설계한다.
3. **Audit Trail**: 봇이 수행한 모든 클릭 이력을 동영상이나 로그로 남겨 감사가 가능하게 한다.
4. **결과**: 직원 3명의 오전 업무가 사라지고, 구매팀은 수집된 데이터를 바탕으로 '단가 협상 전략'을 짜는 가치 있는 업무에만 집중하게 된다.

### 주의사항 및 안티패턴 (Anti-patterns)
- **RPA 만능주의**: API가 있는데도 귀찮아서 RPA로 개발하는 것은 '기술 부채'를 쌓는 지름길이다. UI가 1픽셀만 바뀌어도 봇은 멈춘다. **"API가 가능한 곳은 API로, 안 되는 레거시만 RPA로"** 하는 균형 감각이 필수다.

---

## Ⅴ. 기대효과 및 결론

### 정량적 기대효과
- **생산성 향상**: 수동 작업 대비 처리 속도 **5~10배 향상**.
- **품질**: 전사적인 입력 오류(Typo) 발생률 **0.1% 이하**로 격감.

### 미래 전망 및 진화 방향
RPA는 이제 **'하이퍼 자동화 (Hyper-automation)'**로 가고 있다. 사람이 업무 프로세스를 가르쳐주는 게 아니라, AI가 직원의 업무 로그를 보고 "이 일은 자동화할 수 있겠네요"라고 먼저 제안하는 **Process Mining** 기술과 결합하고 있다. 또한, 생성형 AI(ChatGPT)와 융합하여 "메일 온 거 보고 대충 알아서 답장 보내줘"와 같은 비정형 업무까지 수행하는 **지능형 에이전트**로 진화할 것이다.

## 👶 어린이를 위한 3줄 비유 설명
1. **RPA가 뭔가요?**: 컴퓨터 속에서 나 대신 마우스를 움직이고 키보드를 쳐주는 '투명 인간 로봇'이에요.
2. **왜 좋나요?**: 지루한 숙제를 백 번 반복해도 로봇은 절대로 졸거나 틀리지 않고, 엄청나게 빠른 속도로 끝내주기 때문이에요.
3. **언제 쓰나요?**: 게임 아이템 시세를 매일 아침 수첩에 적는 일처럼, 똑같은 일을 매일 반복해야 할 때 로봇에게 "대신 해줘!"라고 시키면 된답니다!
