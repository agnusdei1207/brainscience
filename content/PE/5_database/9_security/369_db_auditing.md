# 369. 데이터베이스 감사 (DB Auditing) - 데이터 세계의 블랙박스 기록기

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터베이스 내에서 발생하는 모든 활동(접속, 데이터 조회, 수정, 삭제, 권한 변경 등)을 **누가, 언제, 어디서, 어떤 데이터에, 무엇을 했는지 기록하고 감시하는 보안 프로세스**다.
> 2. **혁신**: 단순한 로그 기록을 넘어 사후 장애 분석, 내부자 부정행위 적발, 그리고 규제 기관(GDPR, ISMS 등)의 요구에 대응하는 **객관적인 증거 자료(Audit Trail)**를 자동 생성한다.
> 3. **가치**: 데이터 유출 사고 발생 시 원인 추적의 결정적 단서를 제공하며, "모든 행동은 기록된다"는 심리적 압박을 통해 내부 직원의 고의적인 데이터 오남용을 사전에 억제하는 강력한 통제력을 제공한다.

---

## Ⅰ. 개요 (Context & Background)

### 개념
데이터베이스 감사(Auditing)는 DB에서 일어나는 모든 일을 지켜보는 'CCTV'와 같다. 사용자가 던진 SQL 문장 하나하나를 기록 장부(Audit Log)에 남겨, 나중에 문제가 생겼을 때 범인을 잡거나 시스템 복구의 근거로 삼는다.

### 💡 비유
- **출입 통제 (ID 카드)**: 회사 정문을 통과할 때 카드를 찍는 것이다. (인증/인가).
- **데이터베이스 감사 (사무실 내 CCTV 및 PC 로그)**: 회사에 들어온 직원이 자기 책상에 앉아 어떤 서류를 열어봤고, 어떤 내용을 수정해서 파쇄기에 넣었는지 모든 행동을 녹화하고 기록하는 것이다. 나중에 기밀 서류가 없어지면 CCTV를 돌려보고 "누가 가져갔는지"를 정확히 알아낼 수 있다.

### 등장 배경: '내부자 유출'의 공포
해커에 의한 외부 공격보다 더 무서운 것은 퇴사를 앞둔 직원이나 권한을 가진 관리자가 고객 정보를 엑셀로 긁어가는 내부 유출 사고다. 일반적인 DB 로그는 성능을 위해 최소한만 기록하지만, 감사는 '보안'을 위해 상세한 행동 하나하나를 기록해야 할 필요성 때문에 독립된 기술 영역으로 발전했다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 데이터베이스 감사 시스템 구성 (ASCII Diagram)

```ascii
[ Database User ]
       │
       │ (SQL: SELECT * FROM SALARY WHERE ID='101')
       ▼
┌────────────────────────────────────────────────────────┐
│ [ Database Engine ]                                    │
│                                                        │
│  ① Audit Policy Engine (감사 정책)                      │
│     - "급여 테이블 조회 시 로그 남겨라" 정책 확인         │
│                                                        │
│  ② Trigger / Native Audit (감사 수행)                  │
│     - SQL 문장, 접속 IP, 시간, 결과 성공여부 추출        │
└──────────┬─────────────────────────────────────────────┘
           │ (Async Logging)
           ▼
┌────────────────────────────────────────────────────────┐
│ [ Secure Audit Store (감사 로그 저장소) ]               │
│  - 수정/삭제가 불가능한 독립된 영역 (Write-once)         │
└──────────┬─────────────────────────────────────────────┘
           │
           ▼ (Reporting & Monitoring)
[ Auditor / Security Manager (감사자) ]
  "어제 야간에 급여 조회를 시도한 모든 내역 출력해!"
```

### 3대 핵심 감사 대상 (What to Audit)

#### ① 데이터 접근 감사 (DML Audit)
가장 일반적인 감사다. `SELECT`, `INSERT`, `UPDATE`, `DELETE` 명령을 감시한다. 특히 특정 민감한 테이블(고객 정보, 인사 정보)에 대한 접근을 집중적으로 기록한다.

#### ② 권한 및 스키마 변경 감사 (DDL/DCL Audit)
사용자를 새로 만들거나, 비밀번호를 바꾸거나, 테이블을 통째로 지워버리는(`DROP`) 행위를 기록한다. 관리자(DBA)의 권한 오남용을 막는 핵심 장치다.

#### ③ 로그인 및 접속 감사 (Access Audit)
누가 언제 어떤 IP에서 접속했는지, 몇 번이나 로그인을 실패했는지 기록한다. 'Brute-force' 공격이나 비인가 지역에서의 접속을 탐지하는 데 쓰인다.

## Ⅲ. 융합 비교 및 다각도 분석

### 네이티브 감사 vs 네트워크 스니핑 감사 비교

| 구분 | 네이티브 감사 (DBMS 내장) | **네트워크 스니핑 (DB 방화벽)** |
|:---:|:---|:---|
| **작동 위치** | DB 엔진 내부 | DB 서버 앞단 네트워크 스위치 |
| **성능 영향** | **있음 (DB CPU 점유)** | 없음 (네트워크 복제 방식) |
| **우회 가능성**| 없음 (로컬 접속도 잡음) | **있음 (서버 직접 접속 시 못 잡음)**|
| **정밀도** | 매우 높음 (Bind 변수값까지 확인) | 보통 (패킷 해독 필요) |
| **비유** | 집안의 CCTV | 아파트 단지 정문 CCTV |

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

### 기술사적 판단 (실무 감사 정책 수립 전략)

**상황**: 고객 정보 1,000만 건을 보유한 이커머스 기업. 모든 `SELECT` 쿼리를 감사했더니 로그 저장소 용량이 하루 만에 가득 차고 DB 성능이 30% 하락했다.
**판단 및 효율적 감사 전략**:
1. **표적 감사 (Selective Auditing)**: 모든 테이블이 아닌, 법적 규제 대상인 '개인정보 테이블' 5개에 대해서만 감사를 설정한다.
2. **실패 로그 집중**: 성공한 조회보다 **'권한 부족으로 거부된(Access Denied)'** 실패 로그를 최우선 순위로 기록한다. (해킹 징후 탐지).
3. **권한 기반 차등**: 일반 사용자의 쿼리는 요약해서 남기고, **DBA나 Root 계정**의 모든 행동은 SQL 전문(Full-text)을 남기도록 차등화한다.
4. **결과**: DB 성능 저하를 5% 미만으로 억제하면서도, 법적 감사 요건(ISMS-P)을 100% 충족하고 핵심 보안 가시성을 확보함.

### 안티패턴 및 고려사항
- **감사 로그를 동일 DB에 저장**: 감사 로그를 원본 데이터와 같은 테이블에 저장하면, 해커가 침입한 뒤 증거 인멸을 위해 감사 로그를 직접 지워버릴 수 있다. 감사 로그는 반드시 **물리적으로 분리된 외부 저장소(SIEM, S3 등)**로 즉시 전송하여 보관해야 한다. (Write-once, Read-many).

## Ⅴ. 미래 전망 및 결론

### 결론: 신뢰를 위한 감시
데이터베이스 감사는 "직원을 믿지 못해서"가 아니라, "사고가 났을 때 선량한 직원이 범인이 아님을 증명하고, 회사의 자산을 지키기 위해" 존재한다. 완벽한 감사가 구축된 시스템만이 진정한 비즈니스 신뢰를 얻을 수 있다.

### 미래 전망
앞으로는 **'AI 기반 능동형 감사'**가 대세가 될 것이다. 단순히 로그를 쌓아두는 것이 아니라, AI가 실시간으로 로그를 읽으며 "평소 오전 9시에만 접속하던 이 대리가 왜 갑자기 일요일 새벽 2시에 고객 100만 명을 조회하지?"라는 이상 행동을 스스로 발견하고 그 즉시 세션을 차단하는 지능형 방어 체계로 진화할 전망이다.

## 👶 어린이를 위한 3줄 비유 설명
1. **데이터베이스 감사가 뭔가요?**: 학교 도서관에서 누가 어떤 책을 빌려 갔고, 누가 책 페이지를 찢었는지 매일 기록하는 '대출 장부'와 'CCTV' 같은 거예요.
2. **왜 기록하나요?**: 소중한 책이 없어졌을 때, 장부를 확인해서 누가 가져갔는지 찾아내고 다시는 그런 일이 없게 막으려고 하는 거예요.
3. **가장 좋은 점은?**: "CCTV가 지켜보고 있다!"는 걸 알면 나쁜 마음을 먹었던 사람도 함부로 책을 훔치거나 망가뜨리지 않게 된답니다!
