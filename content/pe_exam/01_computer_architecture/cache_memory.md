# 캐시 메모리 (Cache Memory)

## 핵심 인사이트 (3줄 요약)
> CPU와 주기억장치 사이의 **속도 차이를 완화**하는 고속 메모리. 지역성(Locality) 원리를 활용해 자주 사용하는 데이터를 저장한다. L1→L2→L3 계층 구조로 구성된다.

## 1. 개념
캐시 메모리는 **CPU와 RAM 사이에 위치한 고속 버퍼 메모리**다. 자주 사용하는 데이터를 미리 저장하여 메모리 접근 시간을 줄인다.

> 비유: "책상 서랍" - 자주 쓰는 물건을 가까이 둠

## 2. 등장 배경
- CPU 속도: 수 GHz (매우 빠름)
- RAM 속도: 수백 MHz (상대적으로 느림)
- → 속도 차이로 CPU가 대기하는 병목 발생
- → 캐시로 병목 완화

## 3. 캐시 계층 구조

```
CPU ─── L1 Cache ─── L2 Cache ─── L3 Cache ─── RAM
       (최빠름)      (빠름)        (보통)       (느림)
       (작음)        (중간)        (큼)         (가장 큼)

일반적 크기:
- L1: 32~64 KB (코어별)
- L2: 256 KB~1 MB (코어별)
- L3: 8~64 MB (공유)
```

## 4. 지역성 (Locality)

| 지역성 | 설명 | 예시 |
|--------|------|------|
| 시간 지역성 | 최근 접근 데이터 재접근 | 루프 변수 |
| 공간 지역성 | 인접 데이터 접근 | 배열 순차 접근 |

## 5. 캐시 동작

```
데이터 요청:
1. L1 확인 → Hit: 반환
2. L2 확인 → Hit: 반환 + L1에 복사
3. L3 확인 → Hit: 반환 + 상위 캐시 복사
4. RAM 접근 → 반환 + 캐시에 저장

Hit: 캐시에서 찾음 (빠름)
Miss: 캐시에 없음 (느림)
```

## 6. 캐시 성능 지표

```
Hit Rate = Hit 횟수 / 전체 접근 횟수
Miss Rate = 1 - Hit Rate

AMAT (Average Memory Access Time):
= Hit Time + Miss Rate × Miss Penalty

예시:
- L1 Hit Rate: 95%, Hit Time: 1 cycle
- L2 Hit Rate: 90%, Hit Time: 10 cycles
- RAM: 100 cycles

AMAT = 1 + 0.05 × (10 + 0.1 × 100)
     = 1 + 0.05 × 20
     = 2 cycles
```

## 7. 캐시 매핑 방식

| 방식 | 특징 | 장단점 |
|------|------|--------|
| Direct Mapped | 1:1 매핑 | 빠름, 충돌 많음 |
| Fully Associative | 자유 배치 | 느림, 충돌 없음 |
| Set Associative | N-way | 균형 |

## 8. 교체 알고리즘

```
캐시 Miss 시 교체 대상 선정:
- LRU (Least Recently Used): 가장 오래 안 쓴 것
- FIFO: 먼저 들어온 것
- Random: 무작위
- LFU (Least Frequently Used): 적게 쓴 것
```

## 9. 장단점

| 장점 | 단점 |
|-----|------|
| 메모리 접근 가속 | 추가 비용 |
| 시스템 성능 향상 | 일관성 관리 복잡 |
| 전력 효율 | 면적 증가 |

## 10. 실무에선? (기술사적 판단)
- **CPU 설계**: 캐시 크기가 성능 결정
- **멀티코어**: 캐시 일관성(MESI) 핵심
- **프로그래밍**: 캐시 친화적 코드 작성 필요
- **벤치마크**: 캐시 미스가 병목

## 11. 관련 개념
- 지역성 (Locality)
- 캐시 일관성 (Cache Coherence)
- MESI 프로토콜

### Ⅰ. 개요

**캐시 메모리(Cache Memory)**란 CPU와 주기억장치(RAM) 사이의 속도 차이(CPU: GHz급 vs RAM: MHz급)를 완화하기 위해 고속·소용량의 버퍼 메모리를 계층적으로 배치한 저장 장치다.

- **등장 배경**: CPU는 GHz 속도로 동작하나 RAM은 수십~수백 ns의 접근 시간이 필요 → CPU가 메모리를 기다리는 병목(Memory Wall) 발생 → 지역성(Locality) 원리를 활용한 캐시로 해결
- **핵심 목적**: AMAT(평균 메모리 접근 시간) 최소화, CPU 유휴 사이클 감소

---

### Ⅱ. 구성 요소 및 핵심 원리

#### 1. 캐시 계층 구조

| 계층 | 크기 | 접근 시간 | 위치 | 공유 방식 |
|-----|-----|---------|-----|---------|
| **L1 Cache** | 32~64KB | 1~4 cycle | CPU 코어 내부 | 코어 전용 |
| **L2 Cache** | 256KB~1MB | 4~12 cycle | CPU 코어 근처 | 코어 전용 |
| **L3 Cache** | 8~64MB | 30~40 cycle | CPU 칩 공유 | 모든 코어 공유 |
| **RAM** | 수GB~수TB | 100~300 cycle | 외부 | 시스템 전체 |

#### 2. 지역성(Locality) 원리

```
시간 지역성(Temporal Locality):
  최근 접근한 데이터는 가까운 미래에 다시 접근할 가능성이 높음
  예: for 루프의 카운터 변수 i

공간 지역성(Spatial Locality):
  현재 접근한 주소 근처에 있는 데이터도 곧 접근될 가능성이 높음
  예: arr[0], arr[1], arr[2]... 순차 접근

→ 캐시는 이 원리를 기반으로 Block(Cache Line) 단위로 데이터를 미리 올림
```

#### 3. AMAT 계산 공식

```
AMAT = Hit Time + Miss Rate × Miss Penalty

실제 예:
  L1 Hit Rate=95%, Hit Time=1 cycle
  L2 Hit Rate=90%, Hit Time=10 cycles, RAM=100 cycles

  AMAT = 1 + 0.05 × (10 + 0.1 × 100)
       = 1 + 0.05 × 20 = 2 cycles  (→ RAM 직접 접근 대비 50배 빠름)
```

### Ⅲ. 기술 비교 분석

#### 1. 캐시 매핑 방식 비교

| 비교 항목 | Direct Mapped | Set Associative | Fully Associative |
|---------|-------------|---------------|-----------------|
| **매핑 방법** | 주소 → 1개 라인 | 주소 → N개 중 택 1 | 주소 → 전체 중 택 1 |
| **충돌 빈도** | ★ 높음 (Conflict Miss) | 중간 | ★ 없음 |
| **하드웨어 복잡도** | ★ 낮음 | 중간 | ★ 높음 |
| **검색 속도** | ★ 빠름 (1개만 확인) | 보통 (N개 확인) | 느림 (전체 확인) |
| **적합 상황** | L1 캐시 (속도 우선) | L2/L3 (균형) | TLB, 소형 캐시 |

#### 2. 교체 알고리즘 비교

| 알고리즘 | 원리 | Hit Rate | 구현 복잡도 |
|---------|-----|---------|-----------|
| **LRU** | 가장 오래 미사용 교체 | 높음 | 중간 (사용 시간 추적) |
| **FIFO** | 먼저 들어온 것 교체 | 중간 | 낮음 |
| **LFU** | 사용 횟수 가장 적은 것 교체 | 높음 | 높음 (카운터 관리) |
| **Random** | 무작위 교체 | 낮음 | 매우 낮음 |
| **Clock** | LRU 근사 (참조비트) | LRU에 근접 | 낮음 ★ 실무 사용 |

**★ 선택 기준**: L1/L2는 하드웨어 LRU 근사 알고리즘(Pseudo-LRU), 운영체제 페이지 교체는 Clock 알고리즘이 성능·구현 복잡도 균형점에서 가장 적합.

### Ⅳ. 실무 적용 방안

#### 1. 적용 시나리오

| 분야 | 활용 방법 | 기대 효과 |
|-----|---------|---------|
| **CPU 설계** | L1~L3 계층별 크기·속도 최적화 | AMAT 50~100배 개선 |
| **캐시 친화적 프로그래밍** | 데이터 구조를 Cache Line(64B) 정렬로 설계 | Cache Miss 30~50% 감소 |
| **멀티코어 시스템** | MESI 프로토콜 기반 캐시 일관성 관리 | 데이터 정합성 보장 |
| **게임/실시간 시스템** | Hot Data를 캐시 내에 상주시키는 워밍업 전략 | 응답 지연 90% 감소 |

#### 2. 도입 시 고려사항
- **False Sharing**: 서로 다른 코어가 같은 Cache Line 내 다른 데이터를 수정 시 불필요한 일관성 갱신 발생 → 데이터 패딩으로 분리
- **캐시 오염(Cache Pollution)**: 대용량 스트리밍 데이터가 유용한 캐시 데이터를 밀어내는 현상 → 비시간적(Non-temporal) 스토어 명령 활용
- **멀티코어 일관성**: MESI 프로토콜(Modified/Exclusive/Shared/Invalid) 상태 전이 관리

### Ⅴ. 기대 효과 및 결론

#### 기대 효과

| 효과 영역 | 내용 | 정량적 목표 |
|---------|-----|-----------|
| **처리 속도** | 메모리 접근 시간 단축 | RAM 직접 접근 대비 50~100배 향상 |
| **CPU 효율** | 유휴 사이클(Stall) 감소 | CPU 사용률 20~40% 개선 |
| **전력 절감** | 고속 SRAM이 저주파 DRAM 접근 대체 | 메모리 서브시스템 전력 30% 절감 |

#### 결론
> 캐시 메모리는 **지역성 원리를 기반으로 CPU-RAM 속도 격차를 극복하는 핵심 아키텍처**로, 현대 프로세서 성능의 70% 이상이 캐시 설계에 의해 결정된다. 향후 AI 가속기에서는 HBM(High Bandwidth Memory) 및 PIM(Processing-In-Memory)과 결합하여 메모리 병목을 근본적으로 해소하는 방향으로 진화할 것이다.

> **※ 참고 표준**: IEEE Std 1003.1 (POSIX), Intel 64 Architecture SDM, ARM Cortex 기술 레퍼런스

```
1단계 캐시(L1): 바로 옆 친구 🏃
2단계 캐시(L2): 앞자리 친구 🚶
3단계 캐시(L3): 교실 앞 친구
RAM: 도서관 📚
```

**지역성 비밀**:
- **시간**: 어제 본 책은 오늘도 봐요!
- **공간**: 국어책 10쪽을 보면 11쪽도 봐요!

그래서 캐시는 "이거 곧 또 쓰겠네?" 하고 미리 준비해요! 📖
