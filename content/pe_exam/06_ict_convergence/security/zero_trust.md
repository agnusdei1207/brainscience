# Zero Trust (제로 트러스트)

## 핵심 인사이트 (3줄 요약)
> **"절대 신뢰하지 말고 항상 검증하라"**는 보안 모델. 내부/외부 구분 없이 모든 접근 검증. 클라우드 시대의 필수 보안.

### Ⅰ. 개요

#### 1. 개념
Zero Trust(제로 트러스트)는 **네트워크 내부와 외부를 구분하지 않고, 모든 사용자, 기기, 애플리케이션의 접근을 지속적으로 검증하는 보안 아키텍처**이다.

> 비유: "모든 문에 보안 검색대" - 신분증 없인 아무 데도 못 가요

---

### Ⅱ. 구성 요소 및 핵심 원리

#### 3. 제로 트러스트 핵심 원칙
| 원칙 | 설명 |
|-----|------|
| 명시적 검증 | 모든 접근 요청 인증/인가 |
| 최소 권한 | 필요한 만큼만 접근 허용 |
| 침해 가정 | 이미 침해되었다고 가정 |
| 지속적 모니터링 | 실시간 위협 탐지 |

#### 5. 주요 기술
| 기술 | 설명 |
|-----|------|
| IAM | 신원 및 접근 관리 |
| MFA | 다중 인증 |
| SASE | 보안 접근 서비스 엣지 |
| ZTNA | 제로 트러스트 네트워크 액세스 |
| PAM | 권한 있는 접근 관리 |
| SIEM | 보안 이벤트 관리 |

### Ⅲ. 기술 비교 분석

#### 2. 기존 vs 제로 트러스트
```
┌────────────────────────────────────────────────────────┐
│           기존 경계 보안 vs 제로 트러스트             │
├────────────────────────────────────────────────────────┤
│                                                        │
│  기존 경계 보안 (Perimeter Security):                 │
│  ┌────────────────────────────────────────────────┐   │
│  │                                                │   │
│  │  "내부는 안전, 외부는 위험"                    │   │
│  │                                                │   │
│  │           🔥 방화벽 🔥                         │   │
│  │         ┌─────────────┐                       │   │
│  │  외부 → │   내부망    │ → 모두 신뢰           │   │
│  │  🌍    │  ┌───┐┌───┐│                       │   │
│  │         │  │서버││PC ││                       │   │
│  │         │  └───┘└───┘│                       │   │
│  │         └─────────────┘                       │   │
│  │                                                │   │
│  │  문제:                                         │   │
│  │  • 내부 침해 시 확산 쉬움                     │   │
│  │  • 원격근무 대응 어려움                       │   │
│  │  • 클라우드 경계 모호                         │   │
│  │                                                │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
│  제로 트러스트 (Zero Trust):                          │
│  ┌────────────────────────────────────────────────┐   │
│  │                                                │   │
│  │  "아무도 신뢰하지 않음, 항상 검증"             │   │
│  │                                                │   │
│  │  사용자 ──→ 🔐 검증 ──→ 🔐 검증 ──→ 🔐 검증  │   │
│  │            (신원)     (기기)     (권한)       │   │
│  │                              │                 │   │
│  │                              ↓                 │   │
│  │                         [리소스]               │   │
│  │                                                │   │
│  │  핵심 원칙:                                    │   │
│  │  • 최소 권한 (Least Privilege)                │   │
│  │  • 지속적 검증                                 │   │
│  │  • 마이크로 세그멘테이션                      │   │
│  │                                                │   │
│  └────────────────────────────────────────────────┘   │
│                                                        │
└────────────────────────────────────────────────────────┘
```
