# 결정 트리 (Decision Tree) - 설명 가능한 지능적 분류/회귀 알고리즘

> ⚠️ 이 문서는 머신러닝에서 가장 직관적이고 해석 가능한 분류/회귀 알고리즘인 '결정 트리(Decision Tree)'의 개념, 불순도(Impurity) 측정 지표인 엔트로피(Entropy)와 지니 지수(Gini Index), 그리고 트리 성장(Trade-off) 과정에서 발생하는 과적합 문제와剪枝(Pruning) 전략에 대해 심층 분석한다.

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 결정 트리는 특성(Feature)의 값에 따라 데이터를 순차적으로 분할(Split)하는 트리 구조의 알고리즘으로, 각 내부 노드(Internal Node)에서 특정 피처의 임계값(Threshold)을 기준으로 분할하고, 리프 노드(Leaf Node)에서 최종 예측 결과를 출력한다.
> 2. **가치**: 결정 트리의 가장 큰 장점은 '설명 가능성(Interpretability)'이다. 의사결정 과정을_tree diagram_으로 시각화할 수 있어, 의료 진단이나 금융 심사처럼 의사결정의 이유를 설명해야 하는 도메인에서 널리 사용된다.
> 3. **융합**: 결정 트리는 단독으로도 사용되지만, 다수의 결정 트리를 결합한 '랜덤 포레스트(Random Forest)'와 '그래디언트 부스팅(Gradient Boosting)' 등의 앙상블 기법의 기본 구성 요소(Building Block)로도 활용된다.

## Ⅰ. 개요 및 필요성 (Context & Necessity)

### 1."If-Else" 규칙으로 세상을 이해하려는 시도 (Pain Point)
의사가 환자를 진단할 때, 의사는 일련의 질문과 검사 결과를 통해 진단을 내린다.
- "열이 있습니까?" → 예 → "기침을 합니까?" → ...
- 이 과정은 본질적으로 "결정 트리"와 동일한 구조이다.
- **문제 발생**: 이러한 결정 과정을 컴퓨터로自動化하려면, 어떤 질문(분할 기준)을 먼저 해야 가장 정확한 진단을 할 수 있을까?

### 2. 결정 트리의 필요성
결정 트리는 이러한 "자동화된 의사결정"을 위한 알고리즘이다.
- **학습 과정**: 훈련 데이터에서 최적의 분할 기준(피처와 임계값)을 자동으로 찾는다
- **예측 과정**: 새로운 데이터가 입력되면 트리 구조를 따라 leaf node에 도달하여 예측 결과 출력
- **특징**: Black-box 딥러닝과 달리, 모델의 의사결정 과정을 사람이 직접 확인할 수 있다

- **📢 섹션 요약 비유**: 결정 트리는 "진료 프로세스"와 같다. 환자가进来了면, 의사는 열이 있나요 → 기침을 하나요 → 가슴이 아프나요와 같은 질문(노드)을 통해 진단(leaf node)에 도달한다. 각 질문의 순서와 기준(분할 조건)을 경험과 지식을 통해 의사가 선택하듯이, 결정 트리 알고리즘도 훈련 데이터를 통해 최적의 질문과 기준을 자동으로 학습한다.

### 3. 불순도 측정: 엔트로피와 지니 지수

**엔트로피 (Entropy):**
$$H(S) = -\sum_{i=1}^{c} p_i \log_2(p_i)$$

여기서 $p_i$는 클래스 $i$의 비율, $c$는 전체 클래스 수이다.

**지니 지수 (Gini Index):**
$$G(S) = 1 - \sum_{i=1}^{c} p_i^2$$

**정보 이득 (Information Gain):**
$$IG(S, A) = H(S) - \sum_{v \in Values(A)} \frac{|S_v|}{|S|} H(S_v)$$

| 지표 | 수식 | 특징 | 범위 |
|:---|:---|:---|:---|
| **엔트로피** | $H(S) = -\sum p_i \log_2(p_i)$ | 정보 이론 기반, 완전 순수 시 0 | [0, 1] |
| **지니 지수** | $G(S) = 1 - \sum p_i^2$ | 불순도 측정, 계산 효율적 | [0, 0.5] |
| **분류 오차** | $E(S) = 1 - \max(p_i)$ | 가장 간단하나 마핀分化에 둔감 | [0, 0.5] |

### 4. 트리 성장 과정

| 단계 | 설명 |
|:---|:---|
| **1. Root Node 분할** | 전체 데이터에서 정보 이득이 가장 큰 Feature와 임계값 선택 |
| **2. 하위 노드 분할** | 각 하위 노드에서 동일한 과정 반복 |
| **3. 정지 조건 확인** | 노드의 샘플 수 < 최소 샘플 수, 최대 깊이 도달, 순도 100% 등 |
| **4. Leaf Node 할당** | 정지 조건 만족 시 해당 노드를 leaf로 변환 (다수 클래스 할당) |

### 5. 과적합 문제와 전제/후레 포레스트

- **과적합 원인**: 트리가 너무 깊게 성장하여 훈련 데이터의 노이즈까지 학습
- **Pre-pruning (사전 방지)**: 최대 깊이, 최소 샘플 수, 최소 리프 수 등으로成長 제한
- **Post-pruning (사후 방지)**: 전체 트리를 성장시킨 후, 검증 데이터에서 성능 향상에 기여하지 않는 가지 제거

- **📢 섹션 요약 비유**: 결정 트리의成長 관리는 "키가 너무 크면倒れない庭の木"와 같다. 아무리 성장하게 두면 모든 방향으로 가지를 뻗어 풍성한듯 보여도, 폭풍이 오면 가장 먼저 부러진다. 그러나 너무 적게 성장시키면庭の美观도 떨어지고 음의也因此 줄어든다. 적절한 크기로修剪(포레스트)해야 아름다운同时に음식도 풍성한 과일을 맺는다.

## Ⅲ. 비교 및 기술적 트레이드오프 (Comparison & Trade-offs)

### 결정 트리 vs 다른 알고리즘 비교

| 구분 | 결정 트리 | 로지스틱 회귀 | SVM | KNN |
|:---|:---|:---|:---|:---|
| **학습 방식** | 분할 기반 | 확률적 경사 하강 | 마진 최대화 | 거리 기반 |
| **해석 가능성** | 매우 높음 (트리 시각화) | 높음 (계수 해석) | 낮음 | 낮음 |
| **비선형 처리** | 우수 | 제한적 | 커널 필요 | 우수 |
| **훈련 속도** | 빠름 | 빠름 | 중간 | 느림 (사전 계산) |
| **과적합 위험** | 높음 | 낮음 | 중간 | 데이터 의존적 |

### 지니 vs 엔트로피 비교

| 기준 | 지니 지수 | 엔트로피 |
|:---|:---|:---|
| **계산 복잡도** | 제곱 연산만으로 간단 | 로그 연산 필요, 약간 느림 |
| **민감도** | 약간 편향됨 (큰 클래스 선호) | 균형 잡힌 분리 선호 |
| **일반적 성능** | 둘 다 유사한 성능 | 둘 다 유사한 성능 |
| **실무 권장** | 기본값으로 충분 | 이진 분류에서 선호 |

### Pre-pruning vs Post-pruning 비교

| 기법 | 설명 | 장점 | 단점 |
|:---|:---|:---|:---|
| **Pre-pruning** |成長 과정에서 일찍 중지 | 계산 효율적, 과적합 방지 | 최적 분할을 놓칠 수 있음 |
| **Post-pruning** | 성장 후 불필요한 가지 제거 | 더 정확한 트리 가능 | 계산 비용 높음 |

- **📢 섹션 요약 비유**: 결정 트리의 분할 기준 선택은 "음식 재료의選別"과 같다. 엔트로피는 "각 재료가 담긴 정보(營養)를 최대한 활용하려면 어떻게 자라야 할까"를 계산하고, 지니 지수는 "재료를 나눌 때 불순물(노이즈)을 얼마나 최소화할 수 있을까"를 측정한다. 둘 다 재료의品質을評価하는 다른 관점이며, 결과는 대체로 유사하다.

## Ⅳ. 실무 판단 기준 (Decision Making)

| 고려 사항 | 세부 내용 | 주요 아키텍처 의사결정 |
|:---|:---|:---|
| **도입 환경** | 기존 레거시 시스템과의 호환성 분석 | 마이그레이션 전략 및 단계별 전환 계획 수립 |
| **비용(ROI)** | 초기 구축 비용(CAPEX) 및 운영 비용(OPEX) | TCO 관점의 장기적 효율성 검증 |
| **보안/위험** | 컴플라이언스 준수 및 데이터 무결성 보장 | 제로 트러스트 기반 인증/인가 체계 연계 |

*(추가 실무 적용 가이드 - 금융 카드 승인 모델)*
- **상황**: 은행이 새로운 신용 카드 신청자의 승인 여부를 결정하는 모델을 개발 중. 규제 요건으로 인해 모델의 의사결정을 설명할 수 있어야 한다.
- **실무 의사결정**:
  1. **해석 가능성 요구사항**: 금감원(금융위원회) 규제 요건상 모델이 "왜 이 고객을 거절했는가"를 설명할 수 있어야 함 → 결정 트리 선택
  2. **하이퍼파라미터 설정**: 최대 깊이 5~7, 최소 샘플 수 100~200으로 제한하여 과적합 방지
  3. **지니 지수 선택**: 엔트로피보다 계산이 빠르고 성능 차가 크지 않으므로 지니 지수를 기본값으로 사용
  4. **시각화 활용**: 최종 트리를 PNG로 export하여 심사 담당자가 고객 거절 사유를 설명할 때 활용
  5. **한계 인식**: 단일 결정 트리의 예측 성능이 랜덤 포레스트보다 낮을 수 있으므로, 성능 요구사항에 따라 앙상블 고려

- **📢 섹션 요약 비유**: 금융 분야 결정 트리 적용은 "대출 심사의사결정 프로세스"와 같다.支行장(의사결정 트리)이 심사할 때, "연봉이 5000만 이상인가요? → 예 → 부和相关知识 있는가요? → ..."와 같은 질문을 통해 심사한다. 이 프로세스는 감사 당국에 보고서로 제출될 만큼 명확하고 투명하다. 이것이 결정 트리의 가장 큰 강점—'설명 가능한 인공지능(Explainable AI)'의 реали화이다.

## Ⅴ. 미래 전망 및 발전 방향 (Future Trend)

1. **결정 트리와 딥러닝의 융합 (Neural Decision Trees)**
   전통적인 결정 트리는 사람이 설계한 규칙 기반이었지만, 향후에는 신경망의 표현 학습能力和결정 트리의 해석 가능성을 결합한 'Neural Decision Trees' 연구가 주목받고 있다. 이 접근법은 Feature를 자동으로 학습하면서 동시에 의사결정 과정을 해석 가능한 트리 구조로 나타낼 수 있다.

2. **설명 가능한 인공지능(XAI)에서의 결정 트리 역할**
   딥러닝 모델의 예측을 설명하기 위해, 딥러닝 모델의 근사(Approximation) 역할을 하는 ' المحلية 결정 트리(Local Decision Tree)'를 활용하는 기법이 연구되고 있다. 복잡한 신경망의 예측을 단순한 결정 트리로 근사화하여, 블랙박스 모델의 의사결정을 설명할 수 있게 한다.

- **📢 섹션 요약 비유**: 결정 트리의 미래는 "햇빛과 바람을 동시에 활용하는 미래建筑"과 같다. 과거의 결정 트리는 순수한 自然 에너지만 활용하는 단순한 구조물이었다(전통적 규칙). 그러나 미래에는 태양광 발전(딥러닝의 표현력)과 통풍 설계(결정 트리의 해석 가능성)를 모두 갖춘 획기적인 구조물(Neural Decision Trees)로 발전할 것으로 기대된다.

## 🧠 지식 맵 (Knowledge Graph)

*   **결정 트리 핵심 개념**
    *   **엔트로피 (Entropy)**: 정보의 불확실성 측정, $H(S) = -\sum p_i \log_2(p_i)$
    *   **지니 지수 (Gini Index)**: 불순도 측정, $G(S) = 1 - \sum p_i^2$
    *   **정보 이득 (Information Gain)**: 분할 전후 불순도 감소량, $IG = H_{parent} - \sum H_{child}$
*   **과적합 방지 기법**
    *   Pre-pruning (최대 깊이, 최소 샘플 수 제한)
    *   Post-pruning (검증 데이터 기반 가지 제거)
*   **앙상블 기본 요소**
    *   랜덤 포레스트, 그래디언트 부스팅 등 앙상블 기법의 구성 요소
