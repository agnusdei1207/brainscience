# 28. 감리 계획 수립

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 감리 계획 수립은 감리의品質을 결정하는 가장 중요한 단계로, 예비조사를 통해 식별된 위험을 기반으로 감리 범위, 방법론, 인력, 일정을 전략적으로 설계하는 활동이다.
> 2. **핵심 요소**: 위험 평가 결과, 이용 가능한 감리 자원, 프로젝트 일정 제약이 상호 균형 있게 반영되어야 하며, 지나치게 ambitious한 계획은 실행 불가능하고, 너무 보수적인 계획은 감리 효과를淡化시킨다.
> 3. **동적 관리**: 감리 계획은 고정된 문서가 아니라 프로젝트 진행 과정에서 수집되는新정보에 따라 적응적으로調整되는 living document이다.

---

### Ⅰ. 개요 및 필요성 (Context & Necessity)

감리 계획 수립(Audit Planning)은 감리 업무의品質과 Effectiveness를 결정하는 가장 근본적인 단계이다. 아무리 숙련된 감리인이 투입되더라도, 체계적인 계획 없이 현장에 투입되면 중요한 영역을 누락하거나, 불필요한 영역에 자원을 낭비하게 된다. 감리 계획은 프로젝트의羅針盤과 같이 감리 전 과정의 방향을设定하고, 감리팀 모두가 공유하는共同言語로서 기능한다.

감리 계획 수립의 필요성은 네 가지 측면에서 설명된다. **첫째, 자원 한정성**으로, 감리Budget과 일정은 무한하지 않으므로 가장 위험이 높은 영역에 자원을集中投下해야 한다. **둘째, 범위 명확성**으로, 감리 범위가 모호하면 감리인과 피감리인 간에 범위 논쟁이 발생하여 업무가 지연된다. **셋째, 기대치 조정**으로, 감리 결과에 대한 발주자의 기대와 실제 deliver 가능한 결과의 괴리를 사전에 관리해야 한다. **넷째, 증거 적정성**으로, 감사에서 사용할 방법론과 증거 수집 전략을 사전에 설계해야 사후에証拠不十分问题가 발생하지 않는다.

다음 다이어그램은 감리 계획 수립 프로세스의 입력-처리-출력 흐름을 보여준다.

```text
┌─────────────────────────────────────────────────────────────┐
│ [감리 계획 수립 프로세스]                                    │
│                                                             │
│  [입력 요소]                                                 │
│  ├─ 과업 내용서 (TOR, Terms of Reference)                   │
│  ├─ 제안서 (사업자의 기술/가격 제안)                         │
│  ├─ 기존 감리 이력 (이전 감리 지적 사항)                     │
│  ├─ 프로젝트 위험 평가 결과                                  │
│  └─ 이용 가능한 감리 자원 (인력, 도구, 예산)                  │
│              │                                               │
│              ▼                                               │
│  [계획 수립 활동]                                            │
│  ├─ 1단계: 예비조사 (Preliminary Survey)                    │
│  ├─ 2단계: 위험 평가 및 감리 범위 결정                       │
│  ├─ 3단계: 감리 방법론 선정                                 │
│  ├─ 4단계: 인력 및 일정 계획                                │
│  └─ 5단계: 감리 계획서 작성 및 검토                          │
│              │                                               │
│              ▼                                               │
│  [출력 산출물]                                               │
│  ├─ 감리 계획서 (Audit Plan)                                │
│  ├─ 감리 체크리스트 (Audit Checklist)                       │
│  ├─ 인력 배치 계획 (Resource Assignment)                    │
│  └─ 위험 기반 감리 전략서 (Risk-based Audit Strategy)      │
└─────────────────────────────────────────────────────────────┘
```

이 흐름의 핵심은 '입력에서 출력까지 단방향이 아니라 피드백 루프'가 존재한다는 점이다. 수립된 계획이実行 단계에서 문제가 드러나면 계획으로 돌아가 수정해야 하며, 이는 감리 계획이 고정된 것이 아니라 동적으로 관리되는文書の根拠이다.

📢 **섹션 요약 비유**: 감리 계획 수립은여행旅行策划와 같다.目的地(감리 대상),移動手段(감리 방법론), 경비(감리 자원),schedule(감리 일정)를 미리設計하지 않으면, 아무 장비 없이 모르는 장소로 헛배게 되어 時間과 비용만 낭비한다.入念한 여행 계획이成功的 여행의前提条件이다.

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**2.1 예비조사 (Preliminary Survey)**

예비조사는 감리 계획의 품질을 결정하는 핵심 입력이다. 감리인은 프로젝트의全貌를把握하기 위해 피감리인 인터뷰, 문서 분석, 시스템preview을 수행한다. 예비조사에서는 특히 다음 항목을重点적으로 수집해야 한다.

```text
[예비조사 핵심 수집 항목]
   │
   ├──▶ 사업 개요
   │    ├─ 프로젝트 목적 및 범위 (RFP 대비)
   │    ├─ 사업 규모 (예산, 일정, 인력)
   │    └─ 주요 이해관계자 (발주자 사업담당, 사업자 PM)
   │
   ├──▶ 기술 환경
   │    ├─ 시스템 아키텍처 (온프레미스/클라우드/혼합)
   │    ├─ 적용 기술 스택 (언어, 프레임워크, DB)
   │    └─ 기존 시스템 연동 구조
   │
   ├──▶ 과거 감리 이력
   │    ├─ 이전 감리 지적 사항 및 조치 완료 여부
   │    ├─ 반복 지적 항목 (相同 결함 repeated occurrence)
   │    └─ 피감리인의 감리 대응 역량 평가
   │
   └──▶ 특수 요건
       ├─ 법적/규제 요건 (금융, 의료, 공공 등)
       ├─ 보안 등급 (국가机密, 一般, etc.)
       └─ SLA 수준 요구사항
```

**2.2 위험 기반 감리 범위 결정**

예비조사 결과를 바탕으로 위험 평가를 수행하고, 그 결과에 따라 감리 범위와 자원 배분을 결정한다. 위험 평가는定性的(qualitative) 방법과定量的(quantitative) 방법이 있으며, 대규모 프로젝트에서는两者을병행使用한다.

定性的 위험 평가는 위험 발생 가능성(Probability)과 영향도(Impact)의 2차원 매트릭스로 분류한다. 가능성이 높고 영향도가 큰 영역이 감리의 우선순위가 된다.定量的 위험 평가는 과거 프로젝트 데이터(결함 density, 장애 발생률 등)를 활용하여숫자化한다.

```text
[위험 기반 감리 우선순위 결정 매트릭스]
┌─────────────────────────────────────────────────────────┐
│                    영향도 (Impact)                       │
│           │  낮음    │  보통    │  높음    │  치명적   │
│ 가능성 ─────────────────────────────────────────────────  │
│  높음    │ 보통(M)   │ 高(H)    │ 极高(E)   │ 极高(E)   │
│  보통    │ 低(L)    │ 보통(M)   │ 高(H)    │ 极高(E)   │
│  낮음    │ 低(L)    │ 低(L)    │ 보통(M)   │ 高(H)    │
│  매우 낮음│ 低(L)    │ 低(L)    │ 低(L)    │ 보통(M)   │
└─────────────────────────────────────────────────────────┘
│ E=즉시 감리, H=유사 사업 후 우선 감리, M=표준 감리, L=감리省略検討 │
```

**2.3 감리 방법론 선정**

감리 방법론은 프로젝트 특성에 따라 선택해야 한다. 폭포수 모델 기반 사업에서는단계별 감리(3단계 또는 2단계)가 적합하고, 애자일 모델 기반 사업에서는迭代(Milestone) 감리 또는 지속적 감리가 적합하다. 또한 감리 대상의特性에 따라 문서 검토 중심, 현장检查 중심, 도구 분석 중심 등 감리手法를 조합해야 한다.

**2.4 인력 및 일정 계획**

감리 인력은 프로젝트의 기술 분야와 감리 영역에 맞는 전문성을 갖춘 인원을 배치해야 한다. 예를 들어, 데이터베이스 성능 감리에는 DB 전문 경력의 감리인이 필요하고, 보안 감리에는 시큐어 코딩 진단 능력을 갖춘 정보보안 전문관이 필요하다. 인력 계획에서는 감리인의 독립성(Conflict of Interest check)도 반드시 확인해야 한다.

일정 계획에서는 프로젝트의 마일스톤과 감리 시점을Align해야 한다. 설계 감리의 경우 설계 완료 시점 전에 감리 범위 설정이 완료되어야 하고, 종료 감리의 경우 인수テスト完了 후 바로 감리에 착수할 수 있어야 한다.

📢 **섹션 요약 비유**: 감리 계획은 의사의 진료 계획과 같다. 의사는 환자를 만나기 전에病歷를 읽고(예비조사), 어떤 검사부터 할지(위험 평가), 검사 결과를怎么看解读할지(감리 방법론)를 미리 준비한다. 준비 없이 진료를 시작하면 중대한 병을 놓치거나 불필요한 검사를 반복하게 된다.

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

감리 계획 수립에서 사용되는 주요 방법론들의 장단점을 비교分析하여, 상황에 맞는 적절한 방법론 선택 기준을 제시한다.

**위험 기반 감리 vs 산출물 기반 감리**: 위험 기반 감리(Risk-based Audit)는 리스크가 높은 영역에 자원을集中投下하므로 효율적이지만,某些 영역이 과소평가될 위험이 있다. 산출물 기반 감리(Deliverable-based Audit)는 사전에 정의된 산출물 리스트를 기반으로 검증을 수행하므로漏失는防止할 수 있지만, 리스크가 낮은 영역에도 동일한 자원이投入되어 비효율적이다.

```text
┌──────────────── 감리 방법론 비교 ─────────────────────────┐
│                                                         │
│  방법론           │  장점           │  단점              │
│  ───────────────────────────────────────────────────────  │
│  위험 기반 감리    │ 자원 효율성,    │某些 영역          │
│                   │ 핵심 결함 집중 탐지│과소평가 가능     │
│  산출물 기반 감리  │漏失防止,        │비효율성,          │
│                   │ 객관적 증거 확보 │저위험 과감리      │
│  목적 기반 감리    │ 목표 달성,      │ 범위漂移 위험,    │
│                   │ 이해관계자 합의   │주관성 개입 가능   │
│  지속적 감리      │ 실시간 위험 탐지, │ 자원 과다,        │
│  (애자일 환경)    │ 빠른 피드백     │ 형식화 어려움     │
└─────────────────────────────────────────────────────────┘
```

**폭포수 vs 애자일 프로젝트의 감리 계획 차이**: 폭포수 모델에서는 감리 계획이 프로젝트 초기에一次性으로 완성되며, 이후 큰 변경 없이执行된다. 애자일 모델에서는 각 스프린트 시작 전에 감리 범위를 조정하는，适应적 계획(Adaptive Planning)이 필요하다. 실무에서는 두 모델의 hybrid 형태가 많으므로, 감리 계획도それに合わせて 유연하게 설계되어야 한다.

**내부 감리 vs 외부 감리 계획의 차이**: 내부 감리 조직에서는 연간 감리 계획(Annual Audit Plan)을 수립하여 연간 전체 감리 활동을plan한다. 이는 조직 전체의 감리 자원을年度为单位)로 배분하는 전략적 계획이다. 외부 감리(제3자 감리)에서는 특정 프로젝트에 대한 한정된 감리 계획만을 수립하며, 이는 프로젝트별로 독립적으로作成된다.

감리 계획의质量은감리 결과의質을直接左右한다. Therefore, 감리 계획서 검토 단계에서项目经理(PM)와 senior auditor의入念な_reviewが必不可少である. 계획서 리뷰에서 확인해야 할 핵심 항목은 다음과 같다.

```text
[감리 계획서 리뷰 체크리스트]
   ├──▶ 감리 범위의 충분성 (모든 중요한 영역 coverage)
   ├──▶ 방법론의 적절성 (프로젝트 특성과의 부합성)
   ├──▶ 인력의 전문성 충족 여부 (必需的 기술 스킬 보유)
   ├──▶ 일정의 현실성 (실행 가능 여부, 겹침 检查)
   ├──▶ 증거 수집 전략의 적정성 (6하 원칙 충족 가능?)
   └──▶ 피감리인과의 협력 체계 (인터뷰 일정, 자료 요청 등)
```

📢 **섹션 요약 비유**: 감리 계획 수립은 건축の設計도면 작성과 같다. 지하층,<Layer>, 지붕까지 모든 층을 동일한 깊이로 설계하면 비용이 과다해지고, 중요한 층을 빼먹으면 건물이 무너진다. 적절한 설계도면(감리 계획)이 있어야 施工(감리执行)가 계획대로進行되고, 완성된 건물(시스템)이 견고하다.

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

감리 계획 수립에서 기술사적 판단이 필요한 핵심 영역은 감리 범위의 Grenze 설정, 방법론 선택, 그리고plans의 reality 여부 판단이다.

**4.1 감리 범위의 Grenze 설정**

감리 범위가 넓으면 모든 영역을浅浅하게밖에 검토하지 못하고, 좁으면 중요한 영역을 놓칠 수 있다. 기술사적 판단의 핵심은 프로젝트에서 실패할 때 가장 큰 영향을 미치는 'クリティカル path' 영역을 식별하고, 그 영역에 감리 자원을集中投下하는 것이다.

예를 들어, 전자상거래 시스템에서 결제 처리는クリティカル path이지만, 관리자용 리포트 생성 기능은クリティカル path가 아니다. Therefore, 결제 처리 모듈의 감리는 코드 한 줄 한 줄을 자세히 검토해야 하고, 리포트 생성 기능은 산출물 존재 여부만 확인하면 된다.

```text
[감리 깊이 결정 원칙]
   │
   ├──▶ 시스템 운영에 필수적 (クリティカル path)
   │    └─▶ 코드 100% 리뷰 + 동적 테스트 + 보안 심화
   │
   ├──▶ 중요한 시스템 (重要业务)
   │    └─▶ 주요 시나리오 리뷰 + 샘플링 기반 테스트
   │
   ├──▶ 일반 업무 지원 시스템
   │    └─▶ 산출물 중심 검토 + 인터뷰
   │
   └──▶ 단순 문서化 시스템
        └─▶ 문서 존재 여부 확인만
```

**4.2 감리 일정 딜레마**

감리 일정 계획에서 가장 빈번한 딜레마는 프로젝트 일정이 촉박한데 감리 범위가 넓을 때이다. 기술사적 판단의 원칙은 '중요한 것을 먼저'이다. 모든 영역을同等하게 감리할 시간이 없으면, 가장 위험이 높은 영역부터 감리하고, 시간이 남으면 다음 영역으로 확대한다.

**4.3 계획서의 reality 판단**

감리 계획이 واقع적(실행 가능한 것)인지 판단하는 기준은 세 가지이다. **첫째, 자원 충족 가능성**으로, 계획된 인력 수 × 계획된 일정이 Budget 범위 내에서 충족 가능한가? **둘째, 피감리인 협력 가능성**으로, 감리 과정에서 필요한 자료와 인터뷰에 피감리인이 응할 수 있는가? **셋째, 예기치 못한 상황 대처 가능성**으로, 프로젝트 이슈 발생 시 감리 일정을 조정할 여유가 있는가?

計画が非現実적이라면, 감리项目经理는 발주자에게直ちに 보고하여 현실적인 계획으로 수정해야 한다.無理한 계획을 그대로执行하면 감리의質이 저하되고, 결과적으로プロジェクトに害を及ぼす。

📢 **섹션 요약 비유**: 감리 계획은 음식 만들기 레시피와 같다. 레시피에서 중요한 재료와 시간이 누락되면 맛있는 요리가 만들어지지 않듯이, 감리 계획에서도 중요한 감리 영역과 현실적인 일정이 반드시 설계되어야 한다. 레시피가非現実적이라면 처음부터 다시 써야 하듯이, 계획도 현실적이지 않으면 수정해야 한다.

### Ⅴ. 기대효과 및 결론 (Future & Standard)

체계적인 감리 계획 수립의 기대효과는 감리 업무의效率と品質을 동시에 향상시키는 데 있다. 입念한 계획 덕분에 감리팀은 현장에서 불필요한彷徨なく 곧바로 핵심 영역에 집중할 수 있으며, 피감리인侧에서도 감리 일정을 미리 파악하여資料準備에 대비할 수 있다.

실제로 감리 계획이 잘 수립된 프로젝트와 그렇지 않은 프로젝트의 차이는顕著하다. 계획이 잘 된 프로젝트에서는 감리 지적 사항의 수가 적지만質이 높고,計画缺失의 프로젝트에서는 많은 지적이 나오지만 그것들이 표면적인 것에 그쳐深层 결함은 놓치는傾向이 있다.

| 구분 | 계획 미흡 시 | 계획 충실 시 |
|:---|:---|:---|
| 감리 효율 | 자원이 분산, 핵심 영역 감리 부족 | 핵심 집중, 자원 낭비 최소화 |
| 피감리인 대응 | 갑작스런 자료 요청에 비난 | 사전 준비로 원활한 협력 |
| 감리 품질 |漏失 발생, 증거 불충분 |漏失防止, 증거 확실 |
| 일정 관리 | 잦은遅延, 범위 늘어남 | 계획대로進行, 적시完了 |
| 분쟁 예방 | 결과에 대한 이견 빈발 | 사전 합의로 분쟁 감소 |

미래에는 AI 기반 감리 계획 도구가 예비조사 결과를 분석하여 자동으로 감리 범위와 방법론을 추천하는 것이 가능해질 것이다. 또한 과거 감리 프로젝트의 성과 데이터를 기반으로 성공적인 감리 계획의 패턴을抽出하는 기계학습 모델도 연구되고 있다.

📢 **섹션 요약 비유**: 감리 계획은 축구 경기의 作戦計画과 같다. 감독이 선수의 힘, 상대 팀의 약점, 경기장을 분석하여 전략을 세우면 선수들이それに合わせて 움직인다.作戦がないままグラウンドに立つと混乱が生じ、計画通りに進めれば勝利の可能性が大幅に 향상된다.
