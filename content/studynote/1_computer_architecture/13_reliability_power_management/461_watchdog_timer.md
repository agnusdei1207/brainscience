> **Insight**
> - 워치독 타이머(Watchdog Timer, WDT)는 컴퓨터 시스템이나 마이크로컨트롤러(MCU)가 소프트웨어 오류나 하드웨어 결함으로 인해 무한 루프(Infinite Loop)에 빠지거나 멈추는 현상(Hang-up)을 감지하고, 시스템을 자동으로 재부팅(Reset)하여 정상 상태로 복구하는 하드웨어 타이머 장치입니다.
> - 인간의 개입이 불가능한 원격지 장비, 우주 탐사선, 임베디드 시스템, IoT 기기 등에서 시스템의 무인 연속 가동성(Unattended Continuous Operation)을 보장하는 최후의 방어선 역할을 합니다.
> - 단순히 시스템을 껐다 켜는 것을 넘어, 예기치 않은 소프트웨어 결함으로부터 하드웨어적 수준의 복원력(Resilience)을 제공하는 핵심 결함 허용(Fault Tolerance) 메커니즘입니다.

## Ⅰ. 워치독 타이머(WDT)의 개요 및 동작 원리

### 1. 워치독 타이머(Watchdog Timer)의 정의
워치독 타이머는 시스템(주로 CPU나 MCU)의 정상적인 작동 상태를 감시하는 독립적인 타이머 회로입니다. 시스템은 이 타이머가 만료(Timeout)되기 전에 주기적으로 타이머를 초기화(Clear 또는 "Kick the dog")해야 합니다. 만약 시스템이 먹통이 되어 타이머를 초기화하지 못하고 카운터가 0(Zero)에 도달하면, 워치독 회로는 즉각적으로 하드웨어 리셋 신호(Reset Signal)를 발생시켜 시스템을 강제로 재부팅시킵니다.

### 2. "Kick the Dog" (워치독 밥 주기) 동작 매커니즘
정상적인 프로그램 코드 루프 안에는 워치독 타이머를 리셋하는 명령어가 삽입되어 있습니다. 
* **정상 상태:** 프로그램이 정상 루프를 돌면서 타이머가 만료되기 전에 주기적으로 타이머를 초기값으로 돌려놓습니다.
* **오류 상태:** 소프트웨어가 교착 상태(Deadlock), 무한 루프에 빠지거나 메모리 오류로 프로그램 카운터(PC)가 엉뚱한 곳으로 튀면, 타이머 리셋 코드가 실행되지 않습니다. 결국 타이머가 오버플로우(Overflow)를 일으켜 리셋을 촉발합니다.

> 📢 **섹션 요약 비유:**
> 경비견(워치독)에게 주인이 1시간마다 간식을 주기로 약속했습니다. 만약 1시간이 지나도 주인이 간식을 주지 않으면(주인에게 문제가 생겨 기절했다면), 경비견이 크게 짖고 비상벨을 눌러(리셋 신호) 주인을 깨우는 시스템입니다.

## Ⅱ. 워치독 타이머의 하드웨어 아키텍처

워치독 타이머는 메인 CPU와 독립적으로 동작해야 신뢰성을 보장할 수 있습니다.

```ascii
                      +----------------------------------+
                      |         Microcontroller (MCU)    |
                      |                                  |
                      |   +-----------------------+      |
[ Watchdog Reset ]<---|---| System Reset Logic    |      |
      |               |   +-----------------------+      |
      |               |             ^                    |
      |               |             | (Timeout Signal)   |
      |               |   +-----------------------+      |
      +---------------|-->| Watchdog Timer (WDT)  |      |
                      |   | - Independent Clock   |      |
                      |   | - Counter Register    |      |
                      |   +-----------------------+      |
                      |             ^                    |
                      |             | (WDT Clear Command)|
                      |   +-----------------------+      |
                      |   | CPU Core (Software)   |      |
                      |   | - Main Loop           |      |
                      |   | - "Kick the Dog"      |      |
                      |   +-----------------------+      |
                      +----------------------------------+
```

### 아키텍처의 핵심 요소
1. **독립적인 클럭 소스 (Independent Clock Source):** 메인 시스템 클럭(System Clock)이 고장 나더라도 워치독은 작동해야 하므로, 내부적으로 별도의 저전력 RC 오실레이터(Oscillator) 클럭을 사용하는 경우가 많습니다.
2. **리셋 회로 연결 (Reset Circuitry):** WDT의 만료 신호는 NMI(Non-Maskable Interrupt, 무시할 수 없는 인터럽트)나 하드웨어 리셋 핀과 직접 연결되어, 소프트웨어가 아무리 심각하게 꼬여있어도 하드웨어 레벨에서 전원을 끊었다 켜는 수준의 리셋을 보장합니다.

> 📢 **섹션 요약 비유:**
> 알람 시계를 맞출 때 핸드폰(메인 CPU) 배터리가 방전되어도 울릴 수 있도록, 태엽으로 돌아가는 완전히 독립된 기계식 탁상시계(독립 클럭 워치독)를 하나 더 맞춰놓는 것과 같습니다.

## Ⅲ. 내부 워치독(Internal WDT)과 외부 워치독(External WDT)

워치독 타이머는 칩 내부에 내장되는 방식과 외부의 별도 칩으로 구성되는 방식으로 나뉩니다.

| 구분 | Internal WDT (내부 워치독) | External WDT (외부 워치독) |
| :--- | :--- | :--- |
| **위치** | MCU나 CPU 다이(Die) 내부에 통합됨 | CPU 외부의 별도 IC (Integrated Circuit) |
| **장점** | 비용이 저렴, 공간 제약 없음, 설정이 용이함 | 메인 칩 전체의 전원이나 물리적 결함에도 완벽히 독립적으로 동작하여 신뢰성이 극도로 높음 |
| **단점** | 칩 자체의 전원 결함이나 극단적인 하드웨어 오류 시 WDT도 같이 죽을 수 있음 | 회로 복잡도 증가, BOM(Bill of Materials) 비용 추가 |
| **적용 분야** | 일반적인 IoT 기기, 가전제품 | 우주 항공, 자동차 ECU, 인명 유지 장치 (의료기기) |

> 📢 **섹션 요약 비유:**
> 내부 워치독이 건물 내부 보안팀이 스스로 순찰을 도는 것이라면, 외부 워치독은 건물 밖에서 전문 사설 경비업체가 건물을 감시하다가 연락이 두절되면 외부에서 전기를 차단해버리는 훨씬 더 강력하고 독립적인 통제 방식입니다.

## Ⅳ. 윈도우 워치독 타이머 (Windowed Watchdog Timer)

기본적인 WDT의 맹점을 보완하기 위해 고안된 발전된 형태의 타이머입니다.

* **기존 WDT의 한계:** 만약 소프트웨어 버그로 인해 프로그램이 아주 좁은 무한 루프에 빠졌는데, 공교롭게도 그 루프 안에 "워치독 초기화(Kick)" 코드가 포함되어 있다면, WDT는 시스템이 정상이라고 착각하고 리셋을 발생시키지 않습니다.
* **Windowed WDT의 원리:** 워치독을 초기화할 수 있는 **'허용된 시간 창(Window)'**을 설정합니다. 타이머 값이 너무 늦게 초기화되어도(Timeout) 리셋되지만, 반대로 **너무 빨리 초기화되어도(Too Early)** 리셋을 발생시킵니다.
* **효과:** 프로그램 실행 타이밍이 아주 정확하게 맞을 때만 워치독이 초기화되므로, 무한 루프 내에서의 비정상적인 빠른 연속 초기화 공격(?)이나 소프트웨어 꼬임 현상을 완벽하게 잡아냅니다.

> 📢 **섹션 요약 비유:**
> 경비견에게 밥을 주기로 한 약속 시간이 "정확히 오후 1시부터 1시 10분 사이(Window)"입니다. 주인이 1시 20분에 늦게 줘도 물고, 주인이 미쳐서 오전 11시에 너무 일찍 밥을 주려고 해도 "정상적인 주인이 아니다!"라고 판단하고 경보를 울리는 매우 똑똑한 경비견입니다.

## Ⅴ. 워치독 타이머 적용 시 주의사항 (Best Practices)

워치독 시스템을 제대로 구현하지 않으면 오히려 정상적인 작동을 방해하는 애물단지가 될 수 있습니다.

1. **초기화(Kick) 위치 선정의 중요성:** WDT 초기화 코드는 반드시 프로그램의 '메인 루프(Main Loop)'의 가장 핵심적인 위치에 단 한 곳에만 배치해야 합니다. 타이머 인터럽트(Timer Interrupt) 안에서 WDT를 초기화하면 메인 프로그램이 죽어도 인터럽트만 돌면서 WDT를 속일 수 있으므로 절대 피해야 합니다.
2. **초기 부팅 지연 시간 고려:** 시스템이 처음 전원이 켜지고 OS나 소프트웨어가 부팅을 완료하기 전까지는 WDT 초기화 코드가 실행될 수 없습니다. 따라서 부팅 시간 동안은 WDT가 만료되지 않도록 초기 타임아웃 값을 넉넉하게 주거나, 부팅 후 WDT를 활성화(Enable)하도록 설계해야 합니다.

> 📢 **섹션 요약 비유:**
> 심장 박동기(워치독)를 손가락 끝(인터럽트)에 달아놓으면, 사람이 뇌사 상태에 빠져도 손가락만 움찔거리면 살아있다고 착각할 수 있습니다. 가장 중요한 심장 한가운데(메인 루프)에 달아두어야 시스템의 진정한 생존 여부를 확인할 수 있습니다.
