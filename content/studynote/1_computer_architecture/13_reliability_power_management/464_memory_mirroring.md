> **핵심 인사이트**
> 1. **메모리 미러링(Memory Mirroring)**은 RAID 1과 유사하게, 동일한 데이터를 두 개의 물리적으로 분리된 메모리 채널에 동시에 기록하는 고가용성(High Availability) 기술이다.
> 2. 한쪽 메모리 모듈(DIMM)에 하드 에러나 복구 불가능한 에러(Uncorrectable Error)가 발생하더라도, 미러링된 다른 쪽 메모리를 통해 시스템 중단 없이 계속 서비스를 제공할 수 있다.
> 3. 메모리 용량이 절반으로 줄어들고 쓰기 성능에 일부 오버헤드가 발생하지만, 무중단 운영이 필수적인 미션 크리티컬(Mission-Critical) 서버에서 핵심적인 역할을 한다.



## Ⅰ. 메모리 미러링의 개념과 필요성

**메모리 미러링(Memory Mirroring)**은 시스템 메모리를 절반으로 나누어(예: Channel A와 Channel B), CPU가 데이터를 기록할 때 양쪽 채널에 동일한 데이터를 중복해서 기록하는 기술입니다.

* **문제점 (에러의 한계)**: ECC(Error-Correcting Code) 메모리는 1비트 소프트 에러를 고칠 수 있지만, 메모리 칩 자체가 타버리거나 다중 비트 에러(Multi-bit Error)가 발생하면 시스템이 멈춥니다(Kernel Panic).
* **해결책**: 이를 방지하기 위해 완전히 물리적으로 분리된 두 번째 메모리 공간에 똑같은 복사본을 실시간으로 유지하여, 원본 메모리에 심각한 장애가 발생해도 복사본을 읽어 시스템 다운을 막습니다.

> 📢 **섹션 요약 비유**: 중요한 서류를 작성할 때 먹지를 대고 두 장을 동시에 작성하는 것과 같습니다. 한 장이 찢어지거나 잉크가 번져도 밑에 있는 복사본이 있으니 업무를 계속할 수 있습니다.



## Ⅱ. 메모리 미러링의 동작 방식

### 구조 및 데이터 흐름 (ASCII 다이어그램)

```text
                  ┌─────────────────┐
                  │       CPU       │
                  │ (Memory Controller)
                  └────────┬────────┘
             Write(Data X) │ (동시 기록)
               ┌───────────┴───────────┐
               ▼                       ▼
      [Channel A (Primary)]   [Channel B (Mirror)]
      ┌───────────────────┐   ┌───────────────────┐
      │ DIMM 1 [Data X]   │   │ DIMM 3 [Data X]   │
      │ DIMM 2 [Data Y]   │   │ DIMM 4 [Data Y]   │
      └───────────────────┘   └───────────────────┘
```

1. **쓰기(Write) 작업**: CPU가 메모리에 데이터를 쓸 때, 메모리 컨트롤러가 Channel A와 Channel B에 동시에 동일한 데이터를 기록합니다.
2. **읽기(Read) 작업**: 평소에는 주(Primary) 채널에서 데이터를 읽습니다. 일부 시스템에서는 두 채널을 번갈아 읽어 읽기 대역폭을 향상(Interleaving)시키기도 합니다.
3. **장애 발생(Failover)**: 주 채널의 DIMM에 복구 불가능한 에러(UE, Uncorrectable Error)가 발생하면, 메모리 컨트롤러는 즉시 미러 채널에서 데이터를 읽어오도록 경로를 전환(Failover)합니다. OS나 애플리케이션은 장애를 인지하지 못합니다.

> 📢 **섹션 요약 비유**: 비행기에 두 개의 엔진을 달아두고, 하나가 고장 나면 즉시 다른 하나의 엔진으로만 비행을 계속하는 것과 같습니다. 승객(사용자 프로그램)은 추락하지 않고 안전하게 목적지에 도달합니다.



## Ⅲ. 트레이드오프: 신뢰성 vs 용량 및 비용

메모리 미러링은 궁극의 안정성을 제공하지만, 그에 따른 명확한 비용이 존재합니다.

| 구분 | 일반 모드 (Non-Mirrored) | 미러링 모드 (Mirrored) |
| :--- | :--- | :--- |
| **가용 용량** | 설치된 전체 용량 사용 가능 (예: 128GB) | **설치된 용량의 50%만 사용 가능** (예: 64GB) |
| **가용성 (Availability)** | 단일 DIMM 불량 시 시스템 다운 가능성 높음 | 단일 DIMM 불량 시에도 시스템 무중단 작동 |
| **쓰기 성능** | 단일 채널에 기록 (빠름) | 양쪽 채널에 기록해야 하므로 대역폭 점유/지연 발생 |
| **비용 (TCO)** | 표준 비용 | 동일 용량을 위해 메모리 구매 비용 2배 증가 |

**부분 미러링 (Address Range Mirroring)**: 최근에는 메모리 용량 낭비를 줄이기 위해, 운영체제 커널(Kernel)이나 하이퍼바이저가 적재된 핵심 메모리 영역(일부 주소 대역)만 미러링하고 나머지 사용자 애플리케이션 영역은 일반 모드로 사용하는 유연한 설정도 지원합니다.

> 📢 **섹션 요약 비유**: 모든 데이터를 두 배로 저장하는 것은 보험료를 두 배로 내는 것과 같습니다. 그래서 최근에는 '가장 중요한 핵심 문서'만 복사본을 만들어두는 부분 보험(부분 미러링) 방식을 많이 사용합니다.
