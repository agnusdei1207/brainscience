> **핵심 인사이트**
> 1. **락스텝(Lockstep) 아키텍처**는 두 개(또는 그 이상)의 프로세서가 동일한 클럭 사이클에 동일한 명령어를 실행하고, 그 결과를 실시간으로 비교하여 하드웨어 오류를 즉각적으로 감지하는 기술이다.
> 2. 프로세서 중 하나에 소프트 에러(Soft Error)나 결함이 발생하여 출력 값이 달라지면, 시스템은 즉시 이를 감지하고 오류를 격리하거나 복구 프로세스를 시작한다.
> 3. 항공기 제어, 우주선, 자동차 자율주행(ISO 26262 ASIL-D), 금융 메인프레임 등 '단 한 번의 연산 오류도 치명적인 결과를 낳는' 분야에서 필수적으로 사용된다.

<br>

## Ⅰ. 락스텝 아키텍처의 개념

**락스텝(Lockstep)**이라는 단어는 군인들이 발을 맞춰(Lock) 똑같이 걷는(Step) 모습에서 유래했습니다. 
컴퓨터 구조에서는 **동일한 두 개의 코어(CPU)가 동일한 클럭(Clock) 사이클 단위로 완벽하게 동일한 연산을 수행하는 구조**를 말합니다.

* **목적**: CPU 내부의 레지스터, ALU, 캐시 등에서 발생하는 일시적인 에러(Bit-flip)나 물리적 결함을 100%에 가깝게 감지하기 위함입니다.
* **적용 분야**: 자율주행 자동차의 제어 유닛(MCU), 항공기 플라이트 컨트롤러, 원자력 발전소 제어 시스템 등 하드웨어 결함이 곧 인명 피해나 막대한 금전적 손실로 직결되는 환경(Mission/Safety-Critical).

> 📢 **섹션 요약 비유**: 두 명의 은행원이 똑같은 장부를 놓고 동시에 똑같은 계산기로 계산을 한 뒤, 매 줄마다 답이 맞는지 확인하는 것과 같습니다. 한 명이 실수하더라도 즉시 찾아낼 수 있습니다.

<br>

## Ⅱ. 락스텝의 동작 메커니즘 (Dual-Core Lockstep)

가장 일반적인 형태는 두 개의 코어를 사용하는 **Dual-Core Lockstep (DCLS)** 입니다.

### 구조 (ASCII 다이어그램)

```text
               [ 동일한 입력 데이터 (Input) ]
                     │           │
           ┌─────────▼─┐       ┌─▼─────────┐
           │   Core 0  │       │   Core 1  │
           │ (Primary) │       │ (Checker) │
           └─────────┬─┘       └─┬─────────┘
        결과 A │                   │ 결과 B
               ▼                   ▼
            ┌─────────────────────────┐
            │   비교기 (Comparator)   │
            └────────────┬────────────┘
                         │
             ┌───────────┴───────────┐
        일치 (A == B)           불일치 (A != B)
             │                       │
      [ 정상 출력 (Output) ]     [ 에러 신호 발생! ]
```

1. **동일한 입력**: 시스템 버스에서 들어오는 동일한 명령과 데이터가 두 코어에 동시에 주어집니다.
2. **독립적 연산**: Core 0과 Core 1은 물리적으로 분리된 상태에서 각자 연산을 수행합니다.
3. **클럭 단위 비교**: 매 클럭 사이클마다 양쪽 코어의 출력 버스(Output Bus) 신호를 하드웨어 비교기(Comparator)가 대조합니다.
4. **결과 처리**: 
   * **일치**: 올바른 연산으로 간주하고 결과를 시스템(메모리나 주변기기)으로 내보냅니다.
   * **불일치**: 하드웨어 오류가 발생했음을 즉각 인지하고 오류 신호(Error Signal)를 발생시킵니다. 이후 안전 상태(Safe State)로 진입하거나, 재시도(Retry), 리셋(Reset) 등을 수행합니다.

> 📢 **섹션 요약 비유**: 체조 경기에서 두 명의 심판이 똑같은 동작을 보고 점수를 매긴 후, 점수가 다르면 "문제가 생겼다!"고 즉시 경기를 멈추고 확인하는 시스템입니다.

<br>

## Ⅲ. 다양성과 발전: 지연 락스텝 (Delayed Lockstep)

두 코어가 물리적으로 아주 가까이 있으면, 강력한 전자기파나 외부 노이즈(예: 우주선의 강력한 입자 충돌)가 두 코어를 동시에 타격하여 **같은 종류의 오류(Common Cause Failure)**를 일으킬 위험이 있습니다. 

이를 방지하기 위해 **지연 락스텝(Delayed Lockstep)**이 사용됩니다.
* **시간적 다양성 (Temporal Diversity)**: Checker 코어가 Primary 코어보다 N 클럭(예: 2 사이클) 늦게 동일한 연산을 수행합니다.
* **효과**: 특정 순간에 발생한 노이즈가 Core 0을 고장 내더라도, N 클럭 뒤에 동작하는 Core 1은 정상 환경에서 동작하므로 결과값이 달라져 오류를 확실히 잡아낼 수 있습니다.

> 📢 **섹션 요약 비유**: 두 은행원이 계산을 할 때, 한 명은 1시 정각에 계산하고 다른 한 명은 1시 5분에 똑같은 계산을 하는 것입니다. 1시 정각에 정전이 와서 첫 번째 은행원이 계산을 틀리더라도, 5분 뒤에 하는 은행원은 멀쩡히 계산하여 오류를 발견할 수 있습니다.
