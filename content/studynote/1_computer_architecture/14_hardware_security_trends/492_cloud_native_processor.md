# 468. 도메인 특화 아키텍처 (Domain-Specific Architecture / DSA)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 특정 작업에 최적화된 전용 하드웨어
> 2. **가치**: 효율, 성능, 전력 최적화
> 3. **융합**: ASIC, TPU, NPU, FPGA

---

## Ⅰ. 개요

### 개념 정의

도메인 특화 아키텍처(Domain-Specific Architecture, DSA)는 **범용 CPU 대신 특정 작업 부하에 최적화된 전용 하드웨어 설계로, 에너지 효율과 성능을 극대화하는 아키텍처**다.

### 💡 비유: 전문 요리 도구
DSA는 **전문 요리 도구**와 같다. 만능 칼이 있다. (범용 CPU) 다 한다. (범용) 효율 낮음. 전문 칼이 있다. (DSA) 스테이크용. (특화) 효율 높음. DSA가 똑같다. 특화 설계, 효율 최적화, 단일 목적.

```
┌─────────────────────────────────────────────────────────────────────┐
│                DSA 상세                                                   │

│  ┌───────────────────────────────────────────────────────────────┐ │

│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │                    TPU Core                              │ │ │
│  │  │  ┌───────────────────────────────────────────────────┐ │ │ │
│  │  │  │            Matrix Multiply Unit (MXU)              │ │ │ │

│  │  │  │  Unified Buffer       │  │  Accumulators         │ │ │ │
│  │  │  │  (24 MB SRAM)        │  │  (4 MB)              │ │ │ │

│  │  └─────────────────────────────────────────────────────────┘ │ │

│  │  • 높은 메모리 대역폭                                                      │ │
│  │  • 낮은 정밀도 (bfloat16)                                                 │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【NVIDIA DPU】                                                                  │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  기능:                                                                  │ │
│  │  • 네트워크 패킷 처리                                                     │ │
│  │  • 보안 (IPsec, TLS)                                                     │ │
│  │  • 스토리지 (압축, 암호화)                                                 │ │
│  │  • 가상화 (OVS)                                                          │ │
│  │                                                               │ │
│  │  제품:                                                                  │ │
│  │  • BlueField-2: 200 Gbps                                                │ │
│  │  • BlueField-3: 400 Gbps                                                │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                                      │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  장점:                                                                  │ │
│  │  • 높은 에너지 효율                                                        │ │
│  │  • 특정 작업 최고 성능                                                     │ │
│  │  • 낮은 지연                                                              │ │
│  │                                                               │ │
│  │  단점:                                                                  │ │
│  │  • 낮은 유연성                                                            │ │
│  │  • 높은 개발 비용                                                          │ │
│  │  • 긴 개발 시간                                                            │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘

### 핵심 요약

```

• ASIC: 전용 주문형 집적 회로
• systolic array: 행렬 곱셈 구조
• 장점: 효율, 성능, 전력
• 단점: 유연성, 비용, 시간
