# 469. 인메모리 컴퓨팅 (In-Memory Computing)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 메모리에서 직접 연산 수행
> 2. **가치**: 저지연, 고처리량, 실시간 분석
> 3. **융합**: IMDB, SAP HANA, Redis, MemSQL

---

## Ⅰ. 개요

### 개념 정의

인메모리 컴퓨팅(In-Memory Computing)은 **디스크 대신 메모리에 데이터를 저장하고 메모리에서 직접 연산을 수행하여 디스크 I/O 병목을 해결하는 컴퓨팅 방식**이다.

### 💡 비유: 책상 vs 전자책
인메모리 컴퓨팅은 **책상 vs 전자책**과 같다. 책장이 있다. (디스크) 느림. 찾기 힘듦. 전자책이 있다. (메모리) 빠름. 즉시 검색. 인메모리가 똑같다. 메모리 저장, 직접 연산, 실시간 처리.

## Ⅱ. 인메모리 컴퓨팅 상세

```

│                인메모리 컴퓨팅 상세                                        │

│  │  │  │  (트랜잭션)  │  │  (분석)     │  │ (검색)      │     │ │ │
│  │  │  └─────────────┘  └─────────────┘  └─────────────┘     │ │ │

│  │  • 파티셔닝: 수평/수직 분할                                                 │ │
│  │  • 복제: 동기/비동기 복제                                                   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Redis 아키텍처】                                                                │)
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  데이터 구조:                                                            │ │
│  │  • String: 문자열                                                         │ │
│  │  • Hash: 해시 테이블                                                       │ │
│  │  • List: 리스트                                                           │ │

│  │                                                               │ │
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 인메모리 컴퓨팅: 메모리에서 직접 연산
• IMDB: 인메모리 데이터베이스
• SAP HANA: 엔터프라이즈 IMDB
• Redis: 인메모리 캐시/DB
• OLTP: 온라인 트랜잭션 처리
• OLAP: 온라인 분석 처리
• HTAP: 하이브리드 트랜잭션/분석
• 장점: 저지연, 고처리량, 실시간
• 단점: 비용, 용량, 휘발성
• 활용: 금융, IoT, 게임, 분석
```
