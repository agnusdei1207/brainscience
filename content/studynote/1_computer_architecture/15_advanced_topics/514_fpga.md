# 514. FPGA (Field Programmable Gate Array)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 제조 공정에서 회로가 고정되어 나오는 일반 반도체(ASIC)와 달리, 사용자가 구매한 후(Field) 프로그래밍(Programmable)을 통해 내부 논리 회로를 마음대로 재구성할 수 있는 하드웨어 칩이다.
> 2. **가치**: 소프트웨어의 유연성과 하드웨어(ASIC) 수준의 병렬 처리 및 초저지연 성능 사이의 완벽한 타협점을 제공하여, 알고리즘이 자주 변하는 통신 장비나 AI 가속, 금융권 HFT(초단타 매매)의 핵심 엔진으로 쓰인다.
> 3. **융합**: 과거에는 틈새시장(Niche)의 프로토타이핑 도구였으나, 현재는 클라우드 서버(AWS F1 등)에 탑재되어 CPU의 연산을 오프로딩하거나 스마트NIC의 두뇌로 활용되는 등 데이터센터의 메인스트림으로 진입했다.

---

## Ⅰ. 개요 (Context & Background)

- **개념**: FPGA(Field Programmable Gate Array)는 수백만 개의 기본 논리 소자(Logic Block)와 이를 연결하는 스위칭 네트워크(Routing Matrix)가 바둑판처럼 배열되어 있어, 개발자가 VHDL이나 Verilog 같은 하드웨어 기술 언어(HDL)를 이용해 원하는 논리 회로를 칩 내부에 물리적으로 '구워낼 수 있는(합성)' 반도체다.
- **💡 비유**: 일반 반도체(ASIC)가 이미 방과 복도, 화장실 위치가 콘크리트로 고정된 '지어진 아파트'라면, FPGA는 내 맘대로 블록을 조립해 거실을 넓히거나 방을 없앨 수 있는 '거대한 레고 판'이다. 건물을 허물지 않고도 내일은 병원으로, 모레는 학교로 마음대로 용도 변경이 가능하다.
- **등장 배경 및 발전 과정**:
  1. **초기 프로토타이핑 (1980~1990년대)**: 비싼 ASIC을 만들기 전에, 논리 회로가 맞는지 미리 테스트해보는 에뮬레이터 용도로 주로 쓰였다.
  2. **통신 및 군사 장비의 주력 (2000년대)**: 5G 기지국이나 레이더처럼 표준이 계속 바뀌는 장비는 ASIC으로 만들면 표준이 바뀔 때마다 칩을 버려야 했다. FPGA는 펌웨어 업데이트만으로 새로운 통신 규격을 지원할 수 있어 통신 시장을 제패했다.
  3. **데이터센터 가속기 시대 (2010년대 후반~)**: 마이크로소프트(Project Catapult)가 검색 엔진 알고리즘(Bing) 가속을 위해 전 서버에 FPGA를 달았고, 이를 인텔이 인수(Altera 인수)하고 AMD가 자일링스(Xilinx)를 인수하면서 CPU와 FPGA의 융합 시대가 열렸다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 구성 요소 (FPGA 내부 구조)

| 요소명 | 역할 | 내부 동작 / 특징 | 비유 |
|:---|:---|:---|:---|
| **CLB (Configurable Logic Block)** | 실제 덧셈, AND, OR 연산을 수행하는 핵심 레고 블록 | 내부적으로 LUT, Flip-Flop, MUX로 구성됨 | 레고의 기본 브릭 (1칸, 2칸짜리) |
| **LUT (Look-Up Table)** | 논리 연산의 결과를 미리 저장해 둔 진리표 RAM | 게이트를 통과하지 않고 입력값(주소)에 따른 결과값을 RAM에서 바로 꺼냄 | "1+1=2"라고 적힌 암기 노트 |
| **Routing Matrix (인터커넥트)** | CLB와 CLB 사이를 연결하는 프로그래밍 가능한 전선망 | 스위치 매트릭스를 열고 닫아 신호의 경로를 만듦 | 레고 브릭 사이를 잇는 고무호스 |
| **I/O Block (IOB)** | FPGA 내부 회로와 외부 핀을 연결 | 전압 레벨 조정, 직렬화/역직렬화(SerDes) 수행 | 레고 성의 출입문 |
| **DSP / BRAM (Hard IP)** | 덧셈/곱셈기나 메모리를 미리 ASIC처럼 딱딱하게 굳혀놓은 블록 | 자주 쓰는 무거운 연산을 CLB로 만들면 비효율적이므로 하드와이어링 해둠 | 레고 세트에 들어있는 '통짜 바퀴' 부품 |

### CPU vs GPU vs FPGA vs ASIC 실행 아키텍처 비교

FPGA의 특수성을 이해하려면 폰 노이만 구조의 굴레에서 벗어나야 한다.

```text
  ┌─────────────────────────────────────────────────────────────┐
  │         CPU (Software) vs FPGA (Hardware) 실행 방식 비교          │
  ├─────────────────────────────────────────────────────────────┤
  │                                                             │
  │ [1] CPU/GPU (폰 노이만 방식 - 시간에 따른 순차적 실행)              │
  │                                                             │
  │   [명령어 메모리] ──▶ (Fetch) ──▶ (Decode) ──▶ [ ALU (고정) ]     │
  │   데이터 A ──────▶ (메모리에서 읽음) ───────────▶  (계산) ──▶ 결과 │
  │   * 특징: 하드웨어(ALU)는 가만히 있고, 소프트웨어(명령어)가 바뀌면서  │
  │           ALU에게 "더해라, 빼라" 지시를 내림. (병목 발생)            │
  │                                                             │
  │ ─────────────────────────────────────────────────────────── │
  │                                                             │
  │ [2] FPGA (공간 컴퓨팅 - Space Computing)                       │
  │                                                             │
  │   (명령어를 읽는 과정 자체가 없음. 데이터가 파이프프라인처럼 흐름)     │
  │                                                             │
  │   데이터 A ──▶ [LUT 1 (더하기 회로로 조립됨)] ──┐                   │
  │                                           ▼                   │
  │   데이터 B ──▶ [LUT 2 (조건 검사 회로)] ───▶ [LUT 3 (최종 결과)] ──▶ │
  │                                                             │
  │   * 특징: 알고리즘 자체를 물리적인 '전선과 게이트'의 연결 모양으로     │
  │           바꿔버림. 칩 안에 내 알고리즘만을 위한 맞춤형 컨베이어     │
  │           벨트가 탄생함. 매 클럭마다 데이터가 쏟아져 나옴(100% 병렬). │
  └─────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** [1] CPU나 GPU는 소프트웨어를 실행하는 장치다. 메모리에서 명령어를 가져와 해석하고 연산기를 돌린다. 이 과정(폰 노이만 병목) 때문에 명령어 해석 오버헤드가 발생한다.
반면 [2] **FPGA는 아예 소프트웨어를 하드웨어로 바꿔버린다.** 개발자가 C코드나 Verilog로 `C = A + B`라고 짜서 넣으면(컴파일/합성), FPGA 내부의 수만 개의 레고 블록(LUT)과 스위치가 스스로 길을 찾아 연결되면서 진짜 물리적인 '덧셈기' 기계 덩어리로 변신한다. 이 상태에서는 메모리에서 명령어를 읽어올 필요가 전혀 없이, 데이터가 들어오는 즉시 전선을 타고 흐르며 답이 나온다. 이를 **공간 컴퓨팅(Spatial Computing)**이라 부르며, 딥러닝 추론이나 암호화에서 극한의 초저지연(Latency)이 나오는 이유다.

### 심층 동작 원리 (FPGA의 합성 및 설정 과정)
① **RTL 코딩**: 개발자가 VHDL, Verilog(또는 C/C++ 기반 HLS)로 원하는 로직을 코딩한다.
② **논리 합성 (Synthesis)**: 컴파일러가 코드를 분석해 "이 회로를 만들려면 AND 게이트 100개, MUX 50개가 필요해"라고 계산(Netlist 생성)한다.
③ **배치 및 라우팅 (Place & Route)**: 가장 복잡한 과정. FPGA 칩 안의 수백만 개 빈 방(LUT) 중 어디에 AND 게이트를 넣고, 전선(Routing)을 어떻게 꼬이지 않게 연결할지 소프트웨어가 수 시간에 걸쳐 계산한다.
④ **비트스트림(Bitstream) 다운로드**: 완성된 연결 지도를 '비트스트림'이라는 파일(예: `.bit`)로 뽑아낸다. 전원을 켤 때 이 파일을 FPGA 내부의 구성 SRAM에 덮어씌우면(Programming), 비로소 칩 안의 스위치들이 닫히고 열리며 물리적인 회로가 완성된다. (전원을 끄면 회로 모양이 다시 다 풀어진다.)

## Ⅲ. 융합 비교 및 다각도 분석

### 아키텍처 지형도 (ASIC vs FPGA vs GPU vs CPU)

| 비교 항목 | CPU (범용) | GPU (병렬 연산) | FPGA (재구성 하드웨어) | ASIC (주문제작 반도체) |
|:---|:---|:---|:---|:---|
| **설계 유연성** | **최상** (C/C++ 코드) | 높음 (CUDA 코드) | 보통 (HDL 코딩/합성 시간 김) | **최악** (설계 변경 절대 불가) |
| **개발 비용(NRE)** | 거의 없음 | 거의 없음 | 보통 (칩 가격이 비쌈) | **수백억 원 이상** (마스크 비용) |
| **전력 소모 (효율)** | 높음 (비효율적) | 매우 높음 (전기 먹는 하마) | 낮음 (최적화된 회로만 가동) | **매우 낮음 (극한의 전성비)** |
| **처리 지연 (Latency)**| 밀리초(ms) 단위 | 배치 처리 시 좋으나 단일 지연 김 | **마이크로/나노초(µs~ns)** | 나노초 단위 |
| **적합한 워크로드** | OS, 제어 분기, DB | AI 학습, 3D 그래픽 렌더링 | AI 추론, HFT, 네트워크 패킷 파싱 | 비트코인 채굴, 양산 스마트폰 AP |

### 과목 융합 관점
- **하드웨어 보안 (사이드 채널 및 암호화)**: FPGA는 암호화 알고리즘(AES, RSA)을 하드웨어로 구현하는 데 최고의 도구다. CPU가 암호화를 소프트웨어로 풀면 전력 소모나 실행 시간 패턴이 남아 부채널 공격(Side-channel)에 당하기 쉽지만, FPGA는 물리적 게이트가 항상 일정한 속도로 동작하므로 시간 기반 부채널 공격에 면역력을 갖는다.
- **클라우드 / SDI (SmartNIC)**: 오늘날 데이터센터의 SDN(소프트웨어 정의 네트워크)을 처리하는 SmartNIC(또는 DPU)의 핵심 두뇌는 대부분 FPGA다. 구글과 MS는 자사 클라우드의 방화벽, 암호화, 패킷 라우팅 알고리즘을 매달 업데이트하는데, 이 로직들을 ASIC으로 만들면 업데이트가 불가능하므로, 업데이트 가능하면서도 초저지연을 보장하는 FPGA를 모든 서버 랜카드에 탑재했다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오
1. **시나리오 — 금융권 HFT(초단타 매매) 시스템의 생존 게임**: 시카고 거래소에서 주식 알고리즘 트레이딩을 하는 증권사. 시장 정보가 도착한 후 매수 주문을 내기까지 1마이크로초(1µs = 백만 분의 1초) 늦으면 거래에서 진다.
   - **아키텍처 설계**: 일반 x86 서버에 C++ 프로그램을 짜면 패킷이 랜카드 -> OS 커널(인터럽트) -> 앱으로 올라가는 데만 수십 µs가 걸린다. 증권사는 **FPGA 랜카드**를 꽂고, 트레이딩 전략 알고리즘 자체를 FPGA 하드웨어 칩 안에 구워버린다. 네트워크 선에서 가격 패킷이 들어오자마자, 칩 내부의 논리 게이트를 통과하며 100나노초(ns) 만에 즉시 매수 패킷이 튀어나간다. (OS 개입 0%)
2. **시나리오 — 우주 항공 탐사선(로버)의 하드웨어 업데이트**: 화성 탐사선에 탑재된 카메라 이미지 프로세싱 칩. 발사 후 3년이 지나 더 좋은 화질 개선 알고리즘이 발명되었다.
   - **가치 창출**: 지구에서 새로운 알고리즘의 FPGA 비트스트림 파일을 전파로 쏘아 보낸다. 화성에 있는 탐사선은 전원을 껐다 켜면서 이 파일을 덮어쓴다. 화성 탐사선의 하드웨어 칩 내부 물리 회로가 새것으로 재창조(Re-programmed)되어 새로운 카메라 모듈처럼 동작하게 된다.
3. **시나리오 — 개발 언어의 장벽 (안티패턴)**: AI 스타트업이 성능을 높이겠다며 무작정 FPGA 클라우드(AWS F1)를 임대했다. 
   - **문제 발생**: 데이터 과학자들은 Python, PyTorch밖에 모르는데, FPGA를 다루려면 Verilog 같은 하드웨어 언어를 알아야 하고 컴파일(Place & Route) 한 번에 8시간씩 걸려 개발 생산성이 박살 났다.
   - **해결책**: C/C++ 코드를 하드웨어 코드로 자동 변환해 주는 **HLS(High-Level Synthesis)** 툴이나, 아예 파이토치 모델을 직접 FPGA 비트스트림으로 바꿔주는 Vitis AI 같은 추상화 플랫폼을 적극적으로 도입하여 소프트웨어 엔지니어의 진입 장벽을 낮춰야 한다.

### 도입 체크리스트
- **비즈니스 판단 (ASIC vs FPGA)**: 해당 제품의 예상 판매량이 10만 개 이상인가? 만약 그렇다면 NRE(마스크 제작비 100억)를 태워서라도 ASIC을 파는 게 장기적으로 원가가 싸다. 반대로 예상 판매량이 1만 개 미만이고, 6개월마다 표준(알고리즘)이 바뀌는 시장이라면 무조건 개당 단가가 수백만 원 하더라도 FPGA를 써야 한다.

## Ⅴ. 기대효과 및 결론

### 정량/정성 기대효과

| 구분 | 일반 서버 (CPU 기반) | FPGA 가속기 적용 시 | 개선 효과 |
|:---|:---|:---|:---|
| **정량 (Latency)** | 스레드 스위칭 포함 10~50 µs | **1 µs 미만 (수백 ns 수준)** | 초실시간 응답 (네트워크, 금융 결제) 가능 |
| **정량 (전력 효율)** | 와트당 데이터 처리량 낮음 (소프트웨어 오버헤드) | 필요한 물리적 게이트만 스위칭 (하드웨어 오프로드) | 와트당 성능(Perf/W) **10배 이상 향상** |
| **정성 (타임투마켓)** | 칩을 새로 설계/양산하는 데 2~3년 소요 | 비트스트림 수정 후 즉각 재시작 (수 분 소요) | 고객의 요구사항 변화에 즉각 대응 가능한 민첩성 |

### 미래 전망
- **eFPGA (Embedded FPGA)의 SoC 융합**: 과거에는 CPU 칩 따로, FPGA 칩 따로 메인보드에 꽂았다면, 지금은 인텔, AMD 등 칩 제조사가 자사 CPU 다이 바로 옆(동일 패키지 내부)에 조그만 FPGA 블록(eFPGA)을 칩렛 형태로 박아 넣는 시대다. CPU가 처리하기 힘든 커스텀 암호화나 압축 연산을 만날 때마다 옆에 붙은 eFPGA에 오프로드하는 극강의 유연성을 제공한다.
- **오픈소스 하드웨어의 부상**: 칩 설계 비용이 폭등하면서, RISC-V 코어 같은 오픈소스 하드웨어를 저렴한 개발용 FPGA 보드에 올려 자신만의 커스텀 칩을 뚝딱 만들어보는 '하드웨어 민주화' 운동이 대학과 메이커 생태계를 중심으로 폭발적으로 성장하고 있다.

### 참고 표준
- **OpenCL / SYCL**: 하드웨어 지식이 없는 소프트웨어 개발자도 FPGA를 GPU처럼 쉽게 프로그래밍할 수 있도록 돕는 개방형 병렬 프로그래밍 언어 표준. FPGA 생태계 확장의 핵심 열쇠다.

## 👶 어린이를 위한 3줄 비유 설명
1. 보통 장난감 공장(일반 반도체)은 '자동차'를 만드는 기계를 한번 세팅해 놓으면 평생 자동차만 찍어낼 수 있어요. 비행기로 바꾸려면 공장을 다 부수고 새로 지어야 하죠.
2. 하지만 **FPGA**는 수백만 개의 **'레고 블록'**으로 가득 찬 마법의 공장이에요!
3. 아침에는 레고를 조립해 '자동차 공장'으로 썼다가, 저녁에 뚝딱뚝딱 다시 조립해서 '비행기 공장'으로 바꿀 수 있어요. 그래서 새로운 발명품을 아주 빠르고 쉽게 만들 수 있답니다.
