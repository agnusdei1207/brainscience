# 516. NPU (Neural Processing Unit)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 인공지능(AI) 신경망의 행렬 곱셈과 덧셈(MAC) 연산을 저전력으로 초고속 처리하기 위해 설계된 도메인 특화 반도체(DSA)다.
> 2. **가치**: 범용 GPU가 전기 먹는 하마인 것과 달리, 불필요한 그래픽/제어 로직을 다 걷어내어 극단적인 전성비(Performance per Watt)를 제공하므로 스마트폰, 자율주행차, IoT 기기의 '온디바이스 AI' 구현을 위한 필수 엔진이다.
> 3. **융합**: 시스톨릭 어레이(Systolic Array) 구조를 핵심으로 하며, 최근 메모리와 연산기를 통합하려는 PIM(Processing-in-Memory) 아키텍처와 결합하여 AI 반도체의 최종 진화형으로 나아가고 있다.

---

## Ⅰ. 개요 (Context & Background)

- **개념**: NPU(Neural Processing Unit)는 인간 뇌의 신경망 구조를 모방한 딥러닝 알고리즘의 연산(특히 MAC 연산)에 100% 최적화된 하드웨어 가속기다. (벤더에 따라 TPU, BPU, DPU 등 다양하게 불리나 본질은 같다.)
- **💡 비유**: CPU가 국영수사과를 골고루 잘하는 '범용 엘리트 학자', GPU가 수천 개의 단순 그림 그리기를 동시에 하는 '픽셀 화가 집단'이라면, NPU는 오직 수천 개의 '구구단(행렬 곱)'만 평생 동안 미친 듯이 외우고 계산하는 '구구단 천재 로봇'이다. 밥(전기)도 아주 조금 먹으면서 구구단만큼은 세계에서 제일 빠르다.
- **등장 배경 및 발전 과정**:
  1. **GPU의 오버헤드**: 초기 AI는 그래픽 렌더링용 GPU로 훈련/추론을 했다. 하지만 GPU 내부에는 캐시 메모리, 스레드 제어, 그래픽 파이프라인 등 AI 연산에는 전혀 쓰이지 않는 잉여 회로(Dark Silicon)가 너무 많아 전력 소모가 극심했다.
  2. **모바일/엣지(Edge)의 요구**: 스마트폰에서 얼굴 인식을 하거나 자율주행차가 보행자를 파악할 때 250W짜리 GPU를 달 수는 없다. 배터리로 돌아가는 기기에서 1W 미만의 전력으로 AI를 돌려야만 하는 '온디바이스 AI'의 절대적 필요성이 생겼다.
  3. **ASIC 기반 특화 칩(DSA)의 탄생**: 구글의 TPU, 애플 실리콘의 Neural Engine, 삼성/퀄컴의 NPU 등 각 기업이 자사 AI 모델(소프트웨어)에 딱 맞춰 하드웨어 게이트를 구워버리는(Hard-wired) 전용 칩 시대가 열렸다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 구성 요소

| 요소명 | 역할 | 내부 동작 / 특징 | 비유 |
|:---|:---|:---|:---|
| **MAC (Multiply-Accumulate) Unit** | 딥러닝 연산의 90% 이상을 차지하는 행렬의 곱(Multiply)과 합(Accumulate) 수행 | $a \times b + c$ 연산을 단 1클럭에 하드웨어 레벨에서 처리 | 구구단 계산기 |
| **Systolic Array (시스톨릭 어레이)** | MAC 유닛들의 거대한 2차원/3차원 격자 배열 | 레지스터에 데이터를 다시 쓰지 않고 옆 MAC 유닛으로 혈류처럼 계속 밀어냄 | 계산 결과를 옆 사람에게 바로 넘기는 컨베이어 벨트 |
| **Activation Engine** | 행렬 곱셈이 끝난 값에 비선형 활성화 함수 적용 | ReLU, Sigmoid, tanh 등을 하드웨어 룩업 테이블(LUT)로 빠르게 계산 | 요리가 끝나고 마지막에 뿌리는 특제 소스 기계 |
| **SRAM (Local Memory/Scratchpad)** | 모델의 가중치(Weight)를 임시 보관하는 초고속 내부 메모리 | 외부 DRAM 접근을 최소화하여 전력을 아끼는 핵심 요소 | 요리사 옆에 둔 양념통 |

---

### NPU의 핵심: 시스톨릭 어레이(Systolic Array) 아키텍처

NPU가 GPU를 압도하는 이유는 폰 노이만 병목을 회피하는 데이터 흐름(Data Flow) 구조에 있다.

```text
  ┌─────────────────────────────────────────────────────────────┐
  │              일반 GPU 연산 vs NPU 시스톨릭 어레이 연산               │
  ├─────────────────────────────────────────────────────────────┤
  │                                                             │
  │ [1] 범용 GPU (스레드 기반 반복 연산)                             │
  │                                                             │
  │   [ SRAM (Register) ] ◀──┐ (전력 소모 극심)                   │
  │     │          │         │                                  │
  │     ▼          ▼         │                                  │
  │   [ALU 1]    [ALU 2] ────┘                                  │
  │   * 특징: 곱하고 더할 때마다 레지스터(SRAM)에서 읽고 다시 씀.       │
  │           연산 자체보다 메모리 읽고 쓰는 데 전기를 10배 더 씀.         │
  │                                                             │
  │ ─────────────────────────────────────────────────────────── │
  │                                                             │
  │ [2] NPU (Systolic Array - 심장 박동 구조)                      │
  │                                                             │
  │   (Weight 가중치 고정)                                          │
  │     [W1]       [W2]       [W3]                              │
  │      │          │          │                                │
  │ IN ▶[MAC] ───▶ [MAC] ───▶ [MAC] ───▶ OUT (최종 합산)           │
  │      │          │          │                                │
  │ IN ▶[MAC] ───▶ [MAC] ───▶ [MAC] ───▶ OUT                     │
  │                                                             │
  │   * 특징: 데이터가 왼쪽에서 오른쪽, 위에서 아래로 물 흐르듯 전달됨.   │
  │           SRAM에 결과를 저장하지 않고 바로 옆 MAC 유닛으로 넘김!   │
  │           메모리 병목 타파 = 1/10 전력 소모 달성.                 │
  └─────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 상단의 범용 칩(CPU/GPU)은 연산 한 번 할 때마다 레지스터나 캐시(SRAM)에서 값을 읽어오고 다시 쓴다. 반도체에서 연산(더하기)에 드는 전기보다 이 **데이터 이동에 드는 전기가 100배 이상** 더 크다.
하단의 **NPU(시스톨릭 어레이)**는 이 낭비를 막는다. AI 행렬 곱의 특징은 A, B, C 값을 순차적으로 계속 더해나간다는 것이다. 그래서 첫 번째 MAC(곱셈기)이 계산한 중간 결과를 SRAM에 쓰지 않고, 그냥 전선을 통해 바로 옆에 있는 두 번째 MAC으로 토스해버린다. 피가 심장(Systole)에서 혈관을 타고 한 방향으로 쭉 흐르듯이 데이터가 MAC 배열을 쭉 타고 흐른다. 이 구조 덕분에 외부 메모리 왕복이 사라져 NPU는 기적 같은 전성비(Performance per Watt)를 달성하게 된다.

---

### 심층 동작 원리: Quantization (양자화)와 NPU의 시너지
NPU는 부동소수점(FP32, FP16) 연산을 과감히 버리거나 최소화한다. 대신 소프트웨어 단에서 딥러닝 모델의 가중치를 정수(INT8, INT4 심지어 INT2)로 깎아버리는 **양자화(Quantization)**를 수행한다. NPU 하드웨어는 이 INT8 연산에 최적화된 MAC을 수만 개 때려 박아 칩 크기를 획기적으로 줄인다. 인간의 뇌가 시냅스의 미세한 오차를 신경 쓰지 않고도 고양이와 강아지를 구별하듯, NPU도 정밀도를 희생하고 속도와 에너지 효율이라는 극단적 효율을 취하는 생체 모방(Biomimetic) 철학을 따른다.

## Ⅲ. 융합 비교 및 다각도 분석

### CPU vs GPU vs NPU (인공지능 워크로드 기준)

| 비교 항목 | CPU | GPU | NPU |
|:---|:---|:---|:---|
| **연산 방식** | 직렬(순차) 최적화 | 대규모 병렬 최적화 (SIMT) | **행렬 내적 특화 (Systolic Array)** |
| **코어(ALU) 수** | 수십 개 | 수천 개 | **수만 개 (MAC 단위)** |
| **전력 소모** | 보통 | 매우 높음 (수백 Watt) | **매우 낮음 (모바일은 수 Watt)** |
| **유연성** | 최상 (모든 프로그램 구동) | 높음 (AI, 그래픽, 코인 채굴) | **최악 (정해진 AI 연산 외 불가)** |
| **AI 주요 역할** | 복잡한 제어 로직, 전처리 | **초거대 AI 모델 학습(Training)** | **엣지 디바이스 등 실시간 추론(Inference)** |

### 과목 융합 관점
- **엣지 컴퓨팅 (On-Device AI)**: 과거에는 스마트폰에서 음성 인식을 하면 내 목소리를 녹음해서 클라우드 서버(GPU)로 보낸 뒤 결과를 받아왔다(지연 발생, 개인정보 유출 위험). 현재는 스마트폰 AP 내부에 내장된 NPU가 외부 인터넷 연결 없이 비행기 모드에서도 초당 수조 번 연산(TOPS)을 수행하여 실시간 번역이나 사진 보정을 끝내버린다. NPU가 엣지 컴퓨팅의 진정한 자립을 가져왔다.
- **컴파일러 (TVM / MLIR)**: NPU는 제조사(애플, 구글, 삼성, 테슬라)마다 내부 하드웨어 명령어 구조가 완전히 파편화되어 있다. 파이토치(PyTorch) 코드가 이 수많은 이기종 NPU에서 잘 돌아가게 하려면, 코드를 하드웨어 중립적인 중간 언어(IR)로 바꾸고 각 NPU에 맞게 최적화해 주는 **MLIR이나 Apache TVM 같은 AI 전용 딥러닝 컴파일러 프레임워크**가 절대적으로 필요하다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오
1. **시나리오 — 데이터센터 AI 추론(Inference) 비용 폭발**: 클라우드 서비스 기업이 유저 수백만 명에게 챗봇 서비스를 제공하는데, 추론 서버를 몽땅 엔비디아 A100 GPU로 도배했더니 전기세와 서버 대여료가 수익을 초과해버림.
   - **아키텍처 혁신**: 학습(Training)은 여전히 범용성이 좋은 GPU 클러스터에서 수행하되, 이미 학습이 끝나 가중치가 고정된 모델을 유저에게 서비스(Inference)하는 서버는 AWS Inferentia나 구글 Cloud TPU(NPU 계열 ASIC)로 전면 교체한다. NPU는 추론 시 GPU 대비 전력은 1/10만 쓰면서 처리량(Throughput)은 3~5배 더 뽑아내어 서비스 단위 단가를 획기적으로 낮춘다.
2. **시나리오 — 자율주행 차량의 실시간 사물 인식**: 시속 100km로 달리는 자동차. 카메라 프레임 하나를 분석하는 데 50ms 이상 걸리면 사고가 난다.
   - **해결책**: 트렁크 공간 한계와 자동차 배터리 용량 때문에 일반 GPU 서버를 실을 수 없다. 테슬라(FSD 칩) 같은 제조사는 자체 NPU를 직접 설계한다. 오직 자사 자율주행 신경망 알고리즘에만 딱 맞춘 크기의 SRAM과 MAC 어레이를 하드와이어링(Hard-wiring)하여, 수 ms 이내의 결정적(Deterministic) 저지연으로 보행자와 차선을 100% 인식해 낸다.

### 도입 체크리스트
- **벤더 종속성 (Lock-in)**: 특정 회사의 NPU를 도입하면 그 회사가 제공하는 SDK 전용 생태계에 갇힐 위험이 크다. 만약 회사 AI 모델 아키텍처(예: Transformer에서 Mamba 등으로)가 급격히 바뀌었을 때, 기존 NPU가 지원하는 하드웨어 연산자(Operator) 라이브러리에 새 연산이 없으면 칩 전체가 바보 깡통이 된다. 하드웨어 유연성(Programmability)이 얼마나 남아있는지 따져야 한다.

## Ⅴ. 기대효과 및 결론

### 정량/정성 기대효과

| 구분 | 범용 GPU 기반 추론 | 전용 NPU 기반 추론 | 개선 효과 |
|:---|:---|:---|:---|
| **정량 (TOPS/W)** | 1와트당 약 1~5 TOPS 성능 | 1와트당 약 **20~50 TOPS** 성능 | 극단적인 전력 효율(전성비) 향상 |
| **정량 (Die Area)** | 캐시, 스케줄러로 칩 크기 거대함 | 오직 연산기(MAC)로 면적 꽉 채움 | 칩 제조 단가 및 실장 면적(Form factor) 최소화 |
| **정성 (데이터 프라이버시)** | 클라우드 전송에 따른 보안 취약점 | 로컬 기기 내부에서 모든 AI 연산 완료 | 오프라인 환경 작동 및 완벽한 보안(On-Device) |

### 미래 전망
- **PIM(Processing-In-Memory)과의 궁극적 결합**: NPU가 아무리 똑똑해도 딥러닝 모델의 수천억 개 파라미터를 외부 DRAM에서 가져와야 하는 병목(메모리 월)은 피할 수 없다. 미래의 NPU는 독립된 칩이 아니라, 삼성/SK하이닉스의 HBM 메모리 다이(Die) 안쪽 하단이나 뱅크 사이에 NPU 논리 회로가 콕 박혀있는 PIM(지능형 메모리 반도체) 형태로 융합될 것이다.
- **아날로그 NPU 및 뉴로모픽 도래**: 0과 1의 디지털 연산을 넘어, 빛의 파장(실리콘 포토닉스 NPU)이나 멤리스터 소자의 저항값 자체를 가중치로 사용하는 아날로그 컴퓨팅 기반 NPU가 연구 중이다. 이는 뇌의 뉴런과 시냅스를 완벽히 모방하는 뉴로모픽(Neuromorphic) 칩으로 향하는 징검다리다.

### 참고 표준
- NPU는 아직 x86이나 ARM처럼 천하통일된 명령어 셋(ISA) 표준이 없다. 다만 이 파편화를 해결하기 위해 ONNX(Open Neural Network Exchange) 같은 오픈소스 표준 모델 포맷이 이기종 NPU 간의 생태계를 이어주는 다리 역할을 하고 있다.

- **[도메인 특화 아키텍처 (DSA)](../3_architecture_basics_performance/156_dsa.md)**: 특정 작업에만 효율을 올리는 칩 설계 사상. NPU는 AI 도메인을 가속하는 인류 역사상 가장 성공적인 DSA다.
- **[이기종 집적 (Heterogeneous Integration)](./512_heterogeneous_integration.md)**: 스마트폰 AP 안에 CPU 코어, GPU 코어 옆에 칩렛(블록) 형태로 NPU가 하나로 포장되는 차세대 패키징 기술.
- **[양자화 (Quantization)](../12_accelerators_ai_hardware/434_quantization.md)**: AI 모델의 부동소수점을 8비트 정수로 깎아내는 소프트웨어 기술. NPU 하드웨어는 이 양자화된 데이터를 받아먹어야 극한의 가성비를 낸다.

2. 그런데 새로 나타난 **NPU**라는 친구는 다른 건 하나도 못 하지만, 오직 **'구구단(행렬 곱셈)' 하나만큼은 밥도 안 먹고 우주에서 제일 빨리** 외우는 로봇이에요.
3. 요즘 스마트폰이나 자동차는 인공지능이 필요해서 이 구구단 로봇을 꼭 데리고 다녀요. 배터리도 조금만 먹으면서 눈앞의 사람이나 물건을 1초 만에 알아맞히거든요!
