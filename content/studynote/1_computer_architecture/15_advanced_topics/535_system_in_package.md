# 535. 시스템 온 패키지 (SiP, System in Package)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 여러 개의 서로 다른 반도체 칩(CPU, 메모리, 센서, 통신 모뎀 등)과 수동 소자들을 하나의 거대한 패키지 안에 집적하여, 겉보기에는 단일 칩처럼 동작하게 만드는 융합 패키징 아키텍처다.
> 2. **가치**: 스마트워치나 무선 이어폰처럼 공간이 극도로 부족한 폼팩터에서 메인보드 크기를 획기적으로 줄여주며, 각 칩을 가장 가성비 좋은 공정으로 따로 만들어 조립하므로 SoC 대비 수율과 개발 비용이 압도적으로 우수하다.
> 3. **융합**: 'More than Moore(무어의 법칙 이후)' 시대를 이끄는 핵심 실행 기술이며, 칩렛(Chiplet), TSV 기반 3D 패키징, 이기종 집적(Heterogeneous Integration) 기술을 모두 담아내는 거대한 그릇이다.

---

## Ⅰ. 개요 (Context & Background)

- **개념**: 시스템 온 패키지(SiP)는 로직, 메모리, 아날로그, RF 등 다양한 기능을 수행하는 개별 칩(Bare Die)들을 하나의 패키지 기판(Substrate) 위에 2D, 2.5D 또는 3D로 배열하고 하나로 몰딩(Molding)하여 완전한 시스템을 구현하는 기술이다.
- **💡 비유**: 기존의 SoC가 밀가루, 계란, 초콜릿 칩을 한 반죽에 다 섞어서 거대한 '통 쿠키' 하나를 구워내는 방식이라면, SiP는 빵 공장에서 구워온 빵 조각, 고기 공장에서 가져온 패티, 야채를 층층이 쌓아 포장지로 예쁘게 싼 '햄버거 세트'다. 고객 입장에서는 똑같이 하나의 음식(칩)이지만, 만드는 방식과 재료의 자유도는 완전히 다르다.
- **등장 배경 및 발전 과정**:
  1. **SoC(System on Chip)의 한계**: 스마트폰 시대가 열리며 칩 하나에 모든 걸 다 넣는 SoC가 대세였다. 하지만 메모리나 아날로그 센서는 CPU처럼 미세 공정(3nm 등)으로 작게 만들기 어려웠고, 억지로 같이 구웠다가 수율이 박살 나며 단가가 천정부지로 치솟았다.
  2. **폼팩터의 소형화 압박**: 애플워치나 에어팟 같은 웨어러블 기기가 등장하면서, 부품들을 꽂을 넓은 메인보드(PCB)를 넣을 공간 자체가 사라졌다. 부품들을 아예 칩 껍데기(패키지) 안에 다 욱여넣어야 할 필요성이 생겼다.
  3. **첨단 패키징의 발전**: 단순한 와이어 본딩을 넘어 실리콘 관통 전극(TSV)과 인터포저(Interposer) 기술이 발전하면서, 패키지 안에서 칩과 칩 사이를 연결하는 속도가 SoC 내부의 속도를 따라잡게 되며 SiP가 폭발적으로 성장했다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 구성 요소 (SiP 내부 구조)

SiP는 단순한 플라스틱 껍데기가 아니라 그 자체가 고도로 복잡한 초소형 메인보드다.

| 요소명 | 역할 | 특징 | 비유 |
|:---|:---|:---|:---|
| **Bare Die (베어 다이)** | 패키징(껍데기)이 씌워지지 않은 날것의 반도체 칩들 | CPU, GPU, DRAM, RF 모뎀, PMIC 등 이기종 다이들이 조합됨 | 햄버거의 패티, 양상추, 치즈 |
| **Substrate (패키지 기판)** | 칩들을 올려놓고 전기적으로 연결하는 베이스 판 | 고밀도 미세 회로가 인쇄되어 있으며 메인보드와 연결되는 BGA(솔더볼)를 가짐 | 햄버거를 받치는 트레이 |
| **Interposer (인터포저)** | 다이와 다이 사이를 빛의 속도로 연결하는 중간 다리 (2.5D) | 유기 기판(PCB)보다 수백 배 촘촘한 배선을 뚫을 수 있는 실리콘 재질의 판 | 초고속 데이터 전송 케이블 |
| **수동 소자 (Passive Components)** | 전류/전압을 안정화하는 부품 | 커패시터(MLCC), 저항, 인덕터 등 (SoC에는 넣기 힘든 부품들) | 양념(소스) |
| **EMC (에폭시 몰딩 컴파운드)** | 모든 부품을 덮어 물리적/화학적으로 보호하는 덮개 | 외부의 충격, 습기, 먼지로부터 칩을 보호하며 열을 분산시킴 | 햄버거 포장지 |

### SoC (System on Chip) vs SiP (System in Package) 아키텍처 비교

반도체 집적 기술은 "같은 실리콘 위에 그릴 것인가(SoC)" vs "패키지 안에서 조립할 것인가(SiP)"의 싸움이다.

```text
  ┌─────────────────────────────────────────────────────────────┐
  │                   SoC vs SiP 아키텍처 구조 비교                   │
  ├─────────────────────────────────────────────────────────────┤
  │                                                             │
  │ [1] SoC (System on Chip) - "단일 실리콘 통일"                   │
  │   ┌─────────────────────────────────────────────────────┐ │
  │   │     플라스틱 패키징 (Molding)                             │ │
  │   │  ┌───────────────────────────────────────────────┐  │ │
  │   │  │ CPU │ GPU │ NPU │ SRAM │ I/O Controller │  │  │ │
  │   │  └───────────────────────────────────────────────┘  │ │
  │   │     ↑ (모두 동일한 3nm 최첨단 실리콘 웨이퍼 한 판에서 생성됨) │ │
  │   └──────┬┬┬┬┬┬┬───────────────────────────────────┬┬┬┬┬┬┬──┘ │
  │      * 단점: 공간 낭비, 수율 극악, 아날로그 회로 통합 불리           │
  │                                                             │
  │ ─────────────────────────────────────────────────────────── │
  │                                                             │
  │ [2] SiP (System in Package) - "이기종 레고 블록 조립"             │
  │   ┌─────────────────────────────────────────────────────┐ │
  │   │     플라스틱 패키징 (Molding) + EMI 차폐 쉴드                 │ │
  │   │                                                     │ │
  │   │   [칩 1]     [칩 2]     [수동소자]    [칩 3 (3D 적층)] │ │
  │   │   ┌────┐   ┌────┐     ┌──┐        ┌────┐          │ │
  │   │   │ CPU│   │ RF │     │저항│        │DRAM│          │ │
  │   │   │(3nm)│   │(14nm)│     └──┘        ├────┤          │ │
  │   │   └─┬──┘   └─┬──┘                  │DRAM│          │ │
  │   │     │        │                      └─┬──┘          │ │
  │   │  ┌──┴────────┴──────────────────────────┴────────┐  │ │
  │   │  │     인터포저 (Interposer) 또는 패키지 기판(Substrate)  │  │ │
  │   │  └────────────────────────────────────────────────┘  │ │
  │   └──────┬┬┬┬┬┬┬───────────────────────────────────┬┬┬┬┬┬┬──┘ │
  │      * 장점: 메인보드에 따로 꽂아야 할 수십 개의 부품을 하나의 칩  │
  │              크기로 압축. 각 칩의 최적 공정 혼합 사용 가능.         │
  └─────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 상단의 **SoC**는 스마트폰 두뇌(AP)를 만들 때 쓰던 고전적 방식이다. 디지털 논리 회로든, 외부와 통신하는 I/O 회로든 무조건 비싼 최신 공정 웨이퍼 위에 같이 그려 넣어야 했다.
하단의 **SiP**는 완전히 다른 접근이다. 애플워치의 심장인 S시리즈 칩이 대표적이다. 껍데기를 까보면 그 안에 연산을 담당하는 CPU 다이, 통신을 담당하는 RF 다이, 전력을 관리하는 PMIC, 그리고 수십 개의 미세한 커패시터들이 빼곡하게 들어차 있다. 즉, SiP는 칩이 아니라 **"초소형 메인보드 전체를 플라스틱 껍데기로 덮어 하나의 칩처럼 위장시킨 것"**이다.

---

### 심층 동작 원리: EMI 차폐 (Electromagnetic Interference Shielding)
SiP 내부에는 CPU(고속 디지털 신호)와 RF 모뎀(아날로그 무선 신호)이 1mm도 안 되는 거리에 붙어 있다. 이 둘은 전자기 간섭(EMI)을 일으켜 서로의 신호를 망가뜨린다.
이를 막기 위해 SiP 공정의 마지막 단계에서는 패키지 겉면 전체를 미세한 금속막(Sputtering)으로 코팅하거나, 패키지 내부에 구리 기둥(Trench) 장벽을 세워 칩과 칩 사이의 전파 간섭을 물리적으로 차단(Compartmental Shielding)하는 극한의 소재 공학 기술이 투입된다.

## Ⅲ. 융합 비교 및 다각도 분석

### SoC, SiP, PCB, Chiplet 개념 비교

이 용어들은 헷갈리기 쉽지만 집적(Integration)의 레벨이 다르다.

| 개념 | 집적의 공간 | 특징 | 주요 적용 분야 |
|:---|:---|:---|:---|
| **SoC (System on Chip)** | 단일 실리콘 (Die) 내부 | 모든 회로를 한 판에 굽기. 속도는 가장 빠르나 설계가 경직됨 | 구형 스마트폰 AP, 마이크로컨트롤러(MCU) |
| **SiP (System in Package)**| 단일 패키징 (Molding) 내부 | 여러 다이와 수동 소자(커패시터)까지 하나로 몰딩. 소형화 끝판왕 | 애플워치, 에어팟, IoT 센서 모듈 |
| **Chiplet (칩렛)** | 패키징 내부의 '연산 조각들' | SiP의 일종. 주로 고성능 디지털 칩(CPU/GPU) 분할에 초점 | 서버용 고성능 CPU (AMD EPYC, Intel 뷰트) |
| **PCB (메인보드 기판)** | 패키징 외부 (보드 레벨) | 완성된 패키지(칩)들을 꽂는 거대한 판. 신호 지연이 가장 큼 | 데스크톱 PC, 일반 서버 보드 |

### 과목 융합 관점
- **무선 네트워크 (5G/RF 모듈)**: 스마트폰이 5G로 넘어가면서 안테나 신호를 처리하는 부품이 기하급수적으로 늘었다. 스마트폰 메인보드 공간은 한정되어 있으므로, 퀄컴이나 브로드컴은 안테나, 전력증폭기(PA), 필터 등을 하나의 칩 껍데기 안에 다 구겨 넣은 **AiP (Antenna in Package, SiP의 일종)** 모듈을 만들어 스마트폰 제조사에게 납품한다.
- **소프트웨어 공학 (HW/SW Co-design)**: SiP 설계 시에는 열(Thermal)과 전력(Power)이 가장 큰 문제다. 하드웨어 엔지니어가 칩 배치를 끝내면, 소프트웨어 엔지니어는 이 배치를 기반으로 "A칩과 B칩이 동시에 최고 클럭으로 돌면 패키지가 녹아버리니, OS 스케줄러가 절대 두 칩을 동시에 깨우지 못하게 하라"는 식의 하드웨어-소프트웨어 공동 설계(Co-design) 규칙을 커널에 하드코딩해야 한다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오
1. **시나리오 — 웨어러블 디바이스 (무선 이어폰) 설계**: 지름 1cm 남짓한 무선 이어폰 안에 블루투스 통신 칩, 오디오 디코더(DSP), 노이즈 캔슬링용 마이크 센서, 전력 관리 칩, 램(RAM)을 모두 넣어야 한다.
   - **아키텍처 적용 (애플 H1 칩 사례)**: 이 부품들을 메인보드(PCB)에 따로 납땜하면 이어폰 크기가 사과만 해진다. 따라서 모든 베어 다이(Bare Die)와 수동 소자들을 하나의 **SiP**로 묶는다. 이 SiP 패키지의 크기는 고작 손톱 반의반 만하며, 이 패키지 덩어리 하나만 이어폰 껍데기 안에 쏙 집어넣으면 하드웨어 설계가 끝난다. 극강의 폼팩터 축소가 이루어진다.
2. **시나리오 — 초고해상도 이미지 센서 (CIS) 제작**: 스마트폰 카메라가 1억 화소를 돌파하면서, 빛을 받아들이는 픽셀(Pixel) 영역과 이를 디지털 데이터로 변환하는 아날로그/로직 영역이 서로 다른 공정을 요구하게 되었다.
   - **설계 구조 (3D SiP)**: 위층(Top 다이)에는 빛을 가장 잘 받아들이는 65nm 특수 센서 공정으로 픽셀 다이를 만들고, 아래층(Bottom 다이)에는 데이터를 고속 처리하는 14nm 로직 공정 다이를 만든다. 이 두 다이를 위아래로 포개어 구리 패드(Cu-Cu Bonding)로 붙여버린 뒤 하나의 이미지 센서 칩(SiP)으로 포장한다. 현재 소니와 삼성이 만드는 모든 고급 스마트폰 카메라는 이 방식이다.

### 도입 체크리스트
- **KGD (Known Good Die) 확보 문제**: SiP의 치명적 리스크는 "불량 다이 하나가 전체 패키지를 망친다"는 것이다. 5개의 완벽한 칩과 1개의 불량 칩을 묶어서 패키징하면, 패키징이 끝난 후 테스트할 때 그 햄버거 전체를 쓰레기통에 버려야 한다. 따라서 SiP 제조사는 패키징을 시작하기 전에 각 칩 조각들이 100% 정상 작동하는지 완벽하게 검증(KGD 테스트)하는 것이 원가 절감의 핵심 기술이다.

## Ⅴ. 기대효과 및 결론

### 정량/정성 기대효과

| 구분 | 개별 칩 + PCB 보드 조립 | 시스템 온 패키지 (SiP) 도입 | 개선 효과 |
|:---|:---|:---|:---|
| **정량 (폼팩터, 면적)**| 스마트폰 크기의 거대한 기판 필요 | 손톱만 한 칩 하나로 통합 집적 | 전체 회로 면적 **70~90% 감소 (웨어러블 구현 가능)** |
| **정량 (신호 지연)** | PCB 배선을 타고 밀리초 지연 발생 | 패키지 내부 마이크로미터 단위 통신 | 신호 전송 거리 단축으로 지연율(Latency) 및 전력 소모 급감 |
| **정성 (개발 비용)** | 기능 추가 시 전체 메인보드 재설계 | 이미 만들어진 레고 블록(다이)만 교체 | 고객 맞춤형 칩(Custom Silicon)의 설계/개발 기간 획기적 단축 |

### 미래 전망
- **SiP의 확장, SiIM (System in Integration Module)**: 미래에는 반도체뿐만 아니라, 빛(Photonics), 유체(Micro-fluidics, 바이오 칩), 마이크로 안테나까지 이기종의 극한을 달리는 물리적 요소들이 하나의 패키지 안으로 들어올 것이다. 혈액 한 방울을 SiP 패키지에 떨어뜨리면 칩 내부에서 분석하여 클라우드로 데이터를 전송하는 의료용 IoT 칩이 상용화될 것이다.
- **파운드리와 OSAT의 경계 붕괴**: 과거에는 TSMC가 웨이퍼를 굽고(파운드리), ASE 같은 하청업체가 껍데기를 씌웠다(OSAT). 하지만 SiP와 2.5D/3D 패키징이 너무나 고도의 기술이 되면서, TSMC(CoWoS)나 인텔(Foveros) 같은 초일류 파운드리가 직접 패키징까지 다 해버리는(턴키) 시대로 반도체 생태계가 완전히 재편되었다.

### 참고 표준
- **JEDEC (Joint Electron Device Engineering Council)**: 반도체 패키징 규격(BGA 핀 배열, 패키지 크기, 열 저항 등)을 정하는 국제기구. SiP가 어떤 규격으로 외부 메인보드와 결합할지 이 표준을 따른다.

## 👶 어린이를 위한 3줄 비유 설명
1. 예전에는 빵 따로, 고기 따로, 야채 따로 사 와서 식탁(메인보드) 위를 꽉 채워놓고 밥을 먹었어요.
2. 그런데 책상이 너무 좁아져서(스마트워치), 공장에서 아예 빵, 고기, 야채를 층층이 쌓아서 한 입에 먹을 수 있는 '햄버거 세트(**SiP**)'로 포장해서 팔기 시작했어요.
3. 겉보기에는 그냥 하나의 작은 햄버거지만, 속에는 세상의 온갖 맛있는 재료(칩)들이 꽉꽉 들어찬 마법의 포장 상자랍니다!
