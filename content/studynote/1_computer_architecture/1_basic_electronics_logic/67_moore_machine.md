## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 무어 머신(Moore Machine)은 유한 상태 머신(FSM)의 한 종류로, 시스템의 출력이 **오직 현재의 상태(Present State)**에 의해서만 결정되는 순차 논리 모델이다.
> 2. **가치**: 입력의 변화가 출력에 즉각 반영되지 않고 클럭 에지에 동기화되어 상태가 변할 때만 출력이 갱신되므로, 신호의 글리치(Glitch)가 없고 타이밍 안정성이 매우 뛰어나다.
> 3. **융합**: CPU의 마이크로 명령어나 데이터패스 제어처럼 안정성이 최우선인 로직 설계의 표준이며, 고속 디지털 설계에서 신호의 위상을 정렬하는 리타이밍(Retiming) 전략의 근간이 된다.

---

## Ⅰ. 개요 (Context & Background)

- **개념**: 무어 머신(Moore Machine)은 1956년 에드워드 무어(Edward F. Moore)가 제안한 모델로, 출력값이 상태 노드 내부에 고정되어 있다. 즉, 입력 $X$가 아무리 요동쳐도 현재 상태가 $S_n$인 한 출력은 변하지 않으며, 오직 클럭이 뛰어 상태가 $S_{n+1}$로 바뀔 때만 새로운 출력이 나온다.

- **💡 비유**: 무어 머신은 **"신호등"**과 같다. 보행자가 버튼(입력)을 누른다고 해서 신호등 불빛(출력)이 즉시 바뀌지 않는다. 시스템의 내부 타이머(상태)가 다음 단계로 넘어갈 때까지 기다린 후에야 불빛이 바뀐다. 출력이 오직 '현재 어떤 색깔(상태)인가'에만 매여 있기 때문이다.

- **등장 배경: 동기식 설계의 안정성 확보**:
  - 초기 디지털 회로에서는 입력 노이즈가 출력으로 전파되어 시스템이 오작동하는 일이 잦았다.
  - 이를 방지하기 위해 입력을 즉시 출력으로 내보내지 않고, 반드시 '상태'라는 필터를 거치게 하여 클럭에 완벽히 동기화된 안전한 신호를 얻기 위해 무어 모델이 널리 채택되었다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 무어 머신의 논리 구조 및 데이터 흐름

무어 머신의 핵심은 출력 로직이 입력 변수 $X$와 분리되어 있다는 점이다.

```text
 ┌───────────────────────────────────────────────────────────────┐
 │                  무어 머신 (Moore Machine) 블록 다이어그램          │
 ├───────────────────────────────────────────────────────────────┤
 │                                                               │
 │   입력(X) ──▶┌────────────┐     ┌────────────┐                │
 │              │ 차기 상태  │─────▶│ 상태 저장  │──┬─────────────┐│
 │              │ 조합 논리  │     │ (FF Array) │  │             ││
 │   ┌─────────▶└────────────┘     └────────────┘  │             ││
 │   │                                   ▲         │             ││
 │   │                                   │ 클럭    │             ││
 │   └───────────────────────────────────┴─────────┘             ││
 │                                         현재 상태(Q)          ││
 │                                               │               ││
 │                                               ▼               ││
 │                                         ┌────────────┐        ││
 │                                         │ 출력 조합  │────▶ 출력(Y)│
 │                                         │    논리    │        ││
 │                                         └────────────┘        ││
 │                                                               │
 │   * 핵심 특징: 출력 Y는 입력 X를 참조하지 않고 오직 Q만 참조함    │
 └───────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 위 아키텍처에서 출력 조합 논리(Output Logic)는 오직 상태 저장 장치(플립플롭)에서 나온 현재 상태($Q$)만 입력으로 받는다. 따라서 외부 입력($X$)에서 발생하는 미세한 전기적 잡음(글리치)은 차기 상태 논리에는 영향을 줄 수 있어도, 현재 나오고 있는 출력($Y$)에는 물리적으로 전달될 경로가 없다. 이는 무어 머신이 **'글리치 프리(Glitch-free)'** 특성을 갖게 하는 근본적인 이유이며, 설계자는 이를 통해 매우 깨끗하고 정렬된 제어 신호를 생성할 수 있다.

---

### 무어 머신의 상태 전이 및 출력 표기

상태도(State Diagram)에서 무어 머신은 다음과 같이 표기한다.

- **상태 노드**: `S0 / 0` (상태 이름 / 출력값)
- **천이 선**: `In` (화살표 위에는 오직 입력값만 기재)

## Ⅲ. 융합 비교 및 다각도 분석

### 비교 1: 무어 머신 vs 밀리 머신 (Moore vs Mealy)

| 비교 항목 | 무어 머신 (Moore) | 밀리 머신 (Mealy) |
|:---|:---|:---|
| **출력 결정** | **오직 현재 상태 ($Q$)** | 현재 상태 ($Q$) + 입력 ($X$) |
| **반응 속도** | 1 클럭 지연 (느림) | **즉각적 반응 (빠름)** |
| **타이밍 안정성** | **매우 높음 (글리치 없음)** | 낮음 (입력 노이즈 전파 가능) |
| **상태 수** | 상대적으로 많음 | 상대적으로 적음 (최적화 유리) |
| **주 사용처** | 제어 신호, 동기화 로직 | 고속 데이터 처리, 버스 프로토콜 |

### 비교 2: 하드웨어 설계 관점의 트레이드오프

| 항목 | Moore 선택 시 이점 | Mealy 선택 시 이점 |
|:---|:---|:---|
| **설계 난이도** | 쉬움 (상태에 집중) | 어려움 (입력과 상태의 조합 고려) |
| **시스템 속도** | Fmax(최대 주파수) 향상 유리 | 전체 처리 레이턴시 단축 유리 |
| **안정성** | **임계 경로(Critical Path) 관리 용이** | 글리치 필터 회로 추가 필요 가능성 |

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오: 산업용 컨베이어 벨트 비상 정지 시스템 설계

1. **상황**: 작업자가 비상 버튼을 눌렀을 때 기계가 멈춰야 함. 정전기나 노이즈로 인한 일시적 신호 튀어 오름에 기계가 덜컥거리면 안 됨.
2. **기술적 판단**:
   - 밀리 머신을 쓰면 버튼 신호의 노이즈가 즉시 모터 제어 신호로 나가서 기계 수명을 갉아먹음.
   - **무어 머신** 도입: 버튼 신호를 클럭에 동기화하여 '정지 상태'로 확정된 후에만 모터 전원을 차단.
3. **의사결정**: 한 박자 느리더라도(10ns 지연) 완벽하게 깨끗한 정지 신호를 보장하는 무어 모델을 채택하여 시스템 신뢰성 확보.

### 도입 체크리스트 (Design Reliability)
- [ ] **Synchronization**: 외부 비동기 입력이 무어 머신에 들어가기 전 2-FF 동기화 과정을 거쳤는가?
- [ ] **State Complexity**: 밀리 모델로 설계할 때보다 상태 수가 너무 많아져서 플립플롭 면적이 예산을 초과하지 않는가?
- [ ] **Timing Slack**: 출력 논리가 오직 상태만 참조하므로, 출력단 뒤에 붙는 조합 논리와의 타이밍 마진(Tsu)이 충분히 확보되었는가?

### 안티패턴: 반응 속도만을 강조한 무분별한 밀리 머신 사용
- 모든 제어 로직을 반응이 빠른 밀리 머신으로만 짜면, 입력 신호의 노이즈가 칩 전체로 퍼져나가 원인 불명의 오동작을 일으킨다. 특히 서로 다른 칩 간의 통신 신호는 반드시 무어 머신(혹은 출력 버퍼링된 밀리 머신)을 통해 '정제된' 상태로 내보내야 한다.

## Ⅴ. 기대효과 및 결론

### 정량/정성 기대효과

| 구분 | 기대효과 | 비즈니스 가치 |
|:---:|:---|:---|
| **정량** | 출력 글리치 100% 제거 | 시스템 가동 중단(Downtime) 사고 방지 |
| **정량** | 타이밍 클로저(Closure) 시간 30% 단축 | 반도체 설계 및 검증 기간 단축 |
| **정성** | 설계의 모듈화 및 재사용성 향상 | 플랫폼 아키텍처의 확장성 확보 |

### 미래 전망
- **Formal Verification Friendly**: 무어 머신은 상태 기반의 정형 검증(Formal Verification)이 매우 용이하여, 자율주행이나 의료용 칩처럼 보안과 안전이 생명인 분야에서 표준 모델로 더욱 공고해질 것이다.
- **Clockless Moore Architecture**: 클럭 없이 전하의 평형 상태로만 전이되는 차세대 무어 머신 구조가 연구되어, 기존 대비 1/100의 전력만 소모하는 초저전력 제어 시대를 준비하고 있다.

### 👶 어린이를 위한 3줄 비유 설명
1. 무어 머신은 **"말을 잘 듣는 인형"**과 같아요.
2. 우리가 인형의 팔을 아무리 흔들어도(입력), 인형은 자기가 현재 '웃는 상태'라면 계속 웃는 표정(출력)만 짓고 있죠.
3. 주인님이 "표정을 바꿔!"라고 명령(클럭)을 내린 후에야 비로소 다음 표정으로 바뀌기 때문에, 갑자기 표정이 찌그러지는 일 없이 아주 안정적이에요!
