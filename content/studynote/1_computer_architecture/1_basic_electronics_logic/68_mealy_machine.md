## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 밀리 머신(Mealy Machine)은 유한 상태 머신(FSM)의 한 종류로, 시스템의 출력이 **현재의 상태(Present State)와 현재의 입력(Input) 모두**에 의해 결정되는 순차 논리 모델이다.
> 2. **가치**: 입력 변화가 클럭 에지를 기다리지 않고 즉시 출력에 반영되므로 반응 속도(Responsiveness)가 매우 빠르며, 동일한 논리를 무어 머신보다 더 적은 수의 상태로 구현할 수 있어 회로 면적 효율성이 높다.
> 3. **융합**: 고속 패킷 처리, 실시간 인터럽트 응답, 버스 프로토콜 제어 등 지연 시간(Latency) 최소화가 생명인 현대 고성능 SoC(System on Chip) 설계의 핵심 아키텍처다.

---

## Ⅰ. 개요 (Context & Background)

- **개념**: 밀리 머신(Mealy Machine)은 1955년 조지 밀리(George H. Mealy)가 제안한 모델이다. 출력값이 상태 노드가 아닌 **'상태 전이(Transition)'** 자체에 결합되어 있다. 즉, 시스템이 $S_0$ 상태에 있더라도 입력 $X$가 바뀌는 순간 출력 $Y$도 즉시 바뀔 수 있는 비동기적 통로를 가진다.

- **💡 비유**: 밀리 머신은 **"자동문의 센서"**와 같다. 자동문 시스템이 '닫힘 상태(상태)'에 있더라도, 사람이 다가오는 것(입력)을 감지하는 그 즉시 문이 열리기 시작(출력)해야 한다. 다음 클럭 박자까지 기다렸다가 열리면 사람이 문에 부딪힐 수 있기 때문이다. 상황(입력)에 따라 즉각적인 대처가 필요한 시스템에 최적화되어 있다.

- **등장 배경: 하드웨어 자원 최적화와 속도 경쟁**:
  - 초기 반도체 설계에서 플립플롭은 매우 비싼 자원이었다. 밀리 머신은 입력값을 출력 논리에 직접 활용함으로써, 무어 머신보다 적은 플립플롭으로도 동일한 복잡도의 일을 수행할 수 있게 해주었다.
  - 또한, 통신 속도가 비약적으로 빨라지면서 클럭 한 주기조차 아까운 '초저지연(Ultra-low latency)' 요구사항을 충족하기 위해 밀리 모델이 필수적으로 사용되었다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 밀리 머신의 논리 구조 및 데이터 흐름

밀리 머신의 핵심은 출력 로직이 외부 입력 변수 $X$에 직접 연결되어 있다는 점이다.

```text
 ┌───────────────────────────────────────────────────────────────┐
 │                  밀리 머신 (Mealy Machine) 블록 다이어그램          │
 ├───────────────────────────────────────────────────────────────┤
 │                                                               │
 │   입력(X) ──┬──▶┌────────────┐     ┌────────────┐             │
 │             │   │ 차기 상태  │─────▶│ 상태 저장  │──┬──────────┐ │
 │             │   │ 조합 논리  │     │ (FF Array) │  │          │ │
 │   ┌─────────┼──▶└────────────┘     └────────────┘  │          │ │
 │   │         │                            ▲         │          │ │
 │   │         │                            │ 클럭    │          │ │
 │   │         │                      현재 상태(Q)    │          │ │
 │   │         │                            │         │          │ │
 │   │         └────────────────────────────┼─────────┘          │ │
 │   │                                      ▼                    │ │
 │   │                                ┌────────────┐             │ │
 │   └───────────────────────────────▶│ 출력 조합  │───▶ 출력(Y)  │ │
 │                                    │    논리    │             │ │
 │                                    └────────────┘             │ │
 │                                                               │
 │   * 핵심 특징: 출력 Y가 입력 X와 상태 Q의 조합으로 즉시 결정됨    │
 └───────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 위 아키텍처를 보면 출력 조합 논리(Output Logic)가 상태 저장 장치($Q$)뿐만 아니라 외부 입력($X$)도 직접 받고 있다. 이 '직통 노선' 덕분에 입력이 변하면 클럭 에지(상승/하강)가 오기 전이라도 출력이 즉각적으로 변할 수 있다. 하지만 이 경로는 양날의 검이다. 입력에 섞여 들어온 전기적 노이즈(글리치)가 필터링 없이 그대로 출력으로 나갈 수 있기 때문이다. 따라서 밀리 머신 설계 시에는 이 '조합 논리 경로(Combinational Path)'의 지연 시간과 안정성을 면밀히 계산해야 한다.

---

### 밀리 머신의 상태 전이 및 출력 표기

상태도(State Diagram)에서 밀리 머신은 다음과 같이 표기한다.

- **상태 노드**: `S0` (상태 이름만 기재)
- **천이 선**: `In / Out` (화살표 위에 입력과 그에 따른 출력을 세트로 기재)

## Ⅲ. 융합 비교 및 다각도 분석

### 비교 1: 무어 머신 vs 밀리 머신 (Moore vs Mealy)

| 비교 항목 | 무어 머신 (Moore) | 밀리 머신 (Mealy) |
|:---|:---|:---|
| **출력 결정** | 현재 상태 ($Q$) | **현재 상태 ($Q$) + 입력 ($X$)** |
| **반응성** | 느림 (다음 클럭 에지) | **매우 빠름 (입력 즉시)** |
| **상태 수** | 많음 (복잡함) | **적음 (단순함)** |
| **타이밍 안정성** | **우수 (글리치 프리)** | 낮음 (글리치 전파 위험) |
| **하드웨어 면적** | 상대적으로 큼 | **상대적으로 작음** |

### 비교 2: Mealy 모델의 장단점 분석

| 구분 | 내용 | 기술적 임팩트 |
|:---|:---|:---|
| **장점** | **상태 최적화** | 플립플롭 소모를 줄여 칩 비용 절감 |
| **장점** | **비동기 반응** | 초저지연 시스템 구현 가능 |
| **단점** | **글리치 발생** | 후속 회로에 오동작 유발 가능성 |
| **단점** | **타이밍 클로저** | 조합 논리 경로가 길어져 최대 클럭 제한 |

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오: 고속 통신 버스(PCIe/USB)의 핸드셰이크 제어

1. **상황**: 상대방 칩에서 '데이터 준비 완료(Ready)' 신호가 오면, 우리 쪽에서도 즉시 '수신 가능(Ack)' 신호를 보내야 함. 클럭 한 주기를 기다리면 전송 효율(Throughput)이 20% 이상 하락함.
2. **기술적 판단**:
   - 무어 머신은 1클럭의 레이턴시가 발생하여 고속 버스 규격을 만족하기 어려움.
   - **밀리 머신** 채택: Ready 신호가 들어오는 즉시 조합 논리를 통해 Ack 신호를 생성.
3. **의사결정**: 출력단에 아주 작은 노이즈 필터(Deglitcher)를 추가한 밀리 모델을 사용하여, 레이턴시 제로에 가까운 초고속 인터페이스 구현.

### 도입 체크리스트 (Performance & Safety)
- [ ] **Glitch Sensitivity**: 밀리 머신의 출력이 클럭 에지에 민감한 다른 플립플롭의 입력으로 들어갈 때, 충분한 셋업/홀드 타임 마진이 확보되었는가?
- [ ] **Combinational Loop**: 입력이 출력을 거쳐 다시 자신의 입력으로 돌아오는 '조합 논리 루프'가 형성되어 발진(Oscillation)하지 않는가?
- [ ] **State Reduction**: 밀리 모델의 강점인 상태 수 최소화를 통해 면적 이득을 충분히 보았는가?

### 안티패턴: 복잡한 조합 논리를 가진 밀리 머신
- 출력 논리에 너무 많은 AND/OR 게이트를 때려 박으면, 입력에서 출력까지 가는 지연 시간(Tpd)이 너무 길어진다. 이렇게 되면 클럭 속도를 높일 수 없어 전체 시스템 성능이 오히려 떨어진다. 밀리 머신의 출력 논리는 최대한 단순하게(Shallow logic) 유지해야 한다.

## Ⅴ. 기대효과 및 결론

### 정량/정성 기대효과

| 구분 | 기대효과 | 비즈니스 가치 |
|:---:|:---|:---|
| **정량** | 상태 수 20~30% 감소 | 반도체 다이(Die) 면적 효율 극대화 |
| **정량** | 응답 속도(Latency) 1 Cycle 단축 | 실시간 시스템의 실질 성능 향상 |
| **정성** | 복잡한 외부 이벤트에 대한 유연한 대응 | 다양한 통신 표준과의 호환성 확보 |

### 미래 전망
- **Hybrid FSM**: 무어의 안정성과 밀리의 속도를 결합하여, 중요한 신호는 무어로 내보내고 빠른 응답이 필요한 신호는 밀리로 처리하는 하이브리드 아키텍처가 SoC 설계의 정석으로 자리 잡고 있다.
- **Asynchronous Pulse Logic**: 클럭 자체를 없애고 밀리 머신의 비동기적 특성만을 극대화한 '클럭리스(Clockless)' 프로세서가 차세대 초저전력 가속기로 부상하고 있다.
