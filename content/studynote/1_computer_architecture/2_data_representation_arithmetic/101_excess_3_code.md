# 3초과 코드 (Excess-3 Code)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 3초과 코드(Excess-3 Code, XS-3)는 각 십진수(0~9)에 3을 더한 값(3~12)을 4비트 이진수로 표현하는 BCD 변형으로, 자기 보수(Self-Complementing) 특성을 가진다.
> 2. **가치**: 3초과 코드에서 9의 보수(각 비트 반전)를 취하면 10의 보수가 되어, BCD 뺄셈과 음수 표현이 단순화된다. 이 때문에 초기 컴퓨터(UNIVAC)와 계산기에서 사용되었다.
> 3. **융합**: 3초과 코드는 현대 x86이나 ARM에서 직접 지원하지 않지만, BCD 연산 하드웨어 설계의 역사적 이정표이며, 자기 보수 코드의 대표적인 예시로 교육적 가치가 높다.

---

## Ⅰ. 개요 (Context & Background)

- **개념**: 3초과 코드(Excess-3 Code, XS-3)는 십진수 0~9를 표현하기 위해, 각 숫자에 3을 더한 후 4비트 이진수로 인코딩하는 방식이다. 예를 들어 십진수 0은 0+3=3이므로 `0011`, 십진수 9는 9+3=12이므로 `1100`으로 표현된다. 핵심 특성은 "각 비트를 반전(invert)하면 9의 보수가 된다"는 자기 보수(Self-Complementing) 속성이다.

- **💡 비유**: 3초과 코드는 **'각 숫자를 3만큼 위로 밀어놓은 자판'**과 같다. 일반 BCD는 0~9를 직접 누르지만, 3초과 코드는 모든 버튼이 3칸 위로 올라가 있다. 0을 누르려면 '3' 버튼을, 9를 누르려면 '12' 버튼을 눌러야 한다. 하지만 이렇게 하면 "버튼을 반대로 누르면(0↔1) 보수가 된다"는 편리한 성질이 생긴다.

- **등장 배경 및 발전 과정**:
  1. **UNIVAC의 채택 (1951)**: 최초의 상업용 컴퓨터인 UNIVAC I은 3초과 코드를 내부 데이터 표현으로 사용했다. 당시 하드웨어로 9의 보수를 구하는 것이 BCD 뺄셈에 필수적이었다.
  2. **자기 보수 특성의 활용**: 일반 BCD에서 9의 보수를 구하려면 복잡한 회로가 필요하지만, 3초과 코드는 각 비트를 단순히 NOT 게이트로 반전하면 된다. 이는 게이트 수를 크게 줄였다.
  3. **현대의 역할**: 2의 보수가 표준이 되면서 3초과 코드는 역사적/교육적 목적으로 남아 있다. 그러나 자기 보수 코드의 개념은 코딩 이론과 오류 검출에서 여전히 중요하다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 3초과 코드 인코딩 테이블

3초과 코드는 십진수 + 3을 4비트 이진수로 표현한다.

```text
  ┌──────────────────────────────────────────────────────────────────────────────┐
  │         3초과 코드 (Excess-3 Code) 인코딩 테이블                             │
  ├──────────────────────────────────────────────────────────────────────────────┤
  │                                                                              │
  │   십진수    BCD     십진수+3    3초과 코드    9의 보수    반전된 3초과 코드    │
  │   ────────────────────────────────────────────────────────────────────────   │
  │     0       0000       3        0011          9          1100                │
  │     1       0001       4        0100          8          1011                │
  │     2       0010       5        0101          7          1010                │
  │     3       0011       6        0110          6          1001                │
  │     4       0100       7        0111          5          1000                │
  │     5       0101       8        1000          4          0111                │
  │     6       0110       9        1001          3          0110                │
  │     7       0111      10        1010          2          0101                │
  │     8       1000      11        1011          1          0100                │
  │     9       1001      12        1100          0          0011                │
  │                                                                              │
  │   ⚠ 자기 보수 특성:                                                        │
  │   ┌────────────────────────────────────────────────────────────────────────┐│
  │   │  NOT(3초과 코드) = 9의 보수의 3초과 코드                                ││
  │   │                                                                         ││
  │   │  예: 십진수 2의 3초과 코드 = 0101                                       ││
  │   │      NOT(0101) = 1010                                                  ││
  │   │      1010 = 십진수 7의 3초과 코드                                       ││
  │   │      7 = 9 - 2 (2의 9의 보수) ✓                                        ││
  │   │                                                                         ││
  │   │  → 비트 반전만으로 9의 보수를 얻을 수 있음!                              ││
  │   └────────────────────────────────────────────────────────────────────────┘│
  │                                                                              │
  │   ⚠ 3초과 코드는 0000, 0001, 0010, 1101, 1110, 1111을 사용하지 않음          │
  │      이 값들은 Invalid Excess-3로, 오류 검출에 활용 가능                      │
  │                                                                              │
  └──────────────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 3초과 코드는 십진수에 3을 더한 값을 4비트로 표현한다. 0→3(0011), 9→12(1100). 핵심은 비트 반전이 9의 보수가 된다는 점이다. 예를 들어 2(0101)를 반전하면 1010이 되고, 이는 7(9-2)의 3초과 코드다. 이 자기 보수 특성 덕분에 BCD 뺄셈 시 복잡한 9의 보수 회로가 필요 없고, NOT 게이트만으로 충분하다.

### 3초과 코드 덧셈

3초과 코드 덧셈은 일반 이진수 덧셈 후, 결과에서 3을 빼거나 더하여 보정한다.

```text
  ┌──────────────────────────────────────────────────────────────────────────────┐
  │         3초과 코드 덧셈 알고리즘                                              │
  ├──────────────────────────────────────────────────────────────────────────────┤
  │                                                                              │
  │   알고리즘:                                                                  │
  │   1. 두 3초과 코드를 일반 이진수로 덧셈                                      │
  │   2. 캐리가 발생하면: 결과 + 3 (0011)                                        │
  │   3. 캐리가 없으면: 결과 - 3 (1101, 또는 0011의 2의 보수)                    │
  │   4. 캐리는 다음 자릿수로 전파                                               │
  │                                                                              │
  │   이유: 3초과 코드는 이미 3이 더해져 있으므로, 덧셈 시 3+3=6이 중복됨         │
  │                                                                              │
  │   예시 1: 2 + 3 = 5 (캐리 없음, 결과 - 3)                                    │
  │   ┌────────────────────────────────────────────────────────────────────────┐│
  │   │  십진수: 2 + 3 = 5                                                     ││
  │   │  3초과 코드: 0101 (2+3) + 0110 (3+3) = 1011 (11)                       ││
  │   │                                 ↑                                       ││
  │   │                          2+3=5이지만, 6이 중복됨                        ││
  │   │  1011 - 0011 = 1000 (8) = 십진수 5의 3초과 코드 ✓                       ││
  │   │                                                                         ││
  │   │  검증: 5 + 3 = 8 = 1000 ✓                                              ││
  │   └────────────────────────────────────────────────────────────────────────┘│
  │                                                                              │
  │   예시 2: 7 + 5 = 12 (캐리 발생, 결과 + 3, 캐리 전파)                        │
  │   ┌────────────────────────────────────────────────────────────────────────┐│
  │   │  십진수: 7 + 5 = 12                                                    ││
  │   │  3초과 코드: 1010 (7+3) + 1000 (5+3) = 10010                            ││
  │   │                                 ↑   ↑                                   ││
  │   │                              캐리 1  결과 0010 (2)                       ││
  │   │                                                                         ││
  │   │  캐리 있음 → 결과 + 3: 0010 + 0011 = 0101 (5)                           ││
  │   │  최종: 캐리 1 (10의 자리), 0101 (1의 자리 5)                             ││
  │   │  → 십진수 12의 3초과 코드: 1|0101 (캐리|5)                              ││
  │   │                                                                         ││
  │   │  검증: 1의 자리 2 → 2+3=5=0101 ✓, 10의 자리 1 → 1+3=4=0100              ││
  │   │  하지만 여기서는 캐리를 직접 10의 자리로 사용                            ││
  │   └────────────────────────────────────────────────────────────────────────┘│
  │                                                                              │
  │   ⚠ 3초과 코드 덧셈은 일반 BCD보다 복잡하지만, 뺄셈은 훨씬 간단             │
  │                                                                              │
  └──────────────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 3초과 코드 덧셈은 중복된 3(총 6)을 보정해야 한다. 캐리가 없으면 결과가 6만큼 크므로 3을 빼고, 캐리가 있으면 3만큼 작으므로 3을 더한다. 예를 들어 2+3=5는 0101+0110=1011(=11)이고, 11-3=8=1000이 5의 3초과 코드다. 덧셈은 BCD보다 복잡하지만, 뺄셈은 비트 반전만으로 9의 보수를 얻을 수 있어 훨씬 간단하다.

### 3초과 코드 뺄셈: 자기 보수의 장점

```text
  ┌──────────────────────────────────────────────────────────────────────────────┐
  │         3초과 코드 뺄셈 (자기 보수 활용)                                      │
  ├──────────────────────────────────────────────────────────────────────────────┤
  │                                                                              │
  │   알고리즘:                                                                  │
  │   1. 감수(빼는 수)의 비트를 반전 (NOT) → 9의 보수 획득                       │
  │   2. 피감수와 반전된 감수를 덧셈                                             │
  │   3. 캐리 보정 (순환 캐리)                                                   │
  │                                                                              │
  │   예시: 9 - 3 = 6                                                           │
  │   ┌────────────────────────────────────────────────────────────────────────┐│
  │   │  피감수 9의 3초과 코드: 1100                                            ││
  │   │  감수 3의 3초과 코드: 0110                                              ││
  │   │                                                                         ││
  │   │  Step 1: 감수의 비트 반전                                               ││
  │   │  NOT(0110) = 1001 (9의 보수인 6의 3초과 코드)                           ││
  │   │                                                                         ││
  │   │  Step 2: 덧셈                                                          ││
  │   │    1100 (9)                                                            ││
  │   │  + 1001 (6, 9-3의 보수)                                                ││
  │   │  ──────                                                                ││
  │   │  10101                                                                 ││
  │   │    ↑                                                                   ││
  │   │  캐리 1                                                                ││
  │   │                                                                         ││
  │   │  Step 3: 캐리 보정 (순환 캐리)                                          ││
  │   │  0101 + 1 (캐리) = 0110                                                ││
  │   │                                                                         ││
  │   │  0110 = 십진수 6의 3초과 코드 ✓ (9-3=3 → 3+3=6)                         ││
  │   │                                                                         ││
  │   │  검증: 9 - 3 = 6 ✓                                                     ││
  │   └────────────────────────────────────────────────────────────────────────┘│
  │                                                                              │
  │   ⚠ 일반 BCD에서 9의 보수를 구하려면 복잡한 회로가 필요                     │
  │      3초과 코드는 NOT 게이트 하나로 해결!                                   │
  │                                                                              │
  └──────────────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 3초과 코드 뺄셈의 핵심은 비트 반전만으로 9의 보수를 얻을 수 있다는 점이다. 9-3을 계산하려면, 3(0110)을 반전하여 6(1001)을 얻고, 9(1100)와 더한다. 결과 10101에서 순환 캐리(1)를 더하면 0110(6)이 되어 정답이다. 일반 BCD에서는 9의 보수 회로가 복잡하지만, 3초과 코드는 NOT 게이트 하나로 해결된다. 이것이 초기 컴퓨터가 3초과 코드를 선택한 이유다.

## Ⅲ. 융합 비교 및 다각도 분석

### 비교: 3초과 코드 vs BCD vs 그레이 코드

| 비교 항목 | 3초과 코드 | BCD (8421) | 그레이 코드 |
|:---|:---|:---|:---|
| **인접 수 차이** | 여러 비트 변경 | 여러 비트 변경 | 1비트만 변경 |
| **자기 보수** | O (비트 반전) | X (복잡한 회로) | X |
| **산술 연산** | 덧셈 복잡, 뺄셈 간단 | 덧셈/뺄셈 모두 보정 | 산술 연산 부적합 |
| **오류 검출** | Invalid 코드 존재 | Invalid 코드 존재 | 제한적 |
| **주 용도** | 초기 컴퓨터, 계산기 | 현장 계측, 디스플레이 | 엔코더, 위치 검출 |

### 과목 융합 관점
- **컴퓨터구조(컴구)**: 3초과 코드는 하드웨어 최소화 시대의 산물이다. 게이트 수를 줄이기 위해 설계되었으며, 자기 보수 코드의 대표적 예시다
- **디지털 논리회로**: 3초과 코드는 NOT 게이트로 9의 보수를 구할 수 있어, 가산기 기반 뺄셈기 설계에서 게이트 수를 절약한다
- **코딩 이론**: 자기 보수 코드는 Hamming 거리와 오류 검출 능력 분석의 좋은 예시다

## Ⅳ. 실무 적용 및 기술사적 판단
### 실무 시나리오
1. **시나리오 — 레거시 계산기 분석**: 1970년대 전자 계산기의 ROM을 역설계하는데, 3초과 코드로 데이터가 저장되어 있다. 기술사는 3초과 코드의 인코딩/디코딩과 자기 보수 특성을 이해하여 데이터를 해석한다
2. **시나리오 — 교육용 자기 보수 코드 설명**: 디지털 논리 설계 수업에서 자기 보수 코드의 개념을 설명해야 한다. 기술사는 3초과 코드를 예시로, NOT 게이트만으로 보수를 구할 수 있음을 시연한다
3. **시나리오 — BCD 회로 최적화 연구**: BCD 뺄셈기의 게이트 수를 최소화하는 연구에서, 3초과 코드 기반 설계와 일반 BCD 설계를 비교 분석한다. 기술사는 3초과 코드의 뺄셈 간단함 vs 덧셈 복잡함을 트레이드오프로 제시한다
### 도입 체크리스트
- **기술적**: 뺄셈 연산이 빈번하고, 게이트 수를 최소화해야 하는가? 그렇다면 3초과 코드를 고려할 수 있다
- **운영·보안적**: 현대 시스템과의 호환성이 필요한가? 그렇다면 일반 BCD 또는 2의 보수를 사용해야 한다
### 안티패턴
- **현대 시스템에서 3초과 코드 사용**: x86, ARM 등은 3초과 코드를 직접 지원하지 않아, 소프트웨어 에뮬레이션 오버헤드가 발생한다
- **Invalid 코드 미검증**: 3초과 코드에서 0000, 0001, 1110, 1111 등은 Invalid다. 이를 검증하지 않으면 오류가 발생할 수 있다

## Ⅴ. 기대효과 및 결론
### 정량/정성 기대효과
| 구분 | 일반 BCD | 3초과 코드 | 효과 |
|:---|:---|:---|:---|
| **정량** | 9의 보수 회로 복잡 | NOT 게이트만으로 해결 | 게이트 수 50%+ 절감 |
| **정량** | 덧셈 간단 | 덧셈 복잡 | 트레이드오프 |
| **정성** | 현대 표준 | 역사적/교육적 가치 | 설계 원리 이해 |
### 미래 전망
- **교육적 가치 지속**: 3초과 코드는 자기 보수 코드의 대표적 예시로, 디지털 논리 설계 교육에서 계속 사용될 것이다
- **특수 목적 회로**: 초저전력, 게이트 수 최소화가 필수적인 임베디드 시스템에서는 3초과 코드 기반 설계가 여전히 유효할 수 있다
### 참고 표준

- **[2의 보수 (2's Complement)](./85_2s_complement.md)**: 현대 정수 표현 표준. 3초과 코드와 유사한 편향 개념
- **[해밍 거리 (Hamming Distance)](./110_hamming_distance.md)**: 3초과 코드의 오류 검출 능력 분석

## 👶 어린이를 위한 3줄 비유 설명
1. 3초과 코드는 **'모든 버튼이 3칸 위로 올라간 자판'** 같아요. 0을 누르려면 '3' 버튼을, 9를 누르려면 '12' 버튼을 눌러야 해요
2. 하지만 이상하게도, 버튼을 반대로 누르면(0↔1) "보수"가 돼요! 예를 들어 '5' 버튼(0101)을 반대로 누르면 '4' 버튼(1010)이 되고, 5+4=9예요!
3. 옛날 컴퓨터는 이렇게 하면 뺄셈을 쉽게 할 수 있어서 이상한 자판을 썼어요. 지금은 더 좋은 방법이 생겨서 안 쓰지만, 역사 속의 발명품이에요!
