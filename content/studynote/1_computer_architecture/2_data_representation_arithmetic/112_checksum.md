# 112. 체크섬 (Checksum)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터 무결성 검증을 위해 모든 바이트/워드를 합산하여 생성하는 고정 길이 값으로, 단순 덧셈 연산만으로 빠른 검증 가능
> 2. **가치**: 구현이 극도로 단순하고 처리 속도가 빨라 TCP/IP, 파일 전송, 펌웨어 검증 등에 널리 사용
> 3. **융합**: 네트워크 프로토콜(TCP/UDP/IP), 파일 시스템, BIOS/UEFI 펌웨어의 기본 무결성 검증 메커니즘

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

체크섬(Checksum)은 데이터의 무결성을 검증하기 위해 데이터를 구성하는 모든 요소를 합산하여 생성하는 작은 고정 길이 값이다. 송신 측에서 계산하여 함께 전송하고, 수신 측에서 재계산하여 비교함으로써 데이터 변조나 오류를 검출한다.

### 💡 비유: 영수증의 합계 금액

체크섬은 **영수증의 총액**과 같다. 구매한 물건 5개의 가격을 모두 더하면 총액이 나온다. 나중에 물건 하나의 가격이 바뀌면 총액도 달라져서 오류를 알 수 있다. 단, 두 물건의 가격이 반대로 바뀌면 총액은 같아서 오류를 모르는 것이 한계다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 구성 요소

| 요소 | 설명 |
|:---|:---|
| 데이터 | 체크섬을 계산할 원본 데이터 |
| 합계기 | 모든 바이트/워드를 누적 합산 |
| 오버플로우 처리 | 캐리 비트를 다시 더함(1의 보수) 또는 무시 |
| 체크섬 값 | 최종 계산된 고정 길이 값 |

### 체크섬 종류

```
┌─────────────────────────────────────────────────────────────────────┐
│                    체크섬 종류별 특성                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  1. 단순 합계 체크섬 (Simple Sum)                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  CS = (D1 + D2 + ... + Dn) mod 2^m                           │ │
│  │  • m비트로 잘라서 사용                                        │ │
│  │  • 가장 단순하지만 오류 검출력 낮음                           │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  2. 1의 보수 체크섬 (One's Complement)                             │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  CS = ~(D1 + D2 + ... + Dn + 캐리)                           │ │
│  │  • 오버플로우 캐리를 다시 더함 (end-around carry)            │ │
│  │  • TCP/IP 헤더 체크섬에서 사용                               │ │
│  │  • 결과를 0으로 만들어 검증 단순화                           │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  3. 2의 보수 체크섬 (Two's Complement)                             │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  CS = -(D1 + D2 + ... + Dn)                                  │ │
│  │  • 모든 데이터 + 체크섬 = 0                                   │ │
│  │  • UDP 체크섬에서 사용                                        │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  4. Fletcher 체크섬                                                │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  s1 = (s1 + D[i]) mod 255                                    │ │
│  │  s2 = (s2 + s1) mod 255                                      │ │
│  │  CS = (s2 << 8) | s1                                         │ │
│  │  • 위치 정보 포함으로 단순 합계보다 강력                      │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  5. LRC (Longitudinal Redundancy Check)                           │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  CS = XOR(D1, D2, ..., Dn)                                   │ │
│  │  • 모든 바이트의 XOR                                          │ │
│  │  • 패리티 확장 형태                                           │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

### TCP/IP 체크섬 계산

```c
#include <stdint.h>

// IP/TCP/UDP 체크섬 계산 (1의 보수)
uint16_t ip_checksum(uint16_t *data, int len) {
    uint32_t sum = 0;

    // 16비트 워드 단위로 합산
    while (len > 1) {
        sum += *data++;
        len -= 2;
    }

    // 홀수 바이트 처리
    if (len == 1) {
        sum += *(uint8_t *)data;
    }

    // 캐리를 다시 더함 (end-around carry)
    while (sum >> 16) {
        sum = (sum & 0xFFFF) + (sum >> 16);
    }

    // 1의 보수 반환
    return (uint16_t)~sum;
}

// 체크섬 검증 (데이터 + 체크섬 = 0xFFFF)
int verify_checksum(uint16_t *data, int len) {
    uint16_t result = ip_checksum(data, len);
    return (result == 0);  // 올바르면 0
}
```

## Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

### 체크섬 vs CRC vs 해시

| 특성 | 체크섬 | CRC | 해시(MD5/SHA) |
|:---|:---:|:---:|:---:|
| **연산** | 덧셈 | 다항식 나눗셈 | 복잡한 변환 |
| **길이** | 8~32비트 | 16~64비트 | 128~512비트 |
| **속도** | 매우 빠름 | 빠름 | 느림 |
| **충돌 저항** | 낮음 | 중간 | 높음 |
| **용도** | 통신, 펌웨어 | 네트워크, 저장 | 보안, 서명 |

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

### 실무 시나리오

**시나리오 1: 임베디드 펌웨어 검증**
- 상황: OTA(Over-The-Air) 펌웨어 업데이트
- 선택: 32비트 체크섬 + CRC
- 이유: 단순하고 빠른 검증, 낮은 CPU 부하

**시나리오 2: 네트워크 패킷 검증**
- 상황: 고속 라우터 패킷 처리
- 선택: IP 체크섬 오프로드 (NIC)
- 이유: 하드웨어 가속으로 지연 최소화

**시나리오 3: 파일 다운로드 검증**
- 상황: 대용량 파일 무결성 확인
- 선택: SHA-256 (체크섬보다 강력)
- 이유: 악의적 변조 방지

### 👶 어린이를 위한 3줄 비유 설명

**개념**: 체크섬은 책의 페이지 번호를 모두 더한 것과 같아요. 누가 몰래 페이지를 찢어가면 총합이 달라져서 바로 알 수 있어요!

**원리**: 데이터의 모든 바이트를 더해서 숫자 하나를 만들어요. 데이터가 조금이라도 바뀌면 합계도 달라지니까 오류를 찾을 수 있어요.

**효과**: 인터넷으로 파일을 받을 때 깨진 부분이 없는지 아주 빠르게 확인할 수 있어요. 하지만 나쁜 사람이 일부러 바꾼 건 못 찾아요!
