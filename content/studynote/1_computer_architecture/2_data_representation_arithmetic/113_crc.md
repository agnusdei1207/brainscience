# 113. CRC (Cyclic Redundancy Check)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 다항식 나눗셈 기반의 오류 검출 코드로, 데이터 뒤에 나머지(체크값)를 붙여 전송 후 무결성 검증
> 2. **가치**: 긴 데이터 스트림에서 다중 비트 오류를 높은 확률(99.99%+)로 검출하며, 하드웨어 구현이 시프트 레지스터로 매우 효율적
> 3. **융합**: 이더넷 프레임(FCS), ZIP/ZIP 파일, HDLC/PPP 프로토콜, SSD 컨트롤러의 핵심 오류 검출 메커니즘

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

CRC(Cyclic Redundancy Check, 순환 중복 검사)는 데이터를 다항식으로 취급하여 미리 정해진 생성 다항식으로 나눈 나머지를 체크값으로 사용하는 오류 검출 방식이다. 수신 측에서 동일한 나눗셈을 수행하여 나머지가 0이면 오류 없음, 0이 아니면 오류 발생으로 판단한다.

### 💡 비유: ISBN의 검증 숫자

CRC는 **ISBN의 마지막 검증 숫자**와 비슷하다. 책의 ISBN-13에서 앞 12자리를 특정 공식으로 계산하면 마지막 숫자가 나온다. 숫자 하나가 틀리면 계산 결과가 달라져 오류를 알 수 있다. CRC는 이 원리를 비트 수준에서 다항식으로 확장한 것이다.

### CRC-32 소프트웨어 구현

```c
#include <stdint.h>

// CRC-32 테이블 (미리 계산)
static uint32_t crc32_table[256];

// 테이블 초기화 (한 번만 수행)
void crc32_init(void) {
    uint32_t poly = 0xEDB88320;  // 반전된 다항식

    for (uint32_t i = 0; i < 256; i++) {
        uint32_t crc = i;
        for (int j = 0; j < 8; j++) {
            if (crc & 1)
                crc = (crc >> 1) ^ poly;
            else
                crc >>= 1;
        }
        crc32_table[i] = crc;
    }
}

// CRC-32 계산
uint32_t crc32(const uint8_t *data, int len) {
    uint32_t crc = 0xFFFFFFFF;  // 초기값

    for (int i = 0; i < len; i++) {
        uint8_t index = (crc ^ data[i]) & 0xFF;
        crc = (crc >> 8) ^ crc32_table[index];
    }

    return crc ^ 0xFFFFFFFF;  // 최종 XOR
}

// 테이블 없이 직접 계산 (느리지만 코드 작음)
uint32_t crc32_bitwise(const uint8_t *data, int len) {
    uint32_t crc = 0xFFFFFFFF;

    for (int i = 0; i < len; i++) {
        crc ^= data[i];
        for (int j = 0; j < 8; j++) {
            if (crc & 1)
                crc = (crc >> 1) ^ 0xEDB88320;
            else
                crc >>= 1;
        }
    }

    return crc ^ 0xFFFFFFFF;
}
```

### 👶 어린이를 위한 3줄 비유 설명

**개념**: CRC는 나눗셈을 이용해 비밀 번호를 만드는 거예요. 데이터를 특정 숫자로 나눈 나머지가 CRC예요!

**원리**: 데이터를 아주 긴 숫자로 보고, 정해진 숫자로 나누면 나머지가 남아요. 이 나머지를 데이터 뒤에 붙여서 보내면, 받는 쪽에서 다시 나눴을 때 나머지가 0이어야 정상이에요.

**효과**: 아주 긴 데이터에서도 여러 비트가 틀려도 거의 100%로 오류를 찾아낼 수 있어요. 인터넷으로 사진을 받을 때 깨진 부분이 있는지 확인하는 데 쓰여요!
