# 116. 바이트 오더링 (Byte Ordering)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 멀티바이트 데이터를 메모리에 저장할 때 각 바이트의 배치 순서를 결정하는 규칙으로, 빅 엔디안과 리틀 엔디안 두 가지 방식이 존재
> 2. **가치**: 이기종 시스템 간 통신·데이터 교환 시 필수적으로 이해해야 하는 개념으로, 프로토콜 설계의 기본 전제
> 3. **융합**: 네트워크 프로토콜(TCP/IP), 파일 포맷(ELF, TIFF), 데이터 직렬화(Protobuf)의 핵심 설계 요소

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

바이트 오더링(Byte Ordering) 또는 엔디안(Endianness)은 2바이트 이상의 데이터를 메모리에 저장할 때, 어떤 바이트를 어떤 주소에 배치할지 결정하는 규칙이다. 빅 엔디안(MSB 우선)과 리틀 엔디안(LSB 우선)이 대표적이며, 미들 엔디안(PDP-11) 같은 변형도 존재한다.

### 💡 비유: 여행 가방에 옷 넣기

바이트 오더링은 **여행 가방에 옷을 정리하는 순서**와 같다. 어떤 사람은 자주 쓰는 것(양말)부터 넣고(리틀 엔디안), 어떤 사람은 큰 옷(재킷)부터 넣는다(빅 엔디안). 두 사람이 가방을 공유하면 무슨 순서로 넣었는지 미리 약속하지 않으면 혼란이 온다.

### 등장 배경

| 구분 | 내용 |
|:---|:---|
| **기원** | 《걸리버 여행기》 달걀 깨기 논쟁 (Big-Endians vs Little-Endians) |
| **표준화** | 1980년대 TCP/IP가 네트워크 바이트 순서(빅 엔디안) 채택 |
| **현재** | 대부분 CPU는 리틀 엔디안, 네트워크는 빅 엔디안 |

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 엔디안 비교

| 특성 | 빅 엔디안 | 리틀 엔디안 |
|:---|:---|:---|
| **MSB 위치** | 낮은 주소 | 높은 주소 |
| **LSB 위치** | 높은 주소 | 낮은 주소 |
| **사람 가독성** | 높음 | 낮음 |
| **형변환** | 시프트 필요 | 주소 재해석 |
| **산술 연산** | 역순 접근 | 순차 접근 |
| **네트워크** | 변환 불필요 | 변환 필요 |

### 엔디안 변환 함수

```c
#include <stdint.h>

// 16비트 엔디안 변환
uint16_t swap16(uint16_t value) {
    return (value << 8) | (value >> 8);
}

// 32비트 엔디안 변환
uint32_t swap32(uint32_t value) {
    return ((value & 0x000000FF) << 24) |
           ((value & 0x0000FF00) << 8)  |
           ((value & 0x00FF0000) >> 8)  |
           ((value & 0xFF000000) >> 24);
}

// 64비트 엔디안 변환
uint64_t swap64(uint64_t value) {
    return ((value & 0x00000000000000FFULL) << 56) |
           ((value & 0x000000000000FF00ULL) << 40) |
           ((value & 0x0000000000FF0000ULL) << 24) |
           ((value & 0x00000000FF000000ULL) << 8)  |
           ((value & 0x000000FF00000000ULL) >> 8)  |
           ((value & 0x0000FF0000000000ULL) >> 24) |
           ((value & 0x00FF000000000000ULL) >> 40) |
           ((value & 0xFF00000000000000ULL) >> 56);
}

// 호스트 → 네트워크 (빅 엔디안)
#define htonl(x) swap32(x)
#define htons(x) swap16(x)

// 네트워크 → 호스트
#define ntohl(x) swap32(x)
#define ntohs(x) swap16(x)
```

### 파일 포맷 엔디안 식별

```
┌─────────────────────────────────────────────────────────────────────┐
│                파일 포맷 엔디안 표시                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  ELF 실행 파일                                                      │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  매직: 0x7F 'E' 'L' 'F'                                       │ │
│  │  EI_CLASS: 1=32비트, 2=64비트                                 │ │
│  │  EI_DATA:  1=리틀 엔디안, 2=빅 엔디안                         │ │
│  │                                                               │ │
│  │  → 실행 전 EI_DATA 확인하여 적절히 해석                      │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  TIFF 이미지                                                        │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  "II" (0x4949) → 리틀 엔디안 (Intel)                         │ │
│  │  "MM" (0x4D4D) → 빅 엔디안 (Motorola)                        │ │
│  │                                                               │ │
│  │  → 파일 시작 2바이트로 엔디안 식별                            │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  UTF-16 BOM                                                         │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  FF FE → 리틀 엔디안                                          │ │
│  │  FE FF → 빅 엔디안                                            │ │
│  │                                                               │ │
│  │  → 텍스트 파일의 엔디안 식별                                  │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  PNG 이미지                                                         │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  모든 멀티바이트 필드가 빅 엔디안                             │ │
│  │  청크 길이, CRC, 너비, 높이 등                                │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

### 미들 엔디안 (PDP-11)

```
┌─────────────────────────────────────────────────────────────────────┐
│                미들 엔디안 (PDP-11)                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  32비트 값 0x12345678의 PDP-11 저장 방식                           │
│  ─────────────────────────────────────                             │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  주소:      0x1000  0x1001  0x1002  0x1003                   │ │
│  │             ┌──────┬──────┬──────┬──────┐                    │ │
│  │ PDP-11:     │ 0x34 │ 0x12 │ 0x78 │ 0x56 │                    │ │
│  │             └──────┴──────┴──────┴──────┘                    │ │
│  │             └16비트 워드↘  └16비트 워드↘                     │ │
│  │             (빅 엔디안)     (빅 엔디안)                       │ │
│  │             하지만 워드 순서는 리틀 엔디안                    │ │
│  │                                                               │ │
│  │  16비트 워드는 빅 엔디안으로 저장                             │ │
│  │  하지만 32비트에서 워드 순서는 리틀 엔디안                    │ │
│  │  = 34127856 (혼합)                                            │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  ⚠️ 현대 시스템에서는 거의 사용되지 않음                          │
│  ⚠️ 레거시 데이터 변환 시 고려 필요                               │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

### 실무 시나리오

**시나리오 1: 이기종 시스템 통신**
- 상황: x86 서버 ↔ PowerPC 임베디드
- 해결: 네트워크 바이트 순서(빅 엔디안)로 통일
- 구현: 양쪽 모두 htonl/ntohl 사용

**시나리오 2: 바이너리 파일 포맷 처리**
- 상황: 리틀 엔디안 시스템에서 TIFF(바이엔디안) 읽기
- 해결: 파일 헤더("II"/"MM")로 엔디안 식별 후 처리
- 장점: 호환성 보장

**시나리오 3: 데이터베이스 복제**
- 상황: 리틀 엔디안 서버 → 빅 엔디안 서버 복제
- 해결: 바이너리 로그에 엔디안 명시
- 주의: FLOAT/DOUBLE은 특히 주의

### 설계 가이드라인

| 상황 | 권장 사항 |
|:---|:---|
| **네트워크 프로토콜** | 빅 엔디안 (표준) |
| **내부 저장** | 호스트 엔디안 |
| **파일 포맷** | 명시적 엔디안 표시 |
| **직렬화** | 고정 엔디안 + 문서화 |
