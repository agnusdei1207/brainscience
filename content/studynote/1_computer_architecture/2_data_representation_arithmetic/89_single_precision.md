# 단정밀도 (Single Precision, FP32)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 단정밀도(Single Precision, FP32)는 IEEE 754 표준의 32비트 부동소수점 포맷으로, 1비트 부호·8비트 지수·23비트 가수로 구성되어 약 7자리 십진수 정밀도와 10³⁸ 범위를 제공한다.
> 2. **가치**: 메모리 효율성과 연산 속도의 최적 균형점으로, 3D 그래픽스(PC 게임, WebGL), 머신러닝 추론, 오디오 처리 등 대다수 실시간 애플리케이션의 표준 데이터 타입이다.
> 3. **융합**: GPU(SIMD), CPU(AVX/SSE), TPU/NPU 모두 FP32 연산을 고속으로 지원하며, FP16·BF16으로의 혼합 정밀도 전환 시에도 FP32는 "Master Weight"로 정확성을 보장하는 기준점 역할을 수행한다.

---

## Ⅰ. 개요 (Context & Background)

- **개념**: 단정밀도(Single Precision)는 32비트(4바이트)를 사용하여 실수를 표현하는 IEEE 754 표준 포맷이다. C/C++의 `float`, Java/C#의 `float`, Python의 `numpy.float32`가 모두 이에 해당한다. 내부 구조는 부호(S) 1비트, 지수(E) 8비트, 가수(M) 23비트로 구성되며, 정규화 수에서는 암시적 선행 1로 인해 실제 정밀도는 24비트(약 7.2 십진 자릿수)다.

- **💡 비유**: 단정밀도는 **'일반용 카메라'**와 같다. 전문가용(배정밀도)만큼 해상도가 높지는 않지만, 일상적인 촬영(3D 게임, 오디오, 머신러닝 추론)에는 충분하고, 저장 공간(메모리)과 처리 속도(연산량) 면에서 훨씬 효율적이다. 4K 비디오(배정밀도)가 필요한 전문 촬영이 아니라면, 일반용 카메라로도 충분하다.

- **등장 배경 및 발전 과정**:
  1. **초기 FPU의 등장 (1980년대)**: Intel 8087 코프로세서가 x86에 부동소수점 연산을 추가했을 때, 32비트 포맷은 메모리가 비싼 시대에 '적당한' 정밀도와 범위를 제공하는 실용적 선택이었다.
  2. **3D 그래픽스 혁명 (1990년대)**: OpenGL, DirectX가 FP32를 표준으로 채택하면서, GPU는 수백 개의 FP32 코어를 병렬로 배치하는 방향으로 진화했다. 픽셀 좌표, 색상(RGBA), 깊이(Z-buffer), 법선 벡터(Normal) 모두 FP32로 처리된다.
  3. **머신러닝 시대 (2010년대~현재)**: 초기 딥러닝은 FP32가 유일한 훈련 포맷이었다. 이후 FP16/BF16 혼합 정밀도가 등장했지만, FP32는 여전히 최종 가중치 저장(Master Weights)과 추론 단계의 기본 포맷으로 남아 있다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 비트 레이아웃과 값 계산

FP32의 32비트는 3개의 필드로 나뉘며, 값은 수식 $(-1)^S \times (1 + M) \times 2^{E-127}$으로 계산된다(정규화 수 기준).

```text
  ┌─────────────────────────────────────────────────────────────────────────┐
  │              FP32 (단정밀도) 비트 레이아웃와 값 해석                       │
  ├─────────────────────────────────────────────────────────────────────────┤
  │                                                                         │
  │   31      30           23   22                                 0       │
  │   ┌────┬────────────────┬────────────────────────────────────────────┐ │
  │   │ S  │   Exponent (E) │              Mantissa (M)                  │ │
  │   └────┴────────────────┴────────────────────────────────────────────┘ │
  │    1비트      8비트                         23비트                      │
  │                                                                         │
  │   ─────────────────────────────────────────────────────────────────    │
  │   값 계산 (정규화 수: 0 < E < 255):                                       │
  │                                                                         │
  │   Value = (-1)^S × (1.M) × 2^(E-127)                                    │
  │                  ↑                                                       │
  │            암시적 선행 1 (Implicit Leading 1)                            │
  │            23비트 M 앞에 1이 붙어 24비트 유효숫자가 됨                     │
  │                                                                         │
  │   ─────────────────────────────────────────────────────────────────    │
  │   FP32로 10진수 12.375를 표현하는 과정:                                   │
  │                                                                         │
  │   ① 10진수 → 2진수:  12.375 = 1100.011₂                                 │
  │   ② 정규화:  1100.011₂ = 1.100011₂ × 2³                                 │
  │   ③ 각 필드 결정:                                                       │
  │      S = 0 (양수)                                                       │
  │      E = 3 + 127 = 130 = 10000010₂                                      │
  │      M = 10001100000000000000000₂ (소수점 이하만 저장)                    │
  │   ④ 최종 32비트:                                                        │
  │      0 10000010 10001100000000000000000                                 │
  │      └─┬──┘ └──────┬────────────────────┘                               │
  │        S        E=130             M                                      │
  │                                                                         │
  │   ─────────────────────────────────────────────────────────────────    │
  │   FP32 주요 경계값:                                                      │
  │   ┌─────────────────────────────────────────────────────────────────┐  │
  │   │ 최소 양의 정규화 수: E=1, M=0  → 2^-126 ≈ 1.18 × 10^-38         │  │
  │   │ 최대 정규화 수:     E=254, M=全1 → ≈ 3.40 × 10^38              │  │
  │   │ 최소 서브노멀:     E=0, M=1   → 2^-149 ≈ 1.40 × 10^-45         │  │
  │   │ 정밀도(ulp):       2^-23 ≈ 1.19 × 10^-7 (약 7자리)              │  │
  │   └─────────────────────────────────────────────────────────────────┘  │
  │                                                                         │
  └─────────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** FP32의 핵심은 암시적 선행 1(Implicit Leading 1)이다. 정규화 과정에서 모든 이진수는 `1.xxxx` 형태가 되므로, 굳이 1을 저장할 필요 없이 소수점 이하 23비트만 저장하면 실제로는 24비트 정밀도를 갖게 된다. 이 "1비트 공짜"가 FP32의 효율성 비결이다. 편향값 127은 지수를 -126~+127 범위로 옮겨 음수 지수도 부호 없이 표현 가능하게 한다. E=0과 E=255는 특수값(서브노멀, ±∞, NaN) 예약용이다.

### 정밀도와 ULP(Unit in the Last Place)

FP32의 유효숫자는 24비트(약 7.2 십진 자릿수)이며, 인접한 두 FP32 값 사이의 간격을 ULP라 한다. ULP는 수의 크기에 비례하여 증가한다.

```text
  ┌──────────────────────────────────────────────────────────────────────────┐
  │           FP32 ULP (Unit in the Last Place) 시각화                        │
  ├──────────────────────────────────────────────────────────────────────────┤
  │                                                                          │
  │   ULP = 2^(E-127) × 2^-23 = 2^(E-150)                                   │
  │   → 지수 E에 따라 간격이 2배씩 늘어남                                     │
  │                                                                          │
  │   실수선 (0 주변):                                                       │
  │                                                                          │
  │   0   ...  2^-126      2^-125      2^-124      ...                       │
  │   │        │     │      │    │      │    │                             │
  │   └─ 서브노멀 ─┘    └ULP가 점점 넓어짐┘                                  │
  │      (간격 일정)                                                         │
  │                                                                          │
  │   실수선 (1 주변):                                                       │
  │                                                                          │
  │   0.9    1.0         1.0000001    1.0000002    1.0000003               │
  │   │      │      │      │       │      │       │                        │
  │          └─ ULP ≈ 1.19×10^-7 ─┘                                          │
  │                                                                          │
  │   실수선 (1,000,000 주변):                                               │
  │                                                                          │
  │   999,999  1,000,000    1,000,000.1  1,000,000.2                       │
  │   │         │       │       │        │                                  │
  │             └── ULP ≈ 0.125 ──┘                                           │
  │             (1.19×10^-7 × 2^20 ≈ 0.125)                                  │
  │                                                                          │
  │   ⚠ 의미: 1,000,000 근처에서는 0.1 단위까지 구분 가능                     │
  │           0.01 단위는 반올림 오차로 손실될 수 있음                        │
  │                                                                          │
  └──────────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** ULP가 수의 크기에 비례한다는 것은 FP32의 상대 오차는 일정(약 2⁻²³ ≈ 1.2×10⁻⁷)하지만 절대 오차는 수가 커질수록 증가함을 의미한다. 1.0 근처에서는 약 10⁻⁷ 단위까지 구분되지만, 1,000,000 근처에서는 0.1 단위까지만 구분된다. 이것이 "7자리 정밀도"의 의미다. 8자리 이상의 십진수를 FP32에 저장하면 하위 자릿수가 잘린다.

### FP32 연산의 하드웨어 구현

현대 CPU와 GPU는 FP32 연산을 고속으로 수행하기 위해 파이프라인된 FPU(Floating-Point Unit)를 탑재한다.

| 연산 종류 | 지연 시간(Latency) | 처리량(Throughput) | 비고 |
|:---|:---:|:---:|:---|
| **FP32 덧셈/뺄셈** | 3~5 사이클 | 1~2/cycle | FADD 파이프라인 |
| **FP32 곱셈** | 4~5 사이클 | 1~2/cycle | FMUL 파이프라인 |
| **FP32 FMA (Fused Multiply-Add)** | 4~6 사이클 | 1~2/cycle | a×b+c를 1연산으로 |
| **FP32 나눗셈** | 10~30 사이클 | 1/10~30 cycle | SRT 또는 Newton-Raphson |
| **FP32 제곱근** | 10~30 사이클 | 1/10~30 cycle | Newton-Raphson 기반 |

```text
  ┌──────────────────────────────────────────────────────────────────────────┐
  │           FMA (Fused Multiply-Add) 연산의 정확도 이점                      │
  ├──────────────────────────────────────────────────────────────────────────┤
  │                                                                          │
  │   일반 연산: result = a × b + c                                          │
  │                                                                          │
  │   ┌─────────────────────────────────────────────────────────────────┐   │
  │   │  [FP32 Mul]              [FP32 Add]                             │   │
  │   │  a × b ──────▶ temp ──────▶ temp + c ──────▶ result             │   │
  │   │               (반올림)      (반올림)                               │   │
  │   │                   ↑              ↑                                 │   │
  │   │             1차 오차       2차 오차                                 │   │
  │   └─────────────────────────────────────────────────────────────────┘   │
  │                                                                          │
  │   FMA 연산: result = fma(a, b, c)                                        │
  │                                                                          │
  │   ┌─────────────────────────────────────────────────────────────────┐   │
  │   │  [FMA Unit]                                                      │   │
  │   │  a × b + c ─────────────────────────▶ result                    │   │
  │   │              (단 1회 반올림)            ↑                         │   │
  │   │                                        감소한 오차                │   │
  │   └─────────────────────────────────────────────────────────────────┘   │
  │                                                                          │
  │   ⚠ FMA는 a×b의 중간 결과를 FP32보다 높은 정밀도(약 48비트)로 유지       │
  │      한 후 c를 더하고 마지막에만 반올림하여 정확도 향상                   │
  │                                                                          │
  │   GPU(SIMD)에서는 Tensor Core 또는 CUDA Core가 FMA를 1사이클에 수행      │
  │   예: NVIDIA A100 — 312 TFLOPS (FP32 FMA 기준)                          │
  │                                                                          │
  └──────────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** FMA(Fused Multiply-Add)는 딥러닝과 과학 계산에서 핵심 연산이다. 일반적인 곱셈 후 덧셈은 두 번의 반올림이 발생하지만, FMA는 중간 결과(52비트 이상)를 유지한 채 덧셈까지 완료하고 한 번만 반올림한다. 이것이 행렬 곱(GEMM)의 정확도를 크게 향상시킨다. x86의 AVX2/AVX-512, ARM의 NEON, NVIDIA의 CUDA 모두 FMA 명령어(`vfmadd` 등)를 제공하며, 현대 컴파일러는 `a*b+c` 패턴을 자동으로 FMA로 변환한다.

## Ⅲ. 융합 비교 및 다각도 분석

### 비교 1: FP32 vs FP64 (단정밀도 vs 배정밀도)

| 비교 항목 | FP32 (단정밀도) | FP64 (배정밀도) |
|:---|:---|:---|
| **비트 수** | 32 (4바이트) | 64 (8바이트) |
| **유효숫자** | 24비트 (~7자리) | 53비트 (~16자리) |
| **지수 범위** | 10⁻³⁸ ~ 10³⁸ | 10⁻³⁰⁸ ~ 10³⁰⁸ |
| **메모리 대역폭** | 기준 (1×) | 2배 소모 |
| **GPU 성능** | 기준 (1×) | 1/2 ~ 1/32 (하드웨어에 따라) |
| **주 용도** | 게임, ML, 오디오 | 과학 계산, 금융, 천문학 |

### 비교 2: FP32 vs FP16 vs BF16

| 비교 항목 | FP32 | FP16 (IEEE) | BF16 (Brain Float) |
|:---|:---|:---|:---|
| **총 비트** | 32 | 16 | 16 |
| **지수 비트** | 8 | 5 | 8 |
| **가수 비트** | 23 | 10 | 7 |
| **표현 범위** | 10³⁸ | 6.5×10⁴ (좁음!) | 10³⁸ (FP32와 동일) |
| **정밀도** | ~7자리 | ~3자리 | ~2자리 |
| **ML 훈련** | 안정적 | 오버플로우 위험 | 안정적 (Loss Scaling 불필요) |
| **하드웨어** | 모든 FPU | GPU (Tensor Core) | TPU, GPU (Ampere+) |

### 과목 융합 관점

- **컴퓨터구조(컴구)**: GPU의 FP32 성능은 FLOPS로 측정되며, RTX 4090은 82.6 TFLOPS, A100은 19.5 TFLOPS(기본) 또는 312 TFLOPS(Tensor Core 포함)다. FP32 FMA 1회 = 2 FLOP로 계산한다.

- **운영체제(OS)**: Linux 커널은 FPU 레지스터(x86의 XMM/YMM/ZMM)를 태스크 전환 시 저장/복원한다. FP32 벡터 레지스터 16개(XMM) × 32비트 × 4 = 256바이트가 컨텍스트 스위칭 오버헤드에 추가된다.

- **데이터베이스(DB)**: `REAL` 또는 `FLOAT` 타입은 보통 FP32로 구현된다. 인덱스 생성 시 FP32의 오차로 인해 정렬 순서가 예상과 다를 수 있다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오

1. **시나리오 — 실시간 3D 렌더링에서의 정밀도 부족**: 대규모 오픈 월드 게임에서 카메라가 원점(0,0,0)에서 멀어지면(예: 좌표 10,000,000 이상) 지터링(Jittering)이 발생하여 물체가 흔들려 보이는 현상. FP32의 7자리 정밀도로는 10,000,000 근처에서 ULP가 약 1이 되어, 서브픽셀 이동이 정확히 표현되지 않는다. 기술사는 "카메라 상대 좌표계(Camera-Relative Rendering)"를 사용하여 렌더링 시 모든 좌표를 카메라 위치 기준으로 재계산하거나, FP64를 사용하는 "원점 재설정(Origin Rebasing)" 기법을 적용해야 한다.

2. **시나리오 — 딥러닝 추론에서 FP32 vs FP16 선택**: 엣지 디바이스에서 실시간 객체 탐지(YOLO)를 수행해야 하는 상황. FP32 모델은 95MB, FP16 모델은 48MB지만, 정확도 저하가 우려된다. 기술사는 먼저 FP16으로 양자화하고 mAP(mean Average Precision) 저하가 1% 이내인지 검증한 후, 메모리와 대역폭 이점을 위해 FP16을 선택한다. FP32는 양자화 오차 분석용 "Golden Model"로 유지한다.

3. **시나리오 — 금융 리스크 계산의 FP32 부적합성**: VaR(Value at Risk) 계산을 FP32로 수행했더니, 포트폴리오 규모가 커질수록(10억 달러 이상) 오차가 누적되어 규제 보고서의 숫자가 감사 시 불일치하는 문제. 기술사는 즉시 FP64 또는 Decimal128로 전환하고, 모든 중간 계산 결과를 스냅샷으로 저장하여 재현성을 확보해야 한다. 금융 계산에서 FP32는 규제 위반이 될 수 있다.

### 도입 체크리스트
- **기술적**: 시뮬레이션의 조건 수(Condition Number)가 10⁶ 이하인가? 그 이상이면 FP32의 오차가 결과에 치명적일 수 있다.
- **운영·보안적**: 외부 입력 데이터에 NaN/±∞가 포함될 수 있는가? 이를 감지하고 거부하는 검증 로직이 있는가?

### 안티패턴
- **화폐 계산에 FP32 사용**: 0.1, 0.01, 0.001 등은 FP32에서 정확히 표현되지 않아(순환 소수), 1,000,000번의 0.1 더하기가 100,000이 되지 않을 수 있다.
- **FP32로 천문학적 거리 계산**: 우주 시뮬레이션에서 10²⁰ (광년 단위) 범위를 FP32로 표현하면, 행성 간 거리(10⁹ 수준)가 ULP보다 작아져 구분이 불가능하다.

## Ⅴ. 기대효과 및 결론

### 정량/정성 기대효과

| 구분 | FP64 사용 시 | FP32 사용 시 | 개선 효과 |
|:---|:---|:---|:---|
| **정량** | 메모리 2배, 대역폭 2배 | 메모리 1/2, 대역폭 1/2 | 비용 50% 절감 |
| **정량** | GPU 성능 1/2~1/32 | GPU 성능 1× | 처리량 2~32배 향상 |
| **정성** | 과도한 정밀도 | 적정 정밀도 | 전력 효율 향상 |

### 미래 전망
- **혼합 정밀도(Mixed Precision)의 표준화**: 2025년 현재, FP32는 딥러닝 훈련에서 "Master Weight" 저장용으로만 사용되고, 실제 연산은 FP16/BF16에서 수행되는 것이 표준이 되고 있다. NVIDIA Ampere 이상의 GPU는 BF16을 네이티브로 지원한다.
- **FP8의 부상**: 2024년 NVIDIA Blackwell 아키텍처와 Intel Gaudi3가 FP8(E4M3, E5M2)을 지원하기 시작했다. 추론 단계에서는 FP8로 충분하며, 메모리 대역폭이 1/4로 줄어든다.

- **ISO/IEC 60559**: IEEE 754의 ISO 표준 버전

## 👶 어린이를 위한 3줄 비유 설명
1. 단정밀도(FP32)는 **'일반 가방'**이에요. 너무 크지도 않고 너무 작지도 않아서, 학교 갈 때 딱 좋아요. 책 7권 정도(7자리 숫자)는 편하게 담을 수 있어요.
2. 전문가용 가방(배정밀도)은 책 16권을 담을 수 있지만, 무겁고 비싸요. 우리가 게임이나 유튜브를 볼 때는 일반 가방으로도 충분하답니다!
3. 컴퓨터도 마찬가지예요. 대부분의 게임과 앱은 일반 가방(FP32)으로 충분하고, 과학자들이나 아주 정확한 계산이 필요할 때만 전문가용 가장을 써요.
