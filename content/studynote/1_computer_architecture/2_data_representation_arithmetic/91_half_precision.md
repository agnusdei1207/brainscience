# 반정밀도 (Half Precision, FP16)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 반정밀도(Half Precision, FP16)는 IEEE 754-2008에서 추가된 16비트 부동소수점 포맷으로, 1비트 부호·5비트 지수·10비트 가수로 구성되어 약 3~4자리 십진수 정밀도와 ±65,504 범위를 제공한다.
> 2. **가치**: FP32 대비 메모리 50% 절감, 대역폭 50% 절감, GPU Tensor Core에서의 2~8배 연산 가속을 통해 딥러닝 훈련·추론의 효율성을 획기적으로 높인다.
> 3. **융합**: FP16의 좁은 표현 범위(최대 65,504)는 그래디언트 오버플로우 위험을 야기하므로, Loss Scaling·FP32 Master Weights·Mixed Precision Training과 결합하여 안정성을 확보해야 한다.

---

## Ⅰ. 개요 (Context & Background)

- **개념**: 반정밀도(Half Precision, FP16)는 16비트(2바이트)를 사용하여 실수를 표현하는 IEEE 754-2008 표준 포맷이다. NVIDIA CUDA의 `half`, PyTorch의 `torch.float16`, TensorFlow의 `tf.float16`이 이에 해당한다. 내부 구조는 부호(S) 1비트, 지수(E) 5비트, 가수(M) 10비트로 구성되며, 암시적 선행 1을 포함하면 실제 정밀도는 11비트(약 3.3 십진 자릿수)다.

- **💡 비유**: 반정밀도는 **'여행용 미니 카메라'**와 같다. 일반 카메라(FP32)만큼 선명하지는 않지만, 가볍고 작아서 여행 가방(메모리)에 많이 담을 수 있고, 빠르게 촬영(연산)할 수 있다. 하지만 어두운 밤(작은 그래디언트)이나 아주 밝은 태양(큰 그래디언트) 아래서는 사진이 제대로 안 나올 수 있다.

- **등장 배경 및 발전 과정**:
  1. **IEEE 754-2008의 추가**: IEEE 754-1985에는 32비트(단정밀도)와 64비트(배정밀도)만 있었다. 2008년 개정에서 16비트 반정밀도(binary16)와 128비트 4배정밀도(binary128)가 추가되었다. 당시에는 주로 그래픽스와 오디오 처리를 위한 것이었다.
  2. **딥러닝의 폭발과 메모리 병목 (2015~)**: AlexNet(2012) 이후 모델 크기가 기하급수적으로 증가하면서, FP32 훈련의 메모리와 대역폭이 병목이 되었다. NVIDIA Pascal 아키텍처(GTX 1080, 2016)가 FP16 저장과 FP32 연산을 결합한 Mixed Precision을 지원하기 시작했다.
  3. **Tensor Core의 등장 (2017~)**: NVIDIA Volta(V100) 아키텍처가 FP16 입력에 대해 FP32 누적을 수행하는 Tensor Core를 도입했다. 이로써 FP16 행렬 곱(GEMM)이 FP32 대비 8배 빨라졌다. 오늘날 모든 주요 딥러닝 프레임워크(PyTorch, TensorFlow, JAX)는 FP16 Mixed Precision Training을 기본 옵션으로 제공한다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 비트 레이아웃과 값 계산

FP16의 16비트는 3개의 필드로 나뉘며, 값은 수식 $(-1)^S \times (1 + M) \times 2^{E-15}$으로 계산된다(정규화 수 기준).

```text
  ┌─────────────────────────────────────────────────────────────────────────┐
  │              FP16 (반정밀도) 비트 레이아웃와 값 해석                       │
  ├─────────────────────────────────────────────────────────────────────────┤
  │                                                                         │
  │   15    14        10    9                0                              │
  │   ┌───┬────────────┬────────────────────┐                               │
  │   │ S │ Exponent(E)│    Mantissa (M)    │                               │
  │   └───┴────────────┴────────────────────┘                               │
  │    1비트    5비트           10비트                                       │
  │                                                                         │
  │   ─────────────────────────────────────────────────────────────────    │
  │   값 계산 (정규화 수: 0 < E < 31):                                        │
  │                                                                         │
  │   Value = (-1)^S × (1.M) × 2^(E-15)                                     │
  │                   ↑                                                      │
  │             암시적 선행 1 (Implicit Leading 1)                           │
  │                                                                         │
  │   ─────────────────────────────────────────────────────────────────    │
  │   FP16 주요 경계값:                                                      │
  │   ┌─────────────────────────────────────────────────────────────────┐  │
  │   │ 최소 양의 정규화 수: E=1, M=0  → 2^-14 ≈ 6.10 × 10^-5           │  │
  │   │ 최대 정규화 수:     E=30, M=全1 → ≈ 65,504                     │  │
  │   │ 최소 서브노멀:     E=0, M=1   → 2^-24 ≈ 5.96 × 10^-8           │  │
  │   │ 정밀도(ulp @ 1):   2^-10 ≈ 9.77 × 10^-4 (약 3자리)             │  │
  │   │ 정수 정확 표현:    2^11 = 2,048까지만 정확히 표현 가능           │  │
  │   └─────────────────────────────────────────────────────────────────┘  │
  │                                                                         │
  │   ⚠ FP16의 치명적 제약:                                                  │
  │   ┌─────────────────────────────────────────────────────────────────┐  │
  │   │  최댓값이 65,504뿐임 → 딥러닝 그래디언트가 이를 초과하면 ∞ 됨!    │  │
  │   │  최소 정규화 수가 6×10^-5 → 작은 그래디언트가 0이 될 수 있음     │  │
  │   └─────────────────────────────────────────────────────────────────┘  │
  │                                                                         │
  └─────────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** FP16은 5비트 지수로 인해 표현 범위가 매우 좁다(±65,504). 이는 FP32의 ±3.4×10³⁸과 비교하면 천문학적 차이다. 또한 10비트 가수는 십진수 3~4자리 정밀도만 제공한다. 2,048 이상의 정수는 정확히 표현되지 않으며(2049와 2050이 구분되지 않음), 2,048 이하에서도 ULP가 1보다 커질 수 있다. 이러한 제약 때문에 FP16은 "그대로 사용"할 수 없고, Mixed Precision 기법이 필수적이다.

### FP16의 한계와 Mixed Precision Training

FP16을 딥러닝 훈련에 그대로 사용하면, 그래디언트가 65,504를 초과하여 오버플로우(∞)되거나, 6×10⁻⁵ 미만으로 언더플로우(0)되어 학습이 불안정해진다. 이를 해결하기 위한 핵심 기법이 Mixed Precision Training이다.

```text
  ┌──────────────────────────────────────────────────────────────────────────────┐
  │         Mixed Precision Training (혼합 정밀도 훈련)의 핵심 기법                 │
  ├──────────────────────────────────────────────────────────────────────────────┤
  │                                                                              │
  │  ┌─────────────────────────────────────────────────────────────────────────┐│
  │  │  ① FP32 Master Weights (마스터 가중치)                                   ││
  │  │                                                                         ││
  │  │     [FP32 Weights] ──복사──▶ [FP16 Weights] ──순전파/역전파──▶ [FP16 Grads]││
  │  │          ↑                                               │              ││
  │  │          │                    ┌──────────────────────────┘              ││
  │  │          │                    ▼                                         ││
  │  │          └─────── 업데이트 ◀── [FP32 Grads] ◀── 업캐스팅 ──┘             ││
  │  │                                                                         ││
  │  │  ⚠ 이유: FP16으로 직접 업데이트하면 작은 변화가 사라짐                    ││
  │  │     예: W=1.0, dW=2^-11 → FP16에서 1.0 + 0.00049 = 1.00049 ≈ 1.0        ││
  │  │     FP32 Master Weights는 누적된 작은 변화를 보존                       ││
  │  └─────────────────────────────────────────────────────────────────────────┘│
  │                                                                              │
  │  ┌─────────────────────────────────────────────────────────────────────────┐│
  │  │  ② Loss Scaling (손실 스케일링)                                          ││
  │  │                                                                         ││
  │  │     작은 그래디언트: 10^-5 (FP16 범위 밖)                                ││
  │  │         ↓                                                               ││
  │  │     Loss × 1024 (= Scale Factor)                                        ││
  │  │         ↓                                                               ││
  │  │     그래디언트: 10^-5 × 1024 = 0.01024 (FP16 범위 내)                   ││
  │  │         ↓                                                               ││
  │  │     역전파 완료 후 그래디언트 ÷ 1024                                     ││
  │  │                                                                         ││
  │  │  ⚠ Scale Factor 선택: 너무 작으면 언더플로우, 너무 크면 오버플로우        ││
  │  │     Dynamic Loss Scaling: 오버플로우 감지 시 Scale Factor를 반으로 감소  ││
  │  └─────────────────────────────────────────────────────────────────────────┘│
  │                                                                              │
  │  ┌─────────────────────────────────────────────────────────────────────────┐│
  │  │  ③ FP32 Accumulation (FP32 누적)                                        ││
  │  │                                                                         ││
  │  │     FP16 × FP16 행렬 곱의 결과는 FP32로 누적                             ││
  │  │     Tensor Core: FP16 입력 → FP32 중간 결과 → FP16 출력                 ││
  │  │                                                                         ││
  │  │     ┌─────┐   ┌─────┐   ┌─────┐                                        ││
  │  │     │FP16 │ × │FP16 │ = │FP32 │ 누적 (정확도 유지)                      ││
  │  │     │  A  │   │  B  │   │  C  │                                        ││
  │  │     └─────┘   └─────┘   └─────┘                                        ││
  │  │                              ↓                                          ││
  │  │                         [FP16 출력]                                     ││
  │  └─────────────────────────────────────────────────────────────────────────┘│
  │                                                                              │
  └──────────────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** Mixed Precision Training의 3대 핵심 기법은 ① FP32 Master Weights, ② Loss Scaling, ③ FP32 Accumulation이다. FP32 Master Weights는 FP16의 낮은 정밀도로 인한 가중치 업데이트 손실을 방지한다. Loss Scaling은 작은 그래디언트가 FP16의 언더플로우 범위로 떨어지는 것을 방지한다(10⁻⁵ × 1024 = 0.01). FP32 Accumulation은 Tensor Core가 내부적으로 FP32 정밀도로 중간 결과를 유지하여, FP16 입력의 정확도 손실을 최소화한다. 이 3가지 기법이 결합되어야 FP16 훈련이 FP32와 동등한 정확도를 달성할 수 있다.

### Tensor Core에서의 FP16 연산 가속

NVIDIA Tensor Core는 FP16 행렬 곱셈을 FP32 누적과 함께 수행하여, FP32 연산 대비 최대 8배의 처리량을 제공한다.

```text
  ┌──────────────────────────────────────────────────────────────────────────────┐
  │         Tensor Core의 FP16 행렬 곱셈 (WMMA: Warp Matrix Multiply Accumulate) │
  ├──────────────────────────────────────────────────────────────────────────────┤
  │                                                                              │
  │   연산: D = A × B + C                                                        │
  │                                                                              │
  │   ┌────────────────────────────────────────────────────────────────────────┐│
  │   │  A: 16×16 FP16 행렬 (입력)                                              ││
  │   │  B: 16×16 FP16 행렬 (입력)                                              ││
  │   │  C: 16×16 FP32 행렬 (누적자, Accumulator)                               ││
  │   │  D: 16×16 FP32 행렬 (결과)                                              ││
  │   │                                                                        ││
  │   │  ┌────┐   ┌────┐   ┌────┐   ┌────┐                                    ││
  │   │  │FP16│ × │FP16│ + │FP32│ = │FP32│                                    ││
  │   │  │4×4 │   │4×4 │   │4×4 │   │4×4 │  ← Volta의 기본 타일                ││
  │   │  └────┘   └────┘   └────┘   └────┘                                    ││
  │   │                                                                        ││
  │   │  1 사이클에 4×4×4 = 64개의 FP16 곱셈-누적 연산 수행                     ││
  │   └────────────────────────────────────────────────────────────────────────┘│
  │                                                                              │
  │   Tensor Core 성능 (FP16 기준):                                              │
  │   ┌────────────────────────────────────────────────────────────────────────┐│
  │   │  V100 (Volta):   125 TFLOPS (FP16 FMA 기준)                            ││
  │   │  A100 (Ampere):  312 TFLOPS (Tensor Core FP16)                         ││
  │   │  H100 (Hopper):  989 TFLOPS (Tensor Core FP16)                         ││
  │   │                                                                        ││
  │   │  FP32 CUDA Core 대비 Tensor Core FP16:                                 ││
  │   │  V100:  125 / 15.7 ≈ 8배                                               ││
  │   │  A100: 312 / 19.5 ≈ 16배                                               ││
  │   │  H100: 989 / 67  ≈ 15배                                                ││
  │   └────────────────────────────────────────────────────────────────────────┘│
  │                                                                              │
  │   ⚠ Tensor Core를 활용하려면 행렬 차원이 8 또는 16의 배수여야 함            │
  │      (Padding 필요)                                                         │
  │                                                                              │
  └──────────────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** Tensor Core는 FP16 행렬 A(16×16)와 B(16×16)를 곱하고, FP32 행렬 C(16×16)를 더하여 FP32 결과 D를 출력한다. 핵심은 곱셈의 중간 결과를 FP32로 유지한다는 점이다. 16×16×16 = 4,096개의 FP16 곱셈이 1 Tensor Core 연산으로 처리된다. 이것이 FP16 Mixed Precision Training이 FP32 대비 2~8배 빠른 이유다. 단, Tensor Core를 완전히 활용하려면 행렬의 차원(batch size, hidden dimension 등)이 8 또는 16의 배수여야 한다. 그렇지 않으면 Padding이 필요하며, 성능이 떨어진다.

## Ⅲ. 융합 비교 및 다각도 분석

### 비교 1: FP16 vs FP32 vs BF16

| 비교 항목 | FP16 (IEEE) | FP32 (단정밀도) | BF16 (Brain Float) |
|:---|:---|:---|:---|
| **총 비트** | 16 | 32 | 16 |
| **지수 비트** | 5 | 8 | 8 |
| **가수 비트** | 10 | 23 | 7 |
| **최댓값** | 65,504 (좁음!) | 3.4×10³⁸ | 3.4×10³⁸ (FP32와 동일) |
| **최소 정규화 수** | 6×10⁻⁵ | 1.2×10⁻³⁸ | 1.2×10⁻³⁸ |
| **정밀도** | ~3자리 | ~7자리 | ~2자리 |
| **Loss Scaling 필요** | **필수** | 불필요 | 선택적 (보통 불필요) |
| **하드웨어 지원** | GPU Tensor Core | 모든 FPU | GPU (Ampere+), TPU |

### 비교 2: FP16 vs INT8 (양자화)

| 비교 항목 | FP16 (부동소수점) | INT8 (정수 양자화) |
|:---|:---|:---|
| **비트 수** | 16 | 8 |
| **표현 방식** | 부호+지수+가수 | 부호+정수 (스케일+제로포인트) |
| **동적 범위** | 자동 (지수 조절) | 수동 (스케일 설정 필요) |
| **메모리** | FP32의 50% | FP32의 25% |
| **추론 속도** | FP32 대비 2~4배 | FP32 대비 4~8배 |
| **정확도 저하** | 거의 없음 (Mixed Precision 시) | 있음 ( 양자화 오차) |
| **주 용도** | 훈련 + 추론 | 추론 전용 (훈련은 어려움) |

### 과목 융합 관점

- **컴퓨터구조(컴구)**: GPU의 메모리 계층에서 FP16은 HBM(High Bandwidth Memory) 대역폭을 절반으로 줄인다. A100 HBM2e 2TB/s에서 FP32 모델은 1TB/s의 실질 대역폭을 사용하지만, FP16 모델은 500GB/s만 사용하고 나머지 1.5TB/s는 연산에 집중할 수 있다.

- **운영체제(OS)**: CUDA Driver는 FP16 텐서의 메모리 정렬을 2바이트로 요구한다. 잘못된 정렬은 성능 저하를 유발하며, Tensor Core 사용 시에는 16바이트 정렬이 권장된다.

- **데이터베이스(DB)**: 일부 데이터베이스(PostgreSQL의 `halfvec` 확장)는 벡터 유사도 검색을 위해 FP16 벡터를 지원한다. FP32 대비 인덱스 크기가 절반이다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오

1. **시나리오 — 대규모 LLM 훈련의 메모리 병목**: GPT-3(175B 파라미터)를 FP32로 훈련하면 가중치만 700GB(175B × 4 bytes)가 필요하다. 단일 GPU에 들어가지 않는다. 기술사는 FP16 Mixed Precision을 적용하여 가중치를 350GB로 줄이고, ZeRO-3 + Model Parallelism를 결합하여 다중 GPU 분산 훈련을 설계해야 한다.

2. **시나리오 — 실시간 추론의 지연 시간 요건**: 자율주행차의 객체 탐지 모델이 30 FPS(33ms/frame)를 달성해야 한다. FP32 모델은 20 FPS(50ms/frame)에 불과하다. 기술사는 FP16으로 변환 후 TensorRT로 최적화하여 40 FPS(25ms/frame)를 달성하고, 안전 마진을 확보한다.

3. **시나리오 — FP16 그래디언트 오버플로우로 인한 훈련 실패**: ResNet-50 훈련 중 Loss가 갑자기 NaN으로 폭발했다. 로그를 확인해 보니 FP16 그래디언트가 65,504를 초과하여 오버플로우 발생. 기술사는 Loss Scaling Factor를 1024에서 128로 낮추고, FP32 Master Weights가 올바르게 활성화되어 있는지 점검한다.

### 도입 체크리스트
- **기술적**: 모델의 그래디언트 범위를 확인했는가? FP16 최댓값(65,504)을 초과할 가능성이 있다면 Loss Scaling이 필수다.
- **운영·보안적**: FP16 추론 시 정확도 저하가 1% 이내인가? 그 이상이면 FP32 유지 또는 BF16으로 전환을 고려해야 한다.

### 안티패턴
- **Loss Scaling 없이 FP16 훈련**: 그래디언트 언더플로우로 학습이 진행되지 않거나, 오버플로우로 NaN이 발생한다.
- **FP16 Master Weights 사용**: FP16으로 가중치를 직접 업데이트하면 작은 그래디언트가 누적되지 않아 학습이 정체된다.

## Ⅴ. 기대효과 및 결론

### 정량/정성 기대효과

| 구분 | FP32 사용 시 | FP16 Mixed Precision | 개선 효과 |
|:---|:---|:---|:---|
| **정량** | 메모리 100% | 메모리 50% | 2배 큰 모델 가능 |
| **정량** | 훈련 속도 기준 | 훈련 속도 2~4배 | 시간/비용 절감 |
| **정량** | 추론 지연 기준 | 추론 지연 50~75% | 실시간 처리 가능 |
| **정성** | GPU 메모리 병목 | 메모리 여유 확보 | 배치 사이즈 증가 가능 |

### 미래 전망
- **BF16으로의 전환**: 2023년 이후, NVIDIA Ampere+(A100, H100, L40S)와 Google TPU가 BF16을 네이티브로 지원하면서, FP16 대신 BF16이 딥러닝 훈련의 새로운 표준으로 부상하고 있다. BF16은 Loss Scaling이 필요 없어 구현이 더 간단하다.
- **FP8의 등장**: 2024년 NVIDIA Blackwell, Intel Gaudi3, Qualcomm Cloud AI 100 Ultra가 FP8(E4M3, E5M2)을 지원한다. 추론 단계에서는 FP8로도 충분하며, 메모리/대역폭이 1/4로 줄어든다.

### 참고 표준
- **IEEE 754-2008**: 반정밀도(binary16) 포맷 최초 정의
- **IEEE 754-2019**: 반정밀도 포맷 유지 (변경 없음)
- **NVIDIA Tensor Core**: FP16 WMMA (Warp Matrix Multiply Accumulate) API

## 👶 어린이를 위한 3줄 비유 설명
1. 반정밀도(FP16)는 **'미니 여행 가방'**이에요. 일반 가방(FP32)보다 작지만 가벼워서, 비행기에 더 많이 실을 수 있고, 이동도 빨라요!
2. 하지만 미니 가방은 너무 큰 물건(65,504 이상)은 못 담아요. 큰 물건을 억지로 넣으면 터져버려서(무한대), 그래서 "스케일링"이라는 기술로 물건을 잠깐 작게 만들어서 넣어요.
3. 덕분에 AI가 더 많은 공부를 더 빠르게 할 수 있어요! 요즘은 AI 훈련에 이 미니 가방이 인기랍니다.
