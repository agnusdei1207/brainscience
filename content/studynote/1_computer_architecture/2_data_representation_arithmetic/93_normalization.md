# 정규화 (Normalization)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 부동소수점 정규화(Normalization)는 실수를 $1.M \times 2^E$ 형태(이진수 기준)로 변환하여, 가수(Mantissa)의 최상위 비트가 항상 1이 되도록 소수점 위치를 조정하는 과정이다.
> 2. **가치**: 정규화를 통해 ① 암시적 선행 1(Implicit Leading 1)을 사용하여 1비트를 절약하고, ② 동일한 비트 수로 최대 정밀도를 보장하며, ③ 부동소수점 비교 연산을 정수 비교처럼 단순화한다.
> 3. **융합**: 정규화는 FP32/FP64/FP16 모든 IEEE 754 포맷의 기반이며, 하드웨어 FPU(Floating-Point Unit)의 핵심 로직을 구성한다. 서브노멀(Denormal)은 정규화가 불가능한 예외 영역이다.

---

## Ⅰ. 개요 (Context & Background)

- **개념**: 정규화(Normalization)는 부동소수점 수를 표현하기 전에 소수점을 이동시켜 가수(Mantissa)를 `[1.0, 2.0)` (이진수 기준) 범위로 만드는 과정이다. 예를 들어 십진수 12.5는 이진수 `1100.1`인데, 이를 정규화하면 `1.1001 × 2³`이 된다. 이때 가수부는 `.1001`만 저장하고, 선행 `1`은 암시적으로 생략한다.

- **💡 비유**: 정규화는 **'책장에 책을 세울 때 크기 순으로 정리'**하는 것과 같다. 어떤 책이든 꽂힌 책의 높이가 "1.x cm"가 되도록 맞추는 것이다. 아주 큰 백과사전(12.5)도 "1.25 × 10"으로, 아주 작은 소책자(0.0012)도 "1.2 × 10⁻³"으로 표현하면, 모든 책이 동일한 규칙으로 정리된다.

- **등장 배경 및 발전 과정**:
  1. **과학적 기수법의 컴퓨터 이식**: 인간은 큰 수를 $1.23 \times 10^6$으로 표현한다. 컴퓨터도 동일한 원리를 이진수에 적용하여 $1.M \times 2^E$ 형태를 사용한다.
  2. **비트 효율성 극대화**: 가수의 최상위 비트가 항상 1이라는 점을 이용하여, 이를 실제로 저장하지 않음으로써 1비트를 절약한다. FP32에서 23비트 가수가 실제로는 24비트 정밀도를 갖는 이유다.
  3. **IEEE 754의 표준화**: 1985년 IEEE 754는 정규화 수(Normalized Number)와 서브노멀(Denormal/Subnormal)을 명확히 구분하여, 0에 가까운 수도 표현 가능하도록 했다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 정규화 과정: 10진수 → 2진수 → 정규화된 부동소수점

정규화는 실수를 부동소수점으로 저장하기 위해 반드시 거쳐야 하는 변환 과정이다.

```text
  ┌──────────────────────────────────────────────────────────────────────────────┐
  │         10진수 13.625를 FP32로 저장하는 정규화 과정                           │
  ├──────────────────────────────────────────────────────────────────────────────┤
  │                                                                              │
  │   ① 10진수를 2진수로 변환:                                                   │
  │   ┌────────────────────────────────────────────────────────────────────────┐│
  │   │  13 (정수부) = 8 + 4 + 1 = 1101₂                                       ││
  │   │  0.625 (소수부) = 0.5 + 0.125 = 0.101₂                                 ││
  │   │  → 13.625 = 1101.101₂                                                  ││
  │   └────────────────────────────────────────────────────────────────────────┘│
  │                                                                              │
  │   ② 정규화 (소수점 이동):                                                    │
  │   ┌────────────────────────────────────────────────────────────────────────┐│
  │   │  1101.101₂ = 1.101101₂ × 2³                                            ││
  │   │                ↑          ↑                                             ││
  │   │            가수(M)      지수(E=3)                                       ││
  │   │                                                                        ││
  │   │  ⚠ 소수점을 왼쪽으로 3칸 이동 → 지수 = +3                               ││
  │   │     가수는 항상 1.xxxx 형태가 됨                                        ││
  │   └────────────────────────────────────────────────────────────────────────┘│
  │                                                                              │
  │   ③ FP32 포맷으로 저장:                                                      │
  │   ┌────────────────────────────────────────────────────────────────────────┐│
  │   │  부호(S) = 0 (양수)                                                     ││
  │   │  지수(E) = 3 + 127 (Bias) = 130 = 10000010₂                            ││
  │   │  가수(M) = 10110100000000000000000₂ (소수점 이하만, 선행 1 생략)         ││
  │   │                                                                        ││
  │   │  ┌───┬──────────┬─────────────────────────────────────────────────┐   ││
  │   │  │ 0 │10000010  │10110100000000000000000                         │   ││
  │   │  └───┴──────────┴─────────────────────────────────────────────────┘   ││
  │   │   S     E=130              M (23비트)                                  ││
  │   └────────────────────────────────────────────────────────────────────────┘│
  │                                                                              │
  │   ④ 역변환 (검증):                                                          │
  │   ┌────────────────────────────────────────────────────────────────────────┐│
  │   │  값 = (-1)^0 × (1.101101) × 2^(130-127)                                ││
  │   │     = 1.101101₂ × 2³                                                   ││
  │   │     = 1101.101₂                                                        ││
  │   │     = 13.625 ✓                                                         ││
  │   └────────────────────────────────────────────────────────────────────────┘│
  │                                                                              │
  └──────────────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 정규화의 핵심은 소수점을 이동시켜 가수의 정수부가 항상 1이 되도록 만드는 것이다. 1101.101₂에서 소수점을 3칸 왼쪽으로 이동시키면 1.101101₂ × 2³이 된다. 이때 지수는 이동한 칸 수(+3)가 된다. 가수의 소수점 이하(101101)만 저장하고, 선행 1은 생략(Implicit Leading 1)한다. 이 1비트 절약이 32비트 내에서 24비트 정밀도를 가능하게 한다. 편향값(Bias = 127)을 더해 지수를 양수로 만들면, E = 3 + 127 = 130이 된다.

### 암시적 선행 1 (Implicit Leading 1)

정규화된 부동소수점 수는 가수의 최상위 비트가 항상 1이라는 성질을 이용하여, 이를 실제로 저장하지 않는다.

```text
  ┌──────────────────────────────────────────────────────────────────────────────┐
  │         암시적 선행 1 (Implicit Leading 1)의 원리                             │
  ├──────────────────────────────────────────────────────────────────────────────┤
  │                                                                              │
  │   정규화된 이진수: 항상 1.xxxx 형태                                           │
  │   ┌────────────────────────────────────────────────────────────────────────┐│
  │   │  1.101101₂ × 2³                                                        ││
  │   │  ↑                                                                      ││
  │   │  항상 1 (생략 가능)                                                      ││
  │   └────────────────────────────────────────────────────────────────────────┘│
  │                                                                              │
  │   FP32 저장:                                                                │
  │   ┌────────────────────────────────────────────────────────────────────────┐│
  │   │  저장되는 가수(M): 10110100000000000000000 (23비트)                      ││
  │   │  실제 유효숫자:   1.10110100000000000000000 (24비트)                     ││
  │   │                     ↑                                                   ││
  │   │               암시적 선행 1 (Implicit Leading 1)                        ││
  │   └────────────────────────────────────────────────────────────────────────┘│
  │                                                                              │
  │   ⚠ 이점:                                                                   │
  │   ┌────────────────────────────────────────────────────────────────────────┐│
  │   │  • 23비트 공간에 24비트 정밀도 저장 가능                                 ││
  │   │  • FP64: 52비트 가수 → 53비트 정밀도                                    ││
  │   │  • FP16: 10비트 가수 → 11비트 정밀도                                    ││
  │   │  • "공짜로 1비트 얻는" 효과                                              ││
  │   └────────────────────────────────────────────────────────────────────────┘│
  │                                                                              │
  │   ⚠ 제약:                                                                   │
  │   ┌────────────────────────────────────────────────────────────────────────┐│
  │   │  • 0은 정규화할 수 없음 (별도의 0 표현 필요)                             ││
  │   │  • 0.xxxx 형태의 아주 작은 수는 정규화 불가 → 서브노멀(Denormal)        ││
  │   └────────────────────────────────────────────────────────────────────────┘│
  │                                                                              │
  └──────────────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 암시적 선행 1은 IEEE 754의 천재적인 비트 절약 기법이다. 정규화된 이진수는 항상 1.something 형태이므로, 굳이 1을 저장할 필요가 없다. 대신 가수 필드에는 소수점 이하만 저장하고, 하드웨어가 값을 읽을 때 자동으로 1을 붙인다. 이로써 FP32의 23비트 가수가 실제로는 24비트(약 7.2자리) 정밀도를 제공한다. 단, 이 방식은 0을 표현할 수 없다는 문제가 있다. 0은 정규화할 수 없기 때문에(0.0 = 0.0 × 2^E, 선행 1이 없음), 별도의 특수 표현(E=0, M=0)을 사용한다.

### 정규화 수 vs 서브노멀 (Normalized vs Subnormal)

정규화가 불가능한 아주 작은 수는 서브노멀(Denormal/Subnormal)로 표현된다.

```text
  ┌──────────────────────────────────────────────────────────────────────────────┐
  │         정규화 수 vs 서브노멀 (FP32 기준)                                     │
  ├──────────────────────────────────────────────────────────────────────────────┤
  │                                                                              │
  │   정규화 수 (Normalized Number):                                             │
  │   ┌────────────────────────────────────────────────────────────────────────┐│
  │   │  조건: 0 < E < 255 (지수가 0과 최댓값 사이)                              ││
  │   │  형태: 1.M × 2^(E-127)                                                 ││
  │   │  범위: 2^-126 ≈ 1.18×10^-38 ~ 3.4×10^38                                ││
  │   │  예: 1.000...0 × 2^-126 (최소 정규화 수)                                ││
  │   └────────────────────────────────────────────────────────────────────────┘│
  │                                                                              │
  │   서브노멀 (Subnormal/Denormal Number):                                      │
  │   ┌────────────────────────────────────────────────────────────────────────┐│
  │   │  조건: E = 0, M ≠ 0 (지수가 0이고 가수가 0이 아님)                       ││
  │   │  형태: 0.M × 2^-126  ← 선행 0, 지수 고정                                ││
  │   │  범위: 2^-149 ≈ 1.40×10^-45 ~ 2^-126                                   ││
  │   │  예: 0.000...1 × 2^-126 = 2^-149 (최소 서브노멀)                        ││
  │   └────────────────────────────────────────────────────────────────────────┘│
  │                                                                              │
  │   실수선 (0 주변 확대):                                                      │
  │                                                                              │
  │   0   서브노멀 영역         정규화 수                                        │
  │   │   │    │    │    │     │    │    │    │    │                          │
  │   ├───┼────┼────┼────┼─────┼────┼────┼────┼────┤                          │
  │   0  2^-149            2^-126  2^-125  2^-124                          │
  │       ↑                    ↑                                              │
  │    최소 서브노멀       최소 정규화 수                                        │
  │                                                                              │
  │   ⚠ 서브노멀의 목적:                                                       │
  │   ┌────────────────────────────────────────────────────────────────────────┐│
  │   │  "점진적 언더플로우 (Gradual Underflow)" 구현                           ││
  │   │  정규화 수 최소값과 0 사이의 간격을 메워, 급격한 0으로의 낙하 방지       ││
  │   │  오차가 점진적으로 증가하도록 유도                                      ││
  │   └────────────────────────────────────────────────────────────────────────┘│
  │                                                                              │
  │   ⚠ 서브노멀의 단점:                                                       │
  │   ┌────────────────────────────────────────────────────────────────────────┐│
  │   │  • 하드웨어 구현이 복잡 (별도의 로직 필요)                               ││
  │   │  • 연산 속도가 느림 (일부 CPU에서는 10~100배 느림)                       ││
  │   │  • FTZ(Flush-To-Zero) 모드로 비활성화 가능 (성능 vs 정확성 트레이드오프) ││
  │   └────────────────────────────────────────────────────────────────────────┘│
  │                                                                              │
  └──────────────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 서브노멀(Denormal)은 정규화 수의 최솟값(2⁻¹²⁶)과 0 사이를 메우는 특수 영역이다. 정규화 수는 항상 선행 1이 있어야 하지만, 2⁻¹²⁶보다 작은 수는 이것이 불가능하다. 서브노멀은 선행 1 대신 선행 0을 사용하여, 2⁻¹⁴⁹까지 표현할 수 있게 한다. 이를 점진적 언더플로우(Gradual Underflow)라 하며, 작은 수가 갑자기 0이 되는 대신 점진적으로 정밀도를 잃도록 한다. 단, 서브노멀 연산은 하드웨어적으로 복잡하여 성능 저하가 발생할 수 있다. 이 때문에 일부 시스템에서는 FTZ(Flush-To-Zero) 모드를 활성화하여 서브노멀을 0으로 처리하기도 한다.

## Ⅲ. 융합 비교 및 다각도 분석

### 비교 1: 정규화 수 vs 서브노멀

| 비교 항목 | 정규화 수 (Normalized) | 서브노멀 (Subnormal) |
|:---|:---|:---|
| **지수(E) 조건** | 0 < E < Max | E = 0 |
| **가수 형태** | 1.M (선행 1) | 0.M (선행 0) |
| **유효숫자** | 24비트 (FP32) | 1~23비트 (점진적 감소) |
| **표현 범위** | 2⁻¹²⁶ ~ 10³⁸ | 2⁻¹⁴⁹ ~ 2⁻¹²⁶ |
| **연산 속도** | 빠름 (하드웨어 최적화) | 느림 (소프트웨어 처리 가능) |
| **주 용도** | 일반 부동소수점 연산 | 아주 작은 수, 언더플로우 완화 |

### 비교 2: IEEE 754 포맷별 정규화 특성

| 포맷 | 편향값(Bias) | 최소 정규화 수 | 최소 서브노멀 |
|:---|:---:|:---|:---|
| **FP16** | 15 | 2⁻¹⁴ ≈ 6.1×10⁻⁵ | 2⁻²⁴ ≈ 6.0×10⁻⁸ |
| **FP32** | 127 | 2⁻¹²⁶ ≈ 1.2×10⁻³⁸ | 2⁻¹⁴⁹ ≈ 1.4×10⁻⁴⁵ |
| **FP64** | 1023 | 2⁻¹⁰²² ≈ 2.2×10⁻³⁰⁸ | 2⁻¹⁰⁷⁴ ≈ 5.0×10⁻³²⁴ |

### 과목 융합 관점

- **컴퓨터구조(컴구)**: FPU는 정규화를 하드웨어로 자동 수행한다. 덧셈/뺄셈 후 결과를 다시 정규화하는 "Renormalization" 단계가 파이프라인에 포함된다. 서브노멀 처리는 별도의 마이크로코드로 처리될 수 있다.

- **운영체제(OS)**: OS는 FPU의 FTZ(Flush-To-Zero)와 DAZ(Denormals-Are-Zero) 모드를 제어할 수 있다. 오디오/비디오 실시간 처리에서는 성능을 위해 FTZ를 활성화하고, 과학 계산에서는 정확성을 위해 비활성화한다.

- **컴파일러(Compiler)**: 컴파일러 최적화 `-ffast-math`는 서브노멀을 0으로 처리하는 등 IEEE 754 엄격성을 완화하여 성능을 높인다. `-fno-fast-math`가 기본값이다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오

1. **시나리오 — 오디오 처리에서의 서브노멀 성능 저하**: 디지털 오디오 이펙트(Reverb, Compressor) 계산 중, 신호가 아주 작아져 서브노멀 영역에 진입했다. CPU 사용량이 10%에서 90%로 급증했다. 기술사는 FPU의 FTZ(Flush-To-Zero) 모드를 활성화하여 서브노멀을 0으로 처리함으로써 성능을 회복한다. 오디오에서 10⁻⁴⁵ 수준의 신호는 인간이 들을 수 없으므로 정확성 손실은 무시 가능하다.

2. **시나리오 — 수치 해석에서의 점진적 언더플로우 필요성**: 유한 요소 해석(FEM)에서 스트레스 텐서의 아주 작은 성분이 서브노멀로 표현되어야 한다. FTZ를 활성화하면 0이 되어 응력 분포가 왜곡된다. 기술사는 FTZ를 비활성화하고, 서브노멀 성능 저하를 감수하며 정확성을 우선시한다. 필요시 FP64로 전환하여 서브노멀 영역을 더 낮춘다.

3. **시나리오 — 딥러닝 Activation의 언더플로우**: ReLU Activation 후 아주 작은 양수 값들이 서브노멀로 떨어져 연산 속도가 저하된다. 기술사는 Activation Function을 Leaky ReLU로 변경하여 작은 값들도 일정 수준 이상 유지되도록 하거나, BF16을 사용하여 서브노멀 영역을 FP32와 동일하게 유지한다.

### 도입 체크리스트
- **기술적**: 애플리케이션이 서브노멀 연산을 자주 수행하는가? 그렇다면 FTZ 모드 설정을 검토해야 한다.
- **운영·보안적**: 서브노멀을 0으로 처리할 때 발생하는 정확성 손실이 허용 가능한 범위인가?

### 안티패턴
- **FTZ 모드를 모르고 사용**: 과학 계산에서 FTZ가 활성화되어 있으면, 작은 값들이 0이 되어 결과가 왜곡된다.
- **서브노멀 무시로 인한 발산**: 반복 알고리즘(예: Gradient Descent)에서 작은 값이 0이 되면 수렴하지 않고 발산할 수 있다.

## Ⅴ. 기대효과 및 결론

### 정량/정성 기대효과

| 구분 | 정규화 없이 | 정규화 적용 | 효과 |
|:---|:---|:---|:---|
| **정량** | 비트 낭비 (선행 1 저장) | 1비트 절약 | 정밀도 1비트 향상 |
| **정량** | 비교 연산 복잡 | 정수 비교와 동일 | 하드웨어 단순화 |
| **정성** | 구현마다 다른 표현 | IEEE 754 표준화 | 이식성 향상 |

### 미래 전망
- **AI 전용 포맷과 정규화**: FP8(E4M3, E5M2)도 정규화를 사용하지만, 일부 변형(Non-normalized)도 연구되고 있다. 정규화 하드웨어 오버헤드를 줄이기 위함이다.
- **Posit 포맷**: John Gustafson이 제안한 Posit은 정규화 대신 "tapered precision"을 사용하여, 큰 수와 작은 수 모두에게 더 균등한 정밀도를 제공한다. 아직 IEEE 표준은 아니다.

### 참고 표준
- **IEEE 754-2019**: 정규화 수와 서브노멀의 정의
- **ISO/IEC 60559**: IEEE 754의 ISO 버전

## 👶 어린이를 위한 3줄 비유 설명
1. 정규화는 **'책장에 책을 딱 맞게 정리'**하는 거예요. 어떤 책이든 "1.x cm" 높이가 되도록 맞추면, 모든 책이 같은 규칙으로 정리돼요.
2. 이렇게 하면 책의 "첫 페이지"는 항상 1이니까, 굳이 적지 않아도 돼요! 그래서 23칸짜리 책장에 24칸짜리 책을 넣을 수 있답니다.
3. 하지만 아주 얇은 책(0에 가까운 수)은 이 규칙이 안 통해서, 특별한 칸(서브노멀)에 따로 보관해야 해요!
