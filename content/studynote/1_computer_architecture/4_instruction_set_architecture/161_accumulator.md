## Ⅰ. 개요 (Context & Background)

### 개념 정의

누산기(Accumulator)는 **모든 산술 및 논리 연산의 결과가 저장되는 특수 레지스터**다. 1-주소 명령어 아키텍처에서 핵심 역할을 하며, 피연산자 중 하나가 암묵적으로 누산기로 지정된다.

### 💡 비유: 계산기의 결과창

누산기는 **계산기의 결과창**과 같다. 계산기에서 덧셈, 뺄셈을 할 때마다 결과가 결과창에 표시된다. 마찬가지로 CPU도 모든 연산 결과를 누산기에 모은다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 누산기 vs 범용 레지스터

```
┌─────────────────────────────────────────────────────────────────────┐
│                누산기 vs 범용 레지스터                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【누산기 아키텍처 (Accumulator-based)】                             │
│  ────────────────────────────────────                               │
│                                                                     │
│  장점:                                                              │
│  • 명령어 길이 짧음 (피연산자 1개)                                  │
│  • 코드 크기 작음                                                  │
│  • 하드웨어 단순                                                   │
│  • 전력 소모 적음                                                   │
│                                                                     │
│  단점:                                                              │
│  • 프로그래밍 어려움                                                │
│  • 중간 결과 저장 어려움                                            │
│  • 병렬 처리 어려움                                                  │
│  • 현대 고성능 CPU에서 거의 사용 안 함                              │
│                                                                     │
│  【범용 레지스터 아키텍처 (GPR-based)】                               │
│  ────────────────────────────────────                               │
│                                                                     │
│  장점:                                                              │
│  • 프로그래밍 유연함                                                 │
│  • 중간 결과 다수 저장                                               │
│  • 파이프라이닝 용이                                                  │
│  • 컴파일러 최적화 용이                                               │
│                                                                     │
│  단점:                                                              │
│  • 명령어 길이 김                                                    │
│  • 더 많은 레지스터 필요                                             │
│  • 하드웨어 복잡                                                     │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  비교 예시: ADD 연산                                               │ │
│  │  ──────────────────                                               │ │
│  │  누산기:     ADD mem     ; ACC ← ACC + mem (2바이트)            │ │
│  │  GPR (3-주소): ADD R1,R2,R3 ; R1 ← R2 + R3 (4바이트)            │ │
│  │  GPR (2-주소): ADD R1,R2    ; R1 ← R1 + R2 (2~3바이트)           │ │
│  │                                                               │ │
│  │  → 누산기 방식이 명령어 길이는 짧지만, GPR이 성능에 유리          │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

### 역사적 발전

```
┌─────────────────────────────────────────────────────────────────────┐
│                누산기 아키텍처의 역사                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【초기 컴퓨터 (1940~1960)】                                        │
│  ──────────────────────────                                         │
│  • ENIAC, EDVAC: 누산기 기반                                         │
│  • 모든 연산이 누산기를 통해                                        │
│  • 메모리 참조 방식                                                 │
│                                                                     │
│  【8비트 마이크로프로세서 (1970~1990)】                             │
│  ──────────────────────────────────                                 │
│  • Intel 8080, 8085, 8086: 누산기 사용 (A 레지스터)                │
│  • Motorola 6800: 2개 누산기 (A, B)                                 │
│  • MOS 6502 (Apple II): 누산기 기반                                │
│                                                                     │
│  【현대 임베디드/DSP】                                               │
│  ──────────────────────                                             │
│  • PIC, AVR: 누산기 기반 (간단함)                                   │
│  • 일부 DSP: 누산기 사용                                            │
│  • 저전력, 코드 크기 중시                                           │
│                                                                     │
│  【현대 고성능 CPU】                                                 │
│  ──────────────────────                                             │
│  • x86, ARM: 범용 레지스터 기반                                     │
│  • 누산기 개념은 레지스터 파일 내에 흡수                              │
│  • R0 또는 특정 레지스터가 누산기 역할 가능                        │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  아키텍처별 누산기 여부:                                          │ │
│  │  ──────────────────────                                           │ │
│  │  아키텍처        누산기 여부       설명                           │ │
│  │  ─────────────  ───────────────  ─────────────────              │ │
│  │  Intel 8086    있음 (AX)         16비트 누산기                  │ │
│  │  MOS 6502      있음 (A)          8비트 누산기                   │ │
│  │  x86-64        개념만 있음        RAX가 역할 수행                 │ │
│  │  ARM32         없음              범용 레지스터                    │ │
│  │  ARM64         없음              31개 GPR                         │ │
│  │  RISC-V        없음              32개 GPR                         │ │
│  │  PIC (8비트)   있음 (W)          워킹 레지스터                   │ │
│  │  AVR (8비트)   있음              다양한 방식                        │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오

**시나리오 1: 임베디드 MCU 선택**
- 요구: 작은 코드 크기, 저전력
- 선택: PIC, AVR (누산기 기반)
- 이유: 명령어 짧음, 단순함

**시나리오 2: 고성능 컴퓨팅**
- 요구: 최고 성능
- 선택: ARM, x86 (GPR 기반)
- 이유: 파이프라인, 병렬 처리

**시나리오 3: 센서 데이터 처리**
- 요구: 누적 연산
- 접근: 누산기 활용 알고리즘
- 결과: 효율적 데이터 집계

**시나리오 4: 레거시 코드 유지보수**
- 요구: 8비트 MCU 코드 수정
- 접근: 누산기 아키텍처 이해
- 결과: 효율적 수정

```
│  │  A = B + C       LOAD B       LOAD R1, B                         │ │
```

## Ⅴ. 기대효과 및 결론

### 정량/정성 기대효과

| 환경 | 누산기 적합성 | 이유 |
|:---|:---:|:---|
| **임베디드** | 높음 | 코드 크기, 단순성 |
| **DSP** | 높음 | 누적 연산 특화 |
| **고성능 CPU** | 낮음 | 병렬 처리 어려움 |
| **GPU** | 없음 | 벡터/SIMD 연산 |

### 핵심 요약

```
• 누산기: 모든 연산 결과가 모이는 특수 레지스터
• 1-주소 명령어의 핵심 피연산자
• 장점: 명령어 짧음, 하드웨어 단순
• 단점: 프로그래밍 어려움, 병렬 처리 어려움
• 현대: GPR 기반이 주류, 임베디드에서 일부 사용
```
