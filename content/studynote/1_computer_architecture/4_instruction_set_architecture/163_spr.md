# 163. 특수 목적 레지스터 (SPR, Special Purpose Register)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 특정 기능을 수행하기 위해 전용화된 레지스터, CPU 동작의 핵심
> 2. **가치**: 명령어 실행, 메모리 관리, 상태 추적, 시스템 제어
> 3. **융합**: 운영체제 커널, 인터럽트 처리, 가상화, 디버깅

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

특수 목적 레지스터(SPR, Special Purpose Register)는 **특정 기능을 수행하도록 전용화된 레지스터**로, GPR과 달리 용도가 고정되어 있다. CPU의 동작 상태, 제어, 메모리 관리 등 시스템 핵심 기능을 담당한다.

### 💡 비유: 자동차 계기판

SPR은 **자동차 계기판**과 같다. 속도계는 속도만, 연료계는 연료만, 온도계는 온도만 표시한다. 각각 용도가 정해져 있고, 그 용도에 특화되어 있다. GPR은 빈 짐칸처럼 아무거나 넣을 수 있지만, SPR은 전용 계기다.

### SPR 분류

```
┌─────────────────────────────────────────────────────────────────────┐
│                SPR 분류                                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【1. 프로그램 상태 레지스터 (Program Status)】                      │
│  ─────────────────────────────────────────                          │
│  • PC (Program Counter): 다음 명령어 주소                          │
│  • IR (Instruction Register): 현재 명령어                           │
│  • PSR/FLAGS (Status Register): 연산 결과 상태                      │
│                                                                     │
│  【2. 스택 관련 레지스터】                                           │
│  ──────────────────────────                                         │
│  • SP (Stack Pointer): 스택 최상단 주소                             │
│  • FP (Frame Pointer): 현재 스택 프레임 기준                        │
│  • BP (Base Pointer): 베이스 주소                                   │
│                                                                     │
│  【3. 함수 호출 관련 레지스터】                                      │
│  ──────────────────────────────                                     │
│  • LR/RA (Link/Return Address): 복귀 주소                          │
│                                                                     │
│  【4. 메모리 관리 레지스터】                                         │
│  ──────────────────────────                                         │
│  • MAR (Memory Address Register): 메모리 주소                       │
│  • MDR (Memory Data Register): 메모리 데이터                        │
│  • 페이지 테이블 베이스 (TTBR)                                      │
│                                                                     │
│  【5. 제어 레지스터 (Control Registers)】                            │
│  ──────────────────────────────────                                 │
│  • CR0~CR4 (x86): 제어 플래그                                       │
│  • MSR (Model Specific Register)                                    │
│  • SCTLR (ARM): 시스템 제어                                         │
│                                                                     │
│  【6. 시스템 레지스터】                                              │
│  ────────────────────                                                │
│  • TP (Thread Pointer): 스레드 로컬 저장                            │
│  • GP (Global Pointer): 전역 데이터                                 │
│  • Timer 레지스터                                                   │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  SPR vs GPR 비교:                                                │ │
│  │  ────────────────                                                │ │
│  │  항목         GPR                 SPR                          │ │
│  │  ─────────    ─────────────────   ─────────────────             │ │
│  │  용도         범용                 특수                         │ │
│  │  접근         모든 명령어          특수 명령어                   │ │
│  │  권한         사용자 모드           커널 모드 필요               │ │
│  │  개수         다수 (16~128)        소수 (5~20)                  │ │
│  │  변경         프로그래머 자유       제한적                        │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 주요 SPR 상세

```
┌─────────────────────────────────────────────────────────────────────┐
│                주요 SPR 상세                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【PC (Program Counter)】                                           │
│  ────────────────────────────                                       │
│  • 역할: 다음에 실행할 명령어의 메모리 주소 저장                    │
│  • 크기: 아키텍처 비트 수와 동일 (32/64비트)                        │
│  • 자동 증가: 명령어 인출 후 자동으로 증가                          │
│  • 분기: JMP, CALL 등으로 변경                                      │
│                                                                     │
│  동작:                                                              │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. Fetch: Mem[PC] → IR                                       │ │
│  │  2. PC 증가: PC ← PC + 명령어_길이                             │ │
│  │  3. 분기 시: PC ← 새_주소                                       │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【SP (Stack Pointer)】                                              │
│  ──────────────────────────                                         │
│  • 역할: 스택 최상단(LIFO) 주소 가리킴                              │
│  • 관용: 감소 방향 (full descending)이 일반적                      │
│  • PUSH: SP ← SP - 4; Mem[SP] ← data                               │
│  • POP: data ← Mem[SP]; SP ← SP + 4                                │
│                                                                     │
│  【LR/RA (Link Register / Return Address)】                          │
│  ────────────────────────────────────────                           │
│  • 역할: 함수 호출 후 복귀할 주소 저장                              │
│  • CALL: LR ← PC + 4; PC ← 함수_주소                                │
│  • RET: PC ← LR                                                     │
│  • 중첩 호출 시: 스택에 저장 필요                                   │
│                                                                     │
│  【FLAGS / PSR (Program Status Register)】                           │
│  ────────────────────────────────────────────                       │
│  • 역할: 연산 결과 상태 저장                                        │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  플래그     의미                  예시                          │ │
│  │  ───────   ────────────────       ────────────────              │ │
│  │  Z (Zero)  결과가 0               SUB R0,R0 → Z=1              │ │
│  │  C (Carry) 캐리 발생              ADD 0xFF+0x02 → C=1           │ │
│  │  N (Negative) 결과가 음수         SUB 1,2 → N=1                │ │
│  │  V (Overflow) 오버플로우          127+1 → V=1                   │ │
│  │  I (Interrupt) 인터럽트 마스크     I=1이면 인터럽트 금지        │ │
│  │  T (Thumb) Thumb 모드             ARM에서 16비트 명령어         │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【MAR / MDR (Memory Address/Data Register)】                        │
│  ──────────────────────────────────────────                         │
│  • MAR: 접근할 메모리 주소                                          │
│  • MDR: 읽기/쓰기 데이터                                            │
│  • CPU 내부 버퍼 역할                                               │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  메모리 읽기:                                                    │ │
│  │  1. MAR ← 주소                                                  │ │
│  │  2. Read 신호                                                   │ │
│  │  3. MDR ← Mem[MAR]                                              │ │
│  │                                                               │ │
│  │  메모리 쓰기:                                                    │ │
│  │  1. MAR ← 주소                                                  │ │
│  │  2. MDR ← 데이터                                                │ │
│  │  3. Write 신호                                                  │ │
│  │  4. Mem[MAR] ← MDR                                              │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

### ARM CPSR/SPSR 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                ARM CPSR (Current Program Status Register)           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【CPSR 비트 구조】                                                  │
│  ──────────────────────                                             │
│                                                                     │
│  31 30 29 28 27 26 25 24  23~20    19~16   15~10    9  8  7  6 5~0 │
│  ┌──┬──┬──┬──┬──┬──┬──┬──┬─────┬─────┬─────┬──┬──┬──┬──┬─────┐  │
│  │ N│ Z│ C│ V│ Q│  │  │  │     │     │     │E│A│I│F│T   │  │
│  │  │  │  │  │  │  │  │  │     │     │     │  │  │  │  │ mode│  │
│  └──┴──┴──┴──┴──┴──┴──┴──┴─────┴─────┴─────┴──┴──┴──┴──┴─────┘  │
│                                                                     │
│  【조건 플래그 (비트 31~28)】                                        │
│  ─────────────────────────────                                      │
│  N (Negative): 결과의 MSB (음수 표시)                               │
│  Z (Zero): 결과가 0                                                 │
│  C (Carry): 캐리/보로우                                             │
│  V (Overflow): 부호 있는 오버플로우                                 │
│  Q (Saturation): 포화 연산 발생                                     │
│                                                                     │
│  【인터럽트 마스크 (비트 7~6)】                                      │
│  ─────────────────────────────                                      │
│  I: IRQ (일반 인터럽트) 마스크                                       │
│  F: FIQ (빠른 인터럽트) 마스크                                       │
│  A: 비동기 추억 (asynchronous abort) 마스크                          │
│  E: 빅/리틀 엔디안 제어                                              │
│                                                                     │
│  【모드 비트 (비트 4~0)】                                            │
│  ────────────────────────────                                       │
│  10000 (0x10): User (사용자)                                        │
│  10001 (0x11): FIQ (빠른 인터럽트)                                  │
│  10010 (0x12): IRQ (일반 인터럽트)                                  │
│  10011 (0x13): Supervisor (슈퍼바이저)                               │
│  10111 (0x17): Abort (어보트)                                       │
│  11011 (0x1B): Undefined (미정의)                                   │
│  11111 (0x1F): System (시스템)                                      │
│                                                                     │
│  【SPSR (Saved Program Status Register)】                            │
│  ─────────────────────────────────────                              │
│  • 예외 발생 시 CPSR을 SPSR에 저장                                  │
│  • 모드별 별도 SPSR 존재                                            │
│  • 복귀 시 SPSR → CPSR                                              │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  x86 FLAGS 레지스터 (EFLAGS/RFLAGS):                            │ │
│  │  ────────────────────────────────────                           │ │
│  │  주요 플래그:                                                    │ │
│  │  • CF (Carry Flag): 비트 0                                       │ │
│  │  • PF (Parity Flag): 비트 2                                      │ │
│  │  • AF (Auxiliary Carry): 비트 4                                  │ │
│  │  • ZF (Zero Flag): 비트 6                                        │ │
│  │  • SF (Sign Flag): 비트 7                                        │ │
│  │  • OF (Overflow Flag): 비트 11                                   │ │
│  │  • IF (Interrupt Flag): 비트 9                                   │ │
│  │  • DF (Direction Flag): 비트 10                                  │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오

**시나리오 1: OS 커널 개발**
- 요구: 컨텍스트 스위칭 구현
- 접근: 모든 SPR 저장/복원
- 구현: 어셈블리로 직접 제어
- 결과: 정확한 태스크 전환

**시나리오 2: 디버거 개발**
- 요구: 실행 상태 검사
- 접근: SPR 값 읽기/쓰기
- 구현: ptrace 또는 JTAG
- 결과: 단계별 실행 가능

**시나리오 3: 실시간 시스템**
- 요구: 빠른 인터럽트 응답
- 접근: SPR 직접 조작
- 구현: 인터럽트 마스크 제어
- 결과: 결정적 응답 시간

**시나리오 4: 보안 커널**
- 요구: 권한 분리
- 접근: 모드 전환 제어
- 구현: Secure 모드 SPR
- 결과: 신뢰 실행 환경

### SPR 접근 명령어

```
┌─────────────────────────────────────────────────────────────────────┐
│                SPR 접근 명령어                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【ARM】                                                            │
│  ────────                                                            │
│  ; CPSR 읽기                                                        │
│  MRS R0, CPSR         ; R0 ← CPSR                                  │
│                                                                     │
│  ; CPSR 쓰기                                                        │
│  MSR CPSR, R0         ; CPSR ← R0                                  │
│                                                                     │
│  ; 제어 레지스터                                                    │
│  MRC p15, 0, R0, c1, c0, 0  ; 시스템 제어 레지스터 읽기            │
│                                                                     │
│  【x86】                                                            │
│  ────────                                                            │
│  ; 플래그 읽기                                                      │
│  PUSHF                ; FLAGS를 스택에                              │
│  POP RAX              ; RAX ← FLAGS                                 │
│                                                                     │
│  ; 플래그 쓰기                                                      │
│  PUSH RAX             ; RAX를 스택에                                │
│  POPF                 ; FLAGS ← RAX                                 │
│                                                                     │
│  ; 제어 레지스터                                                    │
│  MOV CR0, RAX         ; CR0 ← RAX                                  │
│  MOV RAX, CR0         ; RAX ← CR0                                  │
│                                                                     │
│  【RISC-V】                                                         │
│  ──────────                                                         │
│  ; CSR 읽기/쓰기                                                    │
│  CSRRW rd, csr, rs   ; rd ← csr; csr ← rs                          │
│  CSRRS rd, csr, rs   ; rd ← csr; csr ← csr | rs                    │
│  CSRRC rd, csr, rs   ; rd ← csr; csr ← csr & ~rs                   │
│  CSRR rd, csr         ; rd ← csr (읽기만)                          │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  주요 CSR (RISC-V):                                              │ │
│  │  ───────────────────                                             │ │
│  │  CSR          주소     용도                                      │ │
│  │  ──────────   ─────    ────────────────────                     │ │
│  │  mstatus      0x300    머신 상태                                 │ │
│  │  mtvec        0x305    머신 트랩 벡터                            │ │
│  │  mepc         0x341    머신 예외 PC                              │ │
│  │  mie          0x304    머신 인터럽트 활성화                      │ │
│  │  mip          0x344    머신 인터럽트 펜딩                        │ │
│  │  satp         0x180    주소 변환/보호                           │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 정량/정성 기대효과

| SPR | 접근 권한 | 주요 용도 |
|:---|:---:|:---|
| **PC** | 사용자 | 실행 제어 |
| **SP** | 사용자 | 스택 관리 |
| **FLAGS** | 사용자 | 조건 분기 |
| **CR/CSR** | 커널 | 시스템 제어 |
| **TTBR** | 커널 | 메모리 관리 |

### 핵심 요약

```
• SPR: 특정 용도로 전용화된 레지스터
• 핵심 SPR: PC, SP, FLAGS, LR, MAR, MDR
• 접근: 특수 명령어 또는 권한 필요
• OS: 컨텍스트 스위칭 시 SPR 저장/복원
• 모드: User/Kernel 모드 전환
```
