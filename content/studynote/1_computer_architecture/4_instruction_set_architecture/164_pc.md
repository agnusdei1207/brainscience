# 164. 프로그램 카운터 (PC, Program Counter)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 다음에 실행할 명령어의 주소를 저장하는 특수 레지스터
> 2. **가치**: 순차적 실행, 분기 처리, 함수 호출/복귀의 핵심
> 3. **융합**: 파이프라인, 분기 예측, OS 스케줄링, 디버깅

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

프로그램 카운터(PC, Program Counter)는 **다음에 실행할 명령어의 메모리 주소를 저장하는 특수 레지스터**다. IP(Instruction Pointer)라고도 부르며(x86), CPU가 명령어를 순차적으로 실행하도록 제어하는 핵심 역할을 한다.

### 💡 비유: 책갈피

PC는 **책갈피**와 같다. 책을 읽을 때 어디까지 읽었는지 책갈피가 표시하듯, PC도 CPU가 어디까지 실행했는지 표시한다. 다음 페이지로 넘어가면 책갈피도 이동하듯, 명령어를 실행하면 PC도 다음 명령어로 이동한다.

### PC 동작 원리

```
┌─────────────────────────────────────────────────────────────────────┐
│                PC 동작 원리                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【기본 동작 (순차 실행)】                                           │
│  ────────────────────────────                                       │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │   메모리                                                       │ │
│  │   ┌─────────────────────────────────────────────────────────┐ │ │
│  │   │ 0x1000: MOV R1, #10                                     │ │ │
│  │   │ 0x1004: MOV R2, #20    ← PC                             │ │ │
│  │   │ 0x1008: ADD R3, R1, R2                                  │ │ │
│  │   │ 0x100C: STR R3, [R4]                                    │ │ │
│  │   └─────────────────────────────────────────────────────────┘ │ │
│  │                                                               │ │
│  │   실행 사이클:                                                 │ │
│  │   1. Fetch: IR ← Mem[PC]     ; 0x1004 명령어 인출            │ │
│  │   2. PC 증가: PC ← PC + 4    ; PC = 0x1008                   │ │
│  │   3. Execute: 명령어 실행                                     │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【분기 동작 (Branch)】                                              │
│  ──────────────────────────                                         │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │   메모리                                                       │ │
│  │   ┌─────────────────────────────────────────────────────────┐ │ │
│  │   │ 0x1000: CMP R1, #0                                      │ │ │
│  │   │ 0x1004: BEQ 0x2000     ; 분기 (Z=1이면)                  │ │ │
│  │   │ 0x1008: MOV R2, #10    ; 분기 안 하면 실행               │ │ │
│  │   │ ...                                                       │ │ │
│  │   │ 0x2000: MOV R3, #0     ; 분기하면 여기로                 │ │ │
│  │   └─────────────────────────────────────────────────────────┘ │ │
│  │                                                               │ │
│  │   분기 실행:                                                   │ │
│  │   • 조건 참: PC ← 0x2000                                      │ │
│  │   • 조건 거짓: PC ← 0x1008                                    │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【PC 크기】                                                         │
│  ────────────                                                        │
│  아키텍처       PC 크기      주소 공간                               │
│  ─────────────  ──────────   ───────────────                        │
│  32비트 CPU      32비트      4 GB                                    │
│  64비트 CPU      48~64비트   256 TB ~ 16 EB                         │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  PC 특징:                                                        │ │
│  │  ──────────                                                      │ │
│  │  • 읽기: 항상 가능                                               │ │
│  │  • 쓰기: 분기, 호출, 인터럽트, 예외 시에만                      │ │
│  │  • 복귀: 서브루틴 복귀 시 LR → PC                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### PC 변경 시나리오

```
┌─────────────────────────────────────────────────────────────────────┐
│                PC 변경 시나리오                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【1. 순차 실행 (Sequential)】                                       │
│  ──────────────────────────────                                     │
│  • 자동 증가: PC ← PC + 명령어_길이                                 │
│  • RISC: PC ← PC + 4 (고정 길이)                                   │
│  • CISC: PC ← PC + 1~15 (가변 길이)                                │
│                                                                     │
│  【2. 무조건 분기 (Unconditional Branch)】                          │
│  ────────────────────────────────────────                           │
│  • JMP label: PC ← label_주소                                      │
│  • BRA: PC ← PC + offset                                           │
│  • 용도: 무한 루프, 상태 머신                                      │
│                                                                     │
│  【3. 조건부 분기 (Conditional Branch)】                            │
│  ──────────────────────────────────────                             │
│  • JZ label: if (Z==1) PC ← label                                  │
│  • BNE label: if (Z==0) PC ← label                                 │
│  • 용도: if-else, 루프                                             │
│                                                                     │
│  【4. 서브루틴 호출 (Call)】                                         │
│  ──────────────────────────                                         │
│  • CALL func: LR ← PC; PC ← func                                   │
│  • BL func: LR ← PC+4; PC ← func                                   │
│  • 용도: 함수 호출                                                  │
│                                                                     │
│  【5. 서브루틴 복귀 (Return)】                                       │
│  ──────────────────────────                                         │
│  • RET: PC ← LR                                                    │
│  • BX LR: PC ← LR                                                  │
│  • 용도: 함수 복귀                                                  │
│                                                                     │
│  【6. 인터럽트/예외】                                                │
│  ──────────────────────                                             │
│  • 인터럽트: LR ← PC; PC ← 인터럽트_벡터                            │
│  • 예외: EPC ← PC; PC ← 예외_핸들러                                │
│  • 복귀: PC ← EPC 또는 LR                                          │
│                                                                     │
│  【7. 시스템 콜】                                                    │
│  ──────────────                                                      │
│  • SVC n: PC ← 시스템_콜_핸들러                                    │
│  • SYSCALL: PC ← 커널_진입점                                       │
│  • 복귀: ERET/SYSRET                                                │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  PC 관련 명령어 (ARM 예시):                                      │ │
│  │  ────────────────────────────                                    │ │
│  │  명령어         동작                    설명                    │ │
│  │  ─────────────  ────────────────────   ────────────             │ │
│  │  B label        PC ← label             무조건 분기              │ │
│  │  BEQ label      if Z==1: PC ← label    조건부 분기              │ │
│  │  BL func        LR←PC+4; PC←func       링크 분기 (호출)         │ │
│  │  BX LR          PC ← LR                레지스터 분기 (복귀)     │ │
│  │  LDR PC, [R1]   PC ← Mem[R1]           간접 분기                │ │
│  │  ERET           PC ← ELR               예외 복귀                │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

### PC와 파이프라인

```
┌─────────────────────────────────────────────────────────────────────┐
│                PC와 파이프라인                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【파이프라인에서 PC의 역할】                                        │
│  ──────────────────────────────                                     │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  5단계 파이프라인 (IF-ID-EX-MEM-WB):                            │ │
│  │  ──────────────────────────────────                             │ │
│  │                                                               │ │
│  │    PC → [IF] → [ID] → [EX] → [MEM] → [WB]                    │ │
│  │            ↑                                                    │ │
│  │            │                                                    │ │
│  │         분기 결정                                                │ │
│  │                                                               │ │
│  │  문제: 분기 결정이 EX 단계에서 발생                             │ │
│  │        → IF, ID 단계의 명령어가 잘못될 수 있음                  │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【분기 예측 (Branch Prediction)】                                   │
│  ──────────────────────────────────                                 │
│                                                                     │
│  • 정적 예측: 항상 분기 / 항상 분기 안 함                            │
│  • 동적 예측: 과거 이력 기반 (BHT, BTB)                             │
│  • 목적: 파이프라인 버블 감소                                       │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  분기 예측 실패 시:                                              │ │
│  │  ────────────────────                                           │ │
│  │  1. 파이프라인 플러시                                           │ │
│  │  2. PC ← 올바른 주소                                            │ │
│  │  3. 처음부터 다시 인출                                           │ │
│  │  4. 페널티: 파이프라인 깊이 × 사이클                             │ │
│  │     (예: 5단계 → 5 사이클 손실)                                 │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【PC 상대 주소 (PC-Relative Addressing)】                           │
│  ────────────────────────────────────────                           │
│                                                                     │
│  • 장점: 위치 독립 코드 (PIC)                                       │
│  • 방식: PC + offset                                                │
│  • 용도: 지역 분기, 데이터 접근                                     │
│                                                                     │
│  예:                                                                │
│  BEQ +100    ; PC ← PC + 100 (현재 PC 기준)                        │
│  LDR R1, [PC+#100]  ; R1 ← Mem[PC+100]                             │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  x86 vs ARM PC 차이:                                             │ │
│  │  ───────────────────────                                         │ │
│  │  x86 (RIP):                                                      │ │
│  │  • 다음 명령어 주소                                              │ │
│  │  • 인출 후 자동 증가                                              │ │
│  │                                                               │ │
│  │  ARM (R15/PC):                                                   │ │
│  │  • 현재 명령어 + 8 (파이프라인 프리페치)                         │ │
│  │  • PC 읽기 시 이미 2명령어 앞                                    │ │
│  │  • ARM7: PC = 현재 + 8                                          │ │
│  │  • ARMv7/8: 파이프라인 깊이에 따라 다름                          │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오

**시나리오 1: 커널 컨텍스트 스위칭**
- 요구: 프로세스 전환
- 접근: PC 저장/복원
- 구현: 어셈블리로 직접 조작
- 결과: 정확한 태스크 전환

**시나리오 2: 디버거 브레이크포인트**
- 요구: 특정 위치에서 정지
- 접근: PC 감시
- 구현: 하드웨어/소프트웨어 BP
- 결과: 정확한 정지점

**시나리오 3: JIT 컴파일러**
- 요구: 동적 코드 생성
- 접근: PC 조작으로 이동
- 구현: 함수 포인터/트램폴린
- 결과: 동적 실행

**시나리오 4: 펌웨어 분석**
- 요구: 실행 흐름 추적
- 접근: PC 로깅
- 구현: JTAG, ETM
- 결과: 완전한 트레이스

```
┌─────────────────────────────────────────────────────────────────────┐

│  │                                                               │ │
│  │  복원:                                                           │ │
│  │    LDR PC, [saved_context]     ; PC 복원                         │ │
```

## Ⅴ. 기대효과 및 결론

### 정량/정성 기대효과

| PC 연산 | 속도 영향 | 용도 |
|:---|:---:|:---|
| **자동 증가** | 0 사이클 | 순차 실행 |
| **분기** | 1~N 사이클 | 제어 흐름 |
| **호출** | 1~N 사이클 | 함수 호출 |
| **복귀** | 1~N 사이클 | 함수 복귀 |

### 핵심 요약

```
• PC: 다음 실행 명령어 주소 저장
• 동작: 인출 후 자동 증가, 분기 시 변경
• 분기: 조건부/무조건/호출/복귀
• 파이프라인: 분기 예측으로 버블 감소
• OS: 컨텍스트 스위칭 시 저장/복원
```
