# 165. 명령어 레지스터 (IR, Instruction Register)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 메모리에서 인출한 명령어를 저장하는 CPU 내부 레지스터
> 2. **가치**: 명령어 디코딩의 입력, 파이프라인의 첫 단계
> 3. **융합**: 제어 유닛, 명령어 디코더, 파이프라인 설계

---

## Ⅰ. 개요 (Context & Background)

### 개념 정의

명령어 레지스터(IR, Instruction Register)는 **메모리에서 인출한 명령어를 일시적으로 저장하는 CPU 내부 레지스터**다. CPU 제어 유닛이 명령어를 디코딩할 때 IR의 내용을 사용한다.

### 💡 비유: 조리법 카드

IR은 **조리법 카드**와 같다. 요리책(메모리)에서 조리법(명령어)을 찾아 카드(IR)에 옮겨 적는다. 그리고 카드를 보고 요리(실행)한다. 카드에 적힌 동안은 같은 요리를 계속 볼 수 있다.

### IR 위치와 동작

```
┌─────────────────────────────────────────────────────────────────────┐
│                IR 위치와 동작                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【CPU 내부 구조에서 IR 위치】                                       │
│  ──────────────────────────────                                     │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                          CPU                                   │ │
│  │  ┌──────────────────────────────────────────────────────────┐ │ │
│  │  │                    제어 유닛 (Control Unit)                │ │ │
│  │  │                                                          │ │ │
│  │  │   ┌───────┐    ┌───────┐    ┌──────────────────────┐    │ │ │
│  │  │   │  PC   │ →  │  IR   │ →  │   명령어 디코더       │    │ │ │
│  │  │   └───────┘    └───────┘    └──────────────────────┘    │ │ │
│  │  │       │            │                    │                │ │ │
│  │  │       │            │                    ▼                │ │ │
│  │  │       │            │            ┌──────────────┐        │ │ │
│  │  │       │            └──────────→ │  제어 신호    │        │ │ │
│  │  │       │                         └──────────────┘        │ │ │
│  │  └──────────────────────────────────────────────────────────┘ │ │
│  │                          │                                     │ │
│  │                          ▼                                     │ │
│  │  ┌──────────────────────────────────────────────────────────┐ │ │
│  │  │    ALU    │    레지스터 파일    │    데이터 경로           │ │ │
│  │  └──────────────────────────────────────────────────────────┘ │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Fetch-Decode-Execute 사이클】                                     │
│  ────────────────────────────────────                               │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  1. Fetch (인출):                                              │ │
│  │     ┌─────────────────────────────────────────────────────┐   │ │
│  │     │  MAR ← PC                     ; 주소 설정            │   │ │
│  │     │  MDR ← Memory[MAR]           ; 메모리 읽기           │   │ │
│  │     │  IR ← MDR                    ; IR에 저장             │   │ │
│  │     │  PC ← PC + 4                 ; 다음 주소              │   │ │
│  │     └─────────────────────────────────────────────────────┘   │ │
│  │                                                               │ │
│  │  2. Decode (해독):                                             │ │
│  │     ┌─────────────────────────────────────────────────────┐   │ │
│  │     │  opcode ← IR[31:26]           ; 연산 코드 추출        │   │ │
│  │     │  rs ← IR[25:21]               ; 소스 레지스터         │   │ │
│  │     │  rt ← IR[20:16]               ; 목적지 레지스터       │   │ │
│  │     │  ...                                                   │   │ │
│  │     │  제어 신호 생성                 ; 디코더 출력          │   │ │
│  │     └─────────────────────────────────────────────────────┘   │ │
│  │                                                               │ │
│  │  3. Execute (실행):                                            │ │
│  │     ┌─────────────────────────────────────────────────────┐   │ │
│  │     │  제어 신호에 따라 ALU/메모리/레지스터 동작             │   │ │
│  │     │  결과 저장/분기                                         │   │ │
│  │     └─────────────────────────────────────────────────────┘   │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  IR 특징:                                                        │ │
│  │  ──────────                                                      │ │
│  │  • 크기: 명령어 길이와 동일 (16/32/64비트)                      │ │
│  │  • 읽기: 명령어 사이클마다 자동 갱신                             │ │
│  │  • 쓰기: Fetch 단계에서만                                        │ │
│  │  • 프로그래머 접근: 불가능 (내부 레지스터)                       │ │
│  │  • 용도: 명령어 디코딩                                           │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### IR과 파이프라인

```
┌─────────────────────────────────────────────────────────────────────┐
│                IR과 파이프라인                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【단일 사이클 vs 파이프라인】                                       │
│  ──────────────────────────────                                     │
│                                                                     │
│  단일 사이클:                                                       │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  명령어 1: [Fetch] [Decode] [Execute] [Memory] [WB]           │ │
│  │                                                               │ │
│  │  명령어 2:                      [Fetch] [Decode] ...          │ │
│  │                                                               │ │
│  │  → 한 번에 하나의 명령어만 처리                                 │ │
│  │  → IR은 하나만 필요                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  파이프라인:                                                        │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  IF    ID    EX    MEM   WB                                    │ │
│  │  ────────────────────────────────────────────────────────      │ │
│  │  I1    I1    I1    I1    I1                                    │ │
│  │        I2    I2    I2    I2    I2                              │ │
│  │              I3    I3    I3    I3    I3                        │ │
│  │                    I4    I4    I4    I4    I4                  │ │
│  │                          I5    I5    I5    I5    I5            │ │
│  │                                                               │ │
│  │  → 각 단계별로 다른 명령어 처리                                 │ │
│  │  → 파이프라인 레지스터(IR/ID/EX 레지스터) 필요                  │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【파이프라인 레지스터】                                             │
│  ──────────────────────────                                         │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │  IF/ID 레지스터:                                                │ │
│  │  ┌──────────────────────────────────────────────────────────┐ │ │
│  │  │  IR (32-bit)    |  PC+4 (32-bit)                          │ │ │
│  │  └──────────────────────────────────────────────────────────┘ │ │
│  │                                                               │ │
│  │  ID/EX 레지스터:                                                │ │
│  │  ┌──────────────────────────────────────────────────────────┐ │ │
│  │  │  Control | Reg1 | Reg2 | SignExt | PC+4                   │ │ │
│  │  └──────────────────────────────────────────────────────────┘ │ │
│  │                                                               │ │
│  │  EX/MEM 레지스터:                                               │ │
│  │  ┌──────────────────────────────────────────────────────────┐ │ │
│  │  │  Control | ALU Result | Write Data | Write Reg            │ │ │
│  │  └──────────────────────────────────────────────────────────┘ │ │
│  │                                                               │ │
│  │  MEM/WB 레지스터:                                               │ │
│  │  ┌──────────────────────────────────────────────────────────┐ │ │
│  │  │  Read Data | ALU Result | Write Reg                       │ │ │
│  │  └──────────────────────────────────────────────────────────┘ │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  파이프라인에서 IR의 역할:                                       │ │
│  │  ────────────────────────                                       │ │
│  │  • IF/ID.IR: 인출된 명령어 저장                                 │ │
│  │  • ID 단계: IR을 디코딩                                         │ │
│  │  • 해저드 검출: IR 분석으로 의존성 확인                         │ │
│  │  • 분기: IR의 분기 정보로 PC 결정                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

### IR과 명령어 디코더

```
┌─────────────────────────────────────────────────────────────────────┐
│                IR과 명령어 디코더                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【명령어 디코딩 과정】                                              │
│  ──────────────────────────                                         │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                               │ │
│  │                  IR (32-bit)                                   │ │
│  │  ┌──────────────────────────────────────────────────────────┐ │ │
│  │  │ 31    26 25    21 20    16 15    11 10     6 5        0  │ │ │
│  │  │┌────────┬────────┬────────┬────────┬────────┬────────┐ │ │ │
│  │  ││ opcode │   rs   │   rt   │   rd   │ shamt  │ funct  │ │ │ │
│  │  │└────────┴────────┴────────┴────────┴────────┴────────┘ │ │ │
│  │  └──────────────────────────────────────────────────────────┘ │ │
│  │         │         │        │        │        │        │        │ │
│  │         ▼         ▼        ▼        ▼        ▼        ▼        │ │
│  │  ┌──────────────────────────────────────────────────────────┐ │ │
│  │  │                    명령어 디코더                            │ │ │
│  │  │  ─────────────────────────────                              │ │ │
│  │  │  • Opcode 디코딩 → 연산 종류                                │ │ │
│  │  │  • rs, rt, rd → 레지스터 주소                               │ │ │
│  │  │  • shamt → 시프트 양                                        │ │ │
│  │  │  • funct → 세부 연산 (R-type)                               │ │ │
│  │  └──────────────────────────────────────────────────────────┘ │ │
│  │                          │                                     │ │
│  │                          ▼                                     │ │
│  │  ┌──────────────────────────────────────────────────────────┐ │ │
│  │  │                    제어 신호 생성                           │ │ │
│  │  │  ───────────────────────                                   │ │ │
│  │  │  • RegDst: rd 또는 rt 선택                                  │ │ │
│  │  │  • ALUSrc: 레지스터 또는 즉시값                              │ │ │
│  │  │  • MemtoReg: ALU 또는 메모리                                │ │ │
│  │  │  • RegWrite: 레지스터 쓰기                                  │ │ │
│  │  │  • MemRead: 메모리 읽기                                     │ │ │
│  │  │  • MemWrite: 메모리 쓰기                                    │ │ │
│  │  │  • Branch: 분기 여부                                        │ │ │
│  │  │  • ALUOp: ALU 연산 선택                                     │ │ │
│  │  └──────────────────────────────────────────────────────────┘ │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  IR → 제어 신호 예시:                                           │ │
│  │  ────────────────────                                           │ │
│  │  IR = ADD R1,R2,R3                                              │ │
│  │  → RegDst=1, ALUSrc=0, RegWrite=1, ALUOp=ADD                   │ │
│  │                                                               │ │
│  │  IR = LW R1,0(R2)                                               │ │
│  │  → RegDst=0, ALUSrc=1, RegWrite=1, MemRead=1, MemtoReg=1       │ │
│  │                                                               │ │
│  │  IR = SW R1,0(R2)                                               │ │
│  │  → ALUSrc=1, MemWrite=1                                        │ │
│  │                                                               │ │
│  │  IR = BEQ R1,R2,label                                           │ │
│  │  → Branch=1, ALUOp=SUB                                         │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

```
│                                                                     │

│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  IR 관련 해저드 검출:                                            │ │
│  │  ────────────────────────                                       │ │
│  │  데이터 해저드:                                                  │ │

│  │                                                               │ │
│  │  제어 해저드:                                                    │ │
│  │  • IF/ID.IR이 분기 명령어 → 분기 예측 필요                      │ │
│  │                                                               │ │
│  │  구조 해저드:                                                    │ │

│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오

**시나리오 1: CPU 설계 검증**
- 요구: 명령어 디코딩 정확성
- 접근: IR → 제어 신호 검증
- 구현: 시뮬레이션
- 결과: 정확한 동작 보장

**시나리오 2: 파이프라인 최적화**
- 요구: 처리량 향상
- 접근: IR 전달 최적화
- 구현: 포워딩 경로 추가
- 결과: CPI 감소

**시나리오 3: 에뮬레이터 개발**
- 요구: 명령어 실행
- 접근: IR 기반 디코딩
- 구현: 소프트웨어 디코더
- 결과: 정확한 에뮬레이션

**시나리오 4: 디버거 개발**
- 요구: 명령어 추적
- 접근: IR 값 표시
- 구현: JTAG/소프트웨어
- 결과: 디버깅 가능

### IR 관련 디버깅

```
┌─────────────────────────────────────────────────────────────────────┐
│                IR 관련 디버깅                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【시뮬레이션 로그】                                                 │
│  ──────────────────────                                             │
│                                                                     │
│  Cycle  PC       IR          Decoded                                │
│  ─────  ──────── ──────────  ─────────────────────                  │
│  1      0x1000   0x3C011000  LUI $1, 0x1000                         │
│  2      0x1004   0x34210004  ORI $1, $1, 0x0004                     │
│  3      0x1008   0x8C220000  LW $2, 0($1)                           │
│  4      0x100C   0x8C230004  LW $3, 4($1)                           │
│  5      0x1010   0x00431020  ADD $2, $2, $3                         │
│  6      0x1014   0xAC220008  SW $2, 8($1)                           │
│                                                                     │
│  【파이프라인 추적】                                                 │
│  ──────────────────────                                             │
│                                                                     │
│  Cycle  IF/ID.IR    ID/EX.IR     EX/MEM.IR   MEM/WB.IR             │
│  ─────  ──────────  ──────────  ──────────  ──────────              │
│  1      0x3C011000  -           -           -                       │
│  2      0x34210004  0x3C011000  -           -                       │
│  3      0x8C220000  0x34210004  0x3C011000  -                       │
│  4      0x8C230004  0x8C220000  0x34210004  0x3C011000              │
│  5      0x00431020  0x8C230004  0x8C220000  0x34210004              │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  IR 디버깅 체크리스트:                                           │ │
│  │  ────────────────────────                                       │ │
│  │  □ IR 값이 올바른 명령어인가?                                    │ │
│  │  □ opcode가 유효한가?                                           │ │
│  │  □ 레지스터 필드가 범위 내인가?                                  │ │
│  │  □ 디코딩된 제어 신호가 맞는가?                                  │ │
│  │  □ 파이프라인에서 IR 전달이 올바른가?                           │ │
│  │  □ 해저드 검출이 작동하는가?                                     │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 정량/정성 기대효과

| 요소 | IR의 역할 |
|:---|:---|
| **Fetch** | 명령어 저장 |
| **Decode** | 디코딩 입력 |
| **Pipeline** | 단계간 전달 |
| **Hazard** | 의존성 검출 |

### 핵심 요약

```
• IR: 인출된 명령어를 저장하는 레지스터
• 동작: Fetch → Decode → Execute 사이클
• 파이프라인: IF/ID 레지스터로 전달
• 디코딩: IR 필드 → 제어 신호
• 프로그래머 접근: 불가능 (내부)
```
