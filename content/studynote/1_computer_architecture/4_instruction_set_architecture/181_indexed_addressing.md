> **핵심 인사이트**
> 1. 인덱스 주소 지정(Indexed Addressing)은 명령어의 오퍼랜드 필드에 **기준 주소(Base Address)**를 두고, 인덱스 레지스터에 저장된 **인덱스 값(Offset)**을 더하여 유효 주소(EA)를 산출하는 방식이다.
> 2. 주로 **배열(Array)이나 문자열(String)과 같이 연속된 메모리 공간에 저장된 데이터 구조를 순차적으로 탐색**하고 처리하는 데 특화되어 있다.
> 3. 루프(Loop) 문과 결합하여 인덱스 레지스터의 값을 1(또는 데이터 크기)씩 자동 증가(Auto-increment)시킴으로써, 컴파일러 최적화와 반복문 실행 성능을 극대화한다.

---

## Ⅰ. 인덱스 주소 지정의 개념과 메커니즘

인덱스 주소 지정은 변위 주소 지정(Displacement Addressing)의 파생 형태 중 하나로, 수식으로는 $EA = (Index Register) + Base$ 로 표현됩니다. 
베이스 레지스터 주소 지정과 덧셈 구조는 같지만, '무엇이 고정되어 있고 무엇이 변하는가'의 역할이 반대입니다.

- **Base (오퍼랜드 필드 상수):** 배열의 시작 주소가 보통 명령어 내부에 상수로 하드코딩됩니다.
- **Index Register (변동 값):** CPU 내부의 전용 인덱스 레지스터(Index Register) 혹은 범용 레지스터(GPR)에 현재 반복문의 `i` 값(예: 0, 1, 2...)이 저장됩니다.

```ascii
[인덱스 주소 지정 방식의 구조]

+------------------+---------+-----------------------+
| Opcode (예: LOAD)| Index R | Base (예: 주소 1000)  |
+------------------+---------+-----------------------+
                       |                |
    [ Index Register ] |                |
    | (i = 3)          |                |
    +------------------+                |
                |                       |
                v                       v
            [ + ] (ALU에서 3 + 1000 연산 수행)
                  \           /
                   \         /
                    v       v
                [ EA = 1003 ] ---> [ 메인 메모리 참조 ]
```

📢 **섹션 요약 비유:**
책(배열)의 10페이지(베이스)를 펴놓고, 손가락(인덱스 레지스터)으로 첫 번째 줄, 두 번째 줄, 세 번째 줄을 차례대로 짚어가며 글을 읽는 것과 같습니다.

## Ⅱ. 배열(Array) 처리를 위한 하드웨어 최적화

고급 프로그래밍 언어에서 `array[i]`를 처리할 때, 컴파일러는 이 인덱스 주소 지정을 사용하여 가장 효율적인 기계어를 생성합니다.

1. **연속된 메모리 접근:**
   - 100개의 배열 요소를 처리하려면 100개의 주소가 필요합니다. 직접 주소 지정(Direct)을 쓴다면 100줄의 `LOAD` 명령어가 필요합니다.
   - 인덱스 주소 지정을 쓰면 `LOAD R1, array(R_index)` 명령어 1줄과 루프(Loop)문만 있으면 됩니다. 루프를 한 번 돌 때마다 `R_index` 값만 1씩 증가시키면 다음 배열 요소를 자동으로 가리키게 됩니다.

2. **자동 증가/감소 (Auto-increment / Auto-decrement):**
   - 일부 CISC 구조나 고급 DSP(Digital Signal Processor)에서는 데이터를 읽어온 직후 하드웨어적으로 인덱스 레지스터의 값을 자동으로 1(또는 워드 크기) 증가시켜 주는 기능을 제공합니다.
   - 이를 통해 인덱스 값을 증가시키는 별도의 ALU 명령어(예: `ADD R_index, 1`)를 파이프라인에서 완전히 제거할 수 있습니다.

📢 **섹션 요약 비유:**
수건 접기 공장에서 기계가 수건 1번(베이스) 자리에 서서, 접고 나면 자동으로 컨베이어 벨트가 한 칸 옆으로 이동(자동 증가)하여 2번, 3번을 계속 접게 해주는 원리입니다.

## Ⅲ. 베이스-인덱스 주소 지정 (Base-Indexed Addressing)

현대의 복잡한 자료구조(예: 2차원 배열, 객체의 배열)를 지원하기 위해, 베이스 방식과 인덱스 방식을 합친 **베이스-인덱스 주소 지정**이 사용됩니다.

- **수식:** $EA = (Base Register) + (Index Register) + Displacement$
- **활용 예시 (2차원 배열 `Matrix[i][j]`):**
  - Base Register: 2차원 배열의 전체 시작 주소
  - Index Register: `i`행으로 가기 위한 오프셋
  - Displacement: `j`열로 가기 위한 상대 거리
- x86 아키텍처에서 이러한 복합 주소 지정 방식을 적극적으로 지원하여, 1개의 명령어로 복잡한 배열 요소에 접근할 수 있게 합니다.

📢 **섹션 요약 비유:**
"A동 건물(베이스 레지스터)의 3층(인덱스 레지스터)에 있는 2번째 방(변위)"을 한 번의 지시로 찾아내는 매우 정교한 길 찾기 시스템입니다.

## Ⅳ. 스케일드 인덱스 (Scaled Indexed Addressing)

데이터의 크기(Byte, Half-word, Word)에 따라 인덱스 값이 실제 메모리 주소 상에서 건너뛰어야 하는 보폭(Stride)이 다릅니다.

- 정수형(`int`) 배열은 보통 1개당 4바이트(32비트)를 차지합니다. 따라서 `array[i]`에서 `i`가 1 증가할 때, 실제 물리적 메모리 주소는 1이 아니라 4가 증가해야 합니다.
- **스케일 팩터(Scale Factor):** 하드웨어가 인덱스 레지스터 값에 자동으로 데이터 크기(x1, x2, x4, x8)를 곱해주는 기능입니다.
- **수식:** $EA = Base + (Index Register \times Scale)$

📢 **섹션 요약 비유:**
징검다리를 건널 때, 아기(1바이트)는 한 칸씩 뛰지만, 어른(4바이트 int)은 보폭이 커서 한 번에 네 칸씩(x4 스케일) 성큼성큼 뛰도록 시스템이 보폭을 자동으로 조절해 주는 것입니다.
