> **핵심 인사이트**
> 1. 산술 연산 명령어(Arithmetic Instruction)는 CPU의 심장인 **ALU(산술 논리 연산 장치)를 구동하여 덧셈, 뺄셈, 곱셈, 나눗셈 등의 수학적 계산을 직접 수행**하는 명령어이다.
> 2. 정수(Integer) 연산과 부동소수점(Floating-Point) 연산으로 엄격히 구분되며, 두 데이터 타입은 구조가 완전히 다르기 때문에 **처리하는 하드웨어 유닛(ALU vs FPU)도 물리적으로 분리**되어 있다.
> 3. 연산 결과에 따라 **상태 레지스터(Status Register/Flags)**의 조건 비트(Zero, Carry, Overflow, Sign)가 자동으로 세팅되어, 이후의 제어 흐름 분기(Branch) 명령어가 동작하는 기준을 제공한다.

---

## Ⅰ. 산술 연산 명령어의 본질과 종류

산술 연산 명령어는 컴퓨터가 단순한 데이터 저장고를 넘어 '계산기(Computer)'로서의 본연의 임무를 수행하게 하는 핵심 명령어 그룹입니다. 데이터가 이미 레지스터에 적재(Load)되어 있다고 가정하고, 그 데이터들을 조합하여 새로운 결과를 창출합니다.

### 1. 기본 산술 명령어
- **ADD (덧셈):** `ADD R1, R2, R3` (R2 + R3 결과를 R1에 저장)
- **SUB (뺄셈):** `SUB R1, R2, R3` (R2 - R3 결과를 R1에 저장). 내부적으로는 2의 보수(2's Complement)를 이용한 덧셈 회로를 재사용합니다.
- **MUL/DIV (곱셈/나눗셈):** 기본 연산에 비해 처리 사이클(Latency)이 수 배~수십 배 길며, 현대 프로세서에서는 파이프라인 지연을 막기 위해 전용 하드웨어 곱셈기(Multiplier)를 사용합니다.

### 2. 단항 산술 명령어
- **INC (증가):** 레지스터 값을 1 증가. `i++` 구현에 사용.
- **DEC (감소):** 레지스터 값을 1 감소. `i--` 구현에 사용.
- **NEG (부호 반전):** 양수를 음수로, 음수를 양수로 변환(2의 보수화).

📢 **섹션 요약 비유:**
요리사가 도마 위에 올려진 두 가지 재료(레지스터)를 칼로 다지고 볶아서(ALU) 완전히 새로운 하나의 요리(결과값)로 만들어내는 실제 요리 과정입니다.

## Ⅱ. 정수 연산 vs 부동소수점 연산

컴퓨터 하드웨어는 데이터를 0과 1로만 보지만, 프로그래머가 그 데이터를 어떻게 해석하느냐에 따라 연산 명령어와 회로가 완벽히 분리됩니다.

1. **정수 연산 (Integer Arithmetic):**
   - 명령어 예: `ADD`, `SUB`, `MUL`
   - **처리 장치:** 일반적인 ALU (Arithmetic Logic Unit).
   - **특징:** 구조가 단순하고 처리 속도가 1사이클 내외로 매우 빠릅니다. 배열 인덱스 계산, 메모리 주소 계산, 반복문 제어 등 프로그램 제어의 근간이 됩니다.

2. **부동소수점 연산 (Floating-Point Arithmetic):**
   - 명령어 예: `FADD`, `FSUB`, `FMUL` (보통 명령어 앞에 F가 붙음)
   - **처리 장치:** 전용 FPU (Floating-Point Unit) 또는 코프로세서.
   - **특징:** 가수부(Mantissa)와 지수부(Exponent)를 분리하여 지수를 맞추고 가수를 더한 뒤 다시 정규화(Normalization)하는 등 복잡한 과정을 거칩니다. 과학 계산, 3D 그래픽, 머신러닝 연산의 핵심입니다.

📢 **섹션 요약 비유:**
정수 연산은 100원, 200원 딱 떨어지는 동전을 세는 빠른 계산원이고, 부동소수점 연산은 소수점 아래 10자리까지 따져야 하는 복잡한 환율 계산을 전담하는 전문 회계사입니다.

## Ⅲ. 상태 레지스터 (Status Register / Flags)와의 상호작용

산술 연산이 끝난 후 그 결과값 자체도 중요하지만, 그 결과가 가진 **'상태 특성'**이 CPU의 상태 레지스터(또는 플래그 레지스터)에 즉시 기록된다는 점이 아키텍처적으로 매우 중요합니다.

- **Zero Flag (Z):** 연산 결과가 0일 때 '1'로 세팅됩니다. (주로 `if (a == b)`를 검사하기 위해 `SUB a, b`를 실행한 후 Z 플래그를 확인합니다.)
- **Sign Flag (S / N):** 결과값의 MSB(최상위 비트)가 1일 때(음수일 때) 세팅됩니다.
- **Carry Flag (C):** 덧셈 시 최상위 비트 밖으로 올림수(Carry)가 발생했거나, 부호 없는 정수의 범위를 넘었을 때 세팅됩니다.
- **Overflow Flag (O / V):** 부호 있는 정수 연산에서 논리적인 한계 범위를 초과하여 부호가 뒤집히는 심각한 오류가 발생했을 때 세팅됩니다.

📢 **섹션 요약 비유:**
요리가 끝난 후 요리사가 결과물에 대해 "이 요리는 싱겁다(Zero)", "이 요리는 맵다(Sign)", "냄비가 넘쳤다(Overflow)"라는 상태표(플래그)에 체크 표시를 해두면, 다음 지시자가 그걸 보고 행동을 결정하는 것과 같습니다.

## Ⅳ. 병렬 산술 연산: SIMD (Single Instruction Multiple Data)

최신 컴퓨터 구조에서는 멀티미디어 데이터나 AI 행렬 계산을 빠르게 처리하기 위해, 한 번의 산술 명령어로 여러 개의 데이터를 동시에 계산하는 SIMD 아키텍처(예: Intel AVX, ARM NEON)를 지원합니다.

- **개념:** 아주 넓은 레지스터(예: 256비트, 512비트)에 32비트 정수 8개나 16개를 팩(Pack) 형태로 담아두고, 단 한 번의 `ADD` 명령어로 16개의 덧셈을 동시에 병렬로 수행합니다.
- **효과:** 이미지의 픽셀 밝기 조절이나, 딥러닝 벡터 내적(Dot Product) 연산 등에서 압도적인(수배~수십 배) 성능 향상(Speedup)을 가져옵니다.

📢 **섹션 요약 비유:**
예전에는 사과 8개를 자를 때 칼질을 8번 해야 했지만(스칼라 연산), SIMD는 칼날이 8개 달린 특수 채칼을 한 번만 내리쳐서 순식간에 사과 8개를 조각내는 마법입니다.

## 📌 관련 개념 맵 (Knowledge Graph)
- `[ALU (Arithmetic Logic Unit)]`: 산술 명령어를 물리적으로 실행하는 논리 회로의 집합
- `[2's Complement (2의 보수)]`: 덧셈기로 뺄셈을 처리하기 위한 음수 표현 방식
- `[FPU / IEEE 754]`: 부동소수점 연산 전용 하드웨어와 데이터 표준 규격
- `[Condition Flags (플래그 레지스터)]`: 산술 연산의 특성(Z, C, V, N)을 저장하여 조건 분기의 근거를 제공하는 레지스터
- `[SIMD / Vector Processor]`: 다중 데이터 산술 연산을 한 클럭에 처리하는 가속기 구조

## 👶 어린이를 위한 3줄 비유 설명
1. 산술 연산 명령어는 컴퓨터 두뇌 속의 아주 빠른 '수학 천재 요리사'예요.
2. 숫자들을 가져다가 더하기(+), 빼기(-), 곱하기(*)를 순식간에 해치워서 정답을 만들어 낸답니다.
3. 계산이 끝나면 "정답이 0이다!", "숫자가 너무 커서 넘쳤다!" 같은 신호(플래그)를 남겨서 컴퓨터가 다음 행동을 할 수 있게 도와줘요!
