> **핵심 인사이트**
> 1. 조건부 분기(Conditional Branch)는 CPU가 이전 산술/논리 연산의 결과로 세팅된 **상태 레지스터(Status Flags)를 검사하여, 특정 조건이 참일 때만 실행 흐름(PC 값)을 변경**하는 명령어이다.
> 2. 프로그램이 외부 입력이나 데이터의 상태에 따라 **'판단'을 내리고 다른 경로를 선택**할 수 있게 해주는 논리적 지능의 원천이다.
> 3. 파이프라인 구조에서는 조건이 참인지 거짓인지 판단되기 전까지 다음 명령어를 확정할 수 없으므로, 성능 저하를 막기 위한 **분기 예측(Branch Prediction)** 기술이 필수적으로 요구된다.

---

## Ⅰ. 조건부 분기의 작동 메커니즘

조건부 분기는 혼자서 작동하지 않고, 반드시 그보다 앞서 실행된 산술 연산(주로 `CMP`나 `TEST`)의 결과에 의존합니다.

1. **조건 생성 (Condition Generation):**
   - 명령어 예: `CMP R1, R2` (Compare)
   - CPU는 내부적으로 `R1 - R2`를 계산합니다. 연산 결과는 저장하지 않지만, 그 결과의 특성(0인지, 음수인지 등)에 따라 상태 레지스터의 플래그(Z, N, C, V 등)를 업데이트합니다.
2. **조건 평가 및 점프 (Condition Evaluation & Branch):**
   - 명령어 예: `BEQ Target_Address` (Branch if Equal)
   - CPU는 앞선 연산이 남긴 `Z 플래그`를 확인합니다. 만약 `Z == 1`이면(두 값이 같았으면), PC(프로그램 카운터)에 `Target_Address`를 덮어써서 실행 흐름을 옮깁니다. 만약 `Z == 0`이면 PC를 원래대로(현재 명령어 길이만큼만) 증가시킵니다.

📢 **섹션 요약 비유:**
앞서 걸어간 정찰병(CMP)이 "앞에 절벽이 있다(플래그 세팅)"라고 깃발을 꽂아두면, 뒤따라오던 본대(분기 명령어)가 그 깃발을 보고 우회로로 갈지 원래 길로 직진할지를 결정하는 것과 같습니다.

---

## Ⅱ. 주요 조건 플래그와 분기 명령어 매핑

상태 레지스터의 4대 핵심 플래그(Z, N, C, V)의 조합에 따라 다양한 조건부 분기 명령어가 파생됩니다.

| 어셈블리 명령어 | 의미 (고급 언어 매핑) | 검사하는 상태 플래그 조건 |
| :--- | :--- | :--- |
| **BEQ / JZ** | 같다 (`a == b`) / 결과가 0이다 | `Z = 1` |
| **BNE / JNZ** | 다르다 (`a != b`) / 결과가 0이 아니다 | `Z = 0` |
| **BGT / JG** | 크다 (`a > b` - 부호 있는 정수) | `Z = 0` AND `N = V` |
| **BLT / JL** | 작다 (`a < b` - 부호 있는 정수) | `N != V` |
| **BGE / JGE** | 크거나 같다 (`a >= b`) | `N = V` |

*※ 아키텍처(ARM, x86, MIPS 등)에 따라 니모닉(명령어 이름)은 다를 수 있으나, 플래그를 검사하는 논리적 기반은 동일합니다.*

📢 **섹션 요약 비유:**
조건부 분기 명령어는 의사의 처방전과 같습니다. "체온(Z 플래그)이 38도 이상(조건)이면 해열제를 먹고(점프), 아니면 그냥 주무세요(직진)."

## Ⅲ. 고급 언어 제어문과의 변환 (Compiler Mapping)

컴파일러는 우리가 작성하는 복잡한 논리 제어문을 아주 단순한 조건부 분기 명령어로 쪼개어 번역합니다.

**1. If-Else 문:**
```c
if (a == b) {
    x = 1;
} else {
    x = 2;
}
```
$\downarrow$ (어셈블리 변환)
```assembly
    CMP R_a, R_b       ; a와 b 비교
    BNE Else_Block     ; 다르면(Z=0) Else로 점프
    MOV R_x, 1         ; 같으면(If 블록) x에 1 대입
    JMP End_Block      ; If 블록 끝났으니 무조건 분기로 Else 건너뜀
Else_Block:
    MOV R_x, 2         ; x에 2 대입
End_Block:
```

**2. While / For 루프:**
루프의 끝에서 조건을 검사하고, 참이면 루프의 시작점 주소로 되돌아가는(Backward) 조건부 분기를 사용합니다.

📢 **섹션 요약 비유:**
우리가 쓰는 세련된 글(고급 언어)을, 기계는 오직 "같냐? 다르면 저기로 가라"라는 아주 원초적이고 무식한 질문(어셈블리) 수십 개로 번역해서 이해합니다.

## Ⅳ. 아키텍처 최적화 기법: 조건부 실행 (Conditional Execution)

조건부 분기가 많아지면 파이프라인 해저드가 심해집니다. 이를 피하기 위해 ARM 같은 프로세서는 **조건부 실행(Predication)**이라는 기법을 도입했습니다.

- **개념:** 분기(Branch) 명령어를 없애버리고, 대신 모든 일반 산술/데이터 전송 명령어 뒤에 조건을 붙이는 방식입니다.
- **예시:** `MOVEQ R1, R2` (만약 앞선 비교에서 Z=1이면, 즉 Equal이면 R2를 R1에 복사하라. 아니면 명령어를 무시하라.)
- **효과:** PC 값을 건드리지 않으므로 파이프라인이 멈추지(Stall) 않습니다. 코드가 짧은 `if` 문을 처리할 때 매우 강력한 성능 향상을 가져옵니다.

📢 **섹션 요약 비유:**
"비가 오면 창고로 가서(분기) 우산을 챙겨라" 대신에, 아예 직원에게 우산을 쥐여주고 "비 올 때만 펴라(조건부 실행)"라고 지시하여 동선을 절약하는 기술입니다.

## 📌 관련 개념 맵 (Knowledge Graph)
- `[Status Register / Flags]`: 조건부 분기의 판단 근거가 되는 상태 보관소
- `[Program Counter (PC)]`: 조건이 참일 때 변경되는 실행 포인터
- `[Control Hazard]`: 조건부 분기가 참/거짓을 결정하기 전까지 파이프라인을 멈추게 하는 병목 현상
- `[Branch Prediction]`: 제어 해저드를 막기 위해 조건부 분기의 결과를 과거 통계로 미리 찍어맞히는 기술
- `[Predication (조건부 실행)]`: 분기 명령어 자체를 없애고 일반 명령어에 조건을 달아 해저드를 원천 차단하는 기법
