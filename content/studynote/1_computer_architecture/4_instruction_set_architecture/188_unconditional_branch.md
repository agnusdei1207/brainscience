> **핵심 인사이트**
> 1. 무조건 분기(Unconditional Branch, Jump)는 상태 레지스터의 **플래그(조건)를 전혀 검사하지 않고, 오퍼랜드에 지정된 타겟 주소로 프로그램 카운터(PC)를 즉시 강제 변경**하는 명령어이다.
> 2. 고급 언어의 `goto`, 루프문의 반복 지점 복귀, `switch/case` 문의 점프 테이블 탐색 등에 필수적으로 사용되며, 제어 흐름의 단절을 만들어낸다.
> 3. 무조건 분기 명령어 자체는 조건 검사 시간이 없어 빠르지만, 타겟 주소가 너무 멀 경우(Far Jump) 명령어 길이의 제약 때문에 **레지스터 간접 주소 지정 방식** 등 다양한 우회 기법이 결합되어 사용된다.

---

## Ⅰ. 무조건 분기의 작동 원리와 구조

무조건 분기 명령어는 수식이나 조건 판단 없이 오직 "저기로 가라"는 명령만을 수행합니다. 
주로 `JUMP` (또는 약어 `JMP`), ARM 계열에서는 `B` (Branch)라는 니모닉을 사용합니다.

- **작동 과정:** 
  1. CPU가 `JMP Target` 명령어를 인출(Fetch)합니다.
  2. 오퍼랜드 필드에 있는 타겟 주소(또는 변위)를 해독합니다.
  3. 현재 PC 레지스터의 값을 해당 타겟 주소로 즉시 덮어씁니다.
  4. 다음 클럭 사이클부터는 새롭게 덮어써진 타겟 주소에서 명령어를 읽어 들입니다.

📢 **섹션 요약 비유:**
조건부 분기가 "파란불이면 건너라"라면, 무조건 분기는 앞뒤 안 보고 눈 감은 채 "일단 반대편으로 뛰어라!"라고 명령하는 것과 같습니다.

## Ⅱ. 무조건 분기의 3가지 점프 방식 (Addressing)

목적지의 거리가 얼마나 먼가에 따라 하드웨어적으로 다른 주소 지정 방식을 사용합니다.

1. **상대 점프 (Relative Jump / Short Jump):**
   - 현재 PC 값을 기준으로 "앞으로 N칸, 뒤로 N칸" 이동하는 방식입니다 (PC 상대 주소 지정).
   - 명령어 내부에 작은 변위(Offset)만 기록하면 되므로 명령어 길이가 짧습니다. (예: 루프의 끝에서 처음으로 되돌아갈 때)
2. **절대 점프 (Absolute Jump / Far Jump):**
   - 메모리의 특정 물리적/논리적 주소로 한 번에 이동하는 방식입니다 (직접 주소 지정).
   - 명령어의 오퍼랜드 필드 전체를 주소 기록에 다 써야 하므로, 가변 길이 명령어를 지원하는 CISC 아키텍처에서 주로 사용됩니다.
3. **간접 점프 (Indirect Jump):**
   - 점프할 목적지 주소가 레지스터나 메모리 안의 특정 공간에 들어 있는 방식입니다. (예: `JMP (R1)`)
   - `switch/case` 문처럼 실행 도중 동적으로 목적지가 바뀌어야 할 때나, 객체지향 프로그래밍의 다형성(가상 함수 테이블, vtable)을 구현할 때 핵심적으로 쓰입니다.

📢 **섹션 요약 비유:**
옆 동네로 갈 때는 자전거(상대 점프)를 타고, 특정 도시의 주소로 갈 때는 기차(절대 점프)를 타며, 목적지가 어딘지 모른 채 "비서(레지스터)가 표 끊어주는 곳"으로 갈 때는 간접 점프를 합니다.

## Ⅲ. 고급 언어 구조와의 매핑 (Compiler Translation)

무조건 분기는 컴파일러가 제어 구조의 '뼈대'를 맞출 때 없어서는 안 될 접착제입니다.

- **If-Else 구조의 마무리:**
  `if` 블록의 코드가 다 끝나면, `else` 블록의 코드가 실행되지 않도록 건너뛰어야 합니다. 이때 `if` 블록 맨 마지막에 무조건 `JMP End` 명령어가 삽입됩니다.
- **Break / Continue:**
  루프(`for`, `while`)를 즉시 탈출하거나, 다음 반복으로 건너뛸 때 무조건 분기를 통해 루프의 바깥(또는 루프의 시작점)으로 점프합니다.
- **Switch 문 (Jump Table):**
  `case`가 아주 많은 경우, 컴파일러는 `if-else`를 수십 번 반복하지 않고 점프할 주소들을 모아놓은 배열(Jump Table)을 만듭니다. 그리고 입력값 `N`에 따라 배열의 `N`번째 주소를 읽어 무조건 간접 점프를 수행하여 속도를 극도로 높입니다. `O(N)`의 탐색 시간을 `O(1)`로 줄이는 마법입니다.

📢 **섹션 요약 비유:**
무조건 분기는 끊어진 길과 길 사이를 이어주는 다리나 순간이동 포탈과 같아서, 프로그램의 복잡한 설계도를 하나의 부드러운 흐름으로 연결해 줍니다.

## Ⅳ. 파이프라인 성능에 미치는 영향

조건부 분기와 마찬가지로 무조건 분기도 파이프라인에서 **제어 해저드(Control Hazard)**를 유발합니다. 

- **조건 판단이 없는데 왜 해저드가 발생할까?**
  명령어가 인출(Fetch)되고 해독(Decode)되어 PC 값이 타겟 주소로 덮어써지기 전까지는 (보통 1~2사이클 소요) CPU가 다음 목적지를 알지 못합니다. 그 짧은 순간 동안 파이프라인은 엉뚱한 명령어(점프 바로 밑에 있는 명령어)를 읽어 들이게 되고, 점프가 확정되면 이를 폐기(Flush)해야 합니다.
- **해결책 (Branch Target Buffer, BTB):**
  현대 CPU는 무조건 분기의 타겟 주소를 BTB에 캐싱해 둡니다. 명령어가 Fetch되자마자 이 명령어가 무조건 점프라는 것과 타겟 주소를 즉시 알아채고, 파이프라인의 낭비(Stall)를 0으로 만들어 버립니다.

📢 **섹션 요약 비유:**
아무리 무조건 뛰어내려야 한다고 해도 절벽 가장자리(해독 단계)에 서서 바닥을 한 번 쳐다보고 뛰면 시간이 지체됩니다. 그래서 아예 지도(BTB)에 "여긴 점프 구간"이라고 적어두고 멈춤 없이 곧바로 뛰어내리는 기술을 씁니다.

## 📌 관련 개념 맵 (Knowledge Graph)
- `[Program Counter (PC)]`: 무조건 분기에 의해 직접적으로 조작되는 레지스터
- `[Conditional Branch]`: 무조건 분기와 짝을 이루어 복잡한 제어 로직을 완성하는 명령어
- `[Jump Table]`: 무조건 간접 분기를 활용하여 `switch` 문을 $O(1)$로 최적화하는 자료구조
- `[Branch Target Buffer (BTB)]`: 점프 목적지를 미리 캐싱하여 제어 해저드를 없애는 예측 하드웨어
- `[Register Indirect Addressing]`: 타겟 주소가 멀거나 동적으로 변할 때 무조건 분기가 사용하는 주소 지정 방식
