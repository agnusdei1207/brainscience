# 감사 (Auditing)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 시스템 활동을 기록하고 추적하여 보안 사건을 조사하고 규정 준수를 확인하는 절차
> 2. **가치**: 침해 사고 시 포렌식 데이터를 제공하고 책임 소재를 명확히 함
> 3. **융합**: 커널 이벤트, 시스템 콜, 파일 시스템, 네트워크의 통합 로깅

---

### Ⅰ. 개요 (Context & Background)

**개념**
감사(Auditing)는 **시스템에서 발생하는 모든 보안 관련 이벤트를 기록, 검토, 분석하는 프로세스**이다. 사용자 로그인, 권한 변경, 파일 접근, 시스템 콜, 네트워크 연결 등 다양한 이벤트를 로깅하며, Linux Auditd, Windows Event Log, Syslog 등의 시스템이 사용된다.


**💡 비유**
감사는 **건물의 CCTV와 방문 기록부**와 같다.
- CCTV: 무엇이 일어났는지 기록
- 방문 기록부: 누가, 언제, 무엇을 했는지
- 목적: 사고 조사, 책임 소재 파악

**등장 배경**
① **1970년**: UNIX syslog 도입
② **1990년대**: C2 Security(Common Criteria) 요구
③ **2000년대**: 리눅스 Auditd, SIEM 등장
④ **현재**: AI 기반 로그 분석, 자동화된 탐지

**해설**
1. **auditctl**: 감사 규칙 설정 도구
2. **libaudit**: 애플리케이션 라이브러리
3. **auditd**: 메인 데몬, 로그 수집/전송
4. **Audit 서브시스템**: 커널 내 이벤트 필터링

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**심층 기술 비교: 감사 시스템**

| 시스템 | 플랫폼 | 장점 | 단점 | 주요 용도 |
|:---|:---|:---|:---|:---|
| **auditd** | Linux | 커널 레벨, 상세 | 설정 복잡 | 보안 감사 |
| **syslog** | Unix | 간단, 표준 | 제한된 정보 | 일반 로그 |
| **Event Log** | Windows | 통합, GUI | 포맷 다름 | 윈도우 감사 |
| **SIEM** | Cross | 분석 강력 | 비용 비쌈 | 엔터프라이즈 |
| **ELK Stack** | Cross | 오픈소스 | 운영 복잡 | 로그 분석 |

**과목 융합 관점**

1. **운영체제**: 커널, 시스템 콜, 파일 시스템
2. **보안**: IDS/IPS, 포렌식, 규정 준수
3. **네트워크**: 원지 로깅, 로그 포워딩

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**실무 시나리오**

**시나리오 1: auditd 설정**
```bash
# auditd 설치
sudo apt install auditd

# 감사 규칙 추가
sudo auditctl -w /etc/passwd -p wa -k identity
sudo auditctl -w /etc/shadow -p wa -k identity

# 로그 확인
sudo ausearch -k identity
```

**시나리오 2: 로그 분석**
```bash
# 실패한 로그인 시도 확인
sudo ausearch -m LOGIN -ts today

# sudo 사용 확인
sudo ausearch -c sudo -ts today

# 파일 삭제 확인
sudo ausearch -m rmdir -m unlink -ts today
```

**도입 체크리스트**

**기술적 측면**
- [ ] auditd 설치/활성화
- [ ] 중요 파일/디렉터리 감시
- [ ] 시스템 콜 감사
- [ ] 네트워크 연결 감시
- [ ] 로그 파일 보호 (쓰기 금지)

**운영·보안적 측면**
- [ ] 로그 정기 검토
- [ ] 이상 탐지 규칙
- [ ] 로그 보관 정책 (최소 90일)
- [ ] 원지 로그 백업
- [ ] 규정 준수 보고

**안티패턴**
1. **"다 기록하면 돼"**: 스토리지, 성능 문제
2. **"로그 확인 안 해도 돼"**: 무용지물
3. **"기본 설정 OK"**: 필요한 이벤트 누락

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**
