# 멀티태스킹 (Multitasking) 주요 용어 및 개념 비교

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: CPU가 여러 작업을 번갈아 가며 실행하여 동시에 수행되는 것처럼 보이게 하거나(동시성), 실제로 여러 코어를 사용해 동시에 실행하는(병렬성) 운영체제의 **다중 작업 관리 능력**.
> 2. **가치**: 자원의 이용률을 극대화하고 사용자에게 멀티미디어 환경 및 대화형 인터페이스를 제공하는 현대 컴퓨팅의 가장 기본적인 패러다임이다.
> 3. **융합**: 하드웨어의 멀티코어 설계, 운영체제의 스케줄링 알고리즘, 그리고 프로그래밍 언어의 비동기 처리 모델이 집약된 시스템 공학의 결정체.

---

### Ⅰ. 멀티태스킹 핵심 용어 정의 (Glossary)

멀티태스킹을 이해하기 위해 반드시 구분해야 하는 4대 유사 개념이다.

| 용어 | 정의 | 핵심 키워드 | 비유 |
|:---|:---|:---|:---|
| **다중 프로그래밍** | 여러 프로그램을 메모리에 적재 | CPU 이용률 극대화 | 요리 냄비 여러 개 올리기 |
| **멀티태스킹** | 여러 작업을 시분할로 번갈아 수행 | **응답 시간** 최소화 | 요리사가 냄비들을 번갈아 젓기 |
| **다중 처리** | 여러 개의 CPU(코어)를 사용 | **병렬 실행 (Parallelism)** | 요리사 여러 명이 함께 요리하기 |
| **시분할 시스템** | 멀티태스킹의 논리적 확장 | 사용자 간 **자원 공유** | 손님들이 요리를 한 입씩 나눠먹기 |

---

### Ⅱ. 동시성(Concurrency) vs 병렬성(Parallelism)

멀티태스킹의 물리적/논리적 구현 차이를 나타내는 핵심 개념이다.

#### 1. 개념도 (ASCII)

```ascii
    [ Concurrency (동시성) ]           [ Parallelism (병렬성) ]
    * Single Core, Interleaved         * Multi Core, Simultaneous
    
    Task A: --  --  --                 Task A: ----------
    Task B:   --  --  --               Task B: ----------
    
    (동시에 실행되는 것처럼 보임)       (실제로 동시에 실행됨)
```

#### 2. 상세 비교

| 구분 | 동시성 (Concurrency) | 병렬성 (Parallelism) |
|:---|:---|:---|
| **논리/물리** | 논리적인 개념 (구조적 설계) | 물리적인 개념 (하드웨어 실행) |
| **목표** | 여러 작업을 동시에 '다루는' 것 | 여러 작업을 동시에 '수행하는' 것 |
| **자원** | 단일 코어에서도 가능 (시분할) | 반드시 멀티 코어/프로세서 필요 |
| **핵심 기술** | 문맥 교환 (Context Switch) | 하드웨어 스케줄링, 데이터 분할 |

### Ⅲ. 멀티태스킹의 핵심 메커니즘 (Mechanism)

#### 1. 문맥 교환 (Context Switch)
- 한 작업에서 다른 작업으로 CPU를 넘길 때, 현재 상태(Registers, PC, Stack Pointer 등)를 저장하고 새 작업의 상태를 복구하는 과정.
- 멀티태스킹의 '유연함'을 제공하지만, 빈번할 경우 성능 저하(Overhead)의 원인이 됨.

#### 2. 선점형(Preemptive) vs 비선점형(Non-preemptive)
- **선점형**: OS가 강제로 CPU를 뺏어올 수 있음 (현대 모든 범용 OS).
- **비선점형**: 작업이 스스로 CPU를 반납할 때까지 기다림 (과거 Windows 3.1 등).

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

#### 1. 개발 시 고려사항 (Thread-safety)
- 멀티태스킹 환경에서는 여러 작업이 공유 자원에 동시에 접근하므로 **경쟁 상태(Race Condition)**가 발생할 수 있다.
- **기술사적 결단**: 뮤텍스(Mutex), 세마포어(Semaphore) 등을 사용하여 임계 구역(Critical Section)을 보호하거나, 함수형 프로그래밍의 불변성(Immutability)을 활용하여 원천적으로 차단한다.

#### 2. 기술사적 인사이트
- **암달의 법칙 (Amdahl's Law)**: 아무리 병렬성을 높여도(코어를 늘려도), 프로그램 내부에 병렬화할 수 없는 순차적 부분이 있다면 성능 향상에는 한계가 있다.
- **처리량 vs 응답성**: 실시간 시스템(RTOS)에서는 전체 처리량보다 특정 작업의 응답 시간(Latency) 보장이 멀티태스킹의 최우선 목표가 된다.

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

#### 1. 정량/정성 기대효과
- **사용자 경험 극대화**: 음악을 들으며 웹 서핑을 하고 문서를 작성하는 멀티미디어 환경 제공.
- **시스템 투자 가치 증대**: 비싼 서버 하드웨어 자원을 놀리지 않고 100% 가깝게 활용.

#### 2. 미래 전망
미래의 멀티태스킹은 **이기종 컴퓨팅(Heterogeneous Computing)**으로 진화하고 있다. CPU뿐만 아니라 GPU, NPU, DSP 등 다양한 연산 장치에 작업을 동적으로 할당하는 '범용 멀티태스킹 스케줄러'가 운영체제의 핵심 경쟁력이 될 것이다.
