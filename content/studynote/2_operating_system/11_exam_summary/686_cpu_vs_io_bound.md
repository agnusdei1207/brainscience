# CPU 바운드 vs I/O 바운드 프로세스 특성 비교

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 프로세스의 실행 주기 동안 **연산(Computation)**에 시간을 더 많이 쓰는지, 아니면 **입출력(I/O)** 대기에 시간을 더 많이 쓰는지에 따른 분류.
> 2. **가치**: 프로세스의 성격을 파악하여 CPU 스케줄러가 누구에게 우선순위를 줄지, 어떤 시간 할당량(Quantum)을 부여할지 결정하는 핵심 근거가 된다.
> 3. **융합**: 현대 시스템은 두 성격이 혼재된 **작업 혼합(Job Mix)** 상태로 운영되며, I/O 바운드 작업에 높은 우선순위를 주어 시스템의 전체적인 반응성(Responsiveness)을 높인다.

---

### Ⅰ. 개념 및 상세 비교

| 구분 항목 | CPU 바운드 (CPU Bound) | I/O 바운드 (I/O Bound) |
|:---|:---|:---|
| **주요 활동** | 복잡한 수치 계산, 데이터 인코딩 | 파일 읽기/쓰기, 네트워크 통신 |
| **버스트 특성** | **긴 CPU 버스트**, 짧은 I/O 버스트 | **짧은 CPU 버스트**, 긴 I/O 버스트 |
| **시간 점유** | CPU 연산 장치 점유가 주 목적 | 주변 장치(Disk, NIC) 응답 대기가 주 목적 |
| **예시** | 비디오 렌더링, 암호화, 과학 연산 | 텍스트 에디터, 웹 서버, DB 조회 |

---

### Ⅱ. CPU-I/O 버스트 사이클 아키텍처 (ASCII)

프로세스는 실행 중 CPU 연산과 I/O 대기를 번갈아 가며 수행한다.

```ascii
    [ CPU Bound Process ]
    +---------------------------+     +--+     +---------------------------+
    |       CPU Burst (Long)    | --> |IO| --> |       CPU Burst (Long)    |
    +---------------------------+     +--+     +---------------------------+
    
    [ I/O Bound Process ]
    +--+     +---------------------------+     +--+     +---------------------------+
    |CP| --> |      I/O Burst (Long)     | --> |CP| --> |      I/O Burst (Long)     |
    +--+     +---------------------------+     +--+     +---------------------------+
```

---

### Ⅲ. 스케줄링 전략 및 기술사적 판단 (Strategy)

#### 1. 스케줄러의 딜레마
- **CPU 바운드 중심**: 처리량(Throughput)은 좋아지지만, I/O 바운드 작업(예: 마우스 클릭)의 응답 속도가 현저히 느려져 사용자가 불만을 느낀다.
- **I/O 바운드 중심**: 응답성(Response Time)은 좋아지지만, 잦은 문맥 교환으로 인해 전체 연산 효율이 떨어질 수 있다.

#### 2. 기술사적 결단: I/O 바운드 우선 정책 (I/O Priority)
- I/O 바운드 프로세스는 CPU를 아주 잠깐만 쓰고 바로 I/O를 위해 CPU를 반납한다. 
- 따라서 이들에게 먼저 CPU를 주면 사용자는 "빠르다"고 느끼면서도, CPU 바운드 작업의 총 실행 시간에는 큰 지장을 주지 않는다. 현대의 **MLFQ(다단계 피드백 큐)** 스케줄러는 이 원리를 이용하여 I/O 바운드 작업의 우선순위를 자동으로 높인다.

---

### Ⅳ. 기대효과 및 결론 (Future & Standard)

#### 1. 정량/정성 기대효과
- **자원 이용률 균형**: CPU와 I/O 장치가 동시에 바쁘게 돌아가는 최적의 상태 달성.
- **체감 응답성 향상**: 대화형 인터페이스(UI)의 부드러운 작동 보장.

#### 2. 미래 전망
최근에는 **메모리 바운드(Memory Bound)** 프로세스라는 개념이 추가되고 있다. 연산 속도보다 메모리 대역폭(Memory Bandwidth)이 병목이 되는 빅데이터나 AI 모델 학습 환경에서, 운영체제는 메모리 스톨(Stall) 시간을 줄이기 위한 새로운 스케줄링 기법을 도입하고 있다.

### 👶 어린이를 위한 3줄 비유 설명
1. **CPU 바운드**는 어려운 수학 문제를 하루 종일 푸는 **'공부 벌레'** 친구와 같아요.
2. **I/O 바운드**는 친구들에게 간식을 나눠주러 돌아다니는 **'심부름꾼'** 친구와 같죠.
3. 선생님(OS)은 심부름꾼 친구가 빨리 일을 끝내게 도와주고, 남는 시간에 공부 벌레 친구가 집중할 수 있게 자리를 마련해 준답니다!
