# 선점 (Preemptive) vs 비선점 (Non-preemptive) 스케줄링 차이

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 현재 CPU를 점유 중인 프로세스로부터 **운영체제가 강제로 자원을 뺏어올 수 있는지(선점)**, 아니면 프로세스가 **스스로 반납할 때까지 기다려야 하는지(비선점)**의 권한 경계 차이.
> 2. **가치**: 선점형은 대화형 시스템의 **응답성**을 보장하며, 비선점형은 문맥 교환 오버헤드가 적고 **일괄 처리** 효율이 높다.
> 3. **융합**: 현대의 모든 범용 운영체제(Linux, Windows 등)는 사용자 경험과 시스템 보호를 위해 **선점형 스케줄링**을 표준으로 채택하고 있다.

---

### Ⅰ. 개념 및 상세 비교

| 구분 항목 | 선점형 스케줄링 (Preemptive) | 비선점형 스케줄링 (Non-preemptive) |
|:---|:---|:---|
| **제어권** | OS가 강제로 CPU 점유를 중단시킴 | 프로세스가 종료되거나 I/O 대기 시에만 반납 |
| **응답성** | 매우 빠름 (높은 우선순위 즉시 실행) | 느림 (앞 작업이 끝날 때까지 대기) |
| **오버헤드** | 높음 (잦은 문맥 교환 발생) | 낮음 (최소한의 문맥 교환) |
| **안정성** | 높음 (무한 루프 프로세스 강제 중단 가능) | 낮음 (하나의 프로세스가 시스템 독점 가능) |
| **대표 알고리즘** | RR, SRTF, MLFQ | FCFS, SJF, HRN |

---

### Ⅱ. 스케줄링 발생 시점 아키텍처 (ASCII)

프로세스의 상태 전이에 따라 스케줄링이 발생하는 시점이 다르다.

```ascii
    [ Running ] --(1)--> [ Waiting ] (I/O call)  : Non-preemptive Possible
    [ Running ] --(2)--> [ Ready ] (Interrupt)   : Preemptive ONLY
    [ Waiting ] --(3)--> [ Ready ] (I/O complete) : Preemptive ONLY
    [ Running ] --(4)--> [ Terminated ] (Finish)  : Non-preemptive Possible
```

- **비선점형**: (1), (4) 상황에서만 다음 프로세스를 선택한다.
- **선점형**: (1), (2), (3), (4) 모든 상황에서 스케줄러가 개입할 수 있다. 특히 (2) 타임 슬라이스 종료나 (3) 더 높은 우선순위 프로세스 등장 시 즉각 개입한다.

---

### Ⅲ. 장단점 및 기술사적 판단 (Strategy & Decision)

#### 1. 선점형 스케줄링의 딜레마
- **장점**: 시분할 시스템 구현이 가능하며, 특정 앱의 고착 상태가 시스템 전체로 번지지 않는다.
- **단점**: 공유 데이터에 접근하는 도중 CPU를 뺏기면 **데이터 불일치(Race Condition)**가 발생할 수 있어, 세마포어 등 복잡한 동기화 메커니즘이 필수적이다.

#### 2. 기술사적 인사이트: 커널 선점 (Kernel Preemption)
- **과거**: 많은 Unix 계열은 사용자 모드는 선점형이었지만, 커널 모드 코드가 실행 중일 때는 비선점형이었다.
- **현재**: 리눅스 2.6 버전 이후로는 **Fully Preemptive Kernel** 옵션을 제공하여, 커널 내부 작업을 수행 중이라도 실시간 작업이 오면 즉시 비우고 비켜주는 구조로 진화했다.

---

### Ⅳ. 기대효과 및 결론 (Future & Standard)

#### 1. 정량/정성 기대효과
- **가용성 증대**: 시스템 자원의 공평한 분배와 독점 방지.
- **멀티미디어 보장**: 비디오/오디오 스트리밍 등 실시간성이 필요한 앱의 끊김 없는 재생.

#### 2. 미래 전망
클라우드 가상화 환경에서는 하이퍼바이저가 게스트 OS를 선점하는 **VCPU 스케줄링**이 핵심 기술이다. 하드웨어 가속 기술과 결합하여 선점 시 발생하는 문맥 교환 비용을 '제로'에 가깝게 줄이는 연구가 지속되고 있다.

### 👶 어린이를 위한 3줄 비유 설명
1. **비선점형**은 화장실 줄 서기와 같아요. 앞사람이 일을 다 보고 나올 때까지 아무리 급해도 밖에서 기다려야 하죠.
2. **선점형**은 놀이터 시소 타기와 같아요. 선생님이 "자, 1분 지났으니 다음 친구에게 양보하자!" 하고 강제로 자리를 바꿔주는 거예요.
