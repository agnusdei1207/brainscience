# I/O 풀링 (Polling) 오버헤드

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: CPU가 장치 상태를 반복적으로 확인하는 방식의 I/O 처리
> 2. **가치**: 구현 단순, 인터럽트 오버헤드 없음, 저속 장치에 적합
> 3. **융합**: 인터럽트, DMA, 버스와 통합된 I/O 모델

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

풀링(Polling)은 CPU가 장치의 상태를 반복적으로 확인하여 I/O를 처리하는 방식입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    I/O 풀링 (Polling)                                │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              풀링 방식                                         │ │
│   │                                                              │ │
│   │   CPU                       장치                             │ │
│   │    │                         │                               │ │
│   │    │───상태 확인(poll)──────→│                               │ │
│   │    │←──준비 안 됨─────────────┤                               │ │
│   │    │                         │                               │ │
│   │    │───상태 확인(poll)──────→│                               │ │
│   │    │←──준비 안 됨─────────────┤      Busy Wait               │ │
│   │    │                         │      (바쁜 대기)              │ │
│   │    │───상태 확인(poll)──────→│                               │ │
│   │    │←──준비 됨!───────────────┤                               │ │
│   │    │                         │                               │ │
│   │    │───데이터 읽기──────────→│                               │ │
│   │    │←──데이터 반환───────────┤                               │ │
│   │    │                         │                               │ │
│   │   특징: CPU가 계속 확인 (busy waiting)                       │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              장단점 비교                                       │ │
│   │                                                              │ │
│   │   장점:                                                       │ │
│   │   • 구현이 단순함                                             │ │
│   │   • 인터럽트 컨텍스트 스위치 오버헤드 없음                    │ │
│   │   • 응답 시간 예측 가능                                       │ │
│   │                                                              │ │
│   │   단점:                                                       │ │
│   │   • CPU 사이클 낭비 (busy waiting)                           │ │
│   │   • 전력 소모 증가                                            │ │
│   │   • 다른 작업 지연                                            │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: 풀링은 단순하지만 CPU를 낭비합니다.

**💡 비유**: 풀링은 택배가 올 때까지 계속 현관문을 확인하는 것과 같습니다.

**📢 섹션 요약 비유**: 계속 확인하는 게 풀링!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **Status Reg** | 상태 확인 | Ready 비트 | 깜빡이 |
| **Loop** | 반복 확인 | while 문 | 기다림 |
| **Timeout** | 제한 | 무한 루프 방지 | 시간 제한 |
| **CPU** | 폴링 주체 | 명령 실행 | 감시자 |

**풀링 vs 인터럽트**

| 특성 | 풀링 | 인터럽트 |
|:---|:---:|:---:|
| **CPU 사용** | 높음 | 낮음 |
| **복잡도** | 낮음 | 높음 |
| **지연** | 일정 | 변동 |
| **전력** | 높음 | 낮음 |

**핵심 알고리즘**: while(!ready); read_data();

**📢 섹션 요약 비유**: 인터럽트가 효율적!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: I/O 방식 선택**

| 상황 | 추천 방식 |
|:---|:---|
| **저속 장치** | 풀링 |
| **고속 장치** | 인터럽트 |
| **대량 전송** | DMA |
| **실시간** | 풀링 (예측 가능) |

**📢 섹션 요약 비유**: 상황에 따라 선택!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 임베디드** → 단순 장치 풀링
**시나리오 2: 고성능 서버** → 인터럽트 + DMA
**시나리오 3: 실시간 시스템** → 혼합 (Hybrid)

**도입 체크리스트**: 장치 속도, CPU 여유, 전력 예산

**📢 섹션 요약 비유**: 하이브리드가 최선!
