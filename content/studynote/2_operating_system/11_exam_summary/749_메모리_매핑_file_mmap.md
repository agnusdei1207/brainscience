# 메모리 매핑 파일 (mmap)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 파일을 메모리 주소 공간에 매핑하여 포인터로 접근하는 기법
> 2. **가치**: read/write 시스템 콜 생략, 페이지 단위 접근, 공유 메모리
> 3. **융합**: 페이지 폴트, 가상 메모리, 파일 시스템과 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

메모리 매핑 파일(mmap)은 파일을 프로세스의 주소 공간에 매핑하여 포인터로 직접 접근하는 기법입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    메모리 매핑 파일 (mmap)                           │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              전통적 파일 I/O vs mmap                          │ │
│   │                                                              │ │
│   │   전통적 방식 (read/write):                                  │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  Application                                         │   │ │
│   │   │      │                                               │   │ │
│   │   │      │ read() 시스템 콜                              │   │ │
│   │   │      ↓                                               │   │ │
│   │   │  ┌────────┐     ┌────────┐     ┌────────┐           │   │ │
│   │   │  │Buffer  │ ←── │Disk    │ ←── │File    │           │   │ │
│   │   │  │(User)  │     │Cache   │     │        │           │   │ │
│   │   │  └────────┘     └────────┘     └────────┘           │   │ │
│   │   │      │                                               │   │ │
│   │   │      │ copy ( Kernel → User )                       │   │ │
│   │   │      ↓                                               │   │ │
│   │   │   데이터 복사 오버헤드 발생                          │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   │   mmap 방식:                                                 │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  Application                                         │   │ │
│   │   │      │                                               │   │ │
│   │   │      │ ptr = mmap(file)                              │   │ │
│   │   │      │ 직접 접근: data = ptr[offset]                 │   │ │
│   │   │      │                                               │   │ │
│   │   │  ┌────────────────────────────────────────┐          │   │ │
│   │   │  │        Virtual Memory Space            │          │   │ │
│   │   │  │  ┌──────────────────────────────────┐ │          │   │ │
│   │   │  │  │   mmap region (file 매핑)        │ │          │   │ │
│   │   │  │  │   ptr[0]  ptr[1]  ptr[2]  ...    │ │          │   │ │
│   │   │  │  └──────────┬───────────────────────┘ │          │   │ │
│   │   │  └─────────────┼─────────────────────────┘          │   │ │
│   │   │                │                                     │   │ │
│   │   │                │ Page Fault 시 디스크에서 로드       │   │ │
│   │   │                ↓                                     │   │ │
│   │   │  ┌────────────────────────────────────────┐          │   │ │
│   │   │  │           File on Disk                 │          │   │ │
│   │   │  └────────────────────────────────────────┘          │   │ │
│   │   │                                                      │   │ │
│   │   │   이점: 시스템 콜 없음, 복사 없음, 페이지 단위 접근  │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: mmap은 파일을 마치 메모리처럼 접근합니다.

**💡 비유**: mmap은 파일을 메모리에 올려두고 바로 읽는 것과 같습니다.

**📢 섹션 요약 비유**: 파일이 메모리처럼!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **mmap()** | 매핑 생성 | VMA 생성 | 연결 |
| **VMA** | 가상 메모리 영역 | vm_area_struct | 구역 |
| **Page Fault** | 지연 로드 | demand paging | 필요시 |
| **msync()** | 동기화 | 디스크 반영 | 저장 |

**mmap 플래그**

| 플래그 | 설명 |
|:---|:---|
| **MAP_SHARED** | 변경 공유 |
| **MAP_PRIVATE** | COW (Copy-on-Write) |
| **MAP_ANONYMOUS** | 파일 없음 |
| **MAP_FIXED** | 주소 고정 |

**핵심 알고리즘**: mmap → 접근 → 페이지 폴트 → 파일 로드

**📢 섹션 요약 비유**: MAP_SHARED가 공유 메모리!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: I/O 방식**

| 방식 | 시스템 콜 | 복사 | 용도 |
|:---|:---:|:---:|:---|
| **read/write** | 많음 | 있음 | 일반 |
| **mmap** | 적음 | 없음 | 대용량 |
| **sendfile** | 적음 | 없음 | 전송 |
| **DMA** | 없음 | 없음 | 고속 |

**📢 섹션 요약 비유**: mmap이 대용량 파일에 효율적!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: DB 버퍼** → mmap 인덱스
**시나리오 2: 공유 메모리** → MAP_SHARED
**시나리오 3: 실행 파일** → 로더

**도입 체크리스트**: 파일 크기, 접근 패턴, 동기화

**📢 섹션 요약 비유**: DB는 mmap을 많이 써요!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 시스템 콜 90% 감소

**미래 전망**: DAX (Direct Access), persistent memory

**참고 표준**: POSIX mmap, Linux VMA

**📢 섹션 요약 비유**: mmap은 제로카피 I/O!
