# 쓰기 시 복사 (COW)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터를 복사하지 않고 공유하다가, 쓰기 시점에 복사하는 최적화 기법
> 2. **가치**: 메모리 절약, fork 최적화, 페이지 공유
> 3. **융합**: 가상 메모리, 페이지 테이블, 스냅샷과 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

쓰기 시 복사(COW, Copy-on-Write)는 데이터를 복사하지 않고 공유하다가, 쓰기가 발생할 때 비로소 복사하는 최적화 기법입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    쓰기 시 복사 (Copy-on-Write)                      │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              fork()와 COW                                     │ │
│   │                                                              │ │
│   │   fork() 직후:                                               │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │   부모 프로세스           자식 프로세스              │   │ │
│   │   │   Page Table              Page Table                 │   │ │
│   │   │   ┌─────────┐             ┌─────────┐               │   │ │
│   │   │   │ P1 → ───┼─────┬──────→│ P1 → ───┤               │   │ │
│   │   │   │ P2 → ───┼─────┤      →│ P2 → ───┤               │   │ │
│   │   │   │ P3 → ───┼─────┘      →│ P3 → ───┤               │   │ │
│   │   │   └─────────┘     │        └─────────┘               │   │ │
│   │   │                   │                                   │   │ │
│   │   │                   ↓                                   │   │ │
│   │   │   ┌─────────────────────────────────────┐            │   │ │
│   │   │   │         Physical Pages (공유)        │            │   │ │
│   │   │   │   [Page1] [Page2] [Page3]           │            │   │ │
│   │   │   │   (Read-Only로 매핑)                │            │   │ │
│   │   │   └─────────────────────────────────────┘            │   │ │
│   │   │                                                      │   │ │
│   │   │   이점: 메모리 복사 없음, 페이지 테이블만 복사        │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   │   자식이 P2에 쓰기 시도:                                     │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │                                                      │   │ │
│   │   │   부모 프로세스           자식 프로세스              │   │ │
│   │   │   Page Table              Page Table                 │   │ │
│   │   │   ┌─────────┐             ┌─────────┐               │   │ │
│   │   │   │ P1 → ───┼─────┬──────→│ P1 → ───┤               │   │ │
│   │   │   │ P2 → ───┼──┐  │       │ P2 → ───┼─→ 새 페이지   │   │ │
│   │   │   │ P3 → ───┼──┼──┴──────→│ P3 → ───┤               │   │ │
│   │   │   └─────────┘  │          └─────────┘               │   │ │
│   │   │                │                                     │   │ │
│   │   │   ┌────────────┴───────────────────────┐            │   │ │
│   │   │   │         Physical Pages              │            │   │ │
│   │   │   │   [Page1] [Page2] [Page3] [Page2'] │            │   │ │
│   │   │   │                  ↑      ↑          │            │   │ │
│   │   │   │               부모    자식(복사)    │            │   │ │
│   │   │   └────────────────────────────────────┘            │   │ │
│   │   │                                                      │   │ │
│   │   │   쓰기 시: Page Fault → 복사 → 쓰기 허용             │   │ │
│   │   │                                                      │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: COW는 필요할 때만 복사합니다.

**💡 비유**: COW는 글을 쓸 때만 새 종이를 쓰는 것과 같습니다.

**📢 섹션 요약 비유**: 쓸 때만 복사해요!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **Read-Only PT** | 공유 표시 | WP 비트 | 잠금 |
| **Page Fault** | 쓰기 감지 | WP 위반 | 알림 |
| **Copy Page** | 실제 복사 | 새 프레임 | 복제 |
| **Update PT** | 매핑 변경 | 쓰기 가능 | 잠금 해제 |

**COW 활용 사례**

| 사례 | 설명 |
|:---|:---|
| **fork()** | 프로세스 복사 |
| **mmap PRIVATE** | 파일 매핑 |
| **스냅샷** | VM 스냅샷 |
| **ZFS** | 파일 시스템 |

**핵심 알고리즘**: 쓰기 → PF → 복사 → PT 갱신 → 재시도

**📢 섹션 요약 비유**: fork가 COW 대표 예시!
