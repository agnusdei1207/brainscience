# SMP 캐시 일관성 / 폴스 셰어링

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 다중 코어 시스템에서 각 코어의 캐시 데이터 일관성 유지
> 2. **가치**: 데이터 정합성, 성능 최적화, MESI 프로토콜 이해
> 3. **융합**: 캐시 계층, 버스 스누핑, 메모리 모델과 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

SMP 캐시 일관성(Cache Coherence)은 다중 코어 시스템에서 각 코어의 캐시 데이터가 동일하도록 유지하는 메커니즘입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    SMP 캐시 일관성                                   │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              캐시 일관성 문제                                  │ │
│   │                                                              │ │
│   │   CPU 0              CPU 1             Memory               │ │
│   │   ┌──────┐          ┌──────┐          ┌──────┐             │ │
│   │   │Cache │          │Cache │          │      │             │ │
│   │   │x = 10│          │x = 10│          │ x=10 │             │ │
│   │   └──┬───┘          └──┬───┘          └──────┘             │ │
│   │      │                 │                                   │ │
│   │      │ x = 20 쓰기     │                                   │ │
│   │      ↓                 │                                   │ │
│   │   ┌──────┐          ┌──────┐          ┌──────┐             │ │
│   │   │Cache │          │Cache │          │      │             │ │
│   │   │x = 20│          │x = 10│ ✗        │ x=?  │             │ │
│   │   └──────┘          └──────┘          └──────┘             │ │
│   │      ↑                                                     │ │
│   │   문제: CPU 1은 여전히 구 데이터 사용                       │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              MESI 프로토콜 (캐시 일관성)                       │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  Modified (M): 수정됨, 내 캐시에만 존재              │   │ │
│   │   │  Exclusive (E): 독점, 다른 캐시에 없음               │   │ │
│   │   │  Shared (S): 공유, 여러 캐시에 있음                  │   │ │
│   │   │  Invalid (I): 무효, 사용 불가                        │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   │   상태 전이:                                                 │ │
│   │   I → E (읽기, 다른 캐시에 없음)                            │ │
│   │   I → S (읽기, 다른 캐시에 있음)                            │ │
│   │   E → M (쓰기)                                              │ │
│   │   S → I (다른 코어가 쓰기)                                  │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              False Sharing (폴스 셰어링)                      │ │
│   │                                                              │ │
│   │   ┌─────────────────────────────────────────────────────┐   │ │
│   │   │  Cache Line (64 bytes)                               │   │ │
│   │   │  ┌───────────────────────────────────────────────┐  │   │ │
│   │   │  │  var A  │  var B  │  var C  │  var D  │ ...  │  │   │ │
│   │   │  └────┬────┴────┬────┴─────────┴─────────┴──────┘  │   │ │
│   │   │       │         │                                   │   │ │
│   │   │    CPU 0       CPU 1                                 │   │ │
│   │   │   (A 사용)    (B 사용)                               │   │ │
│   │   │                                                      │   │ │
│   │   │   문제: A, B가 같은 캐시 라인 → 불필요한 캐시 무효화  │   │ │
│   │   │   해결: 패딩으로 분리, alignas(64)                   │   │ │
│   │   └─────────────────────────────────────────────────────┘   │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: 캐시 일관성은 MESI 프로토콜로 해결합니다.

**💡 비유**: 캐시 일관성은 여러 사람이 같은 문서를 볼 때 버전을 맞추는 것과 같습니다.

**📢 섹션 요약 비유**: MESI가 캐시 일관성 표준!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **MESI** | 일관성 프로토콜 | 상태 머신 | 규칙 |
| **Bus Snooping** | 버스 감시 | 변경 감지 | 감시 |
| **Cache Line** | 전송 단위 | 64 bytes | 묶음 |
| **Invalidation** | 무효화 | I 상태 | 폐기 |

**False Sharing 해결**

| 기법 | 설명 |
|:---|:---|
| **Padding** | 변수 사이 빈 공간 |
| **Align** | 캐시 라인 정렬 |
| **Separate** | 변수 분리 |

**핵심 알고리즘**: 쓰기 → 무효화 브로드캐스트 → 다른 캐시 I 상태

**📢 섹션 요약 비유**: False Sharing은 성능 저하의 주범!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 일관성 프로토콜**

| 프로토콜 | 상태 수 | 복잡도 |
|:---|:---:|:---:|
| **MSI** | 3 | 낮음 |
| **MESI** | 4 | 중간 |
| **MOESI** | 5 | 높음 |
| **MESIF** | 5 | 높음 |

**📢 섹션 요약 비유**: MESI가 가장 널리 쓰여요!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 멀티스레드** → False Sharing 방지
