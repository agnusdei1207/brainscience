# 인터럽트 구동 입출력

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 장치가 준비되면 CPU에 인터럽트하여 I/O를 처리하는 방식
> 2. **가치**: CPU 효율성 향상, 이벤트 기반 처리, 비동기 I/O
> 3. **융합**: 인터럽트 컨트롤러, ISR, DMA와 통합된 I/O

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

인터럽트 구동 입출력(Interrupt-Driven I/O)은 장치가 준비되면 CPU에 인터럽트를 걸어 I/O를 처리하는 방식입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    인터럽트 구동 입출력                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              폴링 vs 인터럽트                                  │ │
│   │                                                              │ │
│   │   Polling:                                                   │ │
│   │   CPU                        Device                          │ │
│   │    │                           │                             │ │
│   │    │──준비됨?────────────────→│                             │ │
│   │    │←─아니오──────────────────│                             │ │
│   │    │──준비됨?────────────────→│                             │ │
│   │    │←─아니오──────────────────│  CPU busy waiting           │ │
│   │    │──준비됨?────────────────→│                             │ │
│   │    │←─예!─────────────────────│                             │ │
│   │    │──데이터 읽기────────────→│                             │ │
│   │                                                              │ │
│   │   Interrupt:                                                 │ │
│   │   CPU                        Device                          │ │
│   │    │                           │                             │ │
│   │    │──다른 작업 수행──         │                             │ │
│   │    │         ┌─────────────────┤                             │ │
│   │    │         │   인터럽트!     │                             │ │
│   │    │←────────┴─(준비 완료)────┤                             │ │
│   │    │──ISR 실행 (데이터 처리)──│                             │ │
│   │    │──다른 작업 재개──         │                             │ │
│   │                                                              │ │
│   │   이점: CPU가 다른 작업 가능                                  │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              인터럽트 처리 과정                                │ │
│   │                                                              │ │
│   │   1. 장치가 인터럽트 신호 (IRQ) 전송                          │ │
│   │                      │                                       │ │
│   │                      ↓                                       │ │
│   │   2. 인터럽트 컨트롤러 (PIC/APIC)가 CPU에 전달               │ │
│   │                      │                                       │ │
│   │                      ↓                                       │ │
│   │   3. CPU가 현재 명령 완료 후 인터럽트 처리                   │ │
│   │                      │                                       │ │
│   │                      ↓                                       │ │
│   │   4. 컨텍스트 저장 (PC, Flags 등)                            │ │
│   │                      │                                       │ │
│   │                      ↓                                       │ │
│   │   5. ISR (Interrupt Service Routine) 실행                   │ │
│   │                      │                                       │ │
│   │                      ↓                                       │ │
│   │   6. 컨텍스트 복원 후 원래 코드 재개                         │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: 인터럽트는 CPU가 기다리지 않고 다른 일을 할 수 있게 합니다.

**💡 비유**: 인터럽트는 택배가 오면 알림을 받는 것과 같습니다.

**📢 섹션 요약 비유**: 알림이 오면 처리해요!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **IRQ** | 인터럽트 번호 | 고유 식별자 | 전화번호 |
| **PIC/APIC** | 컨트롤러 | 우선순위 관리 | 교환원 |
| **IDT** | 테이블 | ISR 주소 | 전화번호부 |
| **ISR** | 처리 루틴 | 실제 처리 | 상담원 |

**인터럽트 우선순위**

| 우선순위 | 유형 |
|:---|:---|
| **높음** | NMI, 하드웨어 오류 |
| **중간** | 타이머, 디스크 |
| **낮음** | 네트워크, 소프트웨어 |

**핵심 알고리즘**: IRQ → IDT 조회 → ISR 실행 → EOI

**📢 섹션 요약 비유**: IDT가 핸들러를 찾아요!

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: I/O 방식**

| 방식 | CPU 대기 | 복잡도 | 용도 |
|:---|:---:|:---:|:---|
| **Polling** | O | 낮음 | 저속 |
| **Interrupt** | X | 중간 | 일반 |
| **DMA** | X | 높음 | 대량 |

**📢 섹션 요약 비유**: 인터럽트가 범용적!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 키보드** → 인터럽트
**시나리오 2: 디스크** → 인터럽트 + DMA
**시나리오 3: 네트워크** → NAPI (인터럽트 + 폴링)

**도입 체크리스트**: ISR 길이 최소화, top/bottom half

**📢 섹션 요약 비유**: ISR은 짧게!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: CPU 활용률 50% 향상

**미래 전망**: MSI-X, 이벤트 기반 I/O

**참고 표준**: x86 IDT, ARM GIC

**📢 섹션 요약 비유**: 인터럽트는 효율적 I/O의 핵심!
