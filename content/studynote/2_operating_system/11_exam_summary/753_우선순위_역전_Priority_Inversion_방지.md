# 우선순위 역전 (Priority Inversion) 방지

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 낮은 우선순위 작업이 높은 우선순위 작업을 간접적으로 블록하는 현상
> 2. **가치**: 실시간 시스템 안정성, Mars Pathfinder 사례
> 3. **융합**: 뮤텍스, 우선순위 상속, 실시간 스케줄링과 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

우선순위 역전(Priority Inversion)은 낮은 우선순위 작업이 높은 우선순위 작업을 간접적으로 블록하는 현상입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    우선순위 역전 (Priority Inversion)                │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              우선순위: H > M > L                              │ │
│   │                                                              │ │
│   │   시나리오 (역전 발생):                                       │ │
│   │                                                              │ │
│   │   시간 ────────────────────────────────────────────────→     │ │
│   │                                                              │ │
│   │   L (낮음): ─[락 획득]──────────────────────[락 해제]─       │ │
│   │                     │                       ↑                │ │
│   │                     │                       │                │ │
│   │   M (중간):         │────[실행]─────────────│                │ │
│   │                     │      (L 선점)        │                │ │
│   │                     │                       │                │ │
│   │   H (높음):         │   [락 필요]──대기───→│                │ │
│   │                     │       ↑               │                │ │
│   │                     │       │               │                │ │
│   │                     └───────┴───────────────┘                │ │
│   │                     L이 락을 가짐    M이 CPU 점유            │ │
│   │                     H가 대기        L이 실행 못 함           │ │
│   │                                                              │ │
│   │   결과: H(높음)이 M(중간)에게 블록됨! (역전)                 │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              해결책: 우선순위 상속 (Priority Inheritance)      │ │
│   │                                                              │ │
│   │   시간 ────────────────────────────────────────────────→     │ │
│   │                                                              │ │
│   │   L (낮음): ─[락 획득]──[상속: H 우선순위]──[락 해제]─       │ │
│   │                     │              │          │              │ │
│   │                     │              │          │              │ │
│   │   M (중간):         │              │   [실행] │              │ │
│   │                     │              │          │              │ │
│   │   H (높음):         │  [락 필요]───┼──[락 획득]             │ │
│   │                     │      ↑       │          │              │ │
│   │                     │      │       │          │              │ │
│   │                     └──────┴───────┴──────────┘              │ │
│   │                     L이 락 보유 중    L이 빨리 끝남           │ │
│   │                     H 우선순위 상속    H가 락 획득            │ │
│   │                                                              │ │
│   │   결과: L이 H의 우선순위로 빨리 실행 → H 대기 최소화          │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: Mars Pathfinder에서 실제 발생한 문제입니다.

**💡 비유**: 우선순위 역전은 일반 차가 비상 차량을 막는 것과 같습니다.

**📢 섹션 요약 비유**: 상속이 해결책!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **PIP** | 우선순위 상속 | 일시 승격 | 양보 |
| **PCP** | 우선순위 천장 | 사전 예약 | 예약 |
| **Mutex** | 상속 지원 | RT Mutex | 잠금 |
| **Priority** | 우선순위 | 0~99 (RT) | 등급 |

**해결 방법 비교**

| 방법 | 설명 | 복잡도 |
|:---|:---|:---:|
| **PIP** | 락 보유 시 상속 | 중간 |
| **PCP** | 천장 할당 | 높음 |
| **DPCP** | 분산 천장 | 높음 |

**핵심 알고리즘**: H 대기 → L이 H 우선순위 상속 → 빨리 완료

**📢 섹션 요약 비유**: PIP가 가장 간단!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 락 메커니즘**

| 락 | 상속 지원 | 용도 |
|:---|:---:|:---|
| **Spinlock** | X | 짧은 구간 |
| **Mutex** | O | 일반 |
| **RT Mutex** | O | 실시간 |

**📢 섹션 요약 비유**: RT Mutex가 필수!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 실시간 OS** → PIP 활성화
**시나리오 2: 임베디드** → PCP
**시나리오 3: Mars Pathfinder** → PIP 패치

**도입 체크리스트**: rt_mutex 사용, 우선순위 설계

**📢 섹션 요약 비유**: Linux PREEMPT_RT 패치!
