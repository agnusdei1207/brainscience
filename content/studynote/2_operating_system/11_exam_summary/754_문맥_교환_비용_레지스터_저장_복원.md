# 문맥 교환 비용 (레지스터 저장/복원)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 프로세스/스레드 전환 시 레지스터 상태를 저장하고 복원하는 비용
> 2. **가치**: 성능 최적화, 스케줄링 이해, 멀티태스킹 오버헤드
> 3. **융합**: PCB, 스케줄러, 캐시 오염과 통합된 문맥 교환

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

문맥 교환 비용(Context Switch Overhead)은 프로세스/스레드 전환 시 레지스터 상태를 저장하고 복원하는 데 드는 비용입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    문맥 교환 비용                                    │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              문맥 교환 과정                                    │ │
│   │                                                              │ │
│   │   Process A                   Process B                      │ │
│   │   (실행 중)                   (대기 중)                       │ │
│   │                                                              │ │
│   │   1. 현재 상태 저장 (Save Context):                          │ │
│   │      ┌─────────────────────────────────────────────────┐    │ │
│   │      │  PCB A에 저장:                                   │    │ │
│   │      │  • General Registers (RAX, RBX, ... )           │    │ │
│   │      │  • Special Registers (RIP, RSP, RFLAGS)         │    │ │
│   │      │  • Floating Point / SIMD (XMM, YMM)             │    │ │
│   │      │  • Address Space (CR3)                          │    │ │
│   │      └─────────────────────────────────────────────────┘    │ │
│   │                                                              │ │
│   │   2. 스케줄러 실행 (Schedule):                               │ │
│   │      • 다음 프로세스 선택 (B)                                │ │
│   │                                                              │ │
│   │   3. 새 상태 복원 (Restore Context):                         │ │
│   │      ┌─────────────────────────────────────────────────┐    │ │
│   │      │  PCB B에서 복원:                                 │    │ │
│   │      │  • CR3 (페이지 테이블)                           │    │ │
│   │      │  • General Registers                            │    │ │
│   │      │  • Special Registers                            │    │ │
│   │      │  • Floating Point / SIMD                        │    │ │
│   │      └─────────────────────────────────────────────────┘    │ │
│   │                                                              │ │
│   │   4. 프로세스 B 실행 시작                                    │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              비용 요소                                        │ │
│   │                                                              │ │
│   │   직접 비용 (Direct Cost):                                   │ │
│   │   • 레지스터 저장/복원: ~100-500 cycles                      │ │
│   │   • 커널 진입/복귀: ~200-300 cycles                          │ │
│   │   • TLB 플러시 (주소 공간 변경): ~1000+ cycles               │ │
│   │                                                              │ │
│   │   간접 비용 (Indirect Cost):                                 │ │
│   │   • 캐시 오염 (Cache Pollution): ~1000+ cycles               │ │
│   │   • 브랜치 예측 실패                                         │ │
│   │   • 파이프라인 플러시                                        │ │
│   │                                                              │ │
│   │   총 비용: ~1-10 µs (프로세스), ~0.5-2 µs (스레드)           │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: 스레드 문맥 교환이 프로세스보다 저렴합니다.

**💡 비유**: 문맥 교환은 책을 덮고 다른 책을 펼치는 것과 같습니다.

**📢 섹션 요약 비유**: 스레드가 더 저렴해요!

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **PCB** | 문맥 저장 | 레지스터 | 책갈피 |
| **CR3** | 페이지 테이블 | 주소 공간 | 색인 |
| **TLB** | 주소 캐시 | 플러시 | 지도 |
| **Cache** | 데이터 캐시 | 오염 | 책상 |

**비용 비교**

| 전환 유형 | 비용 (µs) | 이유 |
|:---|:---:|:---|
| **스레드→스레드** | 0.5-2 | 주소 공간 공유 |
| **프로세스→프로세스** | 1-10 | TLB 플러시 |
| **유저→커널** | 0.1-0.5 | 시스템 콜 |

**핵심 알고리즘**: save_regs → schedule → restore_regs → iret

**📢 섹션 요약 비유**: TLB 플러시가 비싸요!

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 고성능 서버** → 스레드 풀
**시나리오 2: 실시간** → CPU 고정 (Pinning)
**시나리오 3: 컨테이너** → cgroup 스케줄링

**도입 체크리스트**: 문맥 교환 측정, 스레드 vs 프로세스

**📢 섹션 요약 비유**: perf로 측정해요!

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: 스레드 활용으로 50% 비용 절감

**미래 전망**: PCID, ASID (ARM)

**참고 표준**: x86 context switch, ARM exception

**📢 섹션 요약 비유**: 문맥 교환은 멀티태스킹의 비용!
