# 고아(Orphan) 및 좀비(Zombie) 프로세스 처리 매커니즘

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 부모 프로세스가 자식보다 먼저 종료되어 갈 곳을 잃은 **고아 프로세스**와, 실행은 끝났으나 부모가 상태를 확인해주지 않아 시스템 정보(PCB)만 남은 **좀비 프로세스**.
> 2. **가치**: 운영체제는 시스템 자원(PID, 메모리 등)의 고갈을 막기 위해, 고아 프로세스는 `init`(PID 1) 프로세스에게 입양시키고 좀비 프로세스는 부모의 `wait()` 호출을 유도하여 완벽히 제거한다.
> 3. **융합**: 프로세스의 생명주기 관리와 커널의 자원 회수 정책이 충돌하는 지점이며, 안정적인 서버 운영을 위해 반드시 이해해야 할 프로세스 계층 구조의 핵심이다.

---

### Ⅰ. 고아 프로세스 (Orphan Process)

- **정의**: 자식 프로세스가 아직 실행 중인데 부모 프로세스가 먼저 종료된 경우.
- **처리 방식 (Adoption)**:
  - 운영체제는 고아 프로세스를 방치하지 않고, 시스템의 최상위 프로세스인 **`init` (또는 `systemd`)**에게 입양시킨다.
  - `init` 프로세스는 주기적으로 자식들의 상태를 확인(`wait`)하여 고아 프로세스가 종료될 때 자원을 깨끗이 정리해준다.

---

### Ⅱ. 좀비 프로세스 (Zombie Process / Defunct)

- **정의**: 프로세스가 실행을 마치고 종료(exit)되었으나, 부모 프로세스가 아직 자식의 종료 상태를 확인(wait 시스템 콜)하지 않아 시스템 리소스(주로 PID와 PCB 일부)가 남아있는 상태.
- **발생 원인**: 부모 프로세스의 설계 오류나 바쁜 상태로 인해 `wait()` 또는 `waitpid()`를 호출하지 못할 때 발생한다.
- **해결책**:
  - 부모 프로세스가 `SIGCHLD` 시그널 핸들러에서 `wait()`를 호출하도록 수정한다.
  - 부모 프로세스를 강제로 종료하면 좀비 프로세스들은 자동으로 고아 프로세스가 되어 `init`에게 입양되고, `init`이 즉시 정리해준다.

---

### Ⅲ. 프로세스 상태 비교 아키텍처 (ASCII)

```ascii
    [ Normal Life Cycle ]
    Fork -> Exec -> Running -> Exit -> Parent waits -> [ Dead ]
    
    [ Zombie Scenario ]
    Fork -> Exec -> Running -> Exit (PCB remains) -> Parent ignores -> [ ZOMBIE ]
                                                     ^--- Missing wait()
    
    [ Orphan Scenario ]
    Fork -> Exec -> Running -> (Parent dies) -> [ ORPHAN ] -> Adopted by init (PID 1)
```

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

#### 1. 좀비 프로세스의 위험성
- **현상**: 좀비 프로세스는 메모리나 CPU를 거의 쓰지 않지만, **PID(Process ID)**를 점유한다.
- **기술사적 결단**: 시스템의 최대 PID 개수는 한정되어 있으므로, 좀비가 수천 개 쌓이면 새로운 프로세스를 생성할 수 없는 시스템 마비 상태에 빠진다. 따라서 주기적인 프로세스 모니터링(`ps -ef | grep defunct`)과 부모 프로세스의 시그널 처리 로직 검증이 필수적이다.

#### 2. 기술사적 인사이트: Double Fork 기술
- 데몬(Daemon) 프로세스를 만들 때, 의도적으로 자식을 만들고 부모를 바로 죽여서 자식을 고아(init 입양)로 만드는 기술을 쓴다. 이를 통해 부모 프로세스의 관리가 필요 없는 독립적인 배경 서비스를 구현할 수 있다.

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

#### 1. 정량/정성 기대효과
- **시스템 자원 무결성**: 사용되지 않는 PID와 커널 메모리의 즉각적인 회수.
- **서버 가용성 유지**: 좀비 누적으로 인한 프로세스 생성 실패 방지.

#### 2. 미래 전망
최근의 컨테이너 환경에서는 각 컨테이너 내부의 1번 프로세스(Entrypoint)가 `init` 역할을 제대로 수행하지 못해 좀비 프로세스가 쌓이는 문제가 자주 발생한다. 이를 위해 **tini**와 같은 경량 `init` 도구를 컨테이너에 내장하는 것이 표준이 되고 있으며, 운영체제 차원에서도 좀비 프로세스를 더 공격적으로 청소하는 지능형 가비지 컬렉션 기술이 연구되고 있다.

### 👶 어린이를 위한 3줄 비유 설명
1. **고아 프로세스**는 부모님이 먼저 집에 가버려서 길을 잃은 아이와 같아요. 다행히 인자한 경찰관 아저씨(init)가 나타나서 집까지 잘 데려다준답니다.
2. **좀비 프로세스**는 숙제를 다 끝내고 "선생님 검사해주세요!"라고 기다리는데, 선생님(부모)이 딴청을 피우느라 교실을 못 나가고 서 있는 상태예요.
3. 좀비가 너무 많아지면 교실이 꽉 차서 다른 친구들이 공부하러 들어올 수 없기 때문에, 제때제때 검사(wait)를 해주는 게 중요하답니다!
