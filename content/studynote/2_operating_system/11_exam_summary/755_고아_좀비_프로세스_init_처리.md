# 고아 / 좀비 프로세스 (init 처리)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 고아(부모 사망), 좀비(자식 종료 후 미수거) 프로세스
> 2. **가치**: 프로세스 생명주기 이해, 리소스 누수 방지
> 3. **융합**: fork/wait, 시그널, init/systemd와 통합

---

### Ⅰ. 개요 (Context & Background)

**개념 정의**

고아 프로세스(Orphan Process)는 부모가 종료된 자식 프로세스이고, 좀비 프로세스(Zombie Process)는 종료되었지만 부모가 수거하지 않은 프로세스입니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    고아 / 좀비 프로세스                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              정상적인 프로세스 종료                            │ │
│   │                                                              │ │
│   │   Parent Process                                             │ │
│   │       │                                                      │ │
│   │       │ fork()                                               │ │
│   │       ↓                                                      │ │
│   │   Child Process ──→ 실행 ──→ exit() ──→ 좀비 (Zombie)        │ │
│   │       │                                    │                  │ │
│   │       │                                    │                  │ │
│   │       └──── wait() ←───────────────────────┘                  │ │
│   │              부모가 수거 → 좀비 소멸                           │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              좀비 프로세스 (Zombie)                           │ │
│   │                                                              │ │
│   │   Parent Process                                             │ │
│   │       │                                                      │ │
│   │       │ fork()                    (wait() 안 함)             │ │
│   │       ↓                                                      │ │
│   │   Child Process ──→ 실행 ──→ exit() ──→ 좀비 (Zombie)        │ │
│   │                                             │                 │ │
│   │   부모가 wait() 호출 안 함                  │                 │ │
│   │                                             ↓                 │ │
│   │   좀비 상태로 남음 (PCB만 존재)                               │ │
│   │   ps로 보면 <defunct>로 표시                                  │ │
│   │                                                              │ │
│   │   문제: 좀비가 쌓이면 PID 고갈                                │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │              고아 프로세스 (Orphan)                           │ │
│   │                                                              │ │
│   │   Parent Process                                             │ │
│   │       │                                                      │ │
│   │       │ fork()          부모 종료 (exit)                     │ │
│   │       ↓                 ↓                                    │ │
│   │   Child Process ──→ 실행 ──→ 계속 실행                        │ │
│   │       │                  │                                   │ │
│   │       │                  │                                   │ │
│   │       │              고아가 됨                                │ │
│   │       │                  │                                   │ │
│   │       │                  ↓                                   │ │
│   │       │           init/systemd가 새 부모                     │ │
│   │       │                  │                                   │ │
│   │       └──────────────────┴──→ init가 wait() 수행             │ │
│   │                              고아도 정상 종료 가능             │ │
│   │                                                              │ │
│   └──────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

> **해설**: init/systemd가 고아를 입양합니다.

**💡 비유**: 고아는 보호자가 바뀌고, 좀비는 떠도는 유령과 같습니다.

**📢 섹션 요약 비유**: init가 고아를 입양해요!

---

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

**구성 요소 상세 분석**

| 요소명 | 역할 | 내부 동작 | 비유 |
|:---|:---|:---|:---|
| **Zombie** | 종료 대기 | PCB만 남음 | 유령 |
| **Orphan** | 부모 없음 | init 입양 | 고아 |
| **init** | 입양자 | PID 1 | 보호자 |
| **wait()** | 수거 | 상태 회수 | 장례 |

**상태 전이**

| 상태 | 설명 | PCB |
|:---|:---|:---:|
| **Running** | 실행 중 | O |
| **Zombie** | 종료, 미수거 | O |
| **Dead** | 완전 제거 | X |

**핵심 알고리즘**: exit → 좀비 → wait → dead

**📢 섹션 요약 비유**: wait()가 좀비를 없애요!

---

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

**기술 비교: 좀비 처리**

| 방법 | 설명 |
|:---|:---|
| **wait()** | 블로킹 대기 |
| **waitpid()** | 비블로킹 옵션 |
| **SIGCHLD** | 시그널 핸들러 |
| **double fork** | 즉시 분리 |

**📢 섹션 요약 비유**: double fork가 깔끔!

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

**시나리오 1: 데몬** → double fork
**시나리오 2: 셸** → waitpid() WNOHANG
**시나리오 3: 서버** → SIGCHLD 핸들러

**도입 체크리스트**: 좀비 방지, 시그널 처리

**📢 섹션 요약 비유**: SIGCHLD가 실시간 수거!

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

**정량/정성 기대효과**: PID 누수 방지

**미래 전망**: systemd, subreaper

**📢 섹션 요약 비유**: 좀비는 wait()로!
