# 002. 커널 (Kernel)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 운영체제의 핵심, 특권 모드 실행 코드
> 2. **가치**: 하드웨어 제어, 자원 관리, 서비스 제공
> 3. **융합**: 모놀리식, 마이크로, 하이브리드, 시스템 콜

---

## Ⅰ. 개요

### 개념 정의

커널(Kernel)은 **운영체제의 핵심 구성요소로, 하드웨어와 직접 상호작용하며 시스템 자원을 관리하는 특권 코드**다.

### 💡 비유: 자동차 엔진
커널은 **자동차 엔진**과 같다. 자동차가 있다. (컴퓨터) 운전자가 있다. (사용자) 엔진이 있다. (커널) 모든 것을 제어. 연료 분배. 동력 전달. 커널이 똑같다. 하드웨어 제어, 자원 관리, 서비스 제공.

### 과목 융합 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                커널 × 타 과목 융합                                        │

│  【컴퓨터 구조】                                                      │

│                                                                     │

│  • 시스템 콜: 커널 서비스 요청                                              │
│  • API: POSIX, Win32                                                      │
│  • 라이브러리: libc, glibc                                                 │
│  • 드라이버: 커널 모듈                                                      │
│                                                                     │
│  【네트워크】                                                          │

│  • 분산: RPC                                                               │
│                                                                     │
│  【보안】                                                              │
│  • 인증: 사용자 식별                                                        │
│  • 접근 제어: DAC, MAC                                                     │

│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  커널 유형별 OS:                                                                           │ │
│  │  ──────────────────────                                              │ │

│  │  ────            ────              ────                   │ │
│  │  모놀리식        Linux, Unix        성능                    │ │
│  │  마이크로        Minix, QNX         신뢰성                  │ │
│  │  하이브리드      Windows, macOS     균형                    │ │
```

## Ⅳ. 실무 적용

### 커널 분석

```
┌─────────────────────────────────────────────────────────────────────┐
│                커널 분석                                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【커널 정보 확인 (Linux)】                                                        │)
│  ──────────────────                                                  │
│  # 커널 버전                                                               │
│  $ uname -r                                                               │
│  5.15.0-91-generic                                                        │
│                                                                     │
│  # 커널 설정                                                               │
│  $ cat /boot/config-$(uname -r)                                           │
│                                                                     │
│  【커널 모듈】                                                                  │)
│  ──────────────────                                                  │
│  # 로드된 모듈                                                              │
│  $ lsmod                                                                  │
│                                                                     │
│  # 모듈 로드                                                                │
│  $ sudo modprobe <module>                                                 │
│                                                                     │
│  【커널 파라미터】                                                              │)
│  ──────────────────                                                  │
│  # 커널 파라미터 확인                                                         │
│  $ sysctl -a                                                              │
│                                                                     │
│  # 파라미터 변경                                                             │
│  $ sudo sysctl -w <param>=<value>                                         │
│                                                                     │
│  【커널 팁】                                                                    │)
│  ──────────────────                                                  │
│  1. 업데이트: 보안 패치 적용                                                  │
│  2. 모니터링: dmesg, /var/log/messages                                     │
│  3. 튜닝: sysctl로 파라미터 조정                                              │
│  4. 모듈: 필요한 모듈만 로드                                                  │
│  5. 보안: 불필요한 기능 비활성화                                               │
│  6. 백업: 커널 설정 백업                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅴ. 기대효과 및 결론

### 핵심 요약

```
• 커널: OS 핵심, 특권 모드 실행
• 모놀리식: 단일 주소 공간, 성능
• 마이크로: 최소 커널, 신뢰성
• 하이브리드: 혼합형, 균형
• 커널 모드: 특권, 전체 접근
• 사용자 모드: 비특권, 제한 접근
• 시스템 콜: 커널 서비스 인터페이스
• 장점: 추상화, 관리, 보안
• 단점: 오버헤드, 복잡성
• 예시: Linux, Windows, macOS
```
