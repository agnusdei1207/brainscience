# 176. 지수 평균법 (Exponential Averaging)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 과거의 CPU 버스트 시간과 직전의 예측치를 가중 평균하여 다음 CPU 버스트 시간을 예측하는 통계적 기법
> 2. **가치**: 최근의 추세를 더 많이 반영하면서도 과거의 이력을 완전히 배제하지 않아, SJF 스케줄링의 실행 시간 예측 문제를 해결하는 실질적인 대안
> 3. **융합**: 시계열 데이터 분석(Time Series Analysis), 주가 이동평균선 계산, 네트워크 트래픽 예측 등 동적인 변화 감지가 필요한 모든 제어 시스템의 기초 알고리즘

---

### Ⅰ. 개요 (Context & Background)

#### 개념 정의
**지수 평균법(Exponential Averaging)**은 다음 CPU 버스트(Burst) 시간을 예측하기 위해 사용되는 가중 평균 기법이다. 단순 산술 평균과 달리 최근 관측값에 더 높은 가중치를 부여하고, 과거로 갈수록 가중치를 지수적으로 감소시키는 특징이 있다.

#### 💡 비유: 일기 예보와 최근 날씨
지수 평균법은 **내일의 기온을 예측할 때 "어제의 실제 기온"과 "어제 했던 예측"을 섞어서 판단하는 것**과 같다. 만약 요즘 날씨가 급격히 변하고 있다면(가중치 $\alpha$가 높음) 어제 기온을 더 많이 참고하고, 날씨가 평소처럼 일정하다면 과거 데이터의 비중을 높게 유지한다.

#### 등장 배경
1. **SJF의 한계**: 최적의 스케줄링인 SJF를 구현하려면 다음 CPU 사용 시간을 알아야 하지만, 실제로는 실행 전까지 알 수 없음.
2. **동적 변화 대응**: 프로세스의 동작 특성은 실행 중에 변할 수 있으므로, 고정된 평균보다는 최근의 추세를 반영하는 예측이 필요함.
3. **메모리 효율성**: 모든 과거 이력을 저장할 필요 없이 직전 예측치와 현재 관측값만 있으면 되므로 자원 소모가 적음.

#### 📢 섹션 요약 비유
"마치 운전자가 전방 주시를 하면서도 백미러로 지나온 길을 힐끗 보며 속도를 조절하는 것과 같습니다."

### Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

#### 핵심 공식 및 파라미터
지수 평균법은 다음과 같은 재귀적 수식으로 표현된다.

$$\tau_{n+1} = \alpha t_n + (1 - \alpha) \tau_n$$

| 요소 | 의미 | 역할 | 비유 |
|:---:|:---|:---|:---|
| **$\tau_{n+1}$** | 다음 예측치 | 스케줄러가 결정에 사용할 값 | 내일의 예상 기온 |
| **$t_n$** | 최근 실제치 | 직전 단계에서 실제로 사용한 CPU 시간 | 오늘의 실제 기온 |
| **$\tau_n$** | 과거 예측치 | 직전 단계까지의 누적된 예측 정보 | 오늘 했던 예상 기온 |
| **$\alpha$** | 가중치 (0~1) | 최근 정보($t_n$)와 과거 정보($\tau_n$)의 비중 조절 | 정보의 신선도 필터 |

#### 지수적 가중치 감소 구조 (Weight Decay)

공식을 전개하면 과거의 관측값($t_{n-1}, t_{n-2} \dots$)들이 지수적으로 비중이 낮아짐을 알 수 있다.

```
┌─────────────────────────────────────────────────────────────────────┐
│              Exponential Averaging Weight Distribution              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  Weight (%)                                                         │
│    ^                                                                │
│    │  [α] (Recent actual: t_n)                                      │
│    │   █                                                            │
│    │   █  [α(1-α)] (t_{n-1})                                        │
│    │   █       █                                                    │
│    │   █       █  [α(1-α)²] (t_{n-2})                               │
│    │   █       █       █                                            │
│    │   █       █       █  [α(1-α)³] (t_{n-3})                       │
│    │   █       █       █       █   ...                              │
│    └────────────────────────────────────────────> Time (Past)       │
│                                                                     │
│  [Cases of α]                                                       │
│  - α = 0 : τ_{n+1} = τ_n (최근 결과 무시, 과거 이력만 신뢰)            │
│  - α = 1 : τ_{n+1} = t_n (과거 무시, 직전 결과만 반영 - 요동 심함)     │
│  - α = 0.5 : 최근과 과거를 동일하게 중시 (가장 일반적)                 │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

**다이어그램 해설**:
위 다이어그램은 시간의 흐름에 따른 데이터 비중의 변화를 시각화한 것이다. 최근 데이터인 $t_n$은 $\alpha$만큼의 가장 큰 비중을 가지며, 이전 데이터인 $t_{n-1}$은 $\alpha(1-\alpha)$로 줄어든다. 이 비중은 지수 함수적으로 급격히 감소하므로, 아주 오래된 데이터는 전체 예측에 거의 영향을 주지 않게 된다. 이를 통해 알고리즘은 **"최근의 단기적 변화"에 민감하게 반응하면서도 "장기적 평균"의 안정성을 잃지 않는 하이브리드 성격**을 갖게 된다.

#### 예측 시뮬레이션 과정
1. **초기화**: 첫 번째 예측치 $\tau_0$를 시스템 상수로 설정한다.
2. **관측**: 프로세스가 CPU를 사용하고 난 뒤 실제 시간 $t_n$을 측정한다.
3. **계산**: 공식에 대입하여 다음 타임 슬라이스 또는 SJF 우선순위 결정을 위한 $\tau_{n+1}$을 산출한다.
4. **반복**: 프로세스가 종료될 때까지 매 CPU Burst마다 업데이트를 수행한다.

#### 📢 섹션 요약 비유
"최근에 먹은 음식의 맛이 식당의 인상을 크게 좌우하지만, 수년간 쌓아온 명성(과거 이력)도 무시할 수 없는 것과 같습니다."

### Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

#### 예측 기법 비교 분석
| 비교 항목 | 단순 산술 평균 | 이동 평균 (Moving Avg) | 지수 평균 (Exp Avg) |
|:---:|:---|:---|:---|
| **반영 범위** | 전체 이력 동일 비중 | 특정 윈도우 내 데이터 | 전체 이력 지수적 반영 |
| **계산 복잡도** | 중간 (합계/개수 저장) | 높음 (윈도우 데이터 보관) | 낮음 (최근값/예측값만 필요) |
| **급격한 변화 대응** | 매우 느림 | 윈도우 크기에 따라 가변적 | $\alpha$ 값에 따라 매우 신속 |
| **적용 분야** | 정적 통계 | 단기 트래픽 분석 | CPU 스케줄링, 주가 예측 |

#### 과목 융합 관점: 인공지능 및 네트워크
- **딥러닝 옵티마이저**: Momentum이나 Adam 알고리즘에서 그래디언트(Gradient)의 지수 이동 평균(EMA)을 사용하여 학습의 진동을 줄이고 수렴 속도를 높인다.
- **TCP 재전송 타이머**: 네트워크 지연 시간(RTT)을 예측할 때 지수 평균법을 사용하여 재전송 타임아웃(RTO) 값을 동적으로 조절한다.

#### 📢 섹션 요약 비유
"단순 평균이 모든 목소리를 똑같이 듣는 민주주의라면, 지수 평균은 현장의 목소리(최근치)에 더 귀를 기울이는 전문가 자문단과 같습니다."

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

#### 실무 시나리오 및 $\alpha$ 선택 전략
1. **I/O 위주 프로세스**: CPU 사용량이 불규칙하므로 $\alpha$를 높게 설정(0.7 이상)하여 즉각적인 반응성을 확보하는 것이 유리하다.
2. **연산 위주 프로세스 (Batch)**: 일정한 패턴을 보이므로 $\alpha$를 낮게 설정(0.3 이하)하여 일시적인 노이즈에 의한 오판을 방지한다.
3. **시스템 튜닝 (Decision)**: 일반적으로 $\alpha = 0.5$가 최적의 균형점으로 추천되며, 시스템 부하가 높을 때는 예측 오버헤드를 줄이기 위해 업데이트 주기를 조절하기도 한다.

#### 도입 체크리스트
- [ ] 데이터가 시계열적 특성을 가지고 있는가?
- [ ] 최근의 변화 추세가 미래 예측에 중요한가?
- [ ] 과거 모든 데이터를 저장하기에 메모리가 부족한가?
- [ ] $\alpha$ 값을 동적으로 튜닝할 기전이 있는가?

#### 📢 섹션 요약 비유
"지수 평균법은 무거운 역사책을 다 들고 다니지 않고도 과거의 교훈을 현재의 판단에 녹여내는 영리한 메모 전략입니다."

### Ⅴ. 기대효과 및 결론 (Future & Standard)

#### 정량/정성 기대효과 (표)
| 구분 | 도입 전 (고정 시간 할당) | 도입 후 (지수 평균 기반 SJF) |
|:---:|:---|:---|
| **평균 대기 시간** | 큼 (순서에 의존) | 최소화 (최적값 수렴) |
| **처리량 (Throughput)** | 보통 | 향상 (짧은 작업 선처리) |
| **반응성** | 낮음 | 높음 (I/O 작업 감지 및 우선 배치) |

#### 미래 전망
빅데이터와 실시간 스트리밍 처리가 중요해짐에 따라, 지수 평균법은 **Edge Computing**에서의 자원 할당이나 **Serverless Architecture**의 Cold Start 예측 등에서 더욱 정밀하게 사용될 것이다. 단순 $\alpha$ 값을 넘어, 머신러닝 모델이 $\alpha$를 스스로 학습하는 **Adaptive Exponential Smoothing** 형태로 진화하고 있다.

#### 📢 섹션 요약 비유
"지수 평균법은 복잡한 미래를 예측하는 가장 단순하면서도 강력한 '수학적 나침반'입니다."
