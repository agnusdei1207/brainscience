# 194. NUMA (Non-Uniform Memory Access)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: CPU별 지역 메모리가 있는 다중 프로세서 아키텍처
> 2. **가치**: 대규모 시스템의 확장성 제공
> 3. **융합**: SMP, 로드 밸런싱, 메모리 지역성과 연관

---

## Ⅰ. 개요

### 개념 정의

NUMA(Non-Uniform Memory Access)는 **각 CPU가 자신의 지역 메모리(Local Memory)를 가지며, 다른 CPU의 메모리(Remote Memory)에 접근할 때 더 느린 아키텍처**다. 대규모 멀티프로세서 시스템에서 사용된다.

### 💡 비유: 지역 창고
NUMA는 **각 지점에 창고가 있는 체인점**과 같다. 자기 지점 창고(지역 메모리)에서 물건을 가져오는 건 빠르지만, 다른 지점 창고(원격 메모리)에서 가져오는 건 느리다.

## Ⅲ. 실무 적용

### 확인 및 설정
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【NUMA 정보 확인】                                                      │
│  ──────────────────                                                  │
│  # NUMA 하드웨어 정보                                                  │
│  $ numactl --hardware                                               │
│                                                                     │
│  # 현재 프로세스 NUMA 정책                                              │
│  $ numactl --show                                                   │
│                                                                     │
│  # NUMA 노드별 메모리                                                  │
│  $ numastat                                                         │
│                                                                     │
│  # 노드별 CPU                                                         │
│  $ ls /sys/devices/system/node/                                     │
│                                                                     │
│  【NUMA 정책 설정】                                                      │
│  ──────────────────                                                  │
│  # 특정 노드에서만 실행                                                 │
│  $ numactl --cpunodebind=0 --membind=0 ./program                    │
│                                                                     │
│  # 교차 할당                                                          │
│  $ numactl --interleave=all ./program                               │
│                                                                     │
│  # 선호 노드 설정                                                      │
│  $ numactl --preferred=1 ./program                                  │
│                                                                     │
│  【NUMA 밸런싱 제어】                                                    │
│  ──────────────────                                                  │
│  # NUMA 밸런싱 활성화/비활성화                                          │
│  $ sysctl -w kernel.numa_balancing=1                                │
│  $ sysctl -w kernel.numa_balancing=0                                │
│                                                                     │
│  # 현재 설정 확인                                                      │
│  $ cat /proc/sys/kernel/numa_balancing                              │
│                                                                     │
│  【프로그래밍에서의 NUMA】                                                │
│  ──────────────────                                                  │
│  #include <numa.h>                                                   │
│  #include <numaif.h>                                                 │
│                                                                     │
│  // NUMA 사용 가능 확인                                                │
│  if (numa_available() < 0) {                                         │
│      printf("NUMA not available\n");                                 │
│  }                                                                   │
│                                                                     │
│  // 특정 노드에 메모리 할당                                              │
│  void *mem = numa_alloc_onnode(size, node);                          │
│                                                                     │
│  // 현재 노드 확인                                                      │
│  int node = numa_node_of_cpu(sched_getcpu());                        │
│                                                                     │
│  // 메모리 정책 설정                                                    │
│  struct bitmask *mask = numa_parse_nodestring("0");                  │
│  set_mempolicy(MPOL_BIND, mask->maskp, mask->size);                  │
│                                                                     │
│  【NUMA 통계 모니터링】                                                  │
│  ──────────────────                                                  │
│  # 노드별 메모리 할당                                                    │
│  $ cat /sys/devices/system/node/node*/meminfo                       │
│                                                                     │
│  # 노드별 페이지 폴트                                                   │
│  $ numastat -p <pid>                                                │
│                                                                     │
│  # NUMA 히트/미스                                                      │
│  $ numastat -c <pid>                                                │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: CPU별 지역 메모리가 있는 아키텍처

• 스케줄링: 메모리 위치 고려
• 메모리 정책: Default, Bind, Interleave, Preferred

```
