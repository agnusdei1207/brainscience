# 195. 하이퍼스레딩 (Hyper-Threading)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 하나의 물리 코어가 두 개의 논리 코어처럼 동작
> 2. **가치**: 코어 내 자원 공유로 처리량 향상
> 3. **융합**: SMT, 물리 코어, 논리 코어와 연관

---

## Ⅰ. 개요

### 개념 정의

하이퍼스레딩(Hyper-Threading, HT) 또는 SMT(Simultaneous Multithreading)은 **하나의 물리 CPU 코어가 두 개 이상의 논리 코어로 동작하여 동시에 여러 스레드를 실행하는 기술**이다. Intel의 상표명이다.

### 💡 비유: 이중 작업대
하이퍼스레딩은 **하나의 주방에 두 개의 조리대**와 같다. 요리사(코어)는 하나지만, 한쪽이 굽는 동안 다른쪽을 손질하는 식으로 자원을 공유하며 효율을 높인다.

### 하이퍼스레딩 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                하이퍼스레딩 구조                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【물리 코어 vs 논리 코어】                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  하이퍼스레딩 없음:                                             │ │
│  │  ┌──────────────────────────────────────────────────────┐    │ │
│  │  │                   물리 코어 0                          │    │ │
│  │  │  ┌────────────────────────────────────────────────┐  │    │ │
│  │  │  │  ALU  │  FPU  │  캐시  │  레지스터  │   ...    │  │    │ │
│  │  │  └────────────────────────────────────────────────┘  │    │ │
│  │  │                        │                              │    │ │
│  │  │                     1 스레드                          │    │ │
│  │  └──────────────────────────────────────────────────────┘    │ │
│  │                                                             │ │
│  │  하이퍼스레딩 있음:                                             │ │
│  │  ┌──────────────────────────────────────────────────────┐    │ │
│  │  │                   물리 코어 0                          │    │ │
│  │  │  ┌───────────────────┐  ┌───────────────────┐       │    │ │
│  │  │  │  논리 코어 0        │  │  논리 코어 1        │       │    │ │
│  │  │  │ ┌─────────────┐   │  │ ┌─────────────┐   │       │    │ │
│  │  │  │ │ 레지스터    │   │  │ │ 레지스터    │   │       │    │ │
│  │  │  │ │ PC          │   │  │ │ PC          │   │       │    │ │
│  │  │  │ └─────────────┘   │  │ └─────────────┘   │       │    │ │
│  │  │  └───────────────────┘  └───────────────────┘       │    │ │
│  │  │           ▼                    ▼                     │    │ │
│  │  │  ┌────────────────────────────────────────────────┐  │    │ │
│  │  │  │  공유: ALU  │  FPU  │  캐시  │   ...           │  │    │ │
│  │  │  └────────────────────────────────────────────────┘  │    │ │
│  │  │                        │                              │    │ │
│  │  │                    2 스레드                           │    │ │
│  │  └──────────────────────────────────────────────────────┘    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【공유 vs 독립 자원】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  독립 (각 스레드별)           공유 (코어 내)                      │ │
│  │  ────                              ────                          │ │
│  │  • 레지스터                   • ALU                              │ │
│  │  • 프로그램 카운터             • FPU                              │ │
│  │  • 일부 L1 캐시               • L2/L3 캐시                       │ │
│  │  • APIC                       • 메모리 컨트롤러                  │ │
│  │                             • 실행 유닛                          │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                하이퍼스레딩 상세                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【성능 향상 원리】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  단일 스레드 실행 시:                                             │ │
│  │  • 파이프라인 스톨 발생 (캐시 미스, 분기 예측 실패 등)               │ │
│  │  • 실행 유닛 유휴 시간 존재                                        │ │
│  │                                                             │ │
│  │  하이퍼스레딩 시:                                                   │ │
│  │  • 스레드 0이 대기 → 스레드 1 실행                                 │ │
│  │  • 실행 유닛 활용률 증가                                           │ │
│  │  • 전체 처리량 향상                                                │ │
│  │                                                             │ │
│  │  최대 성능 향상: ~30% (이론적 2배, 실제 ~1.3배)                     │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  장점                      단점                                │ │
│  │  ────                        ────                                │ │
│  │  • 처리량 향상              • 단일 스레드 성능 감소 가능          │ │
│  │  • 비용 효율적              • 캐시 경합                          │ │
│  │  • 전력 효율적              • 보안 취약점 (Spectre 등)            │ │
│  │  • 멀티태스킹 개선           • 모든 워크로드에 유용하지 않음        │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【스케줄링 고려사항】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 논리 코어가 아닌 물리 코어 우선                                 │ │
│  │  • 같은 물리 코어의 두 논리 코어보다                                │ │
│  │    다른 물리 코어가 더 나을 수 있음                                 │ │
│  │  • CPU 배치 시 HT 고려                                            │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 확인 및 설정
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【하이퍼스레딩 확인】                                                    │
│  ──────────────────                                                  │
│  # 물리 코어 수                                                        │
│  $ grep "cpu cores" /proc/cpuinfo | head -1                          │
│  cpu cores       : 4                                                 │
│                                                                     │
│  # 논리 코어 수                                                        │
│  $ grep -c ^processor /proc/cpuinfo                                  │
│  8                                                                   │
│                                                                     │
│  # lscpu로 확인                                                       │
│  $ lscpu | grep -E "^CPU\(s\)|Thread|Core|Socket"                    │
│  CPU(s):                8                                            │
│  Thread(s) per core:    2                                            │
│  Core(s) per socket:    4                                            │
│  Socket(s):             1                                            │
│                                                                     │
│  # Thread(s) per core: 2 → 하이퍼스레딩 활성화                          │
│                                                                     │
│  【하이퍼스레딩 비활성화】                                                │
│  ──────────────────                                                  │
│  # 방법 1: BIOS에서 비활성화                                           │
│  # 방법 2: sysfs에서 오프라인 설정                                      │
│  $ echo 0 > /sys/devices/system/cpu/cpu1/online                      │
│  $ echo 0 > /sys/devices/system/cpu/cpu3/online                      │
│  # ... (논리 코어 번호)                                                │
│                                                                     │
│  # 커널 파라미터로 부팅 시 비활성화                                       │
│  # nosmt 또는 nr_cpus=4                                              │
│                                                                     │
│  【스케줄링 고려】                                                        │
│  ──────────────────                                                  │
│  # CPU 토폴로지 확인                                                   │
│  $ lscpu -e                                                          │
│  CPU NODE SOCKET CORE L1d:L1i:L2:L3 ONLINE MAXMHZ MINMHZ             │
│    0    0      0    0   0:0:0:0      yes    4000    800              │
│    1    0      0    1   1:1:1:0      yes    4000    800  # 같은 CORE 0│
│    2    0      1    2   2:2:2:0      yes    4000    800              │
│    3    0      1    3   3:3:3:0      yes    4000    800  # 같은 CORE 1│
│                                                                     │
│  # 같은 코어의 HT 형제 찾기                                             │
│  $ cat /sys/devices/system/cpu/cpu0/topology/thread_siblings_list    │
│  0,1                                                                 │
│                                                                     │
│  【성능 테스트】                                                          │
│  ──────────────────                                                  │
│  # HT 켜진 상태                                                        │
│  $ stress --cpu 8 --timeout 60                                       │
│                                                                     │
│  # HT 꺼진 상태 (4개만 실행)                                            │
│  $ stress --cpu 4 --timeout 60                                       │
│                                                                     │
│  # 비교                                                               │
│  $ time ./cpu_benchmark  # HT on vs off                             │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 한 물리 코어가 여러 논리 코어로 동작
• 기술: SMT (Simultaneous Multithreading)
• 공유 자원: ALU, FPU, 캐시
• 독립 자원: 레지스터, PC
• 성능 향상: 최대 ~30%
• 스케줄링: 물리 코어 우선 고려
```
