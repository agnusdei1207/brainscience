# 198. EDF (최단 데드라인 우선, Earliest Deadline First)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 가장 가까운 데드라인을 가진 태스크 먼저 실행
> 2. **가치**: 최적의 동적 우선순위 실시간 스케줄링
> 3. **융합**: 실시간 스케줄링, RM, CPU 이용률과 연관

---

## Ⅰ. 개요

### 개념 정의

EDF(Earliest Deadline First)는 **가장 가까운 데드라인을 가진 태스크에게 최고 우선순위를 부여하는 동적 우선순위 실시간 스케줄링 알고리즘**이다.

### 💡 비유: 마감일 순 과제
EDF는 **마감일이 가까운 과제부터 하는 것**과 같다. 내일 마감인 과제가 다음주 마감인 것보다 먼저다.

### EDF 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                EDF 구조                                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【핵심 원칙】                                                           │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  데드라인 가까움 → 우선순위 높음                                 │ │
│  │  데드라인 멂 → 우선순위 낮음                                      │ │
│  │                                                             │ │
│  │  동적 우선순위: 실행 시점마다 재계산                                │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【시간에 따른 우선순위 변화】                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  t=0:                                                          │ │
│  │  T1 (d=4), T2 (d=6), T3 (d=8)                                   │ │
│  │  우선순위: T1 > T2 > T3                                          │ │
│  │                                                             │ │
│  │  t=4: (T1 완료 후 T1의 다음 데드라인 = 8)                          │ │
│  │  T1 (d=8), T2 (d=6), T3 (d=8)                                   │ │
│  │  우선순위: T2 > T1=T3                                            │ │
│  │                                                             │ │
│  │  → 우선순위가 시간에 따라 동적으로 변화                              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【스케줄 가능성】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  EDF 스케줄 가능 조건:                                            │ │
│  │                                                             │ │
│  │  U = Σ(Ci/Ti) ≤ 1  (100%까지 스케줄 가능)                         │ │
│  │                                                             │ │
│  │  RM: ≤ 69%       EDF: ≤ 100%                                     │ │
│  │                                                             │ │
│  │  EDF가 RM보다 더 높은 CPU 이용률 달성                              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                EDF 상세                                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【특징】                                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  장점:                                                           │ │
│  │  • 최적의 동적 스케줄링 (100%까지 가능)                             │ │
│  │  • 주기적/비주기적 태스크 모두 지원                                 │ │
│  │  • RM보다 높은 CPU 이용률                                         │ │
│  │                                                             │ │
│  │  단점:                                                           │ │
│  │  • 구현 복잡                                                      │ │
│  │  • 런타임 오버헤드 큼 (우선순위 재계산)                              │ │
│  │  • 과부하 시 동작 예측 어려움                                       │ │
│  │  • 도미노 효과 가능 (한 태스크 지연 → 전파)                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【RM vs EDF 비교】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  구분              RM                   EDF                     │ │
│  │  ────                    ────                     ────           │ │
│  │  우선순위          정적 (주기 기반)        동적 (데드라인 기반)       │ │
│  │  CPU 이용률        ≤ 69%                ≤ 100%                   │ │
│  │  구현              간단                  복잡                     │ │
│  │  오버헤드          적음                  많음                     │ │
│  │  과부하 시         일부 태스크만 지연      예측 어려움              │ │
│  │  예측성            높음                  낮음                     │ │
│  │  적합 환경          간단한 시스템          복잡한 시스템            │ │
│  │                                                               │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【과부하 시 동작】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  U > 1일 때:                                                      │ │
│  │  • RM: 낮은 우선순위 태스크만 지연                                  │ │
│  │  • EDF: 어떤 태스크가 지연될지 예측 어려움                          │ │
│  │    (데드라인 순서가 계속 바뀌므로)                                  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【EDF 스케줄러 구현 개요】                                              │
│  ──────────────────                                                  │
│  #include <time.h>                                                   │
│  #include <stdlib.h>                                                 │
│                                                                     │
│  typedef struct {                                                    │
│      int id;                                                         │
│      int period;                                                     │
│      int exec_time;                                                  │
│      int deadline;        // 절대 데드라인                            │
│      int remaining;       // 남은 실행 시간                           │
│  } EDF_Task;                                                         │
│                                                                     │
│  int compare_deadline(const void *a, const void *b) {                │
│      return ((EDF_Task*)a)->deadline - ((EDF_Task*)b)->deadline;     │
│  }                                                                   │
│                                                                     │
│  EDF_Task* select_edf_task(EDF_Task tasks[], int n) {                │
│      // 데드라인순 정렬                                                │
│      qsort(tasks, n, sizeof(EDF_Task), compare_deadline);            │
│                                                                     │
│      // 실행 가능한 태스크 중 가장 가까운 데드라인 선택                    │
│      for (int i = 0; i < n; i++) {                                   │
│          if (tasks[i].remaining > 0) {                               │
│              return &tasks[i];                                       │
│          }                                                           │
│      }                                                               │
│      return NULL;                                                    │
│  }                                                                   │
│                                                                     │
│  void update_deadlines(EDF_Task tasks[], int n, int current_time) {  │
│      for (int i = 0; i < n; i++) {                                   │
│          if (tasks[i].remaining == 0) {                              │
│              // 다음 주기의 데드라인 설정                               │
│              tasks[i].deadline += tasks[i].period;                   │
│              tasks[i].remaining = tasks[i].exec_time;                │
│          }                                                           │
│      }                                                               │
│  }                                                                   │
│                                                                     │
│  【Linux SCHED_DEADLINE】                                              │
│  ──────────────────                                                  │
│  #include <linux/sched.h>                                            │
│                                                                     │
│  struct sched_attr {                                                 │
│      uint32_t size;                                                  │
│      uint32_t sched_policy;                                          │
│      uint64_t sched_runtime;    // 실행 시간                          │
│      uint64_t sched_deadline;   // 데드라인                           │
│      uint64_t sched_period;     // 주기                              │
│  };                                                                  │
│                                                                     │
│  void setup_deadline_task(int runtime_us, int deadline_us, int period_us) {│
│      struct sched_attr attr = {                                      │
│          .size = sizeof(attr),                                       │
│          .sched_policy = SCHED_DEADLINE,                             │
│          .sched_runtime = runtime_us,                                │
│          .sched_deadline = deadline_us,                              │
│          .sched_period = period_us                                   │
│      };                                                              │
│      sched_setattr(0, &attr, 0);                                     │
│  }                                                                   │
│                                                                     │
│  // 사용 예: 10ms 실행, 30ms 데드라인, 30ms 주기                        │
│  setup_deadline_task(10000, 30000, 30000);                           │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 가장 가까운 데드라인 먼저 실행
• 유형: 동적 우선순위 실시간 스케줄링
• 최적성: CPU 이용률 100%까지 스케줄 가능
• 장점: RM보다 높은 이용률, 비주기 태스크 지원
• 단점: 구현 복잡, 과부하 시 예측 어려움
• Linux: SCHED_DEADLINE으로 지원
```
