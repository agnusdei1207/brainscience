# 241. ABA 문제 (ABA Problem)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 값이 같아도 객체가 다른 CAS 오류
> 2. **가치**: 락 프리 알고리즘의 대표적 함정
> 3. **융합**: CAS, 메모리 재사용, Hazard Pointer와 연관

---

## Ⅰ. 개요

### 개념 정의

ABA 문제는 **CAS 연산에서 값이 A에서 B로 바뀐 후 다시 A로 돌아오면, 아무 변화가 없는 것처럼 보여 CAS가 성공하는 문제**다. 실제로는 객체가 재사용되었을 수 있다.

### 💡 비유: 도둑이 든 집
ABA 문제는 **도둑이 든 집**과 같다. 나갈 때와 들어올 때 물건이 같아 보이지만, 실제로는 도둑이 물건을 바꿨을 수 있다. 겉보기엔 같아 보인다.

### ABA 문제 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                ABA 문제 구조                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【시나리오: 연결 리스트 pop】                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  초기 상태:                                                       │ │
│  │  head → [A] → [B] → [C] → null                                 │ │
│  │                                                             │ │
│  │  1. T1: head 읽음 = A, A->next 읽음 = B                          │ │
│  │  ────────────────────────────────────────────────────────────  │ │
│  │  2. T2: pop() → A 제거                                           │ │
│  │     head → [B] → [C] → null                                     │ │
│  │                                                             │ │
│  │  3. T2: pop() → B 제거, A 재사용하여 push                         │ │
│  │     head → [A] → [C] → null   (A->next = C)                     │ │
│  │                                                             │ │
│  │  4. T1: CAS(head, A, B) 성공!                                    │ │
│  │     head → [B] → ???   (B는 이미 리스트에 없음!)                  │ │
│  │                                                             │ │
│  │  결과: 리스트 손상, C 손실                                         │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【타임라인】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  T1: read(A) ────────────────────────── read(A->next=B) ────  │ │
│  │                                    │                       │    │ │
│  │  T2:         pop(A) ─ pop(B) ─ push(A) ───────────────┘    │    │ │
│  │                    │         │         │                        │ │
│  │  리스트: [A|B|C] → [B|C] → [C] → [A|C]                     │ │
│  │                                    │                           │ │
│  │  T1:                                    CAS(head, A, B) ───────► │ │
│  │                                                             │ │
│  │  CAS 성공! 하지만 B는 이미 free됨 → 리스트 손상                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【문제 원인】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 포인터 값만 비교 (내용 무시)                                   │ │
│  │  • 메모리 재사용 시 같은 주소                                     │ │
│  │  • CAS는 값만 보고 변화 여부 판단                                 │ │
│  │  • "A → B → A" 변화 감지 못 함                                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                ABA 문제 상세                                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【해결 방법 1: 버전 카운터】                                           │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  struct stamped_ptr {                                            │ │
│  │      Node* ptr;                                                  │ │
│  │      uint64_t version;  // 버전 추가                              │ │
│  │  };                                                              │ │
│  │                                                             │ │
│  │  // 더블 워드 CAS로 ptr과 version 함께 변경                       │ │
│  │  bool CAS2(stamped_ptr* addr, stamped_ptr expected,              │ │
│  │             stamped_ptr new_val);                                │ │
│  │                                                             │ │
│  │  // 변경 시마다 version 증가                                       │ │
│  │  A → B → A' (version=2)  // A와 같아 보이지만 version 다름        │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【해결 방법 2: Hazard Pointer】                                       │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  // 접근 중인 포인터를 등록                                        │ │
│  │  hazard_ptr = head;  // 해저드 포인터에 등록                      │ │
│  │  // ... 작업 ...                                                  │ │
│  │  hazard_ptr = null;  // 등록 해제                                 │ │
│  │                                                             │ │
│  │  // 메모리 해제 시                                                 │ │
│  │  // 해저드 포인터에 등록된 주소는 해제하지 않음                      │ │
│  │  // 모든 스레드의 해저드 포인터 확인 후 해제                        │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【해결 방법 3: RCU / Epoch-based】                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  RCU:                                                             │ │
│  │  • Grace Period 후에만 메모리 해제                                │ │
│  │  • 참조 중인 메모리는 재사용 안 됨                                 │ │
│  │                                                             │ │
│  │  Epoch-based:                                                     │ │
│  │  • 전역 epoch로 메모리 해제 시점 관리                              │ │
│  │  • 이전 epoch의 메모리만 해제                                      │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【해결 방법 4: Deferred Reclamation】                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • free 대신 retire list에 추가                                   │ │
│  │  • 안전할 때까지 대기 후 일괄 해제                                  │ │
│  │  • 크기 제한 두어 주기적 해제                                       │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【C++ 버전 카운터 (std::atomic)】                                      │
│  ──────────────────                                                  │
│  #include <atomic>                                                   │
│  #include <cstdint>                                                  │
│                                                                     │
│  // 64비트: 상위 48비트 포인터, 하위 16비트 버전                        │
│  class StampedPointer {                                             │
│      std::atomic<uint64_t> stamped_ptr;                             │
│      static const int VERSION_BITS = 16;                            │
│  public:                                                            │
│      Node* get_ptr() {                                              │
│          return (Node*)(stamped_ptr.load() >> VERSION_BITS);        │
│      }                                                              │
│      bool cas(Node* expected, Node* new_ptr) {                      │
│          uint64_t old = stamped_ptr.load();                         │
│          if ((Node*)(old >> VERSION_BITS) != expected)              │
│              return false;                                          │
│          uint64_t new_val = ((uint64_t)new_ptr << VERSION_BITS) |  │
│                            ((old + 1) & 0xFFFF);                    │
│          return stamped_ptr.compare_exchange_weak(old, new_val);    │
│      }                                                              │
│  };                                                                 │
│                                                                     │
│  【Hazard Pointer 구현 개념】                                           │
│  ──────────────────                                                  │
│  // 전역 해저드 포인터 배열                                            │
│  std::atomic<Node*> hazard_ptrs[MAX_THREADS];                       │
│                                                                     │
│  Node* safe_read(std::atomic<Node*>& ptr) {                         │
│      Node* p;                                                        │
│      do {                                                           │
│          p = ptr.load();                                            │
│          hazard_ptrs[my_thread_id].store(p);  // 등록               │
│      } while (p != ptr.load());  // 더블 체크                        │
│      return p;                                                       │
│  }                                                                  │
│                                                                     │
│  void retire(Node* p) {                                              │
│      retire_list.push(p);                                            │
│      if (retire_list.size() > THRESHOLD) {                          │
│          scan_and_free();  // 안전한 것만 해제                        │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  void scan_and_free() {                                              │
│      // 모든 해저드 포인터 수집                                        │
│      std::set<Node*> hazard_set;                                    │
│      for (auto& hp : hazard_ptrs) {                                 │
│          hazard_set.insert(hp.load());                              │
│      }                                                              │
│      // 해저드 포인터에 없는 것만 해제                                  │
│      for (auto p : retire_list) {                                   │
│          if (hazard_set.find(p) == hazard_set.end()) {              │
│              delete p;                                               │
│          }                                                          │
│      }                                                              │
│  }                                                                  │
│                                                                     │
│  【예방 지침】                                                          │
│  ──────────────────                                                  │
│  1. 락 프리 자료구조에서 메모리 즉시 해제 금지                          │
│  2. 버전 카운터, 해저드 포인터, RCU 중 하나 사용                       │
│  3. 메모리 풀 사용 시 재사용 순서 주의                                 │
│  4. 테스트 시 장시간 실행으로 ABA 발생 가능성 확인                      │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: A→B→A 변화를 CAS가 감지 못함
• 원인: 포인터 재사용, 값만 비교
• 결과: 자료구조 손상, 메모리 오류
• 해결 1: 버전 카운터 (stamped pointer)
• 해결 2: Hazard Pointer
• 해결 3: RCU / Epoch-based
• 해결 4: Deferred Reclamation
```
