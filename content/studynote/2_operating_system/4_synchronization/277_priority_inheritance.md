# 277. 우선순위 상속 (Priority Inheritance)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 락 홀더가 웨이터의 우선순위를 상속
> 2. **가치**: 우선순위 역전 해결의 핵심
> 3. **융합**: rt_mutex, 우선순위 역전과 연관

---

## Ⅰ. 개요

### 개념 정의

우선순위 상속(Priority Inheritance)은 **락을 보유한 낮은 우선순위 스레드가 대기 중인 높은 우선순위 스레드의 우선순위를 일시적으로 상속받는 기법**이다.

### 💡 비유: 응급차 동행
우선순위 상속은 **응급차 동행**과 같다. 일반 차가 응급차 앞에 있으면 응급차처럼 우선通行한다. 환자를 빨리 병원에 데려다주기 위해.

### 우선순위 상속 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                우선순위 상속 구조                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【동작 원리】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  [상속 전]                                                        │ │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │  H (High)   ────►  락 대기                              │ │ │
│  │  │                      ▲                                  │ │ │
│  │  │                      │                                  │ │ │
│  │  │  L (Low)    ◄────────┘  락 보유 (우선순위 낮음)           │ │ │
│  │  │                      M (Medium)가 CPU 선점 가능          │ │ │
│  │  └─────────────────────────────────────────────────────────┘ │ │
│  │                                                             │ │
│  │  [상속 후]                                                        │ │
│  │  ┌─────────────────────────────────────────────────────────┐ │ │
│  │  │  H (High)   ────►  락 대기                              │ │ │
│  │  │                      ▲                                  │ │ │
│  │  │                      │ 상속                             │ │ │
│  │  │  L' (High!) ◄────────┘  락 보유 (우선순위 승격!)          │ │ │
│  │  │                      M이 CPU 선점 불가                   │ │ │
│  │  └─────────────────────────────────────────────────────────┘ │ │
│  │                                                             │ │
│  │  L이 빠르게 락 해제 → H가 즉시 획득                                │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【상속 규칙】                                                         │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 락 획득: 원래 우선순위 유지                                      │ │
│  │  2. 높은 우선순위 웨이터 발생: 상속 시작                              │ │
│  │  3. 상속: max(현재, 웨이터 우선순위)                                │ │
│  │  4. 락 해제: 원래 우선순위로 복귀                                    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                우선순위 상속 상세                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【다중 웨이터 상속】                                                   │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  웨이터들: H1(99), H2(95), H3(90)                                 │ │
│  │  락 홀더 L의 상속 우선순위: max(99, 95, 90) = 99                  │ │
│  │                                                             │ │
│  │  가장 높은 웨이터 우선순위로 승격                                     │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【연쇄 상속 (Chained Inheritance)】                                   │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  H → L1 → L2 (락 체인)                                            │ │
│  │                                                             │ │
│  │  1. H가 L1의 락 대기                                               │ │
│  │  2. L1이 L2의 락 대기                                              │ │
│  │  3. L2가 H의 우선순위 상속                                          │ │
│  │  4. L1도 H의 우선순위 상속                                          │ │
│  │                                                             │ │
│  │  전파: H → L1 → L2                                                │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【구현 복잡도】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 웨이터 우선순위 추적 필요                                        │ │
│  │  • 동적 우선순위 변경                                               │ │
│  │  • 스케줄러 연동                                                    │ │
│  │  • 락 해제 시 복귀 처리                                             │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  장점:                                                              │ │
│  │  • 우선순위 역전 해결                                                │ │
│  │  • 응용 투명 (애플리케이션 수정 불필요)                               │ │
│  │  • 동적 적용                                                        │ │
│  │                                                             │ │
│  │  단점:                                                              │ │
│  │  • 구현 복잡                                                        │ │
│  │  • 오버헤드                                                         │ │
│  │  • 데드락 가능성 (다중 락 시)                                        │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Linux rt_mutex 구현】                                              │
│  ──────────────────                                                  │
│  // kernel/locking/rtmutex.c                                        │
│                                                                     │
│  struct rt_mutex {                                                  │
│      struct rb_root_cached   waiters;  // 우선순위 큐               │
│      struct task_struct      *owner;                               │
│  };                                                                 │
│                                                                     │
│  // 우선순위 상속 로직                                                  │
│  static int rt_mutex_adjust_prio_chain(...) {                       │
│      // 1. 최고 우선순위 웨이터 찾기                                    │
│      // 2. owner 우선순위 조정                                        │
│      // 3. 연쇄 상속 처리                                             │
│  }                                                                  │
│                                                                     │
│  【POSIX 스레드 사용】                                                 │
│  ──────────────────                                                  │
│  #include <pthread.h>                                               │
│  #include <sched.h>                                                 │
│                                                                     │
│  // Priority Inheritance 뮤텍스 생성                                  │
│  pthread_mutexattr_t attr;                                          │
│  pthread_mutexattr_init(&attr);                                     │
│  pthread_mutexattr_setprotocol(&attr, PTHREAD_PRIO_INHERIT);        │
│                                                                     │
│  pthread_mutex_t pi_mutex;                                          │
│  pthread_mutex_init(&pi_mutex, &attr);                              │
│                                                                     │
│  // 실시간 우선순위 설정                                                │
│  struct sched_param param;                                          │
│  param.sched_priority = 50;                                         │
│  pthread_setschedparam(pthread_self(), SCHED_FIFO, &param);         │
│                                                                     │
│  【QNX Priority Inheritance】                                         │
│  ──────────────────                                                  │
│  // QNX는 기본적으로 PI 제공                                          │
│  pthread_mutex_t mutex = PTHREAD_MUTEX_INITIALIZER;                 │
│  // 모든 뮤텍스가 자동으로 PI                                           │
│                                                                     │
│  【VxWorks】                                                          │
│  ──────────────────                                                  │
│  // 세마포어 생성 시 옵션                                               │
│  SEM_ID sem = semMCreate(SEM_Q_PRIORITY | SEM_INVERSION_SAFE);      │
│                                                                     │
│  【Java 우선순위 상속 없음】                                            │
│  ──────────────────                                                  │
│  // Java는 네이티브 PI 미지원                                          │
│  // 대신 순수 Java 구현 사용                                           │
│                                                                     │
│  // 대안: 중간 락 사용                                                  │
│  synchronized (lowPriorityLock) {                                   │
│      // 빠르게 완료하도록 설계                                          │
│  }                                                                  │
│                                                                     │
│  【성능 비교】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  락 타입           락/언락      역전 지연    구현 복잡도            │ │
│  │  ────────           ──────       ──────      ────────           │ │
│  │  일반 mutex         O(1)        무제한      낮음                 │ │
│  │  PI mutex           O(log n)    제한됨      높음                 │ │
│  │  PC mutex           O(1)        없음       중간                 │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【디버깅】                                                            │
│  ──────────────────                                                  │
│  // ftrace로 PI 이벤트 추적                                           │
│  echo 1 > /sys/kernel/debug/tracing/events/rt_mutex/enable          │
│  cat /sys/kernel/debug/tracing/trace_pipe                           │
│                                                                     │
│  // /proc에서 확인                                                    │
│  cat /proc/<pid>/sched                                              │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
