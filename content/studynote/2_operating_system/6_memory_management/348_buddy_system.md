# 버디 시스템 (Buddy System)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 버디 시스템 (Buddy System)은 메모리를 2의 거듭제곱 크기(2^n)의 블록으로 분할하고, 해제 시 크기가 같은 이웃 블록(Buddy)을 자동으로 병합하여 외부 단편화를 줄이는 동적 메모리 할당 알고리즘이다.
> 2. **가치**: O(log N)의 빠른 할당·해제 속도와 자동 병합으로 연속 가변 분할 방식의 압축(Compaction) 오버헤드 없이 메모리를 효율적으로 관리하며, Linux 커널의 페이지 할당자(Buddy Allocator)로 채택되어 수십 년간 검증된 표준 기법이다.
> 3. **융합**: Linux의 `struct zone`과 버디 할당자는 페이지 단위(4KB~2MB)로 동작하며, 슬랩 할당자(Slab Allocator)가 그 위에서 커널 객체 단위의 세밀한 재사용을 처리하는 2계층 메모리 관리 구조를 이룬다.

---

## Ⅰ. 개요 및 필요성

연속 메모리 할당에서 외부 단편화는 해결하기 어렵다. 임의 크기 블록을 할당·해제하다 보면 작은 구멍들이 흩어져 큰 블록을 할당할 수 없게 된다. 압축(Compaction)은 가능하지만 모든 포인터를 갱신해야 하는 막대한 오버헤드가 있다.

버디 시스템은 할당 단위를 2^n으로 제한함으로써 해제된 블록을 자동으로 인접 버디와 병합할 수 있게 한다.

**💡 비유**: 종이를 반씩 접는 것 — 4조각짜리 종이 2개가 있으면 자동으로 8조각짜리 한 장으로 합칠 수 있다.

```text
┌──────────────────────────────────────────────────────────────┐
│         버디 시스템 분할·병합 동작                          │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  초기: 1024KB 연속 가용 블록                                │
│                                                              │
│  70KB 할당 요청 → 2^7=128KB 블록 선택                       │
│                                                              │
│  [512KB] [256KB] [128KB | 128KB]                            │
│                    ↙         ↘                              │
│              [할당됨]     [128KB 가용]                       │
│                                                              │
│  50KB 할당 → 64KB 블록 선택:                                │
│  [512KB] [256KB] [128KB(할당)] [64KB|64KB]                  │
│                                  ↙     ↘                   │
│                             [할당됨] [64KB 가용]            │
│                                                              │
│  70KB 해제 → 인접 128KB 버디와 병합 시도                    │
│  → 버디가 사용 중이면 병합 안 됨 (50KB 블록 처리됨)        │
│  → 버디(128KB)가 가용이면 병합 → 256KB 가용 블록           │
└──────────────────────────────────────────────────────────────┘
```

**📢 섹션 요약 비유**: 버디 시스템은 레고 블록 — 2×2 블록 2개는 4×2로 합칠 수 있고, 4×2 두 개는 4×4로 합칠 수 있어요. 같은 크기 버디끼리만 합쳐지는 깔끔한 규칙이에요.

## Ⅱ. 아키텍처 및 핵심 원리

### 버디 주소 계산

버디 블록의 주소는 XOR 연산으로 간단히 계산 가능:
- 크기 2^k 블록의 버디 주소 = 블록 주소 XOR 2^k
- 예: 주소 0x000, 크기 256KB → 버디 주소 = 0x000 XOR 256K = 0x40000

이 수학적 특성이 O(1) 버디 탐색을 가능하게 한다.

```text
┌──────────────────────────────────────────────────────────────┐
│         Linux 버디 할당자 — order별 free list               │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  order 0:  4KB  블록 free list  ──────▶ [4KB] [4KB] ...     │
│  order 1:  8KB  블록 free list  ──────▶ [8KB] ...           │
│  order 2:  16KB 블록 free list  ──────▶ [16KB] ...          │
│  ...                                                         │
│  order 10: 4MB  블록 free list  ──────▶ [4MB] ...           │
│                                                              │
│  할당: order 계산 → 해당 free list에서 블록 추출             │
│        없으면 상위 order에서 분할 (Split)                    │
│  해제: 블록 반환 → 버디 탐색 → 가용이면 병합(Merge) →      │
│        재귀적으로 상위 order까지 병합 시도                  │
└──────────────────────────────────────────────────────────────┘
```

**📢 섹션 요약 비유**: Linux 버디 할당자는 크기별 서랍장 — 4KB 서랍, 8KB 서랍, 16KB 서랍 각각에서 꺼내고, 반납하면 같은 크기 버디와 합쳐서 더 큰 서랍으로 귀환합니다.

## Ⅲ. 융합 비교 및 다각도 분석

### 버디 시스템 vs 슬랩 할당자

| 항목 | 버디 시스템 | 슬랩 할당자 |
|:---|:---|:---|
| 관리 단위 | 페이지 단위 (4KB~4MB) | 객체 단위 (bytes 수준) |
| 할당 대상 | 연속 물리 페이지 | 커널 자료구조 (task_struct 등) |
| 단편화 | 내부 단편화 (2^n 반올림) | 거의 없음 (객체 크기 맞춤) |
| 관계 | 버디가 슬랩에 메모리 제공 | 버디 위에서 동작 |

**📢 섹션 요약 비유**: 버디는 창고 선반(레이어), 슬랩은 선반 위 작은 바구니(세분화) — 창고 회사(버디)가 선반을 내주면, 바구니 회사(슬랩)가 그 선반을 잘게 나눠 씁니다.

## Ⅳ. 실무 적용 및 기술사적 판단

**실무 시나리오**:
1. **대용량 연속 메모리 할당**: DMA, GPU, 네트워크 버퍼는 연속 물리 페이지 필요. 버디할당자가 order N 블록으로 제공.
2. **메모리 단편화 진단**: `/proc/buddyinfo` 명령으로 order별 가용 블록 수 확인 → 단편화 심각도 진단.

**안티패턴**:
- **장기 운영 후 단편화**: 서버 장기 운영 시 버디 시스템도 order 0~2 블록에 단편화 집중. 주기적 메모리 컴팩션(Memory Compaction) 또는 재부팅 필요.
- **2^n 낭비**: 65KB 할당 시 128KB 블록 사용 → 63KB 내부 단편화(49% 낭비). 이를 슬랩 할당자가 보완.

**📢 섹션 요약 비유**: 65KB 짐을 128KB 상자에 넣는 낭비 — 버디는 딱 맞는 2^n 상자밖에 없어서 어쩔 수 없어요. 이 낭비를 줄이는 게 슬랩 할당자의 역할이에요.

## Ⅴ. 기대효과 및 결론

| 구분 | 가변 분할 | 버디 시스템 |
|:---|:---|:---|
| 외부 단편화 | 심각 | 크게 감소 |
| 내부 단편화 | 없음 | 최대 50% (2^n 반올림) |
| 압축 필요 | 자주 필요 | 병합으로 자동 해결 |
| 시간 복잡도 | O(N) | O(log N) |

버디 시스템은 빠른 할당·해제와 자동 병합으로 커널 메모리 관리의 표준이 되었다. 내부 단편화 한계는 슬랩 할당자로 보완하는 2계층 구조가 현대 Linux 커널 메모리 관리의 핵심이다.

## 📌 관련 개념 맵

| 개념 | 관계 |
|:---|:---|
| 슬랩 할당자 (Slab Allocator) | 버디 위에서 객체 단위 세밀한 재사용 |
| 외부 단편화 (External Fragmentation) | 버디 시스템이 완화하는 핵심 문제 |
| 내부 단편화 (Internal Fragmentation) | 2^n 반올림으로 발생하는 버디의 부작용 |
| Linux `/proc/buddyinfo` | 버디 시스템 상태 진단 인터페이스 |
| 메모리 컴팩션 | 버디 시스템 단편화 해소 보완 수단 |

## 👶 어린이를 위한 3줄 비유 설명
1. 버디 시스템은 종이 접기 규칙 — 반 접기(÷2)만 가능해서 항상 2의 배수 크기 조각만 만들 수 있어요.
2. 두 조각이 원래 한 쌍(버디)이면 다시 합칠 수 있어요 — 혼자 남은 조각은 합칠 수 없어요.
3. 작은 조각들이 짝을 찾아 합쳐지면서 큰 조각이 되니까, 큰 메모리가 필요할 때도 준비가 돼 있어요!
