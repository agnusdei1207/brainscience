# 411. 다단계 페이지 테이블 (Multi-Level Page Table)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 페이지 테이블을 계층적으로 구성
> 2. **가치**: 메모리 절약과 주소 공간 확장
> 3. **융합**: 페이지 테이블, 가상 메모리와 연관

---

## Ⅰ. 개요

### 개념 정의

다단계 페이지 테이블(Multi-Level Page Table)은 **페이지 테이블을 여러 계층으로 나누어 메모리를 절약하는 기법**이다. 사용하지 않는 주소 공간에 대한 페이지 테이블을 유지하지 않는다.

### 💡 비유: 도서관 색인
다단계 페이지 테이블은 **도서관 색인**과 같다. 전체 색인이 너무 크다. 책이 없는 섹션의 색인은 만들지 않는다.

### 다단계 페이지 테이블 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                다단계 페이지 테이블 구조                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【1단계 vs 2단계】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1단계 (단일 페이지 테이블):                                            │ │
│  │  • 32비트 주소, 4KB 페이지                                              │ │
│  │  • 페이지 수: 2^20 = 1M개                                             │ │
│  │  • 테이블 크기: 1M × 4B = 4MB                                         │ │
│  │  • 모든 페이지에 대해 항상 4MB 필요!                                     │ │
│  │                                                             │ │
│  │  2단계 (2-Level Page Table):                                         │ │
│  │  • 외부 페이지 테이블 (Page Directory): 1K개 항목                        │ │
│  │  • 내부 페이지 테이블 (Page Table): 1K개 항목                           │ │
│  │  • 실제 사용하는 주소만 테이블 유지                                        │ │
│  │  • 메모리 크게 절약                                                      │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【2단계 페이지 테이블 구조】                                               │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  32비트 논리 주소:                                                      │ │
│  │  ┌──────────────────┬──────────────────┬──────────────────┐      │ │
│  │  │  페이지 디렉토리    │   페이지 테이블     │     오프셋         │      │ │
│  │  │     (10비트)       │      (10비트)      │     (12비트)       │      │ │
│  │  └──────────────────┴──────────────────┴──────────────────┘      │ │
│  │           │                    │                                   │ │
│  │           │                    │                                   │ │
│  │           ▼                    │                                   │ │
│  │  ┌─────────────────┐          │                                   │ │
│  │  │ 페이지 디렉토리   │          │                                   │ │
│  │  │   (1K 항목)      │          │                                   │ │
│  │  │    항목 0        │──────────┘                                   │ │
│  │  │    항목 1        │──┐                                           │ │
│  │  │    항목 2        │  │  null (할당 안 됨)                         │ │
│  │  │    ...           │  │                                           │ │
│  │  └─────────────────┘  │                                           │ │
│  │                        │                                           │ │
│  │                        ▼                                           │ │
│  │                   ┌─────────────────┐                               │ │
│  │                   │ 페이지 테이블    │                               │ │
│  │                   │   (1K 항목)      │                               │ │
│  │                   │    항목 0 ──► F5 │                               │ │
│  │                   │    항목 1 ──► F2 │                               │ │
│  │                   │    ...          │                                │ │
│  │                   └─────────────────┘                               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                다단계 페이지 테이블 상세                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【64비트 시스템의 4단계】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  x86-64 (4-Level Paging):                                             │ │
│  │                                                             │ │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │ PML4 │ Page Directory Pointer │ Page Directory │ Page Table │Offset││
│  │  │9비트 │       9비트            │     9비트        │    9비트     │12비트││
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                                                             │ │
│  │  5단계 (5-Level Paging):                                               │ │
│  │  • PML5 (9비트) 추가                                                    │ │
│  │  • 57비트 가상 주소 지원                                                   │ │
│  │  • 128 PB 주소 공간                                                     │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【메모리 절약 원리】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  사용하지 않는 주소 공간:                                                 │ │
│  │  • 상위 레벨 항목이 null이면 하위 테이블 없음                              │ │
│  │  • sparse한 주소 공간에 효과적                                            │ │
│  │                                                             │ │
│  │  예: 32비트 주소 공간 중 0-1GB만 사용                                    │ │
│  │  • 단일 테이블: 4MB 필요                                                 │ │
│  │  • 2단계: 4KB + 4KB = 8KB (1/500로 감소!)                               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【주소 변환 과정】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  2단계 예시:                                                            │ │
│  │  논리 주소 = 0x12345678                                                 │ │
│  │                                                             │ │
│  │  1. 추출:                                                              │ │
│  │     페이지 디렉토리 인덱스 = 비트[31:22] = 0x48                        │ │
│  │     페이지 테이블 인덱스 = 비트[21:12] = 0x35                          │ │
│  │     오프셋 = 비트[11:0] = 0x678                                         │ │
│  │                                                             │ │
│  │  2. 조회:                                                              │ │
│  │     CR3 → 페이지 디렉토리[0x48] → 페이지 테이블 주소                      │ │
│  │     페이지 테이블[0x35] → 프레임 번호                                    │ │
│  │                                                             │ │
│  │  3. 합성:                                                              │ │
│  │     물리 주소 = 프레임 번호 × 4KB + 0x678                              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Linux 페이지 테이블 레벨】                                             │
│  ──────────────────                                                  │
│  // x86-64 Linux                                                       │
│  PGD (Page Global Directory)     - 4단계                              │
│  PUD (Page Upper Directory)      - 3단계                              │
│  PMD (Page Middle Directory)     - 2단계                              │
│  PTE (Page Table Entry)          - 1단계                              │
│                                                                     │
│  // 각 레벨은 512개 항목 (9비트)                                          │
│  // 페이지 크기 = 4KB, 각 항목 8바이트                                      │
│                                                                     │
│  【주소 변환 비용】                                                        │
│  ──────────────────                                                  │
│  // 메모리 접근 횟수                                                       │
│  // 1단계: 1회 (페이지 테이블)                                             │
│  // 2단계: 2회 (디렉토리 + 테이블)                                         │
│  // 4단계: 4회 (PML4 + PDP + PD + PT)                                   │
│                                                                     │
│  // TLB 히트 시: 0회 (TLB에서 바로 물리 주소 얻음)                           │
│  // TLB가 중요한 이유!                                                      │
│                                                                     │
│  【페이지 테이블 확인】                                                      │
│  ──────────────────                                                  │
│  // 프로세스의 페이지 테이블 정보                                            │
│  $ cat /proc/<pid>/statm                                             │
│  size resident shared text lib data dt                               │
│  1000   500     100   100  10  100  10                               │
│  // size: 전체 페이지 수                                                   │
│  // resident: 물리 메모리에 있는 페이지 수                                  │
│                                                                     │
│  // 가상 메모리 영역 확인                                                    │
│  $ pmap -x <pid>                                                      │
│  Address           Kbytes     RSS   Dirty Mode  Mapping              │
│  0000555555554000     4       4       0 r---- a.out                  │
│  0000555555555000     4       4       0 r-x-- a.out                  │
│  ...                                                                  │
│                                                                     │
│  【페이지 테이블 크기 최적화】                                               │
│  ──────────────────                                                  │
│  // Huge Page 사용 시                                                    │
│  // 2MB 페이지: 512개 4KB 페이지를 1개로 대체                                │
│  // 페이지 테이블 항목 1/512로 감소!                                         │
│                                                                     │
│  // Transparent Huge Page                                               │
│  $ cat /sys/kernel/mm/transparent_hugepage/enabled                   │
│  [always] madvise never                                                │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 페이지 테이블을 계층적 구성
• 목적: 사용하지 않는 주소 공간의 테이블 제거
• 구조: 2단계, 3단계, 4단계, 5단계
• 장점: 메모리 절약, 큰 주소 공간 지원
• 단점: 주소 변환 단계 증가 (TLB로 완화)
• 활용: x86-64, ARM64 등 현대 CPU
```
