# 516. I/O 스케줄링 (I/O Scheduling)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 디스크 I/O 요청 순서 최적화
> 2. **가치**: 디스크 성능 향상과 공정성
> 3. **융합**: 디스크 스케줄링, 큐, 엘리베이터와 연관

---

## Ⅰ. 개요

### 개념 정의

I/O 스케줄링(I/O Scheduling)은 **디스크 I/O 요청의 순서와 타이밍을 조정하여 디스크 성능을 최적화하는 기술**이다.

### 💡 비유: 엘리베이터 스케줄링
I/O 스케줄링은 **엘리베이터가 층을 방문하는 순서**와 같다. 효율적인 경로로 사람들을 태운다.

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                I/O 스케줄링 상세                                       │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【CFQ (Completely Fair Queer)]                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  • 각 프로세스에 시간 슬라이스 할당                                        │ │
│  │  • 라운드 로빈으로 공정한 처리                                             │ │
│  │  • 읽기 우선 (동기 I/O)                                                  │ │
│  │  • Linux 기본 스케줄러 (구버전)                                            │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【Deadline 스케줄러】                                                   │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  • 읽기/쓰기 각각 마감 시간 설정                                           │ │
│  │  • 기본: 읽기 500ms, 쓰기 5s                                              │ │
│  │  • 기아(starvation) 방지                                                  │ │
│  │  • 실시간/데이터베이스에 적합                                               │ │
│  │                                                             │ │
│  │  큐:                                                                    │ │
│  │  • 정렬 큐 (섹터 순)                                                      │ │
│  │  • 읽기 마감 큐                                                          │ │
│  │  • 쓰기 마감 큐                                                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【noop 스케줄러】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  • 단순 FIFO + 병합만 수행                                                │ │
│  │  • 탐색 최적화 없음                                                       │ │
│  │  • SSD에 적합 (탐색 시간 0)                                               │ │
│  │  • 가상 머신에 적합                                                       │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【최적화 기법】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  1. Merge: 인접한 요청 병합                                              │ │
│  │     [100-109] + [110-119] → [100-119]                                   │ │
│  │                                                             │ │
│  │  2. Sort: 요청을 섹터 순으로 정렬                                          │ │
│  │     [200, 100, 300] → [100, 200, 300]                                   │ │
│  │                                                             │ │
│  │  3. Read-ahead: 미리 읽기                                                │ │
│  │     순차 읽기 시 다음 블록 미리 로드                                        │ │
│  │                                                             │ │
│  │  4. Writeback: 쓰기 지연                                                 │ │
│  │     쓰기 요청을 모았다가 일괄 처리                                          │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【스케줄러 확인/변경】                                                  │
│  ──────────────────                                                  │
│  // 현재 스케줄러 확인                                                    │
│  $ cat /sys/block/sda/queue/scheduler                               │
│  [mq-deadline] none                                                  │
│                                                                     │
│  // 스케줄러 변경                                                        │
│  $ echo noop > /sys/block/sda/queue/scheduler                       │
│  $ echo deadline > /sys/block/sda/queue/scheduler                   │
│  $ echo cfq > /sys/block/sda/queue/scheduler                        │
│                                                                     │
│  // 영구 설정 (udev 규칙)                                                │
│  $ cat > /etc/udev/rules.d/60-scheduler.rules << 'EOF'              │
│  ACTION=="add|change", KERNEL=="sd[a-z]", \                         │
│      ATTR{queue/scheduler}="deadline"                               │
│  ACTION=="add|change", KERNEL=="nvme[0-9]", \                       │
│      ATTR{queue/scheduler}="none"                                   │
│  EOF                                                                 │
│                                                                     │
│  【NVMe 스케줄링】                                                        │
│  ──────────────────                                                  │
│  // NVMe 스케줄러 확인                                                    │
│  $ cat /sys/block/nvme0n1/queue/scheduler                           │
│  [none] mq-deadline                                                  │
│                                                                     │
│  // NVMe는 기본 none 권장 (하드웨어 큐 관리)                               │
│  $ echo none > /sys/block/nvme0n1/queue/scheduler                   │
│                                                                     │
│  【스케줄러 튜닝 파라미터】                                                │
│  ──────────────────                                                  │
│  // deadline 튜닝                                                        │
│  $ cat /sys/block/sda/queue/iosched/read_expire                     │
│  500                                                                 │
│  $ cat /sys/block/sda/queue/iosched/write_expire                    │
│  5000                                                                │
│  $ echo 100 > /sys/block/sda/queue/iosched/read_expire              │
│                                                                     │
│  // CFQ 튜닝                                                             │
│  $ cat /sys/block/sda/queue/iosched/slice_idle                      │
│  8                                                                   │
│  $ cat /sys/block/sda/queue/iosched/quantum                         │
│  8                                                                   │
│                                                                     │
│  【요청 큐 설정】                                                        │
│  ──────────────────                                                  │
│  // 큐 깊이                                                              │
│  $ cat /sys/block/sda/queue/nr_requests                              │
│  128                                                                 │
│  $ echo 256 > /sys/block/sda/queue/nr_requests                       │
│                                                                     │
│  // read-ahead 설정                                                      │
│  $ cat /sys/block/sda/queue/read_ahead_kb                           │
│  128                                                                 │
│  $ blockdev --getra /dev/sda                                         │
│  256                                                                 │
│  $ blockdev --setra 512 /dev/sda                                     │
│                                                                     │
│  【I/O 통계 확인】                                                        │
│  ──────────────────                                                  │
│  // iostat                                                               │
│  $ iostat -x 1                                                       │
│  Device   rrqm/s  wrqm/s   r/s   w/s  rMB/s  wMB/s  await  %util     │
│  sda       0.50   10.00  50.0 100.0   2.00   4.00   5.00   80.0      │
│                                                                     │
│  // 큐 통계                                                               │
│  $ cat /sys/block/sda/queue/iostats                                 │
│  1                                                                   │
│                                                                     │
│  // 블록 장치 통계                                                        │
│  $ cat /proc/diskstats                                               │
│  sda 1234 5678 12345678 901234 5678 1234 5678901 234567 0 123456 78901│
│                                                                     │
│  【워크로드별 추천 스케줄러】                                             │
│  ──────────────────                                                  │
│  용도              추천 스케줄러        이유                                 │
│  ────              ────────────        ────                                 │
│  SSD               none/noop           탐색 시간 없음                       │
│  HDD (일반)         deadline/cfq        탐색 최적화                         │
│  데이터베이스        deadline           지연 예측 가능                       │
│  파일 서버          cfq/bfq            공정성 중요                         │
│  데스크탑           bfq                인터랙티브 응답                       │
│  가상 머신          noop               하이퍼바이저가 스케줄링                │
│  NVMe              none               하드웨어 관리                        │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 디스크 I/O 요청 순서 최적화
• 스케줄러: CFQ, Deadline, noop, BFQ, mq-deadline
• 최적화: Merge, Sort, Read-ahead, Writeback
• HDD: 탐색 시간 최소화 중요
• SSD/NVMe: noop 또는 none 권장
• 확인: /sys/block/sdX/queue/scheduler
• 설정: echo로 변경, udev로 영구 설정
• 튜닝: read_expire, write_expire, nr_requests
```
