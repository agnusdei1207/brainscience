# 562. RS-232/485

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 직렬 통신 표준 인터페이스
> 2. **가치**: 산업 표준, 신뢰성, 거리 통신
> 3. **융합**: UART, 산업 자동화, 모뎀과 연관

---

## Ⅰ. 개요

### 개념 정의
**RS-232/485**는 **직렬 통신을 위한 물리 계층 표준**입니다.

### 💡 비유: 장거리 전화선
RS-232/485는 **장거리 전화선**과 같습니다. 먼 거리에서도 통신합니다.

### RS-232/485 구조
```
┌─────────────────────────────────────────────────────────────────────┐
│                RS-232/485 구조                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【RS-232 vs RS-485 비교】                                           │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  특성              RS-232             RS-485                   │ │   │
│  │  ────              ──────             ──────                   │ │   │
│  │  신호              단형 (싱글엔드)      차동 (Differential)      │ │   │
│  │  선 수             3-25선             2선 (Half), 4선 (Full)    │ │   │
│  │  최대 거리          15m               1200m                    │ │   │
│  │  최대 속도          115.2 kbps        10 Mbps                  │ │   │
│  │  장치 수            1:1 (Point-to-Point) 32-256 (Multidrop)   │ │   │
│  │  전압 레벨          ±3 ~ ±15V         -7 ~ +12V               │ │   │
│  │  용도              콘솔, 모뎀         산업 자동화               │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【RS-232 연결】                                                      │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  DTE (컴퓨터)                      DCE (모뎀/장치)               │ │   │
│  │  ┌─────────┐                       ┌─────────┐               │ │   │
│  │  │ TXD  2  │───────────────────────│ RXD  2  │               │ │   │
│  │  │ RXD  3  │───────────────────────│ TXD  3  │               │ │   │
│  │  │ GND  5  │───────────────────────│ GND  5  │               │ │   │
│  │  │ RTS  7  │───────────────────────│ CTS  8  │               │ │   │
│  │  │ CTS  8  │───────────────────────│ RTS  7  │               │ │   │
│  │  │ DTR  4  │───────────────────────│ DSR  6  │               │ │   │
│  │  │ DSR  6  │───────────────────────│ DTR  4  │               │ │   │
│  │  └─────────┘                       └─────────┘               │ │   │
│  │                                                             │ │   │
│  │  Null Modem: TXD↔RXD, RTS↔CTS 교차 연결                       │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【RS-485 연결】                                                      │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  Half-Duplex (2선):                                             │ │   │
│  │  ┌───────┐      ┌───────┐      ┌───────┐                      │ │   │
│  │  │ 마스터│──A───┤슬레이브│──A───┤슬레이브│                      │ │   │
│  │  │      │──B───┤  1   │──B───┤  2   │                      │ │   │
│  │  └───────┘      └───────┘      └───────┘                      │ │   │
│  │                                                             │ │   │
│  │  Full-Duplex (4선): TX+/TX-/RX+/RX-                            │ │   │
│  │                                                             │ │   │
```

## Ⅱ. 상세 분석
### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                RS-232/485 상세                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【RS-232 핀 정의 (DB-9)】                                           │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  핀   신호      방향              설명                          │ │   │
│  │  ───  ────      ────              ────                          │ │   │
│  │  1    DCD       입력              Data Carrier Detect          │ │   │
│  │  2    RXD       입력              Receive Data                 │ │   │
│  │  3    TXD       출력              Transmit Data                │ │   │
│  │  4    DTR       출력              Data Terminal Ready          │ │   │
│  │  5    GND       -                 Ground                       │ │   │
│  │  6    DSR       입력              Data Set Ready               │ │   │
│  │  7    RTS       출력              Request To Send              │ │   │
│  │  8    CTS       입력              Clear To Send                │ │   │
│  │  9    RI        입력              Ring Indicator               │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【신호 레벨】                                                        │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  RS-232:                                                         │ │   │
│  │  Logic 0 (Space): +3V ~ +15V                                    │ │   │
│  │  Logic 1 (Mark):  -3V ~ -15V                                    │ │   │
│  │  TTL 변환: TTL=1 → RS232=-12V, TTL=0 → RS232=+12V              │ │   │
│  │                                                             │ │   │
│  │  RS-485:                                                         │ │   │
│  │  Logic 0: A > B (by 200mV minimum)                             │ │   │
│  │  Logic 1: B > A (by 200mV minimum)                             │ │   │
│  │  공통 모드 범위: -7V ~ +12V                                     │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  【RS-422】                                                           │
│  ──────────────────                                                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │ │   │
│  │  RS-422 = RS-485 Full-Duplex (1:1 전용)                        │ │   │
│  │  • 4선 전이중 통신                                              │ │   │
│  │  • 1개 송신자, 10개 수신자                                      │ │   │
│  │  • 최대 1200m, 10 Mbps                                         │ │   │
│  │                                                             │ │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용
### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │|
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【RS-232 설정】                                                      │
│  ──────────────────                                                │
│  // 장치 확인                                                        │
│  $ ls /dev/ttyS* /dev/ttyUSB*                                      │
│  /dev/ttyS0   /dev/ttyUSB0                                         │
│                                                                     │
│  // stty로 설정                                                      │
│  $ stty -F /dev/ttyUSB0 9600 cs8 -cstopb -parenb                   │
│  // 9600 baud, 8 data bits, 1 stop, no parity                      │
│                                                                     │
│  // 설정 확인                                                        │
│  $ stty -F /dev/ttyUSB0 -a                                         │
│                                                                     │
│  // 데이터 전송                                                      │
│  $ echo "AT" > /dev/ttyUSB0                                        │
│  $ cat /dev/ttyUSB0                                                │
│                                                                     │
│  【RS-485 설정】                                                      │
│  ──────────────────                                                │
│  // RS-485 모드 활성화                                               │
│  #include <linux/serial.h>                                          │
│                                                                     │
│  struct serial_rs485 rs485;                                        │
│  rs485.flags = SER_RS485_ENABLED | SER_RS485_RTS_ON_SEND;          │
│  ioctl(fd, TIOCSRS485, &rs485);                                    │
│                                                                     │
│  // Linux 명령어                                                     │
│  $ sudo setserial /dev/ttyUSB0 port 0x3F8 irq 4                    │
│                                                                     │
│  【Python RS-232/485】                                               │
│  ──────────────────                                                │
│  import serial                                                      │
│  import time                                                        │
│                                                                     │
│  # RS-232 연결                                                       │
│  ser = serial.Serial(                                              │
│      port='/dev/ttyUSB0',                                          │
│      baudrate=9600,                                                │
│      bytesize=serial.EIGHTBITS,                                    │
│      parity=serial.PARITY_NONE,                                    │
│      stopbits=serial.STOPBITS_ONE,                                 │
│      timeout=1,                                                    │
│      rtscts=True   # Hardware flow control                         │
│  )                                                                  │
│                                                                     │
│  # Modbus RTU (RS-485)                                              │
│  import minimalmodbus                                               │
│                                                                     │
│  instrument = minimalmodbus.Instrument('/dev/ttyUSB0', 1)  # slave 1│
│  instrument.serial.baudrate = 9600                                 │
│  instrument.serial.bytesize = 8                                    │
│  instrument.serial.parity = 'N'                                    │
│  instrument.serial.stopbits = 1                                    │
│                                                                     │
│  # 레지스터 읽기                                                      │
│  value = instrument.read_register(0, 1)  # 주소 0, 1 레지스터        │
│  print(f"Value: {value}")                                          │
│                                                                     │
│  # 레지스터 쓰기                                                      │
│  instrument.write_register(0, 100)                                 │
│                                                                     │
│  【USB-시리얼 변환기】                                                │
│  ──────────────────                                                │
│  // 칩셋별 드라이버                                                   │
│  FTDI FT232: ftdi_sio                                              │
│  Prolific PL2303: pl2303                                           │
│  CP210x: cp210x                                                    │
│  CH340: ch340                                                      │
│                                                                     │
│  // 드라이버 로드                                                     │
│  $ sudo modprobe ftdi_sio                                          │
│  $ sudo modprobe cp210x                                            │
│                                                                     │
│  // USB 장치 확인                                                    │
│  $ lsusb | grep -i serial                                          │
│  Bus 001 Device 005: ID 067b:2303 Prolific Technology, Inc. PL2303 │
│                                                                     │
│  【문제 해결】                                                        │
│  ──────────────────                                                │
│  // 권한                                                            │
│  $ sudo usermod -aG dialout $USER                                  │
│                                                                     │
│  // 포트 점유 확인                                                    │
│  $ lsof /dev/ttyUSB0                                               │
│                                                                     │
│  // 커널 메시지                                                       │
│  $ dmesg | grep tty                                                │
│                                                                     │
│  // 플래그 확인                                                       │
│  $ stty -F /dev/ttyUSB0 -a                                         │
│                                                                     │
```

## Ⅳ. 기대효과 및 결론
### 핵심 요약
```
• 개념: 직렬 통신 물리 계층 표준
• RS-232: 단형, 15m, 1:1, 콘솔/모뎀
• RS-485: 차동, 1200m, 32-256장치, 산업
• RS-422: 차동 전이중, 1:10
• 신호: RS-232 (±12V), RS-485 (차동)
• 커넥터: DB-9, RJ45, 단자대
• Linux: /dev/ttyS*, /dev/ttyUSB*
• Python: pyserial, minimalmodbus
• 흐름 제어: RTS/CTS (하드웨어)
• 용도: 산업 자동화, PLC, 센서
```
