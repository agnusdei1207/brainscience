# 442. 파일 (File)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 논리적 데이터 단위
> 2. **가치**: 데이터 영속성 제공
> 3. **융합**: 파일 시스템, 프로세스와 연관

---

## Ⅰ. 개요

### 개념 정의

파일(File)은 **저장장치에 보관된 관련 정보의 논리적 단위**다. 이름, 속성, 데이터로 구성된다.

### 💡 비유: 서류철
파일은 **서류철**과 같다. 제목(파일명)이 있고 내용(데이터)이 있다. 서랍(디렉토리)에 보관한다.

### 파일 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                파일 구조                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【파일 구성 요소】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  1. 파일 이름 (File Name):                                               │ │
│  │     • 사람이 식별하는 이름                                                 │ │
│  │     • 경로 포함 가능                                                      │ │
│  │     • 예: /home/user/document.txt                                       │ │
│  │                                                             │ │
│  │  2. 메타데이터 (Metadata):                                                │ │
│  │     • 파일 크기                                                           │ │
│  │     • 생성/수정/접근 시간                                                 │ │
│  │     • 접근 권한                                                           │ │
│  │     • 소유자                                                              │ │
│  │     • 파일 형식                                                           │ │
│  │     • 데이터 블록 위치                                                    │ │
│  │                                                             │ │
│  │  3. 데이터 (Data):                                                        │ │
│  │     • 파일의 실제 내용                                                    │ │
│  │     • 바이트 스트림                                                       │ │
│  │     • 블록 단위 저장                                                      │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【파일 속성】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  ┌─────────────────────────────────────────────────────────┐  │ │
│  │  │ 파일 이름    │  document.txt                             │  │ │
│  │  ├─────────────────────────────────────────────────────────┤  │ │
│  │  │ 파일 크기    │  1,024 bytes                              │  │ │
│  │  ├─────────────────────────────────────────────────────────┤  │ │
│  │  │ 파일 형식    │  일반 파일 (regular)                       │  │ │
│  │  ├─────────────────────────────────────────────────────────┤  │ │
│  │  │ 접근 권한    │  rw-r--r-- (644)                          │  │ │
│  │  ├─────────────────────────────────────────────────────────┤  │ │
│  │  │ 소유자       │  user:group                               │  │ │
│  │  ├─────────────────────────────────────────────────────────┤  │ │
│  │  │ 타임스탬프   │  atime, mtime, ctime                       │  │ │
│  │  ├─────────────────────────────────────────────────────────┤  │ │
│  │  │ 아이노드      │  12345                                    │  │ │
│  │  ├─────────────────────────────────────────────────────────┤  │ │
│  │  │ 링크 수      │  1                                        │  │ │
│  │  ├─────────────────────────────────────────────────────────┤  │ │
│  │  │ 블록 수      │  2                                        │  │ │
│  │  └─────────────────────────────────────────────────────────┘  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                파일 상세                                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【파일 형식】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  • 일반 파일 (Regular File, '-'):                                       │ │
│  │    데이터 저장, 실행 파일                                                 │ │
│  │                                                             │ │
│  │  • 디렉토리 (Directory, 'd'):                                            │ │
│  │    파일 목록 저장                                                         │ │
│  │                                                             │ │
│  │  • 심볼릭 링크 (Symbolic Link, 'l'):                                     │ │
│  │    다른 파일 가리킴                                                        │ │
│  │                                                             │ │
│  │  • 하드 링크 (Hard Link):                                                │ │
│  │    같은 아이노드 공유                                                      │ │
│  │                                                             │ │
│  │  • 장치 파일 (Device File):                                              │ │
│  │    블록 장치 ('b'), 문자 장치 ('c')                                       │ │
│  │                                                             │ │
│  │  • 파이프 (Pipe, 'p'):                                                   │ │
│  │    프로세스 간 통신                                                        │ │
│  │                                                             │ │
│  │  • 소켓 (Socket, 's'):                                                   │ │
│  │    네트워크 통신                                                           │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【파일 접근 방식】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. 순차 접근 (Sequential Access):                                       │ │
│  │     처음부터 순서대로 읽기                                                 │ │
│  │                                                             │ │
│  │  2. 직접 접근 (Direct/Random Access):                                    │ │
│  │     원하는 위치로 바로 이동                                                 │ │
│  │     lseek() 사용                                                         │ │
│  │                                                             │ │
│  │  3. 인덱스 접근 (Indexed Access):                                         │ │
│  │     인덱스를 통해 데이터 접근                                              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【파일 정보 확인】                                                      │
│  ──────────────────                                                  │
│  // 파일 상세 정보                                                        │
│  $ stat /etc/passwd                                                   │
│  File: /etc/passwd                                                    │
│  Size: 2847            Blocks: 8          IO Block: 4096             │
│  Device: 801h/2049d    Inode: 12345       Links: 1                   │
│  Access: (0644/-rw-r--r--)  Uid: (    0/    root)  Gid: (    0/root) │
│  Access: 2024-03-08 10:00:00.000000000 +0900                         │
│  Modify: 2024-03-08 10:00:00.000000000 +0900                         │
│  Change: 2024-03-08 10:00:00.000000000 +0900                         │
│                                                                     │
│  // 파일 형식 확인                                                        │
│  $ file /etc/passwd                                                  │
│  /etc/passwd: ASCII text                                             │
│                                                                     │
│  // ls로 파일 정보                                                       │
│  $ ls -la /etc/passwd                                                │
│  -rw-r--r-- 1 root root 2847 Mar  8 10:00 /etc/passwd               │
│                                                                     │
│  【파일 생성/삭제】                                                      │
│  ──────────────────                                                  │
│  // 파일 생성                                                            │
│  $ touch newfile.txt                                                 │
│  $ echo "Hello" > newfile.txt                                        │
│                                                                     │
│  // 파일 복사                                                            │
│  $ cp source.txt dest.txt                                            │
│  $ cp -r src_dir/ dest_dir/                                          │
│                                                                     │
│  // 파일 이동/이름변경                                                    │
│  $ mv oldname.txt newname.txt                                        │
│  $ mv file.txt /home/user/                                           │
│                                                                     │
│  // 파일 삭제                                                            │
│  $ rm file.txt                                                       │
│  $ rm -r directory/                                                  │
│                                                                     │
│  【프로그래밍에서 파일 다루기】                                            │
│  ──────────────────                                                  │
│  #include <stdio.h>                                                  │
│  #include <sys/stat.h>                                               │
│  #include <fcntl.h>                                                   │
│                                                                     │
│  // 파일 정보 얻기                                                        │
│  struct stat st;                                                      │
│  if (stat("/etc/passwd", &st) == 0) {                                │
│      printf("Size: %ld bytes\n", st.st_size);                        │
│      printf("Inode: %ld\n", st.st_ino);                              │
│      printf("Mode: %o\n", st.st_mode);                               │
│  }                                                                   │
│                                                                     │
│  // 파일 열기/읽기/쓰기                                                   │
│  int fd = open("file.txt", O_RDWR | O_CREAT, 0644);                  │
│  write(fd, "Hello", 5);                                               │
│  lseek(fd, 0, SEEK_SET);                                             │
│  read(fd, buffer, 5);                                                │
│  close(fd);                                                           │
│                                                                     │
│  // 또는 stdio 사용                                                      │
│  FILE *fp = fopen("file.txt", "r");                                  │
│  fgets(buffer, 100, fp);                                             │
│  fclose(fp);                                                          │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 저장장치의 논리적 데이터 단위
• 구성: 이름 + 메타데이터 + 데이터
• 형식: 일반, 디렉토리, 링크, 장치, 파이프, 소켓
• 접근: 순차, 직접, 인덱스
• 속성: 크기, 권한, 소유자, 타임스탬프
• 식별: 이름, 아이노드 번호
• 시스템콜: open, read, write, close, stat
```
