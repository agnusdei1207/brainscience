# 448. 링크 (Link)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 파일에 대한 참조
> 2. **가치**: 파일 공유와 공간 절약
> 3. **융합**: 아이노드, 파일 시스템과 연관

---

## Ⅰ. 개요

### 개념 정의

링크(Link)는 **파일에 대한 참조를 생성하여 같은 데이터를 다른 이름으로 접근하는 방법**이다. 하드 링크와 심볼릭 링크가 있다.

### 💡 비유: 바로가기
링크는 **바로가기**와 같다. 원본 파일은 그대로 두고 다른 위치에서 접근할 수 있다.

### 링크 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                링크 구조                                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【하드 링크 (Hard Link)】                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  같은 아이노드를 참조:                                                    │ │
│  │                                                             │ │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │              아이노드 (번호: 12345)                       │    │ │
│  │  │  타입: 일반 파일                                          │    │ │
│  │  │  크기: 100 bytes                                          │    │ │
│  │  │  데이터 블록: [100, 101, ...]                              │    │ │
│  │  │  링크 수: 2                                                │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │           ▲                    ▲                                    │ │
│  │           │                    │                                    │ │
│  │  ┌────────┴───────┐    ┌───────┴───────┐                          │ │
│  │  │ 원본.txt        │    │ hardlink.txt  │                          │ │
│  │  │ (디렉토리 엔트리)│    │ (디렉토리 엔트리)│                          │ │
│  │  └────────────────┘    └───────────────┘                          │ │
│  │                                                             │ │
│  │  • 같은 아이노드 번호                                                    │ │
│  │  • 링크 수 증가                                                          │ │
│  │  • 원본 삭제해도 링크 유효                                                │ │
│  │  • 같은 파일 시스템만 가능                                                │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【심볼릭 링크 (Symbolic Link / Soft Link)】                             │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  파일 경로를 저장:                                                        │ │
│  │                                                             │ │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │          아이노드 (번호: 12346)                           │    │ │
│  │  │  타입: 심볼릭 링크                                         │    │ │
│  │  │  크기: 9 bytes                                            │    │ │
│  │  │  데이터: "원본.txt"                                        │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │           │                                                             │ │
│  │           │ 경로 참조                                                    │ │
│  │           ▼                                                             │ │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │          아이노드 (번호: 12345)                           │    │ │
│  │  │  타입: 일반 파일                                          │    │ │
│  │  │  데이터: 실제 내용...                                      │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │           ▲                                                             │ │
│  │           │                                                             │ │
│  │  ┌────────┴───────┐    ┌────────────────┐                             │ │
│  │  │ 원본.txt         │    │ symlink.txt -> │                             │ │
│  │  └────────────────┘    └────────────────┘                             │ │
│  │                                                             │ │
│  │  • 다른 아이노드                                                          │ │
│  │  • 원본 파일 경로 저장                                                    │ │
│  │  • 원본 삭제 시 링크 끊김                                                  │ │
│  │  • 다른 파일 시스템 가능                                                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                링크 상세                                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【하드 링크 vs 심볼릭 링크】                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  특성           하드 링크          심볼릭 링크                        │ │
│  │  ──────────    ────────────────    ──────────────────             │ │
│  │  아이노드        같음               다름                              │ │
│  │  데이터         공유                경로만 저장                         │ │
│  │  원본 삭제       링크 유효           링크 끊김                          │ │
│  │  파일시스템     같아야 함           달라도 됨                           │ │
│  │  디렉토리       불가능              가능                               │ │
│  │  크기           원본과 같음         경로 문자열 길이                     │ │
│  │  권한           원본과 같음         777 (lrwxrwxrwx)                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【링크 사용 사례】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  하드 링크:                                                               │ │
│  │  • 백업 (원본과 동일한 상태 유지)                                          │ │
│  │  • 버전 관리 (snapshot)                                                  │ │
│  │  • 공간 절약 (같은 데이터 공유)                                            │ │
│  │                                                             │ │
│  │  심볼릭 링크:                                                             │ │
│  │  • 바로가기                                                              │ │
│  │  • 라이브러리 버전 관리                                                   │ │
│  │  • 동적 경로 참조                                                        │ │
│  │  • 크로스 파일시스템 참조                                                 │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【링크 관련 제약】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  하드 링크:                                                               │ │
│  │  • 디렉토리에 하드 링크 불가 (순환 방지)                                    │ │
│  │  • 다른 파일 시스템 간 불가                                                │ │
│  │  • 링크 수가 0이면 아이노드 삭제                                           │ │
│  │                                                             │ │
│  │  심볼릭 링크:                                                             │ │
│  │  • 원본 이동 시 끊김                                                      │ │
│  │  • 순환 참조 가능성                                                       │ │
│  │  • 상대 경로 시 주의 필요                                                  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【링크 생성】                                                          │
│  ──────────────────                                                  │
│  // 하드 링크 생성                                                        │
│  $ ln original.txt hardlink.txt                                     │
│  $ ls -la                                                            │
│  -rw-r--r-- 2 user group 100 Mar 8 10:00 hardlink.txt              │
│  -rw-r--r-- 2 user group 100 Mar 8 10:00 original.txt              │
│  // 링크 수 = 2                                                           │
│                                                                     │
│  // 심볼릭 링크 생성                                                       │
│  $ ln -s original.txt symlink.txt                                   │
│  $ ls -la                                                            │
│  lrwxrwxrwx 1 user group  10 Mar 8 10:00 symlink.txt -> original.txt│
│  // l로 시작, -> 로 원본 표시                                              │
│                                                                     │
│  // 링크 확인                                                            │
│  $ stat hardlink.txt symlink.txt                                    │
│                                                                     │
│  【링크 삭제】                                                          │
│  ──────────────────                                                  │
│  // 하드 링크 삭제 (원본은 유지)                                            │
│  $ rm hardlink.txt                                                   │
│  $ ls -la original.txt                                               │
│  -rw-r--r-- 1 user group 100 Mar 8 10:00 original.txt              │
│  // 링크 수 = 1                                                           │
│                                                                     │
│  // 원본 삭제 시 심볼릭 링크 끊김                                            │
│  $ rm original.txt                                                   │
│  $ cat symlink.txt                                                   │
│  cat: symlink.txt: No such file or directory                        │
│                                                                     │
│  【프로그래밍에서 링크】                                                    │
│  ──────────────────                                                  │
│  #include <unistd.h>                                                 │
│                                                                     │
│  // 하드 링크 생성                                                        │
│  if (link("original.txt", "hardlink.txt") == 0) {                     │
│      printf("Hard link created\n");                                   │
│  }                                                                   │
│                                                                     │
│  // 심볼릭 링크 생성                                                       │
│  if (symlink("original.txt", "symlink.txt") == 0) {                  │
│      printf("Symbolic link created\n");                               │
│  }                                                                   │
│                                                                     │
│  // 심볼릭 링크 읽기                                                       │
│  char buf[256];                                                       │
│  ssize_t len = readlink("symlink.txt", buf, sizeof(buf)-1);          │
│  buf[len] = '\0';                                                     │
│  printf("Links to: %s\n", buf);                                       │
│                                                                     │
│  // 링크 해제                                                          │
│  unlink("hardlink.txt");                                             │
│  unlink("symlink.txt");                                              │
│                                                                     │
│  【링크 탐지】                                                          │
│  ──────────────────                                                  │
│  // 심볼릭 링크인지 확인                                                   │
│  #include <sys/stat.h>                                               │
│  struct stat st;                                                      │
│  lstat("symlink.txt", &st);  // lstat은 링크 자체 정보                   │
│  if (S_ISLNK(st.st_mode)) {                                          │
│      printf("It's a symbolic link\n");                                │
│  }                                                                   │
│                                                                     │
│  // 하드 링크인지 확인 (링크 수 > 1)                                        │
│  stat("file.txt", &st);                                              │
│  if (st.st_nlink > 1) {                                              │
│      printf("Has hard links\n");                                      │
│  }                                                                   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 파일에 대한 참조 생성
• 종류: 하드 링크, 심볼릭 링크
• 하드 링크: 같은 아이노드, 원본 삭제해도 유효
• 심볼릭 링크: 경로 저장, 원본 삭제 시 끊김
• 명령: ln (하드), ln -s (심볼릭)
• 시스템콜: link(), symlink(), unlink()
• 확인: ls -la, stat, lstat
```
