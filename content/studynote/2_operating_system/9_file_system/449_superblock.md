# 449. 슈퍼블록 (Superblock)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 파일 시스템의 메타데이터
> 2. **가치**: 파일 시스템 전체 관리 정보
> 3. **융합**: 파일 시스템, 마운트와 연관

---

## Ⅰ. 개요

### 개념 정의

슈퍼블록(Superblock)은 **파일 시스템 전체에 대한 정보를 저장하는 자료 구조**다. 파일 시스템의 크기, 여유 공간, 아이노드 정보 등을 포함한다.

### 💡 비유: 도서관 안내판
슈퍼블록은 **도서관 안내판**과 같다. 전체 책 수, 여유 선반, 열람실 위치 등 전체 정보가 있다.

### 슈퍼블록 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                슈퍼블록 구조                                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【파일 시스템 구조에서 위치】                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  ┌─────────────────────────────────────────────────────────┐  │ │
│  │  │                    슈퍼블록 (Superblock)                   │  │ │
│  │  │  • 파일 시스템 타입 (ext4, xfs...)                           │  │ │
│  │  │  • 전체 크기                                                │  │ │
│  │  │  • 블록 크기                                                │  │ │
│  │  │  • 여유 블록 수                                              │  │ │
│  │  │  • 아이노드 수                                               │  │ │
│  │  └─────────────────────────────────────────────────────────┘  │ │
│  │                                                             │ │
│  │  ┌─────────────────────────────────────────────────────────┐  │ │
│  │  │           아이노드 비트맵 / 아이노드 테이블                    │  │ │
│  │  └─────────────────────────────────────────────────────────┘  │ │
│  │                                                             │ │
│  │  ┌─────────────────────────────────────────────────────────┐  │ │
│  │  │           블록 비트맵 / 데이터 블록                           │  │ │
│  │  └─────────────────────────────────────────────────────────┘  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【슈퍼블록 주요 필드】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  struct ext4_super_block {                                              │ │
│  │      __le32 s_inodes_count;      // 전체 아이노드 수                     │ │
│  │      __le32 s_blocks_count_lo;   // 전체 블록 수                         │ │
│  │      __le32 s_r_blocks_count_lo; // 예약된 블록 수                       │ │
│  │      __le32 s_free_blocks_count_lo; // 여유 블록 수                      │ │
│  │      __le32 s_free_inodes_count; // 여유 아이노드 수                     │ │
│  │      __le32 s_first_data_block; // 첫 번째 데이터 블록                   │ │
│  │      __le32 s_log_block_size;   // 블록 크기 (로그)                      │ │
│  │      __le32 s_log_frag_size;    // 프래그먼트 크기                       │ │
│  │      __le32 s_blocks_per_group; // 그룹당 블록 수                        │ │
│  │      __le32 s_inodes_per_group; // 그룹당 아이노드 수                    │ │
│  │      __le16 s_magic;            // 매직 넘버 (0xEF53)                    │ │
│  │      // ... 추가 필드                                                   │ │
│  │  };                                                                      │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                슈퍼블록 상세                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【슈퍼블록 백업】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  슈퍼블록 손상 시 복구를 위해 여러 위치에 백업:                             │ │
│  │                                                             │ │
│  │  • 기본 슈퍼블록: 블록 0 또는 1                                             │ │
│  │  • 백업 슈퍼블록: 주기적으로 여러 블록 그룹에 저장                           │ │
│  │  • ext4: 각 블록 그룹의 시작에 슈퍼블록 복사본                              │ │
│  │                                                             │
│  │  손상 시:                                                                │
│  │  $ e2fsck -b 32768 /dev/sda1                                        │
│  │  // 백업 슈퍼블록으로 복구                                                 │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【슈퍼블록과 메모리】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  마운트 시:                                                               │
│  │  • 슈퍼블록을 메모리에 로드                                                │ │
│  │  • 커널 메모리에서 관리                                                   │ │
│  │  • 변경 사항은 주기적으로 디스크에 동기화                                  │ │
│  │                                                             │ │
│  │  마운트 정보:                                                             │
│  │  • struct super_block (커널)                                            │ │
│  │  • 파일 시스템별 슈퍼블록 정보 포함                                        │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【슈퍼블록 동기화】                                                      │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  sync 명령:                                                               │ │
│  │  • 메모리의 슈퍼블록을 디스크에 동기화                                     │ │
│  │  • 깨끗한 언마운트를 위해 필요                                            │ │
│  │                                                             │ │
│  │  주기적 동기화:                                                           │ │
│  │  • 5초마다 (기본값)                                                        │ │
│  │  • /proc/sys/vm/dirty_writeback_centisecs                              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 파일 시스템 전체 관리 정보
• 위치: 파일 시스템 시작 부분
• 내용: 크기, 블록 수, 아이노드 수, 여유 공간
• 백업: 여러 위치에 복사본 저장
• 동기화: sync로 디스크에 반영
• 확인: tune2fs, xfs_info, statfs
• 복구: e2fsck -b (백업으로)
```
