# 452. 디스크 스케줄링 (Disk Scheduling)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 디스크 I/O 요청 순서 최적화
> 2. **가치**: 탐색 시간 최소화와 성능 향상
> 3. **융합**: 디스크 구조, 운영체제와 연관

---

## Ⅰ. 개요

### 개념 정리

디스크 스케줄링(Disk Scheduling)은 **디스크 I/O 요청의 처리 순서를 결정하여 평균 탐색 시간을 최소화하는 기법**이다.

### 💡 비유: 엘리베이터
디스크 스케줄링은 **엘리베이터 스케줄링**과 같다. 엘리베이터가 효율적으로 층을 이동하듯, 헤드가 효율적으로 트랙을 이동한다.

### 디스크 스케줄링 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                디스크 스케줄링 개요                                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【디스크 접근 시간 구성】                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  디스크 접근 시간 = 탐색 시간 + 회전 지연 + 전송 시간              │ │
│  │                                                             │ │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │  탐색 시간 (Seek Time)                                   │    │ │
│  │  │  • 헤드가 원하는 트랙으로 이동하는 시간                    │    │ │
│  │  │  • 가장 오래 걸림 (기계적 이동)                           │    │ │
│  │  │  • 스케줄링의 주요 최적화 대상                            │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                                                             │ │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │  회전 지연 (Rotational Latency)                          │    │ │
│  │  │  • 섹터가 헤드 아래 올 때까지 대기                        │    │ │
│  │  │  • RPM에 따라 결정                                        │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                                                             │ │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │  전송 시간 (Transfer Time)                               │    │ │
│  │  │  • 데이터를 읽거나 쓰는 실제 시간                          │    │ │
│  │  │  • 회전 속도와 섹터 수에 따라 결정                          │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【스케줄링 알고리즘 종류】                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  1. FCFS (First-Come First-Served)                                   │ │
│  │  2. SSTF (Shortest Seek Time First)                                 │ │
│  │  3. SCAN (Elevator)                                                  │ │
│  │  4. C-SCAN (Circular SCAN)                                           │ │
│  │  5. LOOK / C-LOOK                                                    │ │
│  │  6. 기타: N-Step-SCAN, FSCAN                                         │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 스케줄링 알고리즘 비교
```
┌─────────────────────────────────────────────────────────────────────┐
│                스케줄링 알고리즘 비교                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【FCFS (First-Come First-Served)】                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  요청 순서: 98, 183, 37, 122, 14, 124, 65, 67                    │ │
│  │  현재 위치: 53                                                    │ │
│  │                                                             │ │
│  │  이동: 53→98→183→37→122→14→124→65→67                            │ │
│  │  총 이동: 45+85+146+85+108+110+59+2 = 640                        │ │
│  │                                                             │ │
│  │  장점: 공정함, 구현 단순                                          │ │
│  │  단점: 평균 탐색 시간 길음, 헤드 진동 가능성                        │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【SSTF (Shortest Seek Time First)】                                 │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  요청 순서: 98, 183, 37, 122, 14, 124, 65, 67                    │ │
│  │  현재 위치: 53                                                    │ │
│  │                                                             │ │
│  │  이동: 53→65→67→37→14→98→122→124→183                            │ │
│  │  총 이동: 12+2+30+23+84+24+2+59 = 236                           │ │
│  │                                                             │ │
│  │  장점: 평균 탐색 시간 최소                                          │ │
│  │  단점: 기아 현상(Starvation), 탐색 시간 계산 오버헤드               │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【SCAN (Elevator)】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  한 방향으로 계속 이동, 끝에서 방향 전환                              │ │
│  │                                                             │ │
│  │  ┌───┐                                           ┌───┐        │ │
│  │  │ 0 │─┬─┬─┬─┬─┬─┬─┬─┬─┬─┬─┬─┬─┬─┬─→│199│        │ │
│  │  └───┘  ↑                           ↑  └───┘        │ │
│  │         │    53→→→→→→→→→→→→→→→→│              │ │
│  │         └────────────────────────┘              │ │
│  │                                                             │ │
│  │  장점: 기아 현상 없음, 균등한 대기 시간                               │ │
│  │  단점: 양 끝단의 요청 대기 시간 길음                                 │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【C-SCAN (Circular SCAN)】                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │  한 방향으로만 서비스, 끝에서 처음으로 점프                               │ │
│  │                                                             │ │
│  │  ┌───┐                                           ┌───┐        │ │
│  │  │ 0 │←─┬─┬─┬─┬─┬─┬─┬─┬─┬─┬─┬─┬─┬─┬─→│199│        │ │
│  │  └───┘  └────────────────────────────┘  └───┘        │ │
│  │          ↑          (점프)           ↑                       │ │
│  │          └────────────────────────────┘                      │ │
│  │                                                             │ │
│  │  장점: 균등한 대기 시간                                            │ │
│  │  단점: 리턴 시간 낭비                                              │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 리눅스 I/O 스케줄러
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【리눅스 I/O 스케줄러 종류】                                          │
│  ──────────────────                                                  │
│  // 현재 스케줄러 확인                                                  │
│  $ cat /sys/block/sda/queue/scheduler                               │
│  [mq-deadline] kyber bfq none                                       │
│  // [ ] 안이 현재 사용 중인 스케줄러                                     │
│                                                                     │
│  // 스케줄러 변경                                                        │
│  $ echo bfq > /sys/block/sda/queue/scheduler                       │
│                                                                     │
│  1. CFQ (Completely Fair Queuing) - 레거시                          │
│     • 프로세스별 공정한 I/O 대기열                                      │
│     • 데스크탑 시스템에 적합                                            │
│                                                                     │
│  2. Deadline                                                         │
│     • 읽기/쓰기 별도 큐                                                │
│     • 데드라인 보장                                                    │
│     • 데이터베이스에 적합                                                │
│                                                                     │
│  3. NOOP (No Operation)                                              │
│     • 단순 FIFO                                                       │
│     • SSD에 적합                                                       │
│                                                                     │
│  4. BFQ (Budget Fair Queuing)                                       │
│     • 예산 기반 공정 큐잉                                               │
│     • 저지연, 고품질 스트리밍                                            │
│                                                                     │
│  5. mq-deadline (멀티큐 데드라인)                                      │
│     • NVMe/SSD용 멀티큐                                                │
│                                                                     │
│  【SSD와 스케줄링】                                                      │
│  ──────────────────                                                  │
│  SSD는 기계적 탐색이 없음:                                               │
│  • NOOP 또는 none 스케줄러 사용                                         │
│  • deadline도 가능                                                      │
│  • 불필요한 정렬 작업 피하기                                               │
│                                                                     │
│  $ echo none > /sys/block/nvme0n1/queue/scheduler                   │
│                                                                     │
│  【성능 모니터링】                                                      │
│  ──────────────────                                                  │
│  // I/O 통계 확인                                                        │
│  $ iostat -x 1                                                       │
│  Device   await  r_await  w_await  svctm  %util                    │
│  sda      5.23    3.12    8.45   1.23   12.34                      │
│  // await: 평균 대기 시간 (ms)                                           │
│  // svctm: 평균 서비스 시간 (ms)                                          │
│                                                                     │
│  // I/O 대기열 확인                                                       │
│  $ cat /sys/block/sda/queue/nr_requests                             │
│  256                                                                 │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 목적: 디스크 탐색 시간 최소화
• 시간: 탐색 + 회전지연 + 전송
• FCFS: 요청 순서대로, 공정하지만 느림
• SSTF: 가장 가까운 것부터, 기아 가능
• SCAN: 엘리베이터, 한 방향으로 이동
• C-SCAN: 원형, 균등한 서비스
• LOOK: 끝까지 안 감
• SSD: NOOP 또는 none 사용
• 리눅스: CFQ, Deadline, BFQ, NOOP
```
