# 481. 파일 잠금 (File Locking)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 파일 동시 접근 제어 메커니즘
> 2. **가치**: 데이터 무결성 보호
> 3. **융합**: 동시성 제어, NFS와 연관

---

## Ⅰ. 개요

### 개념 정의

파일 잠금(File Locking)은 **여러 프로세스가 동시에 같은 파일에 접근할 때 데이터 무결성을 보장하는 메커니즘**이다.

### 💡 비유: 회의실 예약
파일 잠금은 **회의실 예약**과 같다. 누군가 사용 중이면 다른 사람은 들어갈 수 없다.

### 파일 잠금 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                파일 잠금                                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【잠금 종류】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  1. 권고 잠금 (Advisory Locking):                                      │ │
│  │     • 협력하는 프로세스만 영향                                          │ │
│  │     • 잠금 무시 가능                                                    │ │
│  │     • 유닉스/리눅스 기본                                                │ │
│  │     • flock(), fcntl()                                                 │ │
│  │                                                             │ │
│  │  2. 강제 잠금 (Mandatory Locking):                                     │ │
│  │     • 커널이 강제로 잠금 적용                                            │ │
│  │     • 모든 프로세스에 영향                                              │ │
│  │     • 특별한 권한 설정 필요                                              │ │
│  │     • 성능 오버헤드                                                    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【잠금 모드】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │  읽기 잠금 (공유 잠금, Shared Lock)                       │    │ │
│  │  │  • 여러 프로세스가 동시 읽기 가능                          │    │ │
│  │  │  • 쓰기 잠금 차단                                         │    │ │
│  │  │  • LOCK_SH (flock), F_RDLCK (fcntl)                      │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                                                             │ │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │  쓰기 잠금 (배타 잠금, Exclusive Lock)                      │    │ │
│  │  │  • 하나의 프로세스만 접근 가능                              │    │ │
│  │  │  • 다른 모든 잠금 차단                                    │    │ │
│  │  │  • LOCK_EX (flock), F_WRLCK (fcntl)                      │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                                                             │ │
│  │  호환성:                                                                │
│  │  읽기 + 읽기 = OK                                                      │ │
│  │  읽기 + 쓰기 = 차단                                                     │ │
│  │  쓰기 + 쓰기 = 차단                                                     │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                파일 잠금 상세                                          │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【flock vs fcntl】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  flock (BSD 스타일):                                                  │ │
│  │  • 파일 전체 잠금                                                       │ │
│  │  • flock 구조체 사용                                                   │ │
│  │  • NFS에서 제한적 지원                                                  │ │
│  │  • 간단한 API                                                          │ │
│  │                                                             │ │
│  │  fcntl (POSIX 스타일):                                                │ │
│  │  • 바이트 범위 잠금 (레코드 잠금)                                        │ │
│  │  • struct flock 사용                                                   │ │
│  │  • NFS에서 지원                                                        │ │
│  │  • 더 유연함                                                           │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【잠금 동작 옵션】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  BLOCKING (차단):                                                      │ │
│  │  • 잠금 획득까지 대기                                                   │ │
│  │  • LOCK_SH, LOCK_EX                                                    │ │
│  │                                                             │ │
│  │  NON-BLOCKING (비차단):                                                │ │
│  │  • 잠금 획득 실패 시 즉시 반환                                           │ │
│  │  • LOCK_SH | LOCK_NB, LOCK_EX | LOCK_NB                                │ │
│  │  • F_SETLK (fcntl)                                                     │ │
│  │                                                             │ │
│  │  UNLOCK:                                                               │ │
│  │  • 잠금 해제                                                           │ │
│  │  • LOCK_UN (flock), F_UNLCK (fcntl)                                    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【교착 상태 (Deadlock)】                                              │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  프로세스 A: 파일 1 잠금 → 파일 2 대기                                 │ │
│  │  프로세스 B: 파일 2 잠금 → 파일 1 대기                                 │ │
│  │  → 교착 상태 발생                                                      │ │
│  │                                                             │ │
│  │  해결책:                                                               │ │
│  │  • 항상 같은 순서로 잠금 획득                                           │ │
│  │  • 타임아웃 사용                                                       │ │
│  │  • 비차단 모드 사용                                                    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【flock 사용】                                                      │
│  ──────────────────                                                  │
│  #include <sys/file.h>                                              │
│  #include <fcntl.h>                                                  │
│                                                                     │
│  int fd = open("file.txt", O_RDWR);                                 │
│                                                                     │
│  // 공유 잠금 (읽기)                                                  │
│  if (flock(fd, LOCK_SH) == -1) {                                    │
│      perror("flock");                                                │
│  }                                                                   │
│  // 읽기 작업...                                                      │
│  flock(fd, LOCK_UN);  // 잠금 해제                                   │
│                                                                     │
│  // 배타 잠금 (쓰기)                                                  │
│  if (flock(fd, LOCK_EX) == -1) {                                    │
│      perror("flock");                                                │
│  }                                                                   │
│  // 쓰기 작업...                                                      │
│  flock(fd, LOCK_UN);                                                 │
│                                                                     │
│  // 비차단 모드                                                       │
│  if (flock(fd, LOCK_EX | LOCK_NB) == -1) {                          │
│      if (errno == EWOULDBLOCK) {                                     │
│          printf("File is locked by another process\n");              │
│      }                                                                │
│  }                                                                   │
│                                                                     │
│  【fcntl 사용 (레코드 잠금)】                                          │
│  ──────────────────                                                  │
│  #include <fcntl.h>                                                  │
│                                                                     │
│  struct flock fl;                                                    │
│                                                                     │
│  // 쓰기 잠금 (바이트 0~99)                                           │
│  fl.l_type = F_WRLCK;    // F_RDLCK, F_WRLCK, F_UNLCK               │
│  fl.l_whence = SEEK_SET; // 기준점                                    │
│  fl.l_start = 0;         // 시작 오프셋                                │
│  fl.l_len = 100;         // 길이 (0 = 파일 끝까지)                     │
│  fl.l_pid = 0;           // F_GETLK 시 설정됨                         │
│                                                                     │
│  // 잠금 설정                                                          │
│  if (fcntl(fd, F_SETLK, &fl) == -1) {                               │
│      if (errno == EACCES || errno == EAGAIN) {                       │
│          printf("Cannot acquire lock\n");                            │
│      }                                                                │
│  }                                                                   │
│                                                                     │
│  // 차단 모드로 잠금 대기                                              │
│  fcntl(fd, F_SETLKW, &fl);  // W = wait                               │
│                                                                     │
│  // 잠금 해제                                                          │
│  fl.l_type = F_UNLCK;                                                │
│  fcntl(fd, F_SETLK, &fl);                                            │
│                                                                     │
│  【잠금 상태 확인】                                                    │
│  ──────────────────                                                  │
│  // fcntl로 잠금 정보 얻기                                             │
│  struct flock fl;                                                    │
│  fl.l_type = F_WRLCK;                                                │
│  fl.l_whence = SEEK_SET;                                             │
│  fl.l_start = 0;                                                      │
│  fl.l_len = 0;                                                        │
│                                                                     │
│  if (fcntl(fd, F_GETLK, &fl) == -1) {                               │
│      perror("fcntl");                                                │
│  }                                                                   │
│  if (fl.l_type == F_UNLCK) {                                         │
│      printf("No lock on file\n");                                    │
│  } else {                                                             │
│      printf("Locked by PID %d\n", fl.l_pid);                        │
│  }                                                                   │
│                                                                     │
│  【유틸리티】                                                          │
│  ──────────────────                                                  │
│  // flock 명령어                                                       │
│  $ flock -x /tmp/lockfile -c "command"  # 배타 잠금                   │
│  $ flock -s /tmp/lockfile -c "command"  # 공유 잠금                   │
│  $ flock -n /tmp/lockfile -c "command"  # 비차단                      │
│                                                                     │
│  // 셸 스크립트에서 사용                                               │
│  (                                                                   │
│      flock -x 200                                                    │
│      echo "Critical section"                                         │
│  ) 200>/tmp/lockfile                                                │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
