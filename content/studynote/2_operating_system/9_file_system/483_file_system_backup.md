# 483. 파일 시스템 백업 (File System Backup)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터 복사본 생성으로 보호
> 2. **가치**: 데이터 손실 방지와 복구
> 3. **융합**: 파일 시스템, 아카이빙과 연관

---

## Ⅰ. 개요

### 개념 정의

파일 시스템 백업(File System Backup)은 **데이터 손실 시 복구를 위해 파일 시스템의 복사본을 생성하는 작업**이다.

### 💡 비유: 보험
파일 시스템 백업은 **보험**과 같다. 문제가 생기기 전에는 불필요해 보이지만, 문제 후에는 필수적이다.

### 파일 시스템 백업 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                파일 시스템 백업                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【백업 유형】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  1. 전체 백업 (Full Backup):                                          │ │
│  │     • 모든 데이터 복사                                                │ │
│  │     • 시간 오래 걸림                                                  │ │
│  │     • 복구 간단                                                       │ │
│  │     • 많은 저장 공간 필요                                             │ │
│  │                                                             │ │
│  │  2. 증분 백업 (Incremental Backup):                                   │ │
│  │     • 마지막 백업 이후 변경분만                                        │ │
│  │     • 빠르고 공간 효율적                                              │ │
│  │     • 복구 시 여러 단계 필요                                           │ │
│  │                                                             │ │
│  │  3. 차등 백업 (Differential Backup):                                  │ │
│  │     • 마지막 전체 백업 이후 변경분                                     │ │
│  │     • 증분보다 크지만 복구 쉬움                                        │ │
│  │                                                             │ │
│  │  4. 미러 백업 (Mirror Backup):                                        │ │
│  │     • 정확한 복사본 유지                                               │ │
│  │     • 삭제된 파일도 백업에서 삭제                                      │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【백업 전략 예시】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  GFS (Grandfather-Father-Son) 전략:                                   │ │
│  │                                                             │ │
│  │  일      월      화      수      목      금      토                  │ │
│  │  ┌───┐  ┌───┐  ┌───┐  ┌───┐  ┌───┐  ┌───┐  ┌───┐               │ │
│  │  │증분│  │증분│  │증분│  │증분│  │증분│  │증분│  │전체│               │ │
│  │  │ D │  │ D │  │ D │  │ D │  │ D │  │ D │  │ W │               │ │
│  │  └───┘  └───┘  └───┘  └───┘  └───┘  └───┘  └───┘               │ │
│  │                                                             │ │
│  │  D = 일일 증분, W = 주간 전체                                          │ │
│  │  월간 전체은 매월 1일                                                  │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                파일 시스템 백업 상세                                   │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【백업 방식】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  1. 온라인 백업 (Hot Backup):                                         │ │
│  │     • 서비스 중단 없이 백업                                            │ │
│  │     • 일관성 보장 어려움                                               │ │
│  │     • 스냅샷/저널링 활용                                               │ │
│  │                                                             │ │
│  │  2. 오프라인 백업 (Cold Backup):                                      │ │
│  │     • 서비스 중단 후 백업                                              │ │
│  │     • 일관성 보장 확실                                                 │ │
│  │     • 가용성 저하                                                      │ │
│  │                                                             │ │
│  │  3. 논리적 백업 (Logical Backup):                                     │ │
│  │     • 파일 단위로 백업                                                 │ │
│  │     • tar, rsync 등                                                   │ │
│  │     • 이식성 좋음                                                      │ │
│  │                                                             │ │
│  │  4. 물리적 백업 (Physical Backup):                                    │ │
│  │     • 블록/이미지 단위 백업                                            │ │
│  │     • dd, 파티션 복사                                                 │ │
│  │     • 빠른 복원                                                        │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【백업 대상 선정】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  백업 필요:                                                           │ │
│  │  • 사용자 데이터 (/home, /data)                                       │ │
│  │  • 설정 파일 (/etc)                                                   │ │
│  │  • 데이터베이스                                                        │ │
│  │  • 애플리케이션 데이터                                                 │ │
│  │  • 이메일, 로그 중요분                                                 │ │
│  │                                                             │ │
│  │  백업 불필요/제외:                                                     │ │
│  │  • /proc, /sys, /dev                                                  │ │
│  │  • /tmp, /var/tmp                                                     │ │
│  │  • 캐시 디렉토리                                                       │ │
│  │  • 쉽게 재생성 가능한 것들                                             │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【3-2-1 백업 규칙】                                                  │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  3: 데이터 복사본 3개 유지                                             │ │
│  │  2: 다른 저장 매체 2종류 사용                                          │ │
│  │  1: 오프사이트(원격) 복사본 1개                                        │ │
│  │                                                             │ │
│  │  예: 로컬 디스크 + 외장 HDD + 클라우드                                 │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【tar를 이용한 백업】                                                │
│  ──────────────────                                                  │
│  // 전체 백업                                                          │
│  $ tar -czvf backup_full_$(date +%Y%m%d).tar.gz /home /etc          │
│                                                                     │
│  // 증분 백업 (GNU tar)                                               │
│  $ tar --create --gzip --file=backup_inc_$(date +%Y%m%d).tar.gz \   │
│        --listed-incremental=snapshot.snar /home                      │
│                                                                     │
│  // 복원                                                              │
│  $ tar -xzvf backup_full_20240101.tar.gz                             │
│                                                                     │
│  【rsync를 이용한 동기화 백업】                                        │
│  ──────────────────                                                  │
│  // 기본 동기화                                                        │
│  $ rsync -avz /home/user/ /backup/home/                             │
│                                                                     │
│  // 증분 백업 스타일 (하드링크 활용)                                    │
│  $ rsync -av --link-dest=/backup/yesterday/ \                       │
│        /home/user/ /backup/today/                                   │
│                                                                     │
│  // 원격 백업                                                          │
│  $ rsync -avz -e ssh /home/user/ user@server:/backup/               │
│                                                                     │
│  // 제외 패턴                                                          │
│  $ rsync -avz --exclude='*.tmp' --exclude='cache/' \                │
│        /home/user/ /backup/                                         │
│                                                                     │
│  【dd를 이용한 이미지 백업】                                           │
│  ──────────────────                                                  │
│  // 파티션 이미지                                                      │
│  $ sudo dd if=/dev/sda1 of=/backup/sda1.img bs=4M                   │
│                                                                     │
│  // 압축 포함                                                          │
│  $ sudo dd if=/dev/sda1 | gzip > /backup/sda1.img.gz                │
│                                                                     │
│  // 복원                                                              │
│  $ sudo gzip -dc /backup/sda1.img.gz | dd of=/dev/sda1              │
│                                                                     │
│  【자동화 스크립트】                                                    │
│  ──────────────────                                                  │
│  #!/bin/bash                                                          │
│  # daily_backup.sh                                                     │
│  DATE=$(date +%Y%m%d)                                                 │
│  BACKUP_DIR="/backup"                                                 │
│  SOURCE="/home /etc"                                                  │
│                                                                     │
│  # 전체 백업 (일요일)                                                  │
│  if [ $(date +%u) -eq 7 ]; then                                       │
│      tar -czvf $BACKUP_DIR/full_$DATE.tar.gz $SOURCE                 │
│      # 30일 이상 된 백업 삭제                                           │
│      find $BACKUP_DIR -name "full_*.tar.gz" -mtime +30 -delete       │
│  else                                                                 │
│      # 증분 백업                                                        │
│      tar -czvf $BACKUP_DIR/inc_$DATE.tar.gz $SOURCE                  │
│      find $BACKUP_DIR -name "inc_*.tar.gz" -mtime +7 -delete         │
│  fi                                                                   │
│                                                                     │
│  // crontab 등록                                                       │
│  0 2 * * * /usr/local/bin/daily_backup.sh                            │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 데이터 복사본 생성으로 보호
• 유형: 전체, 증분, 차등, 미러
• 방식: 온라인/오프라인, 논리/물리
• 도구: tar, rsync, dd
• 전략: GFS, 3-2-1 규칙
• 대상: 사용자 데이터, 설정, DB
• 제외: /proc, /sys, /tmp, 캐시
• 자동화: cron, 스크립트
```
