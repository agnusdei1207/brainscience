# 487. 암호화 파일 시스템 (Encrypted File System)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 파일 시스템 레벨에서 데이터 암호화
> 2. **가치**: 저장 데이터 보호
> 3. **융합**: 암호화, 보안, 파일 시스템과 연관

---

## Ⅰ. 개요

### 개념 정의

암호화 파일 시스템(Encrypted File System)은 **파일 시스템 수준에서 데이터를 암호화하여 저장하는 보안 기술**이다.

### 💡 비유: 금고
암호화 파일 시스템은 **금고**와 같다. 내용물을 넣을 때 자동으로 잠기고, 꺼낼 때 열쇠로 연다.

### 암호화 파일 시스템 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                암호화 파일 시스템                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【암호화 계층】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │               응용 프로그램                               │    │ │
│  │  │  read/write 평문 데이터                                   │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                          │                                       │ │
│  │                          ▼                                       │ │
│  │  ═══════════════════════════════════════════════════════════════│ │
│  │  ║           암호화 파일 시스템 (투명한 암호화)               ║    │ │
│  │  ═══════════════════════════════════════════════════════════════│ │
│  │        쓰기: 평문 → 암호화                                      │ │
│  │        읽기: 암호문 → 복호화                                    │ │
│  │                          │                                       │ │
│  │                          ▼                                       │ │
│  │  ┌─────────────────────────────────────────────────────────┐    │ │
│  │  │               디스크                                     │    │ │
│  │  │  암호화된 데이터만 저장                                    │    │ │
│  │  │  [암호문][암호문][암호문][암호문]                          │    │ │
│  │  └─────────────────────────────────────────────────────────┘    │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【암호화 방식】                                                        │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  1. 전체 디스크 암호화 (Full Disk Encryption):                        │ │
│  │     • 파티션/디스크 전체 암호화                                         │ │
│  │     • DM-Crypt + LUKS, BitLocker, FileVault                         │ │
│  │     • 부팅 시 복호화 키 입력                                            │ │
│  │                                                             │ │
│  │  2. 파일 시스템 암호화:                                                │ │
│  │     • 파일 시스템 내에서 암호화                                         │ │
│  │     • eCryptfs, EncFS, fscrypt                                       │ │
│  │     • 파일/디렉토리 단위 제어 가능                                      │ │
│  │                                                             │ │
│  │  3. 스택블 암호화:                                                     │ │
│  │     • 기존 파일 시스템 위에 암호화 계층                                  │ │
│  │     • eCryptfs                                                        │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                암호화 파일 시스템 상세                                 │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【주요 기술】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  DM-Crypt + LUKS (Linux):                                             │ │
│  │  • 커널 레벨 블록 장치 암호화                                          │ │
│  │  • AES-XTS 기본                                                      │ │
│  │  • LUKS 헤더로 키 관리                                                │ │
│  │  • 전체 파티션 암호화                                                  │ │
│  │                                                             │ │
│  │  BitLocker (Windows):                                                 │ │
│  │  • TPM과 통합                                                        │ │
│  │  • AES-128/256                                                       │ │
│  │  • 전체 볼륨 암호화                                                    │ │
│  │                                                             │ │
│  │  FileVault (macOS):                                                   │ │
│  │  • AES-XTS                                                           │ │
│  │  • iCloud 키 복구                                                    │ │
│  │  • APFS와 통합                                                       │ │
│  │                                                             │ │
│  │  fscrypt (Linux):                                                     │ │
│  │  • 파일 시스템 네이티브 암호화                                         │ │
│  │  • ext4, F2FS, UBIFS 지원                                            │ │
│  │  • 디렉토리 단위 암호화                                                │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  장점:                                                               │
│  │  • 저장 데이터 보호 (at-rest encryption)                              │ │
│  │  • 분실/도난 시 데이터 보호                                            │ │
│  │  • 규정 준수 (GDPR, HIPAA 등)                                         │ │
│  │  • 투명한 암호화 (애플리케이션 수정 불필요)                             │ │
│  │                                                             │ │
│  │  단점:                                                               │
│  │  • CPU 오버헤드                                                      │ │
│  │  • 키 분실 시 데이터 복구 불가                                         │ │
│  │  • 부팅 시간 증가                                                     │ │
│  │  • 약간의 성능 저하                                                   │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【LUKS 암호화 볼륨 생성】                                            │
│  ──────────────────                                                  │
│  // 암호화 볼륨 생성                                                    │
│  $ sudo cryptsetup luksFormat /dev/sdb1                             │
│  $ sudo cryptsetup luksOpen /dev/sdb1 encrypted                     │
│  $ sudo mkfs.ext4 /dev/mapper/encrypted                             │
│  $ sudo mount /dev/mapper/encrypted /mnt/secure                     │
│                                                                     │
│  // 상태 확인                                                          │
│  $ sudo cryptsetup status encrypted                                 │
│  $ sudo cryptsetup luksDump /dev/sdb1                              │
│                                                                     │
│  // 마운트 해제                                                        │
│  $ sudo umount /mnt/secure                                           │
│  $ sudo cryptsetup luksClose encrypted                              │
│                                                                     │
│  【fscrypt 설정 (ext4)】                                               │
│  ──────────────────                                                  │
│  // 파일 시스템에 암호화 활성화                                         │
│  $ sudo tune2fs -O encrypt /dev/sda1                                │
│  $ sudo mount -o remount /                                           │
│                                                                     │
│  // fscrypt 설치                                                      │
│  $ sudo apt install fscrypt                                          │
│  $ sudo fscrypt setup                                                │
│                                                                     │
│  // 디렉토리 암호화                                                    │
│  $ fscrypt encrypt /home/user/private                               │
│                                                                     │
│  // 암호화 상태 확인                                                    │
│  $ fscrypt status /home/user/private                                │
│                                                                     │
│  【eCryptfs (홈 디렉토리 암호화)】                                     │
│  ──────────────────                                                  │
│  // 설치                                                              │
│  $ sudo apt install ecryptfs-utils                                  │
│                                                                     │
│  // 디렉토리 암호화                                                    │
│  $ sudo mount -t ecryptfs /secret /secret                           │
│  // 마운트 시 암호 설정                                                 │
│                                                                     │
│  // 홈 디렉토리 암호화 (설치 시)                                        │
│  $ ecryptfs-migrate-home -u username                                │
│                                                                     │
│  【BitLocker (윈도우)】                                                │
│  ──────────────────                                                  │
│  // 관리자 권한으로                                                     │
│  > manage-bde -on C:                                                 │
│  > manage-bde -on C: -rp  // 복구 암호 생성                           │
│                                                                     │
│  // 상태 확인                                                          │
│  > manage-bde -status C:                                             │
│                                                                     │
│  // 잠금 해제 (재부팅 시)                                               │
│  > manage-bde -unlock C: -rp <복구암호>                              │
│                                                                     │
│  【성능 고려사항】                                                      │
│  ──────────────────                                                  │
│  // AES-NI 지원 확인                                                   │
│  $ cat /proc/cpuinfo | grep aes                                     │
│  // 하드웨어 가속 있으면 성능 영향 최소                                  │
│                                                                     │
│  // 벤치마크                                                           │
│  $ sudo cryptsetup benchmark                                         │
│  // 암호화 알고리즘 성능 비교                                           │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 파일 시스템 레벨 데이터 암호화
• 방식: 전체 디스크, 파일 시스템, 스택블
• Linux: DM-Crypt+LUKS, fscrypt, eCryptfs
• Windows: BitLocker
• macOS: FileVault
• 알고리즘: AES-XTS 기본
• 장점: 저장 데이터 보호, 규정 준수
• 단점: CPU 오버헤드, 키 분실 위험
• 팁: AES-NI 지원 CPU 사용
```
