# 488. 분산 파일 시스템 (Distributed File System)

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 여러 서버에 분산된 파일 저장소
> 2. **가치**: 확장성, 가용성, 성능
> 3. **융합**: 네트워크, 클러스터링과 연관

---

## Ⅰ. 개요

### 개념 정의

분산 파일 시스템(Distributed File System)은 **여러 서버에 파일을 분산 저장하고 단일 파일 시스템처럼 접근할 수 있는 시스템**이다.

### 💡 비유: 분점이 있는 체인점
분산 파일 시스템은 **체인점**과 같다. 여러 지점(서버)에 물건이 분산되어 있지만, 고객은 하나의 브랜드로 접근한다.

### 분산 파일 시스템 구조

```
┌─────────────────────────────────────────────────────────────────────┐
│                분산 파일 시스템                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【기본 구조】                                                          │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │              클라이언트들                                             │ │
│  │  ┌─────┐   ┌─────┐   ┌─────┐   ┌─────┐                          │ │
│  │  │ C1  │   │ C2  │   │ C3  │   │ C4  │                          │ │
│  │  └──┬──┘   └──┬──┘   └──┬──┘   └──┬──┘                          │ │
│  │     │         │         │         │                                │ │
│  │     └─────────┴────┬────┴─────────┘                               │ │
│  │                      │                                             │ │
│  │                      ▼                                             │ │
│  │  ═══════════════════════════════════════════════════════════════│ │
│  │  ║              메타데이터 서버 (MDS)                          ║    │ │
│  │  ║  • 파일 위치 정보                                           ║    │ │
│  │  ║  • 디렉토리 구조                                             ║    │ │
│  │  ║  • 접근 제어                                                 ║    │ │
│  │  ═══════════════════════════════════════════════════════════════│ │
│  │                      │                                             │ │
│  │          ┌───────────┼───────────┐                                │ │
│  │          │           │           │                                 │ │
│  │          ▼           ▼           ▼                                 │ │
│  │  ┌───────────┐ ┌───────────┐ ┌───────────┐                       │ │
│  │  │ 저장 서버  │ │ 저장 서버  │ │ 저장 서버  │                       │ │
│  │  │ (OSS 1)   │ │ (OSS 2)   │ │ (OSS 3)   │                       │ │
│  │  │ [A][B][C] │ │ [D][E][F] │ │ [G][H][I] │                       │ │
│  │  └───────────┘ └───────────┘ └───────────┘                       │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅱ. 상세 분석

### 상세 분석
```
┌─────────────────────────────────────────────────────────────────────┐
│                분산 파일 시스템 상세                                   │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【주요 분산 파일 시스템】                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  시스템       특징                    용도                               │ │
│  │  ────────     ────                    ────                                │ │
│  │  HDFS         Hadoop용, 높은 내결함성    빅데이터 분석                      │ │
│  │  Lustre       고성능, 병렬 I/O           슈퍼컴퓨팅, HPC                    │ │
│  │  GlusterFS    소프트웨어 정의 스토리지    일반 분산 스토리지                 │ │
│  │  Ceph         통합 스토리지, 오브젝트     클라우드, 가상화                   │ │
│  │  GPFS/IBM     고성능, 상용               엔터프라이즈                       │ │
│  │  MooseFS      고가용성, 쉬운 확장        일반 목적                         │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【핵심 구성 요소】                                                    │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  1. 메타데이터 서버 (MDS):                                              │ │
│  │     • 파일 시스템 구조 관리                                             │ │
│  │     • 파일-블록 매핑                                                   │ │
│  │     • 권한, 속성 관리                                                  │ │
│  │                                                             │ │
│  │  2. 저장 서버 (OSS/Storage Node):                                      │ │
│  │     • 실제 데이터 저장                                                  │ │
│  │     • 블록/오브젝트 단위                                                │ │
│  │     • 데이터 복제                                                      │ │
│  │                                                             │ │
│  │  3. 클라이언트:                                                        │ │
│  │     • 파일 시스템 인터페이스                                            │ │
│  │     • 캐싱                                                             │ │
│  │     • 네트워크 통신                                                     │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  【장단점】                                                            │
│  ──────────────────                                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │  장점:                                                               │
│  │  • 수평적 확장 (Scale-out)                                             │ │
│  │  • 높은 가용성                                                         │ │
│  │  • 데이터 복제/내결함성                                                 │ │
│  │  • 병렬 I/O로 높은 성능                                                │ │
│  │                                                             │ │
│  │  단점:                                                               │
│  │  • 복잡한 설정/관리                                                    │ │
│  │  • 네트워크 의존성                                                     │ │
│  │  • 일관성 유지 오버헤드                                                │ │
│  │  • 디버깅 어려움                                                       │ │
│  │                                                             │ │
│  └───────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅲ. 실무 적용

### 구현 예시
```
┌─────────────────────────────────────────────────────────────────────┐
│                실무 적용                                            │ |
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【Ceph 설정】                                                          │
│  ──────────────────                                                  │
│  // Ceph 배포                                                          │
│  $ ceph-deploy new node1 node2 node3                                 │
│  $ ceph-deploy install node1 node2 node3                             │
│  $ ceph-deploy mon create-initial                                   │
│  $ ceph-deploy osd create node1:/dev/sdb                            │
│                                                                     │
│  // 상태 확인                                                          │
│  $ ceph -s                                                            │
│  cluster:                                                             │
│    id:     xxx                                                        │
│    health: HEALTH_OK                                                  │
│  services:                                                            │
│    mon: 3 daemons                                                     │
│    osd: 3 osds: 3 up, 3 in                                            │
│                                                                     │
│  // CephFS 생성                                                        │
│  $ ceph fs new myfs metadata data                                    │
│  $ ceph fs ls                                                         │
│                                                                     │
│  // 마운트                                                              │
│  $ mount -t ceph 10.0.0.1:/ /mnt/cephfs                             │
│                                                                     │
│  【GlusterFS 설정】                                                    │
│  ──────────────────                                                  │
│  // 설치                                                              │
│  $ sudo apt install glusterfs-server                                 │
│                                                                     │
│  // 클러스터 구성                                                       │
│  $ sudo gluster peer probe node2                                     │
│  $ sudo gluster peer probe node3                                     │
│  $ sudo gluster peer status                                          │
│                                                                     │
│  // 볼륨 생성                                                          │
│  $ sudo gluster volume create gv0 \                                  │
│        replica 3 node1:/data node2:/data node3:/data                │
│  $ sudo gluster volume start gv0                                     │
│                                                                     │
│  // 마운트                                                              │
│  $ mount -t glusterfs node1:/gv0 /mnt/gluster                       │
│                                                                     │
│  【HDFS 설정】                                                          │
│  ──────────────────                                                  │
│  // Hadoop 설치 후                                                     │
│  $ hdfs namenode -format                                             │
│  $ start-dfs.sh                                                       │
│                                                                     │
│  // 상태 확인                                                          │
│  $ hdfs dfsadmin -report                                             │
│  $ hdfs dfs -ls /                                                     │
│                                                                     │
│  // 파일 저장                                                          │
│  $ hdfs dfs -put localfile /user/hadoop/                            │
│  $ hdfs dfs -get /user/hadoop/remotefile ./                         │
│                                                                     │
│  【적합한 사용 사례】                                                  │
│  ──────────────────                                                  │
│  • HDFS: 빅데이터 분석, Hadoop 생태계                                  │
│  • Lustre: 슈퍼컴퓨팅, 시뮬레이션                                      │
│  • Ceph: 클라우드 스토리지, OpenStack                                  │
│  • GlusterFS: 미디어 스토리지, 백업                                    │
│  • MooseFS: 일반 목적 분산 스토리지                                    │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Ⅳ. 기대효과 및 결론

### 핵심 요약

```
• 개념: 여러 서버에 분산된 파일 저장소
• 구성: MDS(메타데이터) + OSS(저장) + 클라이언트
• 종류: HDFS, Lustre, GlusterFS, Ceph, GPFS
• 장점: 확장성, 가용성, 병렬 I/O
• 단점: 복잡성, 네트워크 의존
• 용도: 빅데이터, 클라우드, HPC
• 마운트: 네이티브 클라이언트 또는 NFS
```
