## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 그룹화(Grouping)는 하나의 블록에 다수의 빈 블록 주소를 모아 관리하는 방식이며, 계수(Counting)는 연속된 빈 블록의 시작 주소와 개수를 쌍으로 관리하는 방식이다.
> 2. **가치**: 연결 리스트의 느린 탐색 속도를 개선하고, 비트맵의 거대한 공간 낭비를 줄이면서도 대용량의 연속된 공간을 빠르게 찾을 수 있게 한다.
> 3. **융합**: 현대 파일 시스템은 대형 파일의 효율적 저장을 위해 계수 방식의 아이디어를 확장한 Extents 기법을 널리 채택하고 있다.

---

## Ⅰ. 그룹화 방식 (Grouping)

- **개념**: 연결 리스트 방식의 변형이다. 첫 번째 가용 블록 내에 n개의 빈 블록 주소를 저장한다.
- **구조**:
  - 처음 n-1개는 실제 빈 블록의 주소다.
  - 마지막 n번째 주소는 '또 다른 주소 목록을 담고 있는 가용 블록'을 가리킨다.
- **장점**: 한 번의 디스크 읽기로 수많은 빈 블록의 위치를 즉시 파악할 수 있어 효율적이다.

📢 **섹션 요약 비유**: 그룹화는 "주소록 한 페이지"와 같다. 사람 한 명의 전화번호만 아는 게 아니라, 한 페이지에 수십 명의 번호를 적어두고 마지막엔 '다음 페이지로'라고 적어두는 것과 같다.

---

## Ⅱ. 계수 방식 (Counting)

- **개념**: 파일이 보통 연속적으로 할당된다는 점에 착안하여, [시작 블록 번호, 연속된 빈 블록 수]의 쌍으로 빈 공간을 관리한다.
- **구조**: `(start_block, count)`
- **장점**:
  - 연속된 공간을 할당할 때 압도적으로 효율적이다.
  - 관리해야 할 엔트리 수가 줄어들어 리스트의 크기가 매우 작아진다.

📢 **섹션 요약 비유**: 계수는 "아파트 동호수 안내"와 같다. "101호, 102호, 103호가 비었어요"라고 말하는 대신 "101호부터 3개 집이 비었어요"라고 짧게 말하는 지혜다.

---

## Ⅲ. 방식별 개념도 (ASCII Diagram)

```text
 [ Grouping Method ]                      [ Counting Method ]
 ┌───────────────────────────┐            ┌────────────────────────────┐
 │ Free Block #1 (Index)     │            │ Free Space Entry List      │
 ├───────────────────────────┤            ├────────────────────────────┤
 │ - Free Address: 12        │            │ - [Start: 50, Count: 5]    │──▶ (50, 51, 52, 53, 54)
 │ - Free Address: 15        │            │ - [Start: 92, Count: 2]    │──▶ (92, 93)
 │ - ...                     │            │ - [Start: 105, Count: 10]  │──▶ (105...114)
 │ - Next Group Block: #100  │──▶ Index   └────────────────────────────┘
 └───────────────────────────┘
```

📢 **섹션 요약 비유**: 그룹화는 "단체 사진"이고, 계수는 "줄자"와 같다. 단체 사진은 여러 명을 한 번에 보여주고, 줄자는 공간이 얼마나 긴지 한 번에 재어준다.

## Ⅳ. 성능 및 효율성 비교 (Performance Comparison)

| 항목 | 그룹화 (Grouping) | 계수 (Counting) |
| :--- | :--- | :--- |
| **탐색 효율** | 높음 (다수 주소 확보) | 매우 높음 (연속 공간 확보) |
| **공간 오버헤드** | 중간 | 낮음 (연속성이 높을수록) |
| **주요 타겟** | 흩어진 빈 공간 관리 | 연속된 빈 공간 관리 |
| **신뢰성** | 중간 | 높음 (구조가 단순함) |

📢 **섹션 요약 비유**: 그룹화는 "쇼핑카트(여러 물건 담기)"이고, 계수는 "패키지 여행(정해진 코스 묶기)"과 같다.

---

## Ⅴ. 현대 파일 시스템으로의 확장: Extents

현대 파일 시스템(Ext4, NTFS 등)은 계수 방식의 원리를 할당 구조에 직접 적용한 **Extents** 기법을 사용한다.
- **원리**: 파일의 주소를 블록 단위가 아닌 [시작, 길이] 단위로 저장하여 인덱스 블록의 크기를 줄이고 성능을 높인다.
- **의의**: 빈 공간 관리뿐만 아니라 실제 할당에서도 '연속성'이 성능의 핵심임을 보여주는 증거다.

📢 **섹션 요약 비유**: Extents는 "기차의 통째 대절"과 같다. 좌석 하나하나(블록) 예약하는 대신, 1번 칸부터 10번 칸까지 통째로(Extents) 빌리는 것이 훨씬 빠르고 관리가 쉬운 것과 같다.
