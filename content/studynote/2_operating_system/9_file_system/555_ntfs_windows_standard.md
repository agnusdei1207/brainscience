## 핵심 인사이트 (3줄 요약)
> 1. **본질**: NTFS(New Technology File System)는 Microsoft Windows의 표준 파일 시스템으로, 모든 객체(파일, 폴더)를 속성(Attribute)의 집합으로 취급하는 관계형 구조를 갖는다.
> 2. **가치**: MFT(Master File Table)를 통한 통합 메타데이터 관리와 트랜잭션 기반 저널링을 통해 데이터의 안정성을 확보하며, 정교한 ACL 기반 보안 설계를 제공한다.
> 3. **융합**: 압축, 암호화(EFS), 할당량 제한 등 엔터프라이즈 환경에 필수적인 부가 기능을 파일 시스템 수준에서 내장하여 윈도우 기반 인프라의 중추 역할을 수행한다.

---

## Ⅰ. NTFS의 정의와 진화 (Definition & Evolution)

- **정의**: NTFS는 Windows NT 3.1부터 도입된 고성능 저널링 파일 시스템이다.
- **진화**: FAT32의 4GB 파일 크기 제한 및 보안 부재를 해결하기 위해 등장했으며, 유니코드 지원과 대용량 하드 드라이브 최적화를 목표로 지속 업데이트되었다.
- **특징**: "Everything is an Attribute". 파일의 데이터조차 파일이 가진 여러 속성 중 하나(Data Attribute)로 취급된다.

📢 **섹션 요약 비유**: NTFS는 "모든 물건에 태그를 붙여 체계적으로 관리하는 똑똑한 백화점 물류 창고"와 같다.

---

## Ⅱ. NTFS의 핵심 구조: MFT (Master File Table)

NTFS 파일 시스템의 심장은 MFT이며, 모든 파일에 대한 정보가 이 테이블에 기록된다.

### 1. MFT 엔트리 구조
- 파일 1개당 최소 1개의 MFT 레코드(보통 1KB) 할당.
- 파일 이름, 시간 정보, 권한(ACL), 데이터 위치 정보가 포함됨.
- **Resident Data**: 파일 크기가 매우 작으면(수백 바이트) MFT 레코드 내부에 데이터를 직접 저장하여 속도를 높임.

### 2. 레이아웃 (ASCII Diagram)

```text
[ NTFS Volume Layout ]
┌──────────────┬──────────────────┬────────────────┬──────────────┐
│ Boot Sector  │ Master File Table│ System Files   │ File Area    │
│ ($Boot)      │ ($MFT)           │ ($LogFile 등)  │ (User Data)  │
└──────────────┴──────────────────┴────────────────┴──────────────┘
       │
       ▼ [ MFT Detail ]
┌──────────────┬──────────────┬──────────────┬──────────────┐
│ Record 0     │ Record 1     │ Record 2     │ Record N     │
│ ($MFT)       │ ($MFTMirr)   │ ($LogFile)   │ (User File)  │
└──────────────┴──────────────┴──────────────┴──────────────┘
```

📢 **섹션 요약 비유**: MFT는 "호텔의 마스터 키와 고객 명부"와 같다. 이것만 있으면 누가 어디 있는지 한눈에 알 수 있고 방(파일) 안으로 들어갈 수도 있다.

## Ⅲ. 신뢰성과 보안 기능 (Reliability & Security)

- **트랜잭션 저널링 (Journaling)**: 파일 시스템 변경 작업을 `$LogFile`에 미리 기록한다. 쓰기 도중 전원이 꺼져도 로그를 보고 파일 시스템의 모순을 해결한다.
- **보안 (ACLs)**: 파일/폴더별로 상세한 읽기, 쓰기, 실행 권한을 사용자 그룹 단위로 지정할 수 있다.
- **복구 (Self-healing)**: 윈도우 부팅 시 시스템 파일 손상을 감지하고 자동으로 수리를 시도한다.

📢 **섹션 요약 비유**: 저널링은 "장부에 기록한 뒤에만 물건을 내주는 꼼꼼한 회계사"와 같다.

---

## Ⅳ. 엔터프라이즈 확장 기능 (Enterprise Features)

- **EFS (Encrypting File System)**: 파일 시스템 수준에서 데이터를 암호화하여 물리적 도난 시에도 데이터를 보호한다.
- **Sparse Files**: 실제로 기록되지 않은 '0' 데이터 영역은 물리 공간을 차지하지 않게 하여 공간 효율을 높인다.
- **Disk Quotas**: 사용자별로 사용할 수 있는 디스크 용량을 제한하여 공유 서버의 자원 독점을 방지한다.
- **Volume Shadow Copy (VSS)**: 사용 중인 파일도 백업하거나 스냅샷을 찍을 수 있게 하는 서비스를 지원한다.

📢 **섹션 요약 비유**: NTFS의 부가 기능은 "보안 카메라, 암호 금고, 무게 제한 저울"이 모두 갖춰진 종합 보안 시설과 같다.

---

## Ⅴ. 기술적 한계 및 미래 (Conclusion)

- **한계**: 비-윈도우(Linux, macOS) 환경에서 쓰기 권한을 완벽히 지원하기 어렵고, 메타데이터 오버헤드로 인해 아주 작은 볼륨에서는 FAT보다 무거울 수 있다.
- **미래**: Microsoft는 NTFS의 한계를 넘는 차세대 파일 시스템 ReFS(Resilient File System)를 서버 제품군부터 점진적으로 도입하고 있다.

📢 **섹션 요약 비유**: NTFS는 "가장 신뢰받는 윈도우 나라의 법전"이지만, 가끔 "다른 나라(OS) 법전과는 말이 잘 안 통할 때"가 있다.

## 👶 아이를 위한 비유 (Child Analogy)
> "NTFS는 윈도우 나라의 **'꼼꼼한 창고지기 할아버지'**야. 누가 장난감을 빌려갔는지, 누가 만질 수 있는지 매일매일 '마법 수첩'(MFT)에 적어두시지. 만약 창고에 불이 꺼져서(전원 차단) 정리를 다 못해도, 할아버지는 수첩을 보고 금방 다시 정리를 끝내신단다. 그리고 비밀 편지(EFS)는 주인이 아니면 아무도 읽지 못하게 꽁꽁 숨겨주기도 해!"
