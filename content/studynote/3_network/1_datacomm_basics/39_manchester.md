# 맨체스터 (Manchester) 코딩 방식 분석

#### 핵심 인사이트 (3줄 요약)
> 1. **본질**: 비트 주기의 중간 지점에서 **반드시 전압 반전(Transition)**이 일어나도록 설계하여, 데이터와 클럭 정보를 동시에 전달하는 **자기 동기화(Self-clocking)** 라인 코딩 방식.
> 2. **가치**: 연속된 0이나 1이 와도 동기화가 깨지지 않으며 직류(DC) 성분이 없어 변압기 결합에 유리하지만, NRZ 대비 2배의 대역폭을 소모하는 특징이 있다.
> 3. **융합**: 고전 이더넷(10Base-T)의 표준으로 자리 잡았으며, 노이즈에 강한 **차동 맨체스터** 방식은 토큰 링(Token Ring) 등 고신뢰성 네트워크의 기반이 되었다.

---

### Ⅰ. 맨체스터 코딩의 종류 및 매커니즘

| 구분 | 표준 맨체스터 (Manchester) | 차동 맨체스터 (Diff. Manchester) |
|:---|:---|:---|
| **비트 결정** | 중간 전압의 **방향**에 따라 결정 | 비트 시작점의 **전환 유무**에 따라 결정 |
| **규칙** | 0: High->Low, 1: Low->High (IEEE 802.3) | **0: 시작점 전환 있음**, 1: 전환 없음 |
| **중간점 전환** | **무조건 발생** (클럭 제공용) | **무조건 발생** (클럭 제공용) |
| **주요 특징** | 구현이 매우 명확하고 단순함. | 잡음에 강하고 위상 반전(Polarity)에 무관함. |

---

### Ⅱ. 방식별 파형 비교 (ASCII)

데이터 '0 1 0'을 전송할 때의 차이다.

#### 1. 표준 맨체스터 (Standard)
```ascii
    Data:      0            1            0
    
    Signal: +---+        |   +---+    +---+
            |   |        |   |   |    |   |
    --------+   +--------+---+   +----+   +------- 0V
                |   |        |        |   |
                +---+        +--------+---+
            (H -> L)     (L -> H)     (H -> L)
```

#### 2. 차동 맨체스터 (Differential)
- 비트 시작 시 전압이 바뀌면 '0', 그대로면 '1' (중간 전환은 항상 있음).
```ascii
    Data:      0            1            0
    
    Signal: (Inv)        (Stay)       (Inv)
            +---+        +---+        |   +---+
            |   |        |   |        |   |   |
    --------+   +--------+   +--------+---+   +--- 0V
```

---

### Ⅲ. 맨체스터 코딩의 공학적 특징 분석

1. **대역폭 오버헤드**: 
   - 비트 하나에 두 번의 신호 변화가 가능하므로, 보드 레이트(Baud)가 비트 레이트(bps)의 2배가 됨. 
   - 대역폭 효율은 50%로 낮음.
2. **DC 밸런스**: 
   - 모든 비트가 +와 -를 균등하게 포함하므로 평균 전압이 0V에 수렴. 
   - 전력 공급과 데이터 전송을 동시에 하는 PoE(Power over Ethernet) 환경의 기초 사상이 됨.
3. **잡음 내성**: 
   - 전압의 절대 레벨이 아닌 '변화'를 읽으므로 외부 노이즈로 인한 레벨 변동에 강함.

---

### Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

#### 1. LAN 표준 선정 (10Mbps Ethernet)
- **현상**: 초기 LAN 환경에서 저렴한 케이블(UTP)로 안정적인 동기화가 필요함.
- **기술사적 결단**: 
  - 별도의 클럭 선을 깔기에는 비용이 너무 많이 듦. 
  - 대역폭 낭비를 감수하더라도 자기 동기화가 완벽한 **맨체스터 코딩**을 10Base-T 표준으로 채택하여 네트워크 구축의 경제성과 신뢰성을 동시에 확보한다.

#### 2. 기술사적 인사이트: 고속 이더넷에서의 변화
- 100Mbps 이상의 고속 이더넷에서는 맨체스터의 2배 대역폭 소모가 치명적임. 
- 따라서 고속망에서는 **MLT-3**나 **PAM-5**와 같은 더 복잡하지만 대역폭 효율적인 코딩으로 전환하고, 동기화는 **블록 코딩(4B/5B)**에 맡기는 분업 구조를 택해야 함을 설명해야 한다.

---

### Ⅴ. 기대효과 및 결론 (Future & Standard)

#### 1. 정량/정성 기대효과
- **완벽한 동기화**: 수신측 클럭 복원 회로(PLL)의 설계 단순화 및 안정화.
- **매체 독립성**: 변압기 절연이 필요한 모든 유선 매체에서 안정적 동작.

#### 2. 미래 전망
맨체스터 코딩은 낮은 효율성 때문에 기가비트급 이상에서는 쓰이지 않지만, 그 **'강력한 동기화'**와 **'DC 밸런스'** 특성 덕분에 RFID, 근거리 무선 통신(NFC), 적외선 리모컨 등 저속·고신뢰성 소형 기기 통신에서는 여전히 독보적인 위치를 차지하고 있다. 향후 초저전력 IoT 환경에서도 이 단순하고 강력한 철학은 계속 유지될 것이다.
