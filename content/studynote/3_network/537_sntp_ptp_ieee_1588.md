# 537. SNTP (Simple NTP) / PTP (Precision Time Protocol, IEEE 1588)

> **핵심 인사이트**: 기존의 NTP가 "1초 내외로 대충 시간 좀 맞춰줘"라면, PTP는 스마트 팩토리의 로봇 팔과 자율주행차가 0.000001초(마이크로초)라도 어긋나면 큰일 나는 환경에서, 스위치 칩셋 단위로 타이머를 잰 정밀도의 끝판왕이다.

## Ⅰ. SNTP (Simple Network Time Protocol)의 개념
NTP(Network Time Protocol)는 네트워크 지연 시간을 정밀하게 수학적으로 보정하느라 CPU 연산이 무겁습니다. **SNMP는 이 복잡한 알고리즘을 싹 빼고, 단순하게 "지금 몇 시야?" ➔ "지금 12시야"라고 시간만 받아오는 아주 가벼운 버전**입니다. PC나 공유기, 스마트 홈 기기처럼 밀리초 단위의 초정밀 오차가 중요하지 않은 일반 기기에서 광범위하게 쓰입니다.

## Ⅱ. PTP (Precision Time Protocol, IEEE 1588)의 등장
스마트 팩토리의 로봇 제어(TSN), 5G 기지국 간 동기화, 주식 초고빈도 거래(HFT) 시스템 등에서는 1밀리초(0.001초)의 오차도 허용되지 않습니다.
이러한 산업용 환경을 위해 IEEE 1588 표준으로 제정된 PTP는 **마이크로초(µs)에서 나노초(ns) 수준의 극단적인 초정밀 시간 동기화**를 제공하는 네트워크 프로토콜입니다.

## Ⅲ. PTP가 나노초 동기화를 이루는 비결 (하드웨어 타임스탬핑)
NTP는 소프트웨어(OS) 레벨에서 패킷의 출발/도착 시간을 기록하기 때문에 운영체제의 인터럽트 지연 등 오차가 큽니다.
반면 PTP는 랜카드(NIC)나 스위치의 **하드웨어(MAC 계층 칩셋) 단위에서 패킷이 선(Wire)을 빠져나가는 그 찰나의 순간을 하드웨어적으로 기록(Hardware Time-stamping)** 하므로, 소프트웨어 지연이 완벽하게 배제됩니다.

## Ⅳ. NTP와 PTP의 비교

| 구분 | NTP (Network Time Protocol) | PTP (Precision Time Protocol) |
|:---|:---|:---|
| **정확도** | 밀리초 (ms, $10^{-3}$초) 수준 | **마이크로초 (µs) ~ 나노초 (ns)** 수준 |
| **타임스탬핑 위치** | 소프트웨어 (OS, 애플리케이션 계층) | **하드웨어** (네트워크 인터페이스 카드 칩셋) |
| **주요 사용처** | 일반 PC, 서버 로그 동기화, 인터넷 통신 | 스마트 팩토리 로봇 제어(TSN), 5G 기지국, 금융망 |
| **운영 방식** | 계층적 서버 구조 (Stratum) | 마스터-슬레이브 구조 (Grandmaster Clock) |

> 📢 **섹션 요약 비유**: SNTP가 대충 "아침 먹을 시간쯤 됐네"라고 손목시계 힐끗 보는 수준이라면, NTP는 육상 경기에서 스톱워치로 달리기 기록을 재는 수준이고, PTP는 100m 달리기 결승선에서 사진 판독기(하드웨어 카메라)로 0.001초 차이까지 완벽하게 갈라내는 올림픽 계측 시스템입니다.
