# 538. SSH (Secure Shell) 포트 22 / Telnet (원격 접속) 포트 23 비교

> **핵심 인사이트**: 예전에는 서버에 원격 접속할 때 Telnet을 썼다. 하지만 해커가 중간에서 가로채면 내가 친 비밀번호 '1234'가 그대로 노출되는 참사가 벌어졌다. 이를 막기 위해 접속부터 비밀번호, 타이핑하는 모든 명령어를 완벽하게 암호화해서 숨기는 통신망이 SSH다.

## Ⅰ. 원격 접속(Remote Access) 프로토콜의 개념
네트워크 너머 멀리 떨어져 있는 서버나 라우터(Linux, Cisco 장비 등)의 터미널(CLI)에 내 PC로 접속하여, 마치 그 장비 바로 앞에 앉아 있는 것처럼 명령어를 입력하고 제어하기 위한 통신 프로토콜입니다. 

## Ⅱ. Telnet (텔넷) - 과거의 유물
- **포트 번호**: TCP **23번**
- **특징**: 인터넷 초창기에 개발된 프로토콜로, 빠르고 가볍습니다.
- **치명적 단점**: 사용자가 입력하는 아이디, 비밀번호, 쳐넣는 명령어(`rm -rf`) 등 모든 통신 내용이 **평문(Cleartext, 암호화되지 않은 텍스트)** 으로 전송됩니다. 와이어샤크(스니퍼)로 누군가 엿보면 해킹에 100% 무방비로 뚫리므로, 오늘날 실무에서는 절대 사용해서는 안 됩니다.

## Ⅲ. SSH (Secure Shell) - 현대의 표준
- **포트 번호**: TCP **22번**
- **특징**: Telnet의 보안 취약점을 완벽히 해결하기 위해 등장했습니다. 접속하는 순간부터 클라이언트와 서버 간에 **암호화된 강력한 보안 터널(Secure Tunnel)** 을 뚫고 그 안에서 통신합니다.

### SSH의 3대 보안 메커니즘
1. **대칭키 암호화**: AES 같은 대칭키 알고리즘을 사용해 통신 내용(명령어, 결과 화면)을 암호화하여 기밀성을 보장합니다.
2. **비대칭키(공개키) 인증**: 처음 접속할 때 서버가 가짜 서버(MitM 공격)가 아닌지 서버의 공개키로 인증하고 암호화 키를 안전하게 교환합니다.
3. **무결성 검사**: 해커가 중간에서 명령어를 변조(`ls` ➔ `rm`)하지 못하도록 MAC(메시지 인증 코드)을 달아 패킷이 온전함을 증명합니다.

## Ⅳ. 포트 포워딩 (SSH 터널링)
SSH의 보안 터널은 단순히 터미널 접속에만 쓰이지 않습니다. 이 터널 속에 VNC(원격 데스크톱)나 내부망 웹사이트 트래픽을 몰래 밀어 넣어(터널링), 밖에서 사내 폐쇄망에 안전하게 우회 접속하는 VPN과 같은 용도로도 널리 활용됩니다.

> 📢 **섹션 요약 비유**: Telnet은 멀리 있는 직원에게 지시를 내릴 때 엽서(평문)에다 "금고 비밀번호는 1234야, 문 열어!"라고 적어 보내서 우체부가 다 읽어볼 수 있는 방식입니다. 반면 SSH는 두꺼운 철제 금고(암호화 터널) 안에 지시서를 넣고 튼튼한 자물쇠를 채워 보내므로, 중간에 도둑이 가로채도 절대 내용을 열어볼 수 없는 안전한 통신망입니다.
