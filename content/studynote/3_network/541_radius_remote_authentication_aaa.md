# 541. RADIUS (Remote Authentication Dial-In User Service) - UDP 기반 AAA

> **핵심 인사이트**: 수백 대의 무선 공유기나 스위치마다 직원들의 계정을 일일이 만들어 관리하는 것은 악몽이다. RADIUS는 "이 사람이 접속해도 되는지"를 한 곳의 중앙 서버에 물어보는 네트워크 출입 통제소이며, 가볍고 빠른 UDP를 사용하지만 '비밀번호만' 암호화한다는 한계가 있다.

## Ⅰ. RADIUS의 개념
네트워크 리소스(라우터, 스위치, VPN 장비 등)에 접근하려는 사용자에 대해 중앙 집중화된 **AAA (Authentication 인증, Authorization 인가, Accounting 과금/로깅)** 서비스를 제공하는 표준 프로토콜입니다.

## Ⅱ. RADIUS의 기술적 특징
- **전송 계층 (UDP 기반)**: 기본적으로 가볍고 빠른 UDP 1812번(인증/인가), 1813번(과금) 포트를 사용합니다. (초기에는 1645/1646 포트를 사용하기도 했습니다.)
- **부분적 암호화**: 사용자의 **패스워드(Password) 부분만 MD5 해시로 암호화**하여 전송하고, 나머지 데이터(사용자 ID, 할당 IP, 인가 정보 등 패킷 본문)는 **평문(Cleartext)**으로 전송합니다. 이로 인해 보안성이 다소 취약합니다.
- **인증과 인가의 결합**: 인증(Authentication)과 인가(Authorization) 절차를 분리할 수 없으며, 하나의 패킷 교환(Access-Request / Access-Accept)으로 묶어서 동시에 처리합니다.

## Ⅲ. 실무 활용 (802.1X 연동 및 NAC)
- **무선/유선 네트워크 접근 제어 (NAC)**: 기업의 사내 무선 와이파이(WPA2/3-Enterprise)나 유선 랜 스위치(IEEE 802.1X 규격)에 직원이 접속할 때 주로 쓰입니다. 
- 접속자가 입력한 ID/PW(또는 인증서)를 스위치(Authenticator)가 가로채어 중앙의 RADIUS 서버로 보내 진짜 직원인지 확인(AD 연동 등)하는 백엔드 인증 용도로 가장 널리 사용됩니다.

> 📢 **섹션 요약 비유**: 전국 체인점 술집에서 신분증 검사를 할 때, 알바생(네트워크 스위치)이 직접 위조 여부를 검사하지 않고 신분증 사진을 찍어 본사 검열팀(RADIUS 서버)에 빠른 문자(UDP)로 묻는 것입니다. 단, 문자를 보낼 때 주민번호 뒷자리(비밀번호)만 모자이크로 암호화하고 이름이나 나이는 그냥 평문으로 보내는 방식입니다.
