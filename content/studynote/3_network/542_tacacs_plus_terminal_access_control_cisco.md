# 542. TACACS+ (Terminal Access Controller Access Control System Plus)

> **핵심 인사이트**: RADIUS가 일반 직원들을 위한 빠르고 가벼운 '와이파이 출입증'이라면, TACACS+는 시스템 관리자들이 '핵심 장비에 접속해 무슨 명령어를 치는지'까지 통제하고 감시하는 철통 보안의 관리자용 출입증이다.

## Ⅰ. TACACS+의 개념
Cisco 시스템즈가 개발한 독자적인(현재는 오픈 표준화 중인) AAA 프로토콜로, 주로 네트워크 장비(라우터, 방화벽, 스위치)에 원격 접속(CLI, SSH, Telnet)하여 설정(Configuration)을 변경하려는 **관리자들의 권한을 통제**하는 데 특화되어 있습니다.

## Ⅱ. TACACS+의 기술적 특징 (RADIUS와의 차별점)
1. **신뢰성 있는 TCP 통신**: UDP를 쓰는 RADIUS와 달리, 연결 지향적이고 패킷 유실을 방지하는 **TCP 49번 포트**를 사용하여 통신 신뢰성을 보장합니다.
2. **패킷 본문 전체 암호화 (Full Payload Encryption)**: 패스워드만 암호화하는 RADIUS와 달리, 사용자의 ID, 패스워드, 주고받는 명령어 등 **패킷의 페이로드(본문) 100%를 암호화**하여 스니핑을 완벽히 차단합니다.
3. **AAA의 완전한 분리**: 인증, 인가, 과금 모듈을 완전히 독립적으로 운영할 수 있습니다. 예를 들어, 인증(Auth)은 A 서버에서 받고, 인가(Authz)는 B 서버의 정책을 따르도록 유연하게 설계할 수 있습니다.

## Ⅲ. 명령어 레벨의 인가 (Command-level Authorization) 🌟
TACACS+의 가장 강력한 기능입니다. 관리자의 직급이나 역할에 따라 장비에서 **사용할 수 있는 명령어를 세밀하게(글자 단위로) 제한**할 수 있습니다.
- *예시*: 주니어 엔지니어 그룹은 장비 상태를 보는 `show` 명령어 계열만 실행 가능하고, 장비 설정을 바꾸는 `config terminal`이나 장비를 재부팅하는 `reload` 명령어는 입력 즉시 차단되도록 중앙에서 통제합니다.

## Ⅳ. RADIUS vs TACACS+ 비교 요약

| 구분 | RADIUS | TACACS+ |
| :--- | :--- | :--- |
| **전송 프로토콜** | UDP (1812, 1813) | **TCP (49)** |
| **암호화 범위** | **패스워드만** 암호화 (나머지는 평문) | **전체 패킷 본문** 암호화 |
| **인증 / 인가** | 결합되어 분리 불가 | **완전히 분리 가능** |
| **주요 사용 대상** | 802.1X 등 **네트워크 접속 일반 사용자** | 라우터/스위치에 접속하는 **네트워크 관리자** |

> 📢 **섹션 요약 비유**: TACACS+는 원자력 발전소 통제실의 깐깐한 보안 시스템입니다. 보안요원(TCP)이 출입자의 가방 안까지 100% 엑스레이 검사(전체 암호화)를 하며, 신원 확인(인증)을 통과한 뒤에도 "이 직원은 모니터 1번 버튼(show)은 누를 수 있지만, 시스템을 끄는 빨간색 폭발 버튼(reload)은 누를 수 없음(명령어 단위 인가)"을 철저하게 감시하고 기록합니다.
