# 545. 커버로스 (Kerberos) - KDC / 티켓 기반 인증 체계

> **핵심 인사이트**: 네트워크에 비밀번호를 직접 날리면 해커가 가로채서(스니핑) 그대로 다시 쓴다(재전송 공격). 커버로스는 비밀번호 대신, 유효기간이 1시간짜리인 '암호화된 일회용 놀이공원 티켓'을 발급해 입장시키는 시스템이다. 티켓에는 타임스탬프(시간)가 찍혀있어 해커가 주워도 금방 휴지조각이 된다.

## Ⅰ. 커버로스(Kerberos)의 개념
- **어원**: 그리스 신화에 나오는 지옥의 문을 지키는 머리 셋 달린 개(Kerberos)에서 유래했습니다.
- **정의**: MIT에서 개발한 분산 환경을 위한 **티켓(Ticket) 기반의 네트워크 인증 프로토콜**입니다. Microsoft Active Directory의 기본 인증 프로토콜로 사용됩니다. (TCP/UDP 88 포트)

## Ⅱ. 핵심 구성 요소 (머리 셋 달린 개)
커버로스는 크게 3개의 논리적 역할(머리)로 구성됩니다.
1. **KDC (Key Distribution Center, 키 분배 센터)**: 커버로스 시스템의 심장부로, 암호키를 관리하고 티켓을 발행하는 신뢰할 수 있는 제3의 중앙 인증 서버입니다. 내부적으로 AS와 TGS 두 가지 역할을 수행합니다.
2. **AS (Authentication Server, 인증 서버)**: 클라이언트의 패스워드(해시)를 확인하여 최초 로그인을 처리하고, 임시 통행증인 **TGT(Ticket Granting Ticket)**를 발급합니다.
3. **TGS (Ticket Granting Server, 티켓 부여 서버)**: 클라이언트가 TGT를 보여주면 확인한 뒤, 최종 목적지(특정 서비스/서버)에 접근할 수 있는 **Service Ticket (서비스 티켓)**을 발급해줍니다.

## Ⅲ. 통신 절차 (티켓 발급 매커니즘)
1. **최초 로그인**: 사용자가 ID를 입력하면 KDC 안의 AS가 확인 후, 사용자만 풀 수 있는 키로 암호화된 **TGT(티켓 발급용 티켓)**를 줍니다.
2. **서비스 요청**: 사용자가 사내 파일 서버에 접속하고 싶을 때, 아까 받은 TGT를 TGS에게 제시합니다.
3. **서비스 티켓 수령**: TGS는 TGT의 유효성을 검사하고, "파일 서버 전용 탑승권"인 **Service Ticket**을 발급합니다.
4. **서비스 접근**: 사용자는 서비스 티켓을 파일 서버에 제출하여 안전하게 접속(인가)합니다.

## Ⅳ. 핵심 보안 메커니즘 (재전송 공격 방지) 🌟
- **대칭키 암호화**: 티켓을 포함한 모든 통신은 노드 간에 공유된 세션 키(대칭키)로 여러 겹 강력하게 암호화됩니다. 사용자의 **패스워드 원문은 네트워크에 절대 전송되지 않습니다**.
- **타임스탬프 (Timestamp)**: 모든 티켓에는 발급된 '시간'과 짧은 '유효 기간(Lifespan, 보통 수 분~몇 시간)'이 찍혀 있습니다. 이를 통해 해커가 중간에 암호화된 티켓 패킷을 통째로 가로채서 나중에 다시 서버로 보내는 **재전송 공격(Replay Attack)**을 완벽히 방어합니다. 
  - *(※ 이 타임스탬프 검증 때문에 커버로스 환경에서는 모든 서버 간의 철저한 시간 동기화(NTP)가 필수적입니다.)*

> 📢 **섹션 요약 비유**: 에버랜드(KDC) 정문 매표소(AS)에서 신분증을 보여주면 손목에 차는 '자유이용권 팔찌(TGT)'를 채워줍니다. 이 팔찌를 차고 롤러코스터 앞의 검표원(TGS)에게 보여주면 '롤러코스터 전용 1회용 탑승권(Service Ticket)'을 줍니다. 탑승권에는 현재 시간(Timestamp)이 적혀 있어서 유효기간 10분이 지나면 무효가 됩니다. 따라서 해커가 땅에 떨어진 탑승권을 주워서 다음 날 쓰려고 해도 절대 입장할 수 없습니다.
