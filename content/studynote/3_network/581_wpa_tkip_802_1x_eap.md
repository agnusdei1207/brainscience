# 581. WPA (TKIP + 802.1X + EAP) - WEP의 대안

> **핵심 인사이트**: 낡은 WEP 암호가 5분 만에 뚫리는 사태가 벌어지자, 전 세계 공유기를 다 갖다 버릴 수는 없으니 급하게 만든 "소프트웨어 땜질용 최강 패치"가 WPA다. WEP처럼 같은 비밀번호를 재탕하지 않고, 10,000개의 패킷마다 비밀번호를 계속 무작위로 갈아치워 도둑이 패턴을 외울 수 없게 만들었다.

## Ⅰ. WPA (Wi-Fi Protected Access)의 등장 배경
1997년에 만들어진 무선랜 보안 표준인 WEP가 심각한 암호학적 결함(RC4 IV 재사용)으로 허무하게 뚫리자, Wi-Fi Alliance는 정식 3세대 표준(802.11i, WPA2)이 완성되기 전까지 시장을 보호할 임시방편으로 **WPA**를 급히 발표했습니다. (기존 WEP 장비들의 소프트웨어 펌웨어 업데이트만으로도 적용할 수 있도록 가볍게 설계됨)

## Ⅱ. 핵심 암호화 기술: TKIP 🌟
- **TKIP (Temporal Key Integrity Protocol)**: WPA의 심장입니다.
- 기존 WEP와 똑같은 RC4 암호화 알고리즘을 그대로 쓰면서도, 치명적인 약점이었던 "정적 키(고정 비밀번호)" 문제를 **동적 키(Dynamic Key)**로 바꾸어 해결했습니다.
- 통신이 맺어지면, 단말기와 공유기가 알아서 **패킷 1만 개(약 몇 분) 단위로 서로 다른 암호화 키를 자동으로 생성하여 계속 바꿔버립니다.** 해커가 허공에서 패킷을 아무리 긁어모아도, 키가 계속 바뀌기 때문에 해독을 위한 수학적 패턴 조각을 맞출 수 없게 됩니다.

## Ⅲ. 무결성 검증 강화 (MIC)
- WEP는 데이터가 조작되었는지 확인하는 기능이 허술(CRC-32)했습니다.
- WPA는 **MIC (Message Integrity Check, 메시지 무결성 검사, 별명: Michael)**라는 강력한 위조 감지 코드를 추가하여, 해커가 암호화된 데이터를 중간에 가로채서 일부 비트를 변조해 기지국으로 다시 쏘는 행위를 완벽히 차단했습니다.

## Ⅳ. 기업용 WPA-Enterprise (802.1X + EAP)
WPA부터는 인증 방식을 두 가지로 깔끔하게 나누어 지원하기 시작했습니다.
1. **WPA-Personal (PSK 방식)**: 가정이나 소규모 카페에서 쓰는 방식. 모두가 똑같은 비밀번호 "12345678"을 치고 들어갑니다.
2. **WPA-Enterprise (802.1X/EAP 방식)**: 대기업에서 쓰는 방식. 공유기 자체 비밀번호가 없고, 직원이 자신의 '사내 사번과 개별 비밀번호'를 입력하면, 공유기가 이를 중앙의 RADIUS 인증 서버로 보내 진짜 직원인지 확인하는 거대한 보안 인프라입니다.

> 📢 **섹션 요약 비유**: WEP는 현관문 비밀번호를 한 번 '0000'으로 맞춰놓고 평생 쓰는 집입니다. 도둑이 옆에서 몰래 쳐다보다가 패턴을 알아내어 털어버립니다. WPA(TKIP)는 현관문에 'OTP(일회용 비밀번호) 생성기'를 달아놓은 것과 같습니다. 문을 10번 열 때마다 비밀번호가 '1592', '8421'로 알아서 계속 바뀌기 때문에, 도둑이 어제 본 비밀번호로 오늘 다시 문을 열려고 해도 절대 열리지 않습니다.
