# 583. WPA3 - SAE(비밀번호 무차별 대입 차단)와 OWE

> **핵심 인사이트**: 지금까지 와이파이는 비밀번호가 '1234'처럼 쉬우면 해커가 사전을 통째로 집어넣어 백만 번씩 찍어보는 공격에 허무하게 뚫렸다. WPA3는 아예 비밀번호를 몇 번 틀리면 자물쇠가 영원히 먹통이 되어버리는 특수 기술(SAE)을 도입했다. 이제 카페 와이파이 비밀번호가 '1234'여도 절대 해킹할 수 없다.

## Ⅰ. WPA3의 등장 배경
2018년에 발표된 무선 LAN 최신 보안 규격입니다. WPA2를 붕괴시킨 KRACK 공격의 충격, 그리고 사용자들이 여전히 "admin", "12345678" 같은 쉬운 비밀번호(사전 대입 공격에 취약)를 사용하는 인간적인 취약점을 완벽하게 보완하기 위해 탄생했습니다.

## Ⅱ. 핵심 기술 1: SAE (Simultaneous Authentication of Equals) 🌟
기존 WPA2의 4-way 핸드셰이크 과정을 버리고 도입된 **가장 혁신적인 인증 과정**입니다.
- **기존 WPA2의 한계**: 해커가 공기 중에서 와이파이 비밀번호가 섞인 인증 패킷 하나만 낚아챈 뒤 집으로 도망갑니다. 그리고 해커의 집 슈퍼컴퓨터로 오프라인에서 비밀번호를 1초에 수백만 번씩 대입(Offline Dictionary Attack)하면 쉬운 비밀번호는 1분이면 뚫립니다.
- **SAE의 해결책 (잠금 기능)**: 비밀번호를 맞히려면 반드시 **'현장에 있는 진짜 공유기(AP)와 실시간으로 실시간 핑퐁 통신(대화)'을 해야만 하도록 강제**합니다. 해커가 패킷을 집에 가져가서 풀어보는 오프라인 공격이 원천 차단되며, 현장(온라인)에서 비밀번호를 몇 번 틀리게 치면 공유기가 접속을 아예 영구 차단(Lock)해 버립니다. 비밀번호가 아무리 짧고 허접해도 해킹이 불가능해집니다.

## Ⅲ. 핵심 기술 2: OWE (Opportunistic Wireless Encryption) - 개방형 망 보안
- **기존의 문제**: 스타벅스나 공항의 "비밀번호 없는 개방형(Open) 와이파이"에 접속하면, 내가 스마트폰으로 검색하는 모든 내용이 공기 중으로 평문으로 날아가 누구나 다 훔쳐볼 수 있습니다.
- **OWE (Enhanced Open)**: 사용자는 비밀번호를 치지 않고 그냥 터치만 해서 편리하게 접속하지만, **내부적으로는 스마트폰과 공유기 간에 1:1로 튼튼한 개별 암호화 터널이 즉시 자동으로 생성**됩니다. 나 말고 다른 사람은 내 패킷을 캡처해도 열어볼 수 없습니다.

## Ⅳ. 코어 보안 스펙 (기업용 강력 보안)
- **PMF (Protected Management Frames) 필수 적용**: 해커가 가짜 '연결 해제(Deauth)' 패킷을 보내어 사람들의 와이파이를 강제로 끊어버리는 디도스 테러를 막기 위해, 접속 유지 제어 프레임도 100% 암호화하여 주고받습니다.
- **192비트 기업급 보안 (CNSA / Suite B)**: 정부 기관, 국방부, 금융권 등 초고도 보안이 필요한 곳을 위해 AES-192 등 훨씬 더 긴 키 길이를 사용하는 강력한 미국 국가보안국(NSA) 스펙을 지원합니다.

> 📢 **섹션 요약 비유**: WPA2 시절에는 현관문 자물쇠 패킷을 복사해서 해커 집으로 가져간 뒤, 드릴(슈퍼컴퓨터)로 100만 번 쑤셔보며 비밀번호를 여유롭게 찾아낼 수 있었습니다. WPA3(SAE)는 복사 자체가 불가능하며, 오직 현관문 앞에 서서만 비밀번호를 누를 수 있습니다. 게다가 5번 틀리면 경찰이 출동하고 키패드가 영원히 부서져 버리므로(Brute-force 무력화), 비밀번호가 '1111'이어도 도둑은 찍어서 문을 열 엄두도 내지 못합니다.
