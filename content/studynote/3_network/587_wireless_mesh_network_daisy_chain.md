# 587. 무선 메시 네트워크 (Wireless Mesh Network) - 데이지 체인 구조

> **핵심 인사이트**: 100평짜리 단독 주택에 와이파이를 터지게 하려고 방마다 공유기를 놨다 치자. 예전에는 이 5대의 공유기까지 일일이 긴 랜선을 끌어와서 꽂아줘야 했다(인프라 모드). 무선 메시 네트워크는 단 1대에만 랜선을 꽂아두면, 나머지 4대는 지들끼리 전파로 징검다리(데이지 체인)를 놔서 100평 전체를 거대한 하나의 와이파이존으로 자동 연결해 버리는 그물망 기술이다.

## Ⅰ. 무선 메시 네트워크 (WMN)의 개념
여러 대의 무선 접속 장치(AP, 라우터)들이 그물눈(Mesh)처럼 촘촘하게 서로 다대다(Multi-to-Multi) 무선 링크로 연결되어, **사용자가 물리적인 유선 케이블의 한계를 벗어나 넓은 지역에서 하나의 네트워크(동일한 SSID)를 로밍하며 사용할 수 있게 해주는 거대 확장형 무선망**입니다.

## Ⅱ. 기존 무선 확장기(Extender)와의 결정적 차이 🌟
집이 넓을 때 쓰던 기존의 '와이파이 증폭기(익스텐더)'와 메시(Mesh)는 근본적으로 다릅니다.
- **기존 증폭기 (Extender/Repeater)**: 메인 공유기 A에서 전파를 받아 단순히 릴레이로 복사해서 B 구역에 뿌려줍니다. 이 과정에서 A와 통신하랴 B에 뿌려주랴 하나의 무선 칩셋이 두 배의 일을 하므로 **속도가 반토막(Halved)** 납니다. 단말기가 A 구역에서 B 구역으로 이동할 때 네트워크 이름(SSID)이 바뀌어 통신이 잠시 끊기기도 합니다.
- **메시 네트워크 (Mesh)**: 여러 대의 공유기(노드)가 서로 통신하기 위한 **'전용 무선 백홀(Backhaul, 뒷단 고속도로)'** 밴드(주파수)를 독립적으로 뚫어 놓습니다. 따라서 A, B, C 공유기가 무선으로 연결되어 있어도 속도 저하가 전혀 발생하지 않으며, 전 구역이 완벽히 동일한 SSID를 공유하여 걸어 다니며 유튜브를 봐도 절대 끊기지 않는 '심리스 로밍(Seamless Roaming)'을 완벽히 지원합니다.

## Ⅲ. 무선 메시 노드의 3가지 역할 구분
1. **Mesh Gateway (Gateway Node)**: 유선 인터넷 케이블(WAN)이 직접 꽂혀 있는 대장 노드입니다. 바깥 세상(인터넷)과 그물망 내부를 이어주는 출입문입니다.
2. **Mesh Router (Router Node)**: 주변의 노드들과 그물망처럼 무선으로 얽혀서, 사용자의 패킷을 목적지까지 가장 빠른 길로 토스(Routing)해주는 중간 중계기입니다.
3. **Mesh Client (Client Node)**: 노트북, 스마트폰 등 사용자의 단말기입니다.

## Ⅳ. 네트워크 특징 및 강점
- **자기 치유 (Self-Healing)**: 만약 C 공유기(노드)의 전원이 실수로 뽑혔다면? 메시 네트워크는 C를 거치던 경로를 스스로 차단하고, 자동으로 D 공유기를 우회하는 새로운 경로(동적 라우팅)를 1초 만에 만들어내어 네트워크 전체의 마비를 막아냅니다.
- **데이지 체인 (Daisy Chain)**: 징검다리처럼 A ↔ B ↔ C ↔ D 순서로 꼬리에 꼬리를 물고 데이터를 무선으로 멀리멀리 전달할 수 있어, 공장이나 농장 같은 야외의 거대한 부지를 공사 없이 커버하는 데 최적입니다.

> 📢 **섹션 요약 비유**: 기존 무선랜 구조가 모든 직원이 부장님(단일 AP) 한 명에게만 일일이 서류를 제출해야 하는 수직적 조직이라면, 메시 네트워크는 직원(여러 개의 Mesh AP)들끼리 촘촘한 다이렉트 핫라인(무선 백홀)을 파놓은 수평적 그물 조직입니다. 부장님이 자리에 없거나 멀리 떨어져 있어도, 옆자리 김 대리와 박 과장을 징검다리 삼아(데이지 체인) 결재 서류를 인터넷망 밖으로 빠르게 무사히 전달할 수 있습니다.
