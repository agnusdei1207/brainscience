# 61. 컴팬딩 (Companding) / 압신 - μ-law, A-law

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 컴팬딩(Companding = Compression + Expanding)은 사람의 귀가 '작은 소리의 변화에는 민감하고 큰 소리의 변화에는 둔감하다'는 생물학적 약점을 역이용하여, **작은 소리는 계단을 촘촘하게, 큰 소리는 듬성듬성하게 양자화하는 비선형(Non-linear) 코딩 기법**이다.
> 2. **가치**: 12비트(선형)로 코딩해야만 들을 만한 깨끗한 작은 소리를, 컴팬딩을 쓰면 단 **8비트**만으로도 완벽하게 구현할 수 있어 통신망 대역폭(비용)을 33%나 절감시킨 통신 역사의 기적이다.
> 3. **표준 규격**: 미국과 한국은 **$\mu$-law (뮤-로우)**를, 유럽은 **A-law** 규격을 사용하여, 전 세계 모든 일반 전화(PSTN)와 VoIP(G.711 코덱)의 뼈대를 이루고 있다.

---

## Ⅰ. 개요 및 필요성 (Context & Necessity)

- **개념**: 
  - 송신단에서 작은 신호는 돋보기로 크게 확폭하고(Compressing), 큰 신호는 억눌러서 일정한 8비트 계단에 구겨 넣은 뒤, 수신단에서 다시 원래 비율로 펴주는(Expanding) 아키텍처.
  - 두 단어를 합쳐서 압신(Companding)이라고 부른다.
- **필요성 (선형 양자화의 낭비)**: 모든 진폭에 똑같은 계단 간격을 부여하는 선형 양자화를 쓰면, 아주 작은 개미 소리를 담기 위해 전체 계단 폭을 극도로 좁혀야 하고, 결과적으로 쓸데없이 12비트(4096개) 이상의 방대한 계단이 필요해져 대역폭이 낭비된다. "어차피 큰 소리(천둥소리)가 날 때는 10%쯤 오차가 나도 사람 귀가 눈치채지 못한다"는 정신물리학(Psychoacoustics) 법칙을 통신 칩에 코딩한 것이 컴팬딩이다.

- **💡 비유**: **컴팬딩은 '돋보기로 작은 글씨 읽기'**와 같다.
  - 책에 코끼리(큰 신호) 그림과 개미(작은 신호) 그림이 있다.
  - 선형 양자화는 똑같은 크기의 카메라 렌즈로 둘을 찍는다. 코끼리는 잘 보이지만 개미는 픽셀 1개로 뭉개져서(양자화 에러) 안 보인다.
  - 컴팬딩(압신)은 개미를 찍을 때는 **돋보기를 대서 확대한 뒤 촘촘하게 그리고**, 코끼리를 찍을 때는 **축소 렌즈로 작게 듬성듬성 그린다**. 이렇게 하면 똑같은 도화지(8비트) 한 장에 코끼리와 개미를 모두 선명하게 담아낼 수 있다!

## Ⅱ. 핵심 아키텍처 및 메커니즘 (Deep Dive)

### 1. 비선형 계단 구조 (Non-linear Step Size)
- **작은 진폭 구간**: 양자화 스텝(계단 높이)을 매우 조밀(촘촘)하게 설정. $\rightarrow$ 미세한 속삭임도 오차 없이 정확히 잡아냄.
- **큰 진폭 구간**: 양자화 스텝을 넓게(듬성듬성) 설정. $\rightarrow$ 어차피 큰 소리에 묻혀 오차가 안 들리므로 대역폭을 아낌.

### 2. 컴팬딩 과정 시각화 (Compressor $\rightarrow$ ADC)

```text
  ┌─────────────────────────────────────────────────────────────┐
  │         [컴팬딩(Companding) 송신단의 압축(Compression) 곡선]           │
  ├─────────────────────────────────────────────────────────────┤
  │                                                             │
  │     변환된 출력 전압 (ADC에 들어가는 값)                          │
  │        ▲                                                    │
  │        │          /─────◀─ (큰 신호는 많이 안 커짐: 듬성듬성)     │
  │        │        /                                           │
  │        │      /                                             │
  │        │    / ◀────── (작은 신호가 엄청나게 증폭됨: 촘촘한 효과)  │
  │        │  /                                                 │
  │        │/                                                   │
  │    ────┼─────────────────────────────────▶ 실제 입력 전압(소리)│
  │                                                             │
  │  * [원리]: 마이크에 들어온 '작은 소리'를 수학적 로그(Log) 함수 곡선을    │
  │           태워서 확 키워버림. 그 후 똑같은 간격의 8비트 ADC에 넣으면, │
  │           결과적으로 작은 소리가 수많은 계단을 차지하게 됨.            │
  └─────────────────────────────────────────────────────────────┘
```

### 3. 글로벌 표준 코덱 (ITU-T G.711)
- **$\mu$-law (Mu-law)**: 북미, 일본, 한국 통신망 표준. 14비트 선형 PCM 수준의 음질을 8비트로 압축해 낸다. 수학적으로 잡음 방어력이 A-law보다 미세하게 더 좋다.
- **A-law**: 유럽 및 국제 통신망 표준. 13비트 선형 PCM 수준의 음질을 8비트로 압축한다. 연산이 좀 더 간단하고 특정 환경에서 유리하다.

## Ⅲ. 실무 융합 및 트러블슈팅

### 실무 시나리오

1. **시나리오 — 국제 SIP(VoIP) 트렁크 연동 시 삐 소리(왜곡) 발생**:
   한국 본사(PBX)와 독일 지사(PBX) 간에 VoIP 전용선을 뚫고 전화를 걸었다. 그런데 서로 목소리는 안 들리고 "삐---" 하는 팩스 소리 같은 끔찍한 괴음만 들린다.
   **[해결책]** 전형적인 **컴팬딩 코덱 불일치(Mismatch)** 장애다. 한국 전화기는 내 목소리를 $\mu$-law라는 암호(로그 곡선)로 압축해서 보냈는데, 독일 전화기는 그걸 A-law라는 다른 지수 곡선 공식으로 풀어버렸다. 압축(Compress)한 방식과 펴주는(Expand) 방식이 어긋나니 파동이 완전한 쓰레기 쇳소리로 박살 난 것이다. 이종 국가 간 통신망을 연동할 때는 중간 세션 보더 컨트롤러(SBC)나 게이트웨이 장비에서 한쪽 코덱을 강제로 변환(Transcoding)해 주는 협상 룰을 세팅해야 한다.

2. **시나리오 — 8비트 IoT 마이크 칩셋 원가 절감 실패**:
   공장 소음 모니터링 센서의 원가를 낮추려고 16비트 ADC 대신 8비트 선형(Linear) ADC 칩을 샀다. 그랬더니 기계가 돌아가는 큰 소리는 잘 찍히는데, 기계가 멈추고 베어링이 갉아먹히는 아주 미세한 "찌긱" 소리(초기 불량음)는 양자화 노이즈에 파묻혀 아예 측정조차 되지 않았다.
   **[해결책]** 8비트 '선형(Linear)' ADC는 쓰레기다. 작은 소리에 대한 분해능이 끔찍하게 낮다. 칩셋을 바꿀 돈이 없다면, 마이크와 ADC 사이에 싸구려 아날로그 로그 앰프(Log Amplifier) 소자를 하나 추가로 납땜하여 아날로그 신호 자체를 **컴팬딩(비선형 증폭)** 시켜버려야 한다. 그러면 작은 소리가 앰프를 통과하며 확 커져서 8비트 ADC의 계단 십여 개를 거뜬히 오르내리게 되고 미세한 긁힘 소리를 디지털로 완벽하게 포획할 수 있다.

- **📢 섹션 요약 비유**: 컴팬딩은 큰 짐(큰 소리)과 작은 짐(작은 소리)을 똑같은 크기의 박스 256개(8비트)에 나눠 담는 정리의 달인입니다. 큰 짐은 발로 꽉꽉 밟아 찌그러뜨려서 대충 1개 박스에 쑤셔 넣고, 아주 섬세한 작은 짐(유리잔)은 뽁뽁이를 칭칭 감아 크게 만들어서 박스 10개에 아주 안전하게 쪼개 담는 극강의 타겟형 패키징 기술입니다.

## Ⅳ. 기대효과 및 요약

- **대역폭(Bandwidth) 구원**: 무식하게 선형으로 자르면 12비트를 써야 할 것을, 컴팬딩의 수학적 트릭 덕분에 8비트로 줄였다. 이로 인해 1가닥의 E1 라인(2.048Mbps)에 30명의 통화(64kbps $\times$ 30)를 꾸역꾸역 집어넣을 수 있었고, 이는 20세기 인류 통신망 대중화의 1등 공신이 되었다.
- 컴팬딩은 단순히 신호를 압축하는 기술이 아니다. **"인간은 결함을 느끼지 못하는 곳(큰 소리)의 데이터는 과감히 버리고, 극도로 예민한 곳(작은 소리)에만 자원을 몰빵한다"**는 공학적 선택과 집중의 가장 완벽한 철학적 성공 사례다.
