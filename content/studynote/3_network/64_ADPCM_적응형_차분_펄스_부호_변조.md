# 64. ADPCM (적응형 차분 펄스 부호 변조, Adaptive DPCM)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: ADPCM(Adaptive DPCM)은 이전 값과의 차이(오차)만을 전송하는 DPCM 기술에 **'인공지능적인 눈치(Adaptive)'**를 추가하여, 입력 파형이 급격히 변할 때는 양자화 계단을 확 늘려주고 완만할 때는 잘게 쪼개는 하이브리드 압축 코덱이다.
> 2. **해결점**: 기존 DPCM과 델타 변조가 파동이 갑자기 튀어 오를 때 쫓아가지 못해 붕괴되는 '경사 과부하(Slope Overload)' 현상을 완벽하게 해결했다.
> 3. **실무 융합**: ITU-T G.726 표준으로 채택되어 기존 64kbps의 전화망 속도를 음질 손상 없이 절반인 **32kbps**로 깎아냈으며, 블루투스 이어폰과 구형 무선 전화기(DECT)의 핵심 오디오 압축 엔진으로 활약했다.

---

## Ⅰ. ADPCM의 개요 및 탄생 배경 (Context & Necessity)

- **개념**: DPCM의 기본 원리인 '이전 샘플과의 차이값(Difference)'만 코딩하는 뼈대는 유지하되, 차이값을 양자화할 때 쓰는 **양자화 스텝의 크기(계단 폭)를 고정하지 않고 입력 신호의 변화량에 따라 능동적으로(Adaptive) 조절**하는 아키텍처.
- **필요성 (DPCM의 치명적 한계 극복)**: 
  - DPCM이나 DM(델타 변조)은 계단의 폭이 고정되어 있었다. 음악을 듣다가 갑자기 심벌즈가 "챙!" 하고 치거나 소리를 확 지르면 파동이 수직 상승하는데, 고정된 작은 계단으로는 이 수직 상승을 쫓아가지 못하고 에러(경사 과부하)를 냈다. 
  - 반대로 쫓아가 보겠다고 계단 폭을 무식하게 크게 만들면, 조용하고 부드러운 목소리 구간에서는 거대한 계단이 오르락내리락하며 쇳소리(그래뉼러 잡음)를 냈다.
  - 결국 엔지니어들은 **"계단 폭이 고무줄처럼 변해야 한다! 파동이 가파르면 계단을 거인처럼 크게 만들고, 완만하면 개미처럼 작게 만들자!"**라는 위대한 발상에 도달했고 이것이 ADPCM이다.

- **💡 비유**: **ADPCM은 '상황에 맞춰 기어 변속을 하는 자전거'**와 같다.
  - **DPCM**: 기어가 1단으로 고정된 자전거. 평지(완만한 파동)는 잘 가지만, 갑자기 가파른 언덕(급격한 파동)이 나오면 아무리 페달을 밟아도 자전거가 못 올라가고 굴러떨어진다.
  - **ADPCM**: 똑똑한 자동 변속기 자전거. 언덕을 만나면 스스로 기어를 낮춰 보폭(계단 폭)을 거대하게 만들어 단숨에 치고 올라가고, 다시 평지가 나오면 기어를 올려서 아주 부드럽고 섬세하게 굴러간다.

## Ⅱ. 핵심 아키텍처 메커니즘 (Deep Dive)

### 1. 적응형(Adaptive) 양자화기 작동 원리
ADPCM 칩셋의 핵심은 '양자화 스텝 제어기(Step Size Controller)'라는 피드백 루프다.
- **신호 급증 (가파른 언덕)**: 차이값($d$)이 계속 큰 양수(+, +)로 연속해서 들어오면, 제어기는 "어? 파동이 급발진하네!" 하고 눈치를 챈다. 즉시 다음번 반올림에 쓸 **계단의 높이($\Delta$)를 2배, 4배로 뻥튀기**한다. 단숨에 꼭대기까지 쫓아간다.
- **신호 정체 (평지)**: 차이값이 (+, -, +, -) 하며 제자리를 맴돌면, 제어기는 "아, 완만한 구간이구나!" 하고 **계단 높이를 아주 얇고 미세하게 줄인다**. 조용한 구간에서 쇳소리(잡음)가 나는 것을 억제한다.

### 2. DPCM과 ADPCM의 에러 방어력 시각화 비교

```text
  ┌────────────────────────────────────────────────────────────────────────┐
  │              [가파른 파동(급격한 변화)을 만났을 때의 추적 능력 비교]             │
  ├────────────────────────────────────────────────────────────────────────┤
  │                                                                        │
  │ [원본 아날로그 파동] : 갑자기 소리를 지름 (급격한 수직 상승)                        │
  │                     /─────                                             │
  │                    /                                                   │
  │                   /                                                    │
  │   ───▶ 1. [DPCM (고정 계단)] : 짧은 다리로 쫓아가다 가랑이가 찢어짐 (에러 폭발)       │
  │         ┌─┐┌─┐┌─┐   (파동은 벌써 저 위에 있는데, 계단은 밑에서 빌빌댐)        │
  │       ┌─┘ └┘ └┘ └    ==> 치명적인 [경사 과부하 잡음 (Slope Overload)] 발생 │
  │     ──┘                                                              │
  │                                                                        │
  │   ───▶ 2. [ADPCM (적응형 고무줄 계단)] : 롱다리로 변신해서 단숨에 멱살 잡음!         │
  │             ┌───────┐                                                │
  │         ┌───┘       │   (첫 계단은 작았지만, 파동이 확 오르자                │
  │     ───┘            │    두 번째, 세 번째 계단의 높이를 거대하게 뻥튀기함)     │
  │                       ==> 원본 파동을 놓치지 않고 100% 추적 성공!           │
  └────────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** DPCM의 가장 큰 약점은 유연성이 없다는 것이다. ADPCM은 소프트웨어 알고리즘을 추가하여, 이전의 오차 발생 패턴을 학습(Prediction)하고 다음 계단의 사이즈를 능동적으로 예측 제어(Adaptation)한다. 고정된 하드웨어의 틀을 소프트웨어 지능으로 깬 혁신적 융합 설계다.

## Ⅲ. 실무 융합 및 사례 분석

### 1. 통신사 백본망의 구원자 (ITU-T G.726 / G.722)
- **과거**: 국제 해저 광케이블(E1/T1) 1가닥에 PCM(64kbps) 30명밖에 전화를 못 했다. 통신비가 1분에 수천 원이었다.
- **혁신**: ADPCM 코덱 알고리즘을 라우터에 탑재하여, 64kbps짜리 무거운 PCM 음성을 거의 동일한 통화 품질의 **32kbps ADPCM으로 압축**했다. 기존 해저 케이블을 하나도 안 깔고도 전화 수용 인원을 2배(60명)로 뻥튀기시켜 막대한 경제적 수익을 창출했다.

### 2. 블루투스 오디오 코덱의 조상
- 무선 이어폰 초기 시절(Bluetooth SCO 채널), 전파는 약한데 마이크로 목소리를 주고받아야 했다. 이때 배터리를 적게 먹으면서도 연산이 아주 가벼운 ADPCM 칩셋이 무선 헤드셋에 대거 채용되어 무선 음성 통신의 표준을 열었다.

### 실무 트러블슈팅 시나리오

1. **시나리오 — 구형 사설 교환기(IP PBX)에서 팩스(Fax) 전송 실패 현상**:
   본사와 지사 간에 대역폭을 아끼기 위해 라우터의 음성 코덱을 G.711(PCM 64k)에서 G.726(ADPCM 32k)으로 바꿨다. 사람 목소리는 잘 들리는데, 팩스를 보내거나 모뎀으로 카드 결제를 할 때마다 신호가 다 깨져서 100% 에러가 터진다.
   **[해결책]** **ADPCM의 '예측(Prediction)' 알고리즘이 팩스 모뎀의 변조 신호(고주파 삐익 소리)를 사람 목소리로 착각하고 멋대로 계단을 넓혀버린 참사**다. ADPCM은 "사람의 목소리는 서서히 변한다"는 전제하에 만들어진 코덱이다. 그런데 팩스나 구형 카드 결제기의 통신음(FSK, QAM 등)은 파동이 극도로 날카롭고 복잡하게 진동한다. 이 날카로운 기계음을 만난 ADPCM 칩은 정신을 못 차리고 계단을 미친 듯이 뻥튀기시켰다가 뭉개면서 원본 팩스 데이터 파형을 완전히 파괴(비선형 왜곡)해 버렸다. 라우터 설정에서 팩스 톤(Fax Tone)을 감지하면 즉시 압축 코덱(ADPCM)을 바이패스(Bypass)하고 64k PCM으로 통과시키는 `T.38 Fax Relay` 기능이나 `Clear Channel` 구동을 켜주어야 한다.

## Ⅳ. 시스템 도입 체크리스트 및 한계

### 도입 체크리스트
- **지연 시간(Latency) 임계치 확인**: ADPCM은 이전 샘플들을 저장해두고 다음 계단을 '계산'하는 알고리즘 루프가 돌아가므로, 단순 무식한 PCM(거의 지연 제로)에 비해 코딩/디코딩 딜레이(약 0.1~1ms)가 추가로 발생한다. 극단적인 실시간 공연장 인이어 모니터망에 이 코덱을 쓴다면 딜레이 위상 캔슬링이 발생하는지 점검했는가?
- **에러 누적(Error Propagation) 대책**: 무선망에서 쓰일 경우 중간에 패킷 하나가 깨지면, 수신단의 ADPCM 계단 폭 계산 공식이 꼬여버려서 뒤따라오는 음성이 줄줄이 금속음으로 박살 나는 도미노 현상이 발생한다. 물리 계층(MAC) 재전송(ARQ) 횟수를 늘리거나 FEC(에러 정정) 비트를 강화했는가?

### ADPCM의 황혼과 유산
- 현재 스마트폰(LTE/5G)이나 최신 넷플릭스 등에서는 ADPCM을 쓰지 않는다. 왜냐하면 CPU 칩셋 성능이 비약적으로 발전해서, 아예 인간의 성대 모양(Vocal Tract)을 수학 모델로 흉내 내어 압축해 버리는 **AMR, EVS (CELP 기반 코덱)**나 MP3, AAC 같은 주파수 분석 코덱들이 대역폭을 8kbps 단위까지 미친 듯이 압축해 주기 때문이다. 
- 하지만 1980년대~90년대 통신 산업의 빙하기에 **"압축(Compression)과 적응(Adaptation)"**이라는 소프트웨어적 지능을 물리 계층에 최초로 부여한 ADPCM은 통신 진화사의 가장 중요한 미싱 링크(Missing Link)로 평가받는다.

## 📌 관련 개념 맵 (Knowledge Graph)

| 개념 명칭 | 관계 및 시너지 설명 |
|:---|:---|
| **DPCM (차분 PCM)** | ADPCM의 아버지. 이전 값과의 차이만 보내는 똑똑함을 가졌지만, 융통성이 없어 계단을 고정시켰다가 가파른 파동에서 장렬하게 전사한 기술. |
| **경사 과부하 (Slope Overload)** | 파동이 롤러코스터처럼 치솟을 때 양자화 계단이 못 쫓아가서 데이터가 깎여나가는 치명적 잡음. ADPCM이 계단을 늘려 이걸 완벽히 죽였다. |
| **적응형 필터 (Adaptive Filter)** | 입력되는 신호의 성질(빠르냐 느리냐)을 실시간으로 감시하고, 그에 맞춰 스스로 수학 공식을 바꾸는 모든 '스마트 회로'의 핵심 사상. |
| **G.726 (ITU-T 코덱 표준)** | 전 세계 엔지니어들이 합의한 통신망 규격. "우리 전화망 압축할 때는 이 ADPCM 알고리즘을 공통으로 써서 32kbps로 맞추자!"라고 약속한 문서. |

## 👶 어린이를 위한 3줄 비유 설명
1. 거인과 소인이 산을 올라간다고 해볼게요. 보통의 계단(DPCM)은 한 칸 높이가 일정해서 완만한 언덕은 소인이 쉽게 오르지만, 가파른 절벽은 절대 못 올라가고 떨어져요(에러 발생).
2. 하지만 **ADPCM 계단**은 마법의 계단이에요! 가파른 절벽이 나오면 계단이 찌익 늘어나서 1m짜리 거대한 블록이 되어 거인이 성큼성큼 단숨에 쫓아 올라가게 해줘요.
3. 그리고 다시 완만한 평지가 나오면, 계단이 1cm짜리로 아주 촘촘하게 변해서 쿵쾅거리는 소음 없이 아주 부드럽게 걸어갈 수 있게 스스로 변신하는 똑똑한 방법이랍니다!
