# 82. 코드 분할 다중화 (CDM, Code Division Multiplexing)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: CDM(Code Division Multiplexing)은 송신자들이 시간(TDM)이나 주파수(FDM)를 쪼개지 않고, **모두가 동일한 시간과 주파수 대역을 100% 겹치게 사용하면서, 각자 고유한 수학적 암호(직교 코드)를 곱해 데이터를 섞어버리는 다중화 기술**이다.
> 2. **원리 (직교성의 기적)**: 수십 명의 신호가 허공에서 하나의 덩어리로 완전히 떡져서 날아와도, 수신기가 '나와 약속된 암호 코드(Walsh Code)'를 곱하는 순간 타인의 신호는 수학적으로 숫자 '0'이 되어 사라지고 내 신호만 마법처럼 튀어나온다.
> 3. **실무 융합**: 주파수 낭비(가드 밴드)와 시간 낭비(버퍼링)를 모두 없애버린 궁극의 공간 활용 기술로, 대한민국이 세계 최초로 상용화한 **3G 이동통신(CDMA)과 현대 GPS 위성 통신망의 절대적인 뿌리**다.

---

## Ⅰ. CDM의 개요 및 탄생 철학 (Context & Necessity)

- **개념**: 
  - 다수의 채널(사용자)이 동일한 시간과 동일한 주파수 대역폭을 공유한다.
  - 각 채널은 자기 자신만 아는 고유한 '의사 잡음 코드(PN Sequence)'나 '직교 코드(Orthogonal Code)'를 원본 데이터에 곱해서 전송한다.
  - 수신단에서는 자기와 약속된 코드를 다시 곱해 원본을 복원한다.
- **필요성 (FDM과 TDM의 찌질함 극복)**:
  - **FDM**은 주파수를 쪼개다 보니 가드 밴드(빈 공간) 때문에 버리는 땅이 너무 많았다.
  - **TDM**은 시간을 쪼개다 보니 내 차례가 올 때까지 기다려야 하는 지연(Latency)이 발생했다.
  - 수학자들은 생각했다. **"시간도, 주파수도 쪼개지 말고 그냥 한방에 다 쏴버려! 대신 각자 다른 언어로 말하게 하고, 듣는 귀에 수학 필터를 달아주자!"** 이것이 대역 확산(Spread Spectrum) 철학을 기반으로 한 CDM의 탄생이다.

- **💡 비유**: **CDM은 '칵테일 파티 효과 (Cocktail Party Effect)'**와 같다.
  - 큰 연회장(동일 주파수)에서 100명의 사람(동일 시간)이 동시에 왁자지껄 떠든다.
  - 한국인 2명, 중국인 2명, 프랑스인 2명이 서로 마주 보고 대화한다.
  - 공기 중에는 온갖 소리가 떡이 되어 날아오지만, 한국인의 귀(수신기)에는 중국어와 프랑스어는 그냥 '웅웅거리는 배경 잡음(0)'으로 무시되고, 오직 내가 아는 '한국어(고유 코드)'만 명확하게 뇌에 꽂힌다.

## Ⅱ. 핵심 아키텍처 및 메커니즘 (Deep Dive)

### 1. CDM의 수학적 마법: 직교성(Orthogonality)
CDM이 성립하려면 각 사용자가 쓰는 코드끼리 절대로 겹치지 않는 **직교성(수학적 내적 = 0)**을 가져야 한다. (예: 왈시 코드, Walsh Code)
- **사용자 A의 코드 ($C_A$)** = `[ 1,  1,  1,  1]`
- **사용자 B의 코드 ($C_B$)** = `[ 1, -1,  1, -1]`
- 두 코드를 곱해서 더하면 (내적): $(1 \times 1) + (1 \times -1) + (1 \times 1) + (1 \times -1) = \mathbf{0}$
- 즉, **A의 코드와 B의 코드는 완벽히 직교하므로, 허공에서 아무리 섞여도 수신기가 계산할 때 서로를 투명 인간 취급(0)**해 버린다.

### 2. 송수신 파이프라인 시각화

```text
  ┌────────────────────────────────────────────────────────────────────────┐
  │                 [CDM (코드 분할 다중화) 송수신 맵핑 원리]                  │
  ├────────────────────────────────────────────────────────────────────────┤
  │                                                                        │
  │ [송신단 - 각자의 코드로 무장]                                                │
  │  Data A (+1) × Code A [1,  1,  1,  1]  =  [ 1,  1,  1,  1] (A의 전파)   │
  │  Data B (-1) × Code B [1, -1,  1, -1]  =  [-1,  1, -1,  1] (B의 전파)   │
  │                                                                        │
  │ [허공 - 전파가 떡이 됨 (가산기)]                                             │
  │  A전파 + B전파 = [ 0,  2,  0,  2]  ◀── (이 쓰레기 파동이 공중에 날아감)        │
  │                                                                        │
  │ [수신단 A - A의 코드만 들이댐]                                               │
  │  허공의 전파 [ 0,  2,  0,  2]                                             │
  │  × A의 코드  [ 1,  1,  1,  1]                                             │
  │  ─────────────────────────────                                             │
  │  (0×1) + (2×1) + (0×1) + (2×1) = 4  ──▶ (4로 나누면) ──▶ (+1) 원본 복원! │
  │                                                                        │
  │ * [수신단 A가 B를 투명인간 취급하는 증명]:                                      │
  │  만약 B의 전파[-1, 1, -1, 1]에 A의 코드를 곱하면? 결과는 완벽한 '0'이 된다.       │
  └────────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 데이터가 아무리 뒤죽박죽 섞여서 [0, 2, 0, 2]라는 외계어 전파가 되어도, 내 코드(A)만 딱 들이대면 다른 놈(B)의 데이터는 수학적으로 완벽하게 소거되고 오직 나에게 보내진 데이터만 딱 떨어진다. 주파수를 자르거나 기다릴 필요가 전혀 없는 우아한 수식의 승리다.

## Ⅲ. 실무 융합 및 트러블슈팅

### 실무 시나리오

1. **시나리오 — 초기 CDMA 기지국의 '원근 문제(Near-Far Problem)'로 인한 통화 끊김**:
   산 위에 세워진 기지국이 있다. 철수는 기지국 바로 밑에서 전화를 걸고 있고, 영희는 5km 밖 산골짜기에서 전화를 걸었다. 이때 철수와 영희는 다른 코드(언어)를 쓰니까 기지국이 둘 다 분리해 낼 줄 알았는데, 영희의 전화는 아예 먹통이 되고 철수 전화만 터졌다.
   **[해결책]** **CDM의 치명적 아킬레스건인 '원근 문제'의 폭발**이다. 직교성 수학 공식이 성립하려면 기지국 안테나에 도착하는 **모든 사용자의 전파 세기(dBm)가 완벽하게 똑같아야 한다**. 기지국 바로 밑에 있는 철수(Near)의 전파 파워가 너무 세서, 5km 밖 영희(Far)의 미약한 전파를 확성기처럼 덮어버리며 수학 공식 자체를 부숴버린 것이다.
   이를 해결하기 위해 CDMA 기지국은 1초에 무려 800번씩 철수 폰에게 **"너 나랑 가까우니까 송신 파워 당장 최하로 깎아! 안 그럼 다 죽어!"**라는 미친듯한 초고속 **전력 제어(Fast Power Control)** 명령을 쏘아댄다. 이 전력 제어 소프트웨어 로직이 없으면 CDM망은 1초도 유지될 수 없다.

2. **시나리오 — 도심지 지하철에서의 다중경로(Multipath) 에러 방어**:
   지하철이나 좁은 도심 골목에서 전파를 쏘면, 빌딩에 부딪혀 튕겨 나온 반사파(메아리)가 0.1초 늦게 수신기에 도착해 원본 전파와 겹치면서 통신이 깨진다 (다중경로 페이딩). 과거 아날로그 FDM 폰은 이때마다 "찌지직" 거렸다.
   **[해결책]** CDM 칩셋 안에는 **레이크 수신기(Rake Receiver)**라는 천재적인 도구가 들어있다. 반사되어서 지각한 전파 덩어리들을 쓰레기통에 버리지 않는다. CDM은 이 지각한 전파들에도 내 코드가 묻어있다는 걸 알고, 갈퀴(Rake)처럼 싹 긁어모아 시간차를 계산해 원본 신호에 다시 더해버린다. 오히려 반사파를 주워 먹고 신호를 더 증폭시켜 버리는(다이버시티 이득) 기적을 행한다. CDMA 폰이 지하실이나 숲속에서 통화가 짱짱하게 잘 터졌던 물리적 이유다.

## Ⅳ. 시스템의 한계와 진화

### CDM의 몰락 원인 (코드 고갈)
- CDM은 코드가 곧 자원이다. 기지국 1개(섹터)가 만들 수 있는 완벽한 직교 코드(Walsh Code)의 개수는 보통 **64개 또는 128개**로 한정되어 있다. 
- 이 말은, 아무리 대역폭이 남아돌아도 **코드 64개를 다 나눠주고 나면 65번째 손님은 기지국에 아예 접속을 못 한다(Capacity Limit)**는 뜻이다.
- 또한 초고속 인터넷 시대가 오면서 한 사람이 넷플릭스를 보려면 코드를 혼자서 10개, 20개씩 묶어서 써야 했다(HSDPA 방식). 결국 코드 자원이 순식간에 고갈되면서, 4G LTE 시대부터는 이 아름다운 수식을 버리고 다시 주파수와 시간을 레고 블록처럼 썰어 쓰는 실용적인 **OFDMA(직교 주파수 분할)**로 대세가 완전히 넘어가게 된다.

## Ⅴ. 결론

코드 분할 다중화(CDM)는 주파수(FDM)와 시간(TDM)이라는 기존 통신 공학의 2차원적 한계를 깨부수고, **'수학적 암호(Code)'**라는 3번째 차원을 도입해 낸 천재적인 파괴적 혁신이다. 비록 데이터 폭식 시대를 감당하지 못하고 코드가 부족해 역사 속으로 물러났지만, 이 기술이 남긴 "강력한 보안성", "잡음과 반사파를 씹어 먹는 생존력", 그리고 "초정밀 전력 제어 로직"은 오늘날의 모든 무선 통신 기기에 그대로 이식되어 인류의 모바일 혁명을 지탱하는 뼈대가 되었다.
