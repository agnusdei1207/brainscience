# 20-27. 형상 관리와 버전 관리 (SCM, Git)

> **핵심 인사이트**: 개발 현장에서 "누가 코드를 이렇게 고쳤어?"라는 질문이 나온다면 이미 늦었다. 형상 관리는 소프트웨어라는 살아있는 생명체의 모든 '변화'를 기록하고 통제하며, 언제든 과거의 안정된 상태로 되돌릴 수 있는 타임머신을 제공하는 필수 안전장치다.

---

## Ⅰ. 형상 관리 (SCM, Software Configuration Management)
소프트웨어 생명주기 전반에 걸쳐 생성되는 모든 산출물의 변경 사항을 체계적으로 관리하는 활동입니다.

### 형상 관리 4대 절차
1. **형상 식별**: 관리할 항목(CI, Configuration Item)을 선정하고 이름표를 붙임.
2. **형상 통제**: 변경 요청이 오면 승인 여부를 결정(CCB 운영)하고 반영함.
3. **형상 감사**: 변경이 계획대로 됐는지, 무결성이 유지되는지 확인.
4. **형상 기록/보고**: 관리 활동의 모든 과정을 기록하여 보고함.

---

## Ⅱ. 기준선 (Baseline)
프로젝트의 중요한 시점마다 공식적으로 승인된 산출물의 집합입니다. 다음 단계로 넘어가기 위한 '약속된 지점'입니다.

* **종류**: 기능적(분석 완료), 설계(설계 완료), 시험(테스트용), 제품(출시용) 기준선.

---

## Ⅲ. 버전 관리 시스템 (VCS)
소스코드의 이력을 관리하는 도구입니다.

1. **중앙 집중형 (CVCS)**: 서버 한 곳에 원본이 있고 사용자는 체크아웃해서 사용. (SVN 등)
2. **분산형 (DVCS)**: 모든 사용자가 전체 이력을 자신의 로컬에 복제하여 소유. 서버가 없어도 작업 가능. (**Git** 등)

---

## Ⅳ. 변경 제어 위원회 (CCB, Configuration Control Board)
* 형상 항목의 변경 여부를 최종적으로 결정하는 협의체입니다. 무분별한 수정을 막아 프로젝트의 안정성을 유지합니다.

---

## Ⅴ. 개념 맵 및 요약

```ascii
[형상 관리의 흐름]

  [ 변경 요청 ] ──> [ CCB 승인 ] ──> [ Check-out ] ──> [ 수정 ]
                                                           │
  [ 기준선 갱신 ] <── [ 형상 감사 ] <── [ Check-in ] <──────┘
```

📢 **섹션 요약 비유**: **형상 관리**는 도서관의 '대출 및 도서 관리 시스템'입니다. 어떤 책(CI)이 있는지 목록을 만들고, 책 내용이 바뀔 때(변경) 관장님(CCB)의 허락을 받으며, 이전 판본도 보관해두는 행위입니다. **기준선**은 책의 '초판', '개정판'처럼 공식적으로 찍혀 나온 버전이며, **Git**은 모든 독자가 도서관의 모든 책을 자기 집에 똑같이 한 세트씩 복사해 두고 자유롭게 읽고 쓰는 혁명적인 복제 기술과 같습니다.
