# 512. 제로 트러스트 3대 원칙 (NIST SP 800-207)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: NIST SP 800-207에서 정의한 제로 트러스트 3대 원칙은 **명시적 검증 (Verify Explicitly), 최소 권한 부여 (Least Privilege), 침해 가정 (Assume Breach)**으로, 신뢰의 근거를 위치가 아닌 증명 가능한 데이터에 두는 보안 설계 지침이다.
> 2. **가치**: 이 3대 원칙은 파편화된 보안 기술들을 하나의 일관된 철학으로 통합하며, 공격자가 일단 내부로 들어왔을 때 수행하는 '횡적 이동'과 '데이터 유출'을 물리적으로 방해하는 강력한 가이드라인을 제공한다.
> 3. **융합**: 현대 아키텍처에서 IAM, micro-segmentation, 실시간 위협 인텔리전스와 결합되어 구현되며, 단순한 기능 구현을 넘어 조직 전체의 보안 성숙도를 평가하는 핵심 잣대가 된다.

---

## Ⅰ. 개요 (Context & Background)

- **개념**: 제로 트러스트 3대 원칙은 제로 트러스트 아키텍처(ZTA)를 구축할 때 반드시 지켜야 할 사상적 뿌리다. NIST(미국 표준기술연구소)는 복잡한 네트워크 환경에서 보안 실효성을 확보하기 위해 이 세 가지 원칙을 준수할 것을 강력히 권고한다.

- **💡 비유**: 제로 트러스트 3대 원칙은 **"국가 중요 시설의 보안 매뉴얼"**과 같습니다. 1) 누구나 신분증을 철저히 검사하고(명시적 검증), 2) 자기 사무실 외에는 절대 못 들어가게 하며(최소 권한), 3) 이미 내부에 간첩이 들어와 있다고 생각하고 상시 감시하는(침해 가정) 고강도 경비 원칙입니다.

- **등장 배경**:
  1. **사라진 경계**: 클라우드와 모바일로 인해 '내부망'이라는 안전지대가 사라지면서, 모든 접근을 의심해야 하는 상황에 직면했다.
  2. **정교한 해킹 기법**: 한번 뚫리면 걷잡을 수 없이 퍼지는 랜섬웨어 등에 대응하기 위해, 공격자가 이미 들어와 있다는 전제하의 방어 체계가 필요해졌다.
  3. **표준화 요구**: 기업마다 제각각인 제로 트러스트의 정의를 하나로 모으기 위해 NIST가 기술적/사상적 기준점을 제시했다.

- **📢 섹션 요약 비유**: 사고가 난 뒤에 후회하지 말고, 처음부터 모든 사람을 의심하고(침해 가정), 꼼꼼히 따져보고(명시적 검증), 딱 필요한 만큼만 허락(최소 권한)하는 깍쟁이 같은 보안 원칙입니다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 제로 트러스트 3대 핵심 원칙 분석

| 원칙 (Tenet) | 상세 정의 | 핵심 구현 기술 | 비유 |
|:---|:---|:---|:---|
| **1. 명시적 검증** | 사용 가능한 모든 데이터 지점을 기반으로 상시 인증/인가 | MFA, 기기 상태 점검, 위치/시간 분석 | 신분증 전수 조사 |
| **2. 최소 권한 부여** | 업무 수행에 필요한 최소한의 권한을 필요한 시간 동안만 부여 | RBAC, JIT (Just-in-Time), 세분화된 ACL | 자기 방 열쇠만 주기 |
| **3. 침해 가정** | 이미 네트워크가 침투당했다고 가정하고 피해 최소화 설계 | Micro-segmentation, 암호화, 행위 분석 | 스파이 상시 감시 |

---

### 원칙 기반의 제로 트러스트 제어 루프 아키텍처

3대 원칙이 실제 시스템에서 어떻게 상호작용하며 작동하는지 시각화한다.

```text
 ┌─────────────────────────────────────────────────────────────────────────────┐
 │                      제로 트러스트 3대 원칙의 실천적 제어 프로세스                │
 ├─────────────────────────────────────────────────────────────────────────────┤
 │                                                                             │
 │ [주체 요청] ──▶ [1. 명시적 검증 (Verify)] ──▶ [2. 최소 권한 (Limit)] ──▶ [자원 접근] │
 │      │                 │                         │                         │      │
 │  ID/기기/상태      - 누구인가? (Auth)        - 꼭 필요한가? (Policy)      접근 승인   │
 │  (Context)        - 안전한 기기인가?         - 이 자원만 열어줌           (Log 기록) │
 │      │                 │                         │                         │      │
 │      └─────────────────┴───────────┬─────────────┴─────────────────────────┘      │
 │                                    ▼                                              │
 │                        [3. 침해 가정 (Monitor)]                                    │
 │                        - 실시간 이상 행위 탐지                                      │
 │                        - 전 구간 암호화 및 격리                                     │
 │                        - 사고 시 즉시 격리 (Blast Radius 축소)                      │
 │                                                                             │
 └─────────────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** 제로 트러스트의 여정은 **명시적 검증**에서 시작된다. 단순히 "비밀번호가 맞네"가 아니라 "이 사람이 지금 쓰는 노트북이 최신 패치가 되었는가?"까지 확인한다. 검증이 끝나면 **최소 권한** 원칙에 따라, 요청한 특정 파일이나 서버에만 접근할 수 있는 '좁은 통로'를 열어준다. 그리고 이 모든 과정 밑바닥에는 **침해 가정** 원칙이 깔려 있다. 사용자가 정상적으로 일을 하는 중에도 "사실 이 사람은 해커가 계정을 탈취한 것일지 모른다"고 생각하며, 실시간으로 행위 로그를 분석하고 데이터 전송량을 감시한다.

---

### 심층 동작 원리: Blast Radius (폭발 반경) 축소

'침해 가정' 원칙의 핵심 기술적 목표다.
1. **격리 (Isolation)**: 네트워크를 마이크로 단위로 세분화하여, 서버 하나가 뚫려도 옆 서버로 공격이 번지지 않게 만든다.
2. **영향 최소화**: 암호화를 생활화하여 데이터가 유출되더라도 내용을 알 수 없게 만든다.
3. **가치**: 불을 끄는 것보다 불이 옆집으로 옮겨붙지 않게 방화벽(물리적 벽)을 촘촘히 세우는 것과 같다.

- **📢 섹션 요약 비유**: 배에 구멍이 나서 물이 들어와도(침해 발생), 배 전체가 가라앉지 않도록 수많은 방수 격실(Blast Radius 축소)을 만들어두는 것과 같습니다.

## Ⅲ. 융합 비교 및 다각도 분석

### 3대 원칙의 상호 보완 관계

| 원칙 A | 원칙 B | 시너지 효과 |
|:---|:---|:---|
| **명시적 검증** | **최소 권한** | 신뢰할 수 있는 사람에게도 권한을 아껴줌으로써 계정 도용 피해 방지 |
| **최소 권한** | **침해 가정** | 애초에 권한이 작으니 해커가 들어와도 할 수 있는 일이 극히 제한됨 |
| **침해 가정** | **명시적 검증** | 이미 뚫렸을지 모르니 접속 중에도 계속 다시 검증(Continuous Auth)함 |

---

### 기술 융합: AI와 3대 원칙의 결합

인간이 수동으로 3대 원칙을 지키기는 불가능에 가깝다.
- **AI-Verify**: 실시간으로 수백 가지 맥락 데이터를 분석하여 지능적으로 신원을 검증한다.
- **AI-Privilege**: 사용자의 업무 패턴을 학습하여, 안 쓰는 권한을 자동으로 회수하고 필요할 때만 준다.
- **AI-Assume**: 아주 미세한 이상 징후를 포착하여 해킹 사고를 초기에 인지한다.

- **📢 섹션 요약 비유**: 깐깐한 경비원(사람) 대신, 24시간 쉬지 않고 모든 사람의 눈빛과 걸음걸이까지 분석하는 인공지능 보안 시스템(AI 결합)이 원칙을 지키는 세상입니다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오: 클라우드 기반 신규 서비스의 보안 설계

- **문제 상황**: 전사 시스템을 클라우드(AWS)로 이전하면서, 기존의 IP 기반 방화벽 정책이 너무 복잡해지고 관리가 안 됨.
- **의사결정 (NIST 3대 원칙 기반 설계)**:
  1. **명시적 검증**: 사내망 접속 여부와 관계없이 모든 API 호출 시 MFA와 IAM Role을 통한 인증을 강제한다.
  2. **최소 권한**: 애플리케이션 서버에 DB 전체 접근 권한을 주지 않고, 특정 테이블의 특정 CRUD 작업만 가능한 세분화된 권한(Fine-grained Policy)을 부여한다.
  3. **침해 가정**: 모든 데이터 저장소에 암호화(KMS)를 적용하고, VPC Flow Log와 CloudTrail을 실시간 분석하여 비정상적인 데이터 유출 패턴을 탐지 시 자동으로 계정을 정지(Auto-remediation)시킨다.

### 도입 체크리스트
- **Verify**: 사용자의 위치나 기기 보안 상태가 변했을 때 세션을 즉시 재검증하는가?
- **Least**: 사용자가 퇴사하거나 부서 이동 시 즉각적으로 권한이 회수되는 프로세스가 있는가?
- **Assume**: 우리 시스템이 뚫렸을 때, 해커가 10분 안에 가져갈 수 있는 최대 데이터량은 얼마인가? (피해 규모 예측)

### 안티패턴
- **예외 구역 (Exception Zone)**: "서버들끼리는 믿어도 되니까 검증 안 해도 돼"라는 안일한 설계. 침해 가정 원칙에 정면 위배된다.
- **영구 권한 (Static Permission)**: 한 번 부여한 권한을 유효기간 없이 영원히 두는 것. 최소 권한 원칙의 시간적 요소를 무시하는 행위다.

- **📢 섹션 요약 비유**: 아무리 친한 가족이라도 내 지갑(데이터)을 열 때는 매번 허락을 받아야 하고(명시적 검증), 지갑 안에서도 꼭 필요한 돈만 가져가게 해야(최소 권한) 평화가 유지된다는 점과 같습니다.

## Ⅴ. 기대효과 및 결론

### 정량/정성 기대효과

| 구분 | 기대효과 | 세부 내용 |
|:---|:---|:---|
| **정량적 (Detection)** | 위협 탐지 속도 향상 | 실시간 감시를 통해 침입 인지 시간(MTTD) 70% 이상 단축 |
| **정량적 (Blast Radius)** | 사고 피해 규모 축소 | 격리 및 권한 제한으로 유출되는 데이터 양을 1/10 수준으로 방어 |
| **정성적 (Standard)** | 일관된 보안 거버넌스 | 부서/시스템마다 달랐던 보안 기준을 국가 표준 원칙으로 통일 |
| **정성적 (Agility)** | 비즈니스 속도 대응 | 복잡한 네트워크 설정 대신 신원 기반 정책으로 유연한 인프라 운영 |

### 미래 전망
- **Self-Governing Zero Trust**: 시스템 스스로 정책 위반 사례를 찾아내고 보안 규칙을 강화하는 자가 통제형 아키텍처가 실현될 것이다.
- **Global Identity Fabric**: 전 세계 어디서든 나를 증명할 수 있는 표준화된 신원 망이 구축되어, 3대 원칙이 국경을 넘어 실시간으로 적용될 것이다.

### 참고 표준
- **NIST SP 800-207**: Zero Trust Architecture (섹션 2.1 제로 트러스트 기본 원칙)
- **CISA Zero Trust Maturity Model**: 원칙 준수 여부에 따른 성숙도 평가 모델

- **📢 섹션 요약 비유**: 성벽을 높이 쌓는 법만 고민하던 과거에서, 성 안의 모든 움직임을 투명하게 관리하고 사고를 미리 대비하는 '지능형 안전 도시'로 보안의 패러다임이 완전히 바뀌었습니다.

## 👶 어린이를 위한 3줄 비유 설명
1. **명시적 검증**: 우리 집에 들어올 때는 누구든지 "진짜 너 맞니?"라고 꼼꼼히 확인하는 약속이에요.
2. **최소 권한**: 친구가 놀러 왔을 때 거실에서만 놀게 하고, 내 비밀 일기장이 있는 안방은 못 들어가게 하는 규칙이에요.
3. **침해 가정**: 도둑이 이미 우리 집 뒷마당에 숨어있을지도 모른다고 생각해서, 항상 조심하고 준비하는 마음가짐이랍니다!
