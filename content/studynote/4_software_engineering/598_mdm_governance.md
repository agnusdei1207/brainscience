# 598. 마스터 데이터 관리 (MDM) 및 데이터 거버넌스

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 데이터 거버넌스는 전사적 차원에서 데이터의 품질, 가용성, 보안을 보장하기 위한 **정책 및 프로세스 체계**이며, MDM (마스터 데이터 관리)은 기업의 핵심 비즈니스 개체(고객, 제품 등)에 대한 **단일하고 정확한 기준 정보(Golden Record)**를 통합 관리하는 기술적 실천이다.
> 2. **가치**: 파편화된 데이터로 인한 의사결정 혼선을 방지하고 데이터의 신뢰성을 확보하며, 법적 규제(개인정보보호법 등) 준수를 위한 관리 체계를 확립하여 데이터 자산의 가치를 극대화한다.
> 3. **융합**: 전사 아키텍처(EA) 및 데이터 아키텍처(DA)의 운영 엔진 역할을 수행하며, 현대 아키텍처에서는 데이터 메시(Data Mesh)와 결합되어 중앙 집중과 분산 관리 사이의 균형을 맞추는 핵심 거버넌스로 진화하고 있다.

---

## Ⅰ. 개요 (Context & Background)

- **개념**: 
  - **데이터 거버넌스 (Data Governance)**: 데이터를 "누가, 어떻게, 어떤 원칙으로" 관리할지 정의하는 전사적 통제 체계다.
  - **MDM (Master Data Management)**: 여러 시스템에 흩어진 고객, 상품, 협력사 등의 핵심 정보를 하나로 통합하여 "이것이 진짜 데이터다"라고 선언하는 기준 정보 관리 체계다.

- **💡 비유**: 데이터 거버넌스와 MDM은 **"국가의 표준 도량형과 법령"**과 같습니다. 시장(시스템 A)과 마트(시스템 B)에서 쓰는 1kg(데이터)의 무게가 다르면 거래가 안 되듯, 나라에서 1kg의 표준 추(MDM)를 만들고 이를 어기면 벌을 주는 법(거버넌스)을 세워 상거래(비즈니스)의 질서를 잡는 것과 같습니다.

- **등장 배경**:
  1. **시스템 파편화 (Silo)**: 부서마다 시스템을 따로 쓰면서 "우리 회사의 고객이 총 몇 명인가?"라는 단순한 질문에도 부서마다 답이 다른 상황이 발생했다.
  2. **데이터 품질의 중요성**: 쓰레기를 넣으면 쓰레기가 나온다는 'GIGO(Garbage In, Garbage Out)' 원리에 따라, 인공지능이나 분석 결과의 신뢰도를 높이기 위해 깨끗한 기초 데이터가 절실해졌다.
  3. **컴플라이언스 대응**: 전사적인 데이터 보안 정책과 개인정보 파기 정책을 일관되게 적용해야 하는 법적 요구사항이 강화되었다.

- **📢 섹션 요약 비유**: 수백 명의 오케스트라 단원들이 각자 자기 악기만 조율(부서별 관리)하는 것이 아니라, 하나의 튜닝기(MDM)에 맞추고 지휘자(거버넌스)의 신호에 따르게 하여 완벽한 화음(데이터 통합)을 만들어내는 과정입니다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

### 1. 데이터 거버넌스의 3대 구성 요소 (Organization, Process, System)

거버넌스는 기술보다 '사람과 규칙'의 조화가 중요하다.

| 요소 | 상세 내용 | 비유 |
|:---|:---|:---|
| **조직 (Org)** | 데이터 관리자(Steward), 거버넌스 위원회 구성 | 법을 만드는 국회와 집행부 |
| **프로세스 (Proc)** | 데이터 표준화, 품질 측정, 승인 절차 정의 | 행정 절차 및 법규 |
| **시스템 (Sys)** | 메타데이터 관리, MDM 솔루션, 품질 모니터링 툴 | 법을 감시하는 CCTV와 전산망 |

---

### 2. MDM 통합 아키텍처 (Golden Record 생성)

흩어진 데이터를 하나로 합쳐서 '진실'을 추출하는 구조를 시각화한다.

```text
 ┌─────────────────────────────────────────────────────────────────────────────┐
 │                      MDM 기반 전사 기준 정보 통합 아키텍처                        │
 ├─────────────────────────────────────────────────────────────────────────────┤
 │                                                                             │
 │ [Source Systems]              [MDM Central Hub]             [Target Systems] │
 │                                                                             │
 │  ┌────────────┐               ┌──────────────────┐             ┌────────────┐   │
 │  │ ERP (고객A)│ ──────▶       │ 1. Data Cleaning │  ──────▶    │ 마케팅 (A) │   │
 │  ├────────────┤               │ 2. Deduplication │             ├────────────┤   │
 │  │ CRM (고객A)│ ──────▶       │ 3. Matching      │  ──────▶    │ 고객센터(A)│   │
 │  ├────────────┤               │ 4. Golden Record │             ├────────────┤   │
 │  │ Web (고객A)│ ──────▶       │    Creation      │  ──────▶    │ DW / BI (A)│   │
 │  └────────────┘               └────────┬─────────┘             └────────────┘   │
 │                                        │                                    │
 │                                        ▼                                    │
 │                    ┌──────────────────────────────────┐                     │
 │                    │      Data Governance Console     │                     │
 │                    │  - 표준 용어 사전 관리 (Metadata)  │                     │
 │                    │  - 데이터 품질 리포팅 (Quality)    │                     │
 │                    └──────────────────────────────────┘                     │
 │                                                                             │
 │  ※ Golden Record: 여러 소스 중 가장 정확하고 신뢰할 수 있는 정보를 선별한 결과물      │
 └─────────────────────────────────────────────────────────────────────────────┘
```

**[다이어그램 해설]** MDM 아키텍처의 핵심은 **Data Cleaning**과 **Deduplication (중복 제거)**이다. 1) ERP, CRM 등에서 '홍길동'이라는 고객 정보가 각기 다른 형태로 들어오면, 2) MDM 허브는 이를 분석하여 동일인임을 판별(Matching)한다. 3) 주소가 가장 최근인 데이터, 전화번호가 인증된 데이터 등을 조합하여 단 하나의 완벽한 **Golden Record**를 생성한다. 4) 이렇게 정제된 '진실의 데이터'는 다시 전사 시스템으로 배포(Publish)되어, 모든 부서가 동일한 고객 정보를 보게 만든다.

---

### 심층 동작 원리: 데이터 품질 (Data Quality) 6대 지표

거버넌스 팀이 매일 체크해야 할 건강 지표다.
1. **정확성 (Accuracy)**: 실제 사실과 데이터가 일치하는가?
2. **완전성 (Completeness)**: 필수 항목이 누락되지 않았는가?
3. **일관성 (Consistency)**: 시스템 간 데이터 형식이 동일한가?
4. **유효성 (Validity)**: 정해진 규칙(도메인) 내의 값인가?
5. **적시성 (Timeliness)**: 필요한 시점에 최신 데이터가 제공되는가?
6. **유일성 (Uniqueness)**: 중복된 데이터가 존재하지 않는가?

- **📢 섹션 요약 비유**: 신선한 식재료를 공급하기 위해 유통기한(적시성), 무게(정확성), 포장 상태(완전성) 등을 꼼꼼히 검수하는 '품질 관리 센터'와 같습니다.

## Ⅲ. 융합 비교 및 다각도 분석

### MDM 구현 방식 (Implementation Styles)

조직의 상황에 따라 세 가지 중 하나를 선택한다.

| 방식 | 설명 | 장점 | 단점 |
|:---|:---|:---|:---|
| **Registry (참조)** | 소스 DB는 그대로 두고 ID만 매핑 | 구축이 빠르고 가벼움 | 실시간 정합성 보장 어려움 |
| **Hub (중앙 집중)** | MDM 서버에 모든 데이터를 물리적 통합 | **데이터 정합성 최상** | 시스템 부하 및 구축 비용 큼 |
| **Hybrid (혼합)** | 주요 속성은 Hub, 상세 정보는 소스 유지 | 성능과 일관성의 절충 | 관리 아키텍처가 복잡함 |

---

### 기술 융합: 데이터 거버넌스와 데이터 메시 (Data Mesh)

중앙 집중형 거버넌스의 한계를 극복하는 현대적 융합이다.
- **Federated Governance**: 모든 데이터를 중앙에서 통제하는 대신, 데이터 관리는 각 부서(도메인)가 직접 하되 '보안 및 표준 규칙'만 중앙에서 내려주는 방식.
- **가치**: 중앙 병목을 없애면서도 전사적인 일관성을 유지하는 '분산형 거버넌스'를 실현한다.

- **📢 섹션 요약 비유**: 모든 요리를 주방장 한 명이 검수하는 대신, 각 요리사가 자기 요리를 책임지되 '위생 복장과 재료 규격'만 본사의 규칙을 따르는 프랜차이즈 시스템과 같습니다.

## Ⅳ. 실무 적용 및 기술사적 판단

### 실무 시나리오: 그룹사 통합 멤버십을 위한 MDM 구축

- **문제 상황**: 계열사(백화점, 마트, 호텔)마다 고객 정보가 파편화되어 있어, 통합 포인트 적립이나 타겟 마케팅이 불가능함.
- **의사결정 (MDM/거버넌스 전략)**:
  1. **거버넌스 조직 구성**: 각 계열사 IT 팀장들이 참여하는 '전사 데이터 표준화 위원회'를 신설한다.
  2. **표준 코드 정립**: 계열사마다 다른 '상품 분류 코드'와 '고객 등급'을 하나의 표준 코드로 통합 정의한다.
  3. **MDM Hub 구축**: 고객 식별자(CI/DI) 기반의 MDM 시스템을 구축하여 계열사 간 중복 고객을 통합(Cleaning)한다.
  4. **Data Steward 배치**: 각 계열사 현업 부서에 데이터 관리자(Steward)를 지정하여, 데이터 품질 오류 발생 시 즉시 교정하는 책임 체계를 마련한다.

### 도입 체크리스트
- **Master Data Scope**: 모든 데이터를 다 마스터로 관리하려 하지 않는가? (비즈니스 핵심 개체 3~5개로 한정 권장)
- **Metadata Management**: 데이터의 정의, 소유자, 수명 주기가 적힌 '메타데이터 카탈로그'가 최신으로 유지되는가?
- **Security Policy**: 통합된 마스터 DB에 대한 접근 권한이 '최소 권한 원칙'에 따라 엄격히 통제되는가?

### 안티패턴
- **기술 중심의 추진**: 비즈니스 부서의 참여 없이 IT 팀끼리 솔루션만 도입하는 경우. (현업이 안 쓰면 죽은 시스템이 됨)
- **품질 관리 없는 통합**: 더러운 데이터(Garbage)를 그대로 합치는 행위. 중복만 늘어나는 거대한 쓰레기통을 만드는 꼴이다.

- **📢 섹션 요약 비유**: 물을 합치기 전에 정수(Data Cleaning)부터 해야 깨끗한 우물이 됩니다. 오염된 물들을 그냥 한 대야에 붓는 것은 통합이 아니라 오염의 확산입니다.

## Ⅴ. 기대효과 및 결론

### 정량/정성 기대효과

| 구분 | 기대효과 | 세부 내용 |
|:---|:---|:---|
| **정량적 (Quality)** | 데이터 오류율 80% 하락 | 자동 정제 및 표준화를 통한 데이터 품질 지수(DQI) 개선 |
| **정량적 (Ops)** | 데이터 수집/가공 시간 단축 | 표준화된 기준 정보 제공으로 분석 리포트 생성 속도 향상 |
| **정성적 (Compliance)** | 개인정보 보호 법규 준수 | 데이터 흐름의 투명성 확보로 감사 및 보안 사고 대응력 강화 |
| **정성적 (Agility)** | 비즈니스 통찰력 확보 | 정확한 통합 데이터를 근거로 한 신속하고 올바른 경영 판단 가능 |

### 미래 전망
- **Self-Healing Data Governance**: AI가 데이터의 입력 패턴을 실시간 감시하여 표준에 어긋나는 데이터를 자동으로 교정하거나 경고하는 기능이 보편화될 것이다.
- **Data Marketplace**: 거버넌스가 잘 잡힌 기업 내부 데이터를 사내외에 상품으로 제공하는 데이터 마켓플레이스 비즈니스가 활성화될 것이다.

### 참고 표준
- **DAMA-DMBOK2**: 데이터 관리 지식 체계 가이드
- **ISO/IEC 38505**: IT 거버넌스 - 데이터 거버넌스 표준
- **CMMI-DMM**: 데이터 관리 성숙도 모델

- **📢 섹션 요약 비유**: 데이터는 기업의 '기억'입니다. MDM과 거버넌스는 이 기억이 왜곡되지 않고 선명하게 유지되도록 돕는 기업의 '기억 저장 및 관리 센터'입니다.
