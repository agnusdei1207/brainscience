# 441-444. 테스트 케이스와 데이터 관리 (TDM)

> **핵심 인사이트**: 테스트는 정교하게 설계된 '실험'이다. 무엇을 넣고(Input) 무엇이 나오길 기대하는지(Output) 명확히 기록한 테스트 케이스는 품질의 객관적 척도가 된다. 또한 실데이터를 안전하게 가공하여 제공하는 TDM은 개인정보 보호와 테스트의 현실성이라는 두 마리 토끼를 잡는 필수 인프라다.

---

## Ⅰ. 테스트 케이스 (Test Case) 구조
특정 요구사항을 준수하는지 확인하기 위해 개발된 입력 값, 실행 조건, 예상 결과의 집합입니다. (ISO/IEC/IEEE 29119 표준)

* **구성 요소**:
  - **식별자**: 고유 ID.
  - **테스트 항목**: 대상 기능.
  - **입력 데이터**: 테스트에 사용될 값.
  - **실행 절차**: 테스트 수행 단계.
  - **예상 결과**: 정답(Expected Result).
  - **합격/불합격 기준**.

---

## Ⅱ. 테스트 시나리오와 절차
1. **테스트 시나리오 (Test Scenario)**: 테스트 케이스들을 비즈니스 흐름에 따라 묶은 집합입니다. (사용자의 이용 흐름 검증)
2. **테스트 절차 (Test Procedure)**: 테스트 케이스를 실행하기 위한 구체적인 순서를 기록한 명세서입니다.

---

## Ⅲ. 테스트 데이터 관리 (TDM)
테스트 수행에 필요한 데이터를 생성, 추출, 변환, 관리하는 활동입니다.

* **개인정보 익명화**: 실데이터 사용 시 성명, 주민번호 등을 **마스킹(Masking)**하거나 **가명화**하여 보안을 준수해야 합니다.
* **데이터 생성 기법**:
  - **현장 데이터 추출**: 운영 DB에서 일부 추출.
  - **합성 데이터 생성**: 도구를 사용하여 가상의 데이터를 대량 생성.

---

## Ⅳ. 개념 맵 및 요약

```ascii
[테스트 설계의 계층]

  [ Test Plan ] (전략)
      │
  [ Test Scenario ] (비즈니스 흐름)
      │
  [ Test Case ] (개별 입력/출력) <─── [ Test Data (TDM) ]
      │
  [ Test Procedure ] (실행 순서)
```

📢 **섹션 요약 비유**: **테스트 케이스**는 요리 시험의 '레시피'와 같습니다. 어떤 재료(데이터)를 넣고 어떤 맛(결과)이 나야 하는지 적혀 있죠. **테스트 시나리오**는 전채 요리부터 디저트까지 이어지는 '코스 요리'의 흐름이며, **TDM**은 요리 연습을 위해 실제 고급 식재료(실데이터) 대신 모양과 성질이 비슷한 '연습용 식재료'를 준비해 주는 재료 관리실과 같습니다.
