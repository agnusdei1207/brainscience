# 445-450. 성능 테스트의 유형과 목적

> **핵심 인사이트**: 시스템이 평상시에 잘 돌아가는 것만으로는 부족하다. 성능 테스트는 시스템이 얼마나 견딜 수 있는지(Load), 언제 무너지는지(Stress), 갑작스러운 폭풍에도 버티는지(Spike), 그리고 오랫동안 지치지 않는지(Soak)를 시험하여 예기치 못한 서비스 마비를 방지하는 활동이다.

---

## Ⅰ. 성능 테스트 (Performance Test) 4대 유형

1. **부하 테스트 (Load Test)**: 시스템의 임계점(목표치)까지 부하를 서서히 증가시키며, 응답 시간과 자원 사용률이 정상 범위 내에 있는지 확인.
2. **스트레스 테스트 (Stress Test)**: 시스템이 견딜 수 있는 **임계점 이상의 과부하**를 주어, 시스템이 언제 붕괴되는지 확인하고 그 과정에서 데이터 유실 없이 우아하게 복구되는지 검증.
3. **스파이크 테스트 (Spike Test)**: 갑작스럽게 사용자가 폭증할 때(이벤트 시작 등) 시스템의 반응과 복구 능력 확인.
4. **내구성 테스트 (Endurance / Soak Test)**: 일정한 부하를 **장시간** 동안 주어, 메모리 누수(Leak)나 자원 고갈이 발생하는지 확인.

---

## Ⅱ. 벤치마크 테스트 (BMT)
* **BMT (Benchmark Test)**: 동일한 환경에서 서로 다른 제품(HW, SW)의 성능을 비교하여 최적의 제품을 선정하기 위한 테스트입니다. 주로 도입 전 의사결정에 활용됩니다.

---

## Ⅲ. 성능 지표
* **Throughput (처리량)**: 단위 시간당 처리하는 트랜잭션 수 (TPS).
* **Response Time (응답 시간)**: 요청 후 첫 응답이 올 때까지의 시간.
* **Turnaround Time (반환 시간)**: 요청부터 최종 결과 출력까지의 시간.

---

## Ⅳ. 개념 맵 및 요약

```ascii
[부하량에 따른 성능 테스트 그래프]

  성능
    ^
    |      [ Load ] (정상 범위)
    |      [ Spike ] (돌발 폭증)
    |      [ Stress ] (한계 돌파!)
    └──────────────────────────────> 부하량 (Load)
```

📢 **섹션 요약 비유**: **부하 테스트**는 승강기에 정원인 15명을 태우고 잘 올라가는지 보는 것이고, **스트레스 테스트**는 20명, 30명을 태워 언제 줄이 끊어지는지 보는 것입니다. **스파이크 테스트**는 15명이 한꺼번에 뛰어 들어올 때의 충격을 견디는지 보는 것이며, **내구성 테스트**는 정원을 태운 채 24시간 내내 운행했을 때 과열되지 않는지 보는 것입니다. **BMT**는 삼성 승강기와 현대 승강기 중 어느 것이 더 조용하고 빠른지 비교하는 시승회와 같습니다.
