# 456-457. 뮤테이션 테스팅과 퍼즈 테스팅

> **핵심 인사이트**: 테스트도 '테스트'가 필요하다. 뮤테이션 테스팅은 일부러 코드에 에러를 심어 내 테스트 케이스가 이를 잡아내는지 확인하는 '테스터를 위한 테스트'다. 한편, 퍼즈 테스팅은 사람이 상상하지 못한 기형적인 데이터를 쏟아부어 시스템의 허를 찌르는 '무차별적 안전 진단'이다.

---

## Ⅰ. 뮤테이션 테스팅 (Mutation Testing)
테스트 케이스의 결함 발견 능력을 측정하기 위해 사용하는 기법입니다.

* **원리**:
  1. 원본 프로그램에 의도적으로 결함이 있는 복제본(**Mutant, 돌연변이**)을 만듭니다. (예: `>` 를 `<` 로 수정)
  2. 기존 테스트 케이스를 돌연변이 프로그램에 실행합니다.
  3. 테스트 케이스가 돌연변이를 잡아내면(Fail) **Kill**되었다고 하며, 못 잡아내면(Pass) 돌연변이가 살아남았다고 합니다.
* **목표**: 살아남은 돌연변이가 없도록 더 강력한 테스트 케이스를 설계하는 것입니다.

---

## Ⅱ. 퍼즈 테스팅 (Fuzz Testing / Fuzzing)
소프트웨어의 취약점을 찾기 위해 **무작위 또는 기형적인 입력 데이터(Fuzz)**를 대량으로 주입하여 예외 상황이나 크래시를 유발하는 기법입니다.

* **특징**:
  - 예상치 못한 에러 처리를 검증하는 데 탁월합니다.
  - 버퍼 오버플로우, 메모리 누수 등 보안 취약점 발견에 널리 쓰입니다.
* **유형**:
  - **Generation-based**: 모델 기반으로 새로운 데이터를 생성.
  - **Mutation-based**: 기존 정상 데이터를 미세하게 변조.

---

## Ⅲ. 개념 맵 및 요약

```ascii
[테스트 완성도 향상 전략]

  [ Mutation Test ] ──> "내 테스트 케이스는 강력한가?" (검증력 평가)
  [ Fuzz Testing  ] ──> "이상한 입력에도 안 죽는가?" (견고성 평가)
```

📢 **섹션 요약 비유**: **뮤테이션 테스팅**은 소방 훈련 때 마네킹을 숨겨놓고 소방관(테스트 케이스)이 찾아내는지 시험하는 것입니다. 마네킹을 못 찾으면 훈련 방식이 잘못된 것이죠. **퍼즈 테스팅**은 기계 부품의 강도를 테스트하기 위해 망치로 무작위로 때려보거나 불에 넣어보는 등 '가혹 행위'를 하는 극한 테스트와 같습니다.
