# 303. мҠӨнҠёлҰј мІҳлҰ¬ DB

## н•өмӢ¬ мқёмӮ¬мқҙнҠё (3мӨ„ мҡ”м•Ҫ)
> 1. **ліём§Ҳ**: мҠӨнҠёлҰј мІҳлҰ¬ DB(Stream Processing Database)лҠ” лҚ°мқҙн„° мҠӨнҠёлҰј(дәӢд»¶ жөҒ)мқ„ мӢӨмӢңк°„мңјлЎң мІҳлҰ¬В·л¶„м„қн•ҳлҠ” лҚ°мқҙн„°лІ мқҙмҠӨлЎң, Kafka, Flink, Kafka Streams, KSQLDB л“ұмқҙ мһҲлӢӨ.
> 2. **к°Җм№ҳ**: мҲҳ л°ҖлҰ¬мҙҲ~мҲҳ мҙҲмқҳ к·№дҪҺ м§Җм—°мңјлЎң мӢӨмӢңк°„ л¶„м„қ, м—°мҶҚ Queries, мӢӨмӢңк°„ лҢҖмӢңліҙл“ң, мқҙмғҒ нғҗм§Җ,дәӢд»¶й©ұеҠЁ м• н”ҢлҰ¬мјҖмқҙм…ҳ м§Җмӣҗмқҙ к°ҖлҠҘн•ҳлӢӨ.
> 3. **мңөн•©**: Kafka, Apache Flink, мҠӨнҠёлҰј мІҳлҰ¬, Complex Event Processing, мӢӨмӢңк°„ л¶„м„қ, Lambda/Kappa м•„нӮӨн…ҚмІҳмҷҖ л°Җм ‘н•ҳкІҢ м—°кҙҖлҗңлӢӨ.

---

## в… . к°ңмҡ” л°Ҹ н•„мҡ”м„ұ (Context & Necessity)

### к°ңл…җ м •мқҳ
мҠӨнҠёлҰј мІҳлҰ¬ DB(Stream Processing Database)лҠ” лҚ°мқҙн„°мқҳ_continuous stream_(м—°мҶҚм ҒмқёдәӢд»¶ жөҒ)мқ„ мӢӨмӢңк°„мңјлЎң мІҳлҰ¬В·л¶„м„қн•ҳлҠ”ж•°жҚ®еә“мқҙлӢӨ. м „нҶөм Ғмқё л°°м№ҳ мІҳлҰ¬(Batch Processing)к°Җ м ҖмһҘлҗң лҚ°мқҙн„°лҘј мқјкҙ„м ҒмңјлЎңеҮҰзҗҶгҒҷгӮӢгҒ®гҒ«еҜҫгҒ—гҖҒгӮ№гғҲгғӘгғјгғ еҮҰзҗҶлҠ” лҚ°мқҙн„°к°Җ лҸ„м°©н•ҳлҠ” мӢңм җм—җ л°”лЎң мІҳлҰ¬н•ңлӢӨ. мҠӨнҠёлҰј мІҳлҰ¬ мӢңмҠӨн…ңмқҖ"time in motion"мңјлЎң лҚ°мқҙн„°лҘјжҚ•гӮүгҒҲгҖҒлҚ°мқҙн„°гҒ® arriving speedгҒ«и·ҹдёҠгҒҷгӮӢгҖӮKafka, Apache Flink, Kafka Streams, KSQLDB, Apache Storm, Amazon Kinesis Data Analytics л“ұмқҙ лҢҖн‘ңм Ғмқё мҠӨнҠёлҰј мІҳлҰ¬ кё°мҲ мқҙлӢӨ.

### н•„мҡ”м„ұ
нҳ„лҢҖ л№„мҰҲлӢҲмҠӨлҠ”гӮҲгӮҠзҙ ж—©гҒ„ мқҳмӮ¬кІ°м •мқ„ мң„н•ҙ мҲҳ л°ҖлҰ¬мҙҲ~мҲҳ мҙҲ лӢЁмң„мқҳгғӘгӮўгғ«гӮҝгӮӨгғ  лҚ°мқҙн„° л¶„м„қмқ„ н•„мҡ”лЎң н•ңлӢӨ. кёҲмңө мӮ¬кё° нғҗм§Җ, л„ӨнҠёмӣҢнҒ¬ м№Ёмһ… к°җм§Җ, мӢӨмӢңк°„ м¶”мІң, IoT м„јм„ң лӘЁлӢҲн„°л§Ғ, кҙ‘кі  нҒҙлҰӯлҘ  мөңм Ғнҷ” л“ұмқҳз”ЁдҫӢгҒ§гҒҜгҖҒл°°м№ҳеӨ„зҗҶмқҳ мҲҳмӢңк°„е»¶иҝҹгҒҜиҮҙе‘Ҫзҡ„гҒӘе•ҸйЎҢгҒЁгҒӘгӮӢгҖӮ мҠӨнҠёлҰј мІҳлҰ¬ DBлҠ” мқҙлҹ¬н•ң мӢӨмӢңк°„ мІҳлҰ¬ лӢҲмҰҲлҘј м¶©мЎұн•ҳл©°, лҚ°мқҙн„°к°Җ мҠӨнҠёлҰјмңјлЎңжөҒе…Ҙн•ҳлҠ” лҸҷм•Ҳ м§ҖмҶҚм ҒмңјлЎң кІ°кіјлҘј мӮ°м¶ңн•ңлӢӨ.

### л°°кІҪ
мҠӨнҠёлҰј мІҳлҰ¬ к°ңл…җмқҖ 1970л…„лҢҖ лҚ°мқҙн„° нқҗлҰ„ м–ём–ҙ(Dataflow Programming)м—җм„ң л№„лЎҜлҗҳм—ҲлӢӨ. мқҙнӣ„з”өдҝЎ л°Ҹ кёҲмңө л¶„м•јгҒ§гҒ®гғӘгӮўгғ«гӮҝгӮӨгғ еҮҰзҗҶ н•„мҡ”жҖ§гҒ« л”°лқјCEP (Complex Event Processing) кё°мҲ мқҙ л°ңм „н–ҲлӢӨ. 2010л…„лҢҖм—җлҠ” LinkedInм—җм„ң к°ңл°ңлҗң Apache Kafka(2011е№ҙ)к°Җ л¶„мӮ° л©”мӢңм§Җ нҒҗмқҳ мӮ¬мӢӨмғҒ н‘ңмӨҖмқҙ лҗҳм—Ҳкі , Kafka Streams, Apache Flink, Apache Storm л“ұмқҳ мҠӨнҠёлҰј мІҳлҰ¬ м—”м§„мқҙ л“ұмһҘн–ҲлӢӨ. мөңк·јм—җлҠ” мҠӨнҠёлҰј мІҳлҰ¬мҷҖ лҚ°мқҙн„°лІ мқҙмҠӨлҘј нҶөн•©н•ң Stream Processing Database(еҰӮ Kafka Streams, KSQLDB, Materialize)гӮӮжіЁзӣ®гӮ’йӣҶгӮҒгҒҰгҒ„гӮӢ.

### л№„мң 
мҠӨнҠёлҰј мІҳлҰ¬лҠ”еӨ§еһӢн•ӯкіөмӮ¬ кҙҖм ңеЎ”мҷҖ к°ҷлӢӨ. л№„н–үкё°к°Җйӣўйҷён•ҳл©ҙ мӢӨмӢңк°„мңјлЎң мң„м№ҳ, мҶҚлҸ„, кі лҸ„ л“ұмқҳ лҚ°мқҙн„°к°Җ нқҳлҹ¬л“Өм–ҙмҳӨкі (лҚ°мқҙн„° мҠӨнҠёлҰј), кҙҖм ңмӮ¬лҠ” мқҙлҘј мӢӨмӢңк°„мңјлЎң лӘЁлӢҲн„°л§Ғн•ҳм—¬ м¶©лҸҢ мң„н—ҳмқҙ мһҲмңјл©ҙеҚіжҷӮ кІҪкі гӮ’зҷән•ңлӢӨ(мӢӨмӢңк°„ л¶„м„қ). м „нҶөм Ғмқё л°°м№ҳеӨ„зҗҶлҠ” л№„н–үкё°к°Җ лӢӨиҗҪең°н•ң нӣ„ м •ліҙлҘјгҒҫгҒЁгӮҒгҒҰзўәиӘҚгҒҷгӮӢгҒҢгҖҒмҠӨнҠёлҰј мІҳлҰ¬лҠ” л№„н–ү мӨ‘еёёгҒ«зӣЈиҰ–гҒҷгӮӢгҖӮ мӢӨмӢңк°„м„ұмқҙ н•„мҡ”н•ң мӢңмҠӨн…ңм—җм„ң л°°м№ҳеӨ„зҗҶлҠ”дҪҝз”Ёл¶Ҳк°ҖлҠҘн•ҳлӢӨ.

рҹ“ў м„№м…ҳ мҡ”м•Ҫ: мҠӨнҠёлҰј мІҳлҰ¬ DBлҠ” лҚ°мқҙн„° мҠӨнҠёлҰјмқ„ мӢӨмӢңк°„мңјлЎң мІҳлҰ¬В·л¶„м„қн•ҳлҠ” DBлЎң, мҲҳ л°ҖлҰ¬мҙҲ~мҲҳ мҙҲмқҳ к·№дҪҺ м§Җм—°мңјлЎң мӢӨмӢңк°„ л¶„м„қ, м—°мҶҚ Queries,дәӢд»¶й©ұеҠЁ м• н”ҢлҰ¬мјҖмқҙм…ҳмқ„ м§Җмӣҗн•ңлӢӨ.

**[лӢӨмқҙм–ҙк·ёлһЁ н•ҙм„Ө]**
мҠӨнҠёлҰј мІҳлҰ¬лҠ” л°°м№ҳ мІҳлҰ¬мҷҖж №жң¬м ҒмңјлЎң лӢӨлҘё нҢЁлҹ¬лӢӨмһ„мңјлЎң, лҚ°мқҙн„°лҘј м ҖмһҘн•ҳм§Җ м•Ҡкі event-by-eventлЎң мӢӨмӢңк°„ мІҳлҰ¬н•ңлӢӨ. Kafka StreamsлҠ” Kafka мғқнғңкі„ лӮҙм—җм„ңи»Ҫлҹүн•ҳкІҢ мҠӨнҠёлҰј мІҳлҰ¬лҘј н•ҳлҠ” лҚ° м Ғн•©н•ҳкі , Apache FlinkлҠ”гӮҲгӮҠ ліөмһЎн•ң лҢҖмҡ©лҹү мҠӨнҠёлҰј мІҳлҰ¬мҷҖ ML нҢҢмқҙн”„лқјмқём—җ м Ғн•©н•ҳлӢӨ. Event Time мІҳлҰ¬лҘј мң„н•ҙм„ңлҠ” WatermarkмҷҖ Window к°ңл…җмқҙ н•„мҲҳм Ғмқҙл©°, мқҙлҘј нҶөн•ҙ л¬ҙн•ң мҠӨнҠёлҰјмқ„ мң н•ң лӢЁмң„лЎңеҲҶжһҗн•  мҲҳ мһҲлӢӨ.

рҹ“ў м„№м…ҳ мҡ”м•Ҫ: мҠӨнҠёлҰј мІҳлҰ¬ DBлҠ”event-by-event мӢӨмӢңк°„ мІҳлҰ¬лЎң к·№дҪҺ м§Җм—°мқ„ м ңкіөн•ҳкі , Kafka StreamsмҷҖ Flink л“ұмқҙ лҢҖн‘ңм Ғмқҙл©°, Event Time мІҳлҰ¬лҘј мң„н•ҙ WatermarkмҷҖ Window к°ңл…җмқҙ нҷңмҡ©лҗңлӢӨ.

## в…ў. кІ°лЎ 

мҠӨнҠёлҰј мІҳлҰ¬ DBлҠ” лҚ°мқҙн„° мҠӨнҠёлҰјмқ„ мӢӨмӢңк°„мңјлЎң мІҳлҰ¬В·л¶„м„қн•ҳлҠ” н•өмӢ¬ кё°мҲ лЎң, Kafka, Flink, Kafka Streams, KSQLDB л“ұмқҙ мһҲлӢӨ. л°°м№ҳ мІҳлҰ¬мҷҖ лӢ¬лҰ¬ лҚ°мқҙн„°лҘј м ҖмһҘн•ҳм§Җ м•Ҡкі event-by-eventлЎң мІҳлҰ¬н•ҳм—¬ мҲҳ л°ҖлҰ¬мҙҲ~мҲҳ мҙҲмқҳ к·№дҪҺ м§Җм—°мқ„ м ңкіөн•ңлӢӨ. Event Time кё°л°ҳ мІҳлҰ¬, Watermark, Window л“ұмқҳ к°ңл…җмқ„ нҶөн•ҙ л¬ҙн•ң мҠӨнҠёлҰјмқ„ мң н•ң лӢЁмң„лЎң л¶„м„қн•  мҲҳ мһҲлӢӨ. кёҲмңө мӮ¬кё° нғҗм§Җ, IoT лӘЁлӢҲн„°л§Ғ, мӢӨмӢңк°„ м¶”мІң, л„ӨнҠёмӣҢнҒ¬ м№Ёмһ… к°җм§Җ л“ұ мӢӨмӢңк°„м„ұмқҙ н•„мҲҳм Ғмқё лӢӨм–‘н•ңй ҳеҹҹм—җм„ң нҷңмҡ©лҗңлӢӨ.

рҹ“ў м„№м…ҳ мҡ”м•Ҫ: мҠӨнҠёлҰј мІҳлҰ¬ DBлҠ”event-by-event мӢӨмӢңк°„ мІҳлҰ¬лҘј нҶөн•ҙ к·№дҪҺ м§Җм—°мқ„ м ңкіөн•ҳл©°, Kafka Streams, Flink л“ұмқҙ лҢҖн‘ңм Ғмқҙкі , Event Time, Watermark, Window л“ұмқҳ к°ңл…җмңјлЎң л¬ҙн•ң мҠӨнҠёлҰјмқ„ л¶„м„қн•ңлӢӨ.
