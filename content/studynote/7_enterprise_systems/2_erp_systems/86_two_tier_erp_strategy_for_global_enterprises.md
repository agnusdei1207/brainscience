# 2-Tier ERP (투티어 ERP 전략)

## 핵심 인사이트 (3줄 요약)
> 1. **본질**: 2-Tier ERP는 본사가 사용하는 무겁고 복잡한 코어 ERP(Tier 1)와, 지사나 자회사의 특수한 요구에 맞춘 가볍고 유연한 ERP(Tier 2)를 이원화하여 운영하는 글로벌 전사적 자원 관리 전략이다.
> 2. **가치**: 본사는 전사 통합 관리와 통제(Control)를 유지하고, 지사는 현지의 법규, 언어, 비즈니스 속도에 맞춰 신속하게 대응(Agility)할 수 있는 상충된 가치의 균형을 제공한다.
> 3. **융합**: 클라우드 ERP(SaaS)의 확산으로 Tier 2 구축 비용이 획기적으로 낮아졌으며, EAI나 API 게이트웨이를 통한 실시간 데이터 동기화 기술이 두 계층을 하나의 유기체로 묶어준다.

+++

## Ⅰ. 개요 (Context & Background)

1. **개념**: 과거에는 전 세계 모든 지사가 본사와 똑같은 ERP(예: SAP)를 쓰는 'Single Instance' 전략이 정답으로 여겨졌다. 하지만 수조 원의 매출을 내는 본사와 수백억 원의 매출을 내는 해외 소규모 지사가 똑같이 비싸고 복잡한 시스템을 쓰는 것은 자원 낭비였다. **2-Tier ERP**는 "본사는 본사답게, 지사는 지사답게" 각자의 체급에 맞는 옷을 입히는 실용주의적 아키텍처다.

2. **💡 비유**: 2-Tier ERP는 **'대형 항공모함과 민첩한 쾌속정 함대'**와 같습니다. 거대한 모선(본사)은 막강한 화력과 지휘 기능(Tier 1)을 갖추고, 주변의 쾌속정(지사)들은 좁은 해안선(현지 시장)을 빠르게 정찰하고 기민하게 움직일 수 있는 엔진(Tier 2)을 다는 것입니다.

3. **등장 배경**:
    - **글로벌 M&A의 활성화**: 인수한 회사의 시스템을 본사 시스템으로 통합하는 데 드는 막대한 시간과 비용 문제 해결.
    - **비즈니스 모델의 다변화**: 본사는 제조 위주인데 지사는 유통이나 서비스 위주일 경우, 하나의 ERP로 두 모델을 다 담기 어려움.
    - **클라우드 ERP의 성숙**: 넷스위트(NetSuite)나 SAP ByDesign처럼 가볍게 시작할 수 있는 SaaS ERP가 등장하며 Tier 2의 대안이 됨.

📢 섹션 요약 비유: 2-Tier ERP는 본사의 '정장'과 지사의 '활동복'을 구분하여 입는 것으로, 격식(통제)과 활동성(민첩성)을 모두 잡는 세련된 경영 코디네이션입니다.

## Ⅱ. 아키텍처 및 핵심 원리 (Deep Dive)

2-Tier ERP의 수직적 구조와 계층별 역할 분담을 분석한다.

### 1. 2-Tier ERP 통합 아키텍처 모델 (ASCII)

```text
    ┌─────────────────────────────────────────────────────────────┐
    │          [ Tier 1 : Headquarters (HQ) ] - Global Core       │
    │  전략 기획, 전사 재무 결산, 통합 구매, 글로벌 HR, 마스터 데이터│
    └───────────────┬─────────────────────────────▲───────────────┘
                    │ (표준/가이드라인)           │ (실적/데이터 취합)
    ┌───────────────▼─────────────────────────────┴───────────────┐
    │          [ Integration Layer : 연계 레이어 ]                │
    │  API Gateway, ESB, Cloud Connector, Master Data Hub         │
    └───────┬───────────────┬───────────────┬───────────────┬─────┘
            ▼               ▼               ▼               ▼
    ┌─────────────┐ ┌─────────────┐ ┌─────────────┐ ┌─────────────┐
    │  [ Tier 2 ] │ │  [ Tier 2 ] │ │  [ Tier 2 ] │ │  [ Tier 2 ] │
    │  A 지사(제조)│ │  B 지사(판매)│ │  C 신설법인 │ │  D 합병회사 │
    └─────────────┘ └─────────────┘ └─────────────┘ └─────────────┘
```

**[다이어그램 해설]**  
- **Tier 1 (HQ)**: 기업의 중심 뼈대다. 주로 재무(FI)와 연결 결산이 핵심이다. 강력한 통제력이 필요한 마스터 데이터(고객, 상품 코드)를 관리한다.
- **Tier 2 (Subsidiaries)**: 현장 중심의 실행 엔진이다. 현지 세법(Tax), 언어, 현장 물류에 최적화된 가벼운 ERP를 쓴다.
- **Integration Layer**: 두 계층 사이의 '통역사'다. 지사의 실적을 본사 장부로 자동 전송하고, 본사의 표준 단가를 지사로 배포한다.

### 2. 계층별 시스템 특성 비교 (Table)

| 구분 | Tier 1 (Global Core) | Tier 2 (Regional/Local) |
|:---|:---|:---|
| **대상 조직** | 본사, 주요 거점 공장 | 해외 법인, 자회사, 판매 거점 |
| **시스템 성격** | 복잡성 높음, 고도의 커스터마이징 | 유연함, 빠른 도입, 표준 기능 위주 |
| **주요 모듈** | 통합 재무, 전략 경영, R&D | 영업, 창고 관리, 현지 인사/급여 |
| **인프라** | 프라이빗 클라우드 / 온프레미스 | **퍼블릭 클라우드 (SaaS)** |
| **핵심 가치** | **Consolidation (통합)** | **Agility (민첩성)** |

### 3. 성공적인 운영 원리: 연합 거버넌스 (Federated Governance)

- **데이터 연동**: 본사가 정한 표준 항목만 취합하고, 세부 운영 데이터는 지사에서 자율 관리.
- **마스터 데이터 일치**: 고객사나 부품 코드는 전사 표준을 따르게 하여 데이터 중복 방지.

📢 섹션 요약 비유: Tier 1은 집의 '기둥과 지붕'이고, Tier 2는 각 방의 '가구와 인테리어'입니다. 기둥만 튼튼하면 방의 가구는 입주자(지사) 취향대로 배치해도 집(기업)은 무너지지 않습니다.

## Ⅲ. 융합 비교 및 다각도 분석 (Comparison & Synergy)

Single Instance 전략과의 차이 및 최신 클라우드 기술과의 시너지다.

### 1. Single Instance vs 2-Tier ERP 비교

| 구분 | Single Instance (통합형) | 2-Tier ERP (이원화형) |
|:---|:---|:---|
| **구축 비용** | 초기 대규모 투자 (High) | 단계적/분산 투자 (Low) |
| **유지보수** | 중앙 집중 관리 (편함) | 다수 시스템 관리 (복잡) |
| **현지 적응력** | 낮음 (본사 표준 강요) | **매우 높음** (현지 특화) |
| **데이터 통합** | 실시간 완벽 통합 | 연계를 통한 부분적 통합 |
| **비유** | 전교생이 똑같은 교복 입기 | 계열별로 어울리는 복장 입기 |

### 2. 과목 융합: SaaS ERP와 2-Tier의 진화

- **변화**: 과거의 Tier 2는 중소기업용 구축형 ERP였으나, 현재는 **NetSuite, Workday, SAP S/4HANA Cloud** 등 검증된 SaaS가 그 자리를 차지함.
- **시너지**: SaaS를 쓰면 지사에 IT 관리 인력이 없어도 본사 아키텍트가 원격으로 거버넌스를 유지할 수 있음. (Low-ops)

📢 섹션 요약 비유: Single Instance가 '큰 솥 하나에 모든 국을 끓이는 것'이라면, 2-Tier는 '큰 솥(본사)과 여러 개의 작은 뚝배기(지사)를 함께 쓰는 것'입니다.

+++

## Ⅳ. 실무 적용 및 기술사적 판단 (Strategy & Decision)

실무에서 2-Tier ERP를 도입할 때의 핵심 판단 기준이다.

### 1. "언제 2-Tier를 택할 것인가?" (Decision)

기술사적 관점에서 다음 조건 중 2개 이상 해당하면 2-Tier가 유리하다.
1. **업종 상이성**: 본사는 중후장대 제조인데, 자회사는 경박단소 서비스업인 경우.
2. **지사 규모**: 지사 인원이 50명 미만이라 거대 ERP 운영이 불가능한 경우.
3. **M&A 빈번**: 회사를 자주 사고팔아 시스템 통합을 기다릴 여유가 없는 경우.
4. **현지 규제**: 본사 시스템이 현지 국가의 세법이나 데이터 주권법을 지원 못 하는 경우.

### 2. 2-Tier ERP 성공 체크리스트

| 점검 항목 | 확인 질문 | 기술사적 제언 |
|:---|:---|:---|
| **통합 결재** | 지사의 실적이 본사 연결 재무제표로 즉시 집계되는가? | 표준 결산 인터페이스(Global Ledger) 설계 |
| **마스터 동기화** | 지사에서 제멋대로 신규 코드를 따고 있지는 않은가? | MDM 허브를 통한 코드 배포 프로세스 강제 |
| **보안 정책** | Tier 2 시스템이 본사의 보안 표준(ISO 27001 등)을 충족하는가? | SaaS 벤더의 보안 인증서 및 암호화 수준 검증 |
| **IT 역량** | 지사 시스템 장애 시 누가 지원하는가? | 중앙 CoE와 현지 MSP(운영업체)의 R&R 확정 |

### 3. 안티패턴
- **"The Wall of Silence"**: 두 시스템이 연결되지 않아 엑셀로 데이터를 주고받는 경우. (2-Tier가 아닌 그냥 '파편화된 시스템')
- **"Two Many Tiers"**: 시스템 종류가 너무 많아져 통합 비용이 관리 이익을 넘어서는 경우. (Tier 2 솔루션을 1~2종으로 단일화 권장)

📢 섹션 요약 비유: 2-Tier ERP의 성공은 두 시스템 사이를 연결하는 '데이터 고속도로'가 얼마나 정체 없이 뚫려 있느냐에 달려 있습니다.

## Ⅴ. 기대효과 및 결론 (Future & Standard)

2-Tier ERP는 거대 기업이 벤처기업처럼 움직이게 만드는 '디지털 유연성'의 핵심이다.

| 기대효과 | 설명 |
|:---|:---|
| **전체 TCO 절감** | 값비싼 Tier 1 라이선스 사용자를 최소화하고 저렴한 SaaS로 대체 |
| **신속한 해외 확장** | 신규 지사 설립 시 수주 내에 ERP 가동 가능 (Fast Roll-out) |
| **현지 최적화 성과** | 현지 고객 요구에 민첩하게 대응하여 시장 점유율 확대 |

미래의 2-Tier ERP는 **"Invisible Tiering"**으로 진화할 것이다. 사용자는 본사 시스템인지 지사 시스템인지 알 필요 없이 하나의 앱(App)을 쓰는 것처럼 느껴지며, 뒤에서는 AI 오케스트레이터가 비즈니스 로직의 중요도에 따라 데이터를 실시간으로 분산 배치하는 '지능형 분산 ERP'가 대세가 될 전망이다. 2-Tier는 이제 전략적 선택을 넘어, 복잡한 글로벌 생태계를 지탱하는 '표준 아키텍처 패턴'이 되고 있다.

📢 섹션 요약 비유: 미래의 2-Tier ERP는 내 폰과 노트북이 하나처럼 연결되는 '클라우드 동기화'와 같아서, 어디서 일을 하든 전 세계가 하나의 거대한 가상 사무실이 될 것입니다.

### 📌 관련 개념 맵
| 관련 개념 | 관계 및 시너지 설명 |
|:---|:---|
| Single Instance | 전 세계를 하나의 시스템으로 통합하는 2-Tier의 대척점 전략 |
| Cloud ERP (SaaS) | Tier 2 구축을 위한 가장 효율적이고 민첩한 인프라 수단 |
| EAI / iPaaS | Tier 1과 Tier 2 간의 데이터 전송과 프로세스를 엮어주는 기술 |
| Hub-and-Spoke | 본사(Hub)와 지사(Spoke)를 연결하는 2-Tier의 네트워크 토폴로지 |
| Post-Merger Integration (PMI) | 인수 합병 후 시스템 통합 시 2-Tier 전략이 가장 빈번히 사용됨 |

### 👶 어린이를 위한 3줄 비유 설명
1. 본부 기지에는 아주 크고 복잡한 컴퓨터가 있고, 밖에서 싸우는 대원들에게는 가볍고 빠른 태블릿을 주는 거예요.
2. 각자 자기에게 맞는 도구를 쓰니까 더 효율적으로 일할 수 있고, 그러면서도 본부에서는 대원들이 무엇을 하는지 다 알 수 있죠.
3. 덩치 큰 거인(본사)과 발 빠른 요정(지사)이 손을 잡고 함께 승리하는 똑똑한 작전이랍니다!
