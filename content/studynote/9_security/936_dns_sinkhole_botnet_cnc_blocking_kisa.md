# 936. DNS 싱크홀 - Zombie PC 봇넷 C&C 서버 질의 블랙리스트 감지 경로 우회 KISA 차단 연계 악성 도메인 룩업망 접속 억제 무효화 처리 방화 정책

> **핵심 인사이트**: 회사 컴퓨터 1,000대가 바이러스에 감염돼 좀비 PC가 되었다. 이 좀비들은 해커의 본부(C&C 서버)인 `hacker.com`으로 "다음 공격 대상 누굴까요?"라고 명령을 받으러 인터넷 접속(DNS 질의)을 시도한다. 백신으로 1,000대를 다 치료하기엔 늦었다. "야! 통신사(ISP) DNS 서버 설정 뜯어고쳐! 좀비 PC가 `hacker.com`의 주소를 물어보면, 진짜 해커 서버 주소 말고 '우리 KISA(한국인터넷진흥원)의 차단 안내 페이지 주소(가짜 주소)'를 던져줘서 그쪽으로 곤두박질치게(싱크홀) 만들어버려!" 해커와 좀비의 통신을 중간에서 원천 봉쇄하는 거대한 블랙홀 방어막, DNS 싱크홀이다.

## Ⅰ. 좀비 PC와 봇넷(Botnet) 통신의 저주
- **C&C (Command & Control) 서버**: 수만 대의 좀비 PC(감염된 컴퓨터)들을 조종해 디도스(DDoS) 공격을 지시하는 해커의 중앙 사령부 서버입니다.
- **해커의 도메인 회피술**: 해커는 경찰이 사령부 IP를 차단할까 봐, 사령부 IP 주소를 10분에 한 번씩 미친 듯이 바꿉니다(Fast Flux). 그래서 좀비 PC들은 IP 대신 `hacker.com`이라는 도메인 주소로만 끈질기게 접속(DNS 질의)을 시도합니다.

## Ⅱ. DNS 싱크홀(Sinkhole)의 개념 🌟
- **개념**: 악성코드(좀비 PC)가 해커의 C&C 서버나 피싱 사이트로 접속하기 위해 DNS(도메인 네임 시스템)에 도메인 이름 풀이(IP 주소 질의)를 요청할 때, **사전에 구축된 블랙리스트를 바탕으로 정상적인 IP 대신 엉뚱한(안전한) IP 주소(싱크홀 서버)를 반환하여, 악성 사이트로의 통신 연결 자체를 100% 무효화(우회 차단)시키는 네트워크 방어 기술**입니다.

## Ⅲ. DNS 싱크홀 동작 원리 (블랙홀로 유도하기) 🌟
한국은 KISA(한국인터넷진흥원)와 통신 3사(KT, SKT, LGU+)가 연합해 이 거대한 싱크홀 망을 국가 단위로 가동 중입니다.

1. **블랙리스트 동기화**: KISA가 밤낮으로 해커들의 C&C 서버 도메인 주소를 수집해 블랙리스트(차단 목록)를 만들어 통신사 DNS 서버에 매일 뿌립니다.
2. **좀비 PC의 질의 (DNS Query)**: 내 컴퓨터가 감염되어 나 몰래 `hacker.com`의 IP를 물어보는 패킷을 KT DNS 서버로 날립니다.
3. **가짜 주소 반환 (Sinkhole 가동)**: KT DNS 서버가 엑셀을 까보니 블랙리스트에 있는 도메인입니다! 정상 IP(예: `1.1.1.1`)를 알려주지 않고, **"어 그거 `127.0.0.1 (루프백, 지 자신)` 이야!" 하거나 "KISA 안내 서버 주소(`200.200.200.2`)"를 던져줍니다.**
4. **연결 무효화 및 감염 탐지**: 
   - 좀비 PC는 가짜 주소로 연결을 시도하다가 허공에 패킷을 날리고 허우적댑니다(해커의 명령 수신 실패).
   - KISA 안내 서버에 접속된 경우, KISA는 "오, 이놈 감염된 좀비네!" 하고 즉각 탐지하여 해당 PC 사용자에게 팝업창으로 **"고객님 PC가 감염되었습니다. 백신을 돌리세요!"**라고 통보(치료 유도)까지 해냅니다.

## Ⅳ. 왜 싱크홀이 방화벽보다 무서운가? (도입 효과)
- **차단의 가성비**: 방화벽은 수억 개의 패킷 데이터를 일일이 다 까보고 차단하느라 장비가 뻗습니다. 싱크홀은 전화번호부(DNS) 단계에서 그냥 **"없는 번호입니다"** 한 줄만 던져주면 끝납니다. 시스템 부하(오버헤드)가 거의 제로에 가까운 극강의 가성비 통제망입니다.
- **치명적 한계 (HTTPS/DoH 시대의 위협)**: 최근 해커들이 DNS 질의 자체를 암호화해서 쏘는 기술(DoH, DNS over HTTPS)을 써서 DNS 서버가 도메인 이름을 못 읽게 맹인으로 만들어 싱크홀을 회피하는 수법이 늘고 있어 창과 방패의 전쟁이 계속되고 있습니다.

> 📢 **섹션 요약 비유**: 해커의 C&C 서버는 사기꾼이 돈을 걷어가는 '대포통장 은행 지점'입니다. 감염된 좀비 PC는 사기꾼에게 돈(데이터)을 보내기 위해 콜센터(DNS 서버)에 전화를 걸어 "대포통장 은행 지점 주소가 어딥니까?"라고 묻습니다. 옛날 콜센터 직원은 멍청하게 진짜 사기꾼의 주소를 알려줬습니다. **DNS 싱크홀**은 콜센터 직원이 경찰(KISA)이 준 수배 전단지(블랙리스트)를 들고 있는 것입니다. 좀비가 전화를 걸면 직원은 진짜 주소를 숨기고 **"아, 그 은행은 '쓰레기장(루프백 IP)'에 있습니다" 또는 "'경찰서 앞마당(안내 페이지)'에 있습니다"**라고 엉뚱한 주소를 알려줍니다. 좀비는 그 주소로 돈을 싸 들고 갔다가 쓰레기통에 돈을 던지거나(연결 무효화) 경찰에게 덜미를 잡히게 되어, 해커의 사기극 네트워크를 원천 차단하는 가장 똑똑하고 가벼운 길안내 조작 방어망입니다.
